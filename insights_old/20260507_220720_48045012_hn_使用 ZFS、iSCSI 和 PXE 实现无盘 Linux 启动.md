## 文章摘要

本文介绍了一种无盘Linux启动方案，使用ZFS存储、iSCSI网络块设备和PXE网络引导技术。作者的动机包括：在游戏PC上测试新的AI模型（llama.cpp在Windows上编译繁琐）、避免Windows更新破坏GRUB、避免重新分区现有硬盘、以及利用现有的NAS进行网络启动。

### 核心技术架构

1. **网络引导协议**：使用Netboot.xyz配合iPXE实现PXE网络启动
2. **存储后端**：ZFS ZVol（32GB）作为iSCSI目标磁盘
3. **iSCSI配置**：使用targetcli-fb配置iSCSI目标，设置CHAP双向认证
4. **DHCP/DNS**：通过Dnsmasq配置不同架构的客户端引导

### 实施步骤

1. 安装配置Netboot.xyz（使用Ansible部署到Apache）
2. 配置TFTP服务提供iPXE引导文件
3. 在路由器配置Dnsmasq实现PXE引导
4. 创建ZFS ZVol卷
5. 配置iSCSI Target和ACL
6. 在目标机器上通过网络安装Debian

## 热门评论分析

### 1. 启动器选择讨论

**高赞评论**：推荐rEFInd替代GRUB

> "Friends don't let friends use grub. rEFInd is so much simpler: one efi entry, one text config file in the efi partition, nothing that needs to change when the kernel updates"

**反驳意见**：
- systemd-boot被认为是更简单的选择
- GRUB支持BIOS、UEFI、x86、ARM等多种平台
- GRUB在故障排查时功能更强大（"GRUB is just a operating system"）

**讨论要点**：
- iSCSI与NBD（网络块设备）的对比
- iSCSI支持标准协议，兼容性更好
- NBD在处理网络中断方面不如iSCSI可靠

### 2. 网络性能优化

用户讨论了在10Gbps网络下运行iSCSI的性能表现：
- 相比本地NVMe仍有4-8倍性能差距
- 建议使用Chelsio或Mellanox网卡配合RDMA加速
- NVMe-oF是最佳协议，开销最小

### 3. iSCSI网络配置建议

- 在拥塞网络环境下iSCSI表现不佳
- 建议使用QoS设置或VLAN优先级
- 建议使用独立网络（控制平面与数据平面分离）

### 4. ZFS自我修复功能

ZFS的校验和纠错功能被讨论：
- 自动修复损坏数据
- scrub操作可扫描整个存储池验证完整性
- 元数据默认有2个副本提供额外保护
