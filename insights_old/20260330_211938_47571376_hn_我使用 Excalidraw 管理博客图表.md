## 核心洞察

### 1. Excalidraw 的优势

- **手绘风格**: 被广泛认为是"恰到好处的限制性、亲和力和专业性"
- **协作友好**: 草图风格能清晰传达这是"概念"而非最终方案
- **用户体验**: UI出色，选择功能比Lucid或Figma更好
- **隐私和加载速度**: 注重隐私，加载速度快
- **灵活性**: 可调整"粗糙度"，草稿阶段使用"完全粗糙"模式，完成后改为正式风格

### 2. Excalidraw 的局限性

- **功能限制**: 缺少基本功能如加粗或斜体文本
- **字体问题**: 默认字体被认为"丑陋、幼稚且无障碍性差"
- **自托管困难**: 多人自托管版本运行困难
- **复杂度限制**: 复杂图表容易过时或难以维护

### 3. 替代工具讨论

| 工具 | 特点 |
|------|------|
| **Grafly** | 更正常的外观，免费工具 |
| **Mermaid** | 代码即图表，可纳入Git版本控制 |
| **PlantUML** | 对LLM更友好，控制力更强 |
| **draw.io** | 功能全面，本地可运行 |
| **TikzMaker** | 仍在开发中 |
| **tldraw** | 与Excalidraw齐名的简约白板工具 |

### 4. AI 集成现状

- **Excalidraw MCP**: 可用于Claude，但生成的图表较为粗糙
- **Mermaid**: AI生成效果更好
- **TikZ**: 通过代码代理可生成图表，但需要多次迭代修正错误箭头、位置等问题
- **结论**: AI图表生成仍需改进，目前无法完全自动化

### 5. 图表维护策略

- 将图表视为" disposable"（可丢弃的）
- 使用图表解释抽象概念，并附加文字说明
- 不要用图表代表系统的精确状态
- Mermaid等代码化图表更适合版本控制

### 6. 工作流集成

- **Obsidian**: 使用Excalidraw插件，可在笔记中添加图表
- **Claude Code**: 集成Excalidraw进行版本控制
- **博客发布**: 自动从Obsidian解析并推送到网站
- **Payload CMS**: 可创建自定义Excalidraw块，支持深色/浅色模式

## 社区共识

1. **手绘风格的价值**: 草图风格传达"这是思考过程，而非最终定稿"
2. **工具选择**: 没有完美的图表工具，需要在"美观的自动布局"、"复杂图表"、"正确性"之间权衡
3. **文档腐化问题**: 文档往往与代码脱节而导致腐化，代码生成的图表（如Mermaid）是解决方案
4. **博客趋势**: 使用Excalidraw管理博客图表已成为一种流行做法
