## 项目简介

Niri 是一个可滚动平铺式 Wayland compositor。窗口被排列在无限向右延伸的列中，打开新窗口不会导致现有窗口调整大小。

## 重要更新

### 1. 项目迁移至 GitHub 组织

Niri 现已迁移到 [GitHub 组织](https://github.com/niri-wm) 而非个人账户。主要原因是可以分配问题分类权限。非常感谢 **@Sempyos** 负责所有问题和 Pull Request 的分类、回答问题和帮助用户诊断问题。

同时迁移了相关项目：
- **awesome-niri**: 相关项目列表，由 @Vortriz 维护
- **artwork**: 艺术作品仓库，包括徽标和壁纸

### 2. 模糊效果（Blur）- 最受欢迎的功能

这是 Niri 有史以来最请求的功能，也是 GitHub 上赞同最多的issue。通过 `ext-background-effect` Wayland 协议支持。

#### 支持的应用程序：
- Dank Material Shell v1.4.5
- Noctalia shell
- Vicinae launcher
- foot terminal v1.26
- kitty terminal v0.46.2
- Ghostty terminal (v1.4)

#### 支持的工具包：
- Quickshell (v0.3)
- winit (v0.31)

#### 两种模糊模式：
- **Xray 模糊**（默认）：显示壁纸的模糊版本，效率极高
- **普通模糊**：需要在帧渲染过程中读取像素

用户可通过 window-rule 或 layer-rule 配置模糊效果。

### 3. 可选包含文件（Optional Includes）

支持可选的配置文件包含，即使文件不存在也不会导致配置加载失败。

```kdl
include optional=true "optional-config.kdl"
```

当可选文件缺失时，niri 会发出警告但仍成功加载配置。

### 4. 滚动时指针 warp

在视图滚动过程中，指针可以从屏幕一侧 warp 到另一侧，类似 Blender 的体验，使滚动多个窗口更加自然。

### 5. 屏幕录制改进

#### 5.1 窗口录制中的指针元数据

现在支持将光标作为单独的帧元数据发送，而非直接嵌入视频帧。这允许应用程序（如 OBS）控制光标可见性。

#### 5.2 动态录制目标延迟启动

动态录制目标现在会延迟启动视频流，直到选择第一个目标，避免出现异常的 1×1 视频。

#### 5.3 屏幕录制 IPC

新增屏幕录制 IPC 支持：
- `niri msg casts` 查看当前活动的屏幕录制
- 可订阅事件流监听录制变化
- 提供详细信息：类型（PipeWire/wlr-screencopy）、目标、节点ID等
- 新增 `niri msg action stop-cast --session-id <ID>` 强制停止屏幕录制

### 6. 动画改进

#### 取消最大化/取消全屏动画同步

修复了取消最大化时视图移动动画不同步的问题。

#### 拖出最大化窗口动画

修复了将最大化窗口拖出时可能跳过水平视图滚动动画的问题。

#### 最左列拖出行为

修复了从工作区拖出最左列时的行为问题。

### 7. 其他改进

- 配置包含现在支持展开 `~` 开头的路径到主目录
- pop-up 菜单支持透明度和其他背景效果配置
- 最低支持的 Rust 版本更新为 1.85
- 修复了 `niri.service` 中的路径硬编码问题
- 简化了 dinit 服务文件
- IME 在 GTK4 pop-up 中的问题修复
- 复制损坏时始终包含光标的问题修复
