# 洞察报告：Mullvad 出口 IP 可被用于用户识别

**洞察链接**: https://news.ycombinator.com/item?id=48143880
**原文链接**: https://tmctmt.com/posts/mullvad-exit-ips-as-a-fingerprinting-vector/
**发布时间**: 2026-05-14
**评分**: 275 points
**评论数**: 138 comments

## 核心发现

### 1. 出口 IP 分配机制存在缺陷

Mullvad 的出口 IP 并非随机分配，而是基于用户的 WireGuard 公钥**确定性**选择：

- 每次连接到同一台服务器，会获得相同的出口 IP
- 公钥每 1-30 天轮换一次（使用第三方客户端则永不轮换）
- 作者测试了 9 台服务器上的 3650 个公钥，仅产生 **284 种 IP 组合**

### 2. 根本原因

Mullvad 使用基于**种子（seed）的随机数生成器（RNG）**来选择出口 IP：

- 相同的种子会产生相同的**相对比例位置**
- 例如：若服务器 IP 池大小为 60，用户获得的 IP 位置始终在池的约 81% 处
- 这导致尽管有超过 8.2 万亿种组合可能，实际组合数量极为有限

### 3. 用户识别风险

通过这种机制，攻击者可以：

| 场景 | 识别精度 |
|------|----------|
| 论坛版主识别马甲账户 | >99% 概率确认同一用户 |
| 数据泄露关联分析 | 可将 VPN 用户去匿名化 |
| IP 日志交叉比对 | 关联多个账号为同一用户 |

**示例**：两个使用不同 Mullvad 服务器的账号，若 IP 浮点值范围重叠（0.4334-0.4428 vs 0.4358-0.4423），则有 >99% 概率为同一人。

## 社区讨论焦点

### 1. VPN 隐私的局限性

- 公共 VPN 的主要目的是隐藏 ISP，而非对访问网站匿名
- 若需真正匿名，应使用 Tor 网络
- VPN 出口 IP 本质上是公开信息

### 2. Mullvad 是否为情报机构前端？

部分评论者提出疑虑：
- 这种设计方式便于识别用户
- 第三方审计未发现此问题
- 但多数认为这更可能是开发者对 RNG 的误解，而非刻意设计

### 3. 解决方案建议

- 避免在同一公钥周期内频繁切换服务器
- 强制轮换公钥：注销并重新登录 Mullvad 应用
- 使用住宅代理或移动代理以获得更好隐私

## 结论

此发现揭示了 Mullvad 出口 IP 分配机制中的一个**隐私漏洞**。尽管不像最初预期的那么严重（需结合其他信息才能完全去匿名化），但确实降低了用户识别门槛。建议 Mullvad 采用真正的随机分配或添加伪随机种子来修复此问题。
