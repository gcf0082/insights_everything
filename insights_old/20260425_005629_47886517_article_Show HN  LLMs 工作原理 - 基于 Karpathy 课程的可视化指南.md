## 核心要点总结

### 1. 训练规模

- **Token 数量**: 15 万亿
- **参数数量**: 4050 亿
- **文本数据**: 44 TB
- **词表大小**: 10 万

### 2. 预训练阶段

#### 2.1 数据收集

- 使用 Common Crawl 从 2007 年开始抓取网页
- 截至 2024 年已索引 27 亿个页面
- 经过多轮过滤后保留约 44 TB 高质量文本（15 万亿 token）

**过滤流程**：
1. URL 过滤 — 屏蔽恶意网站、垃圾网站
2. 文本提取 — 从 HTML 中提取纯文本
3. 语言过滤 — 保留英语内容 ≥65% 的页面
4. 去重 — 移除重复或近似重复的页面
5. PII 移除 — 删除个人身份信息

#### 2.2 分词（Tokenization）

- 神经网络无法处理原始文本，需要转换为数字
- 使用 **BPE（字节对编码）** 算法
- GPT-4 词表包含 100,277 个 token
- 子词分词可以处理新词、未登录词和多语言

#### 2.3 神经网络训练

- 使用 **Transformer** 架构
- 随机初始化数十亿参数
- 通过预测下一个 token 来训练
- 损失值从 ~11.2 下降到 ~2.4

### 3. 基础模型行为

- 基础模型是**互联网模拟器**，本质是一个 sophisticated 的自动补全引擎
- 不是助手，不回答问题，只是延续 token 序列
- 具备few-shot prompting能力，可以通过示例完成任务
- 存在**幻觉**问题——因为训练数据中总是有自信的回答

### 4. 后训练阶段

#### 4.1 监督微调（SFT）

- 人工标注员创建理想对话数据集
- 模型学习模仿"理想助手"的行为
- 现代数据集包含数百万对话（多为合成数据 + 人工审查）

#### 4.2 人类反馈强化学习（RLHF）

- 人类评审员对模型输出排序
- 训练奖励模型预测人类偏好
- 使用强化学习优化生成更好、更诚实、更有结构的回答

### 5. LLM 心理特征

- **记忆机制**：参数 = 长期记忆，上下文窗口 = 工作记忆
- **工具使用**：可以触发外部工具调用
- **无持久自我**：每次对话都是全新开始
- **随机性**：温度参数控制生成随机性
- **知识截止**：有训练数据的最后日期，无法回答该日期之后的事件

### 6. RAG（检索增强生成）

解决知识截止和上下文窗口限制的方案：

1. 将文档转换为向量嵌入
2. 对用户查询也进行嵌入，通过余弦相似度检索相关内容
3. 将检索内容注入上下文窗口，让模型基于真实信息生成回答

## 完整流程图

```
数据收集 → 分词 → 预训练 → 基础模型 → SFT → RLHF → 对话助手
```

1. **数据收集**: Common Crawl → URL过滤 → 文本提取 → 语言过滤 → 去重 → PII移除 → FineWeb (44TB)
2. **分词**: 文本 → UTF-8字节 → BPE → 15万亿token序列
3. **预训练**: Transformer训练预测下一个token → 4050亿参数
4. **基础模型**: 互联网文档模拟器
5. **SFT**: 人工标注对话 → 监督微调（约3小时）
6. **RLHF**: 奖励模型 + 强化学习优化
7. **最终助手**: ChatGPT/Claude/Gemini

---

## 关键洞见

> **最重要的理解**：LLM 生成的每个词都是概率采样——一个 10 万维度的有偏硬币抛掷，经过数十亿次训练得出。模型不是"思考"要说什么，而是计算所有可能下一个 token 的概率分布并从中采样。
