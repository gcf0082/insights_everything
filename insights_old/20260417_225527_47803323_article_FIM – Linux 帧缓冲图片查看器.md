## 项目概述

FIM（Fbi Improved）是一款轻量级且高度可定制的通用图像查看器，深受GNU/Linux设计理念的启发。该软件主要面向熟悉VIM文本编辑器或Mutt邮件用户代理操作方式的用户群体。FIM的目标是成为图像查看领域的“瑞士军刀”，提供强大的自定义和脚本功能。

虽然FIM最初是为GNU/Linux系统开发的，但它也可以构建运行在多个Unix系统上，经过适当移植甚至可以在MS-Windows、WebAssembly（通过emscripten）或Android（通过termux）上运行。FIM具有轻量级的特点，因为它依赖的库很少，而且大多数是可选的。

## 主要特性

### 多平台支持

FIM是一款多设备图像查看器，支持多种输出模式：

- **Linux帧缓冲区控制台**：最初为Linux帧缓冲区控制台开发
- **X窗口系统**：通过SDL和Imlib2库提供X支持
- **ASCII艺术输出**：通过aalib库支持单色ASCII艺术，通过libcaca库支持彩色ASCII艺术
- **SDL2支持**：实验性支持SDL2

### 轻量级设计

FIM设计简洁，依赖的库文件很少，且大多数依赖项是可选的，这使得它非常适合资源有限的环境或追求轻量级解决方案的用户。

### 键盘控制

FIM支持全屏显示，用户通过键盘进行操作和控制。这种设计类似VIM编辑器的操作方式，对于熟悉键盘快捷操作的用户来说非常高效。

### 多种图像格式支持

FIM能够打开并显示多种图像文件格式，具有良好的兼容性。

### 高度可定制和可脚本化

FIM提供了丰富的自定义选项和脚本功能，用户可以根据自己的需求配置和扩展功能。

## 技术架构

### 基于Fbi的改进

FIM最初是Fbi图像查看器（由Gerd Hoffmann开发）的一个补丁，随后发展成为一个独立的分支。FIM保留了Fbi的核心设计理念，同时增加了许多改进功能。

### 开发语言

FIM使用C++语言开发，这使得它具有良好的性能和可维护性。

### 构建系统

FIM支持在多种Unix类系统上构建，具有良好的可移植性。

## 使用方法

### 基本用法

打开单个文件：
```
fim DSC_0001.JPG DSC_0002.JPG
```

打开目录中的所有文件：
```
fim -R /path/to/directory
```

### 配置文件

用户可以通过配置自定义FIM的行为，配置文件通常命名为`.fimrc`。

### 注释功能

可以在图像文件中添加注释，格式如下：
```
#!fim:city=Rome
DSC_0001.JPG    一只大狗。
#!fim:city=Naples
DSC_0002.JPG    一座雕像。
```
