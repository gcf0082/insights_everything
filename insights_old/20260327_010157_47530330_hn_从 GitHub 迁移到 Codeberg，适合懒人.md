## 摘要

本洞察讨论了从 GitHub 迁移到 Codeberg 的可行性和挑战。发帖者认为迁移到 Codeberg 对"懒人"来说很容易，但评论者指出 Codeberg 存在稳定性、功能和社区等方面的问题，使其难以完全替代 GitHub。

---

## 主要讨论观点

### 1. Codeberg 可用性问题

- ** downtime（停机）问题**: Codeberg 经常出现停机情况，这对于需要随时访问代码的开发者来说是一个致命问题
- **Cloudflare 依赖**: GitHub 使用 Cloudflare 来防护 DDoS 攻击，而 Codeberg 使用 Anubis，但防护效果不如 Cloudflare
- **个人经历**: 有用户表示 Codeberg 虽然免费但不稳定，曾在考虑迁移时遇到停机

### 2. Git 分布式架构的正确使用方式

- **离线工作能力**: git 的设计初衷就是支持离线工作，开发者不应该依赖远程仓库始终在线
- **工作流程建议**: 
  - 使用多个远程仓库
  - 在本地保留所有提交
  - 设置 SSH 远程仓库作为备份
- **git init --bare**: 可以创建裸仓库作为远程，通过 SSH 或本地路径进行推送和拉取

### 3. Codeberg 的局限性

- **私有仓库限制**: 私有仓库上限仅为 100MB
- **使用政策**: Codeberg 主要面向自由开源软件（FOSS）项目，不鼓励商业项目或个人非开源用途
- **不支持的功能**: 不提供类似 GitHub Pages 的个人主页托管服务
- **CI/CD 支持**: 直到最近才开始有第三方 CI 支持（如 NixCI）

### 4. GitHub 的优势仍然明显

- **社区效应**: 开发者社区主要在 GitHub 上，这是最大的吸引力
- **免费 CI**: GitHub Actions 提供免费 macOS 运行器和公共仓库无限容量
- **功能完整**: Issues、PR、Discussions 等功能完善
- **可靠性**: 虽然也有停机，但规模更大、经验更丰富

### 5. 自托管解决方案

- **Forgejo**: Codeberg 背后使用的开源软件，可以自托管
- **优点**: 更可靠、可定制、可通过 Tailscale 等工具保护
- **成本**: 可以使用树莓派等低成本硬件运行
- **案例**: 有用户使用 2 台树莓派托管 Forgejo 服务器和运行器

### 6. AI 爬虫问题

- 所有主流 AI 公司（ByteDance、OpenAI、AWS、Claude 等）都在大量爬取代码仓库
- 有用户建议使用 iocaine 或 Cloudflare 的 AI Labyrinth 来阻止 AI 爬虫

---

## 结论

虽然从技术上可以从 GitHub 迁移到 Codeberg，但要完全替代 GitHub 仍然存在以下挑战：

1. **稳定性**: Codeberg 的可用性不如 GitHub
2. **功能完整性**: 缺少一些关键功能如 GitHub Pages
3. **社区**: 开发者社区仍然集中在 GitHub
4. **CI/CD**: 第三方 CI 支持有限

对于愿意投入时间进行自托管的开发者，Forgejo 是一个可行的替代方案。但对于大多数开发者来说，GitHub 仍然是最佳选择，特别是对于需要协作和 CI/CD 功能的团队项目。
