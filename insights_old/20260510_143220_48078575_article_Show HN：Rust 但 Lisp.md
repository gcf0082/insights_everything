## 项目概述

rlisp 是一个将 LISP 语法嫁给 Rust 语义的实验性项目。它是一个透明的 s-expression 前端，能够直接将 LISP 代码编译为 Rust 代码——无需运行时、无需垃圾回收。

核心特性是：`(s-expr → .rs → binary)`，完全保留 Rust 的所有权、借用、生命周期、泛型、特性和模式匹配等语义。

## 核心功能

### 语法转换
rlisp 将 LISP 风格的 s-expression 语法转换为标准 Rust 语法：
- 函数定义、let 绑定、结构体、枚举、模式匹配
- 控制流（if、match、loop、while、for）
- 特性和实现
- 模块系统和可见性控制

### 宏系统
rlisp 的宏是编译时的 s-expression 转换器，无需 proc_macro crate 或 token streaming。支持：
- `quasiquote` - 模板引用，允许内部 unquote
- `unquote` - 插入值
- `unquote-splicing` - 扁平化列表到周围形式
- `&rest` - 捕获剩余参数为列表

### 特殊功能
- **内联 Rust**: 使用 `(rust "...")` 直接嵌入原始 Rust 代码
- **生命周期注解**: 支持 `'a` 生命周期参数
- **turbofish**: 通过 `::` 特殊形式支持
- **类型转换**: `(as x i32)` 进行类型转换
- **不安全代码**: 支持 `unsafe` 块

## 使用方式

```bash
git clone https://github.com/ThatXliner/rlisp.git
cd rlisp
cargo install --path .
```

命令：
- `rlisp compile file.lisp` - 转译为 .rs 文件
- `rlisp build file.lisp` - 转译并用 rustc 编译
- `rlisp run file.lisp` - 转译、编译并运行

## 示例代码

```lisp
(struct Point
  (x f64)
  (y f64))

(impl Point
  (fn distance ((&self) (other &Point)) f64
    (let dx (- (. self x) (. other x)))
    (let dy (- (. self y) (. other y)))
    ((. dx powf 2.0) + (. dy powf 2.0)) sqrt))

(fn main () ()
  (let p1 (new Point (x 0.0) (y 0.0)))
  (let p2 (new Point (x 3.0) (y 4.0)))
  (println! "Distance: {}" (. p1 distance (& p2))))
```

## 项目定位

作者明确表示这是一个**周末项目**，不是生产级编译器。某些 Rust 语法尚未支持（如 lifetime bounds），重点在于探索将 LISP 宏机制与 Rust 语义结合的可能性。

## 为何有趣

1. **宏变得简单**: LISP 宏就是接受 s-expression 并返回 s-expression 的函数，无 token streaming 或 proc_macro 繁文缛节
2. **结构化编辑**: s-expression 易于操作——slurp、barf、transpose、wrap
3. **统一语法**: 表达式、语句、类型和模式无需区分，一切都是 s-expression
