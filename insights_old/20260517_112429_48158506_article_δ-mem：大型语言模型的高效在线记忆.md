## 1. 研究背景与动机

### 问题陈述

大型语言模型（LLM）在长期助手和Agent系统中面临一个关键挑战：需要积累和重用历史信息。现有的解决方案存在明显缺陷：

- **扩展上下文窗口**: 成本高昂，且无法保证有效的上下文利用
- **外部检索模块**: 带来额外开销，与模型内部计算解耦
- **静态参数记忆**: 训练后固定，无法动态更新

### 研究目标

设计一种轻量级的在线记忆机制，使其能够：
1. 在交互过程中动态更新
2. 直接影响模型内部计算
3. 无需全参数微调或替换主干架构
4. 不需要显式扩展上下文

## 2. 方法概述

### 核心创新：δ-mem

δ-mem 是一种轻量级记忆机制，通过以下组件增强冻结的全注意力主干模型：

1. **紧凑的在线关联记忆状态**: 固定大小的状态矩阵（8×8），压缩过去信息
2. **Delta规则学习**: 在线更新记忆状态
3. **低秩修正**: 使用记忆读出生成注意力计算的修正

### 技术细节

- **记忆写入**: 当新token或交互片段到达时，将当前信息投影到低维记忆空间
- **Delta规则**: 动态更新在线状态矩阵
- **注意力修正**: 记忆读出生成低秩修正，直接参与推理时的前向计算
- **写策略**: 支持 TSW (Token-level Segment Write)、SSW (Segment-level Segment Write)、MSW (Mixed Segment Write)

### 与现有方法的区别

| 特性 | LoRA | δ-mem |
|------|------|-------|
| 低秩接口 | ✓ | ✓ |
| 更新方式 | 静态（训练后固定） | 动态（运行时更新） |
| 注意力修正 | 无 | 有 |

## 3. 实验结果

### 基准测试性能

| 基准 | 提升倍数 |
|------|----------|
| 平均得分 | 1.10× (冻结主干) / 1.15× (最强非δ-mem基线) |
| MemoryAgentBench | 1.31× |
| LoCoMo | 1.20× |

### 关键发现

1. **极小的记忆规模**: 仅用 8×8 的在线状态就能实现显著提升
2. **上下文恢复能力**: 在显式上下文删除后，模型仍能恢复有用的历史信息
3. **通用能力保持**: 大部分保持冻结主干的通用能力

## 4. 结论与意义

### 主要贡献

1. **高效记忆机制**: 证明了紧凑的在线状态可以直接与注意力计算耦合，实现有效记忆
2. **无需微调**: 无需全参数微调、主干替换或显式上下文扩展
3. **可扩展性**: 8×8 的极小状态即可生效，表明该方法具有良好扩展潜力

### 局限性

- 记忆容量受状态矩阵大小限制
- 写策略的选择可能影响性能
- 长期使用的效果还需进一步验证

### 未来方向

- 探索更大的记忆状态
- 优化写策略
- 在更多场景中验证有效性
