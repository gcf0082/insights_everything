## 洞察摘要

Google 宣布了一项新的反垃圾邮件政策，针对"返回按钮劫持"(back button hijacking)行为。该政策旨在打击那些通过 JavaScript 操纵浏览器历史记录，导致用户点击返回按钮时被重定向到广告或垃圾内容页面的行为。

### 核心观点

**支持的论点：**
- 这是改善用户体验的积极举措
- 返回按钮劫持违反了用户的基本预期
- 一些大型网站（如 Microsoft、Azure 门户、Amazon、eBay、Reddit）存在此问题
- 技术上很难从浏览器层面完全禁止，因为恶意网站总是能找到绕过方法

**质疑的论点：**
- Google 本身也做过类似的事情（如 YouTube 推荐算法）
- 该政策可能难以有效执行，SEO 相关政策往往会陷入猫鼠游戏
- 营销团队不太可能认真审查技术实现
- 建议的替代方案可能只是增加用户点击次数的门槛

**技术讨论：**
- `window.history.pushState` 有合法用途（如单页应用）
- 浏览器曾尝试限制历史 API，但容易被 cookie 横幅等手段绕过
- 有评论建议将历史 API 限制在用户点击事件之后
- Post/Redirect/Get 模式被推荐为良好的表单处理实践

**延伸讨论：**
- Web 退化问题：现代 Web 充满广告、弹窗、Cookie 横幅等干扰
- 滚动条劫持同样令人困扰
- 一些人怀念 1990 年代简单的静态 HTML 时代
- uBlock Origin 和阅读器模式被推荐为解决方案
