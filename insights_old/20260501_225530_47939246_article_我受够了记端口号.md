## 文章摘要

### 背景问题

作者在本地运行大量项目，包括 Jekyll 博客、Flask API、Next.js 应用、AI 工具等，每个应用都运行在某个端口上。每次打开浏览器时都需要回忆：是在 3000 端口？还是因为其他应用占用了 3000 而移到了 5173？还有一个问题是 Tailscale 的 URL 地址也记不住。

作者将这个问题称为**端口腐朽（port rot）**：服务和被遗忘的端口缓慢累积，以及无用的浏览器标签页堆积，使本地开发环境感觉像一个垃圾抽屉。更糟糕的是，这些应用中许多并不需要一直运行，只在需要时运行然后安静地关闭即可。

### 解决方案：local.vibe

作者构建了 [local.vibe](https://github.com/graiz/local.vibe)——一个友好的仪表板和本地 .vibe 主机名，用于 Mac 上的每个本地 Web 应用。

**核心功能：**

1. **简化启动**：通过 vibe.json 配置文件定义项目，运行 `vibe start` 即可通过 `https://项目名.vibe` 访问
2. **自动端口分配**：自动分配空闲端口，并通过 `$PORT` 环境变量传递给应用
3. **HTTPS 支持**：本地证书自动添加到 Keychain，HTTPS 开箱即用
4. **进程守护**：监控进程，如果崩溃或重新绑定则自动处理
5. **仪表板**：通过 https://local.vibe 集中查看所有运行中的应用
6. **主机名配置**：支持 Tailscale 机器重定向（如 tailscale.vibe）、Home Assistant 代理（hass.vibe）等

### 技术实现

- 使用 Go 语言开发，单个二进制文件
- 安装 dnsmasq、pf 端口转发规则和本地 CA
- 通过 Unix Socket 作为反向代理
- 无外部服务、无账户、无遥测数据

### 代理集成

提供设置指南端点 `curl http://localhost:7999/setup.md`，可粘贴到 Claude Code 或 Cursor 中，让代理了解本地环境并快速注册新路由。

### 使用方法

```bash
git clone https://github.com/graiz/local.vibe.git
cd local.vibe
./setup.sh
```

目前仅支持 macOS，MIT 许可证开源。
