## 项目概述

GenCAD 是一个能够将草图或图片转换为参数化 CAD 模型的项目，输出为 CAD 操作序列而不仅仅是 3D 网格。

## 主要洞察

### 1. 技术可行性受到质疑

多位用户测试后反映：
- 在训练数据以外的图像上几乎无法产生可用结果
- 官方自称训练数据准确率仅约 60%，但首页展示效果具有误导性
- 有人尝试 10 张简单图像、每张运行 50 次，无一成功

### 2. CAD 工作流的真正难点

评论普遍认为：
- CAD 的核心挑战在于尺寸、间距、公差等约束设计，而非绘图本身
- GenCAD 无法处理参数化约束，不解决真正的痛点
- 被认为是"为不存在的问题提供解决方案"

### 3. 潜在应用场景

部分用户看到了价值：
- 木工等从业者快速生成可调整的草图
- 3D 打印爱好者的起点设计
- 结合 LLM 实现自然语言驱动的 CAD

### 4. 现有替代方案

- **OpenSCAD + LLM**: 已被多位用户验证有效的工作流
- **Build123d / CADQuery**: 基于 OpenCASCADE 的 B-Rep 建模
- **zoo.dev**: 自研 CAD 引擎 + KCL 语言 + LLM

### 5. 技术局限性

- CSG vs B-Rep: OpenSCAD 使用 CSG，与真正的参数化 CAD（B-Rep）差距明显
- 容差问题: 复杂模型中曲面交叉处理高度依赖几何内核
- 输出格式: 基于 DeepCAD JSON，非标准 CAD 格式

### 6. 产品质量问题

- 首页视频自动播放影响用户体验
- 移动端显示效果差
- 依赖 Docker 但存在依赖缺失问题

## 社区态度

整体偏负面，主要担忧：
- 夸大宣传，实际效果与演示差距大
- 对非训练数据泛化能力差
- 相比现有 LLM + OpenSCAD 方案无明显优势
