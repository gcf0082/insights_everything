## 内容概述

Zerostack 是一个用纯 Rust 编写的类 Unix 编程代理工具，旨在提供一个轻量级的 AI 辅助编码解决方案。该项目的核心理念是最小化资源占用，同时保持强大的功能特性。

## 核心特性

### 1. 超低内存占用
- **空会话**: 约 8MB
- **工作状态**: 约 12MB
- **对比**: Claude Code 需要多个 GB 内存
- **优化手段**:
  - 使用 Rust 而非 JS/Python（无额外运行时）
  - 按需加载（仅在需要时分配 LLM 连接器）
  - 使用 `smallvec` 存储数组（在栈上存储最多 N 个元素）
  - 使用 `compactstring` 处理字符串（在栈上存储最多 N 个字符）
  - 编译选项 `opt-level=z`（优化二进制大小而非性能）
  - 启用 LTO（链接时优化）

### 2. 权限管理模式
项目实现了四级权限系统：
- **Restrictive**: 无命令执行权限
- **YOLO**: 允许代理执行任意操作
- 支持自定义正则表达式模式来控制 bash 调用的允许/询问/拒绝
- 可通过 `zerostack -R` 强制所有工具请求权限

### 3. 设计决策
- **无 Skills 机制**: 采用提示库方式（~/.config/zerostack/prompts/）替代
- **编译时特性**: 可禁用可能使代理臃肿的功能
- **单线程 tokio**: 使用 `#[tokio::main(flavor = "current_thread")]` 节省约 50% 内存
- **无隐藏遥测**: 仅通过 MCP（via rmcp）和 LLM 连接器（via rig）进行 HTTP 调用

## 社区讨论热点

### 内存占用讨论
社区对超低内存占用反响热烈，多位用户表示这解决了在低端设备上运行编程代理的痛点。有用户提到这使得在 x1 small instances（如 shellbox.dev）上运行成为可能。

### 与其他工具的对比
- **Pi**: 基于 TypeScript，支持自生成工具
- **Claude Code**: 内存占用过高
- **Codex**: 也是 Rust 实现但是闭源
- **airun**: 类似的极简设计理念

### 脚本层方案
讨论中推荐了 **rhai** 作为 Rust 的嵌入式脚本引擎，适合大多数用例（除高性能脚本场景）。作者也考虑过 Zig 的 CompTime 特性或 WASM，但认为需要编译步骤。

### 代码审查反馈
- 所有依赖都是最新版本
- 建议使用 `strum` crate 简化枚举的字符串转换
- panic handler 设置为 "abort" 会导致崩溃时无堆栈跟踪，作者已改回默认设置

## 总结

Zerostack 代表了编程代理领域的"轻量级"趋势，在资源受限环境下具有明显优势。社区对其超低内存占用和简洁设计给予高度评价，同时也期待更多截图和使用演示。
