## 项目概述

SentrySearch 是一个基于人工智能的视频语义搜索工具，专门设计用于在行车记录仪 footage 上进行自然语言搜索。用户只需输入描述性文本，即可获得相关的视频片段。

## 核心功能

### 1. 语义搜索
- 通过文本查询直接搜索视频内容
- 支持如 "red truck running a stop sign" 这样的自然语言描述
- 返回最匹配的视频片段及其相似度分数

### 2. 视频处理
- 将视频分割为重叠的片段（默认30秒，5秒重叠）
- 使用 Google 的 Gemini Embedding 2 模型将视频片段转换为向量
- 自动检测静止帧并跳过，减少不必要的 API 调用

### 3. 向量存储
- 使用 ChromaDB 本地向量数据库存储视频嵌入
- 支持增量索引和统计信息查询

## 技术架构

### 核心技术栈
- **Python**：3.10+
- **向量嵌入**：Google Gemini Embedding 2（原生支持视频）
- **向量数据库**：ChromaDB
- **视频处理**：FFmpeg

### 工作原理
1. 将行车记录仪视频分割为重叠的片段
2. 使用 Gemini Embedding 2 将每个视频片段直接嵌入为 768 维向量
3. 将向量存储在本地 ChromaDB 数据库中
4. 搜索时，将文本查询嵌入到同一向量空间
5. 通过相似度匹配找到最相关的视频片段
6. 自动裁剪并保存匹配的视频片段

## 使用方式

### 安装
```bash
git clone https://github.com/ssrajadh/sentrysearch.git
cd sentrysearch
python -m venv venv && source venv/bin/activate
pip install -e .
```

### 配置
```bash
sentrysearch init
# 输入 Gemini API 密钥
```

### 索引视频
```bash
sentrysearch index /path/to/dashcam/footage
```

### 搜索
```bash
sentrysearch search "red truck running a stop sign"
```

## 成本分析

- **索引费用**：约 $2.50/小时（默认设置：30秒片段，5秒重叠）
- **搜索费用**：可忽略（仅文本嵌入）

### 优化选项
- **预处理**（默认开启）：降采样到 480p@5fps，提高速度
- **静止帧跳过**（默认开启）：跳过无视觉变化的片段，节省 API 调用

## 局限性

1. **静止帧检测**：基于 JPEG 文件大小比较，可能偶尔误判
2. **搜索质量**：取决于片段边界，事件可能跨越两个片段
3. **API 预览版**：Gemini Embedding 2 仍处于预览阶段，行为和定价可能变化

## 适用场景

- Tesla Sentry Mode 视频搜索
- 任何 mp4 格式的行车记录仪 footage
- 需要在大量视频中快速找到特定事件的应用
