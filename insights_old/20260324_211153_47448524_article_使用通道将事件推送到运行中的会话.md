# Claude Code Channels 文档总结

## 概述

Channels 是 Claude Code 的一项功能，允许通过 MCP 服务器将消息、警报和 Webhook 推送到正在运行的 Claude Code 会话中。用户离开时，Claude 可以接收 CI 结果、聊天消息和监控事件并作出响应。

**重要限制**：
- 需要 Claude Code v2.1.80 或更高版本
- 需要 claude.ai 登录（不支持 Console 和 API Key 认证）
- Team 和 Enterprise 组织必须显式启用此功能
- 目前处于研究预览阶段

## 支持的渠道

### Telegram

配置步骤：
1. 通过 BotFather 创建 Telegram 机器人，获取 Token
2. 安装插件：`/plugin install telegram@claude-plugins-official`
3. 配置 Token：`/telegram:configure <token>`
4. 重启 Claude Code：`claude --channels plugin:telegram@claude-plugins-official`
5. 配对账户：发送消息给机器人，获取配对码，然后在 Claude Code 中执行 `/telegram:access pair <code>`
6. 设置白名单：`/telegram:access policy allowlist`

### Discord

配置步骤：
1. 在 Discord 开发者门户创建应用程序和机器人
2. 启用 Message Content Intent
3. 生成邀请 URL 并将机器人添加到服务器
4. 安装插件：`/plugin install discord@claude-plugins-official`
5. 配置 Token：`/discord:configure <token>`
6. 重启 Claude Code：`claude --channels plugin:discord@claude-plugins-official`
7. 配对账户并设置白名单

## 快速开始 (Fakechat)

Fakechat 是一个本地演示通道，无需认证即可测试：

1. 安装：`/plugin install fakechat@claude-plugins-official`
2. 重启：`claude --channels plugin:fakechat@claude-plugins-official`
3. 在浏览器访问 http://localhost:8787 发送消息

## 安全机制

- 每个通道插件维护发送者白名单，只有添加的 ID 才能推送消息
- 通过配对流程将用户 ID 添加到白名单
- `--channels` 参数控制每个会话启用的服务器
- 白名单还控制权限转发功能

## 企业控制

| 计划类型 | 默认行为 |
|----------|----------|
| Pro / Max（无组织） | 渠道可用，用户通过 `--channels` 手动启用 |
| Team / Enterprise | 默认禁用，需管理员显式启用 |

管理员可在 claude.ai 管理设置中启用 channels。

## 研究预览

- 目前仅接受来自 Anthropic 维护白名单的插件
- 批准列表：claude-plugins-official
- 语法和协议可能会根据反馈更改
- 测试自定义通道需使用 `--dangerously-load-development-channels`

## 功能对比

| 功能 | 用途 |
|------|------|
| Web 上的 Claude Code | 在云沙箱中运行任务 |
| Slack 中的 Claude | 从团队对话中启动任务 |
| 标准 MCP 服务器 | 按需访问系统 |
| 远程控制 | 从手机控制本地会话 |
| **Channels** | 将外部事件推送到已运行的本地会话 |
