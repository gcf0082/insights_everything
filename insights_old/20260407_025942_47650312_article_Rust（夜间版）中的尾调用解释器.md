# 洞察报告：Rust 尾调用解释器

## 基本信息

- **链接**: https://www.mattkeeter.com/blog/2026-04-05-tailcall/
- **作者**: Matt Keeter
- **发布日期**: 2026-04-05
- **主题**: 使用 Rust 夜间版 `become` 关键字实现高性能尾调用解释器

## 概述

本文介绍了作者使用 Rust 夜间版新增的 `become` 关键字实现 Uxn CPU 尾调用解释器的经验。该实现不仅超越了作者之前的 Rust 代码，还击败了手写的 ARM64 汇编代码。

## 核心洞察

### 1. Uxn CPU 简介

Uxn 是一个简单的栈式虚拟机，拥有 256 条指令，架构包括：
- 两个 256 字节的栈（数据栈和返回栈）
- 65536 字节的 RAM
- 2 字节的程序计数器
- 256 字节的设备内存

### 2. 尾调用优化的基本原理

尾调用解释器的核心思想：
- 将程序状态存储在函数参数中（映射到寄存器）
- 每个函数结束时调用下一个函数而不是返回

关键实现代码示例：
```rust
match core.inc::<FLAGS>(pc) {
    Some(pc) => {
        let op = core.next(&mut pc);
        become TABLE.0[op as usize](...);  // 使用 become 关键字
    }
    None => (core, pc)
}
```

### 3. 性能对比结果

**ARM64 (M1 MacBook)**:
| 测试项目 | VM (ms) | 汇编 (ms) | 尾调用 (ms) |
|---------|---------|-----------|-------------|
| Fibonacci | 2.41 | 1.32 | **1.19** |
| Mandelbrot | 125 | 87 | **76** |

**x86-64**:
| 测试项目 | VM (ms) | 汇编 (ms) | 尾调用 (ms) |
|---------|---------|-----------|-------------|
| Fibonacci | 4.70 | 1.84 | 3.23 |
| Mandelbrot | 264 | 168 | 175 |

### 4. 关键发现

1. **ARM64 大获全胜**: 尾调用解释器在 ARM64 上全面超越手写汇编
2. **x86 仍有差距**: x86-64 上尾调用仍落后于汇编（Fibonacci 差距明显）
3. **WebAssembly 表现糟糕**: 尾调用在 WASM 上比简单 VM 更慢
   - Firefox: 尾调用 311ms vs VM 264ms
   - Chrome: 尾调用 905ms vs VM 244ms
   - wasmtime: 尾调用 595ms vs VM 128ms

### 5. 代码生成分析

**ARM64 生成的代码**非常高效：
```armasm
and     x8, x0, #0xff
ldr     w9, [x21, x8]
add     w9, w9, #1
strb    w9, [x21, x8]
ldr     x10, [x8, x9, lsl #3]
br      x10
```

**x86-64 生成的代码**存在大量寄存器溢出：
```asm
push   rbp
mov    QWORD PTR [rsp-0x8],r11
; ... 大量寄存器移动 ...
pop    rbp
jmp    rax
```

### 6. 技术要点

- 使用 `extern "rust-preserve-none"` 声明函数以避免默认调用 convention 的寄存器限制
- 宏用于消除样板代码，同时保持 100% 安全 Rust（无 unsafe 代码）
- `become` 关键字确保调用者栈帧直接被被调用者替换，避免栈溢出

## 结论

1. **ARM64 成功**: 尾调用解释器成为 ARM64 上的默认后端
2. **x86 需优化**: x86-64 上仍使用汇编作为首选
3. **WASM 不适用**: 尾调用模式不适合 WebAssembly 堆栈机
