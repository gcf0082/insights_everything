## 项目简介

WSL9x 是一个让现代 Linux 内核（6.19 版本）运行在 Windows 9x 内核中的项目。它允许用户同时利用两个操作系统的完整功能，包括分页、内存保护和抢占式调度。该项目自豪地声明：不使用 AI 编写。

## 技术架构

WSL9x 由三个组件构成：

### 1. 打补丁的 Linux 内核
- 基于 `win9x-um-6.19` 分支
- 内核编译为固定基地址 `0xd0000000`
- 基于 user-mode Linux，但修改为调用 Windows 9x 内核 API
- 运行在 ring 0（内核模式）而非 ring 3（用户模式）
- 主要代码位于 `linux/arch/um/os-Win95`

### 2. VxD 驱动程序
- 负责 WSL9x 的初始化
- 设置内核代码的初始映射
- 使用 DOS 中断从磁盘加载 `vmlinux.elf`
- 在 System VM 中启动新线程
- 分配 16 KiB 栈用于进入 Linux
- 事件循环处理：进入内核、分发 IRQ、返回用户空间、空闲
- 处理页面错误和系统调用
- 系统调用通过通用保护错误（GPF）处理器处理

### 3. wsl.com 客户端
- 一个 16 位 DOS 程序
- 使用 MS-DOS 提示符作为 TTY 窗口
- claim 空闲的控制台
- 事件循环等待 IRQ 并读取键盘
- 使用 `int 0x29` 中断输出字符到 DOS 窗口

## 构建与运行

### 构建要求
1. **交叉工具链**：需要针对 `i386-linux-musl` 的交叉编译工具链，可使用 musl-cross-make 构建
2. **Open Watcom v2**：用于构建 Windows 组件，需设置 `WATCOM` 环境变量
3. **Linux 内核**：需要构建打补丁的 Linux 内核，执行 `make build-linux -j $(nproc)`
4. **Windows 9x 硬盘镜像**：需要预装 Windows 9x 的硬盘镜像 `hdd.base.img`

### 运行步骤
1. 执行 `make` 生成新的 `hdd.img`
2. 在 MS-DOS 提示符下运行 `wsl` 打开 pty
3. 如需 ANSI 颜色支持，建议使用 `nnansi.com` 等驱动程序

## 项目特点

- **双系统融合**：在 Windows 9x 内核中运行完整的现代 Linux 内核
- **无需重启**：同时使用两个操作系统的全部功能
- **先进技术**：支持分页、内存保护和抢占式调度
- **纯手工代码**：不使用 AI，完全由开发者手动编写
