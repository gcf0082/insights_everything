## 项目概述

**WSL9x** (Windows 9x Subsystem for Linux) 是一个能够让现代 Linux 内核（6.19版本）在 Windows 9x 内核中协同运行的项目。该项目由 Hailey Somerville 开发，代码托管于 Codeberg: https://codeberg.org/hails/wsl9x

### 核心特性

- 在 Windows 9x 系统上直接运行 Linux 应用程序
- 无需硬件虚拟化（通过虚拟 8086 模式实现）
- 支持分页、内存保护和抢占式调度
- 可让用户同时使用两个操作系统的完整功能
- 无需重启即可在系统间切换

### 技术实现

该项目是作者在 2020 年发布的 **DOS Subsystem for Linux** (DSL) 的后续作品，相隔 6 年完成。与传统的 WSL（Windows Subsystem for Linux）不同，这个项目是让 Linux 运行在 Windows 9x 上，而非在 Windows NT 上运行。

## 热门评论摘要

### 1. 技术历史背景 (rahen)
在 WSL 之前，在 Windows 内运行未经修改的 Linux 二进制文件的最佳方式是 CoLinux 和 flinux。Cygwin 是正确的方法：Windows 上的原生 POSIX 二进制文件，而不需要任何虚拟化。

### 2. 命名讨论 (vrganj, jeroenhd)
关于 WSL 命名的一贯困惑：WSL 应该是 "Linux subsystem for Windows" 还是 "Windows subsystem for Linux"？这是因为 "Windows subsystem" 是 NT 架构中已有的术语。"It can work either way."
