## 内容摘要

这是一个名为 **Sonar** 的轻量级 CLI 工具的展示帖子，作者因为经常同时运行多个本地项目，经常遇到端口冲突问题（如 Redis 占用、Docker 容器未清理等）而开发了此工具。

### 核心功能

- **单一二进制文件**，无任何依赖
- 显示所有监听 localhost 的进程信息，包括：
  - 进程名称
  - Docker 容器信息
  - 可点击的 URL
- 支持通过端口号直接终止进程，如 `sonar kill 3000`
- 支持代理编排工具的使用场景

### 解决的问题

- 快速查看占用端口的进程
- 一键终止占用特定端口的进程
- 清理遗忘的 Docker 容器
- 处理多个 agent 同时运行时的端口管理

## 社区讨论要点

### 正面反馈

- 许多开发者表示经常忘记 `lsof` 命令的正确用法，这个工具非常实用
- Docker 容器检测功能被评价为「nice touch」
- 工具获得201分，表明社区对此工具有一定认可

### 质疑与讨论

- **名称混淆**：有评论指出「Sonar」可能与 SonarQube 混淆
- **必要性讨论**：部分用户认为简单的 alias 就能实现类似功能
- **替代方案**：提到了其他类似工具如 `murder`、`killport`、`porthole` 等

### 技术讨论

- 讨论了 Unix domain sockets 作为端口的替代方案
- 探讨了进程组管理中的子进程清理问题
- 建议添加系统托盘功能用于监控

## 相关工具推荐

| porthole | 类似的 dotfiles 工具 |
| portless | 将端口映射为 .localhost 域名 |

## 结论

Sonar 是一个针对开发者日常痛点的小工具，解决了多项目同时运行时端口管理的问题。虽然功能可通过 shell alias 实现，但其更好的用户体验和额外功能（如 Docker 集成）使其具有一定的实用价值。该项目现已开源，欢迎贡献和反馈。
