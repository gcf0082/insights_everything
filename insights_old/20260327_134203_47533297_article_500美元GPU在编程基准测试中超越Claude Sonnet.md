## 项目概述

A.T.L.A.S 是一个开源的本地 LLM 推理管道，旨在通过约束驱动生成和自验证迭代优化，使冻结的 14B 模型在单张消费级 GPU 上达到接近前沿 API 模型的性能表现。

### 核心特性

- **无微调、无 API 调用、完全本地部署** — 无需云服务，无数据离开机器，无需 API 密钥
- **成本优势明显** — 仅需电力成本（约$0.004/任务），对比 API 方案可节省 90% 以上费用
- **自验证迭代修复** — 通过 PR-CoT 修复失败任务，使用模型自生成的测试用例进行内部验证
- **Geometric Lens 能量评分** — 使用 5120 维自嵌入进行候选选择（87.8% 准确率）

## 技术架构

### 三阶段流水线

```
Phase 1 (生成)          验证               Phase 3 (修复)
┌─────────────┐        ┌─────────┐        ┌─────────────┐
│  PlanSearch │  k=3   │ Geometric│ 失败   │ Self-Test  │
│  Constraint │──────▶│  Lens   │───────▶│   Gen      │
│  Extraction │        │  C(x)   │        │  I/O pairs │
└─────────────┘        └─────────┘        └─────────────┘
       │                     │                     │
       ▼                     ▼                     ▼
┌─────────────┐        ┌─────────┐        ┌─────────────┐
│   Budget    │        │Sandbox  │        │  PR-CoT     │
│  Forcing    │──────▶│   Code  │        │  Repair     │
│ Thinking    │        │Execution│        │ Multi-persp │
│   Tokens    │        │         │        │ CoT         │
└─────────────┘        └─────────┘        └─────────────┘
```

### 关键组件

- **PlanSearch** — 约束提取与多样化规划生成
- **Budget Forcing** — 思考 token 控制
- **Geometric Lens C(x)** — 能量评分选择最佳候选
- **Sandbox** — 隔离代码执行环境
- **PR-CoT** — 多视角思维链修复

## 已知局限性

1. **仅针对 LiveCodeBench 优化** — V3 针对 LCB 设计和调优，GPQA 和 SciCode 未优化
2. **Phase 2 贡献为 0** — Geometric Lens 训练数据集仅约 60 个样本，无法有效区分候选
3. **G(x) 指标张量未激活** — 因 C(x) 训练不足，G(x) 无法提供有意义的梯度校正
4. **单线程任务管道** — 任务顺序处理，V3.1 将添加并行化
5. **SandboxAdapter stdio bug** — S* 区分输入决胜在 LCB 任务上因 stdio 处理 bug 不可用

## 路线图 (V3.1)

- **模型切换** — Qwen3-14B → Qwen3.5-9B + DeltaNet 线性注意力架构
- **Lens 进化** — 基于基准反馈的在线 C(x) 重新校准
- **Phase 2 重新设计** — 重新训练 C(x)，修复 SandboxAdapter stdio bug
- **任务并行化** — 并行任务执行以加快基准测试
- **更广泛基准套件** — 扩展到 GPQA、AA-LCR、AA-Omniscience 等

**目标:** 80-90% LCB pass@1-v(k=3)，更快的每任务吞吐量

## 总结

ATLAS 是一个创新性的本地 LLM 推理框架，通过基础设施层面的优化（结构化生成、能量验证、自修复）弥补模型规模差距。在仅使用单张消费级 GPU（RTX 5060 Ti 16GB）的情况下，实现了 74.6% 的 LiveCodeBench 得分，接近 Claude 4.5 Sonnet 和 GPT-5 的水平，同时成本降低 90% 以上。

该项目代表了"设备端 AI"的重要进展，证明了通过智能基础设施可以在消费级硬件上实现接近前沿模型的编码能力，而无需依赖云服务或 API 调用。
