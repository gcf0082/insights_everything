# 洞察报告：GitHub 近期服务问题

## 基本信息

- **洞察链接**: https://news.ycombinator.com/item?id=47924775
- **标题**: GitHub is having issues now
- **发布者**: SenHeng
- **发布时间**: 1小时前
- **分数**: 166 points
- **评论数**: 65 comments
- **外部链接**: https://www.githubstatus.com

## 摘要

GitHub 当前正经历服务问题，用户报告 PR 页面和 Issues 页面加载时显示零项目，偶尔会出现"获取数据失败"的错误提示。

## 主要讨论话题

### 1. 微软收购后的服务下滑

有用户指出 GitHub 自微软收购以来一直存在问题，并分享了一个历史正常运行时间图表。有评论认为该图表可能只是反映了微软收购后开始认真追踪宕机事件，但也有人认为最近6个月服务确实明显变差，可能是微软导致的质量下降和AI导致的服务规模增加的综合结果。

### 2. Azure 迁移问题

多位用户提到 GitHub 正在从自有数据中心迁移到 Azure，这一迁移过程似乎不太顺利。用户还指出不能将2019年12月的正常运行时间问题归咎于新冠疫情。

### 3. 替代方案建议

社区强烈建议考虑替代方案：

- **Gitea/Forgejo**: 自托管解决方案，用户反馈拥有更好的正常运行时间和响应速度
- **GitLab**: 提供类似但不同的体验
- **Sourcehut**: Drew DeVault 强烈反对 AI，有良好的运营历史
- **Codeberg**: 许多用户感兴趣的项目所在地
- **Gerrit**: 被描述为最专业的代码审查体验

### 4. 中心化风险

有用户指出，像 git、email 和互联网本身这些设计为去中心化的系统，最终却成为了最中心化、拥有单点故障的系统。评论认为将所有代码集中到 GitHub 并非好主意。

### 5. AI 影响

有用户担心随着越来越多由 AI 生成的代码进入平台，GitHub 的问题可能会进一步恶化。Copilot 的问题也可能延长恢复时间。

### 6. 沉默的失败

最严重的问题之一是 GitHub 会"沉默地失败"——例如显示"没有任何打开的 PR"，而实际上有数十个 PR 存在。这种误导性行为让开发者难以发现问题所在。

## 结论

本次讨论反映了开发者社区对 GitHub 服务稳定性的持续担忧。多数用户认为现在是认真考虑自托管或其他托管替代方案的时候，以避免持续的服务中断影响业务工作流程。
