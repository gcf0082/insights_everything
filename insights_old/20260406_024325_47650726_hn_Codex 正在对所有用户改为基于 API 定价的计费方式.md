## 核心内容

OpenAI Codex 将对所有用户改用基于 API 定价的按 token 使用量计费方式。这一变化意味着：

1. **订阅用户也将受到影响** - 现有的 Plus、Pro 和 Enterprise/Edu 计划用户将在未来几周内迁移到新费率
2. **从按消息计费改为按 token 计费** - 信用额度现在直接与 token 使用量挂钩，而非之前的按消息数估算
3. **额外积分购买保持不变** - 订阅中包含的权益不变，只是额外积分的计算方式从基于消息改为基于 token

## 社区反应分析

### 主要观点

**质疑与批评**：
- 许多用户认为这是"价格上涨"的表现，用"积分"而非美元定价是为了掩盖实际成本
- 有人将这种做法比作游戏中的"宝石包"，旨在混淆用户的实际支出
- 用户担心无法提前知道查询的实际成本

**理解与支持**：
- 部分用户认为这使用量计算更加透明，取代了之前不透明的按消息估算方式
- 有用户指出这与 Google、Anthropic 的定价变化趋势一致

**替代方案讨论**：
- Google AI Pro ($20/月) 被认为是更划算的选择
- Z.ai coder plan ($27/月) 被推荐为替代方案
- 开源模型如 Kimi K2.5、DeepSeek V3.2 等也被提及为便宜替代品

### 市场影响预测

- 有评论认为这标志着"AI 订阅补贴时代正在结束"
- 有人猜测这与 OpenAI IPO 即将到来有关
- 社区普遍认为 AI 公司正从"增长优先"转向"盈利优先"阶段

---

## 总结

此次 Codex 定价调整是 AI 行业从订阅制向按量付费转型的一部分。社区反应混合了对透明度提升的认可和对潜在价格上涨的担忧。随着竞争加剧，用户正在积极寻找更便宜的替代方案，包括 Google AI 和开源模型。
