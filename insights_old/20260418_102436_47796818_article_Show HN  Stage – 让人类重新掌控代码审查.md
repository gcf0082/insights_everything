## 核心产品功能

### 1. 叙事章节 (Narrative Chapters)

- 将 PR 分解为逻辑章节，讲述一个故事
- 理解为什么，而不仅仅是做什么

### 2. AI 驱动的摘要 (AI-Powered Summaries)

- 为每个章节提供即时上下文
- 确切地知道什么改变了以及为什么重要

### 3. 语义分组 (Semantic Grouping)

- 将相关变更分组在一起
- 不再需要在不相关的文件之间跳来跳去

### 4. 更快的审查 (Faster Reviews)

- 自信地审查
- 把时间花在理解上，而不是导航

## 产品示例展示

Stage 展示了一个 PR (PR #142 — feat: Add user authentication) 被分解为以下章节：

| 章节 | 文件数量 | 描述 |
|------|----------|------|
| Authentication Core | 4 files changed | 设置身份验证工具，包括令牌生成、验证和核心 AuthService 类 |
| Database Schema | 2 files changed | 数据库模式变更 |
| API Middleware | 3 files changed | API 中间件 |
| UI Components | 5 files changed | UI 组件 |

## 解决的问题

1. **团队快速迭代但理解不足**: 团队现在比以往任何时候都更快地发布代码，但工程师和审查者都不完全理解他们合并的变更
2. **生产环境 Bug**: 导致生产环境中的 Bug
3. **认知债务**: 造成过多的认知债务

## 市场竞争

Stage 处于 AI 代码审查工具市场的主要竞争对手包括：

- **Octopus** - 使用 LLMs 和向量搜索自动审查 PR
- **Codex (/review)** - 在 CLI、App 和 GitHub PR 上提供代码审查
- **Propel** - GPT-5 审查代理，带有确定性差异和路径感知路由
- **gstack Review Phase** - 具有 18 个专门角色的结构化 AI 审查
- **Archbot** - 代理式 AI 代码审查

## 总结

Stage 是一个专为现代快速迭代团队设计的 AI 代码审查平台。其核心创新在于将代码变更组织成结构化的"叙事章节"，帮助团队理解变更的背景和原因，而不仅仅是查看差异。创始团队来自斯坦福和 Five Rings，已获得 Y Combinator 支持，当前状态为 Active/即将推出。
