## 一、产品概述

Stash 是一个面向 AI Agent 的**持久认知层**（Persistent Cognitive Layer），扮演"AI 的第二个大脑"的角色。它并非简单存储数据，而是将 AI 的原始经验转化为结构化知识，形成不断进化的理解体系。

核心理念：Your AI is the brain. Stash is the life experience.

| 支持的 Agent 数量 | 无限 |

## 二、解决的核心问题

### 2.1 AI "健忘症"

AI 模型推理能力强但缺乏记忆，每个会话都需要用户重新解释背景信息，导致时间浪费和体验碎片化。

### 2.2 上下文窗口成本高昂

将完整对话历史塞入每次提示词的方案速度慢、成本高，且仍然会达到 token 上限，造成重复记忆的资源浪费。

### 2.3 Agent 重复犯错

Agent 尝试某方案失败后，下一个会话仍会尝试完全相同的方案，缺乏将失败经验转化为长期知识的能力。

### 2.4 记忆是平台特权

目前只有少数 AI 平台为自己的模型提供记忆功能，定制 Agent、本地 LLM、Cursor 等工具均从零开始，记忆不应是高价功能。

## 三、核心功能架构

### 3.1 记忆类型层次

Stash 将记忆分为五个层级：

1. **Episodes（片段）**：原始观察记录，仅追加
2. **Facts（事实）**：由 LLM 综合的带置信度的结构化信念
3. **Relationships（关系）**：实体的知识图谱边
4. **Patterns（模式）**：高阶抽象，如目标、失败案例、假设
5. **Goals · Failures · Hypotheses（目标/失败/假设）**：意图、学习、不确定性的追踪

### 3.2 命名空间（Namespaces）

记忆按路径组织，支持层级结构：

```
📁 /users/alice           — 用户信息与偏好
📁 /projects              — 所有项目
📁 /projects/restaurant   — 餐厅 SaaS 项目
📁 /self                  — Agent 自我认知
   /self/capabilities     — 擅长什么
   /self/limits           — 弱点记录
   /self/preferences      — 工作偏好
```

- **递归读取**：从 `/projects` 读取自动包含所有子路径
- **精确写入**：始终写入一个确定的命名空间，避免交叉污染
- **清晰分离**：用户记忆、项目记忆、Agent 自我认知互不干扰

### 3.3 综合流水线（Consolidation Pipeline）

后台进程持续将 Agent 经验综合为结构化知识，共 9 个阶段：

| 阶段 | 功能 |
|------|------|
| 01 Episodes | 原始观察即时存储 |
| 02 Facts | LLM 将片段聚类综合为事实 |
| 03 Relationships | 从事实中提取实体关系边 |
| 04 Causal Links | 事实间因果对抽取 |
| 05 Patterns | 高阶抽象洞察 |
| 06 Contradictions | 自我纠正与置信度衰减 |
| 07 Goal Inference | 事实自动追踪活跃目标，检测进展，暴露矛盾 |
| 08 Failure Patterns | 检测重复错误，提取为新事实 |
| 09 Hypothesis Scan | 新证据被动确认或拒绝开放假设 |

### 3.4 MCP 集成

Stash 原生支持 MCP（Model Context Protocol），提供 **28 个 MCP 工具**，覆盖从原始记忆到因果推理的完整认知栈：

- 基础：`remember`、`recall`、`forget`、`init`
- 高级：`goals`、`failures`、`hypotheses`
- 结构化：`consolidate`、`query_facts`、`relationships`、`causal_links`、`contradictions`
- 自我认知：`namespaces`、`context`、`self_model`

支持 Claude Desktop、Cursor、OpenCode、自定义 Agent、本地 LLM 等任何 MCP 兼容客户端。

### 3.5 Agent 自我模型

通过 `init` 调用，Stash 在 `/self` 命名空间创建脚手架，Agent 可利用自身的记忆层构建自身能力、局限和偏好的模型，实现真正的自我认知。

### 3.6 自主循环（Autonomous Loop）

用户可为 Agent 设置 5 分钟的研究循环：定向 → 研究 → 思考 → 发明 → 综合 → 反思休眠，Agent 以 cron 任务持续运行，每次运行后都会更聪明。

## 四、与 RAG 的对比

| 维度 | RAG | Stash |
|------|-----|-------|
| 定位 | 快速的图书馆管理员 | 不断成长的头脑 |
| 知识来源 | 仅文档 | 对话、决策、成功与失败 |
| 学习能力 | 无 | 自动从每次会话中学习 |
| 目标追踪 | 不支持 | 支持 |
| 因果推理 | 不支持 | 支持 |
| 自我纠正 | 不支持 | 支持矛盾检测与纠正 |
| 连续性 | 无状态，无连续性 | 持续记忆，跨会话接续 |
| 知识创建 | 必须手动写入 | 自动生成知识 |

Stash 适合需要持续学习和上下文记忆的场景，RAG 适合文档检索场景，两者可结合使用。

## 五、技术架构

### 5.1 基础设施

- **数据库**：PostgreSQL + pgvector（向量化存储）
- **部署方式**：Docker Compose（一键启动）
- **配置要求**：设置 `STASH_VECTOR_DIM`（仅需设置一次，之后不可更改）

### 5.2 模型无关性

Stash 使用 OpenAI 兼容的后端，支持任意模型提供商：

| 场景 | 推荐配置 |
|------|----------|
| 云端推荐 | OpenRouter（一个 API key 访问数百模型）|
| 本地部署 | Ollama + Qwen/Llama/Mistral |
| 高速推理 | Groq |
| 自托管 | vLLM、LM Studio、Together AI |

### 5.3 快速启动

```bash
# 1. 克隆
git clone https://github.com/alash3al/stash && cd stash
# 2. 配置
cp .env.example .env  # 编辑并设置 API key 和模型
# 3. 启动
docker compose up
```

## 七、总结与洞察

1. **市场定位**：Stash 填补了 AI Agent 持久记忆层的空白，将记忆从平台特权变为开源基础设施。

2. **差异化价值**：相比 RAG 和平台内置记忆，Stash 提供了真正的认知层——不仅存储，还能学习、推理、自我纠正。

3. **技术优势**：基于 PostgreSQL + pgvector 的架构成熟稳定，Docker 一键部署降低了使用门槛，MCP 原生集成使其具有广泛的兼容性。

4. **应用场景**：适合需要跨会话持续学习的 AI 应用、研究助手、自主 Agent、工作流自动化等场景。

5. **开源意义**：Apache 2.0 许可证确保任何人都可以使用、修改和商业化，推动 AI 记忆功能的民主化。
