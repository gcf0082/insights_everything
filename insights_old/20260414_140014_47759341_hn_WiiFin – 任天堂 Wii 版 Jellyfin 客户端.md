## 洞察摘要

### 1. Jellyfin 生态系统正在壮大

有用户注意到 Jellyfin 在 TrueNAS 应用目录中的安装量已超越 Plex（45,178 vs 42,225），WiiFin 的出现进一步证实了 Jellyfin 开发者生态系统的增强。

### 2. Plex 与 Jellyfin 的对比

**Plex 存在的问题**：
- 定价模式变化，将核心功能付费墙化
- UI 频繁更改，加入广告支持的流媒体频道
- 移动端离线下载功能存在严重 bug
- 移除了插件支持
- 用户感觉 Plex 正在偏离核心媒体共享功能

**Jellyfin 的优势**：
- 完全开源免费
- 社区可以 fork 继续开发（如从 Emby fork 出 Jellyfin）
- 支持多种设备（Samsung Tizen TV、iOS、Android 等）

### 3. Jellyfin 安全问题

用户指出 Jellyfin 安全模型较弱，存在未修复的漏洞。建议：
- 不要直接暴露到公网
- 使用 VPN（如 Tailscale）访问
- 在反向代理后添加独立认证（如 Authentik、Authelia）
- 支持 LDAP 认证

### 4. 技术讨论

**关于 Wii 解码能力**：
- Wii 支持 240p、480i、480p
- 需要服务器端转码（所有媒体都需要转码）
- 现代硬件（如 Intel ARC 380）可以轻松处理 4K 实时转码

**关于集群扩展**：
- Jellyfin 使用 SQLite，不适合多节点集群
- 去年数据库改进为 EF Core，为其他数据库引擎奠定基础
- 可通过 NFS 共享媒体存储，多个 Jellyfin 实例分别服务不同用户群

### 5. 其他评论

- 有用户提到 Wii 早在 PS5 之前就获得了客户端支持
- 讨论了 Wii 作为复古游戏设备的潜力
- Tailscale 可以轻松实现远程访问 Jellyfin
