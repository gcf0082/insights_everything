## 摘要

本报告介绍了 Tailscale 在 macOS 设备上面临的"刘海屏"问题及其解决方案。自 2021 年 MacBook Pro 机型引入刘海屏设计后，部分用户的 Tailscale 应用图标会被隐藏在刘海区域，导致无法正常使用。Tailscale 采用了两种应对策略：一是提供弹出式警告提示；二是推出全新的窗口化 macOS 客户端，从根本上解决了这一设计缺陷。

---

## 问题背景

### 刘海屏带来的挑战

Tailscale 最初在 macOS 上以命令行工具和菜单栏工具的形式呈现。自 2021 年起，部分 MacBook Pro 机型在屏幕顶部中央引入了刘海屏设计。由于菜单栏图标数量的限制，Tailscale 的图标有时会被隐藏在刘海区域，用户无法看到或点击。

苹果公司没有为开发者或用户提供解决这一问题的官方方案。当菜单栏图标数量超过刘海右侧的可用空间时，这些图标就会消失，用户无法获得任何通知或提示。

### 用户反馈

Tailscale 开发团队收到了大量用户报告，用户反馈称 Tailscale 图标"消失"或"无法启动"。开发团队不得不一遍遍解释："它其实在那里，只是被刘海遮住了。"

---

## 解决方案一：弹出式警告提示

### 技术实现

尽管菜单栏图标无法移动，但应用可以检测到自身是否被遮挡。Tailscale 利用 `occlusionState` 数据来判断图标是否处于被遮挡状态，并显示相应的警告信息：

> "太隐蔽了！Tailscale 图标隐藏在刘海后面。由于菜单栏空间有限，Tailscale 图标被隐藏了。请重新排列或移除其他图标来查看 Tailscale 菜单。"

### 代码示例

```swift
self.visibilityObserver = NotificationCenter.default.addObserver(
    forName: NSWindow.didChangeOcclusionStateNotification,
    object: statusItemButton.window,
    queue: .main
) { [weak self] _ in
    guard let self, statusItem.isVisible else { return }
    isIconOccluded = statusItem.button?.window?.occlusionState.contains(.visible) == false
}
```

### 局限性

这种警告机制并不完美，可能会因其他显示问题（如开关笔记本盖在显示器间切换）而触发。

---

## 解决方案二：全新窗口化 macOS 客户端

### 主要功能

Tailscale 推出了窗口化的 macOS 应用（版本 1.96.2 起默认启用），提供以下功能：

- **设备列表**：可搜索的 tailnet 设备列表及其连接状态
- **文件传输**：通过 Taildrop 轻松向设备发送文件
- **出口节点**：可搜索的出口节点，并基于延迟、性能和位置推荐最优节点
- **错误提示**：Dock 图标上的红点标记关键错误
- **迷你播放器**：将 Tailscale 收缩到最小界面

### 设计优势

窗口化客户端不取代菜单栏应用，而是与之并行运行。用户可以从 Dock 或 Spotlight 搜索启动应用，访问更多 Tailscale 数据和功能。

## 未来展望

Tailscale 团队表示正在为 Windows 设备开发类似界面，并持续优化产品功能。

## 总结

Tailscale 通过两种方式应对 macOS 刘海屏设计缺陷：一是利用弹出式警告提示帮助用户了解图标被遮挡的情况；二是通过推出窗口化客户端从根本上解决这一问题，为用户提供更完整的 macOS 使用体验。
