## 洞察摘要

本文是英国网页开发者 David Bushell 撰写的一篇关于网页设计中「滚动淡入效果」（Scroll Fade）的批评文章。作者以幽默犀利的语言表达了对这一流行设计趋势的强烈反对态度。

## 核心观点

### 1. 滚动淡入效果的问题

- **审美疲劳**：过度使用使网站看起来廉价且俗气
- **缺乏目的性**：几乎所有网站都使用相同的效果（1秒透明度过渡 + 100px Y轴位移），毫无创意
- **用户体验差**：只有以极慢的速度滚动才能达到较好的视觉效果

### 2. 反对滚动淡入的论点

**无障碍问题**：
- 可能引发前庭功能障碍用户的不适
- 虽然有 `prefers-reduced-motion` 媒体查询可以缓解，但作者认为应该是「运动选择加入」而非「选择退出」

**认知负担**：
- 滚动淡入会造成认知过载，分散用户注意力
- 在不同平台（macOS、Windows、Linux、Android）上的表现不一致

**性能影响**：
- 可能对 Core Web Vals 造成负面影响，特别是 Largest Contentful Paint (LCP)
- 需要大量 JavaScript 实现，增加了页面复杂性

### 3. 说服客户的策略

- 强调无障碍问题（更容易获得重视）
- 询问项目是否有时间和预算进行真实用户测试
- 测试性能、可用性和任务完成率
- 关注跳出率是否增加

### 4. 技术建议

作者提到可以使用 Scott Jehl 的「自毁 CSS」技术来保护页面在 JavaScript 不可用时的体验。

## 文章结论

作者呼吁开发者社区集体抵制滚动淡入效果，认为这不是一个可以随意添加到项目末尾的「快速解决方案」，而需要从第一天开始就进行规划。建议开发者对这类效果说「不」。
