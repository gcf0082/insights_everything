# OpenSSL 4.0.0 版本洞察报告

## 基本信息

- **洞察链接**: https://github.com/openssl/openssl/releases/tag/openssl-4.0.0
- **发布版本**: OpenSSL 4.0.0
- **发布日期**: 2025年4月14日
- **发布者**: openssl-machine
- **提交记录**: 304次提交至master分支

## 版本概述

OpenSSL 4.0.0 是一个功能发布版本，为 OpenSSL 引入了重要的新功能。该版本包含多个潜在的重大变更和不兼容变更，同时新增了多项新特性。

## 主要变更（不兼容变更）

### 1. 十六进制输出格式调整
- 打印密钥数据（如RSA模数）时，移除了前导的额外"00:"
- 签名十六进制转储宽度标准化为24字节（保持在80字符限制内）
- 其他数据十六进制转储宽度标准化为16字节

### 2. 安全检查增强
- 使用`PKCS5_PBKDF2_HMAC` API与FIPS提供者时，现在强制执行下限检查
- 设置`X509_V_FLAG_X509_STRICT`时添加AKID验证检查
- CRL验证流程新增多项额外检查

### 3. API和行为变更
- `libcrypto`不再通过`atexit()`清理全局分配的数据
- `BIO_snprintf()`现在使用libc提供的`snprintf()`而非内部实现
- `OPENSSL_ERROR`现在在全局析构函数中运行或默认不运行
- `ASN1_STRING`已变为不透明类型
- 多个API函数签名变更，添加了`const`限定符

### 4. 废弃和移除
- 废弃`X509_cmp_time()`、`X509_cmp_current_time()`、`X509_cmp_timeframe()`，推荐使用`X509_check_certificate_times()`
- 移除SSLv2客户端Hello支持
- 移除SSLv3支持（SSLv3自2015年起已废弃，OpenSSL自1.1.0版本起默认禁用）
- 移除引擎支持，`no-engine`构建选项和`OPENSSL_NO_ENGINE`宏始终存在
- 默认禁用TLS中废弃的椭圆曲线支持（RFC 8422），需使用`enable-tls-deprecated-ec`配置选项启用
- 默认禁用显式EC曲线支持，需使用`enable-ec_explicit_curves`配置选项启用
- 移除`c_rehash`脚本工具，改用`openssl rehash`
- 移除`openssl ca`命令中的废弃`msie-hack`选项
- 移除`BIO_f_reliable()`实现（自3.0版本起已损坏）
- 移除自定义`EVP_CIPHER`、`EVP_MD`、`EVP_PKEY`、`EVP_PKEY_ASN1`方法的废弃支持
- 移除废弃的固定SSL/TLS版本方法函数
- 移除废弃的`ERR_get_state()`、`ERR_remove_state()`、`ERR_remove_thread_state()`函数
- 从Configurations中移除`darwin-i386{,-cc}`和`darwin-ppc{,64}{,-cc}`目标平台

## 新增功能

### 1. 加密客户端Hello (ECH)
- 支持RFC 9849定义的加密客户端Hello
- 详见`doc/designs/ech-api.md`

### 2. 中国密码算法支持
- 支持RFC 8998定义的`sm2sig_sm3`签名算法
- 支持`curveSM2`密钥交换群
- 支持`curveSM2MLKEM768`后量子混合群

### 3. 密码学功能
- 支持cSHAKE函数（SP 800-185）
- 支持"ML-DSA-MU"摘要算法
- 支持SNMP KDF和SRTP KDF

### 4. FIPS模块改进
- FIPS自测试现在可以延迟运行
- 使用`openssl fipsinstall`命令的`-defer_tests`选项可实现按需运行

### 5. Windows支持
- 支持在Windows上使用静态或动态VC运行时链接

### 6. TLS 1.2改进
- 支持RFC 7919定义的协商FFDHE密钥交换

## 版本亮点

1. **安全性提升**: 增强了多项安全检查，包括AKID验证和CRL验证
2. **移除废弃协议**: 完全移除了SSLv3和SSLv2支持
3. **现代化**: 移除引擎支持，简化代码库
4. **后量子密码学**: 引入混合密钥交换群`curveSM2MLKEM768`
5. **中国标准支持**: 全面支持SM2/SM3密码算法

## 总结

OpenSSL 4.0.0是一个重要的里程碑版本，带来了显著的安全性改进和新功能。该版本清理了大量废弃功能，简化了API，同时引入了对后量子密码学和中国密码标准的支持。开发者需要注意一些不兼容变更，特别是SSLv3/SSLv2的移除和API签名的变更。
