## 内容摘要

本帖讨论了Anthropic新发布的Claude Code Routines功能，该功能允许用户设置自动化任务，按计划运行、响应API调用或GitHub事件触发。

## 主要观点

### 1. 用户信任问题（最高赞评论）

用户对Anthropic产品的主要担忧：

- **无信任不会削弱工具/模型**
- **无信任不会停止功能**（LLM功能墓地不断增长）
- **对公司长期发展无信任**（担心公司消失或政策突变）

用户表示：不想在"平台"上构建，只想像" dumb pipe"一样使用 commoditize 服务，不愿被锁定。

### 2. 服务条款不清晰

用户对ToS条款的理解混乱：

- 运行 `claude -p <prompt>` 在cron中是否可行？
- 在Telegram机器人中使用是否违规？
- 使用Routines功能的API回调是否违规？

许多用户批评Anthropic沟通策略分散在HN、Reddit、GitHub等多个平台，缺乏一致性。

### 3. 计算资源限制

用户观察到Anthropic的计算资源不足：

- 最近的1百万token上下文窗口导致了巨大负载
- 用户需手动重新启用"clear context and continue"功能
- 有人建议让用户自己裁剪上下文窗口

### 4. 用户用脚投票

一些用户选择降级或离开：

- 从$200/月Max订阅降级到$20
- 转向OpenAI Codex的Pro计划
- 批评adaptive thinking等功能决策

### 5. 积极用例

部分用户分享了Routines的成功应用：

- GitHub issue自动分类和PR创建
- Slack反馈渠道自动处理
- 每日活动汇总（GitHub、Slack、邮件）

## 社区情绪

- **担忧**：对Anthropic平台锁定的不信任
- **困惑**：服务条款不清晰
- **失望**：计算资源限制影响体验
- **期待**：对自动化功能的兴趣
