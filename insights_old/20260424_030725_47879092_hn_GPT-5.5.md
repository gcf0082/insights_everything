## 内容摘要

### 发布背景

OpenAI 于 2026年4月24日 发布 GPT-5.5 模型。该模型在所有三个评估基准上的分数均优于 GPT-5.4，同时使用的 token 数量更少。Codex 通过分析数周的 production traffic patterns 并编写自定义启发式算法来优化工作分配和负载平衡，将 token 生成速度提升了超过 20%。

### 主要特性

- **基准提升**：在 SWE-Bench、CyberGym 等基准上优于 GPT-5.4
- **token 效率**：使用更少的 token 实现更好的性能
- **定价**：API 价格是 GPT-5.4 的 2 倍，是 GPT-5.1 的约 4 倍，是 Kimi-2.6 的约 10 倍
- **推理延迟**：与 GPT-5.4 相当，使用 GB200 NVL72 提供服务
- **CyberGym 分数**：81.8%

### 社区讨论要点

#### 1. 逐步发布策略
由于服务稳定性考虑，GPT-5.5 在 ChatGPT 和 Codex 中的推送将逐步进行，从 Pro/Enterprise 账户开始，逐步扩展到 Plus 用户。

#### 2. 定价争议
有用户指出，GPT-5.5 在 API 定价上明显高于前代版本，虽然效率提升，但价格涨幅引发了对订阅用户实际利益的讨论。部分用户认为价格涨幅过大，特别是考虑到实际性能提升可能不足 10%。

#### 3. 模型"躺平"问题
有用户反映，在使用 GPT-5.4 API 时遇到模型无法驱动执行任务的问题，即使系统提示明确要求"不要省 token，主动思考"也无效。该用户最终转向使用 Kimi 2.6。其他人建议可以通过设置更高的推理预算参数来获得更好的性能。

#### 4. 与 Anthropic Opus 4.7 的比较
- Opus 4.7 在 SWE-Bench Pro 上仍领先于 GPT-5.5
- 有用户提到 Opus 4.7 有时会过度发散或不够投入
- GPT-5.5 在 token 效率上显著优于 Opus 4.7：同样的 AI Index 分数，Opus 4.7 使用 111M output tokens，GPT-5.5 仅使用 22M

#### 5. Three.js 和游戏开发
用户分享使用 GPT 模型配合 Three.js 进行游戏开发的经验。GPT-5.4 在 Three.js 代码生成方面有显著提升，特别是在 GLSL shader 方面。有用户展示了使用 Codex 和 GPT 模型生成的可玩的 3D 地牢竞技场原型。

#### 6. 基础设施优化
Codex 分析 production traffic patterns 并编写自定义启发式算法来优化 GPU 利用率，这是影响 token 生成速度的关键创新。

### 关键评价

- **正面**：token 效率提升显著，对于订阅用户来说实际可用配额可能增加 5 倍
- **负面**：价格上涨 100%，实际性能提升可能有限
- **争议**：模型在实际使用中的行为（如"躺平"）与基准表现存在差距
