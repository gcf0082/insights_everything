## 一、项目概述

Instant 是一个专为 AI 编码应用设计的后端平台，经过 4 年开发，于 2026 年发布 1.0 版本。Instant 将你喜欢的编码代理转变为全栈应用构建器，并且完全开源。

核心主张：Instant 是 AI 编码应用的最佳后端选择。

## 二、核心特性

### 2.1 无限应用

传统方案要么付费使用虚拟机，要么受到限制。许多服务限制免费应用数量，并在闲置时冻结应用，解冻往往需要 30 秒甚至几分钟。

Instant 的解决方案：
- 采用多租户架构，创建新项目不需启动虚拟机，只需在多租户实例中插入几行数据库记录
- 应用不活跃时无计算或内存成本
- 活跃时仅需约几 KB 的额外内存开销（而非虚拟机需要的数百 MB）
- 整个过程仅需几百毫秒

### 2.2 同步引擎

传统 CRUD 应用与现代应用（如 Linear、Notion、Figma）的区别：
- 现代应用是多玩家的、可离线工作的、响应快速的
- 在 Linear 中修改待办事项，所有地方都会同步更新
- 在 Notion 中离线仍可编辑文档
- 在 Figma 中给形状着色无需等待服务器

这些现代应用需要自定义基础设施：
- 实时功能需要状态 WebSocket 服务器
- 离线模式需要在 IndexedDB 中存储缓存
- 乐观更新需要处理客户端的变更应用和撤销

Instant 构建了通用同步引擎，每个应用默认具有多玩家、离线模式和乐观更新功能。

### 2.3 附加服务

Instant 内置了多项常用服务，全部作为单一集成系统工作：
- **文件存储**：文件对象即为数据库中的行，支持 CASCADE 删除规则
- **认证**：支持魔法码、OAuth 和访客认证
- **在线状态**：支持光标、输入指示器和"谁在线"标记
- **流媒体**：支持持久的流传输

## 三、系统架构

系统由三个核心组件构成：

```
┌─────────────┐      ┌─────────────────┐      ┌──────────────────┐
│  Client SDK  │ ───> │ Clojure Backend │ ───> │ Multi-Tenant DB  │
└─────────────┘      └─────────────────┘      └──────────────────┘
```

### 3.1 客户端 SDK

设计动机：需要支持离线工作和乐观更新。

核心组件：
- **IndexedDB**：用于在网页端存储数据，提供数 MB 存储空间和有限查询能力
- **Triple Store**：将数据存储为 `[entity, attribute, value]` 元组，可同时建模属性和关系
- **Datalog**：基于逻辑的查询引擎，可在客户端完全评估查询，无需等待服务器
- **Pending Queue**：存储待处理的变更，支持服务器拒绝时的回滚
- **Reactor**：主要状态机，协调所有流程，通过 WebSocket 与服务器通信

InstaQL 是 Instant 的查询语言，基于普通 JavaScript 对象构建，便于程序化生成查询。

### 3.2 Clojure 后端

设计动机：需要使查询具有响应性，并公平分配多租户资源。

核心组件：
- **Query Store**：存储哪些用户订阅了哪些查询
- **Topics**：将查询转换为"主题"集，描述查询关注的数据部分
- **Invalidator**：跟踪 Postgres 的 WAL（预写日志），生成事务主题，与查询主题匹配以发现过期数据
- **Grouped Queue**：每个应用有独立的子队列，保证应用内事务串行处理，跨应用可并行化
- **Session Manager**：协调所有组件，处理权限和请求路由

技术亮点：
- Clojure 的并发原语和真实线程支持横向扩展
- JVM 生态丰富，如使用 Google CEL 实现权限沙箱
- Clojure 适合 DSL 和实验性编程

### 3.3 多租户数据库

设计动机：需要廉价创建新数据库，且需要关系型支持。

核心方案：单一 `triples` 表，通过 `app_id` 逻辑隔离数据。

优势：
- 创建新数据库几乎免费（仅需几行数据库记录）
- 列创建无锁（Postgres 创建列会锁表）
- 支持软删除列，即使误删也可恢复

实现细节：
- **部分索引**：使用布尔标记实现唯一列约束
- **Count-Min Sketches**：解决 Postgres 在单一表情况下统计信息丢失问题，帮助估计列基数
- **查询引擎**：将 InstaQL 查询和统计信息转换为优化的 SQL 查询计划

## 四、技术栈亮点

| 组件 | 技术选择 |
|------|----------|
| 客户端存储 | IndexedDB |
| 查询语言 | InstaQL (基于 Datalog) |
| 后端语言 | Clojure |
| 数据库 | PostgreSQL (多租户 Triple Store) |
| 权限语言 | Google CEL |

## 五、总结

Instant 是一个创新的后端解决方案，专为 AI 编码时代设计。其多租户架构、强大的同步引擎和内置服务使其成为构建现代应用的理想选择。系统架构展示了如何在保持简单性的同时处理复杂的分布式系统挑战。
