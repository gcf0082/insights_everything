## 核心洞察摘要

### 背景介绍

美国政府公布了一项令人困惑的政策，有效禁止进口新型消费级路由器。作者认为这是荒谬的，并借此机会分享如何将任何类似电脑的设备变成路由器。作者已使用 Linux 迷你电脑作为路由器多年，运行稳定，唯一的问题是磨损了一个 20 美元的 mSATA 驱动器。

### 硬件选择

几乎任何设备都可以作为路由器使用：

- **推荐**: 被动散热设计的迷你电脑
- **备选**: 台式电脑、SBC（单板电脑）、旧笔记本电脑、 rackmount 服务器
- **网络接口**: 需要两个以太网接口，USB-以太网转换器也可凑合使用

即使是很旧的设备（如 Celeron 3205U 双核 1.5GHz）也能轻松处理整个家庭或小型企业的流量。作者曾用 ThinkPad T60、ExpressCard-PCIe 转接桥、古老 Cisco 100M 交换机等垃圾硬件搭建过路由器。

### 所需软件包

基础 Debian 系统安装外，还需要以下软件：

| 软件包 | 用途 |
|--------|------|
| hostapd | 创建 Wi-Fi 无线网络 |
| dnsmasq | DNS 和 DHCP 服务 |
| bridge-utils | 桥接网络端口 |
| firmware-iwlwifi | Intel 无线设备固件 |
| firmware-ath9k-htc | Realtek 无线设备固件 |

### 网络配置

网络接口布局：
- `eth0`: WAN（外部网络）
- `eth1`: LAN（有线网络）
- `wlan0`: LAN（无线网络）

有线和无线 LAN 接口通过桥接（bridge）连接，使两个网络设备可以正常通信。

### 关键配置步骤

1. **接口命名**: 为网络接口创建持久命名文件 `/etc/systemd/network/10-persistent-ethX.link`
2. **Wi-Fi 热点**: 配置 `/etc/hostapd/hostapd.conf`
3. **网络桥接**: 编辑 `/etc/network/interfaces`
4. **IP 转发**: 创建 `/etc/sysctl.d/10-forward.conf` 设置 `net.ipv4.ip_forward=1`
5. **防火墙**: 使用 nftables 配置 NAT 和访问控制规则
6. **DHCP/DNS**: 配置 dnsmasq 提供 IP 分配和域名解析

### 防火墙规则（nftables）

- 允许已建立连接的返回流量
- 允许 ICMP 协议
- 允许 LAN 端的 SSH（22）、DNS（53）、DHCP（67/68）
- 启用 NAT 伪装（masquerade）

### 扩展功能

搭建完成后，还可进一步配置：

- 日志记录和流量分析
- VLAN 网络隔离
- 告警系统
- IPv6 支持
- 实时监控
- 端口转发
- VPN 远程访问
- 动态路由协议（FRRouting）
- 网络过滤和阻止
- 入侵检测

---

## 总结

路由器的本质就是一台特殊的电脑。通过 Linux 系统，任何具备两个网络接口的设备都能充当路由器。这种方法不仅能充分利用现有硬件，还展示了网络设备的本质——它们并不特殊，只是运行特定软件的计算机而已。
