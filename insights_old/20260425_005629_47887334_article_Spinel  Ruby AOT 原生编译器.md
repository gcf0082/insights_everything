## 项目概述

Spinel 是 Ruby 的 AOT (Ahead-of-Time) 编译器，由 Ruby 之父 Matz (Yukihiro Matsumoto) 开发。它将 Ruby 源代码编译为独立的原生可执行文件，通过整程序类型推断和生成的优化 C 代码，实现显著的性能提升。

Spinel 是**自托管**的：编译器后端使用 Ruby 编写，并可将自身编译为原生二进制文件。

## 核心特性

- **自托管编译**: 编译器后端使用 Ruby 编写，能够编译自身
- **整程序类型推断**: 驱动多种编译时优化
- **独立二进制**: 生成的程序无需运行时依赖
- **内置正则表达式引擎**: 无需外部依赖
- **可选大整数**: 通过 mruby-bigint 实现

## 编译流程

1. **Ruby (.rb)** → 使用 Prism 解析器解析并序列化 AST
2. **AST 文本文件** → 类型推断 + C 代码生成
3. **C 源代码 (.c)** → 通过标准 C 编译器编译
4. **原生二进制** → 独立的可执行文件

## 优化技术

- **值类型提升**: 小型不可变类自动在栈上分配为 C 结构
- **常量传播**: 简单字面量常量在使用位置内联
- **循环不变式提升**: 数组/字符串长度在循环前计算一次
- **方法内联**: 短方法使用 static inline
- **字符串链拼接扁平化**: a + b + c + d 编译为单次 malloc 调用
- **Bigint 自动提升**: 循环乘法模式自动提升为大整数
- **静态符号内联**: 字面量符号在编译时解析
- **死代码消除**: 使用 -ffunction-sections 和 --gc-sections

## 构建步骤

```bash
make deps         # 获取 libprism 依赖
make              # 构建解析器 + 正则库 + 引导编译器
make test         # 运行功能测试
make bench        # 运行基准测试
make bootstrap    # 从源码重新构建编译器
sudo make install # 安装到 /usr/local
```

## 已知限制

- **无 eval**: eval、instance_eval、class_eval
- **无元编程**: send、method_missing、define_method
- **无线程**: Thread、Mutex
- **无编码**: 假设 UTF-8/ASCII
- **无一般 lambda  calculus**: 深度嵌套的 -> x { } 调用

## 依赖

- **构建时**: libprism、CRuby (仅引导)
- **运行时**: 无 (仅需 libc + libm)
- **正则表达式**: 内置引擎，无外部库
- **大整数**: 内置 (mruby-bigint)，仅在使用时链接
