## 主题摘要

本讨论围绕 Addy Osmani 发布的 "Agent Skills" 文章展开，主要涉及以下几个核心观点：

### 1. 持怀疑态度的观点

部分开发者认为 Agent Skills 实际上是" snake oil"（华而不实的产品），理由包括：

- LLM 无法完美遵循人类设定的规则，这些框架 pretxtend 可以通过添加更多规则来解决这个问题是一种根本性的认知误区
- slot machine 机制会drop任何硬性要求
- 只有人工审查和监督才能提高可靠性，但这时候所谓的生产力提升承诺同样不可靠

### 2. 积极实践者的观点

另一方面，有开发者分享了积极的使用体验：

- 使用 spec-kit 等 AI 使用方式已长达数月，效果"AMAZING"
- 能够构建真正有价值的东西，没有遇到文中提到的假设性问题
- AI 模型正在不断改进，犯的错误越来越少
- 结合对 AI 的更好引导和 AI 本身的改进，得出相反的结论

### 3. 关于生产力衡量

讨论中还涉及如何衡量 AI 开发的生产力：

- 追踪已发货功能的数量
- 衡量是否满足规范要求
- 常见的衡量指标：缺陷率、MTTR、新工程师入职时间、安全审计周转时间等

### 4. 与其他工具的比较

- 与 Superpowers 的对比
- 与 spec-kit 的对比
- 有人认为 Vanilla（默认设置）可能更好，因为预定义的技能框架可能会增加复杂性

## 结论

这场讨论反映了开发者对 AI Agent 开发工具的两种截然不同的态度：

1. **怀疑派**认为当前的 Agent Skills 框架过于复杂，实际效果有限
2. **实践派**认为这些工具确实提高了生产力，尤其是对有经验的开发者

讨论的共识似乎是：即使使用这些工具，人工审查仍然是必要的，但 AI 作为辅助工具可以显著提高开发效率。
