## 项目概述

**c89cc.sh** 是一个完全使用纯便携式 Shell 脚本（POSIX shell）编写的独立 **C89/ELF64 编译器**。该项目的独特之处在于：整个编译器仅由一个 shell 脚本构成，无需任何外部依赖即可将 C89 代码编译为 x86-64 Linux 可执行文件。

---

## 主要功能与特性

| 功能 | 说明 |
|------|------|
| **C89 解析器** | 支持完整的 C89 标准语法，包括类型说明符、表达式、控制流等 |
| **ELF64 二进制生成** | 输出 Linux x86-64 可执行文件（ELF64 格式） |
| **内置 Mini-libc** | 包含一个极简的 C 运行时库（基于系统调用） |
| **零外部依赖** | 编译器本身不调用任何外部命令，PATH 被清空 |

### 使用方法

```bash
sh c89cc.sh < prog.c > a.out
sh c89cc.sh --no-libc < prog.c > a.out (跳过内置 libc)
```

- **编程语言**：C89（ANSI C）
- **输出格式**：ELF64（x86-64 Linux 可执行文件）
- **实现语言**：纯 POSIX Shell 脚本（兼容 sh、bash、zsh 等）
- **许可证**：ISC License（开源许可证）

---

## 关键亮点与成就

1. **极致轻量**：整个编译器是一个单一脚本文件，实现了完整的 C89 编译器功能
2. **无依赖设计**：编译器不依赖任何外部工具，通过清空 PATH 并内置所有必需功能实现
3. **自包含运行时**：内置 mini-libc 提供基本的 C 运行时支持
4. **技术演示意义**：展示了用 Shell 脚本构建完整编译器系统的可能性

---

## 使用限制说明

根据文件注释，该 mini-libc 非常精简：
- 不包含 `stdio`（因此不支持 `printf` 等标准 IO 函数）
- 不支持预处理指令（如 `#include`、`#ifdef` 等）

如需使用完整标准库，用户需要结合其他 C 源文件一起编译。
