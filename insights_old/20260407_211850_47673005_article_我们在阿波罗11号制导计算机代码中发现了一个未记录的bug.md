## 核心发现

JUXT团队使用行为规范语言Allium从阿波罗制导计算机(AGC)的13万行汇编代码中提炼出1.25万行规范，成功发现了一个57年来未被发现的Bug：**陀螺仪控制代码中的资源锁(LGYRO)在错误路径上发生泄露**，悄然禁用了制导平台重新对齐的能力。

---

## 技术背景

### 阿波罗制导计算机(AGC)

- **历史地位**：历史上被审查最严格的代码库之一
- **代码规模**：74KB核心绳存储，2KB可擦除RAM，时钟频率1MHz
- **开发团队**：MIT仪器实验室，包括Margaret Hamilton等先驱
- **历史验证**：自2003年起源代码公开，志愿者手工转录，Virtual AGC项目通过仿真验证

### IMU子系统

阿波罗制导计算机通过惯性测量单元(IMU)管理陀螺仪平台，该平台负责确定航天器的朝向。AGC通过共享资源锁`LGYRO`管理IMU：当需要校正陀螺仪或执行星体对齐时，程序在开始时获取`LGYRO`，在所有三个轴校正完成后释放该锁。

---

## Bug分析

### 问题根源

当陀螺仪扭矩正常完成时，程序通过`STRTGYR2`退出，`LGYRO`锁被清除。然而，当IMU在扭矩进行中被"锁定"(caging)时，代码通过名为`BADEND`的例程退出，该例程**不清除`LGYRO`锁**。

缺失的两条指令：
```
CAF  ZERO
TS   LGYRO
```

只需4个字节。

### 影响后果

一旦`LGYRO`卡住，后续所有扭矩陀螺仪的尝试都会发现锁被持有，然后等待一个永远不会到来的唤醒信号并挂起。精细对齐、漂移补偿、手动陀螺仪扭矩：全部被阻塞。

### 潜在场景

在阿波罗11号任务中，Michael Collins在命令舱中独自绕月飞行，每两小时消失在月球背面并与地球失联。他定期运行程序52进行星体观测对齐，以保持制导平台指向正确方向。

如果他在扭矩过程中意外触发锁定开关：
1. 第一次失败看起来正常：锁定事件中断了对准，已知恢复方法
2. 第二次尝试挂起：无警报，无程序灯，DSKY接受输入但不执行任何操作
3. 训练有素的反应是解除锁定并重新对齐，但没有任何诊断线索指向问题所在

这看起来像是硬件故障，而非软件Bug。硬重启可以清除它，但在月球背面独自一人，面对一台接受命令但不执行任何操作的计算机，Collins必须自行做出决定。

## 发现方法

### Allium行为规范语言

团队使用Allium从IMU子系统提炼行为规范，该规范模拟每个共享资源的生命周期：获取时间、释放时间、释放路径。

关键规范规则：

```
规则 GyroTorque {
    发送陀螺仪扭矩脉冲命令。预留陀螺仪，启用电源，
    并按轴分发脉冲。
    
    要求：
        imu.mode != caged
        imu.gyros_busy = false
        
    确保：
        imu.gyros_busy = true
        GyroTorqueStarted()
}

规则 GyroTorqueBusy {
    陀螺仪已被另一个扭矩操作预留。
    调用者睡眠直到LGYRO被清除。
    
    要求：imu.gyros_busy = true
    
    确保：
        JobSleep(job: calling_job())
}
```

### 验证方式

- **测试验证代码按编写的方式运行**
- **行为规范询问代码的用途**

规范从另一个方向入手：从`LGYRO`开始，询问是否有任何路径未能清除它。这种方法迫使人们检查每一条路径，包括那些没有人想到要测试的路径。

## 软件工程启示

### 防御性编程的讽刺

AGC的编写非常防御性，以至于像这样的潜在故障会被重启逻辑悄悄纠正——重启作为 erasable-memory 初始化的副作用清除`LGYRO`。任何碰巧在Bug之后触发重启的测试都会看到系统无缝恢复。

防御性编码隐藏了问题，但并未消除问题。没有随后重启的锁定事件仍会使陀螺仪锁定。

### 现代语言的改进

现代语言尝试从根本上防止锁泄露：Go的`defer`、Java的`try-with-resources`、Python的`with`语句、Rust的所有权系统使锁泄露成为编译时错误。

然而，锁泄露仍然存在。MITRE将其分类为CWE-772："有效生命周期后未释放资源"，并将其利用可能性评为高。并非所有资源都由语言运行时管理：数据库连接、分布式锁、 shell脚本中的文件句柄、必须按正确顺序拆除的基础设施——这些仍然通常是程序员的责任。

---

## 结论

一个57岁的Bug隐藏在经过飞行验证的汇编代码中。所有阿波罗宇航员都安全返回，但IMU模式切换例程在多次任务中继续使用，从未被注意到，也从未被修复。
