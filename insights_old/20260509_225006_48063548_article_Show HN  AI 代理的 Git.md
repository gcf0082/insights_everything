## 项目概述

re_gent 是一个专为 AI 编码代理设计的版本控制系统。它能够跟踪 AI 代理的活动，记录哪个提示词修改了哪一行代码，并在出现问题时能够回滚。

## 核心功能

### 已完成功能
- **rgt init** - 在当前目录初始化 .regent/ 目录
- **rgt log** - 显示步骤历史（支持 --session, -n, --since 参数）
- **rgt sessions** - 列出所有活动会话
- **rgt status** - 显示当前仓库状态
- **rgt show <step>** - 显示完整上下文（工具调用 + 对话）
- **rgt blame <path>[:<line>]** - 显示文件的逐行来源
- **rgt cat <hash>** - 按哈希值检查任何对象（调试用）
- **rgt version** - 打印版本信息
- **rgt completion** - 生成 shell 补全脚本

### 即将推出功能
- **rgt rewind <step>** - 非破坏性时间旅行，回滚到之前的步骤
- **rgt gc** - 垃圾回收
- **rgt fork <step>** - 从某个步骤创建新会话

## 技术架构

### 数据存储结构
```
.regent/
├── objects/     # 内容寻址的 blob（BLAKE3 哈希）
├── refs/        # 会话指针（每个代理一个）
├── index.db     # SQLite 查询索引
└── config.toml
```

### 核心概念
- **Step（步骤）**：每次工具调用创建一个步骤，包含父步骤哈希、工作区快照、对话增量、工具信息和会话ID
- **DAG（有向无环图）**：步骤形成 DAG，每个会话有自己的分支，公共祖先去重
- **内容寻址存储**：使用 BLAKE3 哈希，自动去重

## 适用场景

re_gent 解决以下痛点：
- "五分钟前还能正常工作"
- "你为什么改那个文件？"
- "回到重构之前的状态"
- `/compact` 后丢失上下文
- 复制粘贴代码到新对话

- 需要审计 AI 代理活动的团队
