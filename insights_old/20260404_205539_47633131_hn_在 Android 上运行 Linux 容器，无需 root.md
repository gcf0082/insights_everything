## 项目概述

Podroid 是一个能够在 Android 设备上运行 Linux 容器的应用，无需 root 权限。该项目使用 QEMU TCG（软件模拟，非 KVM 硬件加速）来实现容器的隔离运行。

---

## 关键技术点

- **技术方案**：使用 QEMU TCG（libqemu-system-aarch64.so）进行软件模拟，而非硬件虚拟化
- **无需 root**：无需获取 root 权限即可运行
- **支持版本**：支持 Android 14、15 和 16
- **目标**：运行 OCI 标准的容器

---

## 社区讨论焦点

### 1. 与 Android 内置 Linux Terminal 的比较

- **Android 内置 Terminal**：使用硬件加速虚拟化，但仅在部分设备上可用（如 Google Pixel、某些三星 Exynos 设备）
- **限制**：Snapdragon 设备通常不支持非保护虚拟机，会报错"java.lang.UnsupportedOperationException: Non-protected VMs are not supported on this device"
- **Podroid 优势**：支持更广泛的设备，包括那些无法使用内置 Terminal 的手机

### 2. 稳定性问题

- 部分用户报告三星设备上频繁崩溃
- "recovery"功能会删除所有数据，需要从头开始
- 没有会话能持续超过 5 分钟

### 3. 使用场景

- 运行桌面环境
- 运行 Minecraft 服务器和客户端
- 运行 PiHole（使用旧手机）
- 开发/代码工作环境

### 4. 技术实现讨论

- Android 内核的相关参数被禁用，需要 root 才能启用
- PRoot（Termux）可以作为无 root 情况下的替代方案，但 systemd 无法运行
- QEMU TCG 方式有额外的模拟开销

### 5. 相关项目

- **Waydroid**：在 Linux 手机上运行 Android（最新稳定版约一个月前发布）
- **redroid**：在 Linux 上运行 Android 容器
- **Termux + PRoot**：用户空间解决方案

---

## 总结

Podroid 为那些无法使用 Android 内置 Linux Terminal 的用户提供了一个替代方案，尤其适用于较旧或不支持硬件虚拟化的 Android 设备。虽然使用 QEMU TCG 会带来一定的性能开销，但实现了无需 root 即可运行 Linux 容器的功能。社区对其稳定性和实际使用价值仍持观望态度。
