## 洞察摘要

用户报告Anthropic的Claude Code会拒绝请求或额外收取费用，如果用户的git commit中包含"OpenClaw"字符串。具体表现为：当用户在commit消息或代码中提及"OpenClaw"（一个开源的AI Agent库，与Anthropic存在竞争关系）时，Claude Code会立即断开连接并消耗用户的会话配额。

## 核心问题

1. **关键词检测触发**：Claude Code似乎在检测用户输入或commit消息中的特定字符串（如"Openclaw"），一旦检测到相关内容就会触发限制措施
2. **无明确说明**：服务条款中并未明确说明哪些词汇会导致额外收费或被拒绝服务
3. **误伤普通用户**：用户只是在合法的博客文章中提及OpenClaw，也会被模型拒绝或消耗配额
4. **可能的反竞争行为**：有用户认为这是Anthropic针对其竞争对手OpenClaw的反竞争行为

## 用户反馈的主要观点

- **负面评价**：许多用户批评Anthropic的这种做法是"欺诈"或"恶意行为"
- **支持正当限制**：部分用户认为Anthropic有权在其服务条款范围内做出限制
- **要求改进**：用户呼吁Anthropic明确服务限制条款，并提供更好的支持渠道
- **考虑竞争对手**：有用户认为DeepSeek等开源模型可能是更好的替代选择

- OpenClaw发布初期：Anthropic曾允许用户大量使用API，后来突然切断访问
- 近期：持续出现用户抱怨使用限制、AB测试等问题

## 结论

这一事件反映了AI模型提供商与开源竞争对手之间的复杂关系。Anthropic的这种做法引发了用户对服务透明度和公平性的质疑，同时也凸显了AI编程助手市场中竞争加剧的现状。用户在选择AI工具时需要考虑多方面因素，包括服务稳定性、价格透明度以及对竞争对手产品的限制政策。
