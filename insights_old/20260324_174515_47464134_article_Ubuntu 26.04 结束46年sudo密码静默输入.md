# 洞察报告：Ubuntu 26.04 结束 46 年静默 sudo 密码历史

**洞察链接**：https://pbxscience.com/ubuntu-26-04-ends-46-years-of-silent-sudo-passwords/

**发布日期**：2026年3月24日

**来源**：PBX Science

---

## 摘要

Ubuntu 26.04 LTS（代号"Resolute Raccoon"，计划于2026年4月23日发布）将改变sudo命令输入密码时的显示方式——从完全静默改为显示星号反馈。这一改变结束了长达46年的传统，引发了Linux社区的激烈争论。

## 背景

sudo命令最初由Bob Coggeshall和Cliff Spencer于1980年在纽约州立大学布法罗分校创建。静默密码输入是当时有意为之的安全决策，旨在防止"肩窥"——即旁观者通过计算按键次数来猜测密码长度。

## 变化详情

- **旧行为**：在终端输入sudo密码时，屏幕上完全不显示任何内容，只有一个闪烁的光标
- **新行为**：每个输入的字符显示一个星号（*）

## 技术原因

这一变化源于**sudo-rs**，这是经典C语言sudo的Rust重写版本。Canonical从Ubuntu 25.10开始默认使用sudo-rs。2026年2月，上游sudo-rs项目合并了默认启用`pwfeedback`选项的补丁，Canonical将其 cherry-pick 到 Ubuntu 26.04 开发版本中。

## 安全争议

### 批评观点
- 暴露密码长度，增加肩窥风险
- 违反Unix安全传统

### 支持观点
- 密码长度隐藏的安全收益极小——能看清星号的人也能直接看到按键
- 大多数用户的sudo密码与登录密码相同，而图形登录界面已显示占位符 dots
- 在终端隐藏星号而登录界面显示星号是"安全 theater"

## 时间线

| 年份 | 事件 |
|------|------|
| 1980 | sudo创建于SUNY Buffalo，静默输入成为默认设置 |
| 2025年10月 | Ubuntu 25.10用sudo-rs替换经典sudo，用户无感知 |
| 2025年10月 | 有人提交bug报告，请求默认启用pwfeedback |
| 2026年2月 | 上游sudo-rs合并pwfeedback补丁 |
| 2026年4月23日 | Ubuntu 26.04 LTS正式发布 |

## 如何恢复旧行为

用户可以通过以下步骤恢复传统静默提示：

```bash
sudo visudo
```

添加以下行：
```
Defaults !pwfeedback
```

更改立即生效，无需重启。

## 更广泛的背景

这一变化是Ubuntu 26.04更大现代化的一部分：
- 运行GNOME 50（仅Wayland）
- Linux内核7.0
- 采用更多Rust重写的核心工具（包括uutils/coreutils）

## 结论

无论你认为星号显示是迟来的用户体验改进还是危险的传统偏离，选项始终掌握在用户手中。开发者只是决定将默认设置偏向于帮助众多被空白提示困惑的新用户，而非珍视静默提示的少数老派用户。
