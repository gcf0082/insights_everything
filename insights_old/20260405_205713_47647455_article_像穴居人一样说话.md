## 项目概述

**caveman** 是一个 Claude Code 技能（skill），通过让 Claude 以"穴居人"的方式说话，实现**节省约 75% 的 token** 同时保持完整的技术准确性。

该项目基于一个病毒式传播的观察：使用简化的"穴居人"语言可以显著减少 LLM 的 token 使用量，同时不会丢失技术实质内容。

## 核心功能

### 1. Token 节省效果

| 指标 | 正常模式 | Caveman 模式 | 节省比例 |
|------|----------|--------------|----------|
| Token 数量 | 69 | 19 | ~75% |

## 技术实现原理

### Caveman 保留的内容

| 内容类型 | 处理方式 |
|----------|----------|
| 代码块 | 正常书写 |
| 技术术语 | 保持精确（如 polymorphism 保留） |
| 错误信息 | 精确引用 |
| Git 提交和 PR | 正常书写 |

### Caveman 移除的内容

| 内容类型 | 示例 |
|----------|------|
| 冠词 | a, an, the（全部删除） |
| 客套话 | "Sure, I'd be happy to help" |
| 冗余表达 | "The reason this is happening is because" |
| 模糊表达 | "It might be worth considering" |

### 浪费 token 的典型表达（被移除）

- "I'd be happy to help you with that" (8 tokens)
- "The reason this is happening is because" (7 tokens)
- "I would recommend that you consider" (7 tokens)
- "Sure, let me take a look at that for you" (10 tokens)

## 对比示例

### 示例 1：React 组件重新渲染问题

**正常 Claude（69 tokens）：**
> "The reason your React component is re-rendering is likely because you're creating a new object reference on each render cycle. When you pass an inline object as a prop, React's shallow comparison sees it as a different object every time, which triggers a re-render. I'd recommend using useMemo to memoize the object."

**Caveman Claude（19 tokens）：**
> "New object ref each render. Inline object prop = new ref = re-render. Wrap in `useMemo`."
