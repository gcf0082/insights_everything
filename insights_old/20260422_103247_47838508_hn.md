## 核心内容摘要

GitHub 宣布对其 Copilot 个人订阅计划进行重大变更，主要包括：

1. **模型可用性变化**
   - Opus 4.6 不再可用（此前为 3x 倍率）
   - Opus 4.7 现为 Pro+ 专属，使用 7.5x 倍率
   - Pro 计划将转向 Sonnet 4.7 作为新的 3x 模型

2. **计费模式变更**
   - 商业和企业计划也将改为按 token 计费
   - 7.5x 倍率为"促销价"（至4月底），之后可能进一步涨价

3. **价格调整**
   - Pro+ 从 $10/月涨至 $40/月，但权益反而减少

## 社区主要观点

### 负面反应

- **"Rug Pull"（突然撤资）**：大量用户批评这是对付费用户的背叛，没有事先通知
- **价值感受下降**：许多人认为微软作为中间商几乎不提供价值，99%的价值来自 Anthropic 或 OpenAI
- **不满原因**：
  - 有人预先支付了年度订阅却突然变更
  - Opus 4.6 是许多人订阅的主要原因之一
  - 缺乏透明度和沟通

### 为何仍使用 Copilot

- 可直接计入现有 Azure 订阅，避免新供应商审批流程
- 企业已有协议保障 Microsoft 不会用其数据训练模型
- 统一接口，可轻松切换不同模型

### 替代方案讨论

- **直接使用 Claude Code**：但 Anthropic 也被曝将从 Pro 计划移除 Claude Code
- **本地模型**：如 Qwen 3.6
- **其他工具**：OpenCode、Cursor、OpenRouter
- **GLM 5.1**：被提及为性价比高的选择

### 争议性观点

部分用户认为大多数情况下无需使用 Opus：
- Haiku 足以应对 95% 的编码任务
- Sonnet 适合其余 5%
- 用户应学会根据任务选择合适的模型

但这引发争议，有人反驳这取决于代码库复杂度和任务性质。

## 行业影响

1. **AI 编程工具成本上升**：这可能是行业趋势的开始
2. **VC 补贴期结束**：LLM 推理成本高昂，补贴不可持续
3. **用户建议**：
   - 用 AI 构建工具自己使用
   - 不要完全依赖第三方 AI 服务，需确保可替换后端

---

## 总结

此次变更是 AI 编程工具领域的重要转折点，反映了：
- AI 推理成本的真实压力
- 订阅制定价模式向按 token 计费的转变
- 市场竞争格局的变化

用户普遍对这种"先低价吸引、后涨价"的做法感到不满，但行业可能正进入一个新的调整期。
