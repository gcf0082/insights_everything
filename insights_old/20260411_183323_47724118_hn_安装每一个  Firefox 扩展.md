## 内容摘要

这是一篇关于作者尝试安装 Firefox 商店中所有扩展程序（共计 84,194 个）的博文。这种极端实验揭示了浏览器的许多有趣问题。

---

## 主要发现

### 1. 性能问题
- **扩展配置文件问题**：13 年前使用的是 `extensions.sqlite`，如今改为 `extensions.json`
- **写入机制**：每次写入时完整序列化并重写整个文件，防抖间隔为 20ms
- **影响**：15 个扩展时运行良好，但 84,194 个扩展时导致严重性能问题

### 2. 用户界面混乱
- 作者发现可点击的"扩展"按钮实际上是扩展程序提供的假按钮，共发现三个假按钮
- 视频中包含癫痫警告提示，视觉效果极具冲击力
- 伴随金属管音效（由 "Random Metal Pipe" 扩展提供）

### 3. 安全发现
- **钓鱼扩展 "Іron Wаllеt"**：使用西里尔字母伪装，安装 3 秒后从 NocoDB 电子表格获取钓鱼页面 URL
- 作者发现该 API 密钥具有写权限，于是删除了整个电子表格
- 该扩展目前仍存在于 Mozilla 附加组件商店中

### 4. 开发者观察
- **Dr. B**：发布商，拥有 84 个扩展，全部使用" vibe coding"方式编写，使用 Grok 3
- 这些扩展均无图标或截图
- Firefox 附加组件审查可能存在自动化过滤

## 社区评论亮点

### 技术讨论
- 有评论指出 Firefox 的 JSON 存储容易损坏，用户曾遇到标签页丢失问题
- 建议回归 SQLite 存储
- Chrome 和 Edge 也有类似数量的扩展

### 历史对比
- 让网友回忆起 Internet Explorer 时代被工具栏占据 80% 屏幕空间的经历
- 类似当年 Ask.com、Google、Yahoo、Bing 等工具栏之争
- 有人用"Let's Game It Out"风格来形容这次实验

### 趣味观察
- 获得"中指表情贴纸奖"
- 有人建议这适合作为恶作剧（针对未锁定的电脑）

## 结论

这个极端实验展示了大量浏览器扩展同时运行时的各种问题，包括性能下降、UI 混乱、以及潜在的安全风险。社区普遍认为这是一次有趣且有洞察力的尝试。
