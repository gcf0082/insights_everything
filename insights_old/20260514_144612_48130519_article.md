## 核心发现

安全研究员 **Chaotic Eclipse**（又名 Nightmare-Eclipse）发布了两个新的零日漏洞：**YellowKey** 和 **GreenPlasma**。这已是该研究员第二次公开微软零日漏洞——上月发布的 BlueHammer 和 RedSun 漏洞同样获得了 Windows 系统管理员权限。

## YellowKey 漏洞详情

### 攻击原理

YellowKey 漏洞允许攻击者仅通过 U 盘上的几个文件即可解锁 BitLocker 加密的驱动器：

1. 获取 U 盘的写入权限，访问 "System Volume Information" 目录
2. 复制 "FsTx" 文件夹及其内容到该目录
3. 按住 Shift + 点击"重启"进入 Windows 恢复环境
4. 按住 Ctrl 键不放，机器将重启
5. 系统将直接进入提权的命令行界面，完全访问已加密的驱动器，**无需输入任何密钥**

### 关键特征

- **操作极其简单**：不需要复杂的工具或技术知识
- **自动销毁痕迹**：漏洞文件在使用后会自动从 U 盘消失
- **影响范围广**：影响 Windows Server 2022/2025 和 Windows 11（**Windows 10 不受影响**）
- **TPM+PIN 无法防御**：即使使用完整的 TPM 和 PIN 配置也无法阻止该漏洞（研究人员表示有对应变种但未公开）

### 安全风险评估

- BitLocker 作为保护全球数百万家庭、企业和政府机构设备的加密方案，其可信度受到严重挑战
- 攻击者可轻易盗取笔记本电脑或台式机，无需复杂操作即可获取数据

## GreenPlasma 漏洞详情

GreenPlasma 是一个**本地权限提升**漏洞，允许普通用户获得系统级（比管理员更高）的访问权限：

- 通过操纵 CTFMon 进程，将精心构造的内存段对象放入 Windows Object Manager 部分
- 绕过常规访问控制，访问不应访问的内存区域
- 在服务器环境中风险尤其严重：任何普通用户都可获得服务器控制权

## 事件背景

Chaotic Eclipse 表示，其此前向微软安全团队报告 BlueHammer 和 RedSun 漏洞后，报告被忽视和驳回，因此公开了这些漏洞。研究人员强调：

> "这个漏洞隐藏得很好，我本可以靠它赚一大笔钱，但任何金钱都无法阻挡我对抗微软的决心。"

目前，BlueHammer 已获修复，但微软尚未对 YellowKey 和 GreenPlasma 做出官方回应。

---

## 总结与建议

YellowKey 漏洞的发现表明：

1. **BitLocker 的信任度存疑**：作为 Windows 默认加密方案，BitLocker 可能存在隐藏的后门或严重漏洞
2. **物理访问风险加剧**：结合 GreenPlasma 漏洞，攻击者可在短时间内获得系统完全控制权
3. **需要额外防护措施**：依赖 BitLocker 的用户应考虑结合第三方加密工具或硬件级安全方案

截至发稿时，微软尚未就此漏洞发表官方声明。
