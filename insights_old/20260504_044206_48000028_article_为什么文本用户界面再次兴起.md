## 核心摘要

终端用户界面（TUIs）正在卷土重来。文章分析了为什么在 GUI 应用频繁失败的背景下，TUI 成为开发者和用户的新选择。

## 关键内容

### 1. TUI 回归的背景

- **DHH 的 Omarchy 案例**: 该项目采用三种界面类型：TUI（提供即时反馈）、Web 应用和原生应用。TUI 成为首选的交互方式。
- **代码编辑器的发展历程**: 约 10 年前，从 Textmate、Notepad++、Sublime 等原生编辑器转向 Electron 驱动的应用（如 Atom、VSCode），而硬核用户转向 vim 或 emacs。

### 2. 各平台的 GUI 问题

#### Windows
- 微软不断更换 GUI 框架：MFC → Winforms → WPF → Silverlight → WinUIs → MAUI，但始终没有统一的策略。
- Jeffrey Snover 指出：微软每几年就重造一次 OS 和 UI API，导致每次新框架都有功能缺失。

#### Linux
- GTK 和 Qt 两大框架并存，但跨平台支持不佳。
- 很少有公司愿意为 Linux 开发原生应用，多数使用 Electron 或依赖开源社区。

#### macOS
- Apple 曾是界面设计的标杆，但近年来违背了自己的 Human Interface Guidelines：
  - 忽略 Fitts' law
  - 使调整窗口大小变得困难
  - 在菜单中添加图标

#### Electron 应用
- 内存消耗虽有所改善，但仍存在缺乏视觉一致性和键盘驱动工作流的问题。
- 作者的 MacBook Pro 上有 8 个原生应用和 6 个 Electron 应用（Slack、Discord、VSCode、Cursor 等）。

### 3. TUI 的优势

- **快速**: 性能优于 GUI 应用
- **易于自动化**: 可以通过脚本批量处理任务
- **跨平台兼容性好**: 可以在不同操作系统间无缝运行
- **远程友好**: 支持远程运行，无需复杂的 X forwarding
- **一致性**: 提供统一的交互体验，减少学习成本

### 4. 实际应用案例

- **Claude 和 Codex 在命令行上的成功**: 开发者可以专注于交互本身，忽略底层操作系统差异。
- **远程开发**: 可以从 iPad 远程控制 GPU 机器进行开发。

### 5. 未来展望

文章引用 John Loeber 的观点，强调**一致性界面**的重要性：
- 界面越好，需要的思考越少
- 用户期望一致的快捷键（如 Command+C 在所有应用中通用）
- 应该回归基础，重视用户体验设计理论

## 结论

当原生 GUI 框架不断失败、用户体验持续下降时，TUI 提供了一个回归本质的解决方案。它不仅性能优越，还能提供跨平台的一致性体验。文章呼吁开发者重视 UI/UX 设计理论，将用户体验纳入软件工程教育的核心。
