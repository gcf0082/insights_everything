## 漏洞概述

Project Zero团队发布了针对Google Pixel 10的完整零点击(0-click)漏洞利用链。该利用链展示了如何从无需用户交互的上下文逐步获得Android root权限。

## 技术细节

### 1. 漏洞链结构

该利用链包含两个主要环节：

- **Dolby UDC漏洞** (CVE-2025-54957)：影响所有Android设备的零点击漏洞，曾用于Pixel 9的利用链
- **VPU驱动漏洞**：Pixel 10独有的本地权限提升漏洞

### 2. 关键漏洞分析

**VPU驱动中的mmap处理函数存在严重安全问题：**

```c
static int vpu_mmap(struct file *fp, struct vm_area_struct *vm)
{
    unsigned long pfn;
    struct vpu_core *core = container_of(fp->f_inode->i_cdev, struct vpu_core, cdev);
    vm_flags_set(vm, VM_IO | VM_DONTEXPAND | VM_DONTDUMP);
    vm->vm_page_prot = pgprot_device(vm->vm_page_prot);
    pfn = core->paddr >> PAGE_SHIFT;
    return remap_pfn_range(vm, vm->vm_start, pfn, vm->vm_end-vm->vm_start, vm->vm_page_prot) ? -EAGAIN : 0;
}
```

**问题**：该mmap处理器根据VMA大小而非寄存器区域的实际大小来映射物理内存。攻击者可以通过指定大于实际寄存器区域的大小，映射任意范围的物理内存，包括整个内核镜像。

### 3. 利用难度

- 编写完整的内核任意读写漏洞仅需**5行代码**
- 完成整个漏洞利用开发**不到一天**
- 由于Pixel内核始终位于相同的物理地址，无需扫描即可定位内核位置

### 4. 修复时间线

- **漏洞报告日期**: 2025年11月24日
- **严重性评级**: 高危(High)
- **修复周期**: 71天
- **补丁发布**: 2026年2月Pixel安全公告

值得注意的是，这是Project Zero报告的Android驱动漏洞首次在90天内获得修复。

## 社区讨论要点

### 1. AI功能增加攻击面

用户指出，近年来手机AI功能允许用户更好地搜索和理解消息，这种变化导致了零点击攻击面的增加，因为高效的分析通常需要在用户打开消息之前对消息媒体进行解码。

### 2. 漏洞发现效率

有用户提到，AI可以帮助研究人员更好地浏览代码库，但真正的漏洞发现仍需人类专家的深度理解。有用户报告称使用GPT-5.5能够识别出代码中的安全漏洞。

### 3. CVE数量增长

数据显示2024年每天约发布100个CVE，而2024年4月已达到每天约200个。增长原因包括：
- CVE报告规则变化
- Linux内核现在几乎所有bug都提交为CVE
- AI工具增加了漏洞发现数量

### 4. Android更新问题

社区讨论指出：
- 42%的Android设备尚未打补丁
- 品牌Android设备通常可以获得OS安全更新，但驱动和固件更新不确定
- 小品牌手机通常从不更新

### 5. 安全责任讨论

有用户提出应加强对代码编写者的责任，但多数人认为这不现实，可能导致无人愿意编写代码。

## 结论与建议

Project Zero的研究表明：
1. Android在漏洞修复流程上有明显进步
2. 驱动安全问题仍然普遍存在，代码审计需加强
3. AI功能虽然便利，但显著增加了攻击面
4. 厂商需要采取更主动的安全开发实践

该研究强调软件供应商应确保产品在发布时处于合理的无漏洞状态，团队应采取积极的软件安全、代码审计和漏洞修复方法。
