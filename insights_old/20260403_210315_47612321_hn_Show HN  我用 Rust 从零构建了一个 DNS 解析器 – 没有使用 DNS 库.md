## 项目概述

该项目名为 **numa**，是一个从零开始用 Rust 编写的 DNS 解析器，完全不使用任何 DNS 库。作者开发这个项目的初衷是需要将 `https://frontend.numa` 作为主要 DNS 解析地址，而不是使用 `localhost:5173`。

## 核心功能

1. **本地 .numa 域名支持**
   - 可使用 `https://xxx.numa` 替代 `localhost:端口`
   - 自动生成 TLS 证书
   - WebSocket 透传
   - 路径路由

2. **递归解析**
   - 从根域名服务器开始进行递归解析
   - DNSSEC 链式信任验证

3. **广告拦截**
   - 内置 385K+ 个广告域名黑名单
   - 可配置多个阻断列表
   - 支持 StevenBlack 等第三方列表

4. **局域网服务发现**
   - 自动发现运行 numa 的同网络设备
   - 支持静态记录配置

5. **Split DNS**
   - 自动检测 Tailscale 转发规则
   - 查询匹配 `.ts.net` 时自动转发到 Tailscale DNS

### 功能咨询
1. **内存使用**: 作者回复表示 390K 阻断域名时总进程内存约 31MB
2. **Docker 支持**: 可以通过设置容器 DNS 指向主机实现，但反向代理功能还在开发中
3. **自动 TLS**: 运行时生成本地 CA 并存储到系统信任存储，为每个服务生成由该 CA 签名的 TLS 证书
4. **多平台支持**: 支持 Linux 和 macOS，FreeBSD 可以通过 PR 实现
5. **多阻断列表**: 已支持在 numa.toml 中配置多个列表
