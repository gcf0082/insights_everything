## 项目概述

Tilde.run 是一个 Agent 沙箱工具，具有事务性版本文件系统。它基于 lakeFS 构建，提供可扩展的版本化、组合式文件系统解决方案。

## 核心特性

1. **事务性版本文件系统** - 基于 lakeFS 实现，支持大规模数据版本管理
2. **Fuse 挂载** - 文件系统通过 Fuse 挂载到沙箱，无需复制数据
3. **原子提交** - 每次沙箱运行产生新的原子提交到隐藏的主分支
4. **乐观并发** - 使用 lakeFS 检测冲突，支持多人同时操作
5. **Human-in-the-loop** - 支持人工审批流程，类似 Pull Request

## 技术架构

- **底层技术**: lakeFS (开源)
- **存储后端**: S3
- **隔离机制**: 沙箱自动分支挂载
- **当前配置**: 2 核 CPU、4GB RAM（暂不支持 GPU）

## 讨论焦点

### 1. 产品定位争议

有用户认为这类工具过于同质化，缺乏明确差异点。文件版本化功能与 Git、S3 版本控制存在功能重叠。

### 2. 演示效果

用户反馈演示视频缺乏吸引力，80% 内容是基础设置工作。

### 3. 扩展性问题

关于是否支持多分支并行和合并策略存在讨论。lakeFS 以单个文件为单位处理冲突。

### 4. 实际应用场景

- 制药 CAD 设计中的大型文件格式管理
- 蛋白质结构、化合物等科学数据版本控制
- Agent 生成的 Python 脚本缓存和复用

### 5. 替代方案

- btrfs 子卷快照
- S3 Files (AWS)
- InstaVM
- Docker + S3 自建方案

### 6. 网络安全

沙箱网络访问控制和入口流量安全是重要讨论点。

## 结论

Tilde.run 为 AI Agent 提供了可靠的持久化存储和版本控制方案，特别适用于需要管理大量非代码数据（视频、图像、大型数据集）的场景。但其核心功能与现有版本控制工具存在一定重叠，定价策略和大规模并发支持仍是待解决的问题。
