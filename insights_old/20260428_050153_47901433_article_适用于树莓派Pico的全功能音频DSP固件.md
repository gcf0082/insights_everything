## 项目简介

DSPi 是一款功能完整的数字音频处理固件，能够将 Raspberry Pi Pico 或其他基于 RP2040 的开发板转变成一个功能强大且经济实惠的微型数字音频处理器。它作为一个 USB 声卡使用，配备内置 DSP 引擎，可以让用户使用如房间校正、有源分频、参量均衡、时间对齐、响度补偿和耳机交叉馈送等实用工具。

作者希望 RP2040 和 RP2350 能被誉为"比一杯咖啡还便宜的音频瑞士军刀"。

## 核心特性

### 音频接口
- **USB 音频接口**：支持 macOS、Windows、Linux 和 iOS 即插即用。支持 44.1、48 和 96 kHz 的 16 位和 24 位 PCM 输入
- **24 位 S/PDIF 或 I2S 输出**：最多 4 个独立立体声输出插槽（RP2350 上 8 通道，RP2040 上 4 通道），每个插槽可在运行时切换 S/PDIF 和 I2S

### DSP 功能
- **矩阵混音器**：2x9（RP2350）或 2x5（RP2040）路由矩阵，每个交叉点独立增益和反相
- **参量均衡**：每通道最多 10 个 PEQ 频段，支持 6 种滤波器类型。RP2350 总共 110 个滤波频段，RP2040 70 个
- **响度补偿**：基于 ISO 226:2003 等响曲线标准的音量依赖型 EQ
- **耳机交叉馈送**：基于 BS2B 的交叉馈送，带有耳间时间延迟 (ITD)
- **音量均衡器**：基于 RMS 的立体声联动软拐点向上压缩器
- **主音量**：设备端输出上限（-128 至 0 dB）
- **时间对齐**：每输出延迟（最长 85ms）
- **低音炮输出**：专用单声道 PDM 输出

### 系统架构
- **双核 DSP**：双核处理确保多输出时最大吞吐量
- **可配置输出引脚**：所有输出 GPIO 引脚可在运行时重新分配
- **10 槽位预设系统**：保存、加载和管理最多 10 个完整 DSP 配置
- **诊断功能**：每通道峰值/削波监控，USB PHY 错误计数器

## 音频信号链路

### RP2350 (11 通道, 9 输出)
```
USB 输入 (16/24位 PCM 立体声, 44.1/48/96 kHz)
    |
PASS 1: 每通道前置放大器 + USB 音量
    |
PASS 2: 主 EQ (每通道 10 频段)
    |
PASS 2.5: 音量均衡器 (RMS 向上压缩)
    |
PASS 3: 耳机交叉馈送 (BS2B + ITD) + 主峰值表
    |
        响度补偿 (音量依赖型 EQ)
    |
PASS 4: 矩阵混音器 (2 输入 x 9 输出)
    |
PASS 5: 每输出 EQ -> 增益 -> 静音 -> 延迟 -> 输出增益 x 主音量
    |
    +-- 输出 1-2 --> S/PDIF 或 I2S 插槽 0
    +-- 输出 3-4 --> S/PDIF 或 I2S 插槽 1
    +-- 输出 5-6 --> S/PDIF 或 I2S 插槽 2
    +-- 输出 7-8 --> S/PDIF 或 I2S 插槽 3
    +-- 输出 9   --> PDM 低音炮
```

## 开发者参考

### USB 控制协议
配置通过接口 2（Vendor 接口）使用控制传输完成。

主要请求代码：
- `0x42` - 设置 EQ 参数
- `0x44/0x45` - 设置/获取前置放大器
- `0x48/0x49` - 设置/获取延迟
- `0x50` - 获取系统状态
- `0x51-0x53` - 保存/加载/恢复出厂设置
- `0x70/0x71` - 设置/获取矩阵路由
- `0x90-0x9A` - 预设管理命令
- `0xA0/0xA1` - 批量读取/写入全部参数

### 系统架构
- **Core 0**：USB 通信、音频流、DSP 处理（主 EQ、交叉馈送、响度、矩阵混音、输出 EQ 第一对）
- **Core 1**（三种模式）：
  - **PDM 模式**：低音炮输出的 delta-sigma 调制器
  - **EQ 工作模式**：处理较高编号输出的输出 EQ、延迟和 S/PDIF 转换
  - **空闲模式**：无需 Core 1 处理时

## 总结

DSPi 是一个功能完整、性能强大的开源数字音频处理固件，专为 Raspberry Pi Pico 和 Pico 2 设计。它将廉价的开发板转变为专业的音频处理器，支持USB音频输入、多路输出、参量均衡、矩阵混音、响度补偿、耳机交叉馈送等高级功能。两个平台都已生产就绪，RP2350 凭借硬件 FPU 提供更多处理余量和更多输出通道。
