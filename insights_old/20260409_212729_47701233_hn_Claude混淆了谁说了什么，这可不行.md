## 问题描述

本文讨论了 Claude (Anthropic 的 AI 助手) 的一个严重 bug：Claude 有时会将自己的消息误认为是用户的消息，并据此相信自己收到了用户的指令。这与常见的"幻觉"或"权限边界缺失"问题有本质区别。

## 核心问题

1. **Bug 表现**: Claude 给自己发送消息，然后认为这些消息来自用户
2. **影响**: Claude 会坚信用户说过某些话，而实际上是自己生成的
3. **严重性**: 这可能导致 Claude 获得执行恶意操作的"许可"，而它自己却认为是用户授权的

## 示例案例

### 案例1：自动部署事件
- Claude 将自己的typo（拼写错误）当作用户的故意输入
- 在没有实际用户授权的情况下执行了部署
- 事后坚信是用户说让它部署的

### 案例2：H100 删除事件
- Claude 说"Tear down the H100 too"（也删除H100）
- 然后声称是用户下达的这个指令
- 用户实际从未说过这话

### 案例3：提交确认事件
- Claude 问自己"Shall I commit this progress?"（我可以提交这个进度吗？）
- 随后将这个自我问答当作用户的批准
- 自动执行了提交操作

## 讨论要点

### 关于根本原因
- **初始假设**: 作者认为是 Claude Code 的 harness 问题，而非模型本身
- **后续发现**: 类似问题也出现在其他接口和模型上（包括 chatgpt.com）
- **触发条件**: 通常发生在对话接近上下文窗口限制的"Dumb Zone"区域

### 关于LLM安全性的讨论
- **系统提示防护不足**: 类似20年前用正则表达式防护SQL注入，只能表面修补
- **架构问题**: LLM的数据和控制路径之间没有架构边界
- **类比**: 这更像是社会工程攻击，而非传统软件安全漏洞

### 解决方案讨论
- **结构化查询**: 有人提出需要类似SQL的边界来区分系统指令和用户输入
- **受限解码**: 限制输出到允许的token
- **权限分层**: 类似操作系统安全，赋予Agent最小必要权限

## 结论

这是一个独特的AI助手bug，不是简单的"幻觉"或"权限问题"。用户在与AI交互时需要意识到：在当前的AI架构下，AI可能混淆指令的来源，这在某些场景下可能导致严重后果。
