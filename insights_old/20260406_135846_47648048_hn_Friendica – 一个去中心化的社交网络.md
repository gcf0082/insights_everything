## 摘要

这篇文章介绍了 Friendica——一个去中心化社交网络。有用户指出这是最早的去中心化社交平台之一，甚至在"fediverse"这个术语出现之前就存在了（与 Gnu Social 和 status.net 同期）。

## 热门评论摘要

### 关于自托管社交网络

有人询问为家人和朋友自托管一个小规模社交网络的建议，以摆脱 Facebook/Instagram。

**推荐方案：**
- **Pleroma**: 轻量级，支持 ActivityPub，可使用 Mastodon 客户端
- **GoToSocial**: 单二进制文件，易于部署
- **snac**: 代码库简洁
- **Takahē** 和 **bovine**: 其他小型服务器选项
- **Mattermost**: 类似 Slack 的体验
- **BuddyPress**: 基于 WordPress 的自托管方案
- **Nostr + Primal**: 可建立私有的 Nostr relay 只供家人使用
- **atproto/Bluesky**: 可建立独立的 PDS 服务

**不建议的方案：**
- Pleroma 偏 Twitter 风格，不太适合紧密的社区
- atproto 默认公开隐私保护不足

### 关于去中心化社交网络的挑战

1. **用户友好性**: 联邦网络需要技术知识，99% 的普通用户无法驾驭
2. **文档问题**: Friendica 文档不够完善
3. **社区规模**: 小型社交网络才能保持健康，过度开放会退化成像 Facebook 一样的平台
4. **集成性缺失**: 缺少一个集博客、评论、分布式身份、聊天于一体的完整应用

### Friendica 本身

- 支持 PHP + MySQL，可在便宜共享主机上运行
- 无需 Docker 或 node_modules
- 界面设计被认为不够吸引人，首页缺乏优化
- 社区活跃度不够高
- 与 diaspora 类似，存在"领地"和可移植性问题

### 关于"军事级加密"

有评论指出 Friendica 网站上声称的"军事级加密"含义不明，缺乏详细解释。
