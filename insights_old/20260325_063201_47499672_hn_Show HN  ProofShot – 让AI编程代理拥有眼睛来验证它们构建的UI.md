## 项目概述

ProofShot 是一个 CLI 工具，旨在解决 AI 编码 agent 无法直观看到其生成的 UI 在浏览器中实际运行效果的问题。该工具能够：

- 启动浏览器并与页面交互
- 记录整个操作过程的视频和截图
- 收集控制台日志和错误信息
- 将所有内容打包成独立的 HTML 文件供开发者审查

**使用方式:**
```bash
proofshot start --run "npm run dev" --port 3000
# agent 进行导航、点击、截图等操作
proofshot stop
```

**技术特点:**
- 基于 Vercel Labs 的 agent-browser（比 Playwright MCP 更快）
- 支持任意 AI 编码 agent（Claude Code、Cursor、Codex 等）
- 打包为 skill 格式，便于 AI agent 理解使用方式
- 开源且完全免费

**官网:** https://proofshot.argil.io/

## 主要观点

### 正面评价

1. **解决真实痛点:** 工具填补了 AI 编码 agent 视觉反馈的关键空白，agent 可以生成代码但无法验证最终呈现效果

2. **PR 审查价值:** 作为人类代码审查者，截图非常有帮助；可自动为 PR 添加可视化证据

3. **桌面应用支持:** 对于无 DOM 的桌面应用（如绘图软件），截图差异检测是唯一可行的验证方式

4. **与现有工具互补:** 某些用户指出 Playwright 检查结构属性，而 ProofShot 捕获视觉问题，两者可结合使用

### 质疑与比较

1. **与 Playwright 对比:** 部分用户认为 Playwright 本身已具备截图和视频录制功能，ProofShot 的必要性存疑

2. **与 Claude Code/Cursor 内置功能重叠:** 许多用户指出 Claude Code 和 Cursor 已有内置浏览器控制能力

3. **对比 agent-browser:** 有用户询问 ProofShot 与原生 agent-browser 的区别

---

## 技术讨论

1. **视觉测试 vs 结构测试:** Playwright 验证 DOM 结构（元素存在性、可见性等），而视觉测试捕获完全不同类型的错误（颜色错误、布局偏移、间距问题、组件重叠）

2. **截图差异检测的历史:** Mozilla 20 多年前就使用截图回归测试 Gecko 布局，现在可结合视觉模型进行语义分析

3. **移动端支持:** 有用户询问是否支持 iOS 模拟器或 Android 模拟器，Xcode Simulator 有相关支持
