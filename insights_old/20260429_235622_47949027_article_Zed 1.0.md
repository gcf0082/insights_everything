# 洞察报告：Zed 1.0 发布

## 基本信息

- **洞察链接**: https://zed.dev/blog/zed-1-0
- **发布时间**: 2026年4月29日
- **作者**: Nathan Sobo
- **文章标题**: Zed is 1.0

## 文章摘要

本文标志着 Zed 编辑器正式发布 1.0 版本。作者回顾了 Zed 的开发历程，阐述了他们为何选择从零开始构建一个完全不同的桌面编辑器架构，而非继续基于 Web 技术（Electron）进行开发。

## 核心要点

### 1. 技术架构创新

Zed 的开发团队之前创建了 Atom 编辑器，该编辑器基于 Chromium 构建，并催生了 Electron 框架。Electron 最终成为 VS Code 的基础。然而，Web 技术虽然提供了便捷的开发路径，但也带来了性能上限。为此，Zed 团队决定采用类似视频游戏的方式重新构建编辑器，将整个应用围绕 GPU 上运行的着色器数据传输进行组织，并使用 Rust 从头编写了自己的 UI 框架 GPUI。

### 2. 1.0 版本的意义

经过五年的开发，Zed 已经支持：
- 数十种编程语言及其生态系统
- Git 集成
- SSH 远程开发
- 调试器
- 彩虹括号等功能

代码量已超过百万行，支持 Mac、Windows 和 Linux 三大平台。

### 3. AI 原生编辑器

Zed 是一个 AI 原生编辑器，具备以下能力：
- 并行运行多个 AI Agent
- 编辑预测功能，以按键粒度提供下一个修改建议
- 支持 Agent Client 协议，可对接 Claude Agent、Codex、OpenCode、Cursor 等多种 AI Agent

### 4. Zed for Business

即将推出企业版，支持：
- 集中计费
- 基于角色的访问控制
- 团队管理功能

### 5. 未来展望

团队正在开发 DeltaDB，这是一个基于 CRDT 的同步引擎，支持字符级精度的变更追踪。DeltaDB 让多个人类和 AI Agent 能够共享同一个一致的代码库视图，支持邀请团队成员进入与 AI Agent 的对话中，直接在生成代码的上下文中进行代码审查和演进。

## 关键结论

1.0 版本不是一个"完成"或"完美"的标志，而是一个临界点，大多数开发者可以快速适应。Zed 仍然会保持每周发布的节奏，持续改进。
