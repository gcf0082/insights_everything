## 概述

Posit很高兴宣布推出ggsql(alpha版本)，这是一个基于SQL语法的图形语法实现，将丰富的结构化可视化支持带入SQL。ggsql可在Quarto、Jupyter notebooks、Positron和VS Code等环境中使用。

## ggsql语法核心概念

### 1. VISUALIZE子句

用于定义视觉映射，指定数据列与视觉属性的对应关系：

```sql
VISUALIZE bill_len AS x, bill_dep AS y FROM ggsql:penguins
```

### 2. DRAW子句

用于添加可视化图层layer，可叠加多个图层：

```sql
DRAW point
DRAW smooth
```

### 3. PLACE子句

用于添加注释，不从表格获取数据，而是使用提供的字面值：

```sql
PLACE rule
PLACE text
```

### 4. SCALE子句

控制数据值如何转换为视觉美学值：

```sql
SCALE fill TO accent
```

### 5. LABEL子句

用于添加或修改各种文本标签，如标题、副标题、坐标轴和图例标题：

```sql
LABEL
  title => '图表标题',
  subtitle => '副标题'
```

## 开发动机

### 1. 服务SQL用户

许多数据分析师和数据科学家主要使用SQL工作，现有的可视化选择往往不理想：
- 导出数据使用R或Python
- 使用缺乏可重复性支持的GUI BI工具
- 使用功能不够强大的SQL可视化工具

ggsql的设计语法对SQL用户来说非常直观，利用其对可组合声明式子句的期望。

### 2. SQL与图形语法的完美结合

SQL和图形语法都基于声明式、可组合操作的语义，两者的结合可以为从原始数据到最终可视化的完整流程提供非常强大和自然的解决方案。

### 3. 无需运行时

与ggplot2和plotnine需要R和Python不同，ggsql的优势包括：
- 更容易嵌入其他工具
- 更易于沙箱化以防止恶意代码执行
- 可以作为单个SQL查询执行整个数据管道，无需先拉取全部数据

### 4. LLM友好

LLM已被证明能非常有效地将自然语言翻译成SQL，ggsql同样可以：
- 更安全、更轻量的运行时
- 可更自信地将编码代理部署到生产环境

### 5. 18年ggplot2经验的积累

18年的ggplot2开发经验带来了丰富的专业知识，ggsql是一块白板，可以应用这些经验到一个没有现有期望的环境中。

## 完整示例

```sql
WITH astronauts AS (
  SELECT * FROM 'astronauts.parquet'
  QUALIFY ROW_NUMBER() OVER (
    PARTITION BY name
    ORDER BY mission_number DESC
  ) = 1
)
SELECT
  *,
  year_of_selection - year_of_birth AS age,
  'Age at selection' AS category
FROM astronauts
UNION ALL
SELECT
  *,
  year_of_mission - year_of_birth AS age,
  'Age at mission' AS category
FROM astronauts

VISUALIZE age AS x, category AS fill
DRAW histogram
  SETTING binwidth => 1, position => 'identity'
PLACE rule
  SETTING x => (34, 44), linetype => 'dotted'
PLACE text
  SETTING
    x => (34, 44, 60),
    y => (66, 49, 20),
    label => (
      'Mean age at selection = 34',
      'Mean age at mission = 44',
      'John Glenn was 77 on his last mission'
    ),
    hjust => 'left',
    vjust => 'top',
    offset => (10, 0)
SCALE fill TO accent
LABEL
  title => 'How old are astronauts on their most recent mission?',
  subtitle => 'Age of astronauts when they were selected and when they were sent on their mission',
  x => 'Age of astronaut (years)',
  fill => null
```
