## 内容摘要

### 什么是 jj？

`jj` 是 Jujutsu 的命令行工具名称。Jujutsu 是一个分布式版本控制系统（DVCS）。用户可能对其他分布式版本控制系统比较熟悉，比如 `git`，本教程假设读者是从 `git` 迁移过来的。

### 为什么应该使用 jj？

jj 具有一个在编程领域中非常罕见的特性：它既比 `git` 更简单、更易用，同时又更强大。这是一个非常大的claims。在编程中，我们通常被教导（正确地）存在取舍权衡，而「强大但复杂」是一个非常常见的权衡。git 的强大功能是有价值的，因此人们从其前辈系统中纷纷转向了 git。

jj 成功地将 git 的优点与 Mercurial（hg）的优点相结合，创造出一些新的、但又异常熟悉的东西。重要的是， jj 减少了必要的工具数量，同时让这些工具更加强大，因为它们以更清晰的方式协同工作。此外，更高级的 jj 使用可以提供额外的强大工具，这些工具在 git 中很难实现。

还有一个原因是 jj 有一个 git 兼容的后端，你可以自己使用 jj，而不需要求其他人都转换。这意味着尝试 jj 没有任何真正的缺点；如果它不适合你，你不会失去使用它编写的所有历史，可以毫无问题地回到 git。

---

## 核心要点

1. **更简单更强大**: jj 既有简化的工作流程，又提供比 git 更强大的功能
2. **汲取百家之长**: 结合了 git 和 Mercurial 的最佳特性
3. **工具更少但更强**: 必要的工具数量减少，但功能更强大
4. **git 兼容**: 可以与 git 共存，无需全员转换
5. **零风险尝试**: 不满意可以无损失地回到 git

---

## 结论

jj 是一个值得关注的分布式版本控制系统，它通过创新设计实现了「更简单更强大」的理念。对于已经熟悉 git 的开发者来说，jj 提供了一个平滑的迁移路径，同时保留了 git 的兼容性，使得尝试成本极低。
