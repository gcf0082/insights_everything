## 项目概述

这是一个名为 **ymawky** 的项目——完全用 ARM64 汇编语言编写的 macOS 静态文件 Web 服务器。

**主要功能**:
- 支持 GET、PUT、DELETE、HEAD、OPTIONS 请求
- 支持 Range: bytes=X-Y 头（支持视频流拖动）
- 解码百分比编码的 URL
- 严格强制 docroot
- 为任何 HTTP 错误响应提供自定义错误页面
- 支持目录列表
- 包含针对 slowloris 类攻击的（部分）缓解措施

**技术细节**:
- 代码量约 3000 行，注释约 1000 行
- 使用 fork-per-connection 模型（每个连接 fork 一个进程）
- 作者承认性能会比 nginx 等生产级服务器慢，因为架构差异

### 学习资源的讨论
- 作者推荐学习 ARM 汇编的方法：
  - 阅读现有汇编代码来感受其工作方式
  - 大胆搜索一切出错的东西
  - 阅读 ARM 架构参考手册（The ARM ARM）
  - 用 C/C++ 编写代码后用 `gcc -O1 -S file.c` 查看生成的汇编
  - 在 gdb 或 lldb 中调试小程序

### 关于系统调用兼容性
- 用户 `mappu` 指出 macOS 的系统调用不稳定，Go 在 1.12 版本改用调用 libSystem.dylib
- 作者 `imtomt` 回应自己是"受虐狂"，尽量避免使用 libSystem.dylib，只有在 macOS 要求时才链接它

### 关于 Portability
- 关于移植到 Linux 的问题，作者指出：
  - 调用约定不同（macOS 用 x16 寄存器做系统调用号，Linux 用 x8）
  - 系统调用号不同
  - 结构体布局不同（如 sigaction 的 sa_tramp 字段）
  - proc_info() 是 Darwin 特有的，Linux 没有等效物
