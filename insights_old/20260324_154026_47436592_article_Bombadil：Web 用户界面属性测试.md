## 项目概述

Bombadil 是一款面向 Web 用户界面的属性测试（Property-based Testing）工具，也称为模糊测试（Fuzzing）工具。它能够自主探索 Web 应用的状态空间，通过随机和系统化的方式发现问题，从而在开发过程中更早地发现难以察觉的 bug。

该工具可在以下环境中运行：

- 本地开发环境
- CI/CD 持续集成环境
- Antithesis 平台

## 核心功能

### 1. 自动化属性验证

用户无需编写固定的测试用例，而是定义系统应该满足的一般性属性（Properties）。Bombadil 在探索应用的过程中，会持续验证这些属性是否被满足，并报告任何违规情况。

### 2. 智能状态探索

Bombadil 通过以下循环流程进行测试：

1. 从浏览器中提取当前状态
2. 检查所有属性是否满足，记录违规情况
3. 根据当前状态选择下一个操作并执行
4. 等待下一个事件（页面导航、DOM 变化或超时）
5. 返回步骤 1 继续执行

### 3. 灵活的规格定义

用户使用 TypeScript 编写规格（Specification），导出属性和操作生成器。这些可以是领域特定的，用于以自定义方式锻炼和验证系统逻辑，也可以使用 Bombadil 提供的默认属性和操作生成器。

### 4. 多场景支持

Bombadil 适用于任何使用 DOM 的 Web 应用，包括：

- 单页应用（SPA）
- 服务端渲染应用（SSR）
- 静态 HTML 页面

## 为什么选择属性测试？

### 传统示例测试的局限性

- **成本高昂**: 维护 Playwright 或 Cypress 测试套件需要大量工作
- **覆盖有限**: 只测试状态空间的很小部分，通常只覆盖快乐路径、回归测试和部分错误处理场景

### 属性测试的优势

- **随机探索**: 随机且系统地搜索状态空间
- **发现意外**: 能够发现开发者未曾考虑到的意外操作序列
- **异常处理**: 能够测试异常时序和奇怪输入
- **提高效率**: 自动发现手动测试难以覆盖的边界情况

## 使用方式

### 本地开发环境

```bash
# 安装 Bombadil
npm install -g @antithesis/bombadil

# 运行测试
bombadil test --spec your-spec.ts
```

### CI 集成

Bombadil 可以集成到现有的 CI/CD 流程中，自动化执行属性测试。

### Antithesis 平台

在 Antithesis 平台上运行可以获得更强大的测试能力。

## 总结

Bombadil 代表了 Web 测试领域的一种新范式。它将属性测试的力量带入 Web UI 测试，通过自动化探索和验证，帮助开发者在开发早期阶段发现更深层次的 bug。虽然该项目目前仍处于实验阶段，但其创新理念和实用价值值得关注。

对于希望提高 Web 应用测试覆盖率和质量的团队来说，Bombadil 提供了一种有别于传统测试方法的替代方案，特别适合需要处理复杂状态逻辑的现代 Web 应用。
