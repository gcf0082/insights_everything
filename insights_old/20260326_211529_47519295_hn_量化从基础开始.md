## 摘要

本讨论围绕模型量化（Quantization）技术展开，这是一种将大型语言模型从浮点数压缩为小整数的技术，使得模型能够在消费级硬件上运行。文章来自 ngrok 博客，通过可视化方式深入讲解了量化的原理和实现。

## 主要讨论内容

### 1. GPU 硬件支持

- **M1 Max:** 原生支持 FP16，FP8 和 Bfloat16 通过软件模拟（需要反量化）
- **H100:** 原生支持 FP16 和 FP8
- 老旧 GPU 必须将短格式转换为下一个更大的格式才能执行运算

### 2. 极端量化可能性

- Microsoft BitNet 和 Google TurboQuant 证明了可以使用 -1, 0, +1 三元量化
- LLM 本质上是比较 token 之间的距离，极端量化仍有可行性

### 3. 量化的实际意义

- **Qwen 3.5 27B 模型：** FP16 需要 54GB VRAM，Q4_K_M 量化后仅需 16GB
- **14B 模型 Q4 量化：** 约 10GB，RTX 3060 12GB 即可运行
- 量化是让 AI 民主化、让模型在消费级硬件上运行的关键技术

### 4. 动态量化

- 层级别的动态量化是一个 established 的想法
- 使用校准数据集，根据层的重要性调整量化位数
- 参考：EvoPress (https://arxiv.org/abs/2410.14649)

### 5. 模型质量与量化的权衡

- 5-10% 的准确率差异可能就是可用与不可用模型的区别
- 较小模型与较大量化模型的选择：通常更大的量化模型效果更好
- 质量差异在多步骤推理中会累积，单步 perplexity 或 KL divergence 无法完全捕捉
