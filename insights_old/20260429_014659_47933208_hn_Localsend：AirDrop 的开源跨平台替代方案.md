## 核心洞察

Localsend 是一个开源的跨平台文件传输工具，可以作为 AirDrop 的替代方案。然而，它与 AirDrop 存在一个根本性的区别：**Localsend 需要设备连接到同一个本地网络才能工作**，而 AirDrop 能够自动创建和处理本地网络。

## 技术讨论

### AWDL 的工作原理

AirDrop 使用 Apple Wireless Direct Link（AWDL）协议，这是一种专有的点对点层，可以与现有 WiFi 连接同时运行而不会断开。它使用时间片通道切换机制，使无线电可以同时为基础设施 WiFi 和直接对等链接提供服务。

### 复制难度

实现类似 AWDL 功能的主要障碍包括：
- 需要操作系统级别的支持来协商直接无线电链路
- 大多数替代方案只是 TCP/IP，需要现有的共享网络
- Android 的 QuickShare 在较旧的设备上会断开现有 WiFi 连接，因为无线电一次只能关联一个 BSS
- PC 上的 WiFi 卡不支持这种功能

### Wi-Fi Aware 标准

Apple 已将核心逻辑贡献给 Wi-Fi 联盟以构建 Wi-Fi Aware（也称为 NAN - Neighbourhood Aware Networking）。截至 2025 年：
- iOS 的 Airdrop 已与 Wi-Fi Aware 透明地互操作
- iOS 26 已添加开发者 API
- Google 自 Pixel 3 以来已实现该功能
- 三星 S26 Ultra 支持与 iPad 之间的 AirDrop 传输

### EU 法规影响

欧盟的互操作性 mandates 迫使 Apple 在 iOS 26 中采用 Wi-Fi Aware 标准，这让 Android 设备可以与 AirDrop 互操作。

## 用户体验对比

| 特性 | AirDrop | Localsend |
|------|---------|----------|
| 需要相同网络 | 否 | 是 |
| 自动发现设备 | 是 | 是 |
| 跨平台 | 仅 Apple | 是 |
| 传输速度 | 快 | 快 |

## 替代方案

- **KDE Connect**：Linux 上的文件传输
- **PairDrop**：基于 WebRTC P2P 的 PWA
- **FlyingCarpet**：跨平台但不够可靠
- **QuickShare/rquickshare**：Linux 版本存在
- **Sendme/Iroh**：基于 P2P 中继的服务

## 主要观点

1. **AirDrop 的核心优势**：与陌生人进行临时文件分享的便利性是其独特卖点
2. **LocalSend 的适用场景**：在自己控制的设备之间传输文件很方便
3. **实际痛点**：AirDrop 有时不稳定，用户经常遇到无法发现设备的问题
4. **用户体验**：即使 LocalSend 需要手动设置网络，其可靠性往往优于 AirDrop
