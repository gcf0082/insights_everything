## 摘要

这篇文章的作者David Bushell是一位资深Web开发者，他宣布"AI赢了"（The Clankers Won），认为编程和博客已死。但出人意料的是，文章的核心观点恰恰是**反对这种悲观论调**，呼吁开发者继续写博客、保持独特的人类声音。

## 原文核心观点

1. **博客从未如此重要**：在AI内容泛滥的时代，原创人类思考变得更有价值
2. **AI是威胁而非工具**：作者认为AI是"巨大的抄袭机器"，版权已死
3. **保持独特声音**：不要把思考外包给AI，要继续用博客表达原创观点
4. **支持旧互联网**：倡导开放Web、独立Web

## 讨论热点

### 1. 公司是否重视开发者技能提升？

- **正方**：部分公司确实投资于开发者培训
- **反方**：多数公司只关心解决问题，不关心员工技能成长
- **观察**：AI浪潮中，许多公司甚至希望员工技能萎缩

### 2. AI对开发者技能的影响

**担忧派：**
- 技能退化：AI导致开发者无法理解和修复底层系统
- 类似COBOL危机：未来可能找不到懂基础技术的人
- 初级招聘已停止，灾难可能即将发生

**乐观派：**
- 使用AI可以学习更多工具和系统
- 以前没时间学的技术现在可以探索了
- AI帮助完成繁琐工作，让人专注更深层次的学习

### 3. 注意力跨度退化

- 注意力退化早在AI出现前就发生了（互联网时代）
- 浏览器标签页和终端多路复用器是元凶
- 建议：阅读长文、写作、与真人对话

### 4. AI与开源运动

- AI不等于开源：开源运动希望代码可修改，AI恰恰相反
- 开源社区想要的是人们提升技能、获得技术素养
- Closed source不再是护城河，但AI也不提供源代码

### 5. 对抗AI的方式

- 阻止所有爬虫：不贡献给AI训练数据
- 使用本地模型：减少对云端AI的依赖
- 写博客但不被AI爬取

## 多元观点

| 立场 | 观点 |
|------|------|
| 悲观 | AI是永恒运动的骗局，最终会崩溃 |
| 乐观 | AI是老师，帮助学习音乐等新技能 |
| 中立 | AI既没那么好也没那么坏，世界有 nuance |
| 务实 | 不使用AI但也不反对别人使用 |

## 结论

这场讨论反映了开发者对AI改变行业的深切焦虑。无论是悲观还是乐观，大多数人同意：
- 保持学习、避免技能退化很重要
- 原创思考和人类声音仍有价值
- 博客作为自我表达和学习工具不会消失
