# 洞察报告：Linux 7.0 导致 PostgreSQL 性能下降问题

## 基本信息

- **洞察链接**: https://news.ycombinator.com/item?id=47644864
- **原始链接**: https://www.phoronix.com/news/Linux-7.0-AWS-PostgreSQL-Drop
- **标题**: AWS engineer reports PostgreSQL perf halved by Linux 7.0, fix may not be easy
- **发布时间**: 6小时前
- **得分**: 230 points
- **评论数**: 54 comments
- **报告日期**: 2026-04-05

## 事件概述

一位AWS工程师报告称，在Linux 7.0内核上运行时，PostgreSQL的性能下降了一半。该问题与Linux 7.0引入的PREEMPT_LAZY调度机制有关。

## 技术细节

1. **性能影响**: 在96核ARM64机器上，PostgreSQL性能降至原来的0.51倍
2. **平台差异**: 该问题在AMD64平台上无法重现，主要影响ARM64架构
3. **根本原因**: Linux 7.0引入了新的PREEMPT_LAZY预emption机制，移除了PREEMPT_NONE选项
4. **临时解决方案**: 启用huge pages可以减轻性能下降

## 社区讨论要点

- **影响范围**: 多数评论认为这是一个极端工作负载下的问题，普通用户影响有限
- **解决建议**: PostgreSQL需要采用内核7.0的新机制来缓解此问题
- **时间线**: Linux 7.0稳定版将在两周后发布，Ubuntu 26.04 LTS也将于4月发布
- **风险评估**: 使用huge pages的PostgreSQL用户基本不受影响

## 后续发展

PostgreSQL开发者Andres Freund在LKML邮件中表示，问题已被重现但影响程度较小。对于不使用huge pages的工作负载，PREEMPT_LAZY带来的性能下降比不使用huge pages本身的损失要小。社区普遍认为这是一个边缘情况，不会对大多数实际工作负载产生影响。

## 总结

虽然这是一个真实的性能回归问题，但其影响范围可能被夸大。该问题主要影响特定配置下（ARM64架构、大量核心、未启用huge pages）的PostgreSQL实例。大多数生产环境使用huge pages或不会立即升级到Linux 7.0，因此实际影响有限。
