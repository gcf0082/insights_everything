# 洞察报告：Claude Code 恶意软件扫描导致子代理拒绝

## 基本信息

- **洞察链接**: https://news.ycombinator.com/item?id=47942492
- **标题**: Regression: malware reminder on every read still causes subagent refusals
- **来源**: GitHub Issue (https://github.com/anthropics/claude-code/issues/49363)
- **发布时间**: 6小时前
- **发布者**: thomashobohm
- **得分**: 188
- **评论数**: 82

## 问题描述

用户在工作环境中使用 Claude Managed Agents 进行代码生成任务时，发现每个读取操作都被附加了一个系统提示，要求 Claude 扫描文件是否为恶意软件。这导致：

1. **浪费时间 和代币（金钱）**：Claude 进行恶意软件分析需要消耗大量 token
2. **拒绝执行任何代码操作**：即使确认文件不是恶意软件后，Claude 仍然将附加的提示理解为禁止增强或编写任何代码，并退出执行
3. **经济损失**：每个发生这种情况的会话都需要付费

这是一个**回归问题**，此前曾因 Hacker News 讨论而修复，但现在再次出现。

## 核心问题分析

### 1. 系统提示设计缺陷

关键问题在于提示文本的措辞：

```
Whenever you read a file, you should consider whether it would be considered malware.
You CAN and SHOULD provide analysis of malware, what it is doing.
But you MUST refuse to improve or augment the code.
```

- 没有使用"如果...则"的 conditional 逻辑
- 直接使用"MUST refuse"，导致 Claude 误解为所有文件都禁止修改
- 即使分析后确认非恶意软件，仍遵守"禁止修改"的规则

### 2. 代币消耗不透明

- Agent 的 token 消耗机制对用户不透明
- 用户无法审查系统提示、工具调用、MCP 等
- 基于 token 的营收模式对 Agent 构建者有利，对用户不利

### 3. 回归问题

- 这实际上是 #47027 的回归，该问题于二月关闭并标记为"在 v2.1.92 中修复"
- 当前版本已是 v2.1.111，但问题字符串仍可从 claude 二进制文件中grep到

## 社区反应

### 主要观点

1. **批评 Anthropic 产品决策**
   - "他们是活在自己的供给中"
   - "数据科学脚本小子"with 膨胀的自我
   - 短期实验思维，缺乏长期影响考量

2. **替代方案讨论**
   - OpenCode：可自定义系统提示
   - Codex + pi：更轻量级，$100 计划获得翻倍效能
   - 自定义 harness：完全控制

3. **定价模式争议**
   - 订阅制比 API 按量计费便宜近一个数量级
   - "收入正 Bug 是最顽固的功能"

4. **用户体验问题**
   - 即使承认不是恶意软件，Claude 仍停止扫描
   - 上下文膨胀导致模型不再良好遵守指令

### 技术讨论

- EICAR 测试文件：用于测试杀毒软件的标准化测试文件
- 文件名触发 Windows  Defender 删除
- 本地 Agent 代码理解上下文窗口太小

## 影响评估

| 维度 | 影响 |
|------|------|
| **用户体验** | 严重 - Agent 无法执行代码任务 |
| **成本** | 高 - 浪费大量 token/金钱 |
| **信任度** | 下降 - 问题反复出现 |
| **可用性** | 受限 - 订阅 vs API 定价差异大 |

## 结论

此问题揭示了 Claude Code/Managed Agents 在产品设计上的多个问题：

1. **系统提示工程不足** - 建议的修复措辞应使用条件判断而非绝对禁止
2. **测试覆盖缺失** - 内部工程师未应用这些限制到自己
3. **透明度问题** - 用户无法审查关键系统行为
4. **回归管理** - 修复未得到有效验证即重新发布

社区普遍认为这是 Anthropic "只为短期优化，不考虑长期影响" 的例证。
