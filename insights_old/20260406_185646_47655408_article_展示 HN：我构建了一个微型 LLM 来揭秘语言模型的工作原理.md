## 项目概述

GuppyLM 是一个约900万参数的微型语言模型，扮演一只名为Guppy的小鱼。它用简短的小写句子说话，谈论水、温度、食物和鱼缸生活。该项目旨在展示训练自己的语言模型并不神奇，无需PhD学位，无需大规模GPU集群，只需一个Colab笔记本和5分钟即可完成。

---

## 架构规格

| 参数 | 数值 |
|------|------|
| 参数数量 | 8.7M |
| 层数 | 6 |
| 隐藏维度 | 384 |
| 注意力头数 | 6 |
| FFN维度 | 768 (ReLU) |
| 词汇表大小 | 4,096 (BPE) |
| 最大序列长度 | 128 tokens |
| 归一化 | LayerNorm |
| 位置编码 | 可学习嵌入 |
| LM头 | 与嵌入层权重共享 |

**架构特点**：使用 vanilla transformer，无GQA、无RoPE、无SwiGLU、无早退，极其简单。

## 设计决策

1. **为什么没有系统提示？** 每个训练样本使用相同的系统提示。9M模型无法条件跟随指令，性格已固化在权重中。移除可节省约60个token。

2. **为什么仅单轮对话？** 多轮对话在第3-4轮会退化，受限于128 token上下文窗口。记忆丧失的鱼很符合品牌，但输出混乱则不行。单轮更可靠。

3. **为什么用vanilla transformer？** GQA、SwiGLU、RoPE和早退增加复杂性和9M参数下无帮助的代码量。标准attention + ReLU FFN + LayerNorm用更简单的代码产生相同质量。

4. **为什么用合成数据？** 具有一致性格的鱼需要一致的训练数据。模板组合(30个鱼缸物体、17种食物、25种活动)可从约60个模板生成约16K个独特输出。

## 使用方式

### 与Guppy聊天（无需训练）
```python
pip install torch tokenizers
python -m guppylm chat
```

### 本地运行示例
```
You> the cat is looking at you
Guppy> i hide behind the plant when the furry one comes.

You> it is raining outside
Guppy> i think rain is the best thing about outside.
```

## 总结

GuppyLM 是一个教育目的的微型语言模型演示项目，展示了从原始文本到训练权重再到生成输出的完整流程。它使用简单的vanilla transformer架构，在60K合成对话上进行训练，生成了独特的小鱼角色。该项目降低了学习语言模型的门槛，证明了训练LLM不需要昂贵的硬件和专业背景。
