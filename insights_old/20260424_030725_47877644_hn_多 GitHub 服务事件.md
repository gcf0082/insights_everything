## 事件概述

2025年4月，GitHub 再次发生多服务故障，引发社区广泛讨论。根据第三方监控系统 mrshu.github.io/github-statuses/ 的数据，GitHub 整体可用性已跌至 88.15%，远低于其宣称的 99.7% 水准。git 操作的可用性仅为 98.98%，勉强达到「一个九」的标准。

---

## 核心观点

### 1. 自托管成为趋势

多位用户分享了自托管 GitHub 替代方案的经验：

- **Forgejo**：用户 embedding-shape 完成了自托管 Forgejo 实例的搭建，配合 Linux、Windows 和 macOS 运行器，实现完整的 CI/CD 流程
- **Gitea/Forgejo**：用户 bakies 表示自托管实例拥有更好的可用性和速度
- **GitLab**：用户 argee 迁移到 GitLab，享受免费的自托管 CI 运行器

### 2. 可用性质疑

社区对 GitHub 宣称的可用性数据提出强烈质疑：

- 原始数据显示各组件单独可用性看似不错，但综合可用性仅 88.15%
- Git 操作（核心功能）的实际可用性仅为 98.98%，远低于宣传的 99.7%
- 有人指出将服务分割成多个组件是一种让 SRE 数据看起来更好的策略

### 3. 用户流失

部分用户开始放弃 GitHub 服务：

- **jasoncartwright**：取消 GitHub Copilot Pro+ 年订阅（£160），转向 Claude Code
- 用户 surya2006：希望迁移但因生态锁定无法实现

### 4. 替代方案讨论

- **Forgejo 15**：刚刚发布，支持仓库特定访问令牌
- **GitLab**：被认为是最接近 GitHub 的替代方案
- **自建服务**：用户在 Hetzner 盒子搭建 Gogs 或 Forgejo

## 技术细节

### 自托管方案示例

用户 dml2135 的 homelab 配置：

- Dell optiplex mini 运行 Proxmox
- Unraid NAS 用于存储
- Debian VM 运行 Forgejo 和 Komodo
- 使用 NixOS 进行声明式配置
- Claude 协助构建部署流水线

### NixOS 优势

用户 embedding-shape 指出，使用 NixOS 管理自托管服务大幅降低了复杂性：

- 约 100 行配置即可完成 Forgejo + 运行器的部署
- 即使离开数月后也能快速理解系统状态
- 符合「越简单越好」的原则

## 影响分析

### 对开发者

1. **工作中断**：CI/CD 流水线不稳定影响发布流程
2. **心态变化**：对云服务的信任度下降
3. **行动倾向**：加速自托管或迁移至替代平台

### 行业启示

1. **单一依赖风险**：对 SaaS 平台过度依赖存在隐患
2. **可用性质疑**：提供商宣称的可用性与实际体验存在差距
3. **市场机会**：自托管 Git 解决方案存在市场需求

---

## 总结

GitHub 多次故障已引发社区显著不满，综合可用性跌至「两个九」以下。自托管解决方案（尤其是 Forgejo）正在成为部分开发者的替代选择。虽然生态锁定仍是迁移的主要障碍，但随着自托管工具的成熟，这一趋势可能加速。
