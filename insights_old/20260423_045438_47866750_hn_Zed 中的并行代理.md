## 内容摘要

Zed编辑器发布了"Parallel Agents"功能，允许用户并行运行多个AI代理，并支持通过Git worktree创建独立的隔离工作环境。每个worktree可以配置独立的开发环境（如数据库），并在关闭时自动清理资源。这一功能类似于为每个代理创建一个独立的虚拟机环境。

## 核心功能

1. **并行代理线程**：支持同时运行多个AI代理，每个代理可在独立的工作空间中操作
2. **Git Worktree集成**：自动为每个代理创建隔离的worktree，支持生命周期钩子
3. **环境隔离**：可为每个worktree配置独立的本地配置和数据库，实现完全的测试隔离
4. **代理无关性**：支持多种AI提供商（OpenAI、Anthropic、GitHub Copilot等），非单一厂商绑定

## 社区观点

### 正面评价

- **编辑器体验优秀**：Zed仍保持快速、流畅的编辑体验，即使不启用AI功能也很好用
- **可定制性强**：提供一键关闭所有AI功能的选项，用户可选择纯手写代码模式
- **多提供商支持**：代理-agnostic设计，不强制绑定特定AI供应商
- **UI质量高**：相比 Electron 竞品，Zed 基于原生Rust实现，性能更好

### 批评与担忧

- **AI集成过重**：有用户认为Zed的AI功能是VC驱动的营销噱头，核心编辑器开发投入不足
- **MCP支持缺失**：目前不支持 Model Context Protocol 集成，缺少插件和skills支持
- **布局变化**：新版布局将文件树面板移至右侧，可能让新用户不适应
- **TypeScript性能**：有用户反馈TypeScript语言服务器交互速度不如VS Code

### 建议编辑器

部分用户推荐其他仍专注于手写代码体验的编辑器：

- Neovim/Emacs：高度可定制，完全本地控制
- Helix：现代 vim-like 编辑器，专注于纯编辑体验
- Gnome Edit/Kate：轻量级编辑器

## 技术实现亮点

- **生命周期钩子**：通过Zed的tasks和hooks机制支持创建/清理自动化
- **灵活面板布局**：面板可通过右键自定义停靠位置
- **Material图标主题**：提供易读的图标主题选项
