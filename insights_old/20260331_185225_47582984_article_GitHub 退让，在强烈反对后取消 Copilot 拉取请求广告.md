# 洞察报告

**链接**: https://www.theregister.com/2026/03/30/github_copilot_ads_pull_requests/

**来源**: The Register

**发布日期**: 2026年3月30日

**作者**: Brandon Vigliarolo

**主题**: GitHub Copilot PR广告事件

---

## 摘要

2026年3月，GitHub在开发者强烈反对后，紧急撤除了Copilot在Pull Request中插入广告（称为"tips"）的功能。这一事件引发了关于AI工具边界的广泛讨论。

## 事件详情

### 事件起因

澳大利亚开发者Zach Manson在2026年3月发现，当他的同事请求Copilot修正一个PR中的拼写错误时，Copilot在PR中添加了一条推广Raycast生产力应用的信息：

> "Quickly spin up Copilot coding agents from anywhere on your macOS or Windows machine with Raycast"（附带闪电图标和Raycast安装链接）

Manson表示，最初他以为是训练数据被污染或某种新型提示注入攻击。

### 影响范围

- GitHub上已有超过11,400个PR包含类似的广告内容
- 这些广告均由Copilot自动插入
- 开发者发现Copilot可以修改非自己创建的PR的描述和评论

### 开发者反应

Manson批评道："我甚至不知道GitHub Copilot Review集成有能力编辑其他用户的描述和评论。我想不出任何有效的用例来支持这种能力。"

## GitHub的回应

### 官方解释

GitHub开发者关系副总裁Martin Woodward在X上发文解释：
- Copilot在自身创建的PR中插入广告并非新行为，已存在一段时间
- 新功能允许Copilot处理任何被提及的PR，但这一行为"变得令人反感"

### 道歉与修复

Copilot principal产品经理Tim Rogers在Hacker News上承认：
- 添加"tips"功能的初衷是"帮助开发者学习使用代理的新方法"
- 经过反思，"让Copilot在人类不知情的情况下修改PR是错误的判断"
- 已禁用Copilot在PR中插入广告的功能

### 后续声明（3月31日更新）

Martin Woodward发表正式声明："GitHub不会也不计划在GitHub中包含广告。我们发现了一个GitHub Copilot编码代理tips的编程逻辑问题，该问题在Pull Request评论中出现在错误的上下文。我们已从Pull Request评论中移除代理tips。"

## 相关背景

这是GitHub Copilot近期面临的争议之一。其他相关事件包括：
- GitHub恢复使用用户数据训练AI的决定
- 从免费Copilot计划中移除部分模型
- 开发者对强制Copilot功能的抱怨

## 结论

这一事件凸显了AI开发工具在商业化和用户体验之间需要平衡的问题。GitHub的快速响应表明社区反馈对产品决策的重要影响。
