# GitHub 历史正常运行时间洞察报告

## 基本信息

- **洞察链接**: https://damrnelson.github.io/github-historical-uptime/
- **数据来源**: GitHub 官方状态页面历史存档
- **覆盖时间范围**: 2016年至2026年
- **创建者**: DaMrNelson
- **报告生成时间**: 2026年4月1日

## 洞察内容

### 项目概述

该网站由开发者 DaMrNelson 创建，通过抓取 GitHub 官方状态页面的历史数据，重建了 GitHub 多年来的正常运行时间（uptime）指标和事件时间线。网站以图表形式展示 GitHub 每月及近90天的正常运行时间百分比。

### 主要发现

#### 1. 历史正常运行时间数据
- 网站展示了 GitHub 从2016年到2026年的月度正常运行时间数据
- 用户可以查看各服务的90天正常运行时间指标
- 正常运行率范围从97.30%到100%不等

#### 2. 2026年可靠性问题
根据多个来源显示，GitHub 在2026年初经历了多次服务中断：

| 时间 | 事件 |
|------|------|
| 2026年2月 | GitHub 正常运行时间低于90%，仅为87%（来源：Hacker News讨论） |
| 2026年2月 | 发生6起导致服务降级的事件（GitHub官方报告） |
| 2026年2月28日 | 主服务受影响，持续约2小时35分钟 |
| 2026年3月 | 多次服务中断 |

#### 3. SLA合规性问题
- GitHub 企业版云服务承诺99.9%（三个九）的正常运行时间
- 相当于每月允许约8.76小时的停机时间
- 2026年2月的87%正常运行率远低于承诺的SLA目标
- 这意味着大约43小时的停机时间，远超SLA允许范围

### 开发者社区反应

1. **Reddit讨论**: 有用户将 GitHub 的历史停机时间数据进行抓取和可视化
2. **Hacker News讨论**: 社区对 GitHub 近期的可靠性问题表示担忧
3. **媒体关注**: The Register 等技术媒体报道了 GitHub 的可靠性危机

## 结论

GitHub 的历史正常运行时间数据显示，虽然早期年份表现较为稳定，但2026年以来服务可靠性显著下降。这对于依赖 GitHub 进行软件开发的企业和团队来说是一个值得关注的问题。GitHub 官方已承认服务中断对团队和工作流程造成的影响，并表示正在改进。
