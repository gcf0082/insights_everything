## 正文

Cloudflare 正在构建一个统一的 CLI 工具，整合其所有产品线。该 CLI 目前处于技术预览阶段，可通过 `npx cf` 或 `npm install -g cf` 安装使用。核心设计理念包括三个方面：

1. **TypeScript 作为架构语言**：使用 TypeScript 类型系统来定义 API、CLI 命令和参数，确保一致性
2. **本地探索功能**：支持本地调试 Workers，无需部署即可测试
3. **AI 优先设计**：CLI 命令优先于 GUI，因为 AI 智能体更擅长使用 CLI

---

## 关键洞察

### 1. CLI-first 趋势回归
- AI 智能体的发展推动了 CLI 工具的复兴，开发者工具正在回归 CLI 和 API 优先的设计模式
- 智能体擅长使用 CLI 但不擅长诊断命令失败原因，因此清晰的错误提示至关重要

### 2. 权限管理痛点
- 用户期望 `cf permissions check` 命令能在部署前显示所需的 API Token 权限
- 希望有工具能创建狭义作用域的短生命周期 Token
- 代理模式也是一个潜在解决方案，可动态缩窄权限范围

### 3. TypeScript 争议
- Cloudflare 选择 TypeScript 作为"软件工程通用语言"引发讨论
- 部分开发者认为 TypeScript 性能不佳，且被微软控制
- 另观点认为 TypeScript 生态丰富，Deno/Bun 等运行时正在改善性能

### 4. 用户体验建议
- `-h` 和 `--help` 应提供标准化的信息层级区分
- CLI 不应有交互式提示，尤其在帮助命令中
- 命令间的一致性问题（如颜色、选项格式）需改进

### 5. 跨工具对比
- TypeSpec 和 AEP (API Engineering Practices) 被推荐作为更优的 API 定义方案
- Speakeasy 和 Restish 等工具可从 OpenAPI 自动生成 CLI
- Terraform 集成是一大优势，能开箱即用无需编写Provider
