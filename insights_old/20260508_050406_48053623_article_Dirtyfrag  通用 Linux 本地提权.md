# Dirty Frag: 通用 Linux 本地提权漏洞洞察报告

## 基本信息

- **洞察链接**: https://www.openwall.com/lists/oss-security/2026/05/07/8
- **漏洞名称**: Dirty Frag
- **漏洞类型**: 本地提权漏洞 (Local Privilege Escalation, LPE)
- **影响范围**: 所有主流 Linux 发行版
- **披露日期**: 2026年5月8日
- **发布者**: Hyunwoo Kim (imv4bel@...il.com)

## 漏洞概述

Dirty Frag 是一款通用的 Linux 本地提权漏洞，允许攻击者在所有主流 Linux 发行版上获取 root 权限。该漏洞的影响与此前披露的 "Copy Fail" 漏洞类似，都能够在无需身份认证的情况下立即获取 root 权限。

由于公开披露期（embargo）已被破坏，目前没有任何针对此漏洞的官方补丁或 CVE 编号。在与 linux-distros@openwall.org 维护者协商后，维护者请求作者公开披露此漏洞的详细信息。

## 技术细节

### 漏洞链

Dirty Frag 漏洞利用了两个独立的内核漏洞：

1. **第一个漏洞**:
   - 链接: https://git.kernel.org/pub/scm/linux/kernel/git/netdev/net.git/commit/?id=f4c50a4034e62ab75f1d5cdd191dd5f9c77fdff4

2. **第二个漏洞**:
   - 链接: https://lore.kernel.org/all/afKV2zGR6rrelPC7@v4bel/

### 漏洞原理

根据披露的信息，该漏洞利用了 Linux 内核中的两个独立安全缺陷，通过链式利用实现从普通用户到 root 权限的提权。漏洞涉及以下内核模块：

- **esp4**: IPv4 ESP (Encapsulating Security Payload) 模块
- **esp6**: IPv6 ESP 模块
- **rxrpc**: RxRPC 网络协议模块

### 漏洞利用

披露的文章包含了完整的漏洞利用代码（PoC）。利用过程涉及：

1. 创建用户命名空间和网络命名空间
2. 利用 xfrm (IPsec) 子系统创建恶意安全关联 (SA)
3. 通过 splice 系统调用实现内存操作
4. 修改 /usr/bin/su 等系统二进制文件
5. 最终获取 root shell

## 影响范围

该漏洞影响所有主流 Linux 发行版，包括但不限于：

- Ubuntu
- Debian
- Fedora
- CentOS
- Arch Linux
- openSUSE

由于漏洞存在于 Linux 内核的网络子系统，理论上所有运行受影响内核版本的系统都存在风险。

## 缓解措施

由于目前没有官方补丁，用户可以使用以下命令临时缓解漏洞风险：

```bash
sh -c "printf 'install esp4 /bin/false\ninstall esp6 /bin/false\ninstall rxrpc /bin/false\n' > /etc/modprobe.d/dirtyfrag.conf; rmmod esp4 esp6 rxrpc 2>/dev/null; true"
```

此命令将：
1. 创建配置文件禁用加载有漏洞的内核模块
2. 立即移除当前运行的有漏洞模块

## 总结

Dirty Frag 是一个严重的安全漏洞，允许任何本地用户在受影响的 Linux 系统上获取 root 权限。由于公开披露期已被破坏，用户应立即采取缓解措施，禁用有漏洞的内核模块，直到官方发布安全补丁。

建议系统管理员：
1. 尽快应用上述缓解措施
2. 关注各 Linux 发行版的安全公告
3. 持续监控系统安全状态
