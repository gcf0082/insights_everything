## 内容摘要

**原文链接**: https://www.anthropic.com/news/higher-limits-spacex

Anthropic 宣布了三项重大更新，旨在改善最忠实的客户使用 Claude 的体验：

1. **提高 Claude Code 速率限制**: 将 Pro、Max、Team 和基于席位的 Enterprise 计划的五小时速率限制翻倍
2. **取消高峰时段限制降低**: 移除 Pro 和 Max 账户在高峰时段的限制减少
3. **提高 API 速率限制**: 显著提高 Claude Opus 模型的 API 速率限制

**核心交易**: Anthropic 与 SpaceX 达成计算协议，获得 SpaceX Colossus 1 数据中心的计算容量。此外，Anthropic 表示有兴趣与 SpaceX 合作开发多个千兆瓦的轨道 AI 计算容量。

## 主要评论观点

### 1. 关于与 SpaceX 的合作

- **双赢局面**: Anthropic 获得急需的计算资源来持续发展；Elon 将 xAI 无法充分利用的计算（因为 Grok 需求不大）转化为收入，为 IPO 做准备
- **质疑声**: 有评论认为这是 xAI 放弃的迹象，Grok 模型正在陆续停产
- **xAI 前景**: 有人指出 xAI 还有 Colossus 2，将继续运营

### 2. 关于速率限制政策

- **"解决问题 vs 创造问题"**: 许多用户批评 Anthropic 创造了 5 小时窗口限制问题，然后又将其描述为对用户的恩惠
- **周限制未翻倍**: 关键观察是五小时限制翻倍，但周限制未同步增加，意味着整体使用量不变
- **用户体验差异**: 
  - 部分用户频繁触及小时限制但很少达到周限制
  - 部分用户情况相反
  - 有人提到在复杂任务中可以在 5 小时内消耗 10% 的周配额

### 3. 环境与伦理担忧

- **Colossus 环境影响**: 讨论聚焦于 Colossus 使用非法电力、污染孟菲斯附近空气质量、潜在水源污染等问题
- **与 Musk 合作**: 有评论批评 Anthropic 与一位被指控放大极端主义的 CEO 合作
- **轨道数据中心的争议**: 对于在太空建设数据中心的计划存在物理和经济可行性的质疑

### 4. 商业策略分析

- **IPO 时机**: 此时获得数十亿美元收入对 IPO 有利
- **竞争格局**: 有评论认为开源模型如 Gemma 和 Qwen 发展迅速，Anthropic 面临长期价值风险
- **"亲吻戒指"**: 有评论认为对轨道计算的兴趣只是合作的一部分，不必认真对待
