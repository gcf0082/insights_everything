## 什么是 Transducers？

Transducers（转换器）是一种可组合的算法转换。它们独立于输入和输出源头的上下文，仅以单个元素的转换本质来定义转换规则。由于转换器与输入/输出源解耦，它们可以用于多种不同场景——集合、流、通道、观察者等。

## 核心术语

### Reducing Function（归约函数）
传递给 **reduce** 使用的函数类型，接收累积结果和新输入，返回新的累积结果：
```
whatever, input -> whatever
```

### Transducer（转换器）
将一个归约函数转换为另一个归约函数：
```
(whatever, input -> whatever) -> (whatever, input -> whatever
```

## 使用 Transducers

### 创建 Transducer
大多数序列函数省略输入集合时会返回一个 transducer：

```clojure
(filter odd?)  ;; 返回过滤奇数的 transducer
(map inc)      ;; 返回递增映射的 transducer
(take 5)       ;; 返回取前5个元素的 transducer
```

### 组合 Transducers
使用 **comp** 函数直接组合：

```clojure
(def xf
  (comp
    (filter odd?)
    (map inc)
    (take 5)))
```
转换栈从左到右应用（先过滤，再映射，最后取元素）

### transduce
应用 transducer 到集合的主要方式：

```clojure
(transduce xform f coll)
(transduce xform f init coll)
```
立即（非惰性）地对集合进行归约

### eduction
捕获将 transducer 应用到集合的过程：

```clojure
(def iter (eduction xf (range 5)))
(reduce + 0 iter)
```

### into
将 transducer 应用于输入集合并构造新输出集合：

```clojure
(into [] xf (range 1000))
```

### sequence
从 transducer 应用创建序列：

```clojure
(sequence xf (range 1000))
```

## 创建自定义 Transducers

Transducer 的结构：
```clojure
(fn [rf]
  (fn ([] ...)
      ([result] ...)
      ([result input] ...)))
```

内部函数有3种arity：

- **Init** (arity 0): 调用嵌套 transform rf 的 init
- **Step** (arity 2): 标准归约函数，根据需要调用 rf
- **Completion** (arity 1): 产生最终值/刷新状态

### 早期终止

Clojure 提供以下机制：

- **reduced**: 接收值并返回"reduced"值，表示归约应停止
- **reduced?**: 检查值是否由 reduced 创建
- **deref** 或 **@**: 提取 reduced 值内的实际值

### 带归约状态的 Transducers

某些 transducer（如 take、partition-all）需要维护状态：

```clojure
(defn dedupe []
  (fn [xf]
    (let [prev (volatile! ::none)]
      (fn
        ([] (xf))
        ([result] (xf result))
        ([result input]
          (let [prior @prev]
            (vreset! prev input)
            (if (= prior input)
              result
              (xf result input))))))))
```

状态在 transducible process 启动时初始化，状态ful 交互包含在 process 上下文中。

## 支持 Transducer 的函数

以下函数在省略输入集合时产生 transducer：map, cat, mapcat, filter, remove, take, take-while, take-nth, drop, drop-while, replace, partition-by, partition-all, keep, keep-indexed, map-indexed, distinct, interpose, dedupe, random-sample

## 适用场景

Transducers 适用于：
- 需要对同一转换逻辑复用于不同数据源的场景
- 避免创建中间集合，提高内存效率
- 需要早期终止的流式处理
