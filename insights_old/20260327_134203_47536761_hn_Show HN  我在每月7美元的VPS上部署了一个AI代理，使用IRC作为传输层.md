## 项目概述

作者在每月7美元的VPS上部署了一个AI代理，使用IRC作为传输层。主要特点包括：

### 技术栈
- **双代理架构**：两个独立的代理运行在不同的服务器上
- **公共代理 (nullclaw)**：678KB的Zig二进制文件，仅使用~1MB内存，连接到Ergo IRC服务器
- **私有代理 (ironclaw)**：处理邮件和日程安排，仅通过Tailscale和Google的A2A协议访问

### 分层推理策略
- **Haiku 4.5**：用于对话响应（亚秒级响应，低成本）
- **Sonnet 4.6**：仅在需要使用工具时调用
- **每日费用上限**：2美元/天

### A2A穿透
私有端代理借用网关自己的推理管道，因此无论谁发起请求，都只有一个API密钥和一个计费关系。

### 访问方式
- Web客户端: https://georgelarson.me/chat/
- IRC客户端: irc.georgelarson.me:6697 (TLS), 频道 #lobby

## 热门评论总结

### 1. 模型选择讨论

**InitialPhase55** 质疑为何选择Haiku/Sonnet，指出有更便宜且性能相当的选择：
- Haiku 4.5: $1/M输入 / $5/M输出
- MiniMax M2.7: $0.30/M输入 / $1.20/M输出
- Kimi K2.5: $0.45/M输入 / $2.20/M输出

**nl** 推荐Xiaomi Mimo v2-Flash：在他个人的AI代理基准测试中，Mimo v2-Flash得分比Haiku高8%，耗时109秒仅花费0.003美元，而Haiku耗时262秒花费0.24美元。

**ruguo** 表示MiniMax M2.7在编码任务上表现不错，但Opus 4.6仍然领先一个层次。

### 2. IRC作为传输层的优势

**faangguyindia** 分享他也在编码代理中使用IRC：通过切换房间来进入不同的提示词，使用它作为远程控制来更改项目。

**AbanoubRodolf** 指出"房间即上下文"模式被低估了：
- 免费获得命名空间隔离，无需构建会话管理
- 客户端无关性：可以从终端、移动IRC客户端或网页客户端连接同一个代理
- 聊天记录使代理感觉"始终在线"

**stackghost** 解释了IRC的消息长度限制：RFC 1459规定消息不超过512字节，现代IRC实现通过IRCv3扩展支持最多8192字节的元数据。

### 3. 安全与隐私讨论

**chatmasta** 提出挑战：在私有服务器上放置一个CTF标志，看看nully是否能够访问。

**czhu12** 展示了他用 vibe coding 开发的类似项目：一个读取GitHub仓库的网站客服机器人。

**k2xl** 指出这种设计存在潜在问题：可能会被利用来分析漏洞或查找硬编码的密钥。

### 4. 成本与基础设施

**iLoveOncall** 质疑：既然使用的模型是Claude API而非自托管，那基础设施为什么这么重要？

**ekianjo** 认为LLM是关键元素，不是7美元的VPS——模型本身花费数十亿美元训练。

**selcuka** 反驳：LLM如今是商品，可以在不同提供商之间切换，而且作者有2美元/天的上限，每月60+美元可以用于升级VPS运行本地模型。

**chatmasta** 总结：关键（创新）元素是双层沙箱和代理间通信方法。

## 核心洞见

1. **双层架构创新**：使用公共/私有分离加上Tailscale和A2A协议是一种新颖的AI代理安全沙箱方案

2. **IRC作为传输层**：IRC提供了开箱即用的命名空间隔离、会话持久性和多客户端支持，是构建AI代理的可行选择

3. **分层推理成本优化**：使用轻量级模型处理简单任务，仅在需要时切换到更强大的模型，可以有效控制成本

4. **更便宜的替代方案存在**：MiniMax M2.7、Kimi K2.5、Xiaomi Mimo v2-Flash等模型可能以更低成本提供类似或更好的性能
