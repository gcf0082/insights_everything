## 项目概述

Sonar 是一个用于检查和管理本地主机端口上服务的 CLI 工具。作者受够了每次端口被占用时都要运行 `lsof -iTCP -sTCP:LISTEN | grep ...` 命令，然后花时间弄清楚是 Docker 容器还是其他工作树的孤儿开发服务器，于是开发了这款工具。

## 核心功能

### 1. 端口列表查看
- 显示所有监听端口
- 支持显示 CPU、内存、状态、运行时间等统计信息
- 支持按 Docker 容器、进程名等过滤
- 支持 JSON 输出格式
- 支持自定义列显示

### 2. 端口检查
- 查看指定端口的详细信息
- 显示完整命令、用户、绑定地址、CPU/内存/线程数、运行时间、健康检查结果

### 3. 进程管理
- 支持杀死进程（SIGTERM / SIGKILL）
- 支持停止所有 Docker 容器
- 支持按项目停止 Compose 服务

### 4. 日志查看
- Docker 容器：运行 `docker logs -f`
- 本地进程：通过 `lsof` 发现日志文件并跟踪
- 支持 macOS `log stream` 或 Linux `/proc/<pid>/fd`

### 5. 服务附加
- Shell 进入 Docker 容器
- TCP 连接

### 6. 实时监控
- 轮询检测端口变化，显示差异
- 显示实时资源统计
- 支持桌面通知

### 7. 依赖关系图
- 显示服务之间的连接关系

### 8. 配置文件
- 保存项目预期端口快照
- 检查预期端口是否运行
- 批量停止配置中的端口

### 9. 端口等待
- 阻塞直到端口接受连接
- 支持 HTTP 健康检查
- 支持自定义超时和轮询间隔

### 10. 端口映射
- 代理流量，将一个端口的服务映射到另一个端口

### 11. 空闲端口查找
- 查找下一个可用端口
- 支持范围查找
- 支持连续空闲端口查找

### 12. 其他功能
- 浏览器中打开端口
- 菜单栏应用（macOS，仅限 Swift 二进制）

## 支持平台

- **macOS**：使用 `lsof`
- **Linux**：使用 `ss`
- **Windows**：使用 `netstat`

## 安装方式

### Linux/macOS
```bash
curl -sfL https://raw.githubusercontent.com/raskrebs/sonar/main/scripts/install.sh | bash
```

### Windows (PowerShell)
```powershell
irm https://raw.githubusercontent.com/raskrebs/sonar/main/scripts/install.ps1 | iex
```

### Go 安装
```bash
go install github.com/raskrebs/sonar@latest
```

## 适用场景

1. 快速查看本地监听端口
2. 识别占用端口的进程
3. 区分 Docker 容器和本地进程
4. 查看 Docker 容器信息（镜像、Compose 项目等）
5. 实时监控端口变化
6. 批量管理开发服务
7. 脚本化服务启动等待

## 总结

Sonar 是一款功能丰富的本地端口管理 CLI 工具，特别适合需要频繁与多个本地服务（Docker 容器、本地开发服务器）交互的开发者。它将原本分散的多个命令行工具（lsof、docker、netstat 等）的功能整合在一起，提供统一的 CLI 界面，大大提升了开发效率。
