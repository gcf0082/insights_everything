## 洞察摘要

本文讨论了 macOS 27 中即将到来的网络协议变化，主要涉及 AFP (Apple Filing Protocol) 和 Time Machine 的支持变更。

### 主要内容

1. **AFP 协议废弃**: Apple 在 macOS Sequoia 15.5 中警告 AFP 客户端已被废弃，将在"未来版本的 macOS"中移除。虽然 Apple 未明确具体版本，但很可能在 macOS 27 中实施。

2. **Time Capsule 影响**: 原本通过 AFP 使用 Time Capsule 进行网络备份的用户将受到影响。Time Capsule 在 2018 年停售，但其 AFP 支持一直延续到 macOS 26 (Tahoe)。

3. **TLS 1.2 要求**: 另一个变化是要求连接某些服务器时必须使用至少 TLS 1.2。

4. **用户反应**: 社区对此反应不一，有人认为这是合理的安全升级（SMB1 有安全问题），但也有人批评 Apple 放弃旧有协议的惯例做法。

5. **替代方案**: 用户需要迁移到 SMB 协议，或考虑使用第三方 NAS 解决方案（如 Ubiquiti UNAS）。也有用户提及使用 Netatalk 作为 AFP 的开源替代。

### 社区观点

- **支持废弃**: 部分用户认为这是正确的决定，因为 AFP 已过时 10 年以上，继续维护增加安全风险
- **批评声音**: 有用户指出 macOS 的 SMB 支持仍然缓慢且存在问题，希望 Apple 能先修复 SMB 问题
- **Linux 对比**: 有评论提到 Linux 内核最近也在移除旧驱动，但这是因为可能存在安全漏洞
- **备份未来**: 有用户担心未来 Apple 会进一步放弃本地 Time Machine 备份，转向 iCloud 服务收入
