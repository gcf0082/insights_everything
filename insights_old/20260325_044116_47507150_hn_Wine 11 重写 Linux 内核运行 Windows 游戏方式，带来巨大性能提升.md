## 内容摘要

Wine 11 是一个重大更新，它重写了 Wine 在 Linux 上运行 Windows 游戏的方式，引入了 ntsync 内核支持，带来了显著的性能提升。

### 性能基准数据

文章中展示的 benchmark 数据引发了广泛关注：

- Dirt 3: 从 110.6 FPS 提升到 860.7 FPS
- Resident Evil 2: 从 26 FPS 提升到 77 FPS
- Call of Juarez: 从 99.8 FPS 提升到 224.1 FPS
- Tiny Tina's Wonderlands: 从 130 FPS 提升到 360 FPS

### 重要澄清

需要注意的是，这些数字是 Wine+ntsync 与 Wine（未使用任何同步原语）的对比结果。相较于 Wine+fsync（大多数 Linux 游戏玩家已经在使用的方案），性能提升相对温和，通常只有几个百分点。

### 技术背景

- **ntsync**: Linux 内核新增的同步原语支持，更接近 Windows NT 内核的语义
- **fsync/esync**: Wine 之前使用的同步方案，基于 Linux 的 futex
- **Valve**: 大幅资助了 Proton/Wine 的开发，推动了这些改进

## 热门评论观点

### 1. 对 Wine 项目的敬意

> "Wine 是一个让我产生无限敬意项目。试图精确复现 Windows 过去 30 年的所有行为（包括文档化和未文档化的行为）听起来并不有趣，但正是这种对各种边缘情况的处理使得 Wine 成为可行的产品。"

### 2. Wine 运行效果优于 Windows

> "Wine 实际上在一些古老的 Windows 游戏上运行得比 Windows 本身更好。"
> "16 位 Windows 游戏在 Wine 上运行得比 Windows 11 更好，因为 Windows 11 根本无法运行它们。"

### 3. 关于技术深度的讨论

> "阅读这些帖子总是让我觉得自己是个骗子。人们在处理如此底层的东西，而我却在搭建简单的 CRUD 应用。"

这引发了关于底层编程与高级编程各有挑战的讨论。

### 4. Wine 对 Linux 桌面发展的影响

> "这似乎更有可能发生：Windows API 将成为事实上的 Linux 游戏 SDK，为游戏制作 Linux 原生版本的想法将变得毫无意义。"

### 5. 对性能提升的理性看待

> "ntsync 的性能提升（除了少数例外）通常是轻微的，通常在低个位数百分比范围内。这些极端提升是相对于不使用 fsync 的原生 Wine 进行测试的结果，任何在 Linux 上玩 demanding 游戏的人都会使用 fsync。"

### 6. 测试套件的重要性

> "Wine 有大量的测试套件在各个平台上运行以检查兼容性——https://test.winehq.org/data/。这些测试是 Wine 拥有良好兼容性的主要原因。"

## 总结

Wine 11 引入的 ntsync 支持代表了 Linux 游戏的重要进步。虽然原始基准数据看起来令人印象深刻（8 倍提升），但实际用户体验的提升可能较为温和，尤其是对于已经使用 fsync 的用户。这一改进彰显了 Wine 项目三十年来在 Windows API 兼容 性方面的深厚积累，以及 Valve 对开源游戏兼容性的持续投资。
