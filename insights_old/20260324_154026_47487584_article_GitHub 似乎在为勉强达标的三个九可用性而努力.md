## 文章摘要

GitHub 在2026年2月初经历了严重的服务中断问题，涵盖 Actions、pull requests、notifications 以及 Copilot 等核心功能。这引发了关于其服务可用性是否达到"三个九"标准的讨论。

## 主要发现

### 1. 服务可用性问题
- GitHub 在2025年曾出现正常运行时间低于90%的情况
- 其企业云客户的SLA承诺为99.9%正常运行时间
- 距离行业"五九标准"（99.999%）仍有差距

### 2. 受影响的服务
- **GitHub Actions**: 持续数小时的故障
- **Pull Requests**: 无法正常处理
- **Notifications**: 延迟最高达50分钟
- **Copilot**: 策略传播故障，导致新启用模型无法访问

### 3. 行业现状
- 云服务中断频繁发生
- 部分供应商甚至难以维持90%的正常运行时间
- 客户需要为服务中断做好规划

---

## 相关背景

- GitHub 近期修改了状态页面，使其更难直观了解服务可用性
- 第三方通过公共状态订阅重建了非官方的状态页面
- Zig 语言已宣布退出 GitHub，称微软的 AI 优先策略破坏了服务体验

---

## 结论

GitHub 作为全球最大的代码托管平台，近期频繁的服务中断凸显了云服务稳定性的重要性。对于依赖这些平台的企业和开发者而言，需要做好应对服务中断的预案，而非单纯依赖服务商的可用性承诺。
