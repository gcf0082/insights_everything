## 文章总结

### 核心观点

GitHub曾经很好，但现在成了一个"无聊的垃圾墓地"（lame slop graveyard）。微软收购GitHub后，开始走下坡路，经历了典型的"恶化"（enshittification）过程。

### 主要问题

1. **服务质量下降**：官方 uptime 数据持续下降，非官方统计显示情况更糟糕
2. **Copilot 地狱**：微软强推 AI 功能，造成糟糕用户体验
3. **自DoS攻击**：GitHub 自己被垃圾内容淹没
4. **bot 和垃圾内容泛滥**：GitHub 充斥着机器人生成的垃圾内容

### 行业反应

多个人和项目已经开始迁移离开GitHub：
- Lonami: 放弃GitHub
- Ghostty: 离开GitHub
- Armin Ronacher: 怀念"前GitHub时代"
- Jonas Hietala: 迁移到Codeberg/Forgejo

### 核心论点

**Git不等于GitHub**：Git是开源的、分布式的技术，不依赖中心化服务。GitHub只是一个"社交便利"的产物，现在变成了昂贵的负担。

### 替代方案

**集中式托管服务**：
- **Codeberg**: 非营利、社区主导，基于Forgejo，最安全的替代选择
- **Tangled**: 处于alpha阶段的初创公司，集成AT协议
- **Gitea**: 开源项目，提供云托管服务
- **GitLab**: 企业级，功能全面但复杂
- **Bitbucket**: 不推荐，仅仅是"另一个坑"

**自托管方案**：
- 推荐 **Forgejo**，可以自托管CI runner和发布功能
- 也可以考虑Gitea、GitLab的自托管版本

### 建议

不要一次性迁移所有内容，可以逐步进行。简单使用SSH + Git即可：
```bash
git clone user@192.168.1.67:/path/to/repo
```

### 作者立场

作者认为即使其他中心化Git托管服务将来可能也会崩溃，但目前应该"anything but GitHub"（除了GitHub什么都可以）。
