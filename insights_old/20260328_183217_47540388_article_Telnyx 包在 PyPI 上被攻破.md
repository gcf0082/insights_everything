# Telnyx Python SDK 供应链安全通知

**洞察链接**: https://telnyx.com/resources/telnyx-python-sdk-supply-chain-security-notice-march-2026

**来源**: Telnyx 官方博客  
**发布日期**: 2026年3月  
**类型**: 安全公告

---

## 摘要

2026年3月27日03:51:28 UTC，两个未经授权的 Telnyx Python SDK 版本被发布到 PyPI：4.87.1 和 4.87.2。这两个版本均包含恶意代码，并于当天 10:13 UTC 被隔离删除。

此事件是更广泛的供应链攻击的一部分，还影响了 Trivy、Checkmarx 和 LiteLLM。

**Telnyx 平台、API 和基础设施未受到影响。** 此事件仅限于 Python SDK 的 PyPI 分发渠道。

## 受影响版本

| 版本 | 发布时间 |
|------|----------|
| telnyx==4.87.1 | 2026年3月27日 03:51:28 UTC |
| telnyx==4.87.2 | 紧随其后 |

两个版本均已从 PyPI 中删除。

## 可能受影响的人群

以下情况可能受到影响：

- 在 2026年3月27日 03:51 UTC 至 10:13 UTC 期间安装或升级了 telnyx Python 包
- 运行 `pip install telnyx` 时未指定版本，并收到了 4.87.1 或 4.87.2
- 项目中的依赖项将 telnyx 作为传递性、未固定的依赖项引入

## 不受影响的人群

- 使用 4.87.0 或更早版本
- 在 2026年3月27日 03:51 UTC 至 10:13 UTC 期间未安装或升级 telnyx 包
- 直接使用 Telnyx REST API 而未使用 Python SDK

## 如何检查

运行以下命令：

```bash
pip show telnyx
```

**如果显示的版本是 4.87.1 或 4.87.2，请将该环境视为已受感染。**

## 如果受影响该怎么办

1. **立即降级**：
   ```bash
   pip install telnyx==4.87.0
   ```

2. **轮换所有可从受感染环境访问的凭据**：
   - API 密钥
   - 数据库凭据
   - 云提供商令牌
   - SSH 密钥
   - 存储在环境变量或配置文件中的任何密钥

3. **审计系统以检测与攻击者基础设施的出站连接**（参见下面的 IOC）

4. **审查可能已拉取受感染版本的 CI/CD 管道和 Docker 构建**

## 妥协指标 (IOC)

| 类型 | 值 |
|------|-----|
| C2 服务器 | 83.142.209.203:8080 |
| 数据泄露技术 | WAV 隐写术有效载荷传递 |

更多 IOC 将在调查确认后公布。

## 什么未受影响

Telnyx 平台、语音服务、消息传递基础设施、网络、SIP、AI 推理和所有生产 API 均未受到影响。

SDK 是一个封装公共 API 的客户端库，对 Telnyx 基础设施没有特权访问权限。没有任何客户数据通过此事件被访问。

## 正在采取的措施

- 从 PyPI 中删除恶意版本
- 调查发布凭据是如何被获取的

## 相关事件

此攻击是为期数周的供应链活动的一部分：

- **Trivy 事件** (2026年3月19日)
- **LiteLLM 事件** (2026年3月24日)

## 建议的安全措施

1. **始终固定依赖版本**：使用 `pip install telnyx==4.87.0` 而非 `pip install telnyx`
2. **定期审计依赖**：检查项目中的所有依赖项是否安全
3. **实施软件供应链安全最佳实践**：包括签名验证、哈希验证等
4. **监控异常网络活动**：关注与未知 C2 服务器的连接

## 结论

此次事件提醒我们软件供应链安全的重要性。即使是知名的 SDK 提供商也可能成为供应链攻击的目标。开发者应始终保持警惕，使用版本固定，并定期审查依赖项的安全性。
