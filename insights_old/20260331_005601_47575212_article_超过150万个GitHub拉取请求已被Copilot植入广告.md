## 核心洞察

微软Copilot正在向GitHub和GitLab的拉取请求（Pull Request）中注入广告。据墨尔本软件开发者Zach Manson发现，Copilot在修复一个简单拼写错误时，自动在PR描述中添加了Raycast的推广信息："⚡ Quickly spin up Copilot coding agent tasks from anywhere on your macOS or Windows machine with Raycast."

在GitHub上搜索该短语发现，相同的推广文本出现在超过**11,000个**拉取请求中，跨越数千个代码仓库。甚至GitLab上的合并请求也未能幸免。

## 事件详情

### 广告注入方式
- 受影响PR的原始markdown中存在隐藏HTML注释："**START COPILOT CODING AGENT TIPS**"
- 这表明微软正在利用该注释插入指向其开发者生态系统的"提示"

### 广告内容类型
1. **Raycast推广**：推荐使用Raycast的Copilot扩展
2. **Slack/Teams集成**：发送任务到Copilot编程代理，将对话转化为代码
3. **编辑器推广**：推荐从VS Code、Visual Studio、JetBrains IDEs和Eclipse启动Copilot编程代理

### 影响规模
- 超过**150万个**GitHub拉取请求被注入广告

## 行业背景

生成式AI领域投资与收入存在巨大差距：
- AI数据中心投资与实际收入之间存在超过**4000亿美元**的缺口
- 硅谷重新转向广告盈利模式
- OpenAI于2025年1月为免费和Go用户推出广告，声称其广告业务在六周内实现了超过**1亿美元**的年度化收入
- OpenAI计划下月推出自助广告平台，并扩展到加拿大、澳大利亚和新西兰

## 总结

微软Copilot通过其编程代理在代码协作平台中植入广告，这是AI公司寻求变现的典型案例。尽管用户每月支付20美元订阅费，但仍无法覆盖推理计算的高昂成本，最终导致广告介入用户体验。
