## 一、事件概述

2026年3月31日，OpenAI 发现其 GitHub Actions 工作流在 macOS 应用签名过程中下载并执行了被植入恶意代码的 Axios 库（版本 1.14.1）。此次事件是针对 npm 生态系统的广泛供应链攻击的一部分，攻击者据称与朝鲜有关联。

**关键发现：** OpenAI 明确表示，未发现任何证据表明用户数据被访问、系统或知识产权被泄露，或软件被篡改。

---

## 二、事件经过

1. **2026年3月31日**，Axios（广受欢迎的 JavaScript HTTP 请求库）被攻击者入侵，成为更广泛软件供应链攻击的一部分。攻击者获取了 Axios 维护者的账户权限，发布了两 个恶意版本：1.14.1 和 1.14.2。

2. 恶意版本注入了 `plain-crypto-js` 依赖，该依赖作为 **WAVESHAPER.V2 后门** 的投放器（dropper），可感染 macOS、Windows 和 Linux 设备。

3. 恶意版本在约三小时后被移除。

4. OpenAI 的 GitHub Actions 工作流（用于 macOS 应用签名流程）在此期间下载并执行了恶意的 Axios 1.14.1 版本。

5. 该工作流具有访问用于签署 macOS 应用程序的证书和公证材料的权限，涉及 ChatGPT Desktop、Codex、Codex-cli 和 Atlas。

---

## 三、影响范围

### 受影响范围
- macOS 桌面应用程序的签名流程
- 代码签名证书和公证材料面临潜在暴露风险
- 受影响产品：ChatGPT Desktop、Codex、Codex-cli、Atlas（macOS 版本）

### 不受影响范围
- iOS、Android、Windows、Linux 用户
- Web 版本应用
- 用户数据和密码
- OpenAI API 密钥
- OpenAI 内部系统和知识产权

---

## 四、OpenAI 响应措施

1. **证书轮换**：OpenAI 已撤销并轮换受影响的签名证书，为所有 macOS 应用程序颁发新证书。

2. **强制更新**：要求所有 macOS 用户通过官方渠道或应用内更新到最新版本。

3. **工作流修复**：解决了 GitHub Actions 工作流中的错误配置问题。

4. **终止旧版本支持**：自 2026 年 5 月 8 日起，使用旧证书签署的 macOS 应用程序将不再收到更新或支持，并可能无法正常运行。

5. **安全公告**：OpenAI 于 2026 年 4 月 10 日公开发布安全公告，透明披露事件详情。

---

## 五、安全分析与结论

根据 OpenAI 的分析，由于以下缓解因素，签名证书可能未被恶意有效载荷成功窃取：
- 恶意有效载荷执行的时间
- 证书注入到作业中的时机
- 作业本身的执行顺序

尽管如此，OpenAI 出于高度谨慎，仍将证书视为已泄露并采取了完整的轮换措施。

**关键教训：** 开发者工具、代码签名和构建管道现在处于同一攻击面。这次事件再次警示：npm 生态系统中的单个恶意包可以从开源进入 CI/CD，最终影响应用的发布流程——一个微小的依赖项可能成为打开安全防线的钥匙。

---

## 六、建议措施

| 用户类型 | 建议操作 |
|----------|----------|
| macOS 用户 | 立即更新 OpenAI 应用至最新版本 |
| 开发者 | 检查 CI/CD 流程中的第三方依赖，强化供应链安全审计 |
| 企业用户 | 审查内部开发工作流，实施依赖签名验证（如 npm audit、sigstore） |
