## 文章摘要

本文是 Drew DeVault 为 Vim 写的悼词，分享了他对 Vim 的深厚感情、对已故 Vim 作者 Bram Moolenaar 的敬意，以及他决定 fork Vim 的原因。

## 核心要点

### 1. Vim 与作者的深厚渊源

- Drew DeVault 使用 Vim 撰写了他博客上的所有文章、代码、文档和邮件
- 他与 Vim 的关系如同身体的一部分，所有的模式和快捷键都已深深印入肌肉记忆
- 他在手腕上纹了 "hjkl" 以纪念这款编辑器

### 2. 对 Bram Moolenaar 的悼念

- Drew 曾在 Bram 的祖国荷兰居住
- Bram 去世后，葬礼在离他住处不远的地方举行
- Bram 最重要的遗产是为乌干达儿童提供教育和医疗的慈善事业

### 3. 对 AI/GenAI 的强烈批评

- AI 导致贫困社区能源价格上涨
- 破坏野生动物和淡水资源供应
- 增加污染，加剧全球供应链紧张
- 刚果等非洲国家的矿工为供应 AI 芯片所需的钴等稀有金属而承受恶劣工作条件
- AI 数据中心消耗全球 1.5% 的能源
- AI 还助长了全球法西斯主义的抬头

### 4. 为什么要 Fork Vim

- Vim 和 NeoVim 都在使用 LLM 开发软件
- 作者不希望使用包含"slop"（垃圾内容）的软件
- 为了保持对 Vim 的纯粹使用体验，决定创建分支

### 5. Vim Classic 分支详情

- **基于版本**: Vim 8.2.0148（Vim9 Script 引入之前的版本）
- **选择原因**: 保留与旧插件的兼容性
- **维护计划**: 缓慢而安静的维护
- **接受补丁**: 特别是运行时脚本的语言特性更新、bug 修复等

---

## 结论

Drew DeVault 通过创建 Vim Classic 分支，表达了他对纯粹编辑器的坚持，以及对 AI 技术侵入开源软件的担忧。这是一个关于软件哲学和价值观的有趣思考。
