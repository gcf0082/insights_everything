## 内容摘要

本文作者讲述了一次被欺骗的经历：他为一个初创公司开发增强现实（AR）巴士项目，工作了数月却最终未能收到约35,000美元的报酬。作者在项目中投入了大量时间和精力，甚至错过了与两岁孩子相处的宝贵时光，最终却遭遇公司倒闭、资金链断裂的结局。

## 主要讨论主题

### 1. 核心教训：收款或停工

讨论中最受认可的建议是：**要么拿到报酬，要么停止工作**。多位评论者强调，如果发票逾期，就不应继续工作。可以友好地与客户沟通，但必须坚持付款后才能继续工作。

### 2. 付款条款的重要性

- 在项目开始前充分谈判合理的付款条款
- 如果不信任客户，要求预付款
- 避免与对付款犹豫不决的公司合作
- 考虑基于里程碑的付款方式，将大额款项拆分为小额分期

### 3. 合同条款建议

一位用户分享了其公司使用的合同条款范本：
- 发票收到后X天内付款，或解决质量问题后立即付款
- 逾期付款需支付利息和滞纳金
- 逾期时下一期发票立即到期
- 保留在逾期情况下调整交付期限的权利
- 不分享可下载的交付物链接，直至款项付清

### 4. 客户筛选

评论者普遍认为：
- 好的客户通常在2-3天内付款
- 难缠的客户付款周期很长
- 不愿接受合理合同条款的客户可能也不会付款
- 应避免与政治组织和教会等客户，除非预付款

### 5. 小额诉讼的价值

在英国，低于10,000英镑的债务可以通过简化程序追讨，成本约80英镑。发送"行动前信函"通常能立即促使对方付款。

### 6. 关于Y Combinator创始人的警告

有评论者警告：孵化器类型的初创公司往往是最差的客户，他们有资金但经常忘记付款或不尊重合同。

## 关键要点总结

| 主题 | 建议 |
|------|------|
| 付款原则 | 收款或停工，不要白干活 |
| 合同保护 | 设定严格的付款条款和逾期处罚 |
| 风险控制 | 要求预付款或里程碑付款 |
| 客户选择 | 筛选优质客户，避免不良客户 |
| 交付策略 | 不付款就不交付成果 |
| 法律途径 | 考虑小额诉讼追讨欠款 |

## 结论

这次讨论的核心教训是：**在自由职业和合同工作中，付款保护至关重要**。即使项目看起来很有前景，即使客户承诺会付款，也必须建立完善的付款机制来保护自己。合同本身不能保证收款，真正有效的是在交易中控制风险——通过预付款、里程碑付款、停止交付等方式确保自身利益。
