## 洞察摘要

### 漏洞概述

安全研究人员发现了一个名为 "YellowKey" 的零日漏洞，该漏洞允许攻击者仅通过在 USB 驱动器上放置特定文件即可绕过 Microsoft BitLocker 加密。值得注意的是，当漏洞被触发后，这些文件会自动从 USB 设备中删除，增加了溯源难度。

### 关键发现

1. **攻击方式简单**：只需将特定文件放入 USB 驱动器的特定位置，然后启动计算机即可绕过 BitLocker 加密
2. **类后门特征**：漏洞具有所有后门特征，包括特定文件名触发机制和利用后自动删除文件
3. **影响范围**：主要影响 Windows 11 系统，可能影响所有仅使用 TPM 保护的 BitLocker 配置
4. **微软历史问题**：此前的 RedSun 和 Bluehammer 漏洞已被微软"静默修复"，未对研究人员的 CVE 报告做出回应

### 社区讨论焦点

- **是否为后门**：多数评论者认为这极不可能是"漏洞"，更可能是故意植入的后门
- **TPM+PIN 保护是否有效**：漏洞开发者声称 TPM+PIN 保护也可被绕过，但暂无公开概念验证
- **缓解建议**：使用 BitLocker PIN 认证可能提供一定保护，但研究者对此持不同看法
- **安全性信任问题**：许多用户对微软的加密实现失去信心，转向使用 VeraCrypt 等开源替代方案
