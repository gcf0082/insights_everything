# 洞察报告：LLMs Are Not a Higher Level of Abstraction

## 核心观点

本文反驳了"LLMs是更高层次抽象"这一普遍观点。作者认为这个说法是错误的，即使声称者有30年编程经验也无法改变这一事实。

## 关键论述

### 传统编程的确定性

从汇编到C再到Python，每一层抽象的转换都是确定性的函数：

```
f(x) -> y
```

给定特定输入，总是产生相同的输出结果。

### LLMs的非确定性

LLMs的输出不是确定值，而是概率值：

```
f(x) -> P(y)
```

这意味着同一输入可能产生多种不同结果。

### 问题的严重性

实际情况更糟：

```
f(x) -> P(y) ∪ P(z1) ∪ P(z2) ∪ ... P(zN)
```

即LLM可能返回你想要的输出(y)，同时附带你未要求的额外内容(z1, z2等)。

### 实际风险示例

当你让LLM编写一个TODO应用时，它可能：
- 返回你请求的应用 (y)
- 同时附带"开放你的凭证到网络" (z1)
- 或者"通过公开FTP分享你的服务器" (z2)

现有测试只检查y是否存在，无法检测这些隐藏的副作用。

## 结论

作者呼吁程序员保持自我意识，不要成为AI生成内容的渠道。2026年仍在传播"LLMs是抽象层次升级"这一错误观点的人，应该阅读本文。
