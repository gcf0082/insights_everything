## 摘要

VitruvianOS 是一款受 BeOS 启发的桌面 Linux 发行版，它提出了一个独特的问题：用户目前有哪些想用电脑做却做不到的事情？该系统具有以下特点：

- 自定义图形堆栈（非 X11 或 Wayland）
- 实时补丁 Linux 内核，用于低延迟桌面使用
- Nexus 内核桥接器，可在标准 Linux 内核上运行 Haiku 应用程序

## 主要讨论话题

### 1. BeOS 怀旧情结

许多用户分享了他们对 BeOS 的美好回忆：

- BeOS 能够在多个窗口同时播放媒体，每个窗口都能独立播放音频
- 窗口标签分组功能将不同应用的标签合并到单一窗口中，是最佳的用户体验功能
- BeOS 独特的黄色标题栏、可移动的对角图标和可放置在屏幕角落的任务栏设计
- 比当时的 Windows 98SE 更稳定、更快

### 2. Haiku OS 作为替代品

讨论中多次提到 Haiku（BeOS 的现代开源实现）：

- Haiku 是基于 BeOS 的非常不错且可用的系统
- 可以作为日常使用，在老旧硬件上运行良好
- 但开发进展太慢，多年来未能真正离开"梦想时代"
- Ruby 等编程语言在 Haiku 上无法正常工作

### 3. VitruvianOS 与 Haiku 的比较

- VitruvianOS 是 Linux，拥有 Linux 提供的所有支持
- 可以运行现代 Linux 原生二进制文件，同时拥有 BeOS 风格的窗口管理器和 Tracker
- 但无法运行 X11 或 Wayland 应用程序
- Haiku 能运行真正的 BeOS 系统，而不是兼容性层

### 4. 关于实时内核的质疑

- 实时补丁集是否有意义存在疑问
- 对于桌面应用来说可能没有太大区别
- 即使有音视频延迟需求，普通内核在很多情况下也能提供更低的延迟

### 5. 桌面体验讨论

- 用户怀念能够从窗口左上角任意位置拖动窗口的功能
- 在许多 Linux 桌面环境中，可以通过按住 Windows 键 + 鼠标左键拖动来实现
- BeOS 的设计在当时是独特的，现在又重新流行起具有深度感的 UI 元素

### 6. 历史"如果"讨论

- 苹果当年在收购 BeOS 和 NeXT 之间犹豫不决
- 如果选择 BeOS 而不是 Unix 路线，macOS 会是什么样子？
- HP 等供应商放弃预装 BeOS 的计划，转向 Wintel

## 关键观点

1. **实用性 vs 理论优势**: Linux 证明了实际考虑胜于理论优势，除了桌面领域 Linux 一直表现不佳

2. **兼容性挑战**: VitruvianOS 无法运行现有的 X11/Wayland 应用程序，这是其面临的主要挑战

3. **怀旧价值**: BeOS 的设计哲学仍然吸引着众多用户，其独特的用户体验功能（如窗口标签分组）至今仍被怀念

4. **发展困境**: Haiku 和 ReactOS 等项目长期以来停留在虚拟化阶段，未能真正在真实硬件上成为主流
