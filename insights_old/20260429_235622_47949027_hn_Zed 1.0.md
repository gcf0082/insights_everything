## 内容摘要

Zed 编辑器发布 1.0 正式版本，这是来自 Zed Industries 的高性能代码编辑器。帖子引发了大量关于编辑器体验、功能和局限性的讨论。

## 主要讨论话题

### 1. PHP 项目 LSP 警告问题（热门话题）

**问题描述**：
- 用户反映 Zed 对旧版 PHP 项目（2010-2020年代）显示大量警告
- 函数缺少返回类型等历史标准代码被视为错误
- 代码看起来像"红色的海洋"，难以阅读
- 关闭警告的配置过程复杂，需要手动编辑 LSP 配置文件

**用户建议**：
- 希望提供一键式功能，允许用户屏蔽特定类型的警告
- 建议 Zed 自动编辑项目配置来保存用户偏好

### 2. 搜索功能体验（重要话题）

**负面反馈**：
- 搜索时打开新标签页，而非使用模态框或侧边栏
- 搜索速度较慢
- 与 Vim/Helix/JetBrains 的 Telescope 风格搜索体验差距明显
- 引用搜索也在同一文件内打开新标签页

**正面评价**：
- 部分用户喜欢这种设计，可通过 Ctrl+O 返回上一个标签页
- 多缓冲区结果适合"动手"搜索和替换

### 3. 主题和 UI 问题

**反馈意见**：
- 默认主题配色对比度不足，灰色文本配灰色背景
- 图标太小
- 主题命名不够直观（R、C、Go等难以搜索）

**解决方案**：
- 可选择其他主题或自定义主题
- Zed 提供官方主题编辑器

### 4. 其他技术问题

- **Linux 非拉丁键盘布局**：快捷键在非拉丁键盘上无法正常工作
- **LSP 信任模式**：未信任项目目录时，语言服务器功能受限
- **内存占用**：远程开发时内存使用较高

### 5. 正面评价

- **性能优秀**：启动快、响应迅速
- **原生体验**：完全原生实现，无 Electron 负担
- **AI 集成**：Claude Code/Codex 集成使用体验良好
- **远程开发**：SSH 远程编辑功能稳定
- **开放源码**：GPUI UI 库已开源
- **安装便捷**：二进制体积小（Linux x86_64 版本约 136MB）

### 6. 竞品对比

- **Sublime Text**：用户认为 Zed 是"现代化版 Sublime"，但部分老用户仍偏好 Sublime 的操作手感
- **VS Code**：被认为越来"臃肿"，但功能更成熟
- **JetBrains**：功能全面但资源占用大

## 社区情绪

整体情绪偏正面，开发者认可 Zed 团队的工作成果和技术创新。主要抱怨集中在：
1. 用户体验细节（搜索、警告配置）
2. 平台兼容性（Linux 支持）

## 结论

Zed 1.0 标志着这一新兴编辑器进入成熟阶段。其核心优势在于性能、AI 集成和现代化设计，但在某些传统编辑器功能（如搜索 UI、LSP 配置便捷性）上仍有改进空间。对于寻求高性能编辑器的开发者，Zed 是值得关注的选择。
