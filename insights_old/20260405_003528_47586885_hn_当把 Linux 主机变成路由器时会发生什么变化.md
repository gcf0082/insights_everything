## 摘要

本文讨论了将 Linux 主机转变为路由器时需要进行的7项配置变更。主要涵盖网络转发、桥接、IP伪装等关键技术点。

---

## 关键讨论要点

### 1. Linux 路由转发机制

当执行 `sysctl net.ipv4.ip_forward=1` 时，Linux 立即转变为路由器，因为 FORWARD 表的默认策略是 ACCEPT。用户需要显式配置 iptables/nftables 来阻止非预期的流量转发。

### 2. 容器化软件的安全隐患

LXD/Incus 等容器化软件在安装/启动时自动启用转发功能，但未配置防火墙阻止非容器流量，导致机器成为开放路由器。开发者表示这是"设计如此"，但这与其他虚拟化系统的做法不同。

### 3. WiFi 与以太网桥接的限制

在 802.11 基础设施模式下，工作站只能发送具有自己 MAC 地址的帧，AP 不会接受或转发来自未知 MAC 的帧。因此无法透明桥接以太网设备的 MAC 地址通过 WiFi 客户端接口，需要使用 hostapd。

### 4. 路由器与 CPE 的区别

讨论了"路由器"与"CPE（客户 premises 设备）"的术语区别。路由器是连接多个网络并转发数据包的设备，而 CPE 通常指用户端设备，如调制解调器、路由器、机顶盒等。

### 5. IPv6 配置要点

- 启用 IPv6 转发：`net.ipv6.conf.all.forwarding=1`
- nftables 默认只适用于 IPv4，需设置为 `inet` 模式以同时支持 IPv4 和 IPv6
- dnsmasq 可提供路由通告和 SLAAC，部分设备（如 Android）仅支持 SLAAC

### 6. 替代解决方案

| 方案 | 特点 |
|------|------|
| **VyOS** | 类似 Arista/Cisco/Juniper 的路由器操作系统 |
| **OpenWrt** | 主要面向嵌入式 WiFi 设备，x86 版本分区较小 |
| **OPNsense/pfSense** | 基于 FreeBSD |
| **OpenBSD** | 配置更简单，IPv6 开箱即用 |
| **Ipfire/Untangle** | 适合 x86 机器 |

### 7. 虚拟化 vs 专用设备

- **虚拟化方案**: 在 Proxmox 等平台上运行路由器虚拟机，通过 PCI 直通分配网卡
- **专用设备**: 建议将路由器作为独立设备，与其他服务隔离，避免单点故障

---

## 安全建议

1. 启用 IP 转发时，同时配置 iptables/nftables 防火墙规则
2. 启用 NAT (IP 伪装) 以实现双向通信
3. 谨慎使用 LXD/Incus 等自动启用转发的软件
4. 定期检查路由配置，确保无意外流量转发
