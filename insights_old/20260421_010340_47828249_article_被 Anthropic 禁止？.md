## 网站概述

Banned By Anthropic 是一个独立的请愿网站和公共事件记录平台，旨在推动 Anthropic 公司对其账号封禁决定进行人工审核、提供公平的上诉渠道，并恢复合法用户的账号访问权限。

该网站记录了与 Anthropic 账号封禁、限制和访问丢失相关的事件，特别关注与 OpenClaw 等第三方工具相关的用户遭遇。

## 背景事件

### OpenClaw 事件时间线

| 时间 | 事件 |
|------|------|
| **2026年1月9日** | Anthropic 悄然部署服务器端检查，禁止第三方工具使用 Claude 订阅的 OAuth 令牌。OpenCode、OpenClaw、Clawdbot、Roo Code、Zed 等数十款开发工具一夜之间认证失败。 |
| **2026年2月17日** | Anthropic 正式发布文档，明确声明："使用通过 Claude Free、Pro 或 Max 账户获得的 OAuth 令牌用于任何其他产品、工具或服务（含 Agent SDK）均不被允许，违反《消费者服务条款》"。 |
| **2026年4月4日** | Anthropic 正式切断 Claude 订阅与 OpenClaw 的关联，所有订阅用户收到通知。 |
| **2026年4月8日** | Anthropic 开始大规模封禁 OpenClaw 用户的 Claude 账号，而不仅仅是 API 密钥。数以千计的活跃项目受到影响。 |
| **2026年4月** | 出现多起针对整个公司（60人团队）的集体封禁事件，无警告、无解释。 |

## 总结

Banned By Anthropic 网站反映了开发者社区对 Anthropic 在封禁账号问题上缺乏透明度和不公平上诉程序的不满。该事件被视为 AI 领域"vendor lock-in"（供应商锁定）的典型案例，特别是：

1. **无预警执行**: 2026年1月9日的 OAuth 禁令没有任何预先通知
2. **自动封禁系统**: 触发滥用过滤器的账号被即时自动封禁
3. **静默限制**: 用户遭遇"静默限制"（账号功能受限但未收到正式通知）
4. **上传下达**: 整个公司或团队被集体封禁，无个别审核

开发者社区正在通过请愿、Fork 和迁移到其他提供商等方式表达抗议。
