## 项目概述

这是一个创新的Web应用，名为 **Printervention**，通过在浏览器中运行Linux虚拟机来连接老旧打印机。具体实现方式：

- 使用 **v86**（x86模拟器，在JS/WASM中运行）来在浏览器中运行一个精简的Linux系统
- 通过 **WebUSB** API 访问连接到用户电脑的USB打印机
- 使用 **USB/IP** 协议将WebUSB设备桥接到虚拟机内部
- 在虚拟机中运行 **CUPS**（通用Unix打印系统）来处理打印任务
- 目前仅支持Chrome浏览器（其他浏览器不支持WebUSB）

作者表示这个方案是一个"更通用的解决方案"，避免了重新发明轮子。

## 讨论要点

### 1. 技术方案讨论

**替代方案提议**：有评论者提出可以使用LLM来反编译CUPS驱动或捕获USB流量，然后用Go等语言重写一个原生应用，而不是运行整个虚拟机。作者认为他的方案更具通用性。

**树莓派方案**：有人分享了使用Raspberry Pi作为打印服务器的方案，通过USB/IP共享打印机。但作者提到成本和额外线路是障碍。

**Docker转WASM**：讨论了是否有Docker到WASM的流水线，以及与x86模拟的区别。

### 2. 打印机驱动现状

- **Gutenprint**：该项目的MacOS支持已于2024年7月7日正式弃用，没有活跃的MacOS维护者
- **CUPS历史**：Apple曾长期支持CUPS，但于2019年放弃，目前由OpenPrinting社区维护
- **厂商态度**：Xerox删除了大部分旧打印机的Linux驱动，Canon、Brother、HP等厂商仍在基于20年前的代码生产新设备

### 3. WebUSB标准化争议

**Apple的立场**：Apple阻止WebUSB标准化，评论者认为这是为了保护其原生应用商店的业务（30%佣金）。Firefox也不支持WebUSB。

**安全争议**：
- 支持者：用户需要明确授权每个网站访问USB设备，与摄像头/麦克风权限类似
- 反对者：WebUSB暴露了更底层的USB访问权限，可能导致硬件损坏或数据窃取

### 4. 硬件兼容性案例

- 古老的Canon Selphy照片打印机在Windows 7 64位驱动下可以正常工作
- Samsung激光打印机可以通过CUPS添加AirPrint支持
- Epson MX80针式打印机可以通过ESP32实现网络打印

## 总结

这个项目展示了利用现代Web技术（WebAssembly、WebUSB）来解决旧硬件兼容性问题的一种创新方法。虽然方案较为"重型"（运行完整虚拟机），但它能够直接利用现有的Linux打印驱动生态，避免了为每种老打印机编写独立驱动的麻烦。

讨论也反映了当前打印机生态系统的困境：厂商缺乏动力为旧设备提供支持，而开源社区虽在努力维护，但资源有限。WebUSB等Web API的标准化进程受阻于商业利益权衡。
