## 一、收集遥测数据的目的

随着 GitHub CLI 代理功能的采用量不断增长，团队需要了解功能在实际使用中的表现。通过遥测数据来优先排序工作，并评估功能是否满足用户的真实需求。

例如：
- 当发布新的子命令时，了解是否有人使用以及如何使用
- 如果采用率低，需要重新审视功能的可发现性或设计
- 如果某个子命令配合特定标志使用频率高，则需要投资改善相应体验

## 二、如何查看遥测数据

GitHub CLI 是开源项目，可以在 [cli/cli](https://github.com/cli/cli) 仓库中查看遥测实现。如需查看实际发送的内容（不实际发送），可以启用日志模式：

**环境变量方式**：
```bash
export GH_TELEMETRY=log
```

**CLI 配置方式**：
```bash
gh config set telemetry log
```

日志模式下，会将 JSON payload 打印到 stderr，可检查每个字段后再决定是否保持遥测开启。

**示例输出**：
```json
{
  "events": [
    {
      "type": "command_invocation",
      "dimensions": {
        "agent": "",
        "architecture": "arm64",
        "command": "gh repo list",
        "device_id": "1e9a73a6-c8bd-4e1e-be02-78f4b11de4e1",
        "flags": "archived",
        "invocation_id": "eda780f5-27f9-433c-a7ae-7a033361e572",
        "is_tty": "true",
        "os": "darwin",
        "timestamp": "2026-04-16T14:55:13.418Z",
        "version": "2.91.0"
      }
    }
  ]
}
```

收集的维度包括：代理类型、架构、命令名称、标志、设备 ID、调用 ID、是否为 TTY、操作系统、时间戳和版本号。

## 三、如何选择退出遥测

**环境变量方式**（任意假值均有效：`0`、`false`、`disabled`、空字符串）：
```bash
export GH_TELEMETRY=false
```

或使用 `DO_NOT_TRACK` 约定：
```bash
export DO_NOT_TRACK=true
```

**CLI 配置方式**：
```bash
gh config set telemetry disabled
```

**注意**：环境变量优先于配置值。

## 四、数据发送位置

遥测事件发送至 GitHub 内部分析基础设施。更多信息请参阅 [GitHub 通用隐私声明](https://docs.github.com/en/site-policy/privacy-policies/github-general-privacy-statement)。

## 五、附加说明

- GitHub CLI 支持通过安装扩展（包括代理）来添加功能，这些扩展可能收集自己的使用数据，选择退出遥测无法控制
- 本页面描述的是 GitHub CLI（`gh`）的客户端数据收集，不适用于 GitHub Copilot 或 Copilot CLI（它们单独处理数据收集）
