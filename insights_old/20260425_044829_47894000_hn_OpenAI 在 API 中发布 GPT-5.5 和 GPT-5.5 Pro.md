## 核心洞察

### 1. 模型性能表现

**主要发现：**

- **"懒惰"问题**：用户反映 GPT-5.5 在被明确要求使用事务和rollback时，仍给出旧的不完整的代码框架（只有 BEGIN TRAN 和 commit，没有实际查询）
- **代际改进有限**：部分用户认为 GPT-5.3-codex 之后的模型改进不大，只是在不同权衡间调整
- **长任务改进积极意见**：有用户报告 GPT-5.4 在长任务一致性方面有巨大改进，运行了32天、400M+ tokens的任务

**用户体验反馈：**
> "我感觉最近2-3代模型（gpt-5.3-codex之后）没有多大改进，只是调整东西做不同的权衡。"

### 2. 定价与可用性

**定价信息：**

- **GPT-5.5 Pro**：$5/MTok 输入，$0.50 缓存，$30 输出
- **是 GPT-5.4 价格的2倍**
- **OpenRouter 价格**：$30/$180（MTok input/output）

**可用性问题：**

- 企业用户报告昨日仅看到 GPT-5.4，需要等待 rollout
- GitHub Copilot Pro 用户无法使用（仅 Pro+、Business、Enterprise 可用）
- 有用户称 subsidized AI 时代已结束

### 3. 基准测试争议

**WordPress 基准测试结果：**

- 用户 `guilamu` 使用自制 WordPress+GravityForms 基准测试
- GPT-5.5 表现"在排行榜上最差"且"性价比最差"
- 其他模型如 Opus 4.7 表现也低于 Sonnet 4.6

**讨论焦点：**

- 基准测试是否代表整体性能
- 推理与非推理模型的对比

### 4. 知识截止日期问题

**发现：**

- API 页面显示知识截止：2024-06（而非声称的2025-12-01）
- 用户实际测试显示模型知道2024年美国大选结果
- 一般认为实际知识截止在2025年初

### 5. 安全性讨论

**争议观点：**

- 有用户批评OpenAI的safeguards是"把事情变蠢"
- 讨论AI伦理：医生使用ChatGPT翻译诊断、与非英语患者交流
- 反对意见认为 refusal 是"责任转移伪装成安全"

## 关键结论

1. **定价过高**：GPT-5.5 Pro 价格是 GPT-5.4 的2倍，引发用户不满
2. **性能改进不明显**：社区普遍反馈与前代相比改进有限
3. **可用性分层**：高级模型仅面向企业/Pro+用户，普通用户被限制
4. **知识截止存疑**：报告日期与实际知识库存在差异
5. **安全过滤争议**：safeguards引发持续争议
