## 洞察摘要

### 核心发现

DeepSeek V4 是中国开发的大型语言模型，其性能接近顶级模型（如 Claude Opus 4.6/4.7、GPT-5），但价格仅为它们的零头。

### 关键数据

- **成本对比**：用户报告分析一个 TypeScript 代码库（包括 API、DTOs、服务层、数据库模型），DeepSeek V4 Pro 仅花费 **$0.09**，而类似任务在 Claude Opus 上预计需要 **$9-$13**
- **Token 数量**：有用户报告用约 $10 可以获得超过 **200M tokens**
- **性价比**：在当前折扣优惠下（75%折扣至2026年5月31日），DeepSeek V4 Pro 比 Claude/Sonnet 便宜 **91-97%**

### 性能评价

- 多数用户认为 V4 Pro 在编程能力上与 **Claude Sonnet 4.6** 水平相当
- 部分用户认为接近 **Claude Opus 4.6/4.7** 水平
- 适合实现类任务（给定规格进行实现），但在开放式规划和复杂推理方面仍有差距

### 使用体验

- 有用户将 DeepSeek V4 Pro 成功用于前端开发
- 部分用户反映思维链（thinking trace）过于冗长，在代理式编程中会"过度思考"
- 通过 OpenRouter、DeepSeek API 直接调用、Ollama Cloud 等多种方式可用

### 争议与担忧

1. **数据隐私问题**：有用户指出 DeepSeek 明确会使用通过 API 发送的数据进行训练，而 GitHub 用代码训练 Copilot 引发大量抗议，但 DeepSeek 相关讨论较少
2. **开源 vs 闭源**：部分用户认为开源模型（如 DeepSeek）更公平，因为最终会发布模型权重
3. **本地运行**：DeepSeek V4 Pro 约有 25GB 活跃参数，需要大量 RAM 才能本地运行

## 结论

DeepSeek V4 Pro 以极低的价格提供了接近顶级模型的编程能力，对于预算有限或需要大量 token 消耗的开发者来说是一个极具吸引力的选择。但用户需要注意其数据使用政策，并根据自己的隐私需求选择合适的调用方式。
