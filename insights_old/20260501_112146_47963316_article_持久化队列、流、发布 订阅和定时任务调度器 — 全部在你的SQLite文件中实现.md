## 项目概述

Honker 是一个为 SQLite 添加 Postgres 风格 `NOTIFY`/`LISTEN` 语义的扩展，提供持久的发布/订阅、任务队列和事件流功能，无需客户端轮询或独立的守护进程/代理。

## 核心特性

1. **持久队列与流**：在 SQLite 文件内实现队列、流、发布/订阅和 cron 调度器
2. **高性能**：跨进程唤醒延迟约 0.7ms (p50)，在 M 系列笔记本上测试
3. **多语言支持**：Python、Node、Rust、Go、Ruby、Bun、Elixir、C++
4. **统一存储**：队列与业务数据存储在同一 SQLite 文件中，事务一致性
5. **无轮询**：通过轮询 SQLite 的 `PRAGMA data_version` 实现实时唤醒

## 工作原理

- 每毫轮询一次 SQLite 的 `PRAGMA data_version`，这是 SQLite 在每次提交时递增的单调计数器
- 后台线程将信号分发给每个订阅者，执行 `SELECT ... WHERE id > last_seen` 获取新行
- 每个数据库只有一个轮询线程，无论订阅者数量多少
- 空闲成本极低：无页面缓存压力、无写锁竞争、无内核文件监视器

## 解决的问题

当应用使用 SQLite 作为主数据存储后，通常需要添加任务队列。传统方案是引入 Redis + Celery，但这带来了：
- 第二个数据存储及其备份问题
- 业务表与队列之间的双写问题
- 运行代理的运维开销

Honker 的方案是将队列放在同一个 SQLite 文件中，`INSERT INTO orders` 和 `queue.enqueue(...)` 在同一事务中提交，回滚时同时丢弃。

## 安装方式

- Python: `pip install honker`
- Node: `npm install @russellthehippo/honker-node`
- Rust: `cargo add honker`
- Go: `go get github.com/russellromney/honker-go`
- Ruby: `gem install honker`
- Bun: `bun add @russellthehippo/honker-bun`
- Elixir: `{:honker, "~> 0.1"}`
