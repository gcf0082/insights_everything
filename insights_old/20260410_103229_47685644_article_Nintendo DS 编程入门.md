## 一、文档概述

本手册是Nintendo DS homebrew（自制软件）编程的入门指南，由Jaeden Amero编写。本手册涵盖了从基础的homebrew概念到实际游戏开发的完整流程，旨在帮助编程爱好者了解Nintendo DS的内部工作原理并创建自己的游戏。

手册采用案例驱动的方式，通过创建一个名为"Orange Spaceship"的太空射击游戏来教授编程技术。

## 三、关键技术要点

### 3.1 开发工具链

- **devkitPro**: 专业的homebrew开发工具链
- **libnds**: Nintendo DS开发库（前身NDSLIB）
- **devkitARM**: ARM处理器交叉编译工具

### 3.2 硬件概念

- **双屏显示**: Nintendo DS配备两个独立工作的2D图形引擎
- **VRAM配置**: 9个VRAM banks，总容量约656KB
- **图形模式**: 支持Mode 0-6，其中Mode 5支持旋转背景
- **OAM**: 对象属性存储器，最多支持128个sprites

### 3.3 重要概念

- **DLDI** (Dynamically Linked Device Interface): 动态链接设备接口标准
- **Affine Transformation**: 仿射变换，用于背景的旋转、缩放
- **DMA**: 直接内存访问，提高数据传输效率

## 四、设备选择指南

### 4.1 启动设备类型

| 类型 | 说明 | 推荐度 |
|------|------|--------|
| GBA Flash Cart | 传统方式，价格高 | 一般 |
| Slot-1设备 | R4DS、M3 Real、Cyclo DS Evolution | 推荐 |
| Slot-2设备 | M3 Adapter、G6 Flash | 不推荐（用于 piracy） |
| NoPass | 模拟正版卡带 | 推荐 |

### 4.2 推荐设备

- **Slot-1首选**: Cyclo DS Evolution（菜单体验最佳，支持DLDI自动补丁）
- **GBA开发**: GBA Movie Player (GBAMP) + NoPass组合

## 五、Homebrew合法性分析

### 5.1 合法的理由

1. 玩家拥有对自己购买硬件的处置权
2. 反向工程用于学习目的完全合法
3. Homebrew无法与官方开发工具竞争
4. 出版homebrew游戏几乎不可能（无Licensing）

### 5.2 法律边界

- 禁止：破解版权保护、盗版游戏、泄露商业机密
- 允许：学习硬件原理、为社区创作有趣程序

### 5.3 社区态度

- 坚决反对 piracy
- 严格保护版权保护相关信息
- 致力于扩大Nintendo DS功能边界

## 七、代码示例结构

### 7.1 初始化流程

```c

int main() {
    // 开启2D图形核心
    powerOn(POWER_ALL_2D);
    return 0;
}
```

### 7.2 VRAM配置示例

```c
void initVideo() {
    vramSetMainBanks(VRAM_A_MAIN_BG_0x06000000,
                     VRAM_B_MAIN_BG_0x06020000,
                     VRAM_C_SUB_BG_0x06200000,
                     VRAM_D_LCD);
    videoSetMode(MODE_5_2D | DISPLAY_BG2_ACTIVE | DISPLAY_BG3_ACTIVE);
    videoSetModeSub(MODE_5_2D | DISPLAY_BG3_ACTIVE);
}
```
