## 核心事件

GitHub 于2026年4月27日宣布，Copilot 将从2026年6月1日起从高级请求计费（PRU）转为基于使用量计费（GitHub AI Credits）。用户将根据消耗的token数量付费，包括输入、输出和缓存token。

## 主要定价变化

### 个人用户
- **Copilot Pro**: $10/月，包含 $10 AI Credits
- **Copilot Pro+**: $39/月，包含 $39 AI Credits

### 企业用户
- **Copilot Business**: $19/用户/月，包含 $19 AI Credits
- **Copilot Enterprise**: $39/用户/月，包含 $39 AI Credits

### 模型乘数变化（旧版年度计划）
| 模型 | 旧乘数 | 新乘数 |
|------|--------|--------|
| Opus 4.6/4.7 | 3x | 27x |
| GPT 5.4 | 1x | 6x |
| GPT 5.4 mini | 0.33x | 6x |
| Sonnet 4.6 | 1x | 9x |

## 社区关键洞察

### 1. 价格大幅上涨
用户普遍反映价格涨幅惊人：
- Sonnet 4.6 从1x涨至9x
- Opus 模型从3x涨至27x
- 有用户计算：每月$19的Business计划，现在可能只能对Opus说一句话

### 2. "Subsidized Inference"时代终结
社区认为这标志着AI公司补贴推理成本的时代已经结束。之前$10/$20/$100的订阅能获得大量使用量是不可持续的，现在只是回归正常价格。

### 3. 用户可能转向替代方案
- **OpenRouter**: 用户推荐使用OpenRouter来替代Copilot，但需注意其收取5.5%手续费
- **直接API**: 直接使用Anthropic或OpenAI API可能更划算
- **Claude Code / Cursor**: 部分用户考虑转向竞品

### 4. 批评声音
- 这是"enshittification"（价值贬损）的典型案例
- 被比喻为"诱饵-switch"（先低价吸引，后大幅涨价）
- 年度计划用户感到被背叛：已预付费用但6月1日后乘数也会增加

### 5. 企业市场的担忧
- 好奇企业用户是否会被迫接受涨价
- 担心 experimentation cost（试验成本）上升会减缓工具价值的发现过程

### 6. 积极观点
- 部分用户认为这是更透明的定价方式
- 使用量计费比按请求计费更合理（因为agent可能消耗大量token）
- 代码补全和Next Edit建议仍然免费

## 技术细节变化
- 代码补全和Next Edit建议不消耗AI Credits
- 不再提供fallback体验（用完额度后无法降级使用）
- Copilot code review将消耗GitHub Actions分钟数
- 企业可设置预算控制，可在企业、成本中心或用户级别设置

## 总结

社区普遍对此次涨价持批评态度，认为这是GitHub从用户获取阶段转向盈利阶段的标志。用户正在寻找替代方案，OpenRouter和直接API成为热门选择。但也有声音认为这是市场回归理性的标志，之前的低价本就是不可持续的补贴行为。
