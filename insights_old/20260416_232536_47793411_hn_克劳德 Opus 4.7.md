## 内容摘要

### 主要话题

本次讨论围绕 Anthropic 发布的 Claude Opus 4.7 模型展开。该版本使用更新的分词器（tokenizer），可提高文本处理效率，但代价是相同输入可能映射到更多 token（约为 1.0-1.35 倍，具体取决于内容类型）。

### 热点讨论

1. **Caveman 提示词风潮**
   - 用户讨论使用" caveman "提示词风格来节省 token
   - 有评论指出这类工具实际上 savings 极少（不到1%），且可能让模型 reasoning 更加困难
   - 《Compressed Chain of Thought》论文研究表明可减少60%的输出 token

2. **Claude 性能问题**
   - 大量用户抱怨 Claude 近期性能下降
   - 尤其提到 4.6 版本在过去一周表现糟糕
   - 有用户转向 Codex 并获得更稳定的结果
   - 许多人认为是计算资源不足导致的问题

3. **与 OpenAI Codex 对比**
   - 用户讨论 Anthropic 算力不足 vs OpenAI 预先订购大量算力的战略优势
   - OpenAI 推出 $100 计划来争夺 Anthropic 客户
   - Claude Code vs Codex 的具体使用体验差异
   - Codex 在透明度和稳定性方面获得好评

4. **Token 使用与定价**
   - 新 tokenizer 虽然更高效但可能增加 token 使用量
   - 用户对订阅价值的疑虑
   - 对使用限制调整的不满

### 关键观点

- 用户对 Anthropic 缺乏透明度表示不满
- 许多人建议公司更直接地沟通算力限制问题
- 有用户转向竞争对手寻求更稳定的体验
- 对比不同 AI 工具的实际编码质量差异
