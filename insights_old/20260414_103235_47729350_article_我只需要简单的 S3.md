# 简单S3存储方案探索

**来源**: [I Just Want Simple S3](https://blog.feld.me/posts/2026/04/i-just-want-simple-s3/)  
**作者**: feld  
**发布日期**: 2026-04-10  
**最后更新**: 2026-04-13

## 需求概述

作者的需求非常简单：
- 只需要 S3 兼容接口
- 不需要横向扩展
- 不需要数据复制
- 只需要可靠且性能良好的存储方案

## 评估的存储方案

### 1. Minio - 已停止维护

Minio 已经停止维护并归档了代码库，原因是为了追逐 AI 行业的利润。作者指出曾发现 Minio 的测试存在问题——他们使用 mock 响应而不是实际测试代码，导致删除功能曾经出现 bug 但未被检测到，作者对这种态度不满意。

### 2. Garage - Rust 实现但过于复杂

Garage 是一个用 Rust 编写的较新的存储方案，但作者认为它不必要地复杂。开发时间较短（曾因资金问题暂停），缺少一些正常的 S3 功能。

### 3. SeaweedFS - 性能问题

SeaweedFS 是一个有趣的项目，支持 WebDAV 等多层功能。但作者在本地网络中运行 master 和 volume 节点时遇到严重的性能问题——即使在局域网内，下载速度也只有几百 KB/s，最终才缓慢上升到约 10Mbps，远未达到预期。

### 4. CEPH - 过于庞大

CEPH 是一个庞大的存储系统，作者在工作中使用它，但认为对于其简单需求来说过于复杂。如果确实需要构建与 Amazon S3 竞争的系统，CEPH 可能是选择。

### 5. Versity GW - 最终选择 ✅

**推荐方案**

Versity S3 Gateway 是一个较少为人知但被多个机构（包括桑迪亚国家实验室、洛斯阿拉莫斯国家实验室、军队和多所大学）使用的方案。

**优势：**
- 支持本地 POSIX 文件系统作为存储后端
- 提供 Web 界面进行管理
- 支持匿名/公共读 bucket 和策略管理
- 使用 xattrs 存储对象元数据
- 可以代理到其他 S3 后端
- 性能优异——在局域网中能达到线速下载

作者使用 Versity 后，在本地网络中获得了预期的快速性能。

### 6. 后期发现的方案

- **RustFS**: 2023年12月开始开发，2025年7月开源，声称100% S3兼容性，性能优于 Minio
- **rclone**: 可以作为 S3 服务器使用，但不是主要用途
- **Filestash**: 一个存储无关的 Dropbox 风格文件管理器
- **Zenko CloudServer**: Node.js 实现
- **Supabase Storage**: Node.js 实现，元数据存储在 Postgres 中

## 结论

作者最终选择了 Versity GW 作为其简单 S3 存储方案，并期待未来能出现真正的 ZFS 原生对象存储解决方案。
