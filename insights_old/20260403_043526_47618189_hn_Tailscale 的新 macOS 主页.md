## 主题概述

本洞察讨论了 Tailscale 新发布的 macOS 桌面应用，同时引发了对 macOS 菜单栏（Menu Bar）图标被 Notch（刘海）隐藏问题的广泛讨论。

## 主要讨论内容

### 1. macOS Notch 隐藏菜单栏图标问题（核心焦点）

**问题描述**：
- MacBook 的刘海区域会直接隐藏位于其位置的菜单栏图标
- 没有任何视觉提示表明有图标被隐藏
- 用户无法通过下拉菜单访问被隐藏的图标
- 这在 14 英寸显示器上尤为严重，用户可能只能放三个第三方菜单栏应用

**历史背景**：
- 这个问题并非新出现，菜单栏图标溢出隐藏的问题已经存在超过 20 年
- 在 Notch 出现之前，当应用程序菜单过多时也会发生这种情况
- Notch 只是让这个问题变得更早出现

**用户体验影响**：
- 有开发者反馈，其兄弟开发的应用因为菜单栏图标被隐藏，导致用户误以为应用无法启动或没有界面
- 这类问题导致大量退款请求，甚至产生支付纠纷

### 2. 解决方案探讨

**官方方案**：
- macOS Tahoe 在系统设置中提供了半解决方案，可以隐藏选定的菜单栏实用程序
- 但这并不理想，因为被隐藏的应用无法访问

**第三方解决方案**：
- **Bartender**：流行的菜单栏管理应用，但最近被出售给第三方，且因 Apple API 变化在 Tahoe 上损坏
- **Ice**：曾流行的开源方案，目前在 Tahoe 上已无法工作
- **Thaw**：Ice 的维护分支，继续获得更新

**命令行方案**：
```
defaults -currentHost write -globalDomain NSStatusItemSpacing -int 2
defaults -currentHost write -globalDomain NSStatusItemSelectionPadding -int 2
```
可以减少菜单栏图标间距，用户可容纳约 2 倍数量的图标

**绕过 Notch 的方案**：
- 选择"显示所有分辨率"中高度为屏幕高度减去刘海高度的分辨率
- 优点：去掉刘海、无圆角、传统高度菜单栏

### 3. 对 macOS 设计的批评

用户普遍批评 Apple 在这方面的设计：
- "这是 Apple 旗靓产品的严重衰落"
- "一个如此简单的解决方案却拖延了数十年"
- Windows XP 多年前就实现了溢出菜单
- 浏览器的扩展图标也有溢出菜单

### 4. Tailscale 相关讨论

**用户需求**：
- 远程访问家庭网络
- 在国外通过家庭网络串流美国电视
- 便于非技术用户连接（父母、孩子等）
- 远程 Linux 服务器开机自动连接

**Tailscale 方案**：
- 设置美国设备作为出口节点（Exit Node）
- 配置其他设备通过其代理
- 使用 Tailnet 邀请家人

**与其他方案比较**：
- 与 WireGuard 相比：Tailscale 提供更简单的配置和更多企业级功能
- 对于 CGNAT 用户：可使用 VPS 或公共 IPv6 配合 ddns

**Linux 客户端**：
- 目前 Linux 仅有 CLI 版本
- 用户希望有更好的 GUI 支持
- 可使用 tailscale-systray 等第三方方案

### 5. Tailscale 新应用反馈

- 新窗口化应用获得好评，用户喜欢不再受限于 Modal
- 有用户建议将服务与配置应用分离，避免关闭配置应用时停止服务
- 有人期望 Mullvad 等其他 VPN 厂商也跟进窗口化界面

## 社区情绪总结

1. **对 Notch 问题的沮丧**：这是本次讨论的主要情绪，普遍对 Apple 不解决这个问题感到困惑和失望
2. **对 macOS 整体质量下降的担忧**：多个用户提到近年来 macOS 的各种小问题
3. **对 Tailscale 的积极态度**：对新 macOS 应用表示欢迎
4. **对第三方工具的依赖**：社区依赖 Bartender、Thaw 等工具来解决菜单栏问题
