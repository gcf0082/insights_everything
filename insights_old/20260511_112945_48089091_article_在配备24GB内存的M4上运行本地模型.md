## 核心洞察

### 1. 推荐配置方案

在24GB内存的MacBook Pro M4上，作者推荐使用**Qwen 3.5-9B Q4_K_S量化模型**，通过**LM Studio**运行，可实现：
- 约40 tokens/秒的生成速度
- 128K上下文窗口
- 成功启用思考模式和工具调用

### 2. 思考模式推荐参数

```json
{
  "temperature": 0.6,
  "top_p": 0.95,
  "top_k": 20,
  "min_p": 0.0,
  "presence_penalty": 0.0,
  "repetition_penalty": 1.0
}
```

启用思考需在Prompt Template底部添加：`{%- set enable_thinking = true %}`

### 3. 工具选择对比

| 工具 | 特点 |
|------|------|
| LM Studio | 模型运行平台，推荐用于本地部署 |
| Ollama | 另一种本地模型运行选择 |
| llama.cpp | 底层推理库 |
| Pi | 响应更快，但缺乏合理默认值 |
| OpenCode | 自定义性强，适合高级用户 |

### 4. 与SOTA模型的差异

本地模型需要采用**交互式工作流**：
- 需要逐步指导，不能一次性完成复杂任务
- 用户需承担更多思考和规划工作
- 适合作为研究助手、代码审查工具、命令行辅助

### 5. 实际应用案例

**成功案例**：
- 使用credo检查Elixir代码风格问题并自动修复
- 识别git冲突并提供解决方案建议

**失败案例**：
- 工具调用有时会忘记执行编辑操作
- 可能无法正确处理需要编辑器交互的命令（如git rebase --continue）

### 6. 本地模型的优势

1. **无需网络连接**，可在飞行途中使用
2. **零订阅成本**，仅需电费
3. **更环保**，减少数据中心依赖
4. **可玩性强**，适合技术爱好者探索

---

## 关键结论

本地LLM（如Qwen 3.5 9B）虽然无法与SOTA模型相比，但在24GB内存的M4 Mac上可以良好运行，适合执行简单任务、代码辅助和研究工作。关键是要调整工作方式，采用分步骤交互而非一次性复杂指令，以获得最佳效果。本地模型提供了一种更可持续、更环保的AI使用方式。
