## 内容摘要

Fil-C 是一个内存安全的 C 语言实现项目，通过运行时检查来保证内存安全。本帖讨论了这一技术的优缺点。

### 核心观点

1. **性能权衡**: Fil-C 比 Rust 慢 20-30%，但目标是达到可接受的性能水平
2. **安全性**: 提供运行时内存安全，但错误时直接崩溃而非可恢复的异常
3. **与 Rust 对比**: Rust 在编译时提供更多安全检查，Fil-C 是运行时安全
4. **创建者背景**: 项目由前苹果 JavaScript VM 团队负责人、GC 专家开发（如Brucehoult指出的"not some random dad, but a GC expert and former leader of the JavaScript VM team at Apple"）

### 讨论焦点

- **错误处理**: Fil-C 错误时崩溃，Rust 可以 panic 并恢复
- **互操作性**: 无法与非 Fil-C 代码互操作，包括 libc
- **实际应用**: 对于 CLI 工具等场景崩溃是可接受的
- **硬件加速**: 讨论了通过 CHERI 等硬件支持提升性能的可能性

## 结论

Fil-C 为现有 C/C++ 代码提供了一种无需重写的内存安全解决方案，适合不想进行大规模重写的项目，但需要接受性能开销和错误时直接崩溃的设计。
