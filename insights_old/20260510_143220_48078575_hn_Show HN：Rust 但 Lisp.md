## 项目概述

**Rust but Lisp (rlisp)** 是一个将 Rust 语义与 Lisp 语法相结合的项目。该项目允许开发者使用 S-表达式语法编写代码，然后直接编译成 Rust 代码，无需运行时和垃圾回收器。

## 核心特性

### 1. 透明的 S-表达式 前端
- 代码路径：`s-expr → .rs → binary`
- 不增加额外运行时负担
- 利用 `rustc` 进行类型检查、借用检查和优化

### 2. 完整的 Rust 特性支持
项目支持所有 Rust 核心特性：
- 所有权（Ownership）
- 借用（Borrowing）
- 生命周期（Lifetimes）
- 泛型（Generics）
- trait
- 模式匹配（Pattern matching）

### 3. 语法映射示例

| Lisp 语法 | Rust 语法 |
|----------|----------|
| `(add ((x i32) (y i32)) i32 (+ x y))` | `fn add(x: i32, y: i32) -> i32 { (x + y) }` |
| `(let x i32 42)` | `let x: i32 = 42;` |
| `(struct Point (x f64) (y f64))` | `struct Point { x: f64, y: f64 }` |
| `(enum Option (T) (Some T) None)` | `enum Option { Some(T), None }` |
| `(match val ((Some x) (handle x)) (None ()))` | `match val { Some(x) => { handle(x) }, None => { } }` |
| `(if (> x 0) (println! "yes") (println! "no"))` | `if x > 0 { println!("yes") } else { println!("no") }` |

### 4. 宏系统 - 核心亮点

rlisp 的宏是编译时的 S-表达式 转换器，不需要 `proc_macro` crate、token streaming 或 syn/quote。

**宏的优势：**
- 宏本质上是接收 S-表达式并返回 S-表达式 的函数，在编译时执行
- 无需 token streaming 或 `proc_macro` 的复杂仪式
- **这是 Lisp 为 Rust 带来的杀手级特性**

### 5. 同像性（Homoiconicity）
- 代码和数据之间没有区别
- 函数和宏都使用相同的 S-表达式 表示

## 总结

Rust but Lisp 是一个实验性项目，旨在将 Lisp 的宏系统（同像性、强大的编译时元编程）引入 Rust 世界。通过提供一个透明 S-表达式 前端，开发者可以享受 Lisp 风格宏的便利，同时获得 Rust 的安全性和性能保证。该项目的核心理念是"让宏变得简单"，通过消除 `proc_macro` 的复杂性来实现这一目标。
