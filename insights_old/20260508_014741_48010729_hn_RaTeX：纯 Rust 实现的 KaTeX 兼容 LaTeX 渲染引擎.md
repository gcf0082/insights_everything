## 项目概述

RaTeX 是一个用纯 Rust 编写的 LaTeX 数学公式渲染引擎，旨在与 KaTeX 兼容。其主要特点包括：

- **纯 Rust 实现**：核心代码使用 Rust 编写
- **WASM 支持**：可编译为 WebAssembly，在浏览器中运行
- **原生应用支持**：可在没有 JavaScript 引擎的环境中使用
- **零 JS Bundle**：声称 JS bundle 为 0KB（核心是 WASM）

## 主要讨论热点

### 1. LaTeX 的用户体验问题

讨论的核心话题是 LaTeX 的复杂性和可用性问题：

- **学习曲线陡峭**：需要大量hack（如 `\makeatletter \makeatother`）来完成自定义
- **脆弱性**：看似无关的更改可能导致其他部分出问题
- **不一致性**：不同的包有不同的行为和接口

一位用户提到使用 LaTeX 约 10 年，包括本科和硕士论文，"从未感到自然或可靠"。

### 2. Typst 作为现代替代方案

许多用户推荐 Typst 作为 LaTeX 的现代替代品：

- 更易于理解、一致和可定制
- 使用真正的编程语言而非临时宏系统
- 生态较小但可自行实现功能
- 已有用户成功用它写完博士论文和书籍

### 3. RaTeX 的优势与问题

**优势：**
- 原生 Rust，适合 Rust 生态系统
- 适合没有浏览器环境的原生应用
- 可集成到 Rust 静态网站生成器

**问题与质疑：**
- **文档不完整**：没有提及二进制大小和性能
- **可访问性缺失**：生成的图片缺少 alt 文本和 ARIA 标签
- **营销过度**：网站内容看起来像 LLM 生成，包含一些"诚实的谎言"
- **生态系统**：仅关注数学渲染，不是一个完整的 TeX 实现

### 4. 生态系统兼容性讨论

- **Pandoc**：可在 LaTeX 和 Typst 之间转换（但不无损）
- **arXiv 接受度**：如果 arXiv 不接受 Typst，它就难以成为真正替代品
- **LLM 辅助迁移**：有用户报告成功用 LLM 将 LaTeX 模板迁移到 Typst

---

## 关键结论

1. **LaTeX 的历史包袱**：虽然输出质量好，但用户体验差，被认为需要" hardcore 专家"才能可靠使用
2. **Typst 崛起**：作为现代替代方案受到广泛好评，用户愿意放弃 LaTeX 生态转向 Typst
3. **RaTeX 的定位**：主要价值在于原生应用场景（非浏览器），作为 KaTeX 的 Rust 替代品
4. **可访问性关注**：现代排版工具需要考虑无障碍访问，这是 RaTeX 需要改进的方面
