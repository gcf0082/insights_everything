# 洞察报告：Fiverr 客户文件公开可搜索事件

## 基本信息

- **洞察链接**: https://news.ycombinator.com/item?id=47769796
- **标题**: Tell HN: Fiverr left customer files public and searchable
- **发布时间**: 7 小时前
- **得分**: 371 points
- **评论数**: 66 comments
- **发布者**: morpheuskafka

## 事件概述

Fiverr（一个零工经济平台，与 Upwork 竞争）使用 Cloudinary 服务来处理消息中的 PDF 和图片文件，这些文件包括从业者向客户交付的工作产品。然而，Fiverr 选择使用公开 URL 而非签名 URL（signed/expiring URLs），导致敏感的客户资料可以被 Google 搜索索引。

### 主要问题

1. **公开访问的文件存储**: Fiverr 使用 Cloudinary 存储客户文件，但使用了公开可访问的 URL，而非签名或过期的 URL
2. **Google 搜索索引**: 由于使用公开 URL，数百个包含个人身份信息（PII）的文件被 Google 索引
3. **示例查询**: `site:fiverr-res.cloudinary.com form 1040` 可以找到人们的完整税表
4. **Fiverr 投放广告**: Fiverr 故意为 "form 1234 filing" 等关键词购买 Google 广告，尽管知道安全问题存在
5. **责任披露未果**: 向 security@fiverr.com 披露后 40 天未收到回复

### 法律责任影响

这可能导致税务准备人员违反 GLBA/FTC Safeguards Rule（格拉姆-利奇-布林利法案/联邦贸易委员会保障规则）

## 社区评论要点

### 安全问题的严重性

- 用户发现成千上万的社保号码（SSN）、税表、管理员凭据、API tokens
- 税务表格和其他财务文件完全公开
- 存在管理员密码、渗透测试报告、机密 PDF、内部 API 文档
- 有人发现 Dating Site 的管理员密码，这可能导致敲诈

### Fiverr 的回应

Fiverr 回应称：
> "You'er the second person to flag this issue to us. Please note that our records show no contact with Fiverr security regarding this matter ~40 days ago unlike the poster claims. We are currently working to resolve the situation"

发帖者提供了电子邮件记录作为证据。

### 代码安全问题

- 开发者不了解直接对象访问（direct object reference）
- 混淆（obfuscation）被认为足够安全
- 不知道 robots.txt、sitemap 的工作原理
- 很多外包开发人员缺乏基本的安全知识

### 行业问题

- 软件开发工作太容易入门
- 处理数百万人数据的岗位应该有认证要求
- 这种泄漏已经成为常态
- 专业人士需要执照（医生、律师、水管工、电工），但软件行业没有类似要求

### 时间线

- 5 小时后仍未完成敏感文件删除
- 某人在几个月前发送了 DMCA 投诉与此问题相关
- Fiverr 曾在过去对，明显欺诈行为在其平台上发生时毫无作为

- 医疗记录

- API tokens
- 管理员凭据
- 渗透测试报告
- 机密 PDF
- 内部 API 文档
- 书籍手稿
- 课程 PDF

## 技术原因分析

### 为什么文件被 Google 索引

Google 不会随机尝试访问互联网上的每个 URL，它通过链接来抓取。因此，这些文件要被 Google 索引，必须从某个公开的地方链接到它们。

### 应该使用的安全措施

- 签名 URL（Signed URLs）
- 过期 URL（Expiring URLs）
- 禁止爬虫（robots.txt）
- 不使用可预测的 URL 模式

## 结论

这是一起严重的数据泄露事件，涉及：
- 客户敏感财务信息公开
- 数百万人个人信息可能被泄露
- 企业安全实践严重不足
- 对责任披露未作回应

社区普遍认为这应该是"商业模式终结"的信任违约，但担心缺乏监管力度来推动改变。
