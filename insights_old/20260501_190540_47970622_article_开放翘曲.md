## 项目概述

OpenWarp 是一个基于 Warp 终端的社区分支项目，核心目标是为 Warp 解锁自定义 AI 提供商能力（BYOP - Bring Your Own Provider）。该项目允许用户将任意 AI 模型接入终端，保留 Warp 全部交互体验的同时，将 AI 层完全开放——密钥、模型、提示词全部由用户掌控。

---

## 核心特性

### 1. BYOP 自定义提供商
通过 genai 适配层原生支持 6 种 API 协议：
- OpenAI
- OpenAI Responses
- Anthropic
- Gemini
- Ollama
- DeepSeek

用户可在设置中显式选择 API 协议、粘贴 Base URL 与 API Key，实现任意模型接入。

### 2. 动态提示词模板
基于 minijinja 的模板引擎，支持根据当前工作目录、语言、角色实时渲染系统提示词，精准引导模型。

### 3. 隐私优先
- 关闭 Cloud Agent / Computer Use
- 默认不上传云端
- 凭证仅保存在本地 (~/.config/openwarp.toml)
- 不收集遥测数据

### 4. 多语言界面
原生支持简体中文、English、日语、西班牙语等 UI 界面。

### 5. 保留 Warp 体验
持续合并 Warp 上游主线，完整保留 Blocks、AI 命令、Workflows、Keymaps、主题等功能。

---

## 工作方式

1. **接入任意提供商**: 在设置中选择 API 协议、填写 Base URL 与 API Key
2. **编写动态提示词**: 使用 minijinja 模板根据上下文渲染系统提示词
3. **在终端立即启用**: 一键切换模型、对话、命令补全

---

## 常见问题

- **与 Warp 官方的关系**: OpenWarp 是基于 Warp 开源代码的社区分支，与 Warp 官方公司无附属关系
- **API Key 安全**: 所有凭证仅保存在本地配置文件中，直接与 provider 端点通信，不经任何中转
- **上游更新**: 会持续合并 Warp 上游主线，在保留体验的同时叠加 BYOP 与多语言增强

---

## 技术栈

- 后端: Rust
- 前端: Astro + Tailwind + Framer Motion
- 模板引擎: minijinja

---

## 总结

OpenWarp 为希望在终端中使用自定义 AI 模型的用户提供了一个开源解决方案。通过支持多种 API 协议和本地凭证存储，它在保持 Warp 优秀交互体验的同时，给予用户对 AI 模型的完全控制权。该项目适合开发者和技术爱好者使用，但需要注意其目前仍处于早期开发阶段。
