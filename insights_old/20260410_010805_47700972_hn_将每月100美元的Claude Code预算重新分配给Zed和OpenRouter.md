## 内容摘要

本文作者将每月100美元的Claude Code订阅费用转向 Zed 编辑器配合 OpenRouter 的方案。作者认为 Zed 更轻量且反应更快，同时 OpenRouter 提供更灵活的多模型接入和成本控制。

## 主要讨论观点

### OpenRouter 的优势
- 通过单一API密钥访问数十种模型
- 支持追踪每个请求的成本
- 可以在不同模型上运行相同请求并对比结果
- 支持设置路由规则和数据保留策略
- 可匿名化用户身份，保护隐私

### 对比 LiteLLM
- LiteLLM 在集成 Anthropic 模型时性能明显较差
- OpenRouter 作为中间层更稳定
- 但 OpenRouter 收取约5.5%的服务费

### Zed 编辑器的体验
- 初始很喜欢，但长期使用发现一些小问题
- 内存使用（尤其是TS语言服务器）较高
- 一些UX细节不如VSCode完善
- 扩展系统尚未成熟

### 替代方案讨论
- **OpenCode Zen + GitHub Copilot**: 成本更低，可长时间使用
- **Pi**: 轻量级AI编程助手，设计简洁易扩展
- **Kimi K2.5 / GLM 5**: 中国模型性价比高，速度快
- **Ollama Cloud**: $20/月，无使用限制

### 关于Claude订阅
- 用户反映Claude Code消耗配额速度快
- 100美元订阅可获得1000美元以上的使用量
- 存在账户被封禁的风险（与转售或加密货币相关）

---

## 结论

社区普遍认为 Zed + OpenRouter 是可行的 Claude Code 替代方案，尤其适合对成本敏感且需要灵活使用多种模型的用户。但对于追求最佳开发体验的团队，VSCode 或 JetBrains 仍然是更成熟的选择。
