## 一、核心概述

OpenAI 于 2026 年 4 月发布了 **Privacy Filter**，这是一个用于检测和遮盖文本中个人身份信息（PII）的开源模型。该模型体积小（1.5B 总参数，50M 活跃参数），支持本地运行，数据无需离开用户设备即可完成隐私过滤。

**核心定位**：一款前沿级别的个人数据检测小型模型，适用于高吞吐量隐私工作流程，可在 128K token 上下文中进行上下文感知的 PII 检测。

**许可证**：Apache 2.0

**获取地址**：Hugging Face 和 GitHub（均为 openai/privacy-filter）

## 二、核心能力与技术创新

### 2.1 检测范围（八大类别）

| 标签类别 | 说明 |
|---------|------|
| `private_person` | 私人姓名 |
| `private_address` | 私人地址 |
| `private_email` | 私人邮箱 |
| `private_phone` | 私人电话 |
| `private_url` | 私人 URL |
| `private_date` | 私人日期 |
| `account_number` | 账号信息（含信用卡号、银行账户等） |
| `secret` | 密码、API 密钥等 |

### 2.2 技术架构

- **架构类型**：双向 token 分类模型（基于自回归预训练 checkpoint 改造）
- **解码方式**：BIOES span 标签 + 约束 Viterbi 过程
- **处理方式**：单次前向传播完成所有 token 标注
- **上下文长度**：最高 128,000 tokens
- **参数规模**：总参数 1.5B，活跃参数 50M

### 2.3 相比传统方案的优势

传统 PII 检测依赖格式规则（如正则匹配电话号码、邮箱格式），存在以下局限：
- 难以处理上下文敏感的个人信息
- 无法区分公开信息与需遮盖的私人信息

Privacy Filter 的优势：
- **上下文感知**：基于语言先验检测 PII 范围
- **覆盖更广**：支持非结构化文本中的细粒度 PII 检测
- **边界清晰**：通过 BIOES 解码产生更连贯的遮盖边界
- **可配置**：开发者可调整召回率/精确率平衡

## 五、局限性与注意事项

1. **非匿名化工具**：不适用于完全匿名化处理
2. **非合规认证**：不能替代高风险场景下的政策审查
3. **依赖标签体系**：检测效果受限于训练时定义的分类法和决策边界
4. **跨语言/跨领域表现**：可能低于训练分布之外的语言、脚本、命名约定的场景
5. **短文本处理**：在短序列中可能过度或不足遮盖
6. **高敏感领域**：法律、医疗、金融等场景建议结合人工审查和领域微调

## 六、训练方法

2. **架构改造**：将语言模型 head 替换为 token 分类 head，进行监督分类目标的后训练
