## 内容摘要

这是一篇关于 Claude Code 速查表的 Hacker News 讨论帖。帖子作者 phasE89 每天使用 Claude Code，但经常忘记命令，于是让 Claude 研究了所有文档和 GitHub 功能，生成了一份可打印的 A4 横向 HTML 页面。

### 速查表涵盖的内容

- 键盘快捷键
- 斜杠命令（slash commands）
- 工作流（workflows）
- 技能系统（skills system）
- 内存/CLAUDE.md
- MCP 配置
- CLI 标志
- 配置文件

### 速查表特点

- 单个 HTML 文件，轻量级
- 免费，无需注册
- 每日 cron 任务自动检查更新并标记新功能为 "NEW"
- 自动检测 Mac/Windows 系统以显示正确的快捷键
- 显示当前 Claude Code 版本
- 顶部包含可关闭的最近更新日志
- 支持移动端
- Ctrl+P 打印

- **sbinnee**: 认为 Claude Code 更具可扩展性，演进更快

### 2. 关于 CLI 必要性的讨论

- **amai**: "为什么我们仍然需要 AI 的神秘命令？"
- **droidjj**: "这个速查表的存在是一个 UX 红色警告"

- **artyom**: 质疑为什么需要像编程语言一样的速查表

### 3. 速查表的技术细节和修正

- **winternewt**: 询问版本，因为没有 /cost 命令
- **mohsen1**: 指出 ^ 是 Control 键的符号，不是 ⌘
- **kxrm**: 指出 CMD+V 粘贴图片是错误的，Mac 上应该是 CTRL+V
- **zyz**: 确认 CTRL+G 打开外部编辑器，不是 CMD+G
- **bingemaker**: 建议 MCP 部分不应使用 ~ 前缀
- **airstrike**: 在美信纸上可以完美打印成1页

### 4. 关于 AI 交易策略的讨论

- **dataviz1000**: 描述了使用 Claude Code 子代理进行量化交易研究的计划
- **aryehof**: 质疑基于历史数据的策略能否在未来成功
- **nurettin**: 认为这种方法不会找到任何新东西，需要 6-10 年才能验证

### 5. devcontainer 和沙箱环境

- **ffsm8**: 推荐使用 devcontainer.json
- **kstenerud**: 讨论了 yoloAI 项目和 devcontainer 方法的优缺点

### 6. 创意建议

## 总结

这篇讨论展现了开发者社区对 Claude Code 这款 CLI 工具的热情。速查表的出现反映了开发者对掌握复杂命令系统的需求，同时也引发了一场关于 AI 工具应该采用 CLI 还是 GUI 的讨论。大多数开发者认为 Claude Code 的 CLI 设计是合理的，因为它的功能是无限的，而自由的文本输入更适合这种场景。

此外，Claude Code 与 OpenAI Codex 之间的竞争也是讨论的热点，许多开发者认为虽然 Claude Code 的 UX 更好，但 Codex 的模型质量可能更胜一筹。
