## 内容摘要

本帖讨论了 Unix/Linux 系统中 man 手册页标题里数字的含义（如 `sleep(3)` 中的 "(3)"）。

### 核心知识点

- **section 1**: 用户命令（如 `ls(1)`, `cat(1)`）
- **section 2**: 系统调用（如 `read(2)`, `write(2)`）
- **section 3**: C 库函数（如 `printf(3)`, `sleep(3)`）
- **section 4**: 设备文件
- **section 5**: 文件格式（如 `crontab(5)`, `passwd(5)`）
- **section 6**: 游戏
- **section 7**: 杂项
- **section 8**: 系统管理命令

### 实用技巧

1. 使用 `man 5 crontab` 查看文件格式而非命令
2. 使用 `man -k crontab` 可同时搜索多个 section
3. 使用 `apropos --sections=3 .` 查找特定 section 的内容

### 热门评论要点

1. **有趣的彩蛋**: 开发者 Colin Watson 在 man-db 中添加了 Easter egg：当你在凌晨 00:30 运行 `man` 时，会输出 "gimme gimme gimme"
2. **另一个彩蛋**: 如果用户名为 "Tyler" 运行 `shutdown`，会显示 "Oh, good morning Mr. Tyler, going down?"（来自 Aerosmith 的歌曲）
3. **历史渊源**: section 编号源自 AT&T 发布的物理 Binder 系统，每个 section 对应一卷手册
4. **tunefs 彩蛋**: BSD 的 tunefs 手册页包含 "You can tune a file system, but you cannot tune a fish"
5. **关于 Easter eggs 的讨论**: 
   - 支持者认为广泛使用的软件更应该有彩蛋，因为能带来欢乐
   - 反对者认为它们增加了攻击面，且未经过充分测试

### 社区观点

- 许多人认为 man 页面设计对新手不友好，应该有更友好的交互界面
- 有观点认为 Rust 社区倾向于用重写解决所有问题
- 也有用户指出现在很多人转向 StackOverflow 或 LLM 而非阅读 man 手册
