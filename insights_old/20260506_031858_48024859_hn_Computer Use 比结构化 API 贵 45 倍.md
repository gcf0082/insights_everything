## 核心发现

文章主要对比了"计算机使用"(Computer Use)与"结构化API"(structured APIs)在执行任务时的成本差异。研究表明，使用视觉模型进行计算机操作比调用专门的API要贵45倍。

### 关键数据

- **成本差异**: 结构化API比计算机使用便宜45倍
- **时间差异**: 视觉模型需要将近20分钟完成任务，而Sonnet模型只需20秒
- **效率对比**: 结构化API不仅成本低40倍，而且更加确定性，能够构建稳定的产品

## 讨论要点

### 1. Agent技术的现实应用

- 计算机使用适合遗留SaaS软件，例如酒店使用的过时物业管理系统(PMS)
- 对于没有API的专利/锁定桌面应用，计算机使用是更明显或唯一的选择
- 大多数情况下，API/MCP是可用的，且推荐的解决方案

### 2. 混合方法的有效性

- 某些公司(如Smooth)使用混合DOM/视觉方法，并 strongly indexed on 小模型
- UI设计通常为了最小化歧义并尽可能高效地提供上下文
- 下拉框或自动完成等模式使导航更容易，这也是小模型能完成工作的原因
- 精心设计的计算机使用导航实际上可以令人惊讶地 token 高效

### 3. 无障碍访问的协同效应

- AI Agent对有残疾、腕管综合症、RSI、ADHD的人都有帮助
- 界面需要对Agent可访问，不仅仅是人类，这反过来增加了对人类的无障碍访问

### 4. 争议观点

- 有人认为"Agentic engineering"只是为了为其带来更多收入
- 有人指出Anthropic建议Agent解决方案应该是最后的手段，大多数问题用一次调用就能很好地服务
- 关于是否需要完全重新思考操作系统以支持AI Agent，存在分歧

### 5. 技术实现讨论

- 通过可访问性API为Agent提供工具是一个好的解决方案
- OS层面在应用程序正确声明输入时受到保护
- Invoke项目提供macOS应用程序的API，特别关注可访问性相关功能
- Android正在开发AppFunctions功能

## 主要结论

1. **优先使用API**: 尽可能使用结构化API而非计算机视觉使用
2. **成本效益**: 结构化API不仅更便宜，而且更快速、更确定性
3. **混合策略**: 对于没有API的应用程序，可以考虑混合方法或计算机使用作为最后手段
4. **技术趋势**: 虽然方便性通常会获胜，但在生产环境中，成本和可靠性是关键因素
