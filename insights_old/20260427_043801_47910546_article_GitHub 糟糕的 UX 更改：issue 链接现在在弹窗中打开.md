## 问题描述

在某些仓库中，点击任何指向 issue 的链接现在会在弹出层中打开，而不是导航到该 issue。用户质疑这是逐步推出的功能更新，并询问是否有办法关闭或配置此行为。这一变化严重影响了用户体验和生产效率。

## 用户反馈摘要

1. **用户体验问题**：弹出层行为破坏了正常的链接导航功能，影响辅助技术使用，用户希望有选项恢复旧行为
2. **AI Agent 影响**：使用 AI Agents 时，点击问题链接复制 URL 时，得到的是父问题链接而非目标链接
3. **视觉效果**：弹出层未居中显示，看起来很奇怪
4. **兼容性**：有用户创建了 Userscript 临时解决方案来恢复传统链接行为

## 官方回应

**GitHub 工作人员 (azenMatt)** 于 2026年4月26日回复：

> 感谢所有反馈，这是我们尝试的一项功能改进，旨在提高跨仓库链接的加载速度，我们将回滚此更改。

**dewski (功能负责人)** 进一步解释：

> 我对此功能的推出负责。目标是提供更一致的用户体验，让点击 issue 时的行为在不同场景（子issue、issues dashboard、GitHub Projects等）中保持一致。此更改也带来了一些性能改进。初衷是好的，但用户反馈表明我们没有达到目标，已经回滚。

## 结论

GitHub 已确认将回滚此功能更改。用户反馈表明该功能不受欢迎，特别是在以下方面：
- 破坏了传统的链接导航行为
- 影响辅助技术用户体验
- 视觉效果不符合用户预期
