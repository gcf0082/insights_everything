## 核心发现

### 1. 扩展程序扫描概述

LinkedIn 正在对其用户浏览器中安装的扩展程序进行系统性扫描。截至2026年4月，LinkedIn 已识别并跟踪**6,278个扩展程序**。这一行为可追溯至2017年，当时列表包含38个条目。

### 2. 技术实现原理

- LinkedIn 通过发送 `fetch()` 请求到 `chrome-extension://` 类型的URL来检测扩展程序
- 每个错误请求都代表一个用户未安装的扩展程序
- 扫描可在约15分钟内完成对超过6,000个扩展程序的检测
- 扫描以两种模式运行：并行模式（使用 `Promise.allSettled()`）和 последовательный模式（可配置延迟）

### 3. 数据用途

当 LinkedIn 的扩展程序扫描运行时，它并不是为匿名访问者构建设备配置文件。LinkedIn 已知用户的：
- 真实姓名
- 雇主信息
- 职位头衔
- 职业历史
- 薪资范围
- 专业人脉网络
- 地理位置

扩展程序扫描会将详细的软件清单附加到已包含用户验证过的专业身份的配置文件中。

## 隐私风险

### 职业风险

- **求职活动泄露**：数百个求职相关扩展程序在扫描列表中，LinkedIn 可以在用户告知雇主之前就知晓他们正在悄悄找工作
- **个人生活推断**：与政治内容、宗教信仰、残障便利化、神经多样性相关的扩展程序使用情况会被推断并附加到用户的专业身份上
- **组织情报收集**：由于 LinkedIn 知道每个用户的工作单位，扫描结果可以帮助绘制公司的内部工具、安全产品、竞争对手订阅和工作流程

### 系统性风险

- 上述行为均未在 LinkedIn 的隐私政策中披露
- 用户未被告知、也未获得同意
- 这构成了更大的浏览器指纹生态系统问题

## 高级指纹系统

扩展程序扫描只是 LinkedIn 更大的设备指纹系统的一部分。该系统内部称为：
- **APFC** (Anti-fraud Platform Features Collection)
- **DNA** (Device Network Analysis)

该系统收集**48个浏览器和设备特征**，包括：
- Canvas 指纹
- WebGL 渲染器和参数
- 音频处理行为
- 安装的字体
- 屏幕分辨率
- 像素比
- 硬件并发
- 设备内存
- 电池电量
- 通过 WebRTC 获取的本地 IP 地址
- 时区和语言

## 技术细节

### 检测机制

1. **扩展程序列表检测**：LinkedIn 在 JavaScript 文件中包含硬编码的扩展程序 ID 数组，发送 `fetch()` 请求探测每个扩展程序的 `web_accessible_resources`
2. **光谱分析法 (Spectroscopy)**：遍历整个 DOM 树，检查每个文本节点和元素属性中是否引用了 `chrome-extension://` URL

### 数据传输

- 检测到的扩展程序 ID 被封装到 `AedEvent` 和 `SpectroscopyEvent` 对象中
- 使用 RSA 公钥加密
- 传输到 LinkedIn 的 `li/track` 端点
- 加密的指纹作为 HTTP 头注入到用户会话期间的每个后续 API 请求中

## 验证方式

用户可以自行验证：
1. 在 Chrome 中打开 LinkedIn
2. 打开开发者工具 (F12)
3. 查看控制台标签
4. 每个红色错误信息都代表一个被探测的扩展程序，构成您指纹的一部分

## 结论

LinkedIn 正在未经用户同意或知情的情况下，系统性地收集用户浏览器中安装的扩展程序信息，并将其与用户的验证专业身份关联。这一行为违反了用户隐私权，且未在隐私政策中披露。对于记者、律师、研究人员和调查人员来说，这种风险尤其严重，因为他们通常在 LinkedIn 上拥有详细的可验证身份档案。
