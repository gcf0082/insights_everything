---

## 核心观点

### 传统观点的误区

普遍认为**不能直接比较浮点数**，必须使用 epsilon 比较：

```cpp
bool approxEqual(float x, float y) {
    return abs(x - y) < 1e-4f;
}
```

作者认为这种方法**通常是最差的解决方案**。

### 浮点数的本质

- 浮点数是**确定性的**，遵循 **IEEE 754** 标准
- 虽然不能精确表示所有实数，但运算结果是**最接近真实值的可表示数**
- 不精确性不等于"不确定性"或"随机性"

### Epsilon 比较的三大问题

1. **临时 hack**：只是掩盖问题，不解决根本原因
2. **级联问题**：不同模块使用不同 epsilon 导致难以调试的 bug
3. **选择困难**：没有正确的方法选择合适的 epsilon 值
4. **非传递性**：多数算法假设比较是传递的，epsilon 破坏这一假设

## 案例研究

### 1. 基于网格的移动

**问题**：单位移动完成后检测位置是否到达目标单元格中心。

**错误做法**：使用 epsilon 比较
```cpp
if (distance(selectedUnit.position, targetCell.center) > 1e-4)
```

**更好方案**：
- 允许玩家在移动完成前发出下一个命令
- 使用动画队列按顺序播放
- 或使用指数平滑等对目标值突变健壮的插值方案

### 2. 球面线性插值 (Slerp)

**问题**：当两个向量非常接近时，`acos` 可能返回 NaN。

**错误做法**：随机选择 epsilon
```cpp
if (angle < 1e-4) return lerp(a, b, t);
```

**更好方案**：直接检查 `dot(a, b) >= 1.f`，利用浮点数的精确表示特性：
```cpp
if (d >= 1.f) return lerp(a, b, t);
```

### 3. 向量长度计算

**问题**：对小向量返回零，对大向量精度损失严重。

**解决方案**：使用最大值归一化：
```cpp
float length(vec3 v) {
    float M = max(abs(v.x), abs(v.y), abs(v.z));
    if (M == 0.f) return 0.f;
    vec3 u = v / M;
    return M * sqrt(u.x*u.x + u.y*u.y + u.z*u.z);
}
```

直接比较 `M == 0.f`，而非 `M < 1e-4`。

### 4. 求解线性方程组

**问题**：高斯消元中检查主元是否为零。

**错误做法**：
```cpp
if (M < 1e-4) return std::nullopt;
```

**更好方案**：直接检查 `M == 0.f`

**原因**：条件数检查是用户职责，不是库的实现职责。

### 5. 射线与包围盒相交

**问题**：射线方向分量为零时产生无穷值或 NaN。

**结论**：代码**已经正确处理**了这些情况，无需 epsilon。`sort` 函数自动解决正负零的问题。

### 6. 计算凸包

**问题**：三点几乎共线时，`left_turn` 谓词违反循环不变式。

**错误做法**：使用 epsilon
```cpp
return det(b-a, c-a) > 1e-4f;
```

**更好方案**：
- 将输入值四舍五入到固定网格，转为整数运算
- 使用任意精度有理数
- 使用区间算术
- 使用 2Sum 算法计算误差上界

### 7. 用户输入清理

**适合使用 epsilon 的场景！

**原因**：过滤掉太接近的点以避免渲染伪影。

**选择 epsilon 的方法**：
- 根据最大缩放和像素大小计算
- 分析算法估算可接受误差
- 二分搜索查找最佳值

### 8. 编写测试用例

**适合使用 epsilon 的场景！

使用较小的 epsilon（如 `FLT_EPSILON`）进行测试比较。

---

## 结论

**什么时候用 epsilon**：
- 用户输入清理
- 编写测试用例

**什么时候不用 epsilon**：
- 几乎所有其他情况

**核心原则**：不要盲目遵循"永远不用 epsilon"的教条，要**根据具体问题思考最佳解决方案**。
