## 核心内容

该帖子报道了 OpenAI 的 Codex AI 代理成功入侵三星电视系统的消息。事件引发了关于 AI 安全能力、闭源安全性以及物联网设备安全性的广泛讨论。

## 主要观点

### 1. AI 逆向工程能力

- **TP-Link 路由器案例**：用户分享了使用 Codex 逆向工程 TP-Link 路由器的经历。Codex 帮助他绕过了厂商的安全限制，创建了访问自己路由器的 Python API 和 Prometheus 监控指标。
- **固件逆向**：有用户指出，AI 可以使用反汇编工具（如 Ghidra）来处理二进制代码，尽管对机器码的直接推理仍有不确定性。
- **source code 重要性**：关键在于提供了固件源代码，这大大简化了漏洞发现过程。

### 2. 关于闭源安全性的讨论

- 有评论指出，Codex 被提供了源代码才成功完成入侵。这引发了关于"闭源是否真正提供安全保护"的讨论。
- 另一相关讨论（#47780456）认为，闭源在防止 AI 发现漏洞方面并没有实质性帮助。

### 3. 物联网设备的安全现状

- 三星电视的安全问题已被发现多年，有评论戏称"甚至 GPT2 都能黑掉三星电视"。
- 嵌入式设备的 BSP（Board Support Package）领域存在大量安全漏洞，代码质量低下，缺乏安全审计。
- 厂商为了赶在圣诞/黑色星期五等销售窗口出货，往往忽视安全检查。

### 4. AI 对网络安全的影响

- **技能民主化**：AI 使得逆向工程技能变得更易于获取，降低了攻击门槛。
- **规模化攻击**：AI 可以大规模扫描和测试，远超单个安全研究人员的工作量。
- **并非革命性突破**：有评论认为这并非技术突破，而是基于前人工作的大规模应用。

### 5. 相关工具和项目

- **tmpcli**：GitHub 上的一个项目，实现了 TP-Link 设备的 Python 客户端
- **ddecryptor**：一个简单的 JS 工具，可以解密某些 TP-Link 设备配置

## 总结

这次事件展示了 AI 在安全研究领域的潜力，同时也揭示了物联网设备普遍存在的安全漏洞问题。评论普遍认为，虽然 AI 使攻击更容易，但这并非真正的技术突破，而是将已有安全知识规模化应用。关键问题在于厂商对安全性的忽视，以及嵌入式设备领域缺乏安全审计的现状。
