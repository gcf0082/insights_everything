## 主题概述

Anchor Grid 构建了一个 API 和训练模型，用于检测建筑构件、提取进度表和分析建筑文档。该产品主要解决建筑行业长期存在的数据提取难题。

## 核心洞察

### 1. 建筑文档的特殊挑战

- **PDF 是实际通用语言**: 尽管 CAD/BIM 是理想形态，但现实是分包商、小型总包商和贸易商通常只收到 PDF 文件，而非原始 BIM 文件
- **文档来源复杂**: 图纸可能来自 1970 年的软盘、扫描的皱巴巴的纸质文件，或最新的数字导出
- **标准化缺失**: 每个公司、每个行业分支都有不同的约定和标准

### 2. 行业工作流程低效

当前存在大量重复性手工劳动：
- 建筑 technologists 制作基础图纸（含门）
- 总包商收到文档后重新计数验证
- 分包商再次计数
- 供应商也需计数确认
- 最终形成"数数循环"

### 3. 技术挑战

- **解析 DWG/DXF 的局限性**: 当源文件干净且结构良好时，本地解析 DWG/DXF 有意义，但实际中的建筑图纸往往不干净（未解决的 xrefs、爆炸的动态块、版本不兼容、SHX 字体替换等）
- **视觉模型的一致性和可重复性**: 这是使用视觉模型的真正挑战
- **检测精度**: 门检测目前准确率超过 0.87

### 4. 竞争对手与发展方向

- 其他公司也在构建类似平台，如解析 CAD/DXF 为结构化 JSON（但不用 ML）、芬兰建筑文档解析等
- 产品路线：先做门（Division 08），接下来是厕所（Division 10），然后是地板和管道
- 建议：要么接近 100% 准确率找到符号，要么为小众市场做全面工作流

### 5. 相关历史案例

提到 Xerox JBIG2 漏洞（约 2013 年），某些扫描设置会悄悄替换文档中的数字，建筑图纸是发现该问题的案例之一。

## 总结

建筑文档的 OCR 之所以困难，并非因为 PDF 格式本身难以读取，而是因为 OCR 模型缺乏针对建筑文档的专业训练，且这类文档技术性强、细节重要。这是一个真实的市场需求，Anchor Grid 通过视觉 AI 方式解决这一痛点。
