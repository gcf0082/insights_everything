# 洞察报告

**洞察链接**: https://blog.skypilot.co/scaling-autoresearch/

**发布日期**: 2026年3月18日

**作者**: Alex Kim, Romil Bhardwaj

**主题**: 如何扩展Karpathy的Autoresearch到GPU集群

---

## 核心发现

将Autoresearch从单GPU扩展到16 GPU集群后，8小时内完成了约910个实验，将验证损失（val_bpb）从1.003降至0.974，实现了**2.87%的性能提升**。

## 背景

Autoresearch是Andrej Karpathy的项目，让AI代理自动改进神经网络训练脚本。代理修改train.py，运行5分钟的训练实验，检查验证损失，然后循环。默认配置是单GPU、单代理、单实验顺序运行，每小时约12个实验。

## 实验设置

- **硬件**: 16个GPU（13个H100 + 3个H200）
- **基础设施**: Kubernetes集群（通过SkyPilot管理）
- **运行时间**: 约8小时
- **实验数量**: 约910个（700+有效结果）

## 五个实验阶段

### 阶段1：超参数扫描（前200个实验）
- 批量大小减半至2^18有帮助
- Adam betas (0.9, 0.95)优于默认值
- Weight decay 0.08优于0.2
- **结果**: val_bpb从1.003降至0.981

### 阶段2：架构发现（200-420个实验）
- 同时测试6种不同宽高比（AR=48, 64, 72, 80, 90, 96）
- 发现扩展模型宽度比任何超参数调整都更有效
- AR=96是最优选择（model_dim=768）
- **结果**: val_bpb降至0.977

### 阶段3：微调宽模型（420-560个实验）
- 在AR=96基础上微调：warmdown schedule、矩阵学习率等
- **结果**: val_bpb降至0.975

### 阶段4：优化器调优（560-700个实验）
- 最大发现：muon_beta2从0.95提升到0.98
- 这是后期最大的改进，单一改变带来约0.001的提升
- **结果**: val_bpb降至0.974

### 阶段5：边际收益递减（700-910个实验）
- 改进曲线趋于平缓
- 低于0.0001/实验的收益

## 关键洞察

### 1. 并行性改变了代理的研究策略

| 指标 | 顺序（1 GPU） | 并行（16 GPU） |
|------|--------------|---------------|
| 实验/小时 | ~10 | ~90 |
| 策略 | 贪婪爬山 | 每波因子网格 |
| 每决策信息 | 1个实验 | 10-13个同时实验 |

单GPU时，代理被迫做贪婪爬山（试一个，检查结果，再试下一个）。16 GPU时，代理可以运行完整的因子网格——例如同时测试3个weight decay值×4个learning rate值=12个实验，一波完成。

### 2. 涌现的研究策略：利用异构硬件

代理自发开发了**双层策略**：
- 在便宜的H100上并行筛选10+个假设
- 将前2-3名推广到H200进行确认运行

这是因为H200比H200快约9%，在相同5分钟预算下能完成更多训练步骤。代理通过观察结果自己发现了这一点。

### 3. 硬件差异导致排名不一定转移

例如，`FINAL_LR_FRAC=0.03`有时在H100上胜过0.05，但在H200上 consistently 输掉。原因是：更多训练步骤时，模型受益于在训练后期保持更高的学习率。

## 成本

- Claude Code API: 约$9
- GPU计算: 13个H100 × 8小时 ≈ $200，3个H200 × 8小时 ≈ $60
- **总计**: 约$300

## 最佳配置

```python
# 架构
ASPECT_RATIO = 96        # model_dim = 8 * 96 = 768
DEPTH = 8                # 8层transformer
WINDOW_PATTERN = "SL"    # 交替Sliding + Local attention

# 训练
TOTAL_BATCH_SIZE = 2**18  # ~524K tokens/step

# 学习率
MATRIX_LR = 0.05         # Muon优化器权重矩阵学习率
EMBEDDINGL_R = 0.6       # AdamW词嵌入学习率
SCALAR_LR = 0.5          # AdamW残差混合标量学习率

# 优化器
ADAM_BETAS = (0.70, 0.95)
WEIGHT_DECAY = 0.08
WARMDOWN_RATIO = 0.6
FINAL_LR_FRAC = 0.05
# Muon: momentum=0.95, ns_steps=5, beta2=0.98
```

## 结论

并行性不仅加速了实验（9倍），更重要的是改变了代理的搜索策略——从贪婪爬山转变为因子网格搜索，能够发现参数间的交互效应。此外，代理能够自主发现并利用异构硬件的性能差异，这是预期的涌现行为。
