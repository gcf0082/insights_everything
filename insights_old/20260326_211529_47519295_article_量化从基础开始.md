## 摘要

本文深入探讨了大型语言模型（LLM）中的量化技术。量化是一种有损压缩技术，能够将模型体积缩小4倍、速度提升2倍，同时仅损失5-10%的准确率。这使得强大的模型能够在普通笔记本电脑上运行。

---

## 1. LLM为何如此庞大

### 参数与权重

LLM的核心由数十亿个参数（也称为权重）组成。以Qwen-3-Coder-Next为例，该模型拥有800亿参数，体积达159.4GB，需要约159GB RAM才能运行。据传前沿模型拥有超过1万亿参数，需要至少2TB RAM。

### 神经网络结构

LLM采用层叠式神经网络结构：
- 输入层接收数据
- 中间层（隐藏层）处理信息
- 输出层生成预测结果
- 每层包含数千个节点，各节点间全连接

参数数量 = 输入数 × 层数 × 节点数 × 连接密度 = 数十亿

## 2. 数值存储原理

### 整数存储

计算机使用二进制位（bit）存储整数，每个位代表2的幂次方。例如8位无符号整数可表示0-255。

### 浮点数存储

#### float32（32位浮点）
- 1个符号位
- 8个指数位
- 23个尾数位
- 范围：±3.40×10³⁸
- 精度：7位有效数字

#### float16（16位浮点）
- 5个指数位 + 10个尾数位
- 范围：±65504
- 精度：3位有效数字

#### bfloat16（16位浮点变体）
- 8个指数位 + 7个尾数位
- 范围：极宽
- 精度：2位有效数字

### 关键发现

1. **分布特点**：大多数模型参数集中在0附近的小值区域
2. **精度权衡**：大值间跳跃大，小值间跳跃小
3. **实际需求**：LLM不需要32位浮点的全部精度

## 3. 量化技术详解

### 什么是量化

量化是将大范围值压缩到小范围值的过程，本质是**有损压缩**。

### 量化方法对比

#### 方法一：直接舍入（Round-to-Nearest）
将float32值四舍五入到float8或float4最近可表示值。

**问题**：对LLM效果极差
- float8尚可接受
- float4会导致模型完全失效（某些参数变为0，导致输出永远为0）

#### 方法二：对称量化（Symmetric Quantization）

**核心思想**：以0为中心对称压缩

**公式**：
```
scale = max(|values|) / (2^(bits-1) - 1)
quantized_value = round(value / scale)
dequantized_value = quantized_value × scale
```

**示例**（4-bit，参数范围-0.89至0.16）：
- scale = 0.89 / 7 ≈ 0.127
- 参数量化：[-7, 1, 1, -1, 1, -4]
- 平均误差：约18%

**优势**：实现简单，适合参数分布对称的场景

#### 方法三：非对称量化（Asymmetric Quantization）

**核心思想**：围绕数据中点而非0进行压缩

**公式**：
```
scale = (max - min) / (qmax - qmin)
zero = qmin - round(min / scale)
quantized_value = round(value / scale + zero)
dequantized_value = scale × (value - zero)
```

**示例**（4-bit）：
- scale = 0.07
- zero = 5
- 参数量化：[-8, 7, 6, 3, 7, -3]
- 平均误差：约8.5%

**优势**：更高效利用存储空间，误差比对称量化低约10%

## 4. 实际应用中的挑战

### 异常值问题

**问题**：模型中存在少量异常值（outliers），单个异常值会破坏整体量化。

**影响**：
- 参数量化误差从18%飙升到116.8%
- 大部分参数被压缩到极小范围

**解决方案**：分块量化（Block Quantization）
- 每块32-256个参数独立量化
- 保存每块的scale和zero值
- 权衡：块越大开销越小，但误差越大

### 关于异常值的补充说明

研究表明这些"超级权重"对模型质量至关重要。即使删除一个超级权重，模型也可能输出完全无意义的文本。实际量化方案通常会：
- 保留异常值不进行量化
- 或将异常值存储在单独表中

## 5. 量化对模型准确率的影响

### 测试模型

Qwen3.5 9B

### 测试方法

#### 5.1 困惑度（Perplexity）

衡量模型对下一个token预测的确信程度，越低越好。

| 格式 | 困惑度 | 增幅 |
|------|--------|------|
| bfloat16 | 8.186 | - |
| 8位对称 | 8.193 | +0.1% |
| 4位非对称 | 8.563 | +4.6% |
| 4位对称 | 8.71 | +6.4% |
| 2位非对称 | 66.1 | +707.5% |

#### 5.2 KL散度（KL Divergence）

衡量两个概率分布的差异，越接近0越好。

| 格式 | 平均KL散度 |
|------|------------|
| 8位对称 | 0.0008 |
| 4位非对称 | 0.0593 |
| 4位对称 | 0.0675 |
| 2位非对称 | 2.1447 |

#### 5.3 GPQA基准测试

198道高难度选择题（生物、化学、物理），随机猜测正确率25%。

| 格式 | 正确率 | 错误率 | 无答案 |
|------|--------|--------|--------|
| bfloat16 | 66.7% | 33.3% | 0% |
| 8位对称 | 73.2% | 26.8% | 0% |
| 4位非对称 | 62.6% | 36.4% | 1% |
| 4位对称 | 66.2% | 29.3% | 4.5% |
| 2位非对称 | 1% | 2% | 97% |

#### 5.4 实际对话测试

问题："英国首都是什么？"

| 格式 | 回答 |
|------|------|
| bfloat16 | 正确 |
| 8位对称 | 正确 |
| 4位非对称 | 正确 |
| 4位对称 | 正确 |
| 2位非对称 | 拒绝回答/循环输出 |

## 6. 量化对模型速度的影响

### 测试环境

MacBook Pro M1 Max 和 H100 GPU

### 结果（token/秒）

| 格式 | M1 Max | H100 |
|------|--------|------|
| bfloat16 | 19.45 | 106.85 |
| 8位对称 | 32.36 | 141.61 |
| 4位非对称 | 43.32 | 175.70 |
| 4位对称 | 46.05 | 177.06 |
| 2位非对称 | 40.25 | 166.90 |

### 关键发现

- **16位→8位**：速度提升约60-80%
- **16位→4位**：速度提升约80-120%
- 原因：更少的数据移动量

---

## 7. 核心结论

1. **量化效果远超预期**：并非线性劣化，4位量化保持约90%质量
2. **8位量化几无损失**：16位到8位几乎没有质量下降
3. **4位是实用选择**：性能和质量间取得良好平衡
4. **2位不可用**：信息丢失过多，模型基本失效
5. **实际应用价值**：本地运行LLM变得可行
