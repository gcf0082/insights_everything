## 项目概述

ymawky 是一个完全手写的 ARM64 汇编 Web 服务器，通过系统调用实现，不依赖 libc，采用 fork-per-connection（每个连接 fork 一个进程）模式。虽然主要为 MacOS 开发，但代码尽量保持可移植性。

## 主要功能

### 支持的 HTTP 方法
- GET、PUT、DELETE、OPTIONS、HEAD

### 核心特性
1. **静态文件服务**: 支持 MIME 类型自动检测
2. **路径安全保护**: 智能检测和阻止路径遍历攻击 (`..`)
3. **URL 编码解码**: 支持 `%XX` 格式的十六进制编码
4. **防 DoS 攻击**: 内置 slowloris 攻击防护，10 秒超时机制
5. **文件上传**: PUT 支持最大 1GB 文件，使用原子写入（临时文件+重命名）
6. **断点续传**: 支持 Range 请求，支持视频流媒体拖动
7. **错误页面**: 自定义 HTML 错误页面（400+ 状态码）
8. **目录列表**: 访问目录时自动列出文件和子目录

### 安全措施
- 拒绝 >= 4096 字节的路径
- 阻止路径遍历 (`/../..`)
- 要求路径至少 16 字节
- 自动在请求路径前添加 `www/` 限制范围
- 使用 `O_NOFOLLOW_ANY` 拒绝符号链接
- PUT 操作使用临时文件防止部分写入

## 技术架构

### 文件结构
```
├── src/          # 源代码
├── www/          # 文档根目录
├── err/          # 错误页面目录
├── docs/         # 文档和图片
├── Makefile      # 构建文件
└── config.S      # 配置文件
```

### 配置选项
- `DEFAULT_DIR`: 文档根目录（默认 `www/`）
- `ERR_DIR`: 错误页面目录（默认 `err/`）
- `DEFAULT_FILE`: 默认文件（默认 `index.html`）
- `RECV_TIMEOUT`: 接收超时（默认 10 秒）
- `MAX_BODY_SIZE`: 最大 PUT 大小（默认 1GB）
- `MAX_PROCS`: 最大并发进程数（默认 256）

### MacOS 特定实现
- 系统调用使用 `x16` 寄存器和 `svc #0x80`
- 使用 `sigaction` 的 `sa_tramp` 字段直接处理信号
- 使用 `stat64`、`sockaddr_in` 等 MacOS 特定结构
- 使用 Mach-O 重定位操作符（`@PAGE`、`@PAGEOFF`）

## 总结

ymawky 是一个展示汇编编程能力的开源项目，通过纯底层系统调用实现了完整的 HTTP 服务器功能。项目代码简洁但功能完善，包含多项安全保护措施，适合作为学习汇编语言和网络编程的参考资料。
