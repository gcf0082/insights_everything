## 内容概述

本纪录片讲述了 Clojure 从两年的学术休假期间诞生，到成为全球最大金融科技公司之一技术栈的完整故事。主要内容包括：

### 1. Clojure 的起源

- 创始人 Rich Hickey 在为期两年的学术休假中产生了这个"固执的想法"
- 纪录片展示了 Clojure 非传统的起源历程
- 体现了 Clojure 社区的价值观驱动文化
- 记录了 Clojure 对软件思维方式的深远影响

### 2. 核心团队

纪录片邀请了以下核心人物：
- Rich Hickey（Clojure 创始人）
- Alex Miller
- Stuart Halloway
- 以及众多社区贡献者

### 3. 关键技术影响

#### 基础研究论文

- **Out of the Tarpit** (2006) - Ben Moseley & Peter Marks
  - 识别出可变状态是软件意外复杂性的主要来源
  
- **Ideal Hash Trees** - Phil Bagwell
  - Hash Array Mapped Tries (HAMTs) 研究
  - 直接启发了 Clojure 的持久化数据结构
  
- **Composable Memory Transactions** (2005)
  - Tim Harris, Simon Marlow, Simon Peyton-Jones & Maurice Herlihy
  - 软件事务内存 (STM)
  - 影响了 Clojure 的 STM 并发模型

#### 关键书籍

- **On Lisp** - Paul Graham
  - 关于高级 Lisp 技术的经典著作
  
- **Programming Clojure** (第4版)
  - Alex Miller, Stuart Halloway & Aaron Bedra
  - 伴随 Clojure 1.0 发布的第一版
  
- **The Joy of Clojure**
  - Michael Fogus & Chris Houser

### 4. Rich Hickey 的重要演讲

### 5. 重要公司与项目

- **Nubank** - 全球最大的独立数字银行
  - 使用 Clojure 和 Datomic 运行核心基础设施
  - 2020年收购 Cognitect，目前负责 Clojure 的开发
  
- **Datomic** - 分布式数据库
  - 基于不可变事实设计
  - 由 Rich Hickey 和 Cognitect 设计

### 6. Clojure 方言与运行时

#### 核心
- **Clojure** - 运行在 JVM 上
- **ClojureScript** - 编译为 JavaScript
- **ClojureCLR** - 运行在 .NET CLR 上

#### 扩展项目
- **Babashka** - 快速脚本，无需 JVM 启动时间
- **Jank** - 面向 LLVM 实现原生编译
- **ClojureDart** - Flutter 移动应用
- **Squint/Cherry** - 编译为 ES 模块

### 7. 数据科学与 AI

- **libpython-clj** - 直接调用 Python 库
- **Noj** - Clojure 原生数据科学工具包
- **ClojureMCP** - 连接 AI 助手与 Clojure REPL

## 核心概念词汇表

| 术语 | 定义 |
|------|------|
| **Lisp** | 一类将代码表示为嵌套列表（括号包围）的编程语言家族 |
| **REPL** | 读取-求值-打印循环，交互式编程环境 |
| **函数式编程** | 围绕函数构建的编程风格，最小化可变状态和副作用 |
| **值** | 创建后不改变的数据：数字、字符串或持久化集合 |
| **持久化数据结构** | 修改时保留先前版本的集合 |
| **意外复杂性** | 由工具和设计选择导致的困难，非问题本身 |
| **STM** | 软件事务内存，通过事务而非锁来协调共享状态更改 |
| **宿主语言** | 设计运行在现有平台上的语言 |

## 总结

这部纪录片不仅记录了 Clojure 的发展历程，更重要的是展示了其核心理念：
- 强调简单性而非容易性
- 推崇不可变值和函数式编程
- 通过持久化数据结构解决状态管理问题
- 在 JVM 平台上实现 Lisp 的现代复兴

Clojure 由一个"固执的想法"起步，如今已发展成为支撑全球最大金融科技公司基础设施的重要编程语言，其影响深远而持久。
