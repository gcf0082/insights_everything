## 项目概述

**let-go** 是一个用Go语言编写的Clojure方言字节码编译器和虚拟机。它被称为"Almost Clojure written in Go"，是Go语言生态中最小、启动最快的Clojure家族语言。

### 核心理念

- **独立可执行文件** - 编译程序为单个二进制文件，无需运行时环境
- **WASM Web应用** - 编译为自包含的HTML页面，可在浏览器中运行
- **快速启动** - 7ms冷启动，预编译字节码使启动几乎瞬时
- **小体积** - 10MB二进制文件，14MB空闲内存，比Babashka小7倍，比JDK小30倍
- **Go互操作** - 可嵌入Go应用，Go结构与记录互转
- **广泛的Clojure兼容性** - 支持宏、解构、协议、记录、多方法、转换器、惰性序列、持久数据结构、大整数

### Babashka Pods支持

let-go支持Babashka pods，可访问数据库、AWS、Docker、文件监控等生态系统。通过pods/load-pod加载，支持JSON、EDN和transit+jsonpayload格式。

### Go互操作

- **RegisterStruct[T]** - 将Go结构映射到let-go记录
- **ToRecord[T] / ToStruct[T]** - 零开销往返转换
- **BoxValue** - 自动转换注册结构为记录
- boxed Go值通过.method语法暴露方法
- 记录支持.field访问

## 性能基准

### 与其他运行时对比

| 指标 | let-go | babashka | joker | clojure JVM |
|------|--------|-----------|-------|--------------|
| **平台** | Go字节码VM | GraalVM原生 | Go树遍历解释器 | JVM (HotSpot) |
| **二进制大小** | **10MB** | 68MB | 26MB | 304MB (JDK) |
| **启动时间** | **7ms** | 20ms | 12ms | 331ms |
| **空闲内存** | **14MB** | 27MB | 21MB | 92MB |

### 性能亮点

- **最小体积** - 比Babashka小7倍，比JDK小30倍
- **最快启动** - 7ms预编译字节码，适合requestAnimationFrame
- **短任务胜出** - map/filter和转换器管道：8ms vs 19ms (2.4倍快)
- **计算竞争** - fib(35)与Babashka差距在4%内
- **内存最低** - fib(35)用14MB vs 77MB (5.4倍少)
- **比Joker快10倍+** - 字节码VM vs树遍历解释器

## 已知限制

### 未实现功能

- Refs / STM - 使用atoms + channels替代
- Agents - 使用go blocks和channels替代
- Hierarchies (derive, ancestors等) - 仅存根
- with-precision - BigDecimal本身可用
- Chunked sequences - 惰性序列是非分块的
- 读者标签文字 (#inst, #uuid)
- deftype - 使用defrecord替代
- reify - 协议只能扩展到命名类型
- clojure.spec
- alter-var-root
- 数值溢出检测 - 使用+'/'-'/ '*'显式大整数运算
- subseq / rsubseq - 排序集合本身可用但范围查询未实现

### 已知行为差异

- concat*（内部使用）是惰性的 - 用户-facing的concat是惰性的，与Clojure匹配
- 所有通道操作都阻塞 - Go通道始终阻塞
- go blocks是真正的goroutines - 无IOC状态机
- 无BigDecimal - 数值塔为int64 + float64 + BigInt
- 正则是Go风格 - re2语法，不是Java正则
- letfn使用atoms - 内部有轻微开销

## 安装和使用

### 安装方式

**Homebrew (macOS/Linux):**
```bash
brew tap nooga/let-go https://github.com/nooga/let-go
brew install let-go
```

**下载预编译二进制:**
从Releases页面获取Linux、macOS和Windows的amd64/arm64版本

**从源码构建:**
```bash
go install github.com/nooga/let-go@latest
```

### 使用方式

```bash
lg                              # REPL
lg -e '(+ 1 1)'                 # 求值表达式
lg myfile.lg                    # 运行文件
lg -r myfile.lg                # 运行文件后进入REPL
lg -w outdir myfile.lg         # 编译为WASM Web应用
```

### 编译和分发

**编译为字节码:**
```bash
lg -c app.lgb app.lg           # 编译为字节码
lg app.lgb                     # 直接运行字节码
```

**创建独立二进制:**
```bash
lg -b myapp app.lg             # 编译+打包为可执行文件
./myapp                        # 无需lg即可运行
```

**构建WASM Web应用:**
```bash
lg -w site app.lg              # 编译为Web应用
open site/index.html           # 在浏览器中打开
```

## 总结

let-go是一个创新性的项目，它将Clojure的强大功能带入Go生态系统，具有极快的启动速度、极小的体积和优秀的Go互操作性。它适合需要快速脚本编写、嵌入式脚本和WebAssembly应用的场景，是Go开发者在日常工作中使用Clojure风格编程的理想选择。
