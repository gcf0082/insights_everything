## 文章摘要

Colin Percival 是 FreeBSD 安全 Officer，也是 Tarsnap 的创始人。他于 2006 年 4 月 10 日创建了 AWS 账号，是最早使用 AWS 的开发者之一。本文回顾了他在过去 20 年中与 AWS 相关的经历和贡献。

### 主要时间线

1. **2006年4月10日**: 创建 AWS 账号（因 S3 发布而注册）
2. **2007年**: 联系 Jeff Barr，开始参与 FreeBSD/EC2 工作
3. **2008年**: 发现 AWS SimpleDB 签名方案的碰撞漏洞
4. **2010年1月**: 加入 IAM 私有测试版
5. **2010年12月13日**: 宣布 FreeBSD 可在 EC2 t1.micro 实例上运行
6. **2016年10月**: 发表博客警告 EC2 IMDS 的安全性问题
7. **2019年7月**: Capital One 事件后，IMDSv2 发布
8. **2019年5月**: 加入 AWS Heroes 项目
9. **2023年11月**: 成为 FreeBSD 发布工程负责人
10. **2024年4月**: 通过 GitHub Sponsors 获得亚马逊赞助（每周10小时）

### 关键贡献

- **FreeBSD/EC2 支持**: 花费多年时间让 FreeBSD 运行在 EC2 上
- **安全反馈**: 发现并报告多个 AWS 安全问题（签名漏洞、IMDS 风险等）
- **FreeBSD 发布工程**: 2023 年起担任 Release Engineering Lead

## 评论热点分析

### 1. 大科技公司应获得免费劳动？

讨论焦点：开发者是否为亚马逊等大公司提供免费 R&D 劳动的问题。

> "I find myself holding back private research because I don't want to provide free R&D for a value-extraction machine that is already efficient enough."

观点：
- 有人认为开源软件被大公司" commoditize"（商品化）
- 越来越多的项目采用 BSL (Business Source License) 来防止开源工作成为超大规模云厂商的免费输入

### 2. 许可证策略讨论

关于如何防止大公司使用开源软件获利的讨论：

- **AGPL**: 被认为是唯一能有效迫使云厂商开源其软件的许可证
- **GPL3**: 可被放在服务器后面运行而不分发源码
- **BSL**: 公司为了"形象"选择开源，并非真正想做开源

### 3. AWS 价格比较

> "For a small to medium web stack you are looking at roughly $350 a month on AWS versus 20 to 25 euros on Hetzner"

AWS 的真实价值：
- IAM 模型
- 全球覆盖
- 深度集成
- 组织内部"没人会因为选 AWS 而被解雇"的共识

### 4. IAM Roles 安全讨论

关于 EC2 IAM Roles 和 IMDS 安全性的技术讨论：

- IMDSv1 通过未认证的 HTTP 暴露凭证（被 Capital One 事件利用）
- IMDSv2 是有用的改进但只是缓解措施
- 有观点认为 IAM Roles 已经大大改善了安全状况

### 5. Colin 的补偿问题

> "$300/hr was what a mere L6 google engineer would make salaried"

Colin 在 2024 年以 $300/小时的价格获得 AWS 赞助，有人认为这个价格偏低。

## 结论

这篇文章不仅是个人回顾，也引发了关于开源生态系统中贡献者与大公司之间关系的广泛讨论。核心问题包括：
- 开源开发者是否应该为商业公司提供免费劳动
- 如何通过许可证保护开源工作不被云厂商"摘桃子"
- AWS 作为云服务提供商的价值与定价策略
- 安全研究的责任和回报

这些讨论在当前 AI 时代的开源社区中仍然具有重要意义。
