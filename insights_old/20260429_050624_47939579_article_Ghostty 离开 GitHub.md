# 洞察报告：Ghostty 离开 GitHub

## 基本信息

- **洞察链接**：https://mitchellh.com/writing/ghostty-leaving-github
- **发布日期**：2026年4月28日
- **作者**：Mitchell Hashimoto
- **主题**：Ghostty 终端模拟器项目将离开 GitHub平台

## 洞察摘要

### 背景

Mitchell Hashimoto 是 GitHub 第 1299 号用户，自 2008 年 2 月加入以来，几乎每天都会访问 GitHub，持续超过 18 年。他将 Vagrant（首个成功开源项目）的启动归因于希望能以此获得 GitHub 的工作机会。GitHub 是他生活中最热爱的地方，甚至在度假时也会浏览 GitHub 上的项目来学习开源项目的运作方式。

### 离开原因

1. **频繁的服务中断**：在过去一个月，作者记录了几乎每天都有 GitHub 服务中断影响其工作能力。就在撰写本文当天，就因 GitHub Actions 故障导致约 2 小时无法进行 PR 审核。

2. **无法正常完成工作**：作者表示 GitHub 不再是一个可以进行严肃工作的地方，每天因故障损失数小时。

3. **个人情感因素**：尽管作者仍然深爱 GitHub，但这种热爱已经变得不理性。他曾公开严厉批评 GitHub，对相关人员造成伤害，对此表示歉意。

### 迁移计划

1. **逐步移除依赖**：Ghostty 项目将逐步移除对 GitHub 的所有依赖，包括 issues、PRs、Actions 等。

2. **保留只读镜像**：计划在原 URL 保留只读镜像供查阅。

3. **寻找新托管**：作者正在与多个提供商（商业和开源项目）讨论迁移方案，具体细节将在未来几个月公布。

4. **其他项目不受影响**：作者的其他个人项目将继续留在 GitHub，仅 Ghostty 项目因受影响最大而优先迁移。

## 结论

这是 Mitchell Hashimoto 经过 18 年使用 GitHub 后做出的艰难决定。迁移时间与 2026 年 4 月 27 日的大规模故障恰好重合，但作者强调计划已讨论数周，此博客文章一周前已写好。最终决策于本周做出。
