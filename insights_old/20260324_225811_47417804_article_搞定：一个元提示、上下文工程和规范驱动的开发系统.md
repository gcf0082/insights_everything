## 项目简介

**get-shit-done** 是一个轻量级但强大的**元提示工程（meta-prompting）、上下文工程（context engineering）和规格驱动开发（spec-driven development）系统**，专为 Claude Code、OpenCode、Gemini CLI、Codex、Copilot、Cursor 和 Antigravity 等 AI 编码工具设计。

该项目的核心价值主张是解决**"上下文腐败"（context rot）**问题——即 Claude 在填充其上下文窗口时发生的质量下降问题。

## 核心功能

### 1. 上下文工程
GSD 自动管理项目上下文，确保 AI 始终拥有高质量的上下文信息：
- `PROJECT.md` - 项目愿景，始终加载
- `research/` - 生态系统知识（技术栈、功能、架构、陷阱）
- `REQUIREMENTS.md` - 带有阶段可追溯性的 v1/v2 需求
- `ROADMAP.md` - 进度和完成状态
- `STATE.md` - 决策、阻碍、跨会话记忆
- `PLAN.md` - 带有 XML 结构的原子任务
- `SUMMARY.md` - 执行结果和变更历史

### 2. 工作流程命令

| 命令 | 功能 |
|------|------|
| `/gsd:new-project` | 完整初始化：提问→研究→需求→路线图 |
| `/gsd:discuss-phase [N]` | 捕获实施决策 |
| `/gsd:plan-phase [N]` | 研究+规划+验证 |
| `/gsd:execute-phase <N>` | 并行执行所有计划 |
| `/gsd:verify-work [N]` | 人工验收测试 |
| `/gsd:ship [N]` | 从已验证阶段创建 PR |
| `/gsd:next` | 自动进入下一逻辑步骤 |
| `/gsd:quick` | 快速执行临时任务 |
| `/gsd:complete-milestone` | 归档里程碑并标记发布 |
| `/gsd:new-milestone` | 开始下一个版本 |

### 3. 多代理编排
- **研究阶段**：4 个并行研究者调查技术栈、功能、架构、陷阱
- **规划阶段**：规划器创建计划，检查器验证，循环直到通过
- **执行阶段**：按波次分组执行，每个执行者拥有全新的 200k 上下文
- **验证阶段**：验证器检查代码库是否符合目标，调试器诊断失败

### 4. 波浪式执行
- 独立计划 → 同一波 → 并行运行
- 依赖计划 → 后续波 → 等待依赖
- 文件冲突 → 顺序执行

### 5. 原子 Git 提交
每个任务完成后立即获得自己的提交：
```
abc123f docs(08-02): complete user registration plan
def456g feat(08-02): add email confirmation flow
hij789k feat(08-02): implement password hashing
```

## 技术特点

1. **XML 提示格式化** - 每个计划都使用针对 Claude 优化的结构化 XML
2. **模块化设计** - 可添加阶段、插入紧急工作、调整计划而不重建一切
3. **多运行时支持** - 支持 Claude Code、OpenCode、Gemini CLI、Codex、Copilot、Cursor、Antigravity
4. **安全性硬化** - 包含路径遍历防护、提示注入检测、安全 JSON 解析等

## 安装方式

```bash
# 安装所有运行时
npx get-shit-done-cc@latest

# 指定运行时
npx get-shit-done-cc --claude --global   # Claude Code
npx get-shit-done-cc --opencode --global # OpenCode
npx get-shit-done-cc --gemini --global   # Gemini CLI
npx get-shit-done-cc --codex --global    # Codex
npx get-shit-done-cc --copilot --global  # GitHub Copilot
```
