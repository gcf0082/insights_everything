# Anthropic Tokenomics Leaderboard 洞察报告

| 项目 | 内容 |

| **比较模型** | Claude Opus 4.6 vs Opus 4.7 |
| **数据来源** | 社区匿名提交的真实请求token对比 |

## 核心发现

### 1. Tokenizer差异

Claude Opus 4.7采用了新的tokenizer，与Opus 4.6相比：

- **相同输入文本**在Opus 4.7中可能产生**1.0-1.35倍**更多的token
- 这意味着虽然官方定价相同，但实际使用成本可能增加高达**35%**

### 2. 定价信息

| 定价维度 | Opus 4.6 | Opus 4.7 |
|----------|----------|----------|
| 输入（每百万token） | $5.00 | $5.00 |
| 输出（每百万token） | $25.00 | $25.00 |
| 上下文窗口 | 200K | 1M |
| 最大输出 | 128K | 128K |

**注意**：当上下文超过200K tokens时，进入Premium级别定价（输入$10/M，输出$50/M）

### 3. 性能提升

Opus 4.7在各项基准测试中超越Opus 4.6：

| 基准测试 | Opus 4.7 | Opus 4.6 | 提升 |
|----------|----------|----------|------|
| MCP-Atlas | 77.3 | 62.7 | +14.6 |
| CharXiv-R (with tools) | 91.0 | 77.4 | +13.6 |
| SWE-bench Pro | 数据待获取 | - | - |

### 4. 实际成本影响

由于tokenizer变化带来的实际成本影响：

- 同一任务：Opus 4.7可能消耗比Opus 4.6多出**20-35%**的token
- 这被用户称为"隐藏的35%成本陷阱"
- 企业用户（如Uber）已报告AI计算预算消耗显著增加

### 5. 新功能特性

Opus 4.7引入：

- **xhigh effort**模式：介于high和max之间的新 Effort 级别
- **任务预算（Task Budgets）**：用于控制 spending
- **/ultra view命令**：用于深度代码审计
- **auto模式**：完全自主执行

## 技术背景

- **项目开源**：https://github.com/bllchmbrs/tokensmatter
- **数据存储**：仅存储匿名token-count对比记录，不存储原始prompt
- **技术栈**：Cloudflare Workers + D1

## 总结

tokens.billchambers.me/leaderboard 是一个社区驱动的token消耗对比工具，展示了Opus 4.6和Opus 4.7在真实请求中的token消耗差异。虽然官方定价保持不变，但Opus 4.7的新tokenizer导致相同文本消耗更多token，实际成本可能增加35%左右。这对于高volume的企业用户来说是一个需要重点关注的成本因素。
