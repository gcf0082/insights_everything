## 内容摘要

本文是关于Claude Code在2025年2月更新后变得无法用于复杂工程任务的讨论。作者StanAngeloff指出，自2025年2月/3月以来，Claude Code的质量明显下降，用户需要在CLAUDE.md中添加越来越多的指导规则。

### 主要问题

1. **"simplest fix"问题**: 当出现"simplest fix"这类短语时，模型会产生完全无用的代码，会故意破坏代码然后建议最简单的hack

2. **提前终止问题**: 模型会催促用户结束对话，说"我燃烧了太多token"或"这花了太多轮次"

3. **思考被编辑**: 用户无法看到模型的完整思考过程，Anthropic似乎在减少thinking budget

4. **质量退化**: 
   - 建议的代码质量明显下降
   - 在修复bug时倾向于用简单hack而不是正确修复
   - 频繁出现"oh wait"、"actually"等自我纠正
   - 不遵循CLAUDE.md中的指令

5. **OAuth认证问题**: 用户经常遇到OAuth超时错误，无法登录

### 社区反应

- **部分用户认同**: 许多人表示注意到了类似的退化，特别是在Opus 4.6模型中
- **部分用户反对**: 一些用户认为这可能是工作流程问题，而非模型问题
- **替代方案**: 有用户建议使用Codex作为替代
- **批评声音**: 用户对这种"黑箱"性质表示不满，认为AI公司可以 stealthily 降低质量

### 总结

该帖子反映了对Claude Code质量下降的广泛担忧，用户认为Anthropic可能在为了降低成本而减少模型的thinking budget，导致复杂工程任务变得难以完成。许多用户对缺乏透明度表示失望。
