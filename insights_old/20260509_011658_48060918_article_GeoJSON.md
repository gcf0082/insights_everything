## 摘要

GeoJSON 是一种用于编码各种地理数据结构的格式。它通过 JSON 格式表示地理要素，支持点、线、多边形等几何类型，并在 2016 年被 IETF 正式标准化为 RFC 7946。

## 详细内容

### 什么是 GeoJSON？

GeoJSON 是一种基于 JSON 的地理数据编码格式，专门用于表示地理空间数据。它能够编码多种类型的地理数据 structures，包括：

- 点（Point）
- 线（LineString）
- 多边形（Polygon）
- 多点（MultiPoint）
- 多线（MultiLineString）
- 多多边形（MultiPolygon）

### 数据结构示例

```json
{
  "type": "Feature",
  "geometry": {
    "type": "Point",
    "coordinates": [125.6, 10.1]
  },
  "properties": {
    "name": "Dinagat Islands"
  }
}
```

该示例展示了一个包含几何信息（点坐标）和属性信息（名称）的要素（Feature）对象。

### 核心概念

1. **几何对象（Geometry）**：表示地理形状，如点、线、多边形
2. **要素（Feature）**：几何对象附带属性信息的组合
3. **要素集合（FeatureCollection）**：多个要素的集合

### 标准化历程

- **2008年**：最初的 GeoJSON 规范发布
- **2015年**：IETF 成立 GeoJSON 工作组
- **2016年8月**：RFC 7946 正式发布，成为新的标准规范，取代了 2008 年的规范

## 总结

GeoJSON 作为一种轻量级的地理数据交换格式，凭借其基于 JSON 的易读性和广泛支持，已成为 Web 地图和地理信息系统中数据交换的重要标准。其标准化过程体现了地理信息行业对统一数据格式的持续需求。
