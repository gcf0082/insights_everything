## 事件概述

AWS 北弗吉尼亚（us-east-1）数据中心发生宕机，据报道原因为"热事件"(thermal event)，导致冷却系统过热。AWS 官方表示这是一个单一可用区(AZ)的问题，但实际影响范围可能更广。受影响的公司包括 FanDuel 和 Coinbase 等。

## 关键洞察

### 1. US-East-1 的特殊地位与风险

- **历史原因**：us-east-1 是 AWS 最早建立的区域，基于 1990 年代成立的 MAE-East（第一个非政府互联网交换点）
- **控制平面依赖**：许多 AWS 全局服务（如 IAM、STS）的控制平面依赖 us-east-1，即使你在其他区域运行也可能受到影响
- **CloudFormation 证书等必须使用 us-east-1**：这意味着即使采用多区域架构，也无法完全规避该区域的单点故障风险
- **规模最大、负载最高**：us-east-1 是 AWS 最大、最繁忙的区域，也是最复杂的区域

### 2. 冷却系统故障的技术分析

根据有数据中心经验的用户（AdamJacobMuller）的详细解释，冷却系统故障通常是多个小问题叠加的结果：

- 冷却设备通常按 N+1 或 N+3 配置，但设备维修需要专业人员和长 lead time
- 一个部件（如轴承）损坏可能导致相邻设备负载增加
- 负载增加使原本正常的设备（如松动的保险丝）也出现故障
- 如果此时有大型客户开始批量处理作业（如金融公司开盘前的模型训练），会产生大量热量
- 最终形成级联故障：冷却 N-4 → N-5 → N-7 → 0

### 3. 多区域/多可用区冗余的局限性

- 虽然多区域/多可用区架构是 AWS 推荐的实践，但评论指出实际上很少有公司真正实现多 AZ 故障切换
- 用户 sitzkrieg 表示："使用 AWS 这么久，从未见过任何大公司实现多 AZ 故障切换，默认区域就好"
- 当 us-east-1 出现故障时，us-east-2 也会受到"连带影响"（用户迁移造成容量压力）
- 核心 AWS 服务依赖 us-east-1，即使你的服务在其他区域，也可能被波及

### 4. 数据中心选址的考量

关于为何数据中心不建在海边（便于冷却）：

- **盐水的腐蚀问题**：即使使用热交换器，盐水仍会腐蚀设备
- **海洋生物堵塞**：水母、贝类、藻类会堵塞冷却系统（参考 Diablo Canyon 核电站案例）
- **成本因素**：沿海土地昂贵，且可能面临飓风等极端天气风险
- **地理位置的历史因素**：Ashburn VA 成为数据中心枢纽是因为这里是第一个大型 IXP 所在地

### 5. 备份系统的局限

有经验的用户分享了"有备份系统但仍失败"的案例：

- 电源反复开关会耗尽电池，而每次来电会导致发电机关闭
- 操作员电脑有 UPS 但显示器没有（不同电源接口）
- 发电机运行时无法加油

结论：**从未执行的应急计划只是希望和幻想**

### 6. AWS 的应对措施

- AWS 确实进行了负载卸载( load shedding)，但可能不够及时
- Spot 实例价格飙升 → 无新实例可用 → 可抢占实例关闭 → 普通实例关闭
- 降低时钟频率（throttling）很困难，因为现代 Nitro 架构的虚拟机 passthrough 设计

### 7. Coinbase 夸大故障范围

- Coinbase 声称多个 AZ 受到影响，但 AWS 官方表示只有一个 AZ 有问题
- 评论认为"永远不要相信加密公司的诚实"
