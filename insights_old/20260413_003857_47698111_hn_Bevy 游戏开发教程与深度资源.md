## 内容摘要

本帖分享了一个名为 Tainted Coders 的 Bevy 游戏引擎教程网站，该网站提供与 Bevy 0.18 版本保持同步的免费深度学习资源。社区对这些资源表示高度认可，同时也讨论了 Bevy 引擎的优缺点。

## 主要讨论话题

### 1. 资源推荐与评价

- **Tainted Coders 网站**：提供与 Bevy 0.18 版本同步的免费教程，被评价为"目前唯一保持更新的文字教程"
- **Bevy Book**：官方正在开发中的文档，预计在下一个引擎版本发布时首次公开
- **其他资源**：
  - Chris Biscardi 的付费和免费资源 (rustadventure.dev)
  - Bevy 官方示例
  - Pikuma 的 C++ 2D 游戏引擎开发课程
  - Lazyfoo 的 SDL 教程
  - Handmade Hero

### 2. Bevy 的主要优势

- **ECS 架构**：提供良好的数据组织方式
- **状态保存/加载**：方便快速迭代开发
- **易于测试**：可以启动无头应用进行单元测试，可配合 Cucumber 进行验收测试
- **免费资源高质量**：社区贡献的免费教程质量较高

### 3. Bevy 的主要问题

#### 编译时间过长
- 编译时间是开发的主要瓶颈
- 即使使用高性能 CPU（如 7950X），编译仍需等待较长时间
- 建议使用 Bevy 官方提供的快速编译模式

#### 二进制体积过大
- 简单的 "hello world" 程序二进制达 50MB
- 调试版本可达 700MB
- WASM 版本体积甚至超过 Unity 构建的游戏

#### ECS 架构的学习曲线
- 开发者经常困惑于某物应该定义为 Component 还是 field
- ECS 作为较新的架构，其设计模式仍是"民间传说"级别
- 缺乏成熟的最佳实践指导

### 4. 社区反应

- 资源保持更新的做法受到高度赞赏
- 有人指出"Rust 游戏引擎比 Rust 游戏多"的现象
- 对于是否应该使用 Bevy 存在分歧：
  - 支持者认为 ECS 带来的优势值得付出编译时间代价
  - 反对者认为编译速度对游戏开发至关重要，快速迭代是游戏开发的核心

## 总结

Bevy 作为 Rust 语言的游戏引擎，虽然拥有现代化的 ECS 架构和良好的资源支持，但其过长的编译时间和较大的二进制体积仍是主要痛点。社区建议初学者可以利用快速编译模式，并且对于 Web 游戏开发需谨慎考虑。
