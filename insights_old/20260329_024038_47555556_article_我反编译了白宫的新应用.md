**主题**: 白宫Android应用安全与隐私分析

---

## 概述

本文作者对白宫新发布的官方Android应用进行了反编译分析，发现了该应用存在多个严重的隐私和安全问题。

## 应用技术架构

- **框架**: React Native + Expo (SDK 54)
- **JavaScript引擎**: Hermes
- **后端**: WordPress + 自定义REST API
- **构建方**: forty-five-press

应用逻辑编译为5.5 MB的Hermes字节码 bundle，原生Java层仅为薄封装。

## 主要发现

### 1. Cookie/付费墙绕过注入器

应用内置WebView打开外部链接时会注入JavaScript代码，强制隐藏：
- Cookie横幅
- GDPR同意对话框
- OneTrust弹窗
- 隐私横幅
- 登录墙/注册墙
- 付费墙元素

这是美国政府官方应用正在向第三方网站注入脚本以移除其Cookie同意对话框、GDPR横幅、登录门控和付费墙。

### 2. 位置追踪基础设施

尽管Expo配置中有 `withNoLocation` 插件，但OneSignal SDK的原生位置追踪代码已完整编译进APK：
- 前台GPS追踪：每4.5分钟一次
- 后台GPS追踪：每9.5分钟一次

追踪需要满足三个条件：`setLocationShared(true)`、用户授予位置权限、设备有位置提供商。数据同步至OneSignal服务器。

### 3. OneSignal用户画像

OneSignal功能远超推送通知，包括：
- 用户标签添加（addTag）
- 电话号码关联（addSms）
- 跨设备用户识别（addAliases）
- 用户行为和转化追踪
- 通知点击追踪
- 完整的应用内消息生命周期追踪

### 4. 供应链安全风险

**YouTube嵌入**: 使用 `react-native-youtube-iframe` 库，从 `lonelycpp.github.io`（个人GitHub Pages）加载播放器HTML。若该账户被攻破，可向应用用户注入任意代码。

**Elfsight小部件**: 从 `elfsightcdn.com/platform.js` 加载第三方JavaScript来嵌入社交媒体内容，无沙箱隔离。

**其他第三方服务**:
- Mailchimp处理邮件订阅
- Uploadcare托管图片
- Truth Social嵌入
- Facebook页面插件

### 5. 开发产物泄露

生产构建中包含开发 artifacts：
- 本地主机URL（React Native Metro bundler）
- 开发者本地IP：`10.4.4.109`
- Expo开发客户端库
- 导出的Compose PreviewActivity

### 6. 无证书锁定

应用使用标准Android TrustManager，无自定义证书锁定，易受中间人攻击。

## SDK生态

应用集成了68+个库，包括：
- React Native、Expo SDK 54、Hermes
- OneSignal、Firebase Cloud Messaging
- Firebase Analytics、Google Data Transport
- OkHttp 3、Apollo GraphQL
- Fresco、Glide、Coil 3
- ExoPlayer (Media3)
- Google ML Kit Vision
- 25个原生 .so 库

## 总结

白宫官方Android应用存在以下问题：

1. 向用户打开的每个网站注入JavaScript以隐藏Cookie同意、登录墙和付费墙
2. 完整GPS追踪管道已编译进应用，可每4.5分钟获取位置
3. 从随机人员GitHub Pages加载YouTube嵌入代码
4. 从Elfsight加载第三方JavaScript无沙箱
5. 邮件发送至Mailchimp，图片托管于Uploadcare
6. 无证书锁定
7. 生产版本携带开发产物
8. 通过OneSignal进行广泛用户画像

这些做法虽可能不违法，但显然不符合公众对政府应用的期望。
