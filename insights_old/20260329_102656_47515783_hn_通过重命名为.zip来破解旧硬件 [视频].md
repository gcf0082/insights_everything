## 内容摘要

这是一个关于如何通过将文件重命名为.zip来破解旧硬件的视频。讨论很快转向了一个更广泛的话题：智能设备的计划性报废问题。

## 热门评论摘要

### 1. 设备废弃问题（106 points，热门评论）

**morsch** 分享了他收到 Breville Joule Sous Vide 设备即将停止支持的公告：
- 现有应用将停止工作
- 新应用仅在美国、加拿大和部分欧洲地区可用
- 设备的核心功能（设置目标温度、获取当前温度、检查错误状态）本不需要互联网连接
- 用户需要创建新账户并接受新的服务条款

### 2. 讽刺的"功能"（高赞回复）

**red_admiral** 指出新应用的宣传语"With Breville+ Cooking, you'll get: ... The ability to cook with or without WiFi anywhere, anywhere"——人类可悲到需要将"无需WiFi也能烹饪"作为卖点来宣传。

### 3. 计划性报废的阴谋

**sigbottle** 评论道：每次读到这类文章，都会印证那些关于物联网设备走向"租赁化"的担忧，但不希望被验证。

**RajT88** 指出：带WiFi的厨房电器几乎注定在10年后无法使用，需要购买新产品——这是一种变相的计划性报废。

### 4. 正面案例

**nkrisc** 提到他的 Anova Sous Vide 设备（约10年历史）即使没有应用也能完全工作，虽然后来应用转为订阅制，但该公司为所有原用户保留了终身免费订阅。

### 5. 互联网设备的风险

**WalterBright** 分享经历：过去25年买了4台互联网收音机，使用几年后都因为远程服务器关闭而变砖。

**EvanAnderson** 建议：需要立法解决——如果设备需要在线服务才能运行，应明确告知用户这是"租赁"而非"购买"；制造商停止服务时应负责回收电子垃圾。

### 6. 技术讨论：ZIP格式的妙用

评论区还讨论了各种文件格式本质上是ZIP文件：
- **.ipa** (iOS应用) = ZIP
- **.apk** (Android应用) = ZIP  
- **.docx/.xlsx** = ZIP
- **.epub** = ZIP
- **.jar/.war** = ZIP

**charcircuit** 提到可以使用 Claude Code 来逆向Java应用，即使完全混淆也能恢复有意义的名称。

## 核心洞察

1. **物联网设备的寿命问题**：智能设备往往依赖云服务，一旦制造商停止支持，设备即变砖
2. **"拥有权"的消逝**：用户购买的其实是"租赁权"，制造商可以通过停止服务来"回收"设备功能
3. **本地控制的重要性**：不带应用也能工作的设备更具长期价值
4. **逆向工程的力量**：通过分析APK/IPA等文件，可以绕过厂商限制，实现设备再利用
5. **监管需求**：需要立法要求制造商在设备无法使用后承担回收责任
