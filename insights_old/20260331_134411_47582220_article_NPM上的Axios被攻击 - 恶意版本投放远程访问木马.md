# 洞察报告：Axios NPM包遭恶意篡改 — 远控木马植入事件

**洞察链接**：https://www.stepsecurity.io/blog/axios-compromised-on-npm-malicious-versions-drop-remote-access-trojan

**报告日期**：2026年3月31日

**报告来源**：StepSecurity

**事件概述**：2026年3月30日，StepSecurity发现npm上两个恶意版本的axios HTTP客户端库（1.14.1和0.30.4），攻击者通过入侵维护者账户发布恶意包，在依赖中植入远控木马（RAT）dropper，目标是macOS、Windows和Linux系统。

---

## 一、事件背景

### 1.1 什么是axios？

axios是JavaScript生态中最流行的HTTP客户端库，每周下载量超过3亿次，广泛用于Node.js和浏览器应用中。任何运行`npm install`或`npm update`的开发者都可能无意中部署恶意软件。

### 1.2 攻击时间线

| 时间（UTC） | 事件 |
|------------|------|
| 2026-03-30 05:57 | `plain-crypto-js@4.2.0`发布（无害伪装包） |
| 2026-03-30 23:59 | `plain-crypto-js@4.2.1`发布（含恶意postinstall） |
| 2026-03-31 00:21 | `axios@1.14.1`发布（注入恶意依赖） |
| 2026-03-31 01:00 | `axios@0.30.4`发布（39分钟后针对0.x分支） |

---

## 二、攻击手法分析

### 2.1 维护者账户入侵

攻击者入侵了axios项目主要维护者`jasonsaayman`的npm账户，将注册邮箱改为`ifstap@proton.me`（攻击者控制的ProtonMail地址）。利用该账户发布恶意版本，绕过正常的GitHub Actions CI/CD流程。

**关键识别点**：合法版本通过GitHub Actions的OIDC Trusted Publisher发布，而恶意版本`_npmUser`显示为`jasonsaayman`，无`trustedPublisher`字段。

### 2.2 恶意依赖注入

攻击者在`axios@1.14.1`和`axios@0.30.4`中添加了一个从未在axios代码中使用的依赖`plain-crypto-js@4.2.1`，该包唯一目的是执行`postinstall`脚本来投放远控木马。

---

## 三、远控木马dropper技术分析

### 3.1 混淆技术

`setup.js`使用两层混淆：
- **第一层**：`_trans_1(x, r)` — XOR密码，使用键`"OrDeR_7077"`
- **第二层**：`_trans_2(x, r)` — Base64解码+反转

### 3.2 C2服务器信息

- **C2地址**：`http://sfrclak.com:8000/6202033`
- **C2 IP**：`142.11.206.73`

### 3.3 平台特定Payload

| 平台 | 攻击方式 | 持久化位置 |
|------|----------|------------|
| macOS | AppleScript下载RAT | `/Library/Caches/com.apple.act.mond` |
| Windows | VBScript+PowerShell RAT | `%PROGRAMDATA%\wt.exe` |
| Linux | Python RAT | `/tmp/ld.py` |

### 3.4 证据销毁

攻击完成后，dropper会：
1. 删除自身（`setup.js`）
2. 删除`package.json`
3. 将预置的清洁stub（`package.md`）重命名为`package.json`

这使得事后检查`node_modules/plain-crypto-js/`时无法发现恶意痕迹。

## 五、影响评估与检测

### 5.1 是否受影响？

执行以下命令检测：

```bash
# 检查是否安装了恶意版本
npm list axios 2>/dev/null | grep -E "1\.14\.1|0\.30\.4"

# 检查plain-crypto-js目录
ls node_modules/plain-crypto-js 2>/dev/null && echo "POTENTIALLY AFFECTED"

# 检查RAT artifacts
# macOS
ls -la /Library/Caches/com.apple.act.mond
# Linux
ls -la /tmp/ld.py
# Windows
dir "%PROGRAMDATA%\wt.exe"
```

## 六、修复建议

1. **降级到安全版本**：
   ```bash
   npm install axios@1.14.0   # 1.x用户
   npm install axios@0.30.3    # 0.x用户
   ```

2. **添加overrides阻止恶意版本**：
   ```json
   {
     "overrides": {
       "axios": "1.14.0"
     }
   }
   ```

3. **删除恶意依赖**：
   ```bash
   rm -rf node_modules/plain-crypto-js
   npm install --ignore-scripts
   ```

4. **轮换所有凭证**：在运行过恶意包的机器上轮换npm token、AWS密钥、SSH私钥等所有敏感凭证。

5. **CI/CD策略**：在CI/CD中使用`npm ci --ignore-scripts`阻止postinstall脚本执行。

6. **网络封禁**：在防火墙层面封禁C2地址`142.11.206.73`或添加hosts记录。

---

## 七、结论

这是一次极其精密的供应链攻击：
- 攻击者提前18小时预置恶意依赖
- 针对三个操作系统预构建不同payload
- 在39分钟内同时发布两个分支的恶意版本
- 完整的证据销毁机制

**如果已安装axios@1.14.1或@0.30.4，请立即假设系统已遭到入侵，并按照上述修复建议采取行动。**
