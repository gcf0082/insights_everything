## 核心洞察

### 1. 问题背景

作者在撰写技术博客时使用 Excalidraw 制作图表，但每次修改图片都需要繁琐的导出流程：

1. 选中 frame
2. 点击导出
3. 选择正确的文件名 + 深色/浅色模式后缀
4. 导出
5. 切换深色/浅色模式
6. 重复选择文件名
7. 再次导出
8. 检查是否有元素超出 frame 边界
9. 重复以上步骤

整个过程需要约 **45秒**，频繁调整图片时非常痛苦。

### 2. 解决方案演进

#### 方案一：GitHub Action 自动导出

- 检测 Git push 中变化的 `.excalidraw` 文件
- 使用 `jq` 提取 frame 名称
- 自动导出深色和浅色模式的 SVG
- 自动提交到仓库

**存在问题**：
- excalirender 库存在渲染 bug
- 无法在 ARM Mac 本地运行（需要 x86 Docker）

#### 方案二：Fork Excalidraw VSCode 扩展（最终方案）

与 Claude 合作开发，在 VSCode 扩展中实现：

1. 用 frame 包裹要导出的元素
2. 将 frame 命名为 `export_${image_name}`
3. 扩展自动检测变化并导出两个 SVG：
   - `${image_name}.light.exp.svg`（浅色模式）
   - `${image_name}.dark.exp.svg`（深色模式）

### 3. 核心价值

- **本地实时预览**：修改 Excalidraw 后图片立即更新
- **提升效率**：整个项目仅花费数小时
- **解决真实痛点**：自动化繁琐的手动操作
