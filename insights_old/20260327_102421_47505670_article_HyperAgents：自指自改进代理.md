## 项目概述

Hyperagents 是 Meta AI 研究院开发的一种**自指涉自我改进智能体**，能够针对任何可计算任务进行优化。该项目将任务智能体（解决目标任务）和元智能体（修改自身和任务智能体）整合到一个单一的可编辑程序中，实现了**元认知自我修改**能力。

## 核心创新

### 1. 自我指涉架构

Hyperagents 的核心创新在于其自我指涉特性：
- **任务智能体（Task Agent）**：负责解决目标任务
- **元智能体（Meta Agent）**：负责修改任务智能体和自身
- 两个智能体集成到同一个可编辑的程序中

### 2. 元认知自我修改

与传统的自我改进系统不同，Hyperagents 不仅能够改进任务解决行为，还能够改进生成改进的机制本身。这种**元级别的修改程序**也是可编辑的，从而实现了真正的元认知自我修改。

### 3. 消除领域假设

Hyperagents 扩展了 Darwin Gödel Machine（DGM），创建了 DGM-Hyperagents（DGM-H），消除了在编码领域之外任务性能与自我修改能力之间对齐的假设，从而可能支持在任何可计算任务上实现自我加速的进展。

## 技术特性

### 文件结构

```
hyperagents/
├── agent/              # 基础模型代码
├── analysis/           # 绘图和分析脚本
├── domains/           # 各领域代码
├── utils/             # 通用代码
├── meta_agent.py      # 元智能体实现
├── task_agent.py      # 任务智能体实现
├── generate_loop.py   # 算法入口点
└── run_meta_agent.py  # 运行元智能体的脚本
```

### 支持的模型

项目支持多种大语言模型 API：
- OpenAI API
- Anthropic API
- Gemini API

### 实验领域

项目在多个不同领域进行了实验，验证了 Hyperagents 的通用性和有效性。

## 实验结果

根据论文描述，DGM-Hyperagents 在以下方面表现出色：

1. **性能随时间提升**：在多样化领域中，性能随时间推移不断提升
2. **超越基线**：优于没有自我改进或开放探索的基线方法
3. **优于先前系统**：超越此前的自我改进系统
4. **元级改进可迁移**：改进生成新智能体的过程（如持久化内存、性能跟踪）在不同领域之间可迁移
5. **跨运行累积**：元级别的改进可以在不同运行之间累积

## 安全考虑

## 安装与运行

### 环境要求

- Python 3.12
- graphviz
- cmake
- ninja-build

### 快速开始

```bash
# 1. 配置 API 密钥（在 .env 文件中）
OPENAI_API_KEY=...
ANTHROPIC_API_KEY=...
GEMINI_API_KEY=...

# 2. 创建虚拟环境
python3.12 -m venv venv_nat
source venv_nat/bin/activate
pip install -r requirements.txt
pip install -r requirements_dev.txt

# 3. 运行 Hyperagents
python generate_loop.py --domains <domain>
```

## 总结

Hyperagents 代表了自我改进 AI 系统的重要进展，通过引入自指涉架构和元认知自我修改能力，使智能体能够不仅改进其任务解决能力，还能改进其自我改进的机制。这一创新消除了此前自我改进系统对特定领域对齐假设的依赖，为构建真正开放式的 AI 系统开辟了新的可能性。
