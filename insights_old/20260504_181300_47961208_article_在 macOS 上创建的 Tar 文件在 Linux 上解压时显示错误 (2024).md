# 洞察报告

**链接**: https://aruljohn.com/blog/macos-created-tar-files-linux-errors/
**标题**: Tar Files Created on macOS Display Errors When Extracting on Linux
**发布日期**: 2024年12月6日

---

## 问题描述

在macOS上创建的tar.gz文件传输到Linux服务器解压时，会出现以下问题：
- 出现额外的`. `_`开头的文件（似乎是原始文件的副本）
- 出现警告信息，如：
  - `tar: Ignoring unknown extended header keyword 'LIBARCHIVE.xattr.com.apple.quarantine'`
  - `tar: Ignoring unknown extended header keyword 'LIBARCHIVE.xattr.com.apple.lastuseddate#PS'`
  - `tar: Ignoring unknown extended header keyword 'LIBARCHIVE.xattr.com.apple.metadata:kMDItemTextContentLanguage'`

## 原因分析

macOS默认使用BSD tar程序，会自动创建并添加这些额外文件和扩展属性。

## 解决方案

### 方案一：使用 `--no-xattrs` 选项

创建tar文件时添加`--no-xattrs`参数：

```bash
tar -cvzf --no-xattrs pix.tar.gz pix
```

### 方案二：使用 `--disable-copyfile` 选项

```bash
tar -cvzf --disable-copyfile pix.tar.gz pix
```

### 方案三：安装并使用GNU tar（推荐）

1. 通过Homebrew安装GNU tar：
   ```bash
   brew install gnu-tar
   ```

2. 在`~/.bash_profile`中设置路径：
   - Intel处理器Mac：
     ```bash
     export PATH="/usr/local/opt/gnu-tar/libexec/gnubin:$PATH"
     ```
   - Apple Silicon M1/M2/M3/M4：
     ```bash
     export PATH="/opt/homebrew/opt/gnu-tar/libexec/gnubin:$PATH"
     ```

3. 验证安装：
   ```bash
   tar --version
   ```

4. 之后可正常使用：
   ```bash
   tar -cvzf pix.tar.gz pix
   ```

## 结论

推荐使用方案三（安装GNU tar），一劳永逸地解决macOS与Linux之间tar文件的兼容性问题。
