## 项目概述

**Advanced Mac Substitute (AMS)** 是一个在API层面重新实现1980年代Mac OS的项目，类似于"Wine for Classic Mac OS"。它允许经典的Mac应用程序在现代硬件上运行，而无需完整的硬件模拟。

项目网站：https://www.v68k.org/advanced-mac-substitute/

## 核心技术架构

- **后端**: 68K模拟器，采用模块化设计，通过Unix-domain sockets连接各组件，可移植到任何POSIX系统
- **前端**: 支持多种显示方案，包括SDL2、X11、Linux fbdev、VNC以及macOS的五个不同API
- **通信**: 使用FIFO和共享内存进行前后端通信

## 主要讨论点

### 1. 与历史项目的比较

- **ARDI Executor**: 1990年代的商业软件，采用相同的API级重新实现方法，在90年代的PC硬件上运行Mac软件速度惊人。该项目已于2005年停止开发，后开源至GitHub。
- **Basilisk II**: 另一个Mac模拟器，有人正在为其添加ARM64 JIT支持。

### 2. 经典Mac OS的兼容性

- 1980年代软件依赖二进制API兼容性而非特定实现细节
- Mac Toolbox调用主要通过68K的A-line陷阱指令实现
- 大多数应用程序通过OS获取帧缓冲区地址和边界，而非硬编码
- 部分游戏依赖特定CPU时钟速度

### 3. 实际应用场景

- 用户分享了在小学计算机房使用Macintosh SE/Plus的经历
- 讨论了运行经典游戏如Dark Castle的可能性
- 提到Macromedia Freehand等老旧生产力软件的运行需求

### 4. 当前限制

- OpenDF调用尚未完全实现，部分应用启动时会报错但可忽略
- 某些应用程序如Macintosh Common Lisp需要精确的模拟器

## 社区反应

普遍对该项目表示赞赏，认为这是"经典Mac OS的Wine"，为在现代硬件上运行老旧应用提供了可能。有人将其与Web模拟器进行了比较，指出Web架构不适合构建复杂结构，这也是促使创建AMS的原因之一。
