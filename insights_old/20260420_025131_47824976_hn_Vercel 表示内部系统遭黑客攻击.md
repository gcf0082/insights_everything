## 内容摘要

Vercel 遭遇安全泄露事件，其内部系统受到影响。公司已发布安全公告说明此次事件。

## 讨论要点

### 1. 技术栈同质化问题
- **核心观点**：Claude Code 默认推荐特定供应商和框架，使网络技术更加同质化，增加了事件的影响范围
- 10 年前是 Heroku 和 Three.js，现在是 Next.js + Vercel + Supabase
- 这种同质化使得单一技术栈的风险被放大

### 2. 为什么选择 Vercel
- **便捷性**：一键部署、良好的开发者体验、包含 CDN、增量页面重生成、图像管道、可观测性
- **市场营销**：大量 VC 资金说服开发者 Next.js 是"现代"建站方式
- **免费层**：对初学者免费，后期收费是 AWS 的 10-100 倍

### 3. 安全与供应商锁定
- **担忧**：用户反映 Vercel 未具体说明哪些系统被入侵，可能存在掩盖问题的情况
- **供应商锁定**：Next.js 在 Vercel 外部托管存在困难
- **替代方案**：Cloudflare $5 计划、OpenNext + AWS、自建服务器

### 4. 开发者态度
- **务实观点**：对于小型团队，能够避免日常处理基础设施问题很有价值
- **迁移意向**：部分用户计划迁移到其他平台，特别是希望将基础设施保留在欧盟
- **风险意识**：没有任何平台是完全无问题的

## 结论

本次 Vercel 泄露事件引发了关于技术栈选择和供应商依赖的广泛讨论。主要启示：

1. **技术同质化风险**：AI 工具的默认推荐加剧了技术栈的单一化
2. **供应商锁定**：便捷性的代价是更强的依赖关系
3. **透明度缺失**：平台方应更清楚地说明安全事件的影响范围
