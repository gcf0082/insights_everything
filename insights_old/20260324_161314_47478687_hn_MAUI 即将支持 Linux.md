## 核心内容总结

### 公告概要

Avalonia 团队宣布推出 **Avalonia backend for .NET MAUI** 预览版，这一成果使得开发者可以将 .NET MAUI 应用部署到 Linux 和 WebAssembly 平台。自去年秋季以来，团队取得了重大进展，成功将 Avalonia 的强大功能引入 .NET MAUI 生态。

### 技术实现

开发者可以通过以下四个步骤快速上手：

1. 创建一个 .NET MAUI 应用
2. 添加 `Avalonia.Controls.Maui.Desktop` NuGet 包
3. 添加 net11.0 目标框架
4. 在 `MauiBuilder` 中添加 `UseAvaloniaApp`

无需创建 Avalonia 启动器，团队已经提供了完整的解决方案。

### 对 Avalonia 的改进

这一项目为 Avalonia 本身带来了显著改进。为了避免实现 .NET MAUI 特定控件的需求，团队努力缩小 .NET MAUI 和 Avalonia 之间可用控件集的差距。最显著的成果之一是 Avalonia 12 引入了新的导航 API 和控件，这些改进直接源于为 .NET MAUI 提供支持的工作。

### 测试的应用

团队通过移植现有的 .NET MAUI 应用来测试新库，包括：

- **MauiPlanets** - 展示应用
- **2048 游戏** - 游戏示例
- **.NET MAUI Control Gallery** - 用于测试和演示服务和控件
- **AlohaAI** - 与 GitHub Copilot 合作开发的教育应用
- **MyConference** - 演示"Agentic AI"开发的会议应用
- **WeatherTwentyOne** - 使用新开源 WebView 的天气应用

### 原生还是绘制？

 Avalonia 为 .NET MAUI 用户提供了选择：
- **原生控件**：使用操作系统自带控件，与宿主系统更加融合
- **绘制控件**：在所有平台上保持一致的外观和行为

两种方案各有优势，Avalonia MAUI 让开发者有更多控制权。

### 未来计划

- 开发基于 Avalonia 的 Maui.Essentials 自定义实现
- 实现与 WinUI 的互操作性
- 为控件库作者提供扩展到绘制方法的简单模式
- 随着 .NET 11 正式版的发布，目标是在今年推出正式版本

## 社区讨论要点

1. **Wayland 支持**：MikeCodesDotNET 表示正在积极开发 Avalonia 12 的 Wayland 支持，并决定保持 MIT 许可证的简单性

2. **Linux/ Wayland 挑战**：评论者指出，Linux 上渲染窗口有多种方式（XDG Top Level Window、Child Window、Popup Surface、Layer surface 等），很难找到支持所有这些功能的高级工具包

3. **不同桌面环境的差异**：讨论提到 GNOME 和 KDE 在 Wayland 上的实现差异，需要为不同桌面环境编写特定的后端

4. **与 Vulkan 的类比**：评论者将 Wayland 协议扩展的方式与 Vulkan 进行类比，认为这种细粒度和独立的扩展方式在两者中都有些过度设计

## 技术价值

这一发布标志着 .NET 跨平台开发的一个重要里程碑。开发者现在可以使用单一的代码库将应用部署到 Windows、macOS、iOS、Android、Linux 以及 WebAssembly，为 .NET 生态系统的跨平台一致性迈出了重要一步。
