## 核心洞察摘要

**Claude Platform on AWS 现已正式全面上线**，为 AWS 客户提供了全新的方式访问完整的 Claude Platform 功能，包括 AWS 身份认证、计费和承诺额度抵扣。

---

## 详细洞察

### 1. 产品概述

Claude Platform on AWS 将完整的 Claude API 功能套件首次带给 AWS 客户，所有新功能和测试版功能都会在上线当天同步发布到原生 Claude API。

- **身份认证**：通过 AWS IAM 运行
- **审计日志**：通过 CloudTrail 记录
- **计费**：通过单一 AWS 发票，支持与现有承诺额度抵扣
- **可用区域**：大多数 AWS 商业区域
- **推理地域**：支持全球和美国推理地域

### 2. 主要功能

Claude Platform on AWS 包含以下原生平台功能：

- **Claude Managed Agents（测试版）**：大规模构建和部署代理
- **Advisor 策略（测试版）**：通过咨询顾问模型为代理提供智能提升
- **网络搜索和网络抓取**：获取当前网络数据增强 Claude 知识
- **代码执行**：直接在 API 调用中运行 Python 代码、创建可视化、分析数据
- **Files API（测试版）**：上传并在对话中引用文档
- **Skills（测试版）**：教会 Claude 最佳实践，确保一致的结果
- **MCP 连接器（测试版）**：无需编写客户端代码即可连接到任何远程 MCP 服务器
- **Prompt 缓存**：减少重复上下文的成本和延迟
- **引用**：基于源文档为响应提供依据
- **批处理**：高容量异步工作负载

### 3. 可用模型

- Claude Opus 4.7
- Claude Sonnet 4.6
- Claude Haiku 4.5

### 4. Claude Console 访问

客户还可以访问 Claude Console，这是 Anthropic 的 Claude 开发环境，包含：
- Prompt 优化器
- Prompt 生成器
- 评估工具

### 5. 与 Amazon Bedrock 的对比

| 特性 | Claude Platform on AWS | Claude on Amazon Bedrock |
|------|------------------------|---------------------------|
| **服务运营商** | Anthropic | AWS |
| **数据处理** | 在 AWS 边界外处理 | AWS 作为数据处理器 |
| **功能完整性** | 完整原生 API 功能 | 部分功能 |
| **适用场景** | 需要完整 Claude 平台体验的企业 | 有严格区域数据驻留要求的企业 |

### 6. 客户评价

- **ReliaQuest**：简化了 Claude 访问方式，改善了 Claude Code 工程师的体验，提供了将前沿 AI 能力整合到网络安全和工程工作流程的实用路径。
- **OpenRouter**：直接访问最新原生 Claude API 功能，支持通过相同的 AWS IAM 凭证控制访问，提供一致的性能和正常运行时间。
- **Emergent Solutions**：提供了规范的 Anthropic API，AWS 作为访问层，获得完整功能对等性和第一天访问新模型能力，支持服务出色。

### 7. 客户案例亮点

- **ReliaQuest**：Principal Research Scientist Jonathan Echavarria 表示，这有助于简化 Claude 访问，改善了 Claude Code 工程师等关键用户的体验。

## 关键要点

1. **完整功能访问**：AWS 客户现在可以通过 Claude Platform on AWS 访问完整的原生 Claude API 功能，无需切换到其他平台。

2. **集成优势**：
   - 使用现有的 AWS 凭证和 IAM 策略
   - 团队保持在已有的工具和权限管理范围内
   - 单一 AWS 发票计费，支持承诺额度抵扣

3. **差异化选择**：
   - 需要完整功能和快速迭代：选择 Claude Platform on AWS
   - 需要严格数据驻留和 AWS 基础设施：选择 Amazon Bedrock

4. **即时可用**：Claude Platform on AWS 现已正式上线，客户可立即开始使用。
