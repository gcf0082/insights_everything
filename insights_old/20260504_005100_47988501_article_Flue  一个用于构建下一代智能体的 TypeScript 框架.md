## 框架概述

Flue 是一个代理 harness 框架（Agent Harness Framework），其核心定位是帮助开发者构建强大的自主代理（Autonomous Agents）。与传统的 SDK 不同，Flue 提供了一个可编程的 TypeScript harness，允许开发者编写一次代码，即可在任何地方部署。

---

## 核心架构

Flue 的架构遵循现代代理设计模式，将代理定义为 **Model + Harness** 的组合：

```
04 / 文件系统：读取 · 写入 · grep · glob
03 / 沙箱：bash · 安全 · 网络
02 / Harness：技能 · 内存 · 会话
01 / 模型：tokens · 工具 · prompts
```

这种架构使得代理能够规划、获取上下文、编写文件、生成子代理、扮演角色并解决问题，功能远超简单的聊天机器人。

---

## 核心功能

### 1. 可编程的 Agent Harness
- 开发者完全控制代理的关键决策
- 支持技能（skills）作为可重用的工作流，返回结构化输出
- 通过会话来跟踪工作进度（类似 Claude Code 或 Codex）

### 2. 灵活的沙箱选项
- 支持连接远程沙箱（如 Daytona）
- 内置零配置的虚拟沙箱
- 提供细粒度的安全控制，保护敏感 API 密钥和令牌

### 3. 多场景支持
- **AI Issue Triage**：自动分类 GitHub 问题
- **数据分析代理**：分析数据并生成报告
- **编码代理**：处理代码仓库任务
- **客服代理**：响应客户消息

### 4. 部署灵活性
- 打包为 HTTP 服务器，可部署到任何平台
- 支持 CLI 直接运行，适合本地任务和 CI/CD
- 支持：Node.js、Cloudflare Workers、GitHub Actions、GitLab CI/Vercel、Netlify、Railway、Render、Fly.io

## 技术特点

### 代码示例：Issue Triage（22 行 TypeScript）

```typescript
export default async function ({ init, payload, env }: FlueContext) {
  const { issueNumber } = payload;
  const agent = await init({ model: 'anthropic/claude-opus-4-7' });
  const session = await agent.session();

  const triage = await session.skill('triage', {
    args: { issueNumber },
    result: v.object({
      severity: v.picklist(['low', 'medium', 'high', 'critical']),
      reproducible: v.boolean(),
      summary: v.string(),
    }),
  });

  const body = `**Severity:** ${triage.severity}\n**Reproducible:** ${triage.reproducible}\n\n${triage.summary}`;
  await new Octokit({ auth: env.GITHUB_TOKEN }).request(
    'POST /repos/{owner}/{repo}/issues/{num}/comments',
    { owner: 'withastro', repo: 'flue', num: issueNumber, body },
  );
}
```

## 总结

Flue 是一个面向下一代的代理框架，它让开发者能够构建真正自主的 AI 代理，而不是依赖于现成的工具。通过提供完整的堆栈控制（代理、harness、沙箱），开发者可以完全定制自己的工作流程，避免租赁他人代理的局限性。
