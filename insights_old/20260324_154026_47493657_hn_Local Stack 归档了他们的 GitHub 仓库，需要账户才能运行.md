## 事件概述

LocalStack 是一个用于本地 AWS 云服务模拟的开源工具，其 GitHub 仓库已被归档，现在需要账户才能使用。

## 主要讨论观点

### 1. 关于 Open Source 许可与义务
- **支持方观点**：LocalStack 按照 OSS 规则行事，代码仍然可用（Apache 许可证），公司有权决定是否继续开源
- **反对方观点**：公司利用开源社区的贡献来发展商业产品，然后突然关闭开源项目，这是一种"rugpull"（ rug pull）行为
- **许可说明**：MIT、Apache 等宽松许可证不要求永久维护源码；GPL 要求源码提供 3 年（仅当不是版权持有者时）

### 2. 道德争议
- **批评方**：这是"诱饵调包"（bait-and-switch），利用开源作为营销工具，当不再有利可图时即关闭开源项目
- **辩护方**：公司开源 10 年已尽到责任，社区可以 fork 自行维护
- **许可证选择**：如果贡献者希望代码永久开源，应选择 GPL 等保护性更强的许可证

### 3. 经济考量
- 公司需要盈利来维持运营，这并非不道德
- 99% 的公司从不开源任何东西
- 开发者不应期望永久免费维护

### 4. 替代方案讨论
- ** forks **：Terraform → OpenTofu、Redis → Valkey、MinIO 等项目被 fork 后继续开源
- **其他模拟器**：如 botocore、floci 等可作为替代

### 5. LocalStack 商业模式
- 此前已有商业付费版本，提供部分专有功能
- 现转向云端服务模式

## 总结

此事件引发了关于开源项目商业化的广泛讨论。核心争议在于：公司是否有权在利用开源社区后关闭项目？支持者认为代码仍可用且公司有权停止开源；批评者认为这是对社区信任的背叛。这一事件与 Elasticsearch/OpenSearch、MinIO 等案例类似。
