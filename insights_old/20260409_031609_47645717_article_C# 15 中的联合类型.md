## 摘要

C# 15 引入了 `union` 关键字，用于声明联合类型。联合类型是一种值，它必须是固定集合中的**恰好一种**类型，编译器强制进行穷尽性模式匹配。

---

## 核心内容

### 什么是联合类型？

在 C# 15 之前，当方法需要返回多种可能的类型时，存在以下不完美的选择：
- 使用 `object`：无法约束实际存储的类型
- 使用标记接口或抽象基类：无法"关闭"类型集，任何人都可以实现

联合类型解决了这些问题：
- 声明一个**封闭**的类型集合
- 案例类型之间不需要有共同祖先
- 编译器保证 `switch` 表达式是穷尽的

### 基本语法

```csharp
public record class Cat(string Name);
public record class Dog(string Name);
public record class Bird(string Name);

public union Pet(Cat, Dog, Bird);
```

使用示例：
```csharp
Pet pet = new Dog("Rex");
string name = pet switch
{
    Dog d => d.Name,
    Cat c => c.Name,
    Bird b => b.Name,
};
```

### 关键特性

1. **隐式转换**：每个案例类型都可以直接赋值给联合类型
2. **穷尽性匹配**：编译器保证覆盖所有案例类型
3. **模式匹配**：直接对 `Value` 属性进行模式匹配
4. **null 处理**：如果任何案例类型可空，所有 switch 表达式都需要 null 检查

### 带Body的联合类型

联合类型可以包含成员，例如 `OneOrMore<T>`：

```csharp
public union OneOrMore<T>(T, IEnumerable<T>)
{
    public IEnumerable<T> AsEnumerable() => Value switch
    {
        T single => [single],
        IEnumerable<T> multiple => multiple,
        null => []
    };
}
```

### 自定义联合类型

任何带有 `[System.Runtime.CompilerServices.Union]` 属性的类或结构，只要遵循基本模式（单参数构造函数 + Value 属性），都会被识别为联合类型。

### 相关提案

- **Closed Hierarchies**：`closed` 修饰符防止在定义程序集外部派生类
- **Closed Enums**：防止创建声明成员之外的值

这三个功能共同为 C# 提供了完整的穷尽性匹配支持。

## 入门指南

1. 安装 [.NET 11 Preview SDK](https://dotnet.microsoft.com/download/dotnet)
2. 创建或更新 targeting `net11.0` 的项目
3. 在项目文件中设置 `<LangVersion>preview</LangVersion>`

注意：在 .NET 11 Preview 2 中，需要自行声明 `UnionAttribute` 和 `IUnion` 接口。

---

## 总结

联合类型是 C# 长期以来的热门功能请求，现在终于在 C# 15 中实现。它提供了类型安全的联合值，支持编译器强制穷尽性检查，并与现有的模式匹配系统无缝集成。
