## 内容摘要

这篇文章介绍了一个名为 **Surelock** 的 Rust 库，旨在通过类型系统防止死锁。该库基于层级(LockLevel)概念，确保锁的获取顺序是单调的，从而避免传统的锁排序问题。

### 核心设计

1. **层级抽象**: 每个 Mutex 都有一个类型级别，必须按从高到低的顺序获取锁
2. **MutexKey**: 获取锁时返回一个 Key，必须用这个 Key 才能获取更低级别的锁
3. **LockSet**: 支持运行时同时获取多个锁，通过排序避免死锁

### 讨论要点

1. **地址不稳定性问题**: 有人质疑使用地址排序的有效性，指出如果 Mutex 被移动就无法锁定，使用地址是多余的
2. **多锁支持**: 有人担心同时支持多个锁会让 Mutex 变成 monitor lock
3. **共享内存场景**: 进程间共享内存时每个进程可能有不同的地址映射
4. **优先级反转**: 讨论了大锁和小锁的获取顺序可能导致优先级反转
5. **异步支持**: 询问该库是否支持异步代码；回复指出锁获取是阻塞的，且 key/guard 不是 Send，持有锁过 await 点会使 Future 不是 Send

### 关键技术观点

- **TVar 思想**: 有人提到数据库领域的 TVar 思想没有在更多编程语言中普及
- **编译时锁定**: 有人询问是否可以在编译时锁定，回复称使用了宏但因调试困难而放弃
- **与 Fuchsia 的 tree_lock 比较**: 这是对 tree_lock 的改进
