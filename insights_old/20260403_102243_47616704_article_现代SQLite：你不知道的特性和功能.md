# 现代SQLite：你可能不知道的功能

## 处理JSON数据

SQLite附带了JSON扩展，允许你直接在表中存储和查询JSON文档。你可以在保持schema灵活性的同时，使用SQL来组织和分析结构化数据。

示例：从JSON列提取字段：

```sql
CREATE TABLE events (
  id      INTEGER PRIMARY KEY,
  payload TEXT NOT NULL -- JSON
);

SELECT
  json_extract(payload, '$.user.id')   AS user_id,
  json_extract(payload, '$.action')    AS action,
  json_extract(payload, '$.metadata')  AS metadata
FROM events
WHERE json_extract(payload, '$.action') = 'login';
```

你还可以对JSON表达式创建索引，使半结构化数据的查询速度出奇地快。

## 使用FTS5进行全文搜索

SQLite的FTS5扩展使其成为一个功能强大的全文搜索引擎。你可以将所有内容保存在单个数据库文件中，而无需额外安装搜索服务。

示例：构建简单搜索索引：

```sql
CREATE VIRTUAL TABLE docs USING fts5(
  title,
  body,
  tokenize = "porter"
);

INSERT INTO docs (title, body) VALUES
  ('SQLite Guide', 'Learn how to use SQLite effectively.'),
  ('Local-first Apps', 'Why local storage and sync matter.');

SELECT rowid, title
FROM docs
WHERE docs MATCH 'local NEAR/5 storage';
```

你可以在不离开SQLite或不管理单独服务的情况下，获得排名、短语查询、前缀搜索等功能。

## 使用窗口函数和CTE进行分析

SQLite支持公用表表达式（CTE）和窗口函数，这些功能解锁了一类曾经常需要更重量级数据库的分析查询。

示例：使用窗口函数计算累计总量：

```sql
SELECT
  user_id,
  created_at,
  amount,
  SUM(amount) OVER (
    PARTITION BY user_id
    ORDER BY created_at
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
  ) AS running_total
FROM payments
ORDER BY user_id, created_at;
```

结合CTE，你可以在单个SQLite文件之上构建非常丰富的报表和仪表板。

## STRICT表和更好的类型系统

SQLite以其灵活的类型模型而闻名（或臭名昭著）。现代SQLite添加了STRICT表，它更严格地强制类型约束，非常像PostgreSQL或其他传统数据库。

示例：定义严格表：

```sql
CREATE TABLE users (
  id        INTEGER PRIMARY KEY,
  email     TEXT NOT NULL,
  is_active INTEGER NOT NULL DEFAULT 1
) STRICT;
```

使用严格表，无效类型会在插入时被拒绝，使schema更可预测，并减少微妙的bug——特别是在较大的代码库中。

## 生成列用于派生数据

生成列允许你将表达式存储为虚拟列或存储列，将派生数据保存在源数据附近，而无需在应用程序中重复逻辑。

示例：规范化的搜索字段：

```sql
CREATE TABLE contacts (
  id          INTEGER PRIMARY KEY,
  first_name  TEXT NOT NULL,
  last_name   TEXT NOT NULL,
  full_name   TEXT GENERATED ALWAYS AS (
    trim(first_name || ' ' || last_name)
  ) STORED
);

CREATE INDEX idx_contacts_full_name ON contacts(full_name);
```

现在每次插入或更新都会自动保持full_name同步，你可以高效地对其进行索引和查询。

## 预写日志和并发

预写日志（WAL）是一种日志模式，可以提高许多工作负载的并发性和性能。在常见情况下，读者不会阻止写入者，写入者也不会阻止读者。

启用WAL只需一条pragma语句：

```sql
PRAGMA journal_mode = WAL;
```

对于桌面应用程序、本地优先工具和小型服务，WAL模式可以显著提高感知性能，同时保持SQLite的简单性和可靠性。
