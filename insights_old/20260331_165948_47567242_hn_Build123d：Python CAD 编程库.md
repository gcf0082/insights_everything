## 主题概述

Build123d 是一个基于 Python 的 CAD 编程库，构建在 OpenCASCADE (OCCT) 内核之上，可视为 CADQuery 的继任者。该项目在 Hacker News 上引发了关于代码优先 CAD 工作流的广泛讨论。

## 主要讨论热点

### 1. 代码优先 CAD vs GUI CAD

讨论中普遍认为代码优先 CAD 工具与 GUI CAD 是互补而非替代关系。代码优先方法适合高度参数化和规则化的物体（如螺栓、齿轮），而 GUI CAD 更适合自由形式设计。

关键优势：
- 版本控制友好，可通过 diff 查看变更
- 避免重复点击，一个参数修改即可更新模型
- 代码组织和复用更便捷

### 2. OnShape FeatureScript 对比

OnShape 的 FeatureScript 被认为是理想范例：
- 自定义功能与内置功能享有同等地位
- 脚本自动成为历史建模器中的可用的节点
- 内置功能本身也是用 FeatureScript 编写的

Build123d 的定位与 FeatureScript 相似，但缺少与 GUI 的深度集成。

### 3. OpenSCAD 局限性

Build123d 解决了 OpenSCAD 的多个痛点：
- **B-Rep vs CSG**: OpenSCAD 使用 CSG（构造实体几何），无法进行圆角等边界操作；Build123d 基于 B-Rep
- **STEP 导出**: OpenSCAD 无法输出干净的 STEP 文件
- **局部坐标系**: OpenSCAD 缺少本地坐标系统，Build123d 鼓励使用
- **代码组织**: OpenSCAD 难以构建可复用代码

### 4. 混合 GUI/代码 CAD 愿景

用户强烈希望实现混合工作流：
- 在 GUI 中选择边、面、物体
- 将选择结果传递给代码脚本作为输入
- 脚本输出成为历史建模器的新状态

Build123d + OCP CAD Viewer 已提供部分功能：点击特征可复制索引到剪贴板，但存在"拓扑命名"问题导致选择脆弱。

### 5. 约束系统

Build123d 提供了约束系统，与 FreeCAD 的声明式约束求解器不同，采用命令式约束系统。存在第三方项目 solve123d 使用 JAX 添加数值求解器。

### 6. 生态工具

- **OCP CAD Viewer**: VS Code 扩展，用于可视化
- **build123d-sandbox**: Web 端 playground
- **TexoCad.ai**: 基于 Build123d 的封装，可生成 Python 代码和导出 STEP/3MF

## 总结

Build123d 为 Python 开发者提供了一个强大的代码优先 CAD 解决方案，特别适合需要版本控制和参数化设计的场景。与 OpenSCAD 相比，它基于 B-Rep 内核，支持 STEP 导入导出，更适合与主流 CAD 软件互操作。然而，混合 GUI/代码工作流仍是用户期望但尚未完全实现的功能。
