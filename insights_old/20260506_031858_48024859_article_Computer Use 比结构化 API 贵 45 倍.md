## 核心发现

### 成本对比

| 指标 | Vision Agent (Sonnet) | API Agent (Sonnet) | API Agent (Haiku) |
|------|---------------------|-------------------|------------------|-----------------|
| **步骤/调用数** | 53 ± 13 | 8 ± 0 | 8 ± 0 |
| **Wall-clock时间** | 1003s (~17分钟) | 19.7s | 7.7s |
| **Input Tokens** | 550,976 | 12,151 | 9,478 |
| **Output Tokens** | 37,962 | 934 | 819 |

### 关键结论

**Computer Use (视觉代理) 比结构化API调用贵约45倍：**

- **视觉代理消耗**：~551k输入tokens，~17分钟
- **API代理消耗**：~12k输入tokens，~20秒
- **成本差异**：输入tokens减少98%，时间减少98%

## 实验设计

### 测试任务

找到名为"Smith"且订单最多的客户，定位其最新待处理订单，接受所有待处理评论，并将订单标记为已交付。

### 测试应用

基于 react-admin Posters Galore demo 的管理面板（客户、订单、评论管理）。

### 两种路径

- **Path A (视觉代理)**：通过浏览器截图和点击操作 UI
- **Path B (API代理)**：直接调用应用的 HTTP 端点

## 重要发现

### 1. 视觉代理无法独立完成任务

在标准提示下，视觉代理只找到4个待处理评论中的1个，未能翻页处理剩余评论。因为页面上只能看到部分数据，代理没有信号知道还有更多内容需要查看。

### 2. 需要14步详细walkthrough才能完成

为使视觉代理完成任务，需要提供包含14个步骤的详细UI操作说明。这种提示工程本身代表了未被计入的真实成本。

### 3. 视觉代理结果高度不稳定

- 运行时间：749s - 1257s（差异68%）
- 输入tokens：407k - 751k
- 执行周期：43 - 68次

而API代理在5次运行中保持完全一致的结果。

### 4. 结构性差距

视觉代理的成本由界面结构决定，而不是模型能力。更好的模型可以减少每步的错误率，但无法减少所需的截图数量，因为每张截图代表数千个输入tokens。

## 结论

| 维度 | 推荐 |
|------|------|
| **第三方SaaS/无法修改的应用** | 使用视觉代理 |
| **内部工具/自主开发的应用** | 优先使用结构化API |

当API生成的工程成本降为零时，对于内部工具来说，构建结构化API的成本效益比视觉代理更高。
