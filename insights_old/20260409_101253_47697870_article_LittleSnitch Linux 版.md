## 产品概述

Little Snitch for Linux 是一款网络监控工具，能够让用户看到计算机上每个应用程序打开网络连接的活动，并提供选项来控制这些连接。用户可以准确查看哪些应用程序正在与哪些服务器通信，阻止不需要的连接，并持续监控流量历史和数据量。

## 系统要求

- Linux 内核 6.12 或更高版本
- 需要 BTF (BPF Type Format) 内核支持

## 核心功能

### 1. 连接监控
连接视图列出了当前和过去的网络活动，按应用程序分类显示。用户可以查看被规则和黑名单阻止的内容，跟踪数据量和流量历史。支持按最后活动、数据量或名称排序，并可以过滤列表以快速发现异常情况。阻止连接只需一次点击。

底部的流量图表显示数据量随时间的变化。用户可以拖动选择时间范围，缩放并过滤连接列表以仅显示该时段的 activity。

### 2. 黑名单管理
黑名单允许一次切断整类不需要的流量。Little Snitch 从远程来源下载黑名单并自动保持最新状态。支持多种常见格式：
- 每行一个域名
- 每行一个主机名
- /etc/hosts 风格（IP地址后跟主机名）
- CIDR 网络范围

不支持通配符格式、正则表达式或 glob 模式。推荐使用基于域名的列表而非基于主机的列表，处理效率更高。

注意：macOS 版本的 .lsrules 格式与 Linux 版本不兼容。

### 3. 自定义规则
黑名单在域名级别工作，但规则可以更深入。规则可以针对特定进程、匹配特定端口或协议，范围可宽可窄。规则视图允许排序和过滤，方便管理不断增长的规则列表。

### 4. 访问安全
默认情况下，Little Snitch 的 Web 界面对本地运行的任何人或任何应用程序都是开放的。行为不端或恶意应用程序原则上可以添加和删除规则、篡改黑名单或完全关闭过滤器。

如果担心安全问题，可以配置 Little Snitch 需要身份验证。

## 技术实现

Little Snitch 使用 eBPF (Extended Berkeley Packet Filter) 钩入 Linux 网络堆栈，这是一个允许程序观察和拦截内核中发生的情况的机制。eBPF 程序监视出站连接并将数据提供给守护进程，该守护进程跟踪统计信息、预处理规则并提供 Web UI。

eBPF 程序和 Web UI 的源代码托管在 GitHub 上。

## 高级配置

UI 有意只暴露最常见的设置。任何更技术性的配置都可以通过纯文本文件完成，这些文件在重启 `littlesnitch` 守护进程后生效。

默认配置位于 `/var/lib/littlesnitch/config/`。不要直接编辑这些文件，而是将要更改的文件复制到 `/var/lib/littlesnitch/overrides/config/` 并在那里进行编辑。Little Snitch 始终优先使用 override。

主要配置文件：
- **web_ui.toml** - 网络地址、端口、TLS 和身份验证
- **main.toml** - 当连接不匹配任何规则时的行为（默认允许，可改为拒绝）
- **executables.toml** - 用于合理分组的启发式算法

eBPF 程序和 Web UI 都可以替换为自己的构建版本。

## 使用限制

Little Snitch for Linux 是为隐私而非安全而构建的，这一区别很重要。macOS 版本可以做出更强的保证，因为它可以具有更多的复杂性。在 Linux 上，基础是 eBPF，虽然功能强大但有界限：它对存储大小和程序复杂度有严格限制。在高流量下，缓存表可能会溢出，这使得无法可靠地将每个网络数据包与进程或 DNS 名称关联起来。重建给定 IP 地址最初查询的主机名需要启发式方法而非确定性。macOS 版本使用深度数据包检测来更可靠地完成此操作。

Little Snitch for Linux 非常适合监控软件的行为并阻止合法软件打电话回家。对于强化系统以抵御坚定的攻击者，这不是正确的工具。

## 许可证

Little Snitch for Linux 有三个组件：
- eBPF 内核程序：GNU 通用公共许可证版本 2 (GPLv2)
- Web UI：GNU 通用公共许可证版本 2 (GPLv2)
- 守护进程 (littlesnitch --daemon)：专有，但免费使用和再分发

源代码均可在 GitHub 上获取。

## 总结

Little Snitch for Linux 是一款强大的网络监控和隐私保护工具，适用于 Linux 用户想要控制和监控应用程序网络活动的场景。它使用现代化的 eBPF 技术，提供直观的 Web 界面，支持黑名单和自定义规则。但由于其设计目标是隐私而非安全，不适合作为系统安全加固的工具。
