## 文章核心内容

本文由 Prolog 专家 Markus Triska 撰写，揭示了 Prolog 编程中的常见"恐怖陷阱"，并提供了如何编写高质量 Prolog 代码的指导原则。

### 主要问题类型

1. **失去解决方案（ Losing Solutions ）**
   - 程序可能报告错误答案
   - 程序可能未能报告预期解决方案
   - 主要原因：使用 `!/0`、`(->)/2`、`var/1` 等非单调和不纯的语言构造

2. **全局状态（ Global State ）**
   - 使用 `assertz/1` 和 `retract/1` 修改全局数据库
   - 引入隐式依赖，导致程序在不同调用顺序下产生意外结果

3. **不纯输出（ Impure Output ）**
   - 直接使用 `format/2` 在终端打印答案
   - 阻止程序作为真正的"关系"使用，难以测试

4. **低级语言构造（ Low-level Constructs ）**
   - 使用 `(is)/2`、`(=:=)/2` 等低级算术谓词
   - 增加学习和理解的难度

### 解决方案：纯度（Purity）

- 使用 CLP(FD) 约束替代低级算术
- 使用 `dif/2` 等约束处理非线性约束
- 使用 `if_/3` 元谓词处理条件逻辑
- 通过谓词参数传递状态而非全局状态
- 让 toplevel 负责输出而非手动打印

### 代码示例对比

```prolog
/* 不纯版本 - 会失去解决方案 */
horror_factorial(0, 1) :- !.
horror_factorial(N, F) :-
    N > 0,
    N1 is N - 1,
    horror_factorial(N1, F1),
    F is N*F1.

/!* 纯版本 - 完美工作 *!/
n_factorial(0, 1).
n_factorial(N, F) :-
    N #> 0,
    N1 #= N - 1,
    n_factorial(N1, F1),
    F #= N*F1.
```

## 社区讨论亮点

### Prolog 与 Erlang 的对比

多位开发者讨论了 Prolog 与 Erlang 的关系：
- ** Antibabelic **：Erlang 与 Prolog 属于完全不同的范式。早期 Erlang 在 Prolog 中实现，这解释了语法相似性，但本质上差异很大。
- ** xlii **：Erlang 像"当我输入 X，我可以计算出 Y"，而 Prolog 像"如果我想要 Y，X 是什么"。Prolog 更像数据库绑定和查询，Erlang 是函数执行。
- ** kunley **：Joe Armstrong 等人从 Prolog 中借鉴的主要是语法。

### Prolog 的实际应用场景

| 应用 | 描述 |
|------|------|
| **测试场景生成器** | ~50个动作 + ~30个约束生成>200个测试场景 |
| **Maemo（诺基亚）** | 移动操作系统的配置文件管理器 |
| **业务规则引擎** | 处理日志和监控事件的后端 |

| **TerminusDB** | 使用 Prolog 作为查询语言 |
| **AWS 管理** | 与 Ruby 结合解决云管理问题 |

### Datalog 与 Prolog

讨论澄清了两者的关键区别（ **gobdovan** 提供）：

- **Datalog**：基于 Horn 子句作为数据库约束/推理规则，从已知事实开始反复应用规则直到推导完毕，**保证终止**
- **Prolog**：将 Horn 子句作为搜索程序，从查询向后工作直到找到证明或失败，**可能循环**

Datalog 常用于：
- Rust 编译器的 Polonius 借用检查器
- 静态分析
- RDF 三元组存储
- 授权系统（如 Biscuit）
- 图数据库（如 CozoDB）

### 经典引言

- ** Antibabelic **：抽象的优势在于将优化工作集中在一处。使用高度优化的 Prolog 系统比自己重写更高效。
- ** Richard O'Keefe **（《The Craft of Prolog》作者）："优雅不是可选项。"
- ** segmondy **："不了解 Lisp 的人最终会重新发明它，同样的事情也适用于 Prolog。"

## 总结

这篇 HN 热帖围绕 Markus Triska 的经典文章展开，揭示了 Prolog 编程中常见的反模式和最佳实践。社区讨论显示：

1. **纯度优先**：坚持使用纯 Prolog 子集（约束、单调性）是编写高质量 Prolog 的关键
2. **实用价值**：Prolog 在测试生成、规则引擎、配置管理等场景仍有独特价值
3. **Datalog 崛起**：作为 Prolog 的可终止子集，Datalog 在静态分析和授权系统中越来越受欢迎
4. **历史渊源**：Erlang 等语言虽语法相似，但属于完全不同范式
