## 核心发现

### 1. 重大里程碑

Arch Linux 官方 Docker 镜像现已实现**位对位可复现（bit-for-bit reproducible）**，这是继 WSL 镜像实现可复现之后又一重要突破。这一成就通过镜像摘要一致性（digest equality）和 diffoci 工具进行了验证。

### 2. 镜像分发方式

可复现镜像通过新的 **repro** 标签进行分发（`docker.io/ARCHLINUX/ARCHLINUX:REPRO`）。采用专用标签的原因是：

- 为确保可复现性，镜像中必须移除 pacman 密钥
- pacman 在镜像中无法开箱即用
- 需要用户在容器内手动重新生成密钥环

### 3. 使用方法

用户需要在容器启动后运行以下命令初始化 pacman 密钥环：

```bash
pacman-key --init && pacman-key --populate archlinux
```

对于 Distrobox 用户，可使用 pre-init hooks：

```bash
distrobox create -n arch-repro -i docker.io/archlinux/archlinux:repro --pre-init-hooks "pacman-key --init && pacman-key --populate archlinux"
```

### 4. 实现可复现性的关键技术

#### 4.1 设置 SOURCE_DATE_EPOCH

在 Dockerfile 中设置 `SOURCE_DATE_EPOCH` 环境变量，并将其体现在 `org.opencontainers.image.created` LABEL 中。

#### 4.2 移除 ldconfig 辅助缓存文件

`/var/cache/ldconfig/aux-cache` 文件会引入非确定性，必须在构建过程中移除。

#### 4.3 时间戳规范化

在 docker build / podman build 时使用以下选项重写时间戳：

```bash
--source-date-epoch=$SOURCE_DATE_EPOCH --rewrite-timestamp
```

这确保了以下文件的时间戳在多次构建间保持一致：
- `/etc/` 目录及其中所有文件
- `/etc/ld.so.cache`
- `/etc/os-release`
- `/sys/` 目录
- `/var/cache/` 目录
- `/proc/` 和 `/dev/` 目录

### 5. 验证方法

可通过以下命令验证镜像的可复现性：

```bash
# 检查镜像摘要一致性
podman inspect --format '{{.Digest}}' <image>

# 使用 diffoci 比较构建结果
diffoci compare <image1> <image2>
```

## 结论

Arch Linux Docker 镜像的可复现构建是一个重要的技术成就，它确保了多次构建产生完全相同的镜像比特，这对于安全审计、构建验证和供应链安全具有重要意义。尽管目前需要手动初始化 pacman 密钥环，但这是实现完全可复现性所做的必要权衡。
