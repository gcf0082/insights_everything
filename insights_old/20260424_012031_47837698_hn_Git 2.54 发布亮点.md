## 文章摘要

本文介绍了 Git 2.54 版本的主要新特性，包括新的 `git history` 子命令（reword、split）、基于配置的钩子支持、`git add -p` 的改进，以及 `git log -L` 的增强功能。这些更新受到了 Jujutsu（jj）版本控制系统的启发。

---

## 关键洞察

### 1. Git 借鉴 jj 的特性
- `git history reword` ≈ `jj describe`：用于修改提交信息
- `git history split` ≈ `jj split`：用于拆分提交
- 评论普遍对这些"借鉴"表示欢迎，认为这是双赢

### 2. Git 钩子配置的改进
- 新增基于配置的钩子支持（config-based hooks）
- 用户可以直接在 global config 中定义钩子
- 引发了关于本地钩子安全的讨论：有人呼吁更安全的自动安装机制

### 3. 关于本地钩子的讨论
- **反对自动安装**：安全风险，git clone 不应自动运行代码
- **支持 CI 检查**：建议将检查放在 CI 中而非本地钩子
- **替代方案**：pre-commit 工具、Husky、GrumPHP 等
- **建议**：可考虑 `.githooks` 目录 + 用户确认的方案

### 4. 其他新功能
- `git history reword`：轻松修改提交信息，无需 rebase
- `git add -p` 改进：暂存补丁功能增强
- `git log -L` 改进：日志过滤更方便
