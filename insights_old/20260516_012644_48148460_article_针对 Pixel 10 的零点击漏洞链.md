# Project Zero 洞察报告：Pixel 10 零点击漏洞链

**洞察链接**: https://projectzero.google/2026/05/pixel-10-exploit.html

**原始标题**: A 0-click exploit chain for the Pixel 10: When a Door Closes, a Window Opens

**发布来源**: Project Zero

**发布日期**: 2026年5月13日

**作者**: Seth Jenkins

---

## 摘要

本文介绍了 Project Zero 针对 Google Pixel 10 设备发现的零点击（0-click）漏洞链。该漏洞链演示了如何从零点击上下文在仅使用两个漏洞的情况下获得 Android 设备的 root 权限。研究发现，虽然针对 Pixel 9 的 Dolby 漏洞已被修复，但在 Pixel 10 上发现了一个新的高危 VPU 驱动漏洞，允许用户空间直接映射并修改内核内存。

---

## 1. 研究背景

Project Zero 近期发布了一个针对 Google Pixel 9 的漏洞链，演示了如何从零点击上下文仅通过两个漏洞获得 Android 系统的 root 权限。该漏洞利用了影响整个 Android 平台的 Dolby 零点击漏洞（CVE-2025-54957），该漏洞于 2026 年 1 月被修复。

在成功实现 Pixel 9 漏洞链后，研究团队开始探索是否能为 Pixel 10 构建类似的漏洞链。

---

## 2. Dolby 漏洞利用更新

针对 CVE-2025-54957 的漏洞利用代码修改相对直接。主要工作包括：

- **偏移量更新**: 将针对 Pixel 9 库计算出的偏移量更新为 Pixel 10 库中的类似偏移量
- **RET PAC 挑战**: Pixel 10 使用 RET PAC（Pointer Authentication）替代 `-fstack-protector`，导致无法覆写 `__stack_chk_fail` 函数
- **解决方案**: 使用 `dap_cpdp_init` 初始化代码进行覆写，该代码在解码器初始化时调用一次后不再使用

该漏洞利用仅适用于未打补丁的设备（SPL 2025年12月或更早版本）。

---

## 3. 本地权限提升：VPU 驱动漏洞

### 3.1 背景

Pixel 9 上的 BigWave 驱动本地权限提升漏洞无法移植到 Pixel 10，因为该驱动不在 Pixel 10 上部署。然而，在 mediacodec SELinux 上下文中发现了一个新驱动：`/dev/vpu`。

该驱动用于与 Tensor G5 芯片上的 Chips&Media Wave677DV 硅芯片交互，主要用于加速视频解码。根据开源代码中的注释，该驱动与 BigWave 驱动由同一组开发者开发和维护。

### 3.2 发现的高危漏洞

与 Jann Horn 合作，研究团队在 2 小时内审计发现了该 VPU 驱动中的一个极其简单的漏洞。

**漏洞详情**（问题编号：463438263）:

```c
static int vpu_mmap(struct file *fp, struct vm_area_struct *vm)
{
    unsigned long pfn;
    struct vpu_core *core =
        container_of(fp->f_inode->i_cdev, struct vpu_core, cdev);

    vm_flags_set(vm, VM_IO | VM_DONTEXPAND | VM_DONTDUMP);
    vm->vm_page_prot = pgprot_device(vm->vm_page_prot);
    pfn = core->paddr >> PAGE_SHIFT;

    return remap_pfn_range(vm, vm->vm_start, pfn, vm->vm_end-vm->vm_start, vm->vm_page_prot) ? -EAGAIN : 0;
}
```

### 3.3 漏洞原理

这个 mmap 处理器本应用于将 VPU 硬件的 MMIO 寄存器区域映射到用户空间虚拟地址空间。**关键问题**在于：

- `remap_pfn_range` 调用仅基于 VMA 大小，完全不限制于实际寄存器区域的大小
- 通过在 mmap 系统调用中指定大于寄存器区域的大小，调用者可以从 VPU 寄存器区域的物理地址开始，映射任意大小的物理内存
- 由于 VPU 寄存器区域位于物理内存的低地址区域，而整个内核镜像（包括 .text 和 .data 区域）位于更高的物理地址，因此可以被用户空间访问和修改

### 3.4 漏洞利用难度

- **利用简单**: 利用此漏洞实现内核任意读写仅需 5 行代码
- **无需 KASLR 绕过**: Pixel 上的内核始终位于相同的物理地址，VPU 内存区域与内核之间的偏移量始终为已知值，无需在映射的物理内存中扫描内核位置
- **开发时间**: 编写完整漏洞利用不到一天

## 4. 补丁流程

- **漏洞报告时间**: 2025年11月24日
- **严重性评级**: 高危（High）
- **补丁时间**: 漏洞报告后 71 天，在 2026 年 2 月 Pixel 安全公告中修复
- **积极变化**: 相比之前用于 Pixel 9 权限提升的 BigWave 漏洞（相同安全影响但最初被评级为"中等"），这是一个有意义的进步

---

## 5. 结论与启示

### 5.1 积极方面

- Android 漏洞 triage 流程取得明显进展，VPU 漏洞在比以往 BigWave 问题短得多的时间内获得初步修复
- Google 努力确保严重漏洞得到高效修复，有助于保护大量 Android 设备

### 5.2 警示方面

- 审计 BigWave 驱动后 5 个月，研究团队仍在 VPU 驱动中发现了严重且极其简单的漏洞
- 驱动程序安全仍然是确保 Android 生态系统安全的关键优先事项
- 软件供应商需要采取积极主动的软件安全方法、代码审计和漏洞修复措施
