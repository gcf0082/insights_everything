## 项目概述

Pretext 是一个用于多行文本测量和布局的 TypeScript 库。该项目旨在高效计算网页上换行文本的高度，而无需实际先将文本渲染到页面上（这非常昂贵）。它通过预计算单个文本片段（类似于单词）的宽度和高度，并将其缓存起来实现这一功能。然后使用自定义代码实现浏览器构造文本字符串的完整算法，包括换行处理。

## 核心技术实现

该库的工作原理分为两个阶段。prepare 阶段使用 Intl.Segmenter 对文本进行分词，通过 canvas.measureText 测量每个单词的宽度并进行缓存，在需要 emoji 校正时执行一次缓存的 DOM 校准读取。layout 阶段则使用纯算术方法遍历缓存的单词宽度来计算行数和高度，每次调整大小时都可以调用此方法，单次布局耗时约 0.0002 毫秒。

这个问题的难点在于需要处理许多不同类型的换行和字符，包括连字符化、emoji、中文等，还需要考虑不同浏览器（尤其是 Safari）在渲染算法上的细微差异。

## 社区评价

社区对 Pretext 的评价普遍非常积极。许多开发者认为这是一个非常出色的项目，解决了长期困扰 Web 开发者的难题。有评论指出，文本布局引擎开发难度极大，原本以为可以轻松完成的任务，实际上会面临诸如中文标点在列渲染时需要不同旋转等复杂问题。

该项目的 demo 展示了多种应用场景，包括文本气泡换行、手风琴组件高度自适应、形状文本排版等实用案例。

## 性能对比

有开发者将 Pretext 与自己编写的类似库 uWrap.js 进行了对比。对于 ASCII 文本，uWrap.js 在 80ms 内完成，而 Pretext 需要 2200ms。不过 uWrap.js 仅处理拉丁字符，不处理软连字符或 emoji 修正，适用范围较窄。Pretext 的优势在于其广泛的字符类型支持和更高的准确性。

开发者表示，Pretext 适合准备一次然后多次布局的场景，单次布局性能可达到亚毫秒级。目前已有多个性能优化 PR 正在开发中。

## 与现有方案的比较

有开发者询问 Pretext 与 Skia-wasm/Canvaskit 的区别。Pretext 是纯 TypeScript 实现，不需要 WebAssembly，对项目体积要求更高的场景更为适合。

也有开发者指出，该功能本应是浏览器的标准功能。CSS FontMetrics API 提案已经存在，但需要浏览器厂商的支持和实现。

## 局限性与问题

部分用户报告了在特定系统上 demos 显示异常的情况，例如 Fedora + Firefox 上 torus 演示完全变形。还有评论指出，该库实际上是通过在 canvas 上渲染文本来进行测量的，这并不一定比直接使用 DOM 元素更快，唯一的潜在优势是直接渲染到 canvas 而无需构建 DOM 节点。

## 总结

Pretext 是一个雄心勃勃的项目，试图填补 Web 平台长期存在的文本测量功能缺失。虽然有人认为这是 AI 辅助开发的典型案例，但它确实解决了实际痛点，对于 GUI Web 框架和富文本编辑器的未来发展具有重要意义。
