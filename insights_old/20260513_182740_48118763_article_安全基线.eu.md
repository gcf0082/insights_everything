## 核心发现摘要

2026年5月13日，SecurityBaseline.eu 网站正式上线，这是荷兰"基础安全"（Basisbeveiliging）项目的扩展项目。该项目已对基础安全进行超过十年的监控，并已成为政府政策的一部分。本次调查覆盖32个欧洲国家（包括欧盟27国及挪威、瑞士、冰岛、列支敦士登等），对近20万个政府域名和6.7万个地方政府进行安全监测。

### 三大令人担忧的指标

1. **3,081个政府网站非法使用追踪Cookie**：违反GDPR规定，未经用户同意放置追踪Cookie
2. **1,070个phpMyAdmin管理面板公开暴露**：数据库管理界面不应在公共互联网上访问
3. **99%的政府邮件加密不足**：仅1%的邮件服务器符合最新的安全加密标准

---

## 详细发现

### 1. 追踪Cookie问题（3,081个网站）

**现状**：
- 欧洲各国政府网站共放置了357,000个Cookie
- 其中3,081个网站在未经用户同意的情况下放置追踪Cookie
- 这违反了GDPR关于"明确、知情、无条件"同意的要求

**各国差异**：
| 国家 | 占比 |
|------|------|
| 斯洛伐克 | 9.88% |
| 希腊 | 8.16% |
| 葡萄牙 | 7.63% |
| 法国 | 3.88% |
| 德国 | 0.59% |
| 塞浦路斯 | 0% |
| 列支敦士登 | 0% |

**追踪Cookie来源分布**：
- YouTube：2,077个
- Google Ads：842个
- Facebook：293个
- TikTok：20个

**分析**：政府使用追踪技术的原因通常是现代网站技术的副作用，这些技术简单易用但隐藏着广告成本。值得注意的是，追踪Cookie数量未包含"项目型"网站（旅游、建筑、活动等），这类网站更容易使用追踪技术。

---

### 2. phpMyAdmin公开暴露（1,070个）

**现状**：
- 共发现1,070个phpMyAdmin门户分布在3,529个不同域名上
- 许多域名共享同一个面板，通常是同一个服务提供商
- phpMyAdmin是开源工具，但欧洲政府未向该项目提供任何资金支持

**各国暴露数量**：
| 国家 | 数量 |
|------|------|
| 法国 | 513 |
| 波兰 | 499 |
| 匈牙利 | 368 |
| 德国 | 300 |
| 捷克 | 258 |
| 意大利 | 232 |

**安全风险**：2026年4月30日，类似的管理面板cPanel被发现存在严重漏洞。如果这些管理面板不在公共互联网上可达，当发现此类漏洞时造成的危害会小得多。值得注意的是，其中两个面板出现在计算机安全事件响应团队（CSIRT）的地址上，这更是双重违规。

---

### 3. 邮件加密不足（99%）

**现状**：
- 仅1%的政府邮件服务器通过最新加密测试
- 荷兰（58%）和丹麦（44%）是唯一表现相对较好的国家
- 其他国家普遍低于10%

**各国邮件加密通过率**：
| 国家 | 通过率 |
|------|--------|
| 荷兰 | 58% |
| 丹麦 | 44% |
| 葡萄牙 | 8% |
| 欧盟CSIRT | 5% |
| 瑞典 | 4% |
| 德国 | 1% |
| 法国 | 1% |
| 意大利 | 0% |

**测试标准**：本次测量使用internet.nl 1.11版本（2026年4月发布），该版本实施了荷兰政府的传输层安全（TLS）指南。这些指南于2025年5月发布，规定了如何加密以防止邮件被窃听或篡改。

**问题**：目前欧洲层面没有TLS标准。德国和法国各自编写了其他指南，这些指南显然互不兼容，也没有提供简单的测试工具。

## 监测方法与地图可视化

**技术背景**：
- Web Security Map 软件开发历时十余年
- 每晚重新构建全部1,827个地图（32个国家 × 21个指标 × 多种区域类型）
- 监测覆盖近20万个政府域名和6.7万个地方政府

**地图系统**：
- 使用红绿灯颜色系统：红色表示安全问题，橙色表示警告，绿色表示无问题
- 灰色表示未找到该区域的在线地址
- 使用OpenStreetMap作为地图数据

**各国特点**：
- 丹麦： municipalities多为橙色，表示政策正在生效
- 意大利：许多绿色municipalities，其技巧是将市政网站设为子域名，将安全问题转移到上级
- 荷兰：绿色、橙色和红色混杂，归因于政府政策和Basisbeveiliging与Internet.nl的监测

## 結論

本次調査揭示了歐洲政府在網絡安全方面的嚴重問題：

1. **追蹤Cookie違規**：3,081個政府網站未經用戶同意使用追蹤技術，違反GDPR
2. **管理界面風險**：1,070個數據庫管理面板公開暴露，存在被攻擊風險
3. **郵件加密不足**：99%的政府郵件服務器未達到基本加密標準

這些問題需要通過建立持續改進的流程來解決，而非一次性修復。SecurityBaseline.eu的目標是讓政府能夠保護其公民，並可以對本國其他機構提出安全要求。
