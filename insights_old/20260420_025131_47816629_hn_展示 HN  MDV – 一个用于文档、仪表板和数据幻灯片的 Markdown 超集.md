## 项目概述

MDV 是一个 Markdown 超集，声称是严格的 CommonMark 加上四个增项：

1. **YAML front-matter** - 用于标题、主题、命名样式和数据集引用
2. **Fenced blocks** - 用于数据和可视化：```chart type=bar x=region y=sales
3. **::: containers** - 用于样式区域和布局：::: callout / ::: columns
4. **::: toc** - 自动生成目录

## 核心讨论观点

### 1. 表格语法扩展（热门评论）

用户 kevinkoning 提出了一种创新的表格语法扩展：`::` 分隔符表示数据类型

```
| Month::x | Revenue::y1 | Cost::y2 |
| -------- | ----------: | -------: |
| Jan      | $82,000     | $51,000 |
```

这种语法的优点：
- 渲染器理解语法时可生成图表
- 不理解时退化为带点奇怪标题的表格
- 保持良好的可读性

## 总结

这个 HN 帖子反映了开发者社区对 Markdown 扩展的持续兴趣。MDV 作为又一个 Markdown 超集，面临着与众多竞品相似的挑战：

1. **渲染器可移植性**���规范是否足够开放
2. **编辑器支持**：IDE/编辑器能否原生支持
3. **与现有工具竞争**：djot、Markdoc、Marp、Quarto 等
4. **实用性**：是否解决真实问题而非过度工程化

核心张力在于：在保持 Markdown 可读性的同时添加富功能，最终可能只是重新发明 HTML。这种批评在讨论中多次出现。
