## 内容摘要

这是一篇关于 Asahi Linux 7.0 进展报告的 Hacker News 讨论帖，原始文章来自 Asahi Linux 官方博客。

### 主要技术亮点

**音频驱动突破**：Asahi 团队通过逆向工程发现 CS42L84 和 CS42L42 音频芯片虽然针对 macOS 只支持 48/96 kHz 采样率，但 CS42L42 实际上支持所有常见采样率。经过研究，将 CS42L42 的寄存器值应用到 CS42L84 驱动中，成功实现了 44.1、88.2、176.4 和 192 kHz 采样率的支持。这项补丁已提交至上游内核并被合并。

### 讨论热点

1. **采样率与电池续航**：用户讨论苹果为何只支持两种采样率，这会导致 PipeWire 需要进行软件重采样，消耗 CPU 资源进而影响电池寿命。

2. **项目可持续性担忧**：有用户质疑 Asahi 仍是独立项目，未被主流内核和发行版直接支持。团队回应称他们正在积极向上游提交代码，努力减少与主线的差异。

3. **Linux 与 macOS 对比**：讨论中爆发了关于 Linux 与 macOS 体验的争论，涉及兼容性、稳定性、定制化等方面。

4. **右维修讨论**：用户探讨苹果不提供文档是否属于"右维修"问题。
