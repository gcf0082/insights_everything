## 内容摘要

这是一个 Hacker News 上的 Ask HN 帖子，发帖人表示自己不使用 OpenClaw，身边的人也不使用，尽管他觉得自己“非常关注 AI 领域”。他想知道是否有其他人真的在使用 OpenClaw。

## 主要讨论观点

### 正面观点

1. **个人助理用途**
   - 与 Obsidian 结合作为记忆存储，通过 WhatsApp 访问
   - 用于卡路里/体重/锻炼追踪、待办事项、生日提醒等生活管理任务
   - 优势在于记忆和系统提示词可以版本控制，便于切换 LLM

2. **商业应用案例**
   - 一位园丁使用 NanoClaw 管理日程：从 Gmail 获取工作订单、安排报价访问、分析照片、生成 PDF 提案、创建发票
   - 每周节省约20分钟，提案生成只需20分钟

3. **自动化工作流**
   - 自动将待办事项滚动到第二天
   - 检查天气并提醒收取草坪家具
   - 从邮件中提取发票数据并录入成本追踪表
   - 检查 Outlook 日历中已被拒绝的会议

4. **学习助手**
   - 从 Obsidian 笔记自动生成 Anki 闪卡
   - 夜间同步笔记并创建复习卡片

### 负面观点

1. **价格问题**
   - 约 $3.50/天 或 $100/月，成本过高
   - 许多人转向使用更便宜的替代方案如 MiniMax

2. **稳定性问题**
   - 经常出现故障，需要人工干预修复
   - WhatsApp 集成尤其不稳定
   - 设置复杂，需要大量调试

3. **过度炒作**
   - 很多人只是在社交媒体上宣传，但没有实际的使用案例
   - 被认为是“制造出来的 bot 热度”
   - 主要受益者是托管提供商和课程卖家，而非实际用户

4. **安全隐患**
   - 需要授予全面的 API 访问权限
   - 存在删除仓库、系统文件的风险
   - 有人将其比作“有史以来最大的安全退步”

### 中立观点

1. **技术门槛**
   - 对于非技术人员有价值，但工程师可以自己用 cron + Playwright 实现
   - 类似于 Dropbox 时刻——为普通用户简化了复杂操作

2. **替代方案**
   - 有人转向 Hermes Agent、NanoClaw 或自建 Claude Code 方案
   - 更倾向使用确定性的自动化脚本

## 结论

OpenClaw 引发了激烈讨论。支持者认为它简化了AI助手的使用门槛，适合非技术用户；批评者则认为它过度炒作、价格昂贵且不稳定。多数人承认其有一定用途，但认为对于技术人员来说，用 cron job 和脚本能以更低成本实现类似功能。
