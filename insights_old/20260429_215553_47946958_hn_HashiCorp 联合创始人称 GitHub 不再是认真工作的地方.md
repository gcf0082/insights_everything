## 摘要

HashiCorp 联合创始人 Mitchell Hashimoto 在其博客中宣布，其终端模拟器项目 Ghostty 将离开 GitHub，并批评 GitHub "不再是一个适合认真工作的地方"。这一表态在 Hacker News 上引发了热烈讨论，近 120 名开发者参与评论，普遍对 GitHub 近年来的稳定性问题表示不满，并讨论了潜在的替代方案。

## 核心论点

### 1. GitHub 稳定性问题加剧

- GitHub 可用性已下降至 86%
- 2025年10月计划扩容10倍，2026年2月已需要30倍容量
- 大规模流量增长主要来自 AI 生成代码（"vibe coding"）项目
- 多起服务中断影响 CI/CD 工作流

### 2. 迁移至 Azure 基础设施

- 正在将 GitHub 迁移至 Azure
- 迁移期间出现了稳定性问题
- 用户反映 Azure Repos 实际上更稳定

### 3. 微软收购后的影响

- 2018年微软收购 GitHub 后，问题逐渐加剧
- 裁员导致核心工程师流失
- 产品决策更注重新功能而非稳定性

## 社区反应与替代方案

### 选择离开的项目

- **Ghostty**: 正在离开 GitHub
- **Zig**: 已迁移至 Codeberg
- **Bookstack-app**: 已离开
- **HardenedBSD**: 已离开

### 推荐的替代平台

| 平台 | 类型 | 特点 |
|------|------|------|
| Forgejo | 自托管 | 开源，Codeberg 使用 |
| SourceHut | 托管服务 | 极简设计 |
| GitLab | 自托管/云服务 | 功能完整 |
| Azure Repos | 云服务 | 基础但稳定 |

### 自托管案例

- 用户从 GitHub 迁移至自托管 Forgejo，CI 构建时间减少 80%
- 自托管 GitLab 6 年无计划外停机

## 分析

### 根本原因

1. **AI 代码潮**: "vibe coding" 创造了大量低质量仓库，消耗基础设施资源
2. **规模挑战**: GitHub 原本设计无法应对 30 倍流量增长
3. **运营能力下降**: 微软收购后运维关注度下降

### 行业影响

- 可能引发更多项目迁移浪潮
- 对 GitHub 市场主导地位构成挑战
- 开源社区对中心化平台的信任正在动摇
