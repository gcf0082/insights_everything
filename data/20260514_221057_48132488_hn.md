## 摘要

Bun 项目已将其核心代码从 Zig 重写为 Rust 并合并到主仓库。此次重写涉及超过 **100万行代码**，在短短9天内完成。Rust 版本修复了内存泄漏问题，并将二进制大小减少了 3-8MB。

---

## 详细分析

### 事件背景

- **收购关系**: Anthropic 于 2025年12月收购了 Bun，并将其作为 Claude Code 的运行时基础设施
- **技术栈变更**: Bun 最初使用 Zig 开发，现在正在向 Rust 迁移
- **实验性质**: Jarred Sumner 此前曾表示"代码很可能被完全丢弃"，这仍是一个实验性项目

### 技术细节

1. **代码规模**: 超过100万行 Rust 代码
2. **内存安全性**: Rust 编译器能够自动捕获以下问题：
   - use-after-free（使用后释放）
   - double-free（双重释放）
   - forgot-to-free-on-error-path（错误路径忘记释放）
3. **二进制优化**: 体积减少 3-8MB
4. **测试兼容性**: 达到 99.8%（Linux x64 glibc）

### 有趣细节

- 后续删除 60万行 Zig 代码的 PR 被 Bun 自己的 CI 系统自动标记为 **"AI slop"**
- Claude Code 本身也是"vibe-coded"（ vibe 编程，指完全依赖 AI 生成代码）

---

## 社区反应

### 主要观点

1. **担忧**: 9天内完成如此大规模的重写太过仓促，缺乏充分测试
2. **质疑**: 这是营销手段还是真正的技术实验？
3. **讽刺**: Bun 现在 literally 成为"vibe-coded"的产物
4. **关注点**: Claude Code 依赖的这个运行时现在没有一个人有完整的心智模型

### 技术评论

- Rust 对 Zig 的优势在于生态成熟度更高
- Zig 的"无 AI 代码"政策与 Anthropic 的 AI 驱动开发文化存在冲突
- 这可能是 Anthropic 展示 AI 代码生成能力的一种方式

---

## 影响与意义

1. **行业影响**: 这是迄今为止最大规模的 AI 代码重写实验之一
2. **工程实践**: 展示了 AI 可以在极短时间内完成大规模代码迁移
3. **技术趋势**: 验证了 AI 辅助编程在大规模系统级项目中的可行性
4. **风险警示**: 90万行代码的质量和长期可维护性仍需验证

---

## 结论

Bun 从 Zig 到 Rust 的重写是 Anthropic 收购后的重大技术转型。这一实验性的重写已在短时间内取得显著进展（99.8% 测试兼容性），但最终是否会成为正式版本仍有待观察。Jarred Sumner 本人也表示"这很可能被完全丢弃"。
