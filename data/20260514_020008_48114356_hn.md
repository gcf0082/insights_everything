## 项目概述

Scrcpy 是一个开源的 Android 设备屏幕镜像和控制工具，由 Genymobile 开发维护（注意：不是 Google）。该工具通过 USB 或 TCP/IP 无线连接 Android 设备，将手机屏幕投射到电脑上，并允许使用键盘和鼠标控制设备。无需 root 权限，也无需在设备上安装任何应用。支持 Linux、Windows 和 macOS 三大平台。

截至目前，GitHub 仓库已获得约 14 万颗星标，是 Android 开发者和测试人员常用的工具。

## v4.0 主要新功能

### 1. SDL3 迁移
从 SDL2 迁移到 SDL3，以获得持续维护、错误修复和新功能支持。SDL3 的新 API 实现了调整窗口大小时自动保持画面宽高比，避免出现黑边。

### 2. 弹性显示（Flex Display）
新增 `--flex-display`（或 `-x`）选项，允许虚拟显示与客户端窗口同步动态调整大小。

### 3. 相机手电筒和缩放控制
新增相机控制功能：
- `MOD + t`：开启相机手电筒
- `MOD + Shift + t`：关闭相机手电筒
- `MOD + ↑`（上）：放大
- `MOD + ↓`（下）：缩小

也可通过命令行参数 `--camera-torch` 和 `--camera-zoom` 设置。

### 4. 窗口宽高比锁定
现在调整窗口大小时会自动保持内容宽高比，不会出现黑边。可通过 `--no-window-aspect-ratio-lock` 恢复旧行为。

### 5. 保持设备活跃
新增 `--keep-active` 选项，定期向系统发送用户活动信号，防止设备因 inactivity 而自动熄屏。

### 6. 自定义背景色
默认背景色从纯黑改为深灰色。可通过 `--background-color` 自定义（支持 3 位或 6 位十六进制颜色码）。

### 7. 断开连接提示
设备断开连接时，窗口关闭前会显示断开图标 2 秒，避免用户误以为是应用崩溃。

### 8. 快捷键新增
- `F11`：切换全屏模式
- `MOD + q`：退出 scrcpy

### 9. Meta Quest 支持修复
解决了 Meta Quest 固件升级后的屏幕闪烁问题。

### 10. 其他改进
- 修复 OPUS 音频解码时的高 CPU 占用问题
- 支持检测 mDNS 提供的 TCP 设备
- 修复设备序列号包含空格的兼容性问题
- Windows 控制台代码页设置为 UTF-8
- 升级 ADB 平台工具至 37.0.0
- 升级 FFmpeg 至 8.1.1
- 升级 SDL 至 3.4.8
