## 项目概述

claude-token-efficient 是一个通用的 CLAUDE.md 配置文件，通过优化 Claude Code 的输出行为，可以将输出 token 减少约 63%。该工具采用"即插即用"方式，无需修改任何代码，只需将文件放入项目根目录即可生效。

> **注意**：大部分 Claude 费用来自输入 token，而非输出 token。该文件主要针对输出行为进行优化——包括谄媚倾向、冗长回复和格式噪音。它不会解决最大的费用问题，但会改善最恼人的响应问题。

## 使用场景

### 适用场景

- 自动化管道（高输出量：恢复机器人、代理循环、代码生成）
- 重复的结构化任务（跨数百次调用的冗长累积）
- 需要跨会话一致、可解析输出的团队

### 不适用场景

- 单次短查询——文件每次消息都会加载，低输出交换时净 Token 增加
- 随意的一次性使用——低量时开销不划算
- 深度失败模式——需要钩子、门控和机械强制
- 每个任务使用多个新鲜会话——新鲜会话不会以相同方式继承 CLAUDE.md 开销
- 大规模解析可靠性——使用 API 内置的结构化输出更稳健

## 使用方法

**方法一：通用（任意项目）**
```bash
curl -o CLAUDE.md https://raw.githubusercontent.com/drona23/claude-token-efficient/main/CLAUDE.md
```

**方法二：克隆并选择配置**
```bash
git clone https://github.com/drona23/claude-token-efficient
cp claude-token-efficient/profiles/CLAUDE.coding.md your-project/CLAUDE.md
```

**方法三：手动复制**
将 CLAUDE.md 的内容复制到项目根目录。

## 结论

claude-token-efficient 是一个简单高效的 Claude Code 优化工具，通过优化输出行为可以显著减少 token 使用量。对于高频使用 Claude 的开发者和团队来说，这是一个值得尝试的工具。但需要注意，该文件本身会在每次消息时消耗输入 token，因此只有当输出量足够高时才能实现净节省。
