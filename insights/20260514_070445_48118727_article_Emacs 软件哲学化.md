# 洞察报告：软件的 Emacs 化

**来源链接**：https://sockpuppet.org/blog/2026/05/12/emacsification/

**发布日期**：2026年5月12日

**作者**：Thomas Ptacek（Sockpuppet 博客）

---

## 核心洞察

### 1. Markdown 阅读体验的困境

- Markdown 已成为软件开发的事实标准
- AI 代理催生了 TUI（终端用户界面）工具的复兴，但阅读体验变得令人难以忍受
- 现有的 TUI Markdown 查看器（如 glow、Markless）受限于终端的单调字体，阅读疲劳
- 图形界面编辑器（如 Obsidian、Typora、Bear）体验良好，但会干扰已有的编辑器窗口布局
- App Store 的 Markdown 查看器功能残缺：缺少文本搜索、不支持复制到粘贴板等

### 2. AI 构建个性化工具的突破

- 作者用 Claude 在约 30 分钟内构建了一个优于 App Store 所有产品的 Markdown 查看器 MDV.app
- MDV 功能包括：文本选择与复制、SQLite FTS 全文搜索、书签、目录导航、记住阅读位置、精美配色主题
- 核心体会：寻找好工具不如让 AI 生成一个

### 3. Electron 的问题与原生 UI 的复兴

- Signal 等 Electron 应用导致屏幕闪烁（每个应用都携带独立的 Chromium 副本）
- 过去原生 UI 开发人才稀缺，现在 AI 代理可以胜任
- AI 代理可以构建与专业打包程序质量相当的原生用户界面

### 4. 什么是"Emacs 化"？

**定义**：软件变成了像 Emacs 配置一样个人化、可定制的事物

关键特征：
- **个人软件**：大多数工具只对创建者有用，之后被遗忘（如 Emacs 中的小 elisp 程序）
- **偶尔逃逸**：极少数工具会超越个人需求，被更多人安装使用
- **理念比代码重要**：源代码的价值下降，提示词和想法变得更重要
- **配置而非构建**：用 AI 构建软件更像是"配置"而非"构建"

### 5. Emacs 文化的扩展

- 历史上 Emacs 包的体验通常很糟糕：丑陋、慢、难发现
- AI 代理"破解"了 Emacs 文化，使其泄漏到更广阔的世界
- 现在任何有屏幕和输入的人都能可靠地构建原生用户界面

### 6. 未来展望

- **nerd 软件将变得更有趣**：大量陈旧的终端应用可以被大幅改进
- **工具将变得可编程**：不仅是技术上的可编程，更是实践上的可编程
- **Magit 仍然是最佳选择**：至少目前仍是

---

## 结论

作者认为，尽管 AI 编码在漏洞研究领域带来突破令研究者兴奋，但对大多数人可能伴随着恐惧。然而，"Emacs 化"代表了一种真正积极的变化：**构建原生 UI 变得有趣，比构建 web 界面有趣得多**。建议读者尝试构建一些针对自己问题的极Specific 工具，享受它，然后分享截图和使用的提示词。
