## 项目概述

lib0xc是微软开发的一套C标准库邻接API，旨在提高系统编程的安全性。尽管C语言无法在语言层面完全做到类型安全和边界安全，但lib0xc通过提供精心设计的API来使C编程变得更加安全。

## 核心目标

### 1. 启用严格编译警告
lib0xc的设计目标是使项目能够开启尽可能多的编译警告（如`-Wall -Wextra -Werror`），并在代码引入新警告时构建失败。许多高价值的警告通常因可移植性问题被禁用，lib0xc在API设计中优先考虑了这些问题。

### 2. 熟悉易用
lib0xc的API命名和设计刻意模仿标准库函数，在适当的地方可以作为直接替代品，实现平滑过渡。

### 3. 拥抱静态边界
lib0xc的大多数API不假设存在内存分配器（除非API专门提供分配功能）。许多API设计用于固定大小的数据结构（如struct或数组类型），在编译时即可确定大小信息。库大量使用C预处理器，许多API是宏的形式。限制代码使用固定大小对象（避免动态分配）使C编程更加安全。

### 4. 支持clangbounds-safety
lib0xc的API完全支持clang的`-fbounds-safety`扩展，这些扩展利用宏安全地展开以表示指针引用的内存边界，与现有C代码源兼容。

### 5. 标准化、文档化、测试
lib0xc提供的许多API在行业中以各种形式存在了数十年。库不声称原创这些想法，而是寻求提供标准化、良好文档化和彻底测试的版本。

### 6. 扩大C的成功陷阱
lib0xc封装的模式应该具有较大的"成功陷阱"，即正确使用比滥用更容易。许多C语言的缺陷源于设计不佳的API契约，lib0xc在这些API存在的地方提供更好的替代方案。

## 使用示例

### 边界追踪格式化（CURSOR）
```c
#include <0xc/std/cursor.h>

char buf[256];
CURSOR cur;
cbuffopen(&cur, buf, "w");
cprintf(&cur, "hello %s", "world");  // 剩余空间自动追踪
```

### 边界检查的上下文指针
```c
#include <0xc/std/context.h>

struct my_state state;
context_t ctx = __context_export(struct my_state *, &state);

// 导入时验证大小 - 类型大小不匹配将陷阱:
struct my_state *s = __context_import(struct my_state *, ctx);
```

### 安全整数转换
```c
#include <0xc/std/int.h>

// 溢出时在运行时陷阱，而不是静默截断:
size_t n = __cast_signed_unsigned(size_t, file_stat.st_size);
```

### 可移植printf格式说明符
```c
#include <0xc/std/io.h>

uint32_t v32 = 42;
uint64_t v64 = 100;
printf("%u %lu\n", oxou(v32), oxolu(v64));  // 无需PRIu32/PRIu64
```

## 构建要求

- **C标准**: C11 with GNU extensions (`-std=gnu11`)
- **编译器**: clang或gcc（推荐clang以支持`-fbounds-safety`）
- **构建工具**: GNUMake >= 3.81
- **支持平台**: macOS (arm64, x86_64), Linux (arm64, x86_64)

## 构建和测试

构建POSIX静态库:
```bash
make lib
```
生成`build/public/lib0xc.a`。

运行完整测试套件:
```bash
make test
```
测试按模块组织，使用`sys/unit.h`API进行注册，使用`sys/check.h`API进行测试用例断言。

## 总结

lib0xc是微软为提高C语言安全性而创建的重要项目。通过提供边界安全、类型安全的API和工具，它帮助开发者编写更安全的系统级代码。项目的设计理念强调编译时检查、静态边界和清晰的错误处理，是C语言安全编程的重要参考。
