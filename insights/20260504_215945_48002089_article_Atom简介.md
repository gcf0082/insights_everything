# Atom 订阅格式洞察报告

## 基本信息

- **洞察链接**: https://validator.w3.org/feed/docs/atom.html
- **文档来源**: W3C Feed Validation Service
- **格式类型**: Atom Syndication Format (RFC 4287)
- **媒体类型**: application/atom+xml
- **报告日期**: 2026-05-04

## 什么是 Atom？

Atom 是一种基于 XML 的网页内容和元数据同步格式，也是一种用于发布和编辑定期更新网站的应用程序级协议。所有 Atom feed 必须是格式良好的 XML 文档。

### 一般要求

- 所有元素必须位于 http://www.w3.org/2005/Atom 命名空间
- 所有时间戳必须符合 RFC 3339 格式
- 除特别说明外，所有值必须是纯文本（不能包含实体编码的 HTML）
- 可使用 xml:lang 标识人类可读文本的语言
- 可使用 xml:base 控制相对 URI 的解析方式

## Feed 必需元素

| 元素 | 说明 |
|------|------|
| id | 使用永久且全局唯一的 URI 标识 feed |
| title | 人类可读的 feed 标题，不能为空 |
| updated | feed 最后一次重大修改的时间 |

## Feed 推荐元素

| 元素 | 说明 |
|------|------|
| author | feed 的作者。feed 必须至少包含一个 author 元素，除非所有 entry 元素都包含 author 元素 |
| link | 关联的网页，通过 rel 属性定义关系类型。feed 应包含指向自身的链接 |

## Feed 可选元素

| 元素 | 说明 |
|------|------|
| category | feed 所属的分类 |
| contributor | feed 的贡献者 |
| generator | 生成 feed 的软件 |
| icon | feed 的小图标（方形） |
| logo | feed 的大图标（宽度是高度的两倍） |
| rights | 版权信息 |
| subtitle | feed 的人类可读描述或副标题 |

## Entry 必需元素

| 元素 | 说明 |
|------|------|
| id | 使用永久且全局唯一的 URI 标识 entry |
| title | 人类可读的 entry 标题，不能为空 |
| updated | entry 最后一次重大修改的时间 |

## Entry 推荐元素

| 元素 | 说明 |
|------|------|
| author | entry 的作者 |
| content | entry 的完整内容。如果不存在 alternate 链接，则必须提供 |
| link | 关联的网页。如果不存在 content 元素，则必须提供 alternate 链接 |
| summary | entry 的简短摘要。如果 content 不在内联或使用 base64 编码，则应提供 |

## Entry 可选元素

| 元素 | 说明 |
|------|------|
| category | entry 所属的分类 |
| contributor | entry 的贡献者 |
| published | entry 的初始创建时间 |
| rights | 版权信息 |
| source | 如果 entry 是副本，则包含源 feed 的元数据 |

## 通用构建块

### Category（分类）

- term（必需）：分类标识
- scheme（可选）：分类方案 URI
- label（可选）：人类可读的显示标签

### Content（内容）

content 元素可以包含或链接到 entry 的完整内容。根据 type 属性：
- text/html/xhtml：内容以文本形式定义
- src 属性：内容的 URI
- type 以 +xml 或 /xml 结尾：内联 XML 文档
- type 以 text 开头：转义文档
- 其他：base64 编码的文档

### Link（链接）

- href（必需）：引用的资源 URI
- rel（可选）：链接关系类型（默认为 alternate）
  - alternate：替代表示
  - enclosure：大型相关资源
  - related：相关文档
  - self：feed 自身
  - via：信息来源
- type：资源的媒体类型
- hreflang：引用资源的语言
- title：人类可读的链接信息
- length：资源长度（字节）

### Person（人物）

author 和 contributor 描述人员或实体：
- name（必需）：人类可读的名称
- uri（可选）：个人主页
- email（可选）：电子邮件地址

### Text（文本）

title、summary、content 和 rights 包含人类可读文本。type 属性决定编码方式：
- text：纯文本
- html：实体转义的 HTML
- xhtml：内联 XHTML（包裹在 div 元素中）

## 扩展性

Atom 内容元素设计为支持直接包含其他 XML 词汇表。任何完全限定的 URI 都可用作 link 元素的 rel 属性值。基本上可以使用 RSS 1.0 和 RSS 2.0 的模块。

## 示例 Feed

```xml
<?xml version="1.0" encoding="utf-8"?>
<feed xmlns="http://www.w3.org/2005/Atom">
  <title>Example Feed</title>
  <link href="http://example.org/"/>
  <updated>2003-12-13T18:30:02Z</updated>
  <author>
    <name>John Doe</name>
  </author>
  <id>urn:uuid:60a76c80-d399-11d9-b93C-0003939e0af6</id>
  <entry>
    <title>Atom-Powered Robots Run Amok</title>
    <link href="http://example.org/2003/12/13/atom03"/>
    <id>urn:uuid:1225c695-cfb8-4ebb-aaaa-80da344efa6a</id>
    <updated>2003-12-13T18:30:02Z</updated>
    <summary>Some text.</summary>
  </entry>
</feed>
```
