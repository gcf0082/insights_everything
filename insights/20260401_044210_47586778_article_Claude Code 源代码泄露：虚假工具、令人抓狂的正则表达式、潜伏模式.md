## 事件概述

2026年3月31日，Anthropic 在发布 Claude Code 的 npm 包时意外附带了 `.map` 文件，其中包含完整的可读源代码。该包随后被撤回，但代码已被广泛镜像。本文作者在获知消息后立即进行了代码审查，并整理了以下主要发现。

---

## 核心发现

### 1. 反蒸馏机制：注入虚假工具

在 `claude.ts` (第301-313行) 中存在一个名为 `ANTI_DISTILLATION_CC` 的标志。当启用时，Claude Code 会在 API 请求中发送 `anti_distillation: ['fake_tools']`，告诉服务器静默注入诱饵工具定义到系统提示中。

**工作原理**：如果有人录制 Claude Code 的 API 流量来训练竞争模型，这些虚假工具会污染训练数据。

**激活条件**（需全部满足）：
- `ANTI_DISTILLATION_CC` 编译时标志
- `cli` 入口点
- 第一方 API 提供商
- `tengu_anti_distill_fake_tool_injection` GrowthBook 标志返回 true

**绕过方法**：设置环境变量 `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS` 为真值可禁用整个机制。

### 2. 隐形模式 (Undercover Mode)

文件 `undercover.ts`（约90行）实现了一种模式，当 Claude Code 在非内部仓库中使用时，会去除所有 Anthropic 内部痕迹。模型被指示永远不要提及内部代号如 "Capybara" 或 "Tengu"、内部 Slack 频道、仓库名称或 "Claude Code" 本身。

**关键设计**（第15行）：
> "There is NO force-OFF. This guards against model codename leaks."
> （不存在强制关闭选项。这是为了防止模型代号泄露。）

可以通过 `CLAUDE_CODE_UNDERCOVER=1` 强制开启，但无法强制关闭。在外部构建中，整个函数会被死代码消除。

### 3. 沮丧情绪检测（使用正则表达式）

`userPromptKeywords.ts` 包含一个用于检测用户沮丧情绪的正则表达式：

```
/\b(wtf|wth|ffs|omfg|shit(ty|tiest)?|dumbass|horrible|awful|
piss(ed|ing)? off|piece of (shit|crap|junk)|what the (fuck|hell)|
fucking? (broken|useless|terrible|awful|horrible)|fuck you|
screw (this|you)|so frustrating|this sucks|damn it)\b/
```

这是 Hacker News 讨论中最受关注的发现。评论普遍认为：一家 LLM 公司使用正则表达式进行情感分析是极具讽刺意味的。

### 4. 原生客户端认证

在 `system.ts` (第59-95行) 中，API 请求包含占位符 `cch=00000`。在请求离开进程前，Bun 的原生 HTTP 栈（使用 Zig 编写）将这五个零覆盖为计算哈希值。服务器验证哈希以确认请求来自真实的 Claude Code 二进制文件，而非伪造客户端。

**技术细节**：
- 使用相同长度的占位符以避免更改 Content-Length 头
- 计算发生在 JavaScript 运行时之下，因此对 JS 层不可见
- 本质上是 API 调用的 DRM

**已知限制**：
- 整个机制由编译时特性标志 `NATIVE_CLIENT_ATTESTATION` 控制
- 可以通过设置 `CLAUDE_CODE_ATTRIBUTION_HEADER` 为假值来完全禁用
- 仅在官方 Bun 二进制中有效

### 5. 每日250,000次浪费的 API 调用

`autoCompact.ts` (第68-70行) 的源代码注释说明：

> "BQ 2026-03-10: 1,279 sessions had 50+ consecutive failures (up to 3,272) in a single session, wasting ~250K API calls/day globally."
> （2026年3月10日：1,279个会话出现50次以上连续失败，单个会话最多3,272次，全球每天浪费约25万次API调用。）

**修复方案**：`MAX_CONSECUTIVE_AUTOCOMPACT_FAILURES = 3`。连续3次失败后，会话剩余时间内自动压缩功能被禁用。

### 6. KAIROS：未发布的自主代理模式

整个代码库中多处引用了名为 `KAIROS` 的特性门控模式。基于 `main.tsx` 中的代码路径，这似乎是一个未发布的自主代理模式，包括：

- 用于"夜间记忆蒸馏"的 `/dream` 技能
- 每日追加日志
- GitHub webhook 订阅
- 后台守护进程 worker
- 每5分钟刷新一次的定时任务

### 7. 其他 Notable 发现

- **Companion 系统**：`buddy/companion.ts` 实现了类似 Tamagotchi 的伴侣系统，每用户根据用户 ID 通过 Mulberry32 PRNG 生成确定性生物（18个物种，稀有度等级，1%闪亮几率，RPG 统计属性）
- **终端渲染优化**：`ink/screen.ts` 和 `ink/optimizer.ts` 使用游戏引擎技术，声称可减少约50倍的 token 流式传输期间的 `stringWidth` 调用
- **安全检查**：每个 bash 命令都经过 `bashSecurity.ts` 中的23个编号安全检查，包括18个被阻止的 Zsh 内建命令
- **提示缓存经济**：有14个跟踪的缓存破坏向量，代码中存在标记为 `DANGEROUS_uncachedSystemPromptSection()` 的函数警告开发者
- **多代理协调器**：整个编排算法是一个提示而非代码

## 安全隐患分析

1. **源码地图暴露**：Anthropic 收购的 Bun 工具在生产模式下仍会提供源码地图（已知 Bug，#28001）
2. **Axios 漏洞**：代码库使用 Axios，而就在事件期间，npm 上的 Axios被发现存在恶意版本，投放远程访问木马
3. **代码质量问题**：`print.ts` 单个函数长达3,167行，嵌套12层

---

## 影响评估

1. **产品路线图泄露**：KAIROS、反蒸馏机制等是竞争对手现在可以看到的产品路线图细节
2. **战略优势丧失**：代码可以重构，但战略惊喜无法撤销
3. **法律与合规**：这是 Anthropic 一周内的第二次意外泄露（上次是模型规范泄露）

---

## 结论

这次源代码泄露事件暴露了 Anthropic Claude Code 内部运作的多个层面，包括：
- 反竞争的反蒸馏技术
- 隐藏 AI 身份的隐形模式
- 基于正则的情感检测
- 原生级别的客户端认证
- 未发布的产品功能

虽然技术层面的保护措施可以被绕过（通过 MITM 代理、环境变量配置等），但真正的保护可能来自法律层面而非技术层面。
