## 项目概述

Wander 是一个微小且完全去中心化的工具，用于探索"小网站"(small web)。该工具受 Kagi Small Web 启发而创建。Kagi Small Web 目前仅接受博客、漫画和 YouTube 频道，不接受任意小型网站，这一局限性促使开发者创建了 Wander。

## 技术特点

- **完全去中心化**: 任何人都可以在自己的网站上托管
- **极简架构**: 仅包含两个文件（`index.html` 和 `wander.js`）
- **无需服务端**: 无服务器端代码、无数据库、无需安装
- **易于部署**: 可托管在 GitHub Pages 或 Codeberg Pages 上
- **工作方式**: 类似 webring，但更加灵活。每个控制台可以链接任意数量的网站和其他控制台

## 社区反馈亮点

1. **去中心化优势**: 评论者赞赏 Wander 避免了 Kagi 使用 GitHub（一个高度中心化平台）的问题

2. **用户体验问题**:
   - 有人指出如果只链接到某个控制台而不在链接其他控制台，可能会"困住"用户
   - 开发者迅速响应，在下一个版本中实现了记录已发现控制台列表的功能

3. **功能建议**:
   - 添加"在新标签页打开"按钮（已实现）
   - 支持自定义 UI 样式（已支持）
   - 修复移动端 iframe 边距问题（已修复）

4. **浏览器兼容性**: 有用户反映在较老的浏览器（如 Palemoon 26.5）上无法运行，原因是使用了 ES6 语法（如 `let`）

5. **潜在问题**:
   - 部分网站设置了 `X-Frame-Options: SAMEORIGIN`，无法被嵌入
   - 刷新页面后会丢失之前的浏览历史

6. **类似项目比较**:
   - 与 StumbleUpon 相似，但无需中心化服务
   - 是 Webring 的进化版，从环形结构变为去中心化图结构

7. **扩展建议**:
   - 考虑支持 Gemini 和 Gopher 协议（但这些协议本身不支持 JavaScript）
   - 有建议增加轻量级信任图机制

## 总结

Wander 是一个有潜力的去中心化小网站探索工具，延续了 Webring 和 StumbleUpon 的精神。其完全去中心化的设计使其避免了单一平台依赖的问题。该项目在 HN 上获得了积极响应，开发者也积极采纳用户建议进行改进。
