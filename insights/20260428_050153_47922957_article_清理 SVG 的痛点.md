## 摘要

本文深入探讨了 Scratch 在处理用户生成的 SVG 内容时面临的安全挑战。Scratch 将用户创建的 SVG 内容解析并插入到主文档中，这一操作本质上是不安全的。多年来，Scratch 通过构建日益复杂的清理基础设施来移除危险内容，但作者认为这种方法注定失败。

## 核心问题

Scratch 的核心问题在于：**将用户生成的 SVG 内容解析并插入到主文档中是不可安全操作**。无论 SVG 在主文档中停留多长时间，这种做法都存在固有风险。Scratch 的做法是构建越来越复杂的解析基础设施来移除危险部分，但这种方法已被多次证明存在漏洞。

## 历史漏洞梳理

### 2019年：通过 script 标签的 XSS

2019年，发现 SVG 可以包含 `<script>` 标签，加载时会被执行。使用正则表达式移除 script 标签来修复。

**绕过方式**：大小写绕过（如 `<SCRIPT>`）

### 2020年：XSS 漏洞再次出现 (CVE-2020-27428)

之前的修复存在严重缺陷，可以通过大小写和其他方式绕过。DOMPurify 被引入来移除脚本。

**新攻击方式**：内联事件处理器
```xml
<svg>
  <foreignObject>
    <img src="data:invalid" onerror="alert(1)" />
  </foreignObject>
</svg>
```

### 2022年：通过 image href 的 HTTP 泄漏

DOMPurify 不保护 HTTP 泄漏，因为"方式太多，无法可靠处理"。攻击者可以记录加载项目的用户 IP 地址。

### 2023年：通过 CSS @import 的 HTTP 泄漏

使用 `<style>` 元素中的 CSS `@import` 语句可以发起外部请求。集成了 CSS 解析器来移除 @import。

### 2024年：通过 Paper.js 的 XSS

Scratch 在 scratch-svg-renderer 之前对 SVG 进行了清理，但未清理的 SVG 仍被传递给 Paper.js（在服装编辑器中使用）。

### 2025年：通过 CSS url() 的 HTTP 泄漏

在特定 CSS 规则中使用 `url()` 可以发起外部请求。

### 2026年：之前代码的多个 Bug

发现三个独特的漏洞：
- 未处理 CSS 转义代码
- 未处理 style 属性中的多个 url()
- 未处理 CSS 变量中的 url()

### 2026年：通过长转换的全页面重新样式

利用超长转换和强制浏览器重新样式所有元素，攻击者可以向整个 Scratch 页面应用任意样式。此漏洞**未被修复**。

### 2026年：通过 image-set() 的 HTTP 泄漏

作者于 2025 年报告此漏洞，但未被修复。这是**未修复**漏洞。

### 20XX：通过新 CSS 功能的 HTTP 泄漏

使用了未来可能实现的 CSS 功能（src() 和 image()），目前只有 Ladybird 浏览器实现。这是**未修复**漏洞。

### 2026-04-12：通过 CSS 嵌套宽松语法的 HTTP 泄漏

Claude 发现的新漏洞：css-tree 无法解析 CSS 嵌套的宽松语法，导致整个块被解析为"原始文本"节点而未被清理。

## 核心观点

1. **方法不可持续**：在清理方面堆叠越来越多的复杂性显然是注定失败的方法。已经 more than 5 major revisions deep and yet there are still known holes.

2. **新漏洞不断出现**：人们正在积极共享绕过 SVG 清理的项目。

3. **并非所有问题都有明确的解决方案**：对于全页面重新样式，两个 SVG 看起来完全无害。

## TurboWarp 的替代方案

TurboWarp（作者工作的 Scratch 分支）采用了一种替代方法：**使用 iframe 进行沙箱化处理**。

1. 使��带有 `sandbox="allow-same-origin"` 的 iframe
2. 设置严格的 Content-Security-Policy：
   - `default-src 'none'`
   - `style-src 'unsafe-inline' data:`
   - `font-src data:`
   - `img-src data:`

**优势**：
- 浏览器使用其现有代码来完成繁重的工作
- SVG 无法影响主文档
- 不需要了解所有可能的外部请求方式

## 结论

清理方法的根本问题在于依赖 css-tree 等第三方库来完美解析 CSS，而这种方法是绝望的路径，会继续导致更多漏洞。TurboWarp 的 iframe 沙箱方法提供了一种更可靠的替代方案。
