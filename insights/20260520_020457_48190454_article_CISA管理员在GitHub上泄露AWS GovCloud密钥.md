## 事件概述

2026年5月15日，安全研究公司GitGuardian的研究人员Guillaume Valadon发现，CISA的一名承包商在GitHub上维护了一个名为"Private-CISA"的公开代码仓库，其中暴露了多个高权限AWS GovCloud账户的凭证，以及大量内部CISA系统的敏感信息。安全专家表示，这是近年来最严重的政府数据泄露事件之一。

## 暴露的敏感信息

### 泄露的凭证类型

1. **AWS GovCloud管理员凭证**: 一个名为"importantAWStokens"的文件包含了三个Amazon AWS GovCloud服务器的管理凭证
2. **内部系统密码**: "AWS-Workspace-Firefox-Passwords.csv"文件包含数十个内部CISA系统的明文用户名和密码
3. **内部代码仓库凭证**: CISA内部"artifactory"（代码包仓库）的明文凭证
4. **日志和其他敏感文件**: 包括云密钥、令牌、明文密码、日志等

### 安全问题分析

安全专家Guillaume Valadon指出，此次泄露堪称安全意识薄弱的典型案例：
- CSV文件中存储明文密码
- 将备份数据存储在Git仓库中
- 明确禁用GitHub的秘密检测功能

Valadon表示："在我职业生涯中，这是见过的最严重的泄露。这显然是个人的错误，但可能反映出内部存在的安全实践问题。"

## 事件时间线

| 时间 | 事件 |
|------|------|
| 2018年9月 | 承包商的GitHub账户创建 |
| 2025年11月13日 | "Private-CISA"仓库创建 |
| 2026年5月15日 | GitGuardian发现并通知CISA |
| 2026年5月15日后 | 仓库被下线，但AWS密钥仍有效48小时 |

## 涉及方及回应

### 承包商
- 仓库由Nightwing公司（位于弗吉尼亚州杜勒斯）的员工维护
- Nightwing拒绝置评，将问题转交给CISA

### CISA声明
CISA发言人表示："目前没有迹象表明任何敏感数据因该事件而泄露。虽然我们要求团队成员保持最高标准的诚信和操作意识，但我们正在努力确保实施额外的保障措施，以防止未来发生类似事件。"

## 背景：人员裁减影响

CISA目前仅以正常预算和人员配置的一小部分运营。自第二届特朗普政府成立以来，该机构已失去近三分之一的劳动力，导致各部门的早期退休、买断和辞职。

安全专家Philippe Caturegli推测："我认为该承包商使用GitHub在工作和个人电脑之间同步文件，因为他自2025年11月以来定期向该仓库提交内容。这对任何公司来说都是尴尬的泄露，但考虑到这是CISA，情况更糟。"

## 安全风险

1. **横向移动风险**: 安全专家警告称，暴露的artifactory凭证是攻击者寻找持久立足点的理想目标
2. **供应链攻击风险**: 攻击者可能在软件包中植入后门，每次构建新项目时都会部署后门
3. **弱密码问题**: 仓库显示承包商对内部资源使用容易猜测的密码，例如使用平台名称加上当前年份作为密码

## 结论

这起事件暴露了CISA在承包商管理和内部安全实践方面的严重缺陷。虽然CISA表示目前没有证据表明敏感数据被泄露，但这一事件引发了对其安全实践和人员管理的广泛质疑。
