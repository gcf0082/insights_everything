## 报告摘要

Git 2.54 是开源 Git 项目的最新版本，包含了众多新特性和改进。本报告涵盖从 Git 2.52 和 2.53 以来最重要的功能更新。

---

## 一、主要新特性

### 1. git history 命令

Git 2.54 引入了全新的实验性命令 `git history`，专门用于简单的历史重写操作。

**支持的操作**：
- `git history reword <commit>`：修改提交消息，不影响工作树和索引，可以在 bare 仓库中操作
- `git history split <commit>`：交互式将一个提交拆分为两个

**设计限制**：
- 不支持包含合并提交的历史
- 拒绝可能导致合并冲突的操作

### 2. 基于配置的 Hooks

Git 2.54 引入了在配置文件中定义 hooks 的新方式：

```gitconfig
[hook "linter"]
   event = pre-commit
   command = ~/bin/linter --cpp20
```

**优势**：
- 可以在用户级别、系统级别或仓库级别配置
- 支持同一事件运行多个 hooks
- 可以通过 `git hook list` 查看配置的 hooks
- 可以通过 `hook.<name>.enabled = false` 禁用特定 hook

### 3. Geometric Repacking 成为默认维护策略

在 Git 2.52 中引入的 `geometric` 策略现已成为手动维护的默认策略。

**优势**：
- 避免昂贵的全量垃圾回收
- 增量合并 pack 文件
- 自动维护 commit-graph、reflogs 等辅助数据结构

## 二、其他重要更新

### 1. git add -p 改进

- 使用 `J`/`K` 键导航时显示之前接受或跳过的 hunks
- 新增 `--no-auto-advance` 选项，允许在文件间手动导航

### 2. git replay 成熟化

- 默认执行原子引用更新
- 新增 `--revert` 模式
- 支持丢弃在 replay 过程中变为空的提交
- 支持 replay 到根提交

### 3. HTTP 传输改进

- 支持 HTTP 429 "Too Many Requests" 响应处理
- 支持 `Retry-After` 头或通过 `http.retryAfter` 配置重试延迟
- 新增 `http.maxRetries` 和 `http.maxRetryTime` 配置选项

### 4. git log -L 改进

- 现在与标准 diff 管道兼容
- 支持 `-S` 和 `-G` pickaxe 选项
- 支持 patch 格式选项

### 5. 增量多包索引（ MIDX ）改进

- 支持紧凑化功能，合并较小的 MIDX 层
- 对长期仓库更加实用

### 6. git status 改进

- 新增 `status.compareBranches` 配置选项
- 可以同时比较上游和 push 远程

### 7. git rebase 改进

- 新增 `--trailer` 选项
- 自动为每个 rebased 提交追加 trailer

### 8. 签名验证改进

- 使用已过期 GPG 密钥的有效签名现在正确显示为有效签名

### 9. git blame 改进

- 新增 `--diff-algorithm` 选项
- 支持选择不同的 diff 算法

### 10. git backfill 改进

- 支持指定 revision 和 pathspec 参数
- 可以限定下载特定历史区域的 blobs

### 11. 别名配置改进

- 支持非 ASCII 字符的别名
- 使用新的子配置节语法

---

## 三、底层改进

1. **对象数据库（ ODB ）重构**：使用可插拔的后端设计，为未来功能奠定基础

2. **Histogram Diff 算法修复**：修复了输出质量问题的边界情况

---

## 总结

Git 2.54 带来了许多实用的改进，特别是 `git history` 命令、基于配置的 hooks 以及 geometric repacking 成为默认维护策略等变化，这些都显著提升了开发者的工作效率。
