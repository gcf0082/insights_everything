## 概述

Routines 是 Claude Code 的自动化运行配置，包含提示词、一个或多个代码仓库以及一组连接器，打包后可自动执行。Routines 在 Anthropic 管理的云基础设施上运行，因此即使关闭笔记本电脑也能持续工作。

## 核心功能

### 触发类型

Routines 支持三种触发方式，可组合使用：

1. **定时触发（Scheduled）**: 按周期性计划运行，如每小时、每天、每周或工作日
2. **API 触发**: 通过 HTTP POST 请求调用专属端点，携带 bearer token 启动 routine
3. **GitHub 事件触发**: 响应代码仓库事件，如 PR 创建、推送、问题更新或 GitHub Actions 工作流

### 可用计划

Routines 在 Pro、Max、Team 和 Enterprise 计划中可用，需启用 Claude Code on the web。

## 使用场景

| 场景 | 触发类型 | 描述 |
|------|----------|------|
| 待办事项维护 | 定时 | 每周晚间运行，读取新增问题并应用标签、分配负责人 |
| 告警分类 | API | 监控系统调用 routine API，关联错误日志并自动创建 PR |
| 自定义代码审查 | GitHub | PR 打开时运行，应用团队审查清单并留下评论 |
| 部署验证 | API | CD 流水线调用 routine 运行冒烟检查和回归扫描 |
| 文档同步 | 定时 | 每周扫描已合并的 PR，标记需要更新的文档 |
| 库移植 | GitHub | PR 合并时将变更移植到其他语言的 SDK |

## 创建方式

### 1. 从网页创建

访问 claude.ai/code/routines，点击 **New routine**，填写：
- Routine 名称和提示词（最重要的部分，需自包含且明确成功标准）
- 选择一个或多个 GitHub 仓库
- 选择云环境（控制网络访问、环境变量、安装脚本）
- 选择触发类型（定时、API、GitHub 事件）
- 配置连接器（默认包含所有已连接的 MCP 连接器）

### 2. 从 CLI 创建

在任意会话中运行 `/schedule` 命令，可通过描述直接创建，如 `/schedule daily PR review at 9am`。Claude 会引导完成配置过程。

### 3. 从 Desktop 应用创建

打开 Desktop 应用的 **Schedule** 页面，点击 **New task** 选择 **New remote task**。

## 配置详情

### 定时触发

预设频率：每小时、每天、工作日、每周。时间以本地时区输入会自动转换。

自定义间隔（如每两小时）可通过 cron 表达式配置，最小间隔为一小时。

### API 触发

创建后生成专属端点和 token。调用示例：

```bash
curl -X POST https://api.anthropic.com/v1/claude_code/routines/trig_01ABCDEFGHJKLMNOPQRSTUVW/fire \
  -H "Authorization: Bearer sk-ant-oat01-xxxxx" \
  -H "anthropic-beta: experimental-cc-routine-2026-04-01" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{"text": "Alert body text"}'
```

成功返回包含 session_id 和 session_url 的 JSON 响应。

### GitHub 事件触发

需在目标仓库安装 Claude GitHub App。可订阅的事件包括：

- Pull request（打开、关闭、同步等）
- Pull request review
- Push
- Release
- Issues
- Issue comment
- Discussion
- Check run/suite
- Workflow run/dispatch
- Repository dispatch

支持过滤器：作者、标题、正文、基础分支、头部分支、标签、草稿状态、是否合并、是否来自 fork。

## 管理功能

在 routine 详情页可以：
- 查看历��运行记录
- 点击 **Run now** 立即运行
- 暂停/恢复计划
- 编辑配置（名称、提示词、仓库、环境、连接器、触发器）
- 删除 routine

### 仓库和分支权限

默认情况下，Claude 只能推送到 `claude/` 前缀的分支，以防止意外修改保护分支。可在配置中启用 **Allow unrestricted branch pushes** 移除此限制。

### 连接器

Routines 可使用 MCP 连接器与外部服务交互。创建时默认包含所有已连接的连接器，可按需移除。

### 环境

每个 routine 在云环境中运行，控制网络访问、环境变量和安装脚本。需提前配置环境。

## 使用限制

Routines 消耗订阅使用量，有每日运行次数上限。超过限制时：
- 启用额外使用量的组织可继续按量计费运行
- 未启用额外使用量的账户会拒绝额外运行，直到窗口重置

可在 claude.ai/code/routines 或 claude.ai/settings/usage 查看当前消耗和剩余次数。
