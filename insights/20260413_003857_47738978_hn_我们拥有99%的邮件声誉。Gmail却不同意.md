## 原文摘要

Font Awesome 在博客中抱怨他们在 Gmail 的邮件投递 reputation 达到 99%，但 Gmail 仍然将他们的邮件标记为垃圾邮件。作者认为他们只会在有重要内容时发送邮件（约每几个月一次），但 Gmail 不这么认为。

## 主要洞察

### 1. 用户体验与商业利益的冲突

- **核心问题**：Font Awesome 在用户注册时强制要求邮箱，但这对于使用图标来说并非必要
- 用户只是想使用图标库，却被要求提供邮箱，这造成了期望与现实的脱节
- **关键洞察**：拥有邮箱地址 ≠ 获得用户同意接收营销邮件

### 2. 自动订阅的暗箱操作

- 许多公司在用户注册时默认勾选订阅选项，或者根本不询问是否愿意订阅
- 用户在不知不觉中被添加到多个邮件列表
- **这可能违反 GDPR 等隐私法规**

### 3. 用户对营销邮件的容忍度已耗尽

- 评论者普遍表示：即使在设置中关闭了促销邮件，仍然会收到营销内容
- 公司会添加新的邮件类别并默认启用
- 取消订阅后，公司会从其他列表继续发送，形成"无限取消订阅"的困境

### 4. Stripe 等大公司也采用类似策略

- 提供取消订阅选项，但会自动将用户添加到新的邮件列表
- 使取消订阅功能变得无效

### 5. Gmail 的 spam 检测逻辑

- Gmail 有基于用户反馈的 reputation 系统
- 如果足够多的用户标记为 spam，算法会自动调整投递策略
- **用户点击 "report spam" 是最直接的信号**

### 6. 邮件发送的技术问题

- Font Awesome 的邮件来自不同的子域名（可能是为了绕过垃圾邮件过滤器）
- "确认邮箱" 和 "完成账户设置" 邮件来自不同的子域名
- 这被视为试图绕过 spam 检测的负面行为

## 社区观点

### 反对方（认为 Font Awesome 应受谴责）

- "Opt-out is not consent. If I didn't opt in, I mark it as spam."
- 认为 Font Awesome 使用欺骗性手段获取用户
- 只要发送非用户请求的非事务性邮件，就是 spam

### 支持方（理解商业现实）

- 企业需要客户关系来生存
- "If you want to grow a business, you have to do this stuff"
- B2B 比 B2C 更容易，因为企业用户不会将营销邮件视为 spam

### 中立观点

- 认为 Google 的 spam 检测有时会误判合法邮件
- 建议使用 RSS 作为替代方案，但承认几乎没人使用 RSS

## 关键教训

1. **明确同意至关重要**：必须在注册时明确让用户选择是否订阅
2. **分离事务邮件和营销邮件**：使用不同的子域名和发送策略
3. **尊重用户取消订阅**：取消订阅应该立即生效，而非需要 60-90 天处理
4. **用户主导权**：用户有权决定他们想收到的内容类型和频率
5. **声誉是集体行为的结果**：单个用户的反馈会累积影响整体投递率

## 结论

Gmail 的判断基于**用户实际行为**而非发送者的自我认知。即使 Font Awesome 认为他们的内容"值得"收到，如果用户不这么认为，算法会站在用户一边。这个案例说明了现代电子邮件营销的核心挑战：**在用户注意力有限的环境中，强行发送用户不感兴趣的内容只会导致被标记为垃圾邮件，无论内容本身的质量如何。**
