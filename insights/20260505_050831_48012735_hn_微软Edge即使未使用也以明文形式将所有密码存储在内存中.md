# 洞察报告

**链接**: https://news.ycombinator.com/item?id=48012735
**标题**: Microsoft Edge stores all passwords in memory in clear text, even when unused
**发布时间**: 2小时前
**得分**: 223 points
**评论数**: 93 comments

## 内容摘要

该洞察揭示了微软Edge浏览器将所有保存的密码以明文形式存储在内存中，即使这些密码在当前会话中并未被使用。安全研究人员发现这一问题后，引发了社区广泛讨论。

## 主要观点

### 1. 威胁模型分析
- 如果攻击者能够读取任意进程内存，他们通常也已经能够通过其他方式获取密码
- 管理员权限可以附加调试器到Chrome进程并强制解密所有密码
- 冷启动攻击可能是唯一真正利用此问题的场景

### 2. 密码管理器的局限性
- 任何密码管理器都无法真正解决此问题，因为操作系统进程模型本身不安全
- 密码必须以明文形式存在才能使用，这是无法回避的事实
- Chrome、Firefox、Edge以及KeePass、Bitwarden等密码管理器都存在相同问题

### 3. 安全防护建议
- 使用硬件绑定的Passkey（密钥）而非传统密码
- 使用专用密码管理器（如KeePass）而非浏览器内置密码管理
- Windows的Credential Guard可以提供额外的保护层
- 在不同设备上备份多个Passkey

### 4. 社区反应
- 许多人认为这是"安全 vs 便利"的经典权衡
- 部分用户建议使用macOS的Keychain或Windows Hello等系统级凭据管理
- 有人指出即使密码被加密，解密密钥也必须存在于内存中

## 结论

虽然Edge在内存中以明文存储所有密码这一做法不够规范，但从技术角度来看，这是桌面操作系统架构的固有限制。只要攻击者能够访问进程内存，任何安全措施都难以完全保护明文密码。真正的解决方案可能需要依赖硬件级安全机制（如TPM、安全 enclave）或完全转向Passkey等无密码认证方案。
