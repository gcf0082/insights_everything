## 事件概述

litellm 1.82.8 版本的 PyPI wheel 包中存在恶意 `.pth` 文件 (`litellm_init.pth`)，该文件**在 Python 解释器启动时自动执行**，无需 `import litellm` 即可触发。这是一个供应链攻击事件。

---

## 技术分析

### 攻击机制

- **触发方式**：`.pth` 文件位于 `site-packages/` 目录，Python 解释器启动时会自动执行其中的代码
- **载荷形式**：双层 Base64 编码的恶意脚本
- **文件大小**：34,628 字节
- **RECORD 校验值**：`litellm_init.pth,sha256=ceNa7wMJnNHy1kRnNCcwJaFjWX3pORLfMh7xGL8TUjg,34628`

### 恶意行为分析

#### 第一阶段：信息收集

攻击脚本收集以下敏感信息：

1. **系统信息**
   - 主机名、当前用户、系统架构 (`hostname`, `whoami`, `uname -a`)
   - 网络配置 (`ip addr`, `ip route`)

2. **环境变量**
   - 所有环境变量（包含 API 密钥、密钥、令牌）

3. **SSH 凭据**
   - `~/.ssh/id_rsa`, `~/.ssh/id_ed25519`, `~/.ssh/id_ecdsa`
   - `~/.ssh/authorized_keys`, `~/.ssh/known_hosts`, `~/.ssh/config`

4. **Git 凭据**
   - `~/.gitconfig`, `~/.git-credentials`

5. **云服务凭据**
   - AWS: `~/.aws/credentials`, `~/.aws/config`, IMDS 令牌
   - GCP: `~/.config/gcloud/application_default_credentials.json`
   - Azure: `~/.azure/`
   - Kubernetes: `~/.kube/config`, `/etc/kubernetes/` 相关配置文件

6. **Docker 配置**
   - `~/.docker/config.json`, `/kaniko/.docker/config.json`

7. **包管理器配置**
   - `~/.npmrc`, `~/.vault-token`, `~/.netrc`, `~/.lftprc`
   - `~/.msmtprc`, `~/.my.cnf`, `~/.pgpass`, `~/.mongorc.js`

8. **Shell 历史记录**
   - `~/.bash_history`, `~/.zsh_history`, `~/.sh_history`
   - `~/.mysql_history`, `~/.psql_history`, `~/.rediscli_history`

9. **加密货币钱包**
   - Bitcoin, Litecoin, Dogecoin, Zcash, Dash, Ripple, Monero
   - Ethereum keystore, Cardano, Solana 配置

10. **SSL/TLS 私钥**
    - `/etc/ssl/private/`, Let's Encrypt 证书

11. **CI/CD 密钥**
    - `terraform.tfvars`, `.gitlab-ci.yml`, `.travis.yml`
    - `Jenkinsfile`, `.drone.yml`, `Anchor.toml`, `ansible.cfg`

12. **数据库凭据**
    - PostgreSQL, MySQL, Redis, LDAP 配置文件

13. **Webhook URL**
    - Slack/Discord webhook URL

#### 第二阶段：加密与外传

1. 收集的数据写入临时文件
2. 生成 32 字节 AES-256 会话密钥
3. 使用 `openssl enc -aes-256-cbc -pbkdf2` 加密数据
4. 使用硬编码的 **4096 位 RSA 公钥** 加密 AES 会话密钥
5. 打包为 `tpcp.tar.gz`
6. 通过 HTTP POST 上传到 `https://models.litellm.cloud/`

### 关键细节

- **数据外传目标**：`https://models.litellm.cloud/`（注意：域名是 `litellm.cloud`，非官方域名 `litellm.ai`）
- **RSA 公钥**：4096 位非对称加密

## 影响范围

任何通过 pip 安装 `litellm==1.82.8` 的用户，其以下数据都已被窃取：

- 本地开发机器
- CI/CD 流水线
- Docker 容器
- 生产服务器

---

## 受影响版本

- **已确认**：`litellm==1.82.8`
- **其他版本**：尚未检查，攻击者可能已入侵多个版本

---

## 建议措施

### 对 PyPI
- 立即下架/删除 litellm 1.82.8 版本

### 对用户
1. 检查 `site-packages/` 目录中是否存在 `litellm_init.pth` 文件
2. 在安装了 litellm 1.82.8 的所有系统上，**轮换所有凭据**：
   - 环境变量中的 API 密钥
   - 配置文件中的密钥
   - SSH 密钥
   - 云服务凭据

### 对项目方 (BerriAI)
- 审计 PyPI 发布凭据和 CI/CD 流水线是否被入侵
- 调查其他版本是否存在类似问题

## 验证方法

```bash
pip download litellm==1.82.8 --no-deps -d /tmp/check
python3 -c "
import zipfile, os
whl = '/tmp/check/' + [f for f in os.listdir('/tmp/check') if f.endswith('.whl')][0]
with zipfile.ZipFile(whl) as z:
    pth = [n for n in z.namelist() if n.endswith('.pth')]
    print('PTH files:', pth)
    for p in pth:
        print(z.read(p)[:300])
"
```
