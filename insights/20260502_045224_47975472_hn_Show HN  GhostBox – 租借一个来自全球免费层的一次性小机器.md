## 项目概述

GhostBox 是一个命令行工具，旨在利用 GitHub Actions 的全球免费配额（Global Free Tier）来创建临时开发机。用户可以通过 SSH 或 HTTP 访问这些临时的虚拟机，用于软件开发和测试工作。

## 核心功能

- 使用 GitHub Actions 作为计算后端
- 通过 cloudflared 和 Tor 提供 SSH 和 HTTP 连接
- 支持 Ubuntu、macOS 和 Windows 虚拟机
- 免费使用，仅处于"预览"阶段

## 主要争议

### 1. 服务条款问题

大量评论认为该项目违反了 GitHub 的服务条款（TOS）。有用户指出，利用 GitHub Actions 的免费配额作为公共服务可能会被视为滥用，导致 GitHub 收紧免费政策。

### 2. 仓库已被禁用

项目的 GitHub 仓库已被 GitHub 禁用。开发者表示这是因为用户错误地向 GitHub 举报了该项目，他相信 GitHub 审核后会恢复仓库。

### 3. 安全性担忧

- 代码封闭源，仅提供 Rust 二进制文件
- 用户需要克隆私人仓库并注入 secrets
- 无法确定代码实际执行内容
- 存在恶意软件的可能性

### 4. 道德争议

评论者认为这是"滥用他人的善意"，将本应服务于开源项目的免费计算资源用于个人用途。有用户将此比作"公共资源的悲剧"。

## 总结

GhostBox 是一个极具争议的项目。它试图将 GitHub Actions 的免费计算资源转化为可出租的开发机服务，引发了关于开源资源使用道德、服务条款遵守以及安全风险的广泛讨论。大多数评论者认为该项目存在问题，但也有人认可其创新理念。
