# 洞察报告：Open Source Resistance

## 基本信息

- **洞察链接**：https://ossresistance.com/
- **洞察日期**：2026-05-14
- **报告生成时间**：2026-05-14 02:00:08
- **来源网站**：Open Source Resistance

## 网站概述

Open Source Resistance（开源抵抗）是一个关于开源软件维护者行动宣言的网站，由 Mike McQuaid 创建。该网站主张维护者应该在公司时间内，利用工作时间来维护公司依赖的开源软件，而不是在业余时间免费贡献。

## 核心观点

### 主要论点

网站的核心论点是：开源软件赖以运行的基础设施是由维护者在工作时间维护的，而非仅靠业余时间。维护者不应再向那些赚取巨额利润的公司请求许可、争取少量时间或点击"赞助"按钮。当开源软件出现问题时，"抵抗者"会在公司时间内静默、专业地修复它。

### 宣言要点

1. **停止请求许可**：开源软件不是"业余爱好"。几乎所有公司都无法在不使用开源软件的情况下运营业务。公司每小时都在提取价值，却要求维护者请求周五下午的时间、捐赠按钮或几句好话。维护者应该而且可以在公司时间内直接利用工作时间处理这些必要的开源代码工作。

2. **三点行动指南**：
   - **做工作**：审查 PR、更新依赖并交付修复
   - **保护自己**：验证合同、保守工作机密、确保拥有开源 IP
   - **不要愚蠢**：不要把 100% 的工作时间花在开源上而被解雇

### 替代方案

网站也承认其他积极的做法，包括 Open Source Pledge（要求公司支付维护者每年每位开发者 2000 美元）和 Open Source Friday（要求公司捐赠时间）。作者认为这些都是有价值的，但"开源抵抗"是演进的下一步。

## 回应反对意见

网站针对常见的反对意见进行了反驳：

- **"这是偷窃"**：公司已经在从开源维护者身上提取价值。股东已经免费享受开源补贴数十年。
- **"你应该请求许可"**：请求许可会维持权力不平衡。维护者不需要经理的允许来维护雇主已经依赖的基础设施。
- **"这只是安静离职"**：安静离职是减少工作，而这个行动产出的是互联网基础设施。
- **"这会惩罚好雇主"**：好雇主通过允许开源维护时间来避免这个问题。

## 作者背景

Mike McQuaid 是 Homebrew 项目负责人，自 2009 年起担任 Homebrew 维护者，曾在 GitHub 担任 GitHub Sponsors 和 Open Source Friday 的联合创建者。他表示自己从未因为主要工作获得过报酬来从事开源项目，但在每一家雇主那里都这样做了，并且已经写了 "Open Source Maintainers Owe You Nothing" 一文。

## 法律警示

网站明确声明作者不是律师，并提供以下重要警示：

- 阅读雇佣合同，确保雇主不拥有你打算发布的开源 IP
- IP 转让通常是可协商的
- 不要泄露私人仓库、凭证、客户数据、路线图、未发布的漏洞或内部讨论
- 使用自己的判断，当政策、合同、客户义务或安全规则改变风险时
- 对于初级或不稳定工程师来说，这个论点最弱；对于修复雇主已经使用的公共依赖的高级维护者来说，这个论点最强

## 结论

Open Source Resistance 代表了一种激进但务实的方法来解决开源维护的可持续性问题。它主张维护者应该将开源维护视为工作的一部分，而不是额外牺牲的业余活动。这是一种"直接行动"的方式，让维护者自己掌握主动权。
