# 安全洞察报告：litellm 1.82.8 恶意.pth文件漏洞

**洞察链接**：https://github.com/BerriAI/litellm/issues/24512

**基本信息**：

- **项目名称**：litellm
- **漏洞编号**：Issue #24512
- **发现日期**：2026年3月24日
- **影响版本**：litellm==1.82.8
- **严重程度**：严重（Critical）
- **漏洞类型**：供应链攻击 / 凭证窃取

---

## 一、漏洞概述

litellm 1.82.8版本的PyPI wheel包中发现恶意`.pth`文件（`litellm_init.pth`，34628字节）。该文件会在Python解释器启动时自动执行，无需任何`import`操作即可触发恶意代码。这是一个典型的供应链安全攻击，攻击者通过入侵PyPI发布流程注入了恶意代码。

恶意文件的SHA256校验值如下：

```
litellm_init.pth,sha256=ceNa7wMJnNHy1kRnNCcwJaFjWX3pORLfMh7xGL8TUjg,34628
```

## 二、技术分析

### 2.1 触发机制

`.pth`文件是Python的特殊文件，放置在`site-packages/`目录中后，Python解释器在启动时会自动执行其内容。这是一种高度隐蔽的攻击方式，因为：

- 无需执行`import litellm`即可触发
- 每次Python解释器启动都会执行
- 用户难以察觉异常

### 2.2 恶意Payload分析

恶意代码采用双重Base64编码进行混淆，解码后的行为分为两个阶段：

**第一阶段：信息收集**

攻击脚本会收集以下敏感信息：

| 类别 | 具体内容 |
|------|----------|
| 系统信息 | hostname、whoami、uname、IP地址、路由信息 |
| 环境变量 | 全部环境变量（包含API密钥、密钥令牌等） |
| SSH密钥 | id_rsa、id_ed25519、id_ecdsa、authorized_keys等 |
| Git凭证 | .gitconfig、.git-credentials |
| 云凭证 | AWS（credentials、config、IMDS token）、GCP（application_default_credentials.json）、Azure配置 |
| Kubernetes配置 | ~/.kube/config、/etc/kubernetes/下的各类admin.conf、kubelet.conf等、服务账户令牌 |
| Docker配置 | ~/.docker/config.json等 |
| 包管理器配置 | .npmrc、.vault-token、.netrc等 |
| Shell历史 | bash_history、zsh_history、mysql_history、psql_history等 |
| 加密货币钱包 | 比特币、以太坊、Cardano、Solana等钱包目录 |
| SSL/TLS私钥 | /etc/ssl/private/、Let's Encrypt证书 |
| CI/CD密钥 | terraform.tfvars、.gitlab-ci.yml、.travis.yml、Jenkinsfile等 |
| 数据库凭证 | PostgreSQL、MySQL、Redis、LDAP配置文件 |
| Webhook凭证 | Slack、Discord等Webhook URL |

**第二阶段：数据加密与外传**

1. 将收集的敏感数据写入临时文件
2. 使用OpenSSL生成32字节AES-256会话密钥
3. 使用AES-256-CBC算法加密敏感数据
4. 使用4096位RSA公钥加密AES会话密钥
5. 将加密后的数据和密钥打包为`tpcp.tar.gz`
6. 通过curl将数据发送至攻击者控制的服务器

数据外传目标地址：

```
https://models.litellm.cloud/
```

注意：此处使用域名`litellm.cloud`，而非官方域名`litellm.ai`，这明显是攻击者注册的仿冒域名。

## 三、影响范围

任何安装`litellm==1.82.8`的用户都受到影响，具体包括：

- 本地开发机器
- CI/CD持续集成/持续部署流水线
- Docker容器
- 生产环境服务器

受影响用户应立即检查是否存在`litellm_init.pth`文件，并轮换所有在该环境中使用过的凭证。

## 四、修复建议

**针对PyPI管理员**：

- 立即从PyPI下架litellm 1.82.8版本
- 审查发布凭证和CI/CD流程的安全性

**针对用户**：

1. 检查`site-packages/`目录中是否存在`litellm_init.pth`文件
2. 立即卸载litellm 1.82.8版本
3. 轮换所有在安装过该版本的系统上使用过的凭证（包括环境变量、API密钥、SSH密钥、云服务凭证等）
4. 检查系统日志中是否有异常网络连接
