# 洞察报告：UK Biobank 健康数据持续被错误上传至 GitHub

**洞察链接**：https://biobank.rocher.lc  
**报告日期**：2026-04-25  
**数据来源**：GitHub DMCA 档案 (github.com/github/dmca)

---

## 摘要

UK Biobank 是一项英国生物医学数据库，持有50万英国志愿者的遗传、健康和生活方式数据。该数据库已向全球约20,000名研究人员提供数据访问权限，严格禁止进一步共享数据。然而，研究人员 repeatedly 将参与者数据错误地上传到公共 GitHub 仓库。本报告追踪了 UK Biobank 发出的 110 份 DMCA 删除通知，涉及 197 个代码仓库和 170 名开发者。

| 指标 | 数值 |
|------|------|
| 删除通知总数 | 110 份 |

## 目标文件类型分布

- **Jupyter/R 笔记本**：近 50%，可能包含少量数据行
- **遗传和基因组数据文件**：25%（PLINK、BOLT-LMM、BGEN）
- **表格数据集**：CSV、TSV、Excel 和序列化 R 对象
- **其他**：分析脚本、文档、压缩档案

## 关键事件时间线

- **2026年3月**：Guardian 调查报道曝光后，通知恢复
- **2026年3月14日**：The Guardian 披露 UK Biobank 参与者数据暴露
- **2026年4月17日**：最新通知日期

## 数据隐私风险

根据 The Guardian 的调查，仅凭两个信息（近似出生日期和单次大手术日期）即可重新识别暴露数据集中的志愿者身份。《英国医学杂志》(BMJ) 发表评论指出，UK Biobank 通过淡化重新识别风险来损害参与者利益。

## 结论

UK Biobank 面临持续的数据泄露治理挑战。尽管已发出大量删除通知，但数据暴露问题仍未得到有效解决。建议 Biobank 听取隐私专家意见，展现对数据保护问题的谦逊态度。
