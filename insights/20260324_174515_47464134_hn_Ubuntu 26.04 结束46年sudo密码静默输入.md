## 洞察摘要

Ubuntu 26.04 将改变 sudo 命令输入密码时不显示任何字符（静默输入）的传统行为，改为显示星号（*）作为反馈。这结束了 Unix/Linux 系统中长达 46 年的静默密码输入历史。

## 主要讨论观点

### 支持这一改变的理由

1. **用户体验改善**: 在高延迟 SSH 连接下，用户经常无法确定自己的按键是否被正确识别，静默输入造成了很多困扰。

2. **安全考量已过时**: 这一设计源于早期 Unix 时代的多用户办公环境，当时终端是共享的，有实体打印记录。但现代场景下，如果有人能通过肩后看到你的屏幕，他们同样可以观察到键盘输入，密码长度信息并不会显著增加风险。

3. **历史背景**: 早期 Unix 使用 56 位 DES 加密，实际只使用密码的前 8 个字符，因此 8 字符密码曾是标准做法。

### 反对或质疑的观点

1. **安全风险**: 透露密码长度可能帮助攻击者缩小暴力破解范围，尤其对于使用弱密码的用户。

2. **特殊场景**: 有用户提到父亲的密码只有一个字符，之前因为静默输入而受到"保护"。

3. **跨平台一致性**: 如果 Ubuntu 采用显示星号的方式，但其他 Linux 发行版或 SSH 连接的远程服务器不这样做，仍会造成混乱。

### 技术细节

- 这一改变发生在 sudo 二进制程序层面
- 如果 SSH 到 Ubuntu 26.04 主机，会看到星号反馈
- SSH 到其他操作系统则不会显示
- 用户可以通过配置将行为改回静默模式

## 结论

大多数 Hacker News 用户认为这是受欢迎的改进，是结束一个"令人困惑的怪癖"的好决定。虽然从安全角度不完全显示密码仍有其道理，但在现代计算环境下，这种保护的实际意义已经大大降低。
