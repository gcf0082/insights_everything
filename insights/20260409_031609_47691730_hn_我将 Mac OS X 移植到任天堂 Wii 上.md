## 项目概述

作者成功将 Mac OS X 10.0 Cheetah 原生移植到任天堂 Wii 上运行。这是一个极具挑战性的项目，涉及编写自定义引导程序、内核补丁和驱动程序。

## 关键信息

### 硬件兼容性分析

- **CPU**: Wii 使用 PowerPC 750CL 处理器，与 G3 iBooks 和某些 G3 iMacs 使用的 PowerPC 750CXe 有密切的演进关系
- **内存**: Wii 配备 88 MB 总内存（24 MB 1T-SRAM + 64 MB GDDR3 SDRAM），Mac OS X Cheetah 官方要求 128 MB，但实际可使用更少内存启动

### 技术实现方案

#### 1. 引导程序开发
- 基于 ppcskel 编写自定义引导程序
- 实现了 Mach-O 格式内核加载器
- 构建设备树（Device Tree）
- 使用 LED 指示灯作为调试工具（在串口和视频输出不可用时）

#### 2. 内核补丁
- 修复 BAT（Block Address Translations）设置
- 修改控制台输出重定向到 USB Gecko
- 使用 QEMU 在 Mac OS X Cheetah 虚拟机中进行开发

#### 3. 驱动程序开发

**IOKit 驱动模型：**
- 编写 NintendoWiiHollywood 驱动程序作为主驱动
- 为每个子设备创建 nub（ attach-point）
- 实现了 SD 卡驱动、帧缓冲驱动和 USB 支持

**SD 卡驱动：**
- 继承 IOBlockStorageDevice
- 通过 IPC 与 Starlet 协处理器通信
- 解决了缓存内存问题（使用非缓存内存）

**帧缓冲驱动：**
- 继承 IOFramebuffer
- 解决 Wii 视频硬件使用 YUV 颜色空间与 Mac OS X 使用 RGB 的不兼容问题
- 采用双帧缓冲策略：每秒60次将 RGB 像素数据转换为 YUV

**USB 支持：**
- 解决 AppleUSBOHCI 与 IOPCIDevice 的依赖问题
- 创建假的 PCI 设备 nub
- 解决字节序问题（reversed-little-endian）
- 通过 IRC 从 CVS 仓库获取了 IOUSBFamily 源代码

### 最终成果

- Mac OS X 10.0 Cheetah 在 Wii 上成功启动到桌面
- 可以运行 Mac OS X 安装程序
- 支持 USB 键盘和鼠标
- 实现了完整的安装流程
