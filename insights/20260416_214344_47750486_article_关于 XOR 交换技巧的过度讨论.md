## 1. XOR 的基本概念

### 1.1 什么是 XOR？

XOR 是 "Exclusive OR"（异或）的缩写，与"Inclusive OR"（或）相对应。两者的区别在于：当说"A 或 B"时，是否允许同时拥有 A 和 B？数学和计算领域需要更精确的定义，因此区分了 XOR（不允许同时）和 IOR（允许同时）。

**现实世界示例**：

- 迪士尼世界规则："禁止骚扰或伤害野生动物"——这是包含性或，因为你可能同时做两件事
- 餐厅菜单："牛肉热狗配橙子或薯条"——这是异或，你只能选择其中之一

### 1.2 逻辑 XOR 真值表

| A | B | A XOR B |
|---|---|---------|
| false | false | false |
| false | true | true |
| true | false | true |
| true | true | false |

**核心理解**：A XOR B 等同于问"A 和 B 不同吗？"

### 1.3 位运算 XOR

在大多数编程语言中，`^` 运算符执行按位 XOR。举例：

```
  12 = 1100
  10 = 1010
  ----------
  ^  = 0110 = 6
```

**关键性质**：

1. **自反性**：`a ^ a == 0`（任何值与自身异或等于零）
2. **恒等性**：`a ^ 0 == a`（与零异或不改变值）

组合使用：`a ^ b ^ b = a` 或 `a ^ b ^ a = b`

## 2. XOR swap trick 详解

### 2.1 原理

XOR swap trick 利用上述性质，在不使用临时变量的情况下交换两个变量：

```c
a ^= b;
b ^= a;
a ^= b;
```

### 2.2 具体示例（a=5, b=3）

| 步骤 | 操作 | a | b |
|------|------|---|---|
| 开始 | | 5 | 3 |
| 第1行 | a ^= b | 5 ^ 3 = 6 | 3 |
| 第2行 | b ^= a | 6 | 3 ^ 6 = 5 |
| 第3行 | a ^= b | 6 ^ 5 = 3 | 5 |

## 3. 实用性分析

### 3.1 场景一：交换局部变量

**测试代码**：

```c
int div_direct(int a, int b) {
    return a / b;
}

int div_xor_swap(int a, int b) {
    a ^= b;
    b ^= a;
    a ^= b;
    return a / b;
}

int div_temp_swap(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
    return a / b;
}
```

**编译结果（clang -O2, x86-64）**：

```asm
div_direct:
    mov   eax, edi
    cdq
    idiv  esi
    ret

div_xor_swap:
    mov   eax, esi
    cdq
    idiv  edi
    ret

div_temp_swap:
    mov   eax, esi
    cdq
    idiv  edi
    ret
```

**结论**：编译器完全看穿了交换操作，三种版本生成的代码完全相同！没有 XOR 指令，没有临时变量，编译器只是调整了寄存器中值的对应关系。

### 3.2 场景二：通过指针交换

**测试代码**：

```c
void swap_xor(int* a, int* b) {
    *a ^= *b;
    *b ^= *a;
    *a ^= *b;
}

void swap_temp(int* a, int* b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}
```

**编译结果**：

```asm
swap_xor:
    mov   eax, dword ptr [rdi]
    xor   eax, dword ptr [rsi]
    mov   dword ptr [rdi], eax
    xor   eax, dword ptr [rsi]
    mov   dword ptr [rsi], eax
    xor   dword ptr [rdi], eax
    ret

swap_temp:
    mov   eax, dword ptr [rdi]
    mov   ecx, dword ptr [rsi]
    mov   dword ptr [rdi], ecx
    mov   dword ptr [rsi], eax
    ret
```

**分析**：

- 临时变量版本：4 条指令
- XOR 版本：6 条指令（更差）

### 3.3 为什么编译器不能优化 XOR swap？

**根本原因**：当两个指针指向同一地址时，行为不同！

- `swap_temp(&x, &x)`：正常，结果不变
- `swap_xor(&x, &x)`：数据被销毁（第一次 `*a ^= *b` 变成 `x ^ x = 0`）

这意味着编译器不能用一种实现替换另一种，因为它们在别名情况下行为不同。

### 3.4 使用 restrict 关键字

```c
void swap_xor_restrict(int* restrict a, int* restrict b) {
    *a ^= *b;
    *b ^= *a;
    *a ^= *b;
}
```

**结果**：编译器生成 4 条指令的优化代码，与临时变量版本完全相同！

## 4. 加法交换技巧

### 4.1 原理

```c
a = a + b;
b = a - b;
a = a - b;
```

### 4.2 问题

- **有符号整数溢出**：在 C 中是有未定义行为（UB）
- **浮点数精度丢失**：当两个值相差过大时会丢失数据

**浮点数示例**：

```
a = 1.0, b = 1e16
a = a + b = 1e16
b = a - b = 1e16 - 1e16 = 0.0
a = a - b = 1e16 - 0.0 = 1e16

结果：a = 1e16, b = 0.0（1.0 丢失！）
```

## 5. 实际应用场景分析

### 5.1 手写汇编场景

在汇编中交换两个已在寄存器中的值：

- **传统方法（需要临时寄存器）**：
  ```
  move  $t2, $t0
  move  $t0, $t1
  move  $t1, $t2
  ```

- **XOR swap（无需临时寄存器）**：
  ```
  xor   $t0, $t0, $t1
  xor   $t1, $t1, $t0
  xor   $t0, $t0, $t1
  ```

### 5.2 真实架构分析

| 架构 | 是否需要 XOR swap |
|------|-------------------|
| MIPS | 32 个通用寄存器，在寄存器压力高的内循环中偶尔有用 |
| Z80 | `XOR` 只能写累加器，无法在任意寄存器间使用 |
| x86 | 从 1978 年起就有 `XCHG` 指令，完全不需要 |

## 6. 其他 XOR 技巧

### 6.1 找出唯一元素

**问题**：一个列表中每个值出现两次，除了一个唯一值，求找出唯一值。

```c
int find_unique(int* values, int n) {
    int result = 0;
    for (int i = 0; i < n; i++) {
        result ^= values[i];
    }
    return result;
}
```

**示例**：`{4, 7, 2, 7, 4}` → `4 ^ 7 ^ 2 ^ 7 ^ 4 = 2`

**复杂度**：O(n) 时间，O(1) 空间

## 7. 结论

### 7.1 核心发现

1. **局部变量交换**：编译器完全优化掉 XOR swap，生成相同代码
2. **指针交换（无 restrict）**：XOR 版本比临时变量版本更慢（6 vs 4 条指令）
3. **指针交换（带 restrict）**：编译器将 XOR swap 优化为临时变量版本
4. **历史价值**：在某些 1980 年代的架构（如 MIPS）可能有价值
5. **现代环境**：几乎完全无用，仅作为面试题和趣闻存在

### 7.2 建议

- **不要**在实际代码中使用 XOR swap trick
- 现代编译器会自动优化传统的临时变量交换
- 使用 `restrict` 关键字可以帮助编译器优化指针操作
- 加法交换技巧有更多问题（有符号溢出、浮点数精度丢失），应避免使用
