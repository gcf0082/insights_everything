## 项目概述

Cook 是一个简洁的命令行界面（CLI）工具，用于编排 Claude Code、Codex 和 OpenCode。该工具由开发者 RJ Corwin 创建，旨在解决 AI 代理工作流中的"一次性提示"问题——用户不再需要期望代理一次就做对，而是可以设计一个迭代直到达到质量阈值的系统。

## 核心功能

### 1. 原语（Primitives）

Cook 解析三类令牌：

- **Work（工作）**: 一个提示词，一次代理调用，核心单元
- **Loop operators（循环操作符）**: 用迭代包装工作：`xN`、`review`、`ralph`
- **Composition（组合操作符）**: 并行运行分支并解析结果：`vN`、`vs`、`pick`

### 2. 循环操作符

#### repeat（xN）
`xN` 运行工作 N 次顺序执行，每次都能看到之前的输出：
```sh
cook "Add dark mode" x3           # 3次顺序执行
cook "Add dark mode" repeat 3     # 长格式
```

#### review
`review` 添加审查→门控→迭代循环。工作完成后，审查者检查质量，门控决定 DONE 或 ITERATE：
```sh
cook "Add dark mode" review                    # 默认提示，最多3次迭代
cook "Add dark mode" "Review for accessibility" "DONE if WCAG AA, else ITERATE"
cook "Add dark mode" review --work-agent codex --work-model gpt-5-codex --review-agent claude --review-model opus
```

#### ralph
Ralph 用外部门控包装 cook，用于任务列表进度。工作提示是自指向的——它读取项目状态并决定下一步：
```sh
cook "Work on next task in plan.md" ralph 5 "DONE if all tasks complete, else NEXT"
```

### 3. 组合操作符

组合操作符在隔离的 git worktrees 中并行运行多个 cooks，然后使用解析器合并结果。

#### versions（vN / race N）
`vN` 并行运行 N 个相同的 cooks。`pick` 是默认解析器：
```sh
cook "Add dark mode" v3 "least code wins"     # 带 pick 标准
cook "Add dark mode" review v3 "cleanest"      # race 3，每个都有审查循环
```

#### versus（vs）
`vs` 并行运行两种方法，然后选择：
```sh
cook "Implement auth with JWT" vs "Implement auth with sessions" pick "best security"
```

### 4. 配置

运行 `cook init` 在项目根目录创建配置文件：
```sh
cook init
```

这会创建：
- `COOK.md` — 项目说明和代理提示模板
- `.cook/config.json` — 代理/模型/沙箱默认值和每步骤覆盖
- `.cook/Dockerfile` — Docker 沙箱模式的项目依赖
- `.cook/logs/` — 会话日志（gitignored）

示例 `.cook/config.json`：
```json
{
  "agent": "claude",
  "sandbox": "agent",
  "steps": {
    "work":   { "agent": "codex",  "model": "gpt-5-codex" },
    "review": { "agent": "claude", "model": "opus" }
  },
  "env": ["CLAUDE_CODE_OAUTH_TOKEN"]
}
```

### 5. 沙箱模式

| 模式 | 命令 | 说明 |
|------|------|------|
| Agent（默认） | `--sandbox agent` | 代理使用自己的 OS 级沙箱，不需要 Docker |
| Docker | `--sandbox docker` | 代理在具有受限网络访问的 Docker 容器中运行 |

注意：OpenCode 仅在 Docker 模式下受支持。

### 6. 速率限制恢复

当代理遇到 token 配额或速率限制时，cook 会自动等待并重试，而不是退出。TUI 中显示倒计时。默认启用。

## 安装方式

### Skill 方式（无 CLI）
```sh
mkdir -p .claude/skills/cook && curl -fsSL https://raw.githubusercontent.com/rjcorwin/cook/main/no-code/SKILL.md -o .claude/skills/cook/skill.md
```

### CLI 方式
```sh
npm install -g @let-it-cook/cli
mkdir -p .claude/skills && cp -r $(npm root -g)/@let-it-cook/cli/skill .claude/skills/cook
```

要求：Node.js 20+ 和 Claude Code、Codex 或 OpenCode。

## 社区反馈

### 正面观点
- 为 Claude Code 新手提供了"友好的入门点"
- 解决了可重复多步骤工作流的需求
- 文档简洁易读
- 支持灵活的代理和模型配置

### 质疑观点
- 有用户认为可以直接用 bash 脚本调用 Claude Code
- 对于非常特定的工作流，自定义脚本可能更灵活
- 有人指出 token 成本是真实问题，建议将 80% 的任务切换到更小的模型

### 有趣评论
- 有人打趣说这是一个"元"工具：用提示引导工作流，然后用提示将其转换为 opencode 插件，然后用提示生成网站来宣传工具，然后再用提示构建自动整合反馈的工具
- 开发者回应说唯一的区别是"我不在工作中消耗我的 tokens"

## 技术特点

1. **隔离执行**: 每个并行分支在隔离的 git worktrees 中运行，确保不会相互干扰
2. **速率限制恢复**: 自动处理 API 限流，可配置等待行为
3. **灵活的代理选择**: 支持 Claude、Codex、OpenCode，可为不同步骤选择不同代理
4. **任务列表支持**: Ralph 操作符支持基于项目计划文件的任务进度管理

## 总结

Cook 代表了 AI 辅助编码工作流的演进方向——从"一次提问，期待最好结果"转变为"设计一个迭代直到达到质量阈值的系统"。它填补了 Claude Code 在可重复多步骤工作流方面的空白，为开发者提供了一种结构化的方式来编排 AI 代理。随着 AI 编码工具的普及，这类编排工具可能会变得越来越重要。
