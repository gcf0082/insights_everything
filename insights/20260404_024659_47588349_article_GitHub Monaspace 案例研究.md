## 项目背景

GitHub Next（GitHub 旗下的实验性研究孵化器）于 2021 年 8 月首次联系 Lettermatic。GitHub Next 负责人 Idan Gazit 对精美和功能性排版有强烈兴趣，这促使双方探讨了专门为代码编辑器设计的字体问题。

**核心问题**：
- 为什么现代开发工具出现以来，代码编辑器的字体几乎没有改变？
- 为什么开发者几乎没有选择来定制他们的代码编写体验？
- 为什么开发者每天必须使用单一字体？

这些问题的探索最终促成了 Monaspace 的诞生。

## Texture Healing（纹理修复）技术

Texture Healing 是 Lettermatic 为本项目开发的上下文感知功能，旨在解决所有等宽字体的固有纹理问题：当所有字母被强制放入相同大小的框时，会产生挤压的字母和大量的负空间。

### 工作原理

1. **识别宽窄字母组合**：当宽字母（如 `m`、`w`）出现在窄字母（如 `i`、`l`）旁边时
2. **自动调整**：让窄字母（如 `i`）收缩，为宽字母（如 `m`）腾出空间
3. **保持等宽网格**：字母变宽或变窄时，始终向同一侧变化，保持字形的边界框完全相同

### 技术渊源

- 上下文感知字体的概念已存在多年
- Donald Knuth 在 1970 年代曾尝试过类似技术
- 自 1990 年代起，理论上已可以实现类似 Texture Healing 的机制
- 但由于字体渲染底层技术陈旧且复杂，此前的等宽字体中很少见到这类实验

### 扩展应用

Texture Healing 不仅限于字母，还包括标点符号（如下划线、逗号、句号、冒号和分号），这些符号在代码中经常出现，可以从它们那里借用一些额外空间来帮助更麻烦的压缩字母。

## 五种字体家族

| 字体 | 风格描述 | 设计特点 |
|------|----------|----------|
| **Argon** | 人文主义无衬线 | 融入书法灵感的小细节，参考人类手写字母的历史，但经过理性化处理，冷静而不引入尖锐的笔触细节 |
| **Neon** | 哥特式无衬线 | 遵循"国际风格"的传统，是数字设计中最常用的字体类型之一 |
| **Xenon** | Slab 衬线 | 提供尖锐、活泼的编辑风格，衬线更加棱角分明，让人联想到打字机的文学风格 |
| **Radon** | 手写体 | 灵感来自手写形式，模仿用圆珠笔速记的方式，让代码注释具有人的温度 |
| **Krypton** | 机械无衬线 | 采用 OCR（光学字符识别）传统，提供冷冰冰的机器语气，强调"这是计算机在说话" |

## 个性化功能

### 可变字体轴
- **字重（Weight）**：控制字体的粗细
- **字宽（Width）**：控制字体的宽窄
- **倾斜（Slant）**：控制字体的倾斜程度

### 字符变体
提供九类字符变体选项，包括：
- 零的多样化形式：点状零、斜杠零、反斜杠零、切割斜杠零、传统零
- 星号点数选择：五角或六角星号

### 编程连字
- 添加了近 100 个自定义连字
- 示例：`-->` 自动转换为完整箭头，`###` 连接起来，`===` 替换为三横等号
- 通过 OpenType 控制，可单独启用十类连字

### 语言支持扩展
- 初始版本支持超过 200 种语言
- 1.4 版本新增西里尔文、希腊文和越南文支持
- 总计增加超过 6400 个全新绘制的字形
- 所有新语言版本均包含 Texture Healing 功能

## 无障碍设计

从项目最初阶段，团队就希望将无障碍功能作为 Monaspace 字体系统的标准配置：

- **低视力用户**：可通过调整字重和字宽来减少整体字号，在屏幕上显示更多内容，减少滚动
- **色盲用户**：可创建完全不依赖颜色的层级结构，通过字体的混合搭配来实现

---

## 核心价值

1. **高可定制性**：通过字体风格、字重、字宽和倾斜提供前所未有的定制能力
2. **Texture Healing 技术**：为等宽字体提供前所未有的可读性提升
3. **共享网格设计**：五种字体在同一网格上设计，可以在同一文档中混合搭配而不破坏等宽网格
4. **开源免费**：遵循 GitHub 的开源使命，任何人都可以免费下载使用
