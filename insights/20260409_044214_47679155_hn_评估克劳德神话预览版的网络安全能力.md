## 内容摘要

该洞察讨论了 Anthropic 发布的关于 Claude Mythos Preview 网络安全能力的评估报告。该模型展示了自主发现和利用软件漏洞的能力，包括 Linux 本地权限提升、浏览器漏洞利用等。

## 主要发现

### 漏洞发现成果
- Mythos Preview 在测试中发现了多个系统的新漏洞
- 自主获取了 Linux 等操作系统的本地权限提升漏洞
- 通过利用微妙的竞态条件和 KASLR 绕过技术实现攻击
- 在 OpenBSD、FFmpeg、FreeBSD 等软件中发现漏洞

### 关键漏洞
- 在生产级内存安全 VMM 中发现内存损坏漏洞（未公开细节）
- OpenBSD TCP 实现中的关键 DOS 漏洞
- Linux 内核中的多个本地权限提升漏洞

## 社区讨论要点

### 1. 嵌入式设备安全问题
- **avsm**：数亿嵌入式设备难以升级，将运行有漏洞的二进制文件。唯一实际防御是让前沿模型生成"有益"攻击来为旧二进制文件接种。
- **gmuslera**：坏人也更容易发现漏洞。应尽快淘汰未维护且可远程访问的设备。
- **linzhangrun**：许多非专业软件公司运行 Windows Server 2012 和 PHP 5.3，从不升级。存在 SQL 注入漏洞。

### 2. 质疑与批评
- **staticassertion**：希望看到针对非数十年历史的 C/C++ 代码库的目标。KASLR 对防止 LPE 基本无用。
- **rfoo**：Mythos 在生产级内存安全 VMM 中发现漏洞，这是个重大发现。
- **Aloisius**：希望他们针对自己的代码库，因为经常看到 Claude 代码中的安全漏洞。

### 3. AI 安全担忧
- **awestroke**：这开始变得有些可怕，希望 LLM 智能能尽快达到某种 plateau。
- **lebovic**：RL 在网络安全领域扩展良好，且模型并非专门针对网络安全训练，仍有很大提升空间。
- **vonneumannstan**：我们几乎没有在冷战中幸存下来，有时是因为纯粹运气。AI 至少和冷战一样危险。

### 4. 积极观点
- **hibikir**：在网络安全领域，我们从不通过不关注来获胜。需要顶尖的破解系统知识才能保护它。
- **torginus**：LLM 在奖励函数已知的领域（如利用漏洞）要强大得多。
- **michaelbuckbee**：由于立法和商业压力，情况确实有所改善。

### 5. 断开连接的讨论
- **creata**：另一个实际防御是断开这些设备的连接。
- **WhyNotHugo**：这就是为什么这么多设备不应该连接到互联网。互联网连接的中央供暖系统看起来绝对疯狂。
- **oytis**：应该实现 OTA 更新，否则不要将设备连接到网络。
