## 洞察摘要

2026年4月23日，我们的纯OCaml CCSDS协议栈在低地球轨道启动运行！该项目代号*Borealis*，运行在DPhi Space的ClusterGate-2有效载荷模块上，具有端到端加密的指令控制和后量子密钥轮换功能，全部采用安全的OCaml实现。

---

## 核心要点

### 1. 项目背景

- **首次运行**: 2026年4月23日，纯OCaml CCSDS协议栈首次在太空运行
- **运行平台**: DPhi Space的ClusterGate-2有效载荷模块
- **卫星轨道**: 卫星每90分钟绕地球一圈

### 2. 技术架构

Borealis是一个守护进程，在地面和卫星上使用相同的客户端-服务器协议（遥测查询、命令和响应、OTAR密钥更新请求）。

**协议栈包含**:
- 纯OCaml实现的CCSDS协议族（连接航天器与地面）
- 从无线电帧到Bundle Protocol的每一层
- BPSec安全扩展（加密和认证）

**安全特性**:
- BPSec为每个Bundle添加两个扩展块：一个加密有效载荷，一个进行认证
- 序列号拒绝重放攻击
- 预共享密钥（通过OTAR轮换）将路由路径排除在信任路径之外

### 3. 为什么选择OCaml

**安全考量**:
- 在卫星上运行不受信任的代码是巨大的安全风险
- OCaml是理想的太空安全语言
- 约70%的C/C++严重CVE源于内存损坏（缓冲区溢出、释放后使用、整数溢出）
- NASA CryptoLib曾有堆缓冲区溢出漏洞（TC帧解析器中的整数下溢）

**性能优势**:
- 使用Jane Street的OxCaml编译器分支
- 模式系统允许标记堆栈绑定的分配
- p99.9延迟从**29ns降至9ns**
- 25百万个数据包上GC事件从**394次降为0次**

### 4. 后量子密钥轮换（OTAR）

- 支持ML-DSA-65后量子签名密钥的OTAR
- 密钥生命周期为卫星寿命（10-15年）
- 这将是**首个公开的轨后量子OTAR演示**

### 5. 技术细节

**卫星端循环**:
```ocaml
let send_telemetry t ~prefix payload =
  let bundle = make_bundle t ~source:Eid.sat_telem
    ~destination:Eid.ground_telem ~payload
  in
  match protect_bundle t bundle with
  | Ok protected ->
    ignore (write_bundle t ~dir:(downlink_dir t.config) ~prefix protected)
  | Error _ -> log t "Failed to protect telemetry bundle"
```

**命令调度**:
```ocaml
type cmd = Ping | Check | Capture | Halt

let dispatch t = function
  | Ping    -> send_response t ~prefix:"pong" "PONG"
  | Check   -> run_self_check t
  | Capture -> capture_and_send t
  | Halt    -> t.shutdown_requested <- true
```

### 6. OxCaml的性能提升

| 指标 |  stock OCaml 5.3.0 | OxCaml 5.2.0+ox |
|------|-------------------|-----------------|
| p99.9延迟 | 29 ns | 9 ns |
| GC事件 | 394次 | 0次 |

**优化方法**:
- 将每次迭代的堆分配转为堆栈分配
- 使用`exclave_ stack_`注解
- 类型系统证明记录不能逃逸调度范围

### 7. 防御层次

1. **类型检查**: 协议状态机编码为GADT，类型检查器在编译时拒绝无效转换
2. **形式化验证加密原语**: libcrux、fiat-crypto
3. **互操作测试**: 针对现有参考实现运行
4. **依赖审计**

## 结论

Borealis在轨道上作为一个二进制文件运行。MirageOS核心库已在地面生产运行十年，现在在ClusterGate-2上运行于太空。这证明了OCaml作为安全、可靠的太空软件语言的可行性，同时展示了函数式编程在航天领域的实际应用价值。
