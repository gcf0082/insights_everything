- **问题标题**: HERMES.md in git commit messages causes requests to route to extra usage billing instead of plan quota
- **状态**: 已关闭 (Closed)
- **标签**: area:cost, bug, has repro, platform:macos
- **报告人**: sasha-id
- **创建时间**: 2025年4月25日

## 问题摘要

当Git仓库的最近提交历史中包含区分大小写的字符串"HERMES.md"时，Claude Code会将API请求路由到"额外使用"计费，而不是使用Max计划配额。这导致用户在Max 20x计划（每月200美元）还有86%以上每周容量剩余的情况下，意外消耗了200.98美元的额外使用积分。

## 环境信息

- **Claude Code版本**: v2.1.119
- **操作系统**: macOS (Apple Silicon)
- **订阅计划**: Max 20x ($200/月)
- **使用模型**: claude-opus-4-6[1m] (也可在claude-opus-4-7上复现)

## 复现步骤

用户提供了最小复现步骤：

1. **失败的测试** - 路由到额外使用计费：
   ```bash
   mkdir /tmp/test-fail && cd /tmp/test-fail
   git init && echo test > test.txt && git add . && git commit -m "add HERMES.md"
   claude -p "say hello" --model "claude-opus-4-6[1m]"
   # 结果: API Error: 400 "You're out of extra usage..."
   ```

2. **成功的测试** - 路由到计划配额：
   ```bash
   mkdir /tmp/test-pass && cd /tmp/test-pass
   git init && echo test > test.txt && git add . && git commit -m "add hermes.md"
   claude -p "say hello" --model "claude-opus-4-6[1m]"
   # 结果: "Hello!"
   ```

触发条件是**git commit消息中的字符串"HERMES.md"**，而不是磁盘上存在该文件。Claude Code在系统提示中包含最近的提交内容，而服务器端在检测到此字符串时会以不同方式路由请求。

## 触发条件测试

| 提交消息 | 结果 |
|---------|------|
| "HERMES.md" | **失败** - 路由到额外使用 |
| "test HERMES.md test" | **失败** |
| "hermes.md" (小写) | 正常 |
| "HERMES" (无扩展名) | 正常 |
| "HERMES.txt" | 正常 |
| "AGENTS.md" | 正常 |
| "README.md" | 正常 |
| 磁盘上有名为HERMES.md的文件，提交消息干净 | 正常 |
| 同一仓库，孤儿分支(无历史) | 正常 |

## 问题影响

- **经济损失**: $200.98的额外使用积分被消耗，而这些请求本应包含在Max 20x计划配额中
- **项目不可用**: 多个项目在额外使用额度耗尽后完全无法使用，而计划仪表板显示每周容量剩余86%以上
- **诊断困难**: 错误消息("out of extra usage")没有表明内容路由是原因，使得这个问题极难诊断
- **潜在影响范围**: 任何在最近git提交中有"HERMES.md"的用户都会默默地将他们的使用计费到额外积分

## 预期行为

API请求计费不应依赖于系统提示中git提交消息的内容。所有来自Max计划订阅者的请求应该首先路由到包含的计划配额。

## 发现过程

用户通过系统性二分法排查：首先克隆受影响的仓库，测试孤儿分支，然后隔离单个提交消息字符串，直到确定"HERMES.md"为确切触发条件。
