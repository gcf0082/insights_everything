## 核心发现

本文探讨了 Ruby 中 `Date::ITALY` 常量的含义及其背后的历史背景。该常量值为 `2299161`，代表意大利从儒略历切换到格里历的儒略日数。

---

## 详细总结

### 一、Ruby 中的日历常量

Ruby 的 Date 类提供了多个看似奇怪的常量：

- `Date::ITALY` — 2299161，代表意大利及部分天主教国家采用格里历的日期
- `Date::ENGLAND` — 英格兰及其殖民地的日历改革日期
- `Date::GREGORIAN` — 格里历改革的儒略日数（值为 `-Infinity`）
- `Date::JULIAN` — 儒略历改革的儒略日数（值为 `Infinity`）

这些常量用于定义不同地区从儒略历向格里历过渡的日期。

### 二、格里历转换历史

儒略历由凯撒大帝创立，每128年误差约一天。到1582年教皇格里高利十三世时期，复活节的日期已偏离约两周。1582年，教皇发布通谕规定：1582年10月4日（星期四）之后直接跳到10月15日（星期五），跳过了10天。

在 Ruby 中，这段时间的日期会引发异常：

```ruby
Date.new(1582, 10, 5)  # 抛出 Date::Error 异常
```

而日期迭代会自动跳过这十天：

```ruby
Date.new(1582, 10, 4).next_day  # 返回 1582-10-15
```

### 三、儒略日数系统

儒略日系统起源于公元前4713年1月1日正午 UTC，这一时刻三个最重要的多年周期完美同步：28年太阳周期、19年月球周期、15年罗马税务周期（指示周期）。儒略日周期为 28 × 19 × 15 = 7980 年，下一次周期轮换将在3268年。

儒略日作为不同日历系统之间的“通用中间人”，解决了各国采用格里历时间不同的问题。例如英格兰直到1752年才采用格里历（因此存在 `Date::ENGLAND`）。

### 四、Date 类的实现细节

Ruby 的 Date 初始化器默认参数：

```ruby
new(year = -4712, month = 1, mday = 1, start = Date::ITALY)
```

默认日期 `-4712-01-01` 等同于 `Date.jd(0)`。可选的 `start` 参数表示格里历切换的起始日期。

使用示例：

```ruby
# 在英国SaaS产品中表示牛顿生日
Date.new(1642, 12, 25, Date::ENGLAND)  # 而非 Date.new(1643, 1, 4)
```

如果想始终使用儒略历，可以将“格里历切换”推向无穷远的未来，因此 `Date::JULIAN` 为 `Float::INFINITY`。而 `Date::GREGORIAN` 为 `-Float::INFINITY`，表示“一直在使用格里历”。

如果要模拟俄罗斯的日历转换（1918年2月14日采用格里历）：

```ruby
Date.new(1918, 1, 31, 2421639).next_day  # 返回 1918-02-14
```

### 五、Date 对象的内部表示

在 Ruby 的 `inspect` 格式中，例如：

```ruby
#<Date: 1582-10-15 ((2299161j,0s,0n), +0s,2299161j)>
```

这表示：儒略日数、秒和纳秒（时间组件）、时区偏移、格里历切换的儒略日数。`DateTime` 会使用秒和纳秒部分表示时间。

## 有趣知识点

1. **术语"推算"（Proleptic）**：指外推日历日期到其采用或使用之前的时间。

2. **儒略日的命名来源**：并非来自儒略历或凯撒本人，而是来自约瑟夫·斯加利格的父亲朱利叶斯·凯撒·斯加利格，而朱利叶斯·凯撒·斯加利格的名字又来自凯撒。这与“凯撒沙拉”命名来源类似（由凯撒·卡尔迪尼发明，其名字又来自凯撒）。

3. **三循环系统**：儒略周期是三个其他周期的同步，因此被称为“三循环系统”。

4. **历史记录中的指示周期**：历史记录往往与指示周期数相关联，而非具体日期。
