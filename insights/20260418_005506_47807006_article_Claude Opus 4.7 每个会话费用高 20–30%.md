## 核心发现

Anthropic 的 Claude Opus 4.7 迁移指南声称新分词器使用"大约 1.0 到 1.35 倍"的 tokens。但作者实际测量发现：**在真实内容上达到 1.47 倍，在 CLAUDE.md 文件上达到 1.45 倍**。这意味着大多数 Claude Code 用户的实际内容位于 Anthropic 声明范围的**上限**而非中位数。

## 成本实测

### 真实 Claude Code 内容测试

| 用户提示词 | 4,405 | 1,122 | 1,541 | **1.373** |
| 博客文章摘要 | 5,000 | 1,209 | 1,654 | **1.368** |
| Git 提交日志 | 2,853 | 910 | 1,223 | **1.344** |
| 终端输出 | 2,210 | 652 | 842 | **1.291** |
| Python 堆栈跟踪 | 5,255 | 1,736 | 2,170 | **1.250** |
| 代码差异 | 4,540 | 1,226 | 1,486 | **1.212** |

**加权平均比率**：1.325x（8,254 → 10,937 tokens）

## 分词器变化分析

1. **CJK、Emoji 和符号内容**：移动 1.005–1.07x，非拉丁语部分变化较小
2. **英文和代码**：移动 1.20–1.47x，4.7 使用更短或更少的子词合并
3. **代码比独特散文受影响更大**（1.29–1.39x vs 1.20x）

每字符 token 数：英文从 4.33 降至 3.60，TypeScript 从 3.66 降至 2.69。

## 指令遵循能力测试

使用 IFEval 基准测试（541 个提示词，采样 20 个）：

| 指标 | 4.6 | 4.7 | 变化 |
|-----|-----|-----|------|
| 严格模式（提示词级） | 17/20 (85%) | 18/20 (90%) | **+5pp** |
| 严格模式（指令级） | 25/29 (86%) | 26/29 (90%) | +4pp |
| 宽松模式（提示词级） | 18/20 (90%) | 18/20 (90%) | 0 |
| 宽松模式（指令级） | 26/29 (90%) | 26/29 (90%) | 0 |

**结论**：4.7 在严格指令遵循上略有提升（+5pp），但样本量较小（仅 20 个），效果真实但幅度不大。

## 费用计算（80 轮会话场景）

### 4.6 会话成本

| 项目 | 计算 | 费用 |
|-----|------|------|
| 首次缓存写入 | 8K × $6.25/MTok | $0.05 |
| 缓存读取（79轮） | 79 × 86K × $0.50/MTok | $3.40 |
| 新鲜用户输入 | 79 × 500 × $5/MTok | $0.20 |
| 输出 | 80 × 1,500 × $25/MTok | $3.00 |
| **总计** | | **~$6.65** |

### 4.7 会话成本

| 项目 | 计算 | 费用 |
|-----|------|------|
| 首次缓存写入 | 10K × $6.25/MTok | $0.06 |
| 缓存读取（79轮） | 79 × 115K × $0.50/MTok | $4.54 |
| 新鲜用户输入 | 79 × 660 × $5/MTok | $0.26 |
| 输出 | 80 × 1,500-1,950 × $25/MTok | $3.00-$3.90 |
| **总计** | | **~$7.86-$8.76** |

### 成本差异

**约 $6.65 → $7.86-$8.76，每会话增加约 20-30%**

## 对提示缓存的影响

1. **首次会话需重建缓存**：4.7 的缓存分区与 4.6 分离，冷启动成本更高
2. **缓存容量增长 1.445 倍**：CLAUDE.md 部分需要支付更多缓存写入和读取费用
3. **相同内容不同计数**：从 4.6 切换到 4.7 后，日志显示的 token 数量会发生变化

## 总结

- **Tokens 成本**：英文和代码内容增加 1.3-1.45 倍
- **收益**：严格指令遵循提升约 +5pp
- **价格**：单 token 价格不变，但每会话有效成本增加约 20-30%

**是否值得？** 取决于你的使用场景。对于英文和代码密集型任务，你支付了更多 tokens，换来了轻微但真实的指令遵循改进。
