## 技术说明

### Twitter Snowflake ID 格式

Twitter 使用 Snowflake ID 系统来生成唯一的推文、用户和消息 ID。Snowflake ID 的结构如下：

- **第 63-22 位 (41 bits)**: 时间戳 - 自 Twitter Epoch 以来的毫秒数
- **第 21-17 位 (5 bits)**: 数据中心 ID
- **第 16-12 位 (5 bits)**: 工作器 ID
- **第 11-0 位 (12 bits)**: 序列号

Twitter Epoch 设置为 2010-11-04 01:42:54 UTC (1288834974657 ms)，这个自定义 Epoch 使得 41 位时间戳可以使用约 69 年（到 2080 年左右）。
