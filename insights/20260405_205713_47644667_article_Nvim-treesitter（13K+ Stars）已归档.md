## 讨论摘要

### 核心问题

用户 **pickle-and-pork** 提问：为什么 nvim-treesitter 插件没有正式的发布版本？用户认为通常的做法是应该有稳定版本和可能的破坏性变更版本，但该插件却没有使用 tags 和 releases。

### 主要讨论内容

1. **用户反馈**（pickle-and-pork）:
   - 注意到该插件没有真正的发布版本
   - 要求用户始终保持最新的构建版本
   - 认为这有点奇怪，询问为何不使用 tags 和 releases

2. **用户反馈**（shushtain）:
   - 批评在 v0.12 发布的第一天就放弃对 v0.11 的支持
   - 认为移除兼容性检查的做法"很疯狂"
   - 提到 Arch Linux 包需要通过安全检查才能更新，导致用户被限制无法更新
   - 引用了 commit c82bf96 作为问题根源

3. **维护者回应**（clason）:
   - 该插件很早以前就正式要求 Nvim 0.12
   - 因为用户无法阅读说明，持续提交关于 Nvim 0.11 的 issue 和 PR，所以被迫将 0.11 设为硬性错误
   - 强调该插件仍然是实验性的
   - 声称有固定的路线图问题，当达到稳定状态时才会创建发布版本
   - 建议用户锁定到任意想要的 commit 并停止升级

4. **后续讨论**:
   - shushtain 反驳称插件在 Neovim 0.11 上一直可用，直到那个 commit
   - clason 最终回应"OK"后离开对话

---

## 结论

该讨论揭示了 nvim-treesitter 项目的以下特点：

1. **无正式发布版本**：该插件采用持续更新的开发模式，不提供传统的语义化版本发布
2. **实验性状态**：维护者明确表示插件仍处于实验阶段，尚未达到稳定状态
3. **强制升级政策**：用户被建议锁定特定 commit 以避免自动更新带来的兼容性问题
4. **Nvim 版本要求**：插件要求 Neovim 0.12，已移除对旧版本的支持
