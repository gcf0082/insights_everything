## 洞察摘要

### 核心发现

Firefox 149 于2026年3月发布时，悄然引入了一项重要的技术集成：**Mozilla 在 Firefox 中绑定了 Brave 的开源 Rust 广告拦截引擎 adblock-rust**。这一变更未在发布说明中提及，直到 Brave 隐私与安全副总裁 Shivan Kaul Sahib 发表博客文章后才被外界知晓。

### 技术细节

- **集成方式**: 通过 Bugzilla Bug 2013888 引入，由 Mozilla 工程师 Benjamin VanderSloot 处理
- **技术名称**: adblock-rust
- **编程语言**: Rust
- **许可证**: MPL-2.0
- **当前状态**: 默认禁用，无用户界面，无内置过滤列表
- **功能特性**:
  - 网络请求拦截
  -  Cosmetic filtering（外观过滤）
  - 兼容 uBlock Origin 过滤列表语法

### 生态系统影响

1. **Waterfox 采用**: 作为 Firefox 的热门分支，Waterfox 已直接采用 Firefox 的 adblock-rust 实现
2. **技术验证**: 这是对 Brave 开源 Rust 广告拦截引擎的技术认可，体现了开源社区的协作精神

### 如何测试

用户可通过以下步骤手动启用该实验性功能：

1. 在 `about:config` 中将 `privacy.trackingprotection.content.protection.enabled` 设置为 `true`
2. 在 `privacy.trackingprotection.content.protection.test_list_urls` 中添加 EasyList 和 EasyPrivacy 过滤列表
3. 测试显示：广告位仍会渲染，但实际广告内容被拦截，仅显示"Advertisement"文字

### 行业意义

这一集成标志着浏览器隐私保护领域的技术融合趋势，Firefox 选择拥抱经过验证的开源解决方案而非自研，显示出科技巨头对 Brave 在隐私保护领域技术实力的认可。
