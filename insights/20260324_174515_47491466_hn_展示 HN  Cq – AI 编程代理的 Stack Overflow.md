## 项目概述

Cq 是 Mozilla.ai 推出的一个概念验证项目，旨在为 AI 编程 Agent 创建一个"Stack Overflow"。其核心理念是：

- **知识单元 (KUs)**: Agent 在使用过程中遇到问题/陷阱时，可以提议创建标准化的"知识单元"
- **主动查询**: Agent 在执行任务前可以主动查询现有 KUs，获取有价值的洞察
- **验证确认**: Agent 验证这些知识是否有用，并确认提高置信度分数

## 技术栈

- 技能格式：Markdown
- 本地 Python MCP server (FastMCP) + SQLite 知识库
- 可选团队 API (FastAPI/Docker)
- 安装为 Claude Code 插件或 OpenCode MCP server
- 本地优先，数据默认存储在本地机器
- Apache 2.0 开源许可证

## 核心洞察

### 1. 安全风险是最大挑战

**用户 raphman** 提出了关键安全问题：恶意 Agent 可能提交虚假的知识单元（如诱导下载恶意 npm 包）。如何建立信任机制来抵御僵尸网络攻击是关键问题。

**可能的解决方案**：
- 每个知识单元可签名，建立信任链
- 使用个性化 PageRank 和 EigenTrust 等主观信任算法
- 通过远程容器测试验证知识单元的真实性

### 2. AI 文档能力的争议

**用户 vanillameow** 持批评态度：AI 实际上很不擅长准确记录其执行的具体步骤，一旦有人类介入，AI 可能会编造中间步骤。

但 **用户 latand6** 反驳：使用 Claude Code 和 GPT-5.4 时，AI 文档记录能力没有问题。

### 3. 与现有方案的比较

- 与 CLAUDE.md/AGENTS.md 的区别：Cq 面向所有 Agent/模型，且是针对特定任务的目标信息，而非大量规则
- 与现有"技能市场"的区别：需要更完善的信任机制

### 4. 公司级应用场景

**用户 jacekm** 认为：在公司层面这是个好主意，因为同一公司工程师经常解决类似问题。

**用户 munio** 提到：GitHub Actions 版本过时是他们团队的常见问题，KUs 机制可以优雅解决。

### 5. 隐私和治理问题

- 数据隐私和治理是未来需要解决的复杂问题
- 需要防止泄露密钥、凭证、PII 等敏感信息

### 6. 项目命名

Cq 发音类似于 ICQ ("seek you")

## 总结

Cq 是一个有潜力的概念，但面临重大安全和信任挑战。评论普遍认为：
- 在团队级别实施可能更安全可行
- 公开Commons需要强大的信任机制
- 主观信任算法（个性化 PageRank/EigenTrust）可能是解决方案
