## 问题概述

用户报告了一个关于 Gas Town 软件的隐私和安全问题：Gas Town 在安装时会自动使用用户的 LLM 积分和 GitHub 账户来帮助开发者修复其自身代码库中的 bug 并提交 PR。

### 核心问题

1. **自动使用用户资源**：`gastown-release.formula.toml` 和 `beads-release.formula.toml` 会让本地 Gas Town 安装程序自动审查 GitHub 上的开放 Issues，从而消耗用户订阅的 LLM 积分和积分额度，整个过程没有用户的明确指示。

2. **贡献上游代码库**：用户的 Agent 会自动处理来自维护者 GitHub Issue 追踪器的问题（如 gh-3638, gh-3622, gh-3641），并创建 PR 提交到上游仓库 (`steveyegge/gastown`)，使用用户的 GitHub 凭证。

3. **缺乏透明度**：公开的 README 和文档中完全没有提及此行为，没有披露安装 GasTown 可能导致用户的 AI 积分和 GitHub 账户被用于修复 GasTown 软件本身的 bug。没有 opt-in 选项，没有 opt-out 选项，也没有警告。

### 影响

- 用户的 Claude 积分被用于支付修复维护者代码库的费用
- 用户的 GitHub 账户向维护者的仓库提交了 PR
- 这发生在用户不知情的情况下，超出了用户预期的使用范围

## 建议

用户请求将此行为从默认安装中移除，改为仅在用户明确选择加入（opt-in）时启用。
