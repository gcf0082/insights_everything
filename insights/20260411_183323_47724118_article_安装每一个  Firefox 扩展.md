## 文章摘要

本文记录了作者尝试安装所有Firefox扩展程序的完整过程。作者共发现了84,194个Firefox扩展程序，总大小约49.3GB，平均每个扩展约585KB。通过多次尝试，作者最终成功在虚拟机中安装并运行了所有扩展程序。

### 3. 安全问题发现

#### 钓鱼攻击
- 发现多个使用同形文字（homoglyph）攻击的钓鱼扩展
- 最有意思的案例："Іron Wаllеt"（使用西里尔字母伪装）
- 扩展会从NocoDB电子表格获取钓鱼页面URL

#### SEO垃圾
- 发现大量"Code Injector"类垃圾扩展
- 大量2025年发布的AI生成扩展
- 作者主页链接中包含赌博网站

#### PUA（潜在有害应用）
- 发现Innover Online Group Ltd公司发布的系列垃圾扩展
- 超过70万用户受影响
- 包含Yahoo搜索联盟代码

### 4. 用户分布统计

- **34.3%** 的扩展没有日活用户
- **25.1%** 的扩展超过10个日活用户
- **10.6%** 的扩展超过100个日活用户
- **3.2%** 的扩展超过1000个日活用户
- **0.7%** 的扩展超过10000个日活用户

### 5. 其他发现

- **76.7%** 的扩展是开源的
- **23%** 的扩展是作者写这篇文章后创建的
- **2.4%** 的扩展需要付费

## 安装过程

### 尝试次数与结果

| 尝试 | 环境 | 扩展数量 | 结果 |
|------|------|----------|------|
| 0 | Windows VM | 65,335 | 内存溢出，系统冻结 |
| 1 | Windows | ~1,000 | 成功但无法继续 |
| 2 | Windows | 65,335 | Firefox冻结 |
| 3 | Mac | 65,335 | 无法运行，内存不足 |
| 4-10 | Windows | 1000-6000 | 部分可用 |
| 11 | Windows VM | 84,194 | 成功运行 |

### 最终成功配置

- **硬件**: 16GB RAM（桌面）/ 24GB统一内存（笔记本）
- **下载时间**: 约1小时43分钟
- **extensions.json大小**: 189 MB
- **内存占用**: 27-37 GB

### 关键步骤

1. 使用Mozilla公共API抓取所有扩展数据
2. 直接下载.xpi文件到Firefox配置文件夹
3. 删除addonStartup.json.lz4和修改extensions.json来启用所有扩展

## 使用体验

- **about:addons**: 页面加载耗时**6小时**
- **example.com**: **24小时后仍未加载**
- **about:telemetry**: 加载后崩溃
- **moz-extension**: 唯一成功加载的非about页面

### 结论

**完全不可用**。安装所有扩展后，浏览器基本上无法正常使用，内存占用巨大，许多页面无法加载。

## 结论

虽然技术上可以安装所有84,000个Firefox扩展程序，但实际上毫无意义。浏览器在加载如此多的扩展后基本处于不可用状态，仅about:addons页面就需要6小时才能加载完成。这个实验展示了Firefox扩展生态的庞大规模，同时也暴露了浏览器在处理大量扩展时的性能瓶颈。
