## 洞察摘要

这是一篇关于将 Rust WASM 解析器重写为 TypeScript 后性能反而提升的技术博客。文章引发了热烈讨论，核心观点如下：

### 性能提升的关键原因

1. **算法改进 > 语言选择**：从 O(N²) 到 O(N) 的流式处理优化带来了 3.3x 的性能提升，比从 WASM 切换到 JS 的 2-4x 提升更显著
2. **WASM 边界开销**：频繁跨 JS/WASM 边界调用会产生显著开销，特别是在处理流式小数据块时
3. **缓存和内存布局**：跨边界数据传递会破坏缓存局部性，增加内存分配开销

### 讨论中的其他观点

- **性能的反直觉性**：有评论者分享了从 C++ 迁移到 Python 1.4 后性能提升 10x 的经历，原因是修复了一个隐藏的缓存键计算 bug
- **语言选择的权衡**：选择 Python 可以快速开发，但长期维护性差；Rust/Go 等可兼得性能与开发效率
- **LLM 时代的影响**：有观点认为未来多数代码会用 Python/JS 编写（因 LLM 擅长），但真正需要性能的部分仍需用 C/C++/Rust 优化

## 关键结论

> 算法复杂度改进主导语言级优化。从 O(N²) 到 O(N) 的流式处理优化比从 WASM 切换到 TypeScript 有更大的实际影响。
