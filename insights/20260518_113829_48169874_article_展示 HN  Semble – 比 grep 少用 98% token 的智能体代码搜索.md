## 项目概述

Semble 是一个专为 AI 代理设计的代码搜索库，旨在提供快速、准确的代码检索能力。该项目使用比传统 grep+read 方法少约 98% 的令牌（tokens），显著降低了延迟和成本。

---

## 核心特性

| 特性 | 说明 |
|------|------|
| **高速** | 平均代码库索引仅需 ~250ms，查询响应 ~1.5ms，全部在 CPU 上运行 |
| **准确** | NDCG@10 达到 0.854，与代码专用 Transformer 模型相当，但成本更低 |
| **令牌高效** | 仅返回相关代码块，比 grep+read 节省约 98% 的令牌 |
| **零配置** | 无需 API 密钥、GPU 或外部服务，纯本地运行 |
| **MCP 服务器** | 可作为 MCP 服务器使用，支持 Claude Code、Cursor、Codex、OpenCode 等 |
| **本地与远程** | 支持本地路径或 git URL |

## 技术架构

### 核心工作原理

1. **代码分块**：使用 Chonkie 将每个文件分割成代码感知的块
2. **双重检索**：
   - 静态 Model2Vec 嵌入（使用 potion-code-16M 模型）用于语义相似度
   - BM25 用于标识符和 API 名称的词汇匹配
3. **结果融合**：使用倒数排名融合（RRF）合并两个检索结果
4. **重排序**：使用代码感知信号进行重排序

### 排序信号

- **自适应权重**：符号类查询获得更多词汇权重，自然语言查询保持平衡
- **定义提升**：定义查询符号的代码块排名更高
- **标识符词干**：查询词干与代码块中的标识符词干匹配
- **文件一致性**：同一文件的多个匹配块会获得提升
- **噪音惩罚**：测试文件、兼容层、示例代码等会被降权

## 使用方式

### MCP 方式（推荐）

```bash
# Claude Code
claude mcp add semble -s user -- uvx --from "semble[mcp]" semble

# Codex
# 在 ~/.codex/config.toml 中配置

# OpenCode
# 在 ~/.opencode/config.json 中配置

# Cursor
# 在 ~/.cursor/mcp.json 中配置
```

### Bash 集成

```bash
pip install semble  # 或 uv tool install semble
semble search "authentication flow" ./my-project
semble find-related src/auth.py 42 ./my-project
```

### Python API

```python
from semble import SembleIndex

index = SembleIndex.from_path("./my-project")
results = index.search("save model to disk", top_k=3)
```

## 性能基准

| 方法 | NDCG@10 | 索引时间 | 查询延迟 |
|------|---------|----------|----------|
| CodeRankEmbed Hybrid | 0.862 | 57s | 16ms |
| **Semble** | **0.854** | **263ms** | **1.5ms** |
| CodeRankEmbed | 0.765 | 57s | 16ms |
| ColGREP | 0.693 | 5.8s | 124ms |
| BM25 | 0.673 | 263ms | 0.02ms |

Semble 实现了 CodeRankEmbed Hybrid 性能的 99%，索引速度快 218 倍，查询速度快 11 倍。

## 令牌效率

- Semble 平均节省 **98%** 的令牌
- Semble 在仅 2k 令牌预算下达到 94% 召回率

## 应用场景

Semble 适用于以下场景：

1. **AI 代理代码搜索**：为 Claude Code、Cursor、Codex、OpenCode 等代理提供即时代码搜索能力
2. **本地开发**：快速索引和搜索本地代码库
3. **远程仓库搜索**：通过 git URL 搜索远程代码库
4. **自定义工具开发**：通过 Python API 集成到自定义工具中

## 总结

Semble 是一个高性能、高效率的代码搜索解决方案，专为 AI 代理设计。它通过创新的双检索器架构（语义 + 词汇）实现了与大型 Transformer 模型相当的检索质量，同时保持了极低的延迟和资源消耗。对于需要快速、准确代码搜索的 AI 应用，Semble 是一个值得考虑的选择。
