# 洞察报告

## 基本信息

- **链接**: https://donottrack.sh/
- **标题**: DO_NOT_TRACK - 一个尊重用户隐私的软件标准
- **日期**: 2026年5月3日

## 摘要

该网站提出了一个名为"DO_NOT_TRACK"的隐私保护标准，旨在为软件开发者提供一个统一的环境变量来尊重用户拒绝追踪的意愿。

## 核心内容

### 问题背景

许多命令行工具、SDK和框架默认收集遥测数据。每个工具都有自己的退出追踪方式：

- .NET: `DOTNET_CLI_TELEMETRY_OPTOUT=1`
- AWS SAM CLI: `SAM_CLI_TELEMETRY=0`
- Azure CLI: `AZURE_CORE_COLLECT_TELEMETRY=0`
- Go: `go telemetry off`
- Google Cloud SDK: `gcloud config set disable_usage_reporting true`
- Homebrew: `HOMEBREW_NO_ANALYTICS=1`

这些方法数量繁多且各不相同，给用户带来了极大的困扰。

### 解决方案

提出使用单一标准环境变量 `DO_NOT_TRACK=1`，让用户明确表达拒绝以下追踪的意愿：

- 广告追踪
- 使用报告（无论是否匿名）
- 遥测数据
- 崩溃报告
- 向软件创建者或第三方发出的非功能性网络请求

用户只需在shell配置文件中添加 `export DO_NOT_TRACK=1` 即可生效，支持 Bash、Zsh、Fish、PowerShell 和 Windows CMD。

### 对软件作者的建议

如果开发的工具会收集遥测、分析数据或发出非必要的网络请求，请检查该变量：

- 如果 `DO_NOT_TRACK` 设置为 `1`，则禁用所有追踪
- 在现有退出机制旁尊重该变量
- 建议将遥测改为默认关闭而非默认开启

## 结论

DO_NOT_TRACK 标准旨在简化用户隐私保护流程，通过单一环境变量让用户能够一键拒绝所有追踪行为，同时为软件开发者提供明确的实现指南。
