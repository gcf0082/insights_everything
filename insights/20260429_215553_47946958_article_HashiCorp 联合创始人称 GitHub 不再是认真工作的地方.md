## 核心内容

### 事件概述

Hashicorp 联合创始人 Mitchell Hashimoto 宣布 GitHub 不再是"认真工作的地方"，因此他将把当前项目 Ghostty 迁移到其他平台。

### 背景信息

- **GitHub 用户 ID**：1299（2008年2月加入）
- **当前项目**：Ghostty（终端模拟器）
- **使用历史**：18年来几乎每天使用 GitHub

### 主要原因

**GitHub 服务不稳定**：
- 过去一个月，Hashimoto 记录了几乎每天都有 GitHub 宕机影响其工作
- 在撰写文章当天，因 GitHub Actions 宕机，他无法进行 PR 审核约 2 小时
- 4月28日还发生了一次因 Elasticsearch 问题导致的 PR 失败事件

**个人情感**：
> "我想要在那里，但它不想让我在那里。我想要完成工作，但它不想让我完成工作。我想要发布软件，但它不想让我发布软件。"

### 迁移计划

- 正在与多个提供商（包括商业和开源）讨论
- 将逐步移除对 GitHub 的所有依赖
- 将在 GitHub 上保留 Ghostty 的只读镜像
- 个人项目仍保留在 GitHub 上

### 回归条件

如果 GitHub 能提供"真正的结果和改进，而不是言语和承诺"，他会考虑回归。

## 分析与评论

文章指出，Hashimoto 的经历表明 GitHub 作为代码协作首选平台的地位可能受到威胁。这一担忧正值微软承认 Windows 存在严重质量问题（部分原因是强行在过多工具中注入 AI）之际。GitHub 频繁出现问题的时间也与微软对 AI 的执着追求相吻合。
