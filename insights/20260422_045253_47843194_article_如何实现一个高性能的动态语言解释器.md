## 摘要

本文讲述作者如何将一个**极其简单的AST解释器**（为名为Zef的动态语言编写）优化到可与Lua、QuickJS和CPython竞争的水平。

## 关键技术

文章涵盖21个优化技巧，最终实现**16.6倍加速**（使用Yolo-C++可达67倍）：

| 优化 | 效果 |
|------|------|
| #1 直接调用操作符 | +17.5% |
| #2 直接调用RMW操作符 | +3.7% |
| #3 避免IntObject检查 | +1% |
| #4 符号系统 | +18% |
| #5 值内联 | +2.8% |
| #6 对象模型+内联缓存+监视点 | **4.55倍** |
| #7 参数优化 | +33% |
| #8-10 Getter/Setter特化 | +9% |
| #11 全局哈希表 | +15% |
| #12-13 避免optional+特化参数 | +5.5% |
| #14 值慢路径优化 | +10% |
| #16-19 sqrt/toString/数组特化 | +17% |

## 性能对比

| 实现 | 相对基线 | 相对Python | 相对Lua | 相对QuickJS |
|------|----------|------------|---------|-------------|
| Zef基线 | 1x | 35.4x更慢 | 79.6x更慢 | 22.6x更慢 |
| Zef优化后 | 16.6x | 2.1x更慢 | 4.8x更慢 | 1.35x更慢 |
| Zef+Yolo-C++ | 67x | 1.9x更快 | 1.2x更慢 | 3x更快 |

## 核心要点

1. **值表示**：使用64位标记值（NuN tagging），避免整数类型的堆分配
2. **内联缓存**：记住上次访问的类型和偏移，加速后续访问
3. **对象模型**：用Storage替代哈希表存储字段
4. **监视点**：处理子类覆盖场景
5. **无SSA/无GC/无字节码/无机器码**，但效果显著

## 结论

文章证明了即使没有JIT，在简单解释器上应用常识性优化也能实现巨大性能提升。
