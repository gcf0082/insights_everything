## 核心要点

DuckDB v1.5.2 是一个补丁版本，主要包含错误修复、性能改进以及对 DuckLake v1.0 lakehouse 格式的支持。这是 DuckDB v1.5 系列的第二个补丁版本。

## 详细洞察

### 1. DuckLake 湖仓格式

DuckDB 正式发布了 stable 和 production-ready 的 lakehouse 规范及其参考实现。

**DuckLake v1.0 主要特性：**

- 数十个错误修复
- 向后兼容性保证
- 数据内联（data inlining）
- 排序表（sorted tables）
- 桶分区（bucket partitioning）
- 删除缓冲区作为 Iceberg 兼容的 Puffin 文件

### 2. Iceberg 扩展更新

Iceberg 扩展新增了以下功能支持：

- GEOMETRY 类型
- ALTER TABLE 语句
- 分区表的更新和删除操作
- 分区 truncate 和 bucket 操作

### 3. Jepsen 测试结果

DuckDB 团队与 Jepsen 展开了合作以提高数据库健壮性。测试套件发现了一个由 INSERT INTO 语句在主键冲突解析时触发的错误，并在本版本中修复了该问题。

### 4. 在线 Shell 重大更新

WebAssembly 在线 shell (shell.duckdb.org) 经历了全面重做。新功能包括：

- 使用 `.files` 命令存储和列出文件
- 支持拖放文件上传
- 支持使用 COPY ... TO 语句创建新文件并下载结果
- 内置数据集供用户实验

### 5. 性能基准测试

在 Linux v7 内核、r8gd.8xlarge 实例（32 vCPUs、256 GiB RAM、NVMe SSD）上测试：

- TPC-H 复合分数提升约 **10%**
- 从 778,041 提升至 854,676（QphH@Score 指标）

测试系统从 Ubuntu 24.04 LTS 升级到 Ubuntu 26.04 beta。

## 技术栈亮点

- **核心版本**：v1.5.2
- **新格式支持**：DuckLake v1.0、Iceberg
- **测试工具**：Jepsen
- **WebAssembly**：在线 Shell 全面重构
- **测试环境**：Ubuntu 26.04 beta、AWS r8gd.8xlarge

## 总结

DuckDB v1.5.2 是一个重要的补丁版本，不仅修复了多个错误，还引入了对DuckLake v1.0湖仓格式的正式支持，并显著提升了性能表现（约10%的TPC-H分数提升）。与Jepsen的合作进一步增强了数据库的健壮性，而在线Shell的重大更新则为用户提供了更好的Web端使用体验。
