# 洞察报告：你可以在自己的服务器上运行 DNS 服务

**洞察链接**：https://simonsafar.com/2025/running_dns/  
**发布日期**：2025年5月3日  
**作者**：Simon Safar  

---

## 概述

本文探讨了运行自建 DNS 服务器的可行性，作者认为这并不困难，并推荐使用 PowerDNS 作为解决方案。

## 核心内容

### DNS 服务器选择

作者推荐使用 **PowerDNS**，原因如下：
- 支持从数据库查询 DNS 记录
- 可配合 PostgreSQL 后端使用

### 配置示例

作者展示了通过 PostgreSQL 数据库管理 DNS 记录的方式，示例表结构包含以下字段：
- `id`、`domain_id`、`name`、`type`、`content`、`ttl`、`prio`、`change_date`、`disabled`、`ordername`、`auth`

记录类型包括：
- **CNAME 记录**：用于 `some-service`、`webhooks`、`calendars`、`whisper` 等子域
- **A 记录**：指向 IP 地址 `100.99.98.97`

### 部署策略

作者建议采用**子域名**方式进行部署：
1. 创建正式域名的子域名（如 `your.example.com`）
2. 将该子域的 NS 记录指向自建的 DNS 服务器

**优势**：
- 主要域名（如 email）仍由域名注册商托管，保持高可用性
- 自建 DNS 可用于添加 CNAME 记录等额外服务
- 无需登录域名供应商（如 Namecheap）后台即可管理记录

## 总结

运行自建 DNS 服务器不仅可行，而且实用。通过 PowerDNS + PostgreSQL 的组合，可以轻松实现数据库驱动的 DNS 记录管理，结合子域名部署策略，可以在保持主域名稳定的同时，获得更高的灵活性。
