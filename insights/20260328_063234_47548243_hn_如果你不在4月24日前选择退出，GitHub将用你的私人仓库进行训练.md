## 内容摘要

### 核心事件

GitHub宣布将在2025年4月24日后，默认对用户的Copilot交互数据进行AI模型训练。用户需要主动选择退出（opt-out）才能避免自己的数据被用于训练。

### 官方回应

GitHub员工martinwoodward在评论中澄清：
- 仅对Free、Pro和Pro+订阅用户的Copilot交互数据进行训练
- Business和Pro订阅用户的数据不会被用于训练
- 不会训练静态私有仓库的内容，仅训练Copilot交互数据
- 不使用Copilot的用户不受影响
- 退出链接：https://github.com/settings/copilot/features

### 社区反应

1. **隐私担忧**：用户批评这是"opt-out"而非"opt-in"模式，认为侵犯了用户隐私权

2. **不信任感**：
   - 用户指出"inputs"（输入）包括代码片段、文件结构等，本质上等同于训练私有仓库代码
   - 有人质疑企业版用户是否真的安全
   - 有人提到欧盟法规下opt-out模式可能不合法

3. **建议方案**：
   - 使用GitHub企业版（但价格昂贵，起价$21.99/用户）
   - 自托管替代方案：Forgejo、Gitea

4. **讽刺评论**：
   - "Microsoft would never do this"（暗指Microsoft确实会这样做）
   - 用户建议应该让用户主动选择加入（opt-in）而不是主动退出

### 关键讨论点

- 私有仓库的"私有"定义被质疑
- 用户担心即使自己不用Copilot，团队成员使用也会导致代码被用于训练
- 有用户提到即使在GitHub网页上操作（如AI生成commit信息）也会触发Copilot功能
