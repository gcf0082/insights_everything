## 贝叶斯与频率统计学的恩怨情仇

频率学派的方法是如此主导，以至于当你学习统计学时，它并不被称为"频率学派"，它**就是**统计学。而贝叶斯学派则是这个奇怪的 niches，只有少数人似乎**非常**喜欢。它是统计学中的Haskell。就像它的编程对应物一样，这群小小的贝叶斯派确实有理由喜欢它。

在本质上，贝叶斯和频率统计学的区别在于**概率**在框架中扮演的**哲学角色**。在两个框架中，你都有**参数**（通常是决定事物行为的某些未知量）和**数据**（或**观测值**），即你测量到的事物。

一个简单的例子是掷骰子。这里的参数是面数 n（直观上，我们都知道面数越多，给定面出现的概率越低），而数据就是你掷骰子时看到的点数。让我告诉你，为了让我的例子有任何意义，你必须让这个场景更复杂一些。所以假设你在玩DnD或某种基于骰子的游戏，但你的游戏主持人正在帘子后面掷骰子。所以你不知道骰子有多少面（也许游戏主持人在骗你，也许不是），只知道它是一个骰子，以及掷出的值。频率学派在这种情况下会告诉你参数 n 是固定的（虽然未知），而数据是从均匀分布 X∼U(n) 中随机抽取的。另一方面，贝叶斯学派会说参数 n 本身是从另一个分布 P 中抽取的随机变量，有自己的不确定性，而数据会告诉你那个分布到底是什么。

我在这里暂停一下，让你深吸一口气，对屏幕大喊这完全没有意义。当然，面数是固定的，它是一个骰子！贝叶斯统计学用分布 P 量化的不是面数的随机性，而是你对其**不确定**的程度。这是关键的区别，也是贝叶斯统计学如此强大的整个原因。在频率学派方法中，不确定性往往是事后才考虑的，只是用一些样本到总体的公式后才附加上去。也许如果你觉得 fancy，你会用一些 bootstrapping 方法。而你从中得到的任何区间都是*置信区间*，它不告诉你参数在其中的可能性，而是告诉你以这种方式构造的区间有多少次会包含参数。这是一个经常混淆的点，使置信区间成为一个被严重误解的概念。另一方面，在贝叶斯统计学中，参数不是一个*点*而是一个*分布*。该分布的展开已经考虑了你对参数的不确定性，而你从中得到的*可信区间*实际上告诉你参数在其中的可能性。

在更数学的层面上，两种方法之间的区别在于贝叶斯著名定理，它告诉你条件概率如何相互关联：

**P(A|B)P(B) = P(B|A)P(A)**

就是这样！如果你把这个方程代入参数 θ 和数据 X，你会得到 **P(θ|X) = P(X|θ)P(θ)/P(X)**，这是贝叶斯推理的基石。这可能看起来没有直接用处，但确实如此。记住 X 只是一堆观测值，而 θ 是参数化你模型的东西。所以 **P(X|θ)**，即*似然*，只是对于给定的参数实现，看到你所拥有数据的可能性有多大。同时，**P(θ)**，即*先验*，是你对参数应该是什么样子的某种直觉。我会回到这一点，但它通常是你选择的东西。最后，你可以只把 P(X) 看作归一化常数，而人们在贝叶斯推理中做的主要事情之一就是想办法不计算它！目标是估计*后验*分布 P(θ|X)，它告诉你参数服从什么分布。后验分布很有用，因为

1. 它让你对模型参数化中的不确定性有一个清晰的认识，
2. 你可以用它来构建**后验预测分布** P(Y|X) = ∫P(Y|θ)P(θ|X)dθ，其中 Y 是新数据。

让我们回到掷骰子的例子，假设你观察到以下值及其频率：

| 值 | 计数 | 频率 |
|---|------|------|
| 1 | 2 | 0.250 |
| 2 | 1 | 0.125 |
| 3 | 2 | 0.250 |
| 4 | 3 | 0.375 |

如果你是频率学派，你会寻找面数的*最大似然估计*，这实质上是最大化上面介绍的项 P(X|θ)。让我们花一秒钟来了解一下：如果你的骰子有 n 面，那么 X∼U(n)，而观察到*正好*这个数据的概率是

**P(X|n) = P(X=1|n)² × P(X=2|n) × P(X=3|n)² × P(X=4|n)³ = (1/n)⁸**

这显然在 n 是可能的最小值时最大化，这里是 4（因为用3面骰子掷出4是不可能的）。到目前为止这很容易，但置信区间是另一回事，说明了"附加"的概念。一种找到它的方法是找到所有满足 P(X_max ≤ 4|n) ≥ α/2 的 n 值，其中 α 是置信水平（通常选择为5%）。对于给定的 n，这个概率等于 (4/n)⁸，得到形式为 [4,6] 的置信区间，所以我们就得到了！

现在让我们戴上贝叶斯帽子，看看能做什么。首先，我们已经看到对于 k 次观测，P(X|n) = 1/n^k（k=8），所以我们已经有了*似然*。*先验*，正如我之前提到的，是你选择的东西。你基本上要决定一个你认为参数可能服从的分布。但听我说：只要它是合理的，不必是完美的！先验的作用基本上是给你的贝叶斯模型一些初始信息，就像一个 boost。你唯一应该确保的是为你认为可能相关的任何值提供支持（所以总是选择一个相对宽的分布）。例如在这里，我将选择一个超级无信息先验：均匀分布 P(n) = 1/N，其中 n ∈ [4, N+3]，N 非常大（比如100）。然后使用贝叶斯定理，后验分布是 P(n|X) ∝ 1/n^k。符号 ∝ 意味着它在归一化常数之前成立，所以我们可以将整个分布重写为

**P(n|X) = n⁻ᵏ / Σ(m≥4) m⁻ᵏ = n⁻ᵏ / ζ(k,4)**

其中分母叫做 Hurwitz zeta 函数，一个快速收敛的系列。在这个阶段，贝叶斯统计学家会计算由分布的最大值给出的*最大后验估计*（MAP）（在 n=4 处），或者平均值 n̄ = Σ(n≥4) n^(1-k) / Σ(m≥4) m⁻ᵏ = ζ(k-1,4)/ζ(k,4) ≈ 4.26。现在可以通过查看后验分布的累积分布函数 F(N) = Σ(s=4 to N) P(n=s|X) 并找到覆盖95%概率质量的 [4, n_R] 值来获得可信区间。对于这个问题，我们只需对几个值这样做，看看它在哪里停止，得到区间 [4,5]：

| n | F(n) |
|---|------|
| 4 | 0.816 |
| 5 | 0.953 |
| 6 | 0.985 |

这个结果与频率学派方法相当一致，这里的不确定性可以解释为

- 使用了*非常*无信息的先验 P(θ) ∝ 1
- 观测数据很少

如果似然和先验都携带很少的信息，那么后验将是非常不确定的。这是一个完美的例子，说明使用不同的先验（包括一些关于问题的知识）如何提供帮助。由于 n 是一个可能接近4的整数，我将使用几何分布作为先验 n ~ 3 + Geom(q)，q = 0.5。在下面的代码中，我用 `pymc` 数值化地做这个，发现 n̄ = 4.10，可信区间为 [4,5]。虽然区间相同，但重要的是分布更接近4（参见平均值），表明我们的不确定性在缩小。

```python
import pymc as pm
import numpy as np
import arviz as az
import matplotlib.pyplot as plt

# Observations
observations = np.array([1, 1, 2, 3, 3, 4, 4, 4])
k = len(observations)
x_max = int(observations.max())

with pm.Model() as model:
    # Geometric prior on excess faces beyond x_max
    excess = pm.Geometric("excess", p=0.5) - 1  # 0, 1, 2, ...
    n = pm.Deterministic("n", excess + x_max)
    
    # Likelihood: (1/n)^k, valid by construction since n >= x_max
    pm.Potential("likelihood", -k * pm.math.log(n))
    
    # Use NUTS sampler with target_accept=0.9 for discrete variables
    trace = pm.sample(10000, tune=2000, chains=4)

posterior_n = trace.posterior["n"].values.flatten()
hdi = az.hdi(trace, var_names=["n"], hdi_prob=0.95)
print(f"Posterior mean:    {posterior_n.mean():.2f}")
print(f"95% HDI:           {hdi['n'].values}")
```

> 但你之前说先验不重要。它明显很重要！

是的，这是贝叶斯统计学的一个关键方面。由于后验直接依赖于先验，它当然有一些效果。然而，你拥有的数据越多，你的后验就越会被似然项决定。如果你选择"宽"先验（宽高斯、均匀等），情况尤其如此。原因是：你拥有的数据越多，你的似然将具有更多的结构（即局部峰值）。当与先验相乘时，先验的平坦部分将几乎不会扰动这些，它们将是后验的特征。但当你数据很少时，情况相反，你的后验更多地反映先验数据。这是贝叶斯统计学的优势之一。先验在这里补偿数据的缺乏，当有足够的数据时，它就退出了。

**简而言之：** 虽然频率学派统计学认为参数是固定的、数据是随机的，贝叶斯统计学则相反。这主要是一个解释上的差异，但它确实对框架本身有一些影响。贝叶斯统计学特别适合对采样数据固有的不确定性进行建模。

## 贝叶斯统计学的实践

如果上一节感觉有点冗长，这是正常的。通常对于像上面这样的简单例子，贝叶斯统计学感觉没有特别有用，切换框架的复杂性似乎不值得。然而，这种情况在现实中很少以原样出现。我最近遇到了一个更有趣的用例，其中先验和似然建模之间的权衡以非常有趣的方式出现。

想象你是一家零售公司，你想基于历史数据生成代表你的销售订单的合成数据。一个相当困难的方面是如何在地理上分布合成数据。最简单的方法是基于过去类似订单的频率随机选择一个位置（比如邮政编码）。现在，相似可能只是指同一类别，或在同一渠道销售（实体店、在线等）。这个问题的一种频率学派方法通常首先根据你选择的分组对历史数据进行聚类，并使用销售计数来估计每个聚类的邮政编码分布。如果你按类别归一化计数，你会得到条件概率分布 P(邮政编码|类别)，然后你可以从中抽样。

虽然这是一种完全有效的方法，但它并非没有问题。例如，它对于新类别或新邮政编码不是很健壮。同样，如果你的数据稀疏，估计的分布可能相当嘈杂。在数据科学中，这种类型的情况通常需要特定的正则化方法。在贝叶斯方法中，历史邮政编码分布控制似然（我基于 Dirichlet-Multinomial 分布），但你仍然必须提供一个先验。正如我上面提到的，先验将接管你的数据不够准确以提供强似然的地方。当然，与前面的例子不同，你不想在这里使用无信息先验，而是想利用一些领域知识。否则，你不妨使用频率学派方法。对于这个问题，一个好的先验是基于人口的分布（或任何与销售有某种关联的东西）。关键点是，不同于我们的数据，人口分布并不稀疏，所以每个邮政编码都有机会被抽样，这导致了一个更健壮的模型。当你这样做时，你得到一个充分利用数据的模型，同时通过使用先验作为某种回退来优雅地处理新区域。

## 贝叶斯统计学中的数值方法

我希望到目前为止你能看到贝叶斯统计学有多有用。但同时，你可能也注意到计算可能有点困难。我们很幸运，之前一切都可以用众所周知的数学级数来表示。一般来说，情况并非如此。幸运的是，有一些强大的数值算法可以帮助你找到后验分布，而不必做这些工作，实际上我已经在上面向你展示了一个基于 Python 包 `pymc` 的例子。

这个想法基于马尔可夫链蒙特卡洛（MCMC）方法：

- '马尔可夫链'本质上是参数空间中的随机游走，没有记忆（所以每一步只依赖于当前状态），
- '蒙特卡洛'指的是使用随机抽样来估计某些量（在本例中，是后验分布）的算法。

为了抽样后验分布，有几种 MCMC 算法（`pymc` 使用 NUTS 算法），但在这里我将聚焦于 Metropolis 算法，我之前曾用它来解决伊辛自旋模型。算法从参数空间中的某一点 θ₀ 开始。然后在每个时间步 t，算法提议一个新点 θ_{t+1}，其接受概率为 min(1, P(θ_{t+1}|X) / P(θ_t|X))。因为这个概率只依赖于后验分布的比值，它独立于归一化项 P(X)，而只依赖于似然和先验分布。这是一个巨大的优势，因为两者通常都是众所周知的且易于计算的。算法继续一段时间，直到链收敛到后验分布，观察到的数据点显示后验分布的形状。

在 `pymc` 中，这样做的意思是使用 `pm.Model()` 定义一个模型。你可以使用 `pm.Uniform`、`pm.Normal`、`pm.Binomial` 等为你的先验定义一些分布。要指定你的似然，你可以直接使用 `pm.Potential`（就像我上面做的那样）如果你有封闭式表单，否则你可以基于你的参数使用任何分布方法指定一个模型，并使用 `observed` 参数提供观测数据。最后，你可以调用 `pm.sample()` 来运行 MCMC 算法并从后验分布中获取样本。然后你可以使用 `arviz` 来分析结果并获取可信区间、后验均值等。

作为一个例子，假设你想用一些数据 (x_i, y_i) 拟合线性回归模型 y = ax + b。在贝叶斯方法中，我们首先为参数 a、b 定义先验。由于所有参数都是连续实数，宽正态分布先验是一个好选择。对于似然，我们可以关注残差 r_i = y_i - (ax_i + b)，我们通过正态分布 r_i ∼ N(0, σ²) 来建模（我们也为 σ 提供先验）。在 `pymc` 中，这可以实现如下：

```python
import pymc as pm

with pm.Model() as model:
    # Priors
    a = pm.Normal("a", mu=0, sigma=10)
    b = pm.Normal("b", mu=0, sigma=10)
    sigma = pm.HalfNormal("sigma", sigma=10)
    
    # Likelihood
    y_obs = pm.Normal("y_obs", mu=a * x + b, sigma=sigma, observed=y)
    
    # Sample from the posterior
    trace = pm.sample(1000, tune=1000, chains=4)
```

一旦链收敛，我们可以为 a、b 和 σ 提取后验分布（为了获得均值或 CI）：

```python
import arviz as az

a_mean = trace.posterior["a"].values.mean()
b_mean = trace.posterior["b"].values.mean()
sigma_mean = trace.posterior["sigma"].values.mean()
```

这种方法的一个很酷的好处是，如果你的数据有异常值，它也非常有效。在这种情况下，你可以为每个数据点添加一个干扰参数 g_i ∈ [0,1]，它在我们的高斯似然和另一个具有更大方差的高斯分布之间插值，对背景噪声进行建模。这大大增加了未知参数的数量，但作为回报，每个参数都被加权，模型可以轻松识别异常值。在 `pymc` 中，这可以这样做：

```python
with pm.Model() as model:
    # Priors
    a = pm.Normal("a", mu=0, sigma=10)
    b = pm.Normal("b", mu=0, sigma=10)
    
    # Variance of the two mixture Gaussians
    sigma = pm.HalfNormal("sigma", sigma=10)
    delta = pm.HalfNormal("delta", sigma=5)
    sigma_bgd = pm.Deterministic("sigma_bgd", sigma + delta)
    # For the background variance, ensure it's larger
    # than the inlier variance by adding a positive delta.
    # This helps breaking the symmetry between
    # the two components and allows the model to better identify outliers.
    
    # Mixture weights
    gs = pm.Beta("gs", alpha=2, beta=2, shape=len(x))
    
    # Likelihood as a weighted mixture
    mu = a * x + b
    w = pm.math.stack([gs, 1 - gs], axis=1)

        "likelihood",
        w=w,
        comp_dists=[

            pm.Normal.dist(mu=mu, sigma=sigma_bgd),
        ],
        observed=y,
    )

```

所以当 g_i 接近1时，似然由回归高斯支配，而当 g_i 更接近0时，混合物倾向于背景高斯。在下图中，你可以看到这种方法与最小二乘线性回归的比较。

作为数据科学家，你可能习惯于使用正则化线性回归（如 Lasso (L1) 或 Ridge (L2) 回归）来解决问题。在幕后，这相当于基于拉普拉斯或高斯先验找到参数的 MAP。如果你将贝叶斯定理的日志版本与回归似然一起使用，那么最大化后验分布变成最小化

**θ_MAP = min_θ (-log P(θ|X)) = min_θ Σ_i (y_i - θ_0 - θ₁x_i)²/σ² + F(θ)**

对于高斯先验 P(θ) ∼ N(0, τ)，所以 F(θ) = (1/τ²) Σ_i θ_i²，而对于拉普拉斯先验 P(θ) ∼ Laplace(0, τ)，则 F(θ) = (1/τ) Σ_i |θ_i|。所以一直以来，这两种正则化技术只是贝叶斯先验的不同选择！
