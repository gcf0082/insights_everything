## 洞察摘要

### 背景

macOS Tahoe（预计为 macOS 26）引入了菜单图标功能，但这一设计变更引发了部分用户的不满。主要问题包括：

- **可读性降低**：菜单图标使菜单难以快速浏览
- **图标选择不当**：Apple 选择的许多图标毫无意义
- **不一致性**：不同系统应用程序之间的图标风格不统一

### 解决方案

Steve Troughton-Smith 发现了一个 Terminal 命令，可以禁用菜单图标：

```bash
defaults write -g NSMenuEnableActionImages -bool NO
```

### 效果

- 执行命令后，重启应用程序即可生效
- 该命令会保留少数仍有用的图标（如窗口缩放/调整大小图标）
- 作者已运行此命令，表示对结果非常满意

### 作者观点

Stephen Hackett 认为 Apple 应该在 macOS 27 中撤销这一更改，或者为用户提供一个正式的设置选项来禁用这些图标，以满足那些觉得图标分散注意力的用户的需求。

## 结论

通过简单的终端命令，用户可以恢复 macOS Tahoe 菜单的简洁外观，移除不必要的图标，同时保留真正有用的图标。这为不喜欢菜单图标用户提供了一个实用的解决方案。
