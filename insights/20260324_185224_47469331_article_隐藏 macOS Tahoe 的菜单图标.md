## 洞察内容总结

### 背景

作者强烈不喜欢苹果公司在 macOS Tahoe 菜单中添加图标的做法，原因包括：

- **难以浏览**：图标使菜单难以快速扫描
- **图标选择不当**：苹果选择的部分图标没有意义
- **不一致性**：不同系统应用程序之间的图标不统一

### 解决方案

Steve Troughton-Smith 发现了一个终端命令，可以禁用这些菜单图标：

```bash
defaults write -g NSMenuEnableActionImages -bool NO
```

### 特点

- 该命令保留了少数需要图标的实例，如窗口缩放/调整大小
- 应用程序重新启动后生效
- 作者已使用此命令并表示满意

### 建议

作者认为苹果应该在 macOS 27 中撤销这一更改，或者提供一个 proper 设置选项，让不喜欢菜单图标的用户可以禁用它们。
