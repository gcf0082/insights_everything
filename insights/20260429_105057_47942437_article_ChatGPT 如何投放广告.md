## 核心发现

### 1. ChatGPT广告投放架构

OpenAI的广告平台由两部分组成：

- **ChatGPT端**：后端通过SSE流将结构化的`single_advertiser_ad_unit`对象注入对话响应中
- **商户端**：名为OAIQ的跟踪SDK在访客浏览器中运行，将产品浏览数据回传至OpenAI

两部分通过Fernet加密的点击令牌连接，每个广告包含四个令牌。

### 2. 广告注入机制

当用户发送消息时，后端在`chatgpt.com/backend-api/f/conversation`开启SSE响应。部分事件为模型输出，部分为广告单元。广告单元的关键字段：

- `advertiser_brand.id`：格式为`adacct_<32位十六进制>`，为商户账户的稳定标识符
- 图片和favicon托管在`bzrcdn.openai.com`，OpenAI代管广告主创意素材
- `target.open_externally: false`使链接在ChatGPT应用内webview打开，OpenAI可观测点击后行为

### 3. 上下文定位机制

研究发现同一账户在不同话题对话中收到不同广告：

| 对话主题 | 广告主 |
|---------|--------|
| 北京 trip 规划 | Grubhub - 中餐外卖 |
| 北京 tour 预订 | GetYourGuide - 长城 tour |
| 北京航班 | Axel - 航班搜索 |
| NBA季后赛 | Gametime - 门票 |
| 春季时尚趋势 | Aritzia - 服装 |
| 生产力/演示文稿 | Canva - 设计工具 |

广告投放基于对话上下文，尚未发现利用历史对话记录进行定向的证据。

### 4. 四令牌归因链

每个广告携带四个不同的Fernet加密令牌：

| 令牌 | 位置 | 用途 |
|------|------|------|
| `ads_spam_integrity_payload` | SSE数据中 | 服务器端防伪造点击完整性校验 |
| `oppref` | 点击URL + Cookie `__oppref` | 前向归因令牌，TTL 720小时(30天) |
| `olref` | 点击URL |  impression端/出站链接参考日志 |
| `ad_data_token` | SSE载荷中，base64包装 | 服务器端核对 |

**关键点**：Fernet的前9字节为公开信息（版本字节 + 8字节时间戳），因此令牌的创建时间可在无密钥情况下还原。

### 5. 商户端归因闭环

用户点击广告卡片后，URL包含：
```
https://www.grubhub.com/?utm_source=chatgptpilot&...&oppref=gAAAA<...>&olref=gAAAA<...>
```

页面加载OAIQ SDK (`bzrcdn.openai.com/sdk/oaiq.min.js`，版本0.1.3)：
- 读取URL中的`oppref`参数，写入第一方Cookie `__oppref`，TTL 720小时
- 设置探测Cookie `__oaiq_domain_probe`
- 每次`measure`调用POST至：
  ```
  POST https://bzr.openai.com/v1/sdk/events?pid=<商户>&st=oaiq-web&sv=0.1.3
  ```

## 技术要点

- 加密算法：AES-128-CBC + HMAC-SHA256完整性校验
- 追踪域名：`bzrcdn.openai.com`（资源）、`bzr.openai.com`（事件）
- 关键Cookie：`__oppref`（归因令牌）、`__oaiq_domain_probe`（域探测）

---

## 总结

本文揭示了ChatGPT广告归因的完整技术链条：从SSE流中的广告注入，到商户端SDK的事件回传，通过四个Fernet加密令牌实现跨平台点击归因。广告投放基于实时对话上下文，实现了与用户意图的动态匹配。
