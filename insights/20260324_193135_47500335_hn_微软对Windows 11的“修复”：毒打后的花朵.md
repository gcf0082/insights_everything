## 洞察摘要

这篇文章批评了微软宣布的Windows 11"修复"计划，认为这类似于"殴打后的鲜花"——先伤害用户，再道歉索取掌声。作者梳理了微软过去四年对Windows 11的种种"虐待"行为，并指出所谓的7点修复计划只涵盖了最显眼UI问题，而真正重要的隐私侵犯和用户锁定问题并未被修复。

## 核心观点

### 微软的"虐待"时间线

1. **Copilot强制整合** (2023年9月)：微软将AI助手注入Windows 11，占据任务栏位置，无法移除或移动
2. **广告注入** (2024年4月)：更新KB5036980在开始菜单、锁屏、设置、文件资源管理器中投放广告
3. **账户锁定** (2021-2025)：逐步封堵所有本地账户绕过方法，强制用户使用微软账户
4. **OneDrive劫持** (2024年)：未经同意自动启用文件夹备份，将用户文件转移至云端
5. **Windows Recall** (2024年5月)：每隔几秒截屏并存储，最初以明文存储，存在严重安全风险
6. **硬件淘汰** (2025年10月)：Windows 11的TPM 2.0等要求导致约2.4亿台PC被标记为"过时"

### 修复计划的局限性

| 修复项目 | 原始问题时间 | 计划修复时间 | 间隔 |
|----------|-------------|-------------|------|
| 任务栏位置 | 2021年10月 | 2026年3月 | 4.5年 |
| Copilot强制安装 | 2023年9月 | 2026年3月 | 2.5年 |
| 开始菜单广告 | 2024年4月 | 2026年3月 | 2年 |
| 本地账户 | 2021-2025 | **未计划** | - |
| 遥测数据 | 2021年10月 | **未计划** | - |
| OneDrive自动同步 | 2024年 | **未计划** | - |

### 关键结论

- 微软的"修复"只针对产生负面新闻的UI问题
- 真正的商业模式——数据收集、用户锁定、遥测——被刻意保留
- 用户花了139美元购买的操作系统却包含广告和强制AI功能
- 这种"先伤害再道歉"的循环是微软的惯用策略

## 社区讨论要点

1. **渐进式侵蚀**：公司总是试图一步步挑战用户容忍度的极限
2. **缺乏选择**：普通消费者几乎没有真正的替代选项
3. **监管缺失**：反垄断监管机构未能有效约束微软的行为
4. **Linux可行性**：虽然Linux在技术上可行，但对普通用户来说门槛仍然较高
5. **竞争对手**：微软Teams被广泛认为体验不佳，但因捆绑销售而占据市场份额
