## 讨论摘要

### 主要话题

这是关于 Cursor 3 发布功能的 Hacker News 讨论。Cursor 是一款 AI 增强的代码编辑器，基于 VS Code 构建。

### 用户反馈汇总

#### 1. 对新 UI 方向的担忧

多数评论者表达了对 Cursor 3 新设计方向的担忧：

- **chat 界面优先**: 用户担心新版本将聊天窗口作为主要界面，而代码本身变成次要关注点
- **"氛围编程"(vibe coding) 担忧**: 有用户认为这种设计方向会让开发者变成"只是随着感觉走"地完成工单，而不是真正理解和编写代码
- **失去代码优先哲学**: 怀念原来"开发者主导，AI 辅助"的理念

#### 2. Claude Code vs Cursor 对比

- Cursor 用户认为它仍然是更好的选择，因为：
  - 支持多种模型（多模型支持）
  - 保留了传统 IDE 功能（代码补全、文件编辑）
  - 支持远程 SSH
  - 有云平台产品
- Claude Code 的优势：
  - 更纯粹的 CLI 体验
  - 心理模型更清晰
  - 可以嵌入 IDE 使用

#### 3. 商业模型讨论

- 讨论 Cursor 如何盈利（token 消费 vs 订阅制）
- 企业版采用按席位收费 + 按量计费的模式
- 有评论认为"代理是消耗 token 的地方，也是收费更高的地方"

#### 4. 正面反馈

- Cursor 员工 (leerob) 澄清：新界面是一个独立窗口，旧界面仍然可用
- 部分用户认为 Composer 2 功能很好，速度快
- 有用户喜欢云代理功能，可以在不同设备间切换

#### 5. 产品定位困惑

- 有用户表示不清楚 Cursor 现在到底是什么：
  - 是 IDE？
  - 是代理界面？
  - 是 CLI 工具？
  - 是云平台？

### 关键评论精选

**seamossfet**:
> 我真的很希望他们保持原来的理念——让开发者主导，AI 辅助。这种设计方向更像是把聊天界面作为一等公民，而代码本身变成次要关注。

**davnicwil**:
> 我的猜测是这更多是由产品哲学驱动，而非试图在竞争激烈的市场中最大化投资回报。Cursor 有着优秀的代码优先理念，但这个市场可能太小。

**leerob (Cursor 员工)**:
> 我们仍然非常相信这一点（代码优先理念）。这就是为什么即使在新界面中，你仍然可以查看/编辑文件、做远程 SSH、跳转到定义和使用 LSP 等。

## 结论

Cursor 3 的发布引发了关于 AI 编程工具设计方向的广泛讨论。用户主要担忧新版本过于强调聊天/代理模式，可能削弱开发者对代码的直接控制。但 Cursor 方面表示传统 IDE 模式仍然保留，用户可以根据偏好选择使用。
