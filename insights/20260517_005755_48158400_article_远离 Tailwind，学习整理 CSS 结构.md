---

## 摘要

Julia Evans分享了她将网站从Tailwind迁移到原生CSS的经验。8年前她曾热情地介绍Tailwind，但如今决定转向更语义化的HTML和原生CSS。她认为Tailwind教会了她很多CSS结构化的方法，现在可以自己实现这些系统。

## 主要内容

### 1. 为什么迁移

- Tailwind现在更依赖构建系统，需要使用构建系统才能使用新版本
- 作者的很多项目中 Tailwind.min.css 文件达2.8MB（gzip后270KB）
- 作者的CSS技能比8年前大有提升
- Tailwind存在限制：无法做一些"奇怪"的CSS操作
- 项目中混合使用原生CSS和Tailwind，难以维护
- 对更语义化的HTML感到好奇

### 2. CSS结构系统

作者建立了以下CSS组织系统：

| 类别 | 说明 |
|------|------|
| Reset | 直接复制Tailwind的preflight样式（约200行） |
| Components | 每个组件有独立CSS文件，使用嵌套选择器 |
| Colors | 所有颜色都定义为CSS变量 |
| Font Sizes | 定义字体大小变量，使用Tailwind的刻度 |
| Utilities | 保留必要的工具类（如.sr-only） |
| Base | 全局基础样式，如section和链接样式 |
| Spacing | 使用子选择器管理间距 |
| Responsive | 使用CSS Grid减少媒体查询 |

### 3. 关键技术要点

- **组件化**：每个组件有唯一class，互不覆盖，使用嵌套选择器如`.zine.horizontal`
- **颜色管理**：所有颜色集中在colors.css，使用CSS变量
- **字体大小**：定义--size-xs、--size-sm等变量，避免记忆使用em还是px
- **响应式设计**：使用auto-fit自动调整列数，减少媒体查询
- **构建系统**：开发时可用CSS的@import和嵌套选择器，生产环境用esbuild打包

## 结论

作者认为Tailwind曾经是很好的工具，但CSS近年来新增了很多强大的功能（如Grid、嵌套选择器等），使得原生CSS更加易用。她强调尊重CSS作为一项技术的重要性，反对"CSS是简单的"这种偏见。
