# GCC 16 发布系列：变更、新特性和修复

**洞察链接**: https://gcc.gnu.org/gcc-16/changes.html

**洞察来源**: GNU Project

**生成时间**: 2025-05-01

---

## 概述

GCC 16 是 GNU 编译器集合的最新主要版本，带来了大量的改进、新特性和错误修复。本报告将详细总结 GCC 16 发布系列的主要变化。

## 注意事项

- 在 Solaris 上，`int8_t` 等类型现在为 `signed char`，以符合 C99 标准。但这是一个不兼容的变更。
- `-pthread` 选项不再在 Solaris 上预定义 `_REENTRANT`。
- 所谓的 "json" 格式的 `-fdiagnostics-format` 已被移除，用户应使用 SARIF 格式。

## 通用改进

### 链接时优化（LTO）

- 链接时优化现在支持使用 `-flto-toplevel-asm-heuristics` 更好地处理顶层汇编语句。

### 向量化器改进

- 投机去虚拟化现在可以处理一般间接函数调用，并支持推测多个目标。
- 向量化器在识别循环内归约的并行性方面更加灵活。
- 向量化器现在支持向量化未计数循环或无法确定迭代次数的循环。
- 向量化器现在支持使用掩码对向量长度无关循环进行对齐剥离。
- 向量化器现在支持相互对齐剥离。
- 向量化器为带有提前退出的循环生成更高效的代码，消除了向量归纳计算。

### 文档改进

- GCC 命令选项和选项索引的文档已得到纠正，包含了许多之前遗漏的选项。
- GCC 特定属性的文档已现代化，更加注重标准属性语法。
- 参数和选项规范文件的文档已移至 GCC 内部手册。

## 新语言和语言特定改进

### OpenMP

- 内存分配支持增强：对于具有 `pinned` 特性的分配器（包括 `ompx_gnu_pinned_mem_alloc`），如果 CUDA API 可用，则使用 CUDA API，提高在 Nvidia GPU 上访问此内存的性能。
- 新增了 `ompx_gnu_managed_mem_alloc` 分配器和 `ompx_gnu_managed_mem_space`（均为 GNU 扩展）。
- OpenMP 5.0：添加了对 `declare mapper` 的有限支持。
- OpenMP 5.1：添加了对 `iterator` 修饰符的初始支持。
- OpenMP 6.0：添加了 `omp_target_memset` 和 `omp_target_memset_async` API 例程。
- 对 OpenMP 5.0、5.1 或 5.2 中已弃用的指令和子句显示弃用警告。

### OpenACC

- 添加了 `acc_memcpy_device` 和 `acc_memcpy_device_async` API 例程。
- OpenACC 3.0：`wait` 指令现在接受 `if` 子句。
- OpenACC 3.4：在 Fortran 中，命名常量（`PARAMETER`）现在允许在数据子句中使用。

### Ada

- 添加了 Constructor 和 Destructor 扩展，提供了新的构造/终结机制。
- 添加了 Implicit with 功能，允许独立使用子句隐式导入库单元。
- 添加了 Structural Generic 实例化功能。
- 添加了 `Extended_Access` 方面，支持创建对数组切片的访问。
- 添加了 VAST（Ada 语义树验证器），可用于调试编译器。
- 改进了 Android 支持。

### C++

- **C++20 成为默认标准**：GCC 16 将 C++ 默认语言版本从 `-std=gnu++17` 改为 `-std=gnu++20`。如果代码依赖较旧版本的 C++ 标准，需要添加 `-std=` 到构建标志。
- **实现了多个 C++26 特性**：
  - P2996R13：反射（通过 `-std=c++26 -freflection` 启用）
  - P3394R4：反射注解
  - P3293R3：拼接基类子对象
  - P3096R12：函数参数反射
  - P3491R3：`define_static_string`、`define_static_object`、`define_static_array`
  - P1306R5：扩展语句
  - P2900R14：合约
  - P2795R5：未初始化读取的错误行为
  - P1061R10：结构化绑定可以引入包
  - P3068R5：`constexpr` 异常
  - P3533R2：`constexpr` 虚继承
  - P1494R5：部分程序正确性
- **改进了实验性 C++20 模块支持**：
  - 新增 `--compile-std-module` 命令行选项
  - 自动将 `#include` 转换为对 `<bits/stdc++.h>` 的 `import`
- **改进了约束失败诊断**，为标准库类型特征提供更详细的错误说明。

### 运行时库（libstdc++）

- 对于支持 128 位整数的目标，`std::is_integral<__int128>` 等特征始终为 true。
- 实现了 P0952R2：`std::generate_canonical` 的新规范。
- 更新了 `std::variant` ABI 以符合 C++20 及更高版本。
- `std::regex` 执行已重写为使用基于堆的栈，避免匹配较大字符串时的栈溢出。
- **C++20 支持不再是实验性的**。
- 改进了 Windows 上的 `std::chrono::current_zone()`。
- **改进了实验性 C++23 支持**：`std::mdspan`、`ranges::starts_with`、`ranges::ends_with` 等。
- **改进了实验性 C++26 支持**：`std::simd`、`std::inplace_vector`、`std::optional<T&>`、`std::copyable_function`、`std::function_ref` 等。

### Fortran

- 使用本机共享内存多线程的 Coarrays，处理 Fortran 2018 的 `TEAM` 特性。
- 改进了 Fortran 2003 的参数化派生类型支持。
- 支持 Fortran 2018 对 `IMPORT` 语句、`REDUCE` 内在函数和 `GENERIC` 语句的扩展。
- 添加了 Fortran 2023 三角函数扩展（如 `sinpi` 内在函数）。
- 添加了 `-fexternal-blas64` 选项，用于使用 64 位整数参数调用外部 BLAS 例程。

### Modula-2

- 实现了拼写提示功能。
- 实现了新的宽集合实现，伴随库模块 `M2WIDESET`，这改变了 ABI。
- 添加了二进制字典模块 `BinDict`。
- 添加了 `-fm2-pathname-root=` 选项。

### Algol 68

- GCC 现在包含一个实验性的 Algol 68 编译器 ga68。
- 它旨在实现修订报告所述的语言，包括所有 IFIP WG2.1 子委员会批准的勘误表。

## 新目标和目标特定改进

### IA-32/x86-64

- **支持 AMD Zen6 核心**：通过 `-march=znver6`。
- **支持 Intel Wildcat Lake**：通过 `-march=wildcatlake`。
- **支持 Intel Nova Lake**：通过 `-march=novalake`。
- 添加了新的 `--enable-x86-64-mfentry` 配置选项，默认启用。
- 添加了 `--enable-tls=DIALECT` 支持。
- 移除了 `-mavx10.1-256`、`-mavx10.1-512` 和 `-mevex512`。

### AMD GPU（GCN）

- 大幅降低了 OpenMP 目标区域和 OpenACC 计算区域的启动开销。
- 添加了对 AMD Instinct MI300（`gfx942`）设备的实验性支持。
- 默认构建的 multilib 包括：`gfx908`、`gfx90a`、`gfx9-generic`、`gfx9-4-generic`、`gfx10-3-generic` 和 `gfx11-generic`。

### LoongArch

- 支持位精确整数类型（`_BitInt(N)` 和 `unsigned _BitInt(N)`）。
- 添加了函数多版本（FMV）支持，使用 `target_clones` 属性。
- 添加了对 LoongArch32 架构的支持，包括 `ilp32d`、`ilp32f` 和 `ilp32s` ABI。

### S/390、System z、IBM z Systems

- 支持位精确整数类型。
- 支持浮点类型 `_Float16`。
- 添加了全局栈保护器支持，通过 `-mstack-protector-guard=global`。
- 弃用了 `-m31` 支持，将在未来版本中移除。

## 操作系统改进

### Solaris

- 支持使用 `-gsctf` 选项生成 Solaris CTF（紧凑 C 类型格式）。

### Windows

- 支持在 Windows 上使用本机 TLS（线程本地存储）。

## GCC 诊断改进

- GCC 现在可以通过 `-fdiagnostics-add-output=experimental-html` 输出 HTML 格式的诊断信息。
- GCC 的 SARIF 输出现在尊重转储目录。
- GCC 的 SARIF 输出现在捕获逻辑位置的分层。
- GCC 诊断现在可以有与之关联的有向图。
- GCC 诊断现在可以引用 XML 和 JSON 文件中的逻辑位置。
- 添加了 `GCC_DIAGNOSTICS_LOG` 环境变量支持。
- 添加了 `EXPERIMENTAL_SARIF_SOCKET` 环境变量支持。

## 插件作者的改进

- GCC 获得了一个"发布/订阅"框架，允许松耦合的发送者和接收者之间传递强类型消息。
- GCC 诊断接收器现在可以关联带有 `finalizer` 钩子的 `extension` 对象。

## 静态分析器改进

- 分析器现在可以用于简单的 C++ 示例，因为它现在处理 C++ 的命名返回值优化并具有一些异常初始支持。
- 添加了 `-fanalyzer-assume-nothrow` 选项。
- 重写了用于表示用户代码的数据结构。
- 重实现了用于模拟用户程序内存内容的数据结构。
- 分析器开始使用 GCC 的 `value_range` 机制，消除了一些误报。

## libgdiagnostics 改进

添加了 37 个新入口点：

- 5 个用于处理逻辑位置的新入口点
- 2 个用于支持命令行选项和 SARIF 回放的新入口点
- 12 个用于处理有向图的新入口点
- 17 个用于通过缓冲区构建诊断文本的新入口点

## 总结

GCC 16 是一个重要的版本更新，带来了许多令人兴奋的新特性和改进。主要亮点包括：

1. **C++20 成为默认标准**，推动了 C++ 语言的现代化
2. **大量 C++26 特性的实现**，包括反射、合约等前沿特性
3. **对更多架构的支持**，包括 AMD Zen6、Intel 新处理器、LoongArch32 等
4. **编译器诊断能力的显著增强**，包括 HTML 和 SARIF 输出
5. **静态分析器的改进**，更好地支持 C++ 代码分析

这个版本展示了 GCC 持续推动编译器技术边界，为开发者提供更强大、更现代的工具集的努力。
