**来源**: The Register
**发布日期**: 2026年5月13日
**作者**: Connor Jones
**类型**: 安全新闻

## 核心要点

一名匿名安全研究人员（绰号"Nightmare-Eclipse"或"Chaotic Eclipse"）在微软月度补丁 Tuesday 后公布了两个新的 Windows 零日漏洞。这是该研究人员今年发布的第五和第六个零日漏洞。

## 新披露的漏洞

### 1. YellowKey（BitLocker 绕过）

- **描述**: 可绕过 BitLocker 加密，将文件加载到 U 盘上后执行正确按键序列即可获得对 BitLocker 保护设备的无限制 shell 访问权限
- **影响**: 研究人员称这是"最疯狂的发现之一"。由于 BitLocker 是 Windows 设备被盗后的最后防线，此漏洞使窃贼能够访问加密文件
- **缓解措施**: 可通过设置 BitLocker PIN 和 BIOS 密码锁定来防御
- **争议**: 研究人员暗示 YellowKey 是微软植入的后门，但无法验证

### 2. GreenPlasma（权限提升）

- **描述**: 权限提升漏洞，可授予攻击者 SYSTEM 访问权限
- **状态**: 发布了部分漏洞代码而非完整概念验证（PoC）
- **当前状态**: 在默认 Windows 配置下会触发 UAC 同意提示，静默漏洞仍在开发中
- **缓解措施**: 目前无已知补丁，等待微软修复

## 历史背景

该研究人员今年已发布五个零日漏洞：

1. **BlueHammer** (CVE-2026-32201，评分 6.5) - 已于 4 月修复
2. **RedSun** - 管理员权限提升漏洞 - 未修复
3. **UnDefend** - 拒绝服务漏洞 - 未修复

据传该研究人员可能是前微软员工，其首次博客文章显示漏洞泄露始于"违反协议"导致其"无家可归"。

## 安全专家警示

- **Forescout 安全情报副总裁 Rik Ferguson**: 如果研究人员 claim 成立，被盗笔记本不再只是硬件问题，而是数据泄露事件
- **Bridewell 网络威胁情报主管 Gavin Knapp**: YellowKey 对使用 BitLocker 的组织来说是"巨大的安全问题"。这些权限提升漏洞常被攻击者在获得初始立足点后用于凭证窃取、数据 harvesting 和横向移动

## 实际威胁

根据 Huntress 的报告，研究人员此前发布的 PoC 代码已被快速用于真实攻击。Ferguson 警告这可能是"升级的报复活动"，并暗示未来还有更多 RCE 漏洞披露。该研究人员声称有"死亡开关"机制确保更多漏洞会被公开。

## 建议

1. 立即为使用 BitLocker 的设备配置 PIN 和 BIOS 密码
2. 监控是否有利用这些漏洞的概念验证代码的恶意变体
3. 关注微软即将发布的补丁更新
4. 加强端点检测和响应能力
