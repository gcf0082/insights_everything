## 核心发现

该项目成功复现了David Ng的RYS（重复Y块）方法，发现了一种无需训练、无需权重修改即可提升大语言模型推理能力的方法。

### 主要成果

1. **Qwen2.5-32B**: 复制3个特定层（7, 8, 9层）可将推理能力提升17%
2. **Devstral-24B**: 复制第12-14层可将BBH基准上的逻辑推理从0.22提升至0.76

## 技术原理

### 转换器的"神经解剖学"

Transformer模型在训练过程中会自我组织成功能电路——执行完整认知操作的多层处理单元。这些电路是不可分割的：复制单层几乎无效，但复制正确的3-4层块可以让模型进行第二次推理。

### 关键发现

- **Devstral-24B（40层）**: 推理电路位于第12-14层
- **Qwen2.5-32B（64层）**: 推理电路位于第7-9层

边界非常精确：向任一方向移动一层，改进就会消失或反转。

## 实验结果

### Devstral-Small-2-24B（第12、13、14层复制一次）

| 指标 | 基础模型 | 手术模型 | 变化 |
|------|----------|----------|------|
| bbh/causal_judgement | 0.5775 | 0.6364 | +0.0588 |
| bbh/date_understanding | 0.9440 | 0.9000 | -0.0440 |
| gsm8k_cot (strict-match) | 0.8408 | 0.8704 | +0.0296 |
| ifeval (prompt_level_strict_acc) | 0.6229 | 0.5767 | -0.0462 |
| mbpp (pass_at_1) | 0.7000 | 0.6700 | -0.0300 |

**结论**: 手术模型提升了数学推理和因果推理能力，但牺牲了指令跟随和代码生成能力。模型思考更深但遵循指令的精确度下降。

### Qwen2.5-Coder-32B（第7、8、9层复制一次）

| 探测类型 | 基础模型 | +3层 | 变化 |
|----------|----------|------|------|
| 推理（因果+逻辑+导航） | 76.5% | 94.1% | +23% |
| EQ（情商） | 92.1 | 93.6 | +1.6% |

## "模式"发现

不同的复制模式会从相同权重产生不同的认知配置：

| 模式 | 数学 | EQ | 特点 |
|------|------|-----|------|
| 双次通过 13-16 | ↑↑ | ↑ | 数学专家 |
| 三次通过 13-16 | ↑ | ↑↑ | EQ专家 |
| 交错 13,13,14,14,15,15,16 | ↑↑↑ | ↓ | 纯数学模式 |
| 四次通过 13-16 | — | ↑↑ | EQ模式，数学中立 |

## 工具套件

该项目提供以下Python工具：

- **sweep.py**: 主要扫描工具——找到最优层复制配置
- **layer_path.py**: 构建具有明确层执行路径的任意GGUF
- **gguf_surgery.py**: 低级GGUF层复制
- **math_probe.py**: 硬算术探测
- **eq_probe.py**: 情商探测
- **reasoning_probe.py**: BBH衍生的因果/逻辑/导航/数学应用题
- **compare_eval.py**: 比较lm-evaluation-harness结果
- **visualize.py**: 扫描结果的热力图可视化

## 快速开始

### 在模型中查找电路

```bash
python sweep.py \
    --model /path/to/model.gguf \
    --llama-server /path/to/llama-server \
    --tmpdir /dev/shm/rys \
    --results pass.jsonl \
    --block-sizes 3 4 5 \
    --stride 1 \
    --start-min 10 --start-max 20
```

### 应用已知电路

```bash
# 复制Devstral的第12-14层
python layer_path.py model.gguf improved.gguf \
    -p "0..14,12,13,14,15..39" -v

# 复制Qwen2.5-32B的第7-9层
python layer_path.py model.gguf improved.gguf \
    -p "0..9,7,8,9,10..63" -v
```

## 常见问题

**是否使用更多VRAM？**
是的，复制层在GGUF中是物理副本。对于24B模型上的3个额外层，预计需要额外~1.5 GiB。

**这会降低推理速度吗？**
是的，与额外层数量成比例。40层模型上的3个额外层≈慢7.5%。

**这对我的模型有效吗？**
可能有效。已在Mistral架构（Devstral）和Qwen2架构上测试。所有transformer模型都存在电路——问题在于它们在哪里、有多大。

## 总结

LLM电路发现器展示了一种无需微调或权重修改即可提升模型特定能力的创新方法。通过识别和复制transformer模型中的"推理电路"，可以在数学推理、逻辑推理和情商方面获得显著提升，同时保持相同的权重和磁盘占用。这是一种正交于微调的方法，两者可以叠加使用。
