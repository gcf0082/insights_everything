# 洞察报告：六层暗色模式

**洞察链接**: https://cssence.com/2024/six-levels-of-dark-mode/

**发布来源**: CSSence.com

**发布日期**: 2024年4月16日

**作者**: Matthias Zöchling

---

## 概述

本文探讨了实现网页暗色模式（Dark Mode）的六个技术层次，作者将其命名为"Barebone"到"Ballistic"六个级别，后来又补充了第七层（Beyond）和第八层（Beguiling）。这个分类帮助开发者理解从基础到高级的暗色模式实现方式。

---

## 核心内容总结

### 层级一：Barebone（基础层）

- 使用HTML的`<meta>`标签声明支持的配色方案
- 代码示例：`<meta name="color-scheme" content="light dark">`
- 浏览器会自动遵循用户操作系统的颜色偏好设置
- 无需编写任何CSS即可实现基础的亮/暗模式切换

### 层级二：Basic（基本层）

- 使用CSS的`color-scheme`属性
- 代码示例：`html { color-scheme: light dark; }`
- 与meta标签效果类似，但可以在CSS中声明
- 可以结合CSS系统颜色（system-color）实现简洁的设计
- 与meta标签不同，`color-scheme`可以应用到非根元素

### 层级三：Benign（温和层）

- 使用CSS的`light-dark()`颜色函数
- 代码示例：`background-color: light-dark(black, white);`
- 第一个参数用于浅色模式，第二个参数用于深色模式
- 可以使用自定义属性（CSS变量）
- 撰写时浏览器支持度不佳，但未来将被广泛支持

### 层级四：Bold（大胆层）

- 使用`prefers-color-scheme`媒体查询
- 代码示例：
  ```css
  @media (prefers-color-scheme: dark) {
    html { background-color: black; color: white; }
  }
  ```
- 提供最大的自定义灵活性，不仅限于颜色调整
- 可以在暗色模式下使用滤镜使图像 desatuate，或将阴影改为边框

### 层级五：Bisectional（二分层）

- 在HTML中使用媒体查询属性的`<link>`标签
- 代码示例：
  ```html
  <link media="screen and (prefers-color-scheme:light)" rel="stylesheet" href="light.css">
  <link media="screen and (prefers-color-scheme:dark)" rel="stylesheet" href="dark.css">
  ```
- 适用于大量自定义样式的场景
- 浏览器会忽略不匹配的样式文件，减少下载量
- 可用于隐藏打印机的暗色样式

### 层级六：Ballistic（动态层）

- 使用JavaScript的`matchMedia`函数
- 代码示例：`const isDarkScheme = window.matchMedia('(prefers-color-scheme:dark)');`
- 可以动态检测和响应颜色模式变化
- 可用于需要JavaScript处理的场景

### 层级七：Beyond（超越层）

- 实现颜色模式切换器（Color Scheme Switcher）
- 不再仅仅依赖用户系统偏好，允许用户手动选择
- 三种模式：自动（跟随系统）、浅色、深色
- 需要在自动模式基础上构建切换器

### 层级八：Beguiling（迷惑层）

- 使用CSS的`:has()`选择器来实现模式检测
- 无需添加`.dark`类或`data-theme`属性
- 代码示例：
  ```css
  html:has(meta[name="color-scheme"][content="dark"]) {
    --color-bg: black;
    --color-text: white;
  }
  ```
- 这是一种更现代的方法，可以直接检测meta标签

---

## 实用建议

1. **混合使用**：实际项目中，通常需要混合使用多个层级的技术，而非局限于单一方法

2. **可访问性注意**：在实现暗色模式时，需确保文字对比度足够，特别是在旧版Safari浏览器中

3. **打印样式**：`prefers-color-scheme`在打印预览中始终报告为`light`，因此暗色模式不会影响打印输出

---

## 结论

本文为开发者提供了一个全面的暗色模式实现指南，从最基础的meta标签声明到高级的JavaScript和`:has()`选择器技术。随着CSS功能的不断增强（如`light-dark()`和`:has()`），实现暗色模式变得越来越简单和灵活。开发者应根据项目需求选择合适的实现层级。
