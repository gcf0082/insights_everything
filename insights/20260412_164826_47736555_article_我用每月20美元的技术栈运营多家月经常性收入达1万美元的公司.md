## 一、核心观点

Steve Hanov是一位 bootstrapped（自力更生型）创业者，运营着多家月经常性收入(MRR)达10,000美元的公司，但每月技术成本仅20美元。他的核心观点是：

- **保持极低成本 = 获得与融资相同的生存期**：用接近零的成本，达到与获得百万美元融资、燃烧率很高的情况下相同的生存时间
- **架构简单，压力更小**：没有复杂的基础设施，有更多时间寻找产品-市场契合度
- **没有投资者压力**：不需要面对董事会的压力

已建成的产品包括：websequencediagrams.com、eh-trade.ca、rhymebrain.com、rapt.ink、zwibbler.com等。

## 二、技术栈方案

### 2.1 使用精简服务器（Lean Server）

**问题**：2026年启动Web应用的常见方式是启动AWS、配置EKS集群、RDS实例、NAT网关等措施，这在你看到landing page之前就可能花费300美元/月。

**解决方案**：租用单独的虚拟专用服务器(VPS)

- **推荐服务商**：Linode或DigitalOcean
- **成本**：每月5-10美元
- **配置**：1GB RAM足够（可通过swapfile扩展）
- **优势**：只有一台服务器时，你知道日志在哪里、为什么会崩溃、如何重启

> 目标是服务请求，而不是维护基础设施。

### 2.2 使用精简语言（Lean Language）

在1GB内存限制下，使用Python或Ruby会消耗大量RAM来启动解释器和管理gunicorn workers。

**解决方案**：使用Go语言

- **优势**：
  - 性能极高，专为Web任务设计
  - 静态类型
  - 易于LLM推理
  - 编译为单个静态链接二进制文件
  - 无依赖地狱，无虚拟环境
  - 直接scp到5美元服务器即可运行

```go
package main

import (
    "fmt"
    "net/http"
)

func main() {
    http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprintf(w, "Hello, your MRR is safe here.")
    })
    
    http.ListenAndServe(":8080", nil) 
}
```

### 2.3 使用本地AI处理长时间任务

如果有一张显卡，就可以获得"无限"的AI积分。

**场景**：构建eh-trade.ca时，需要对数千家公司进行深入的股票市场研究，总结大量季度报告。

**方案选择**：

1. **Ollama（起步）**：
   - 一条命令安装 (`ollam a run qwen3:32b`)
   - 可以即时尝试数十种模型
   - 非常适合迭代提示词

2. **VLLM（生产环境）**：
   - Ollama在高并发请求时会成为瓶颈
   - VLLM使用PagedAttention，速度更快
   - 一次发送8或16个异步请求，VLLM会将它们批处理在一起

3. **Transformer Lab（高级任务）**：
   - 需要模型预训练或微调时使用

**构建的工具**：
- **laconic**：针对8K上下文窗口优化的智能代理，像操作系统管理虚拟内存一样管理LLM上下文
- **llmhub**：将任何LLM抽象为简单的provider/endpoint/apikey组合

**成本**：一块900美元的RTX 3090显卡（24GB VRAM），一次前期投资，后续无API费用。

### 2.4 使用OpenRouter处理快速/智能LLM

不是所有任务都能在本地完成，有时需要Claude 3.5 Sonnet或GPT-4o等顶级模型进行用户交互。

**方案**：使用OpenRouter

- **优势**：
  - 只需编写一个OpenAI兼容的集成代码
  - 立即可访问每一个主流前沿模型
  - **自动故障转移路由**：如果Anthropic的API在周二下午宕机，应用会自动切换到等效的OpenAI模型，用户永远不会看到错误屏幕

### 2.5 使用Copilot而非昂贵的AI IDE

开发者每周都在花费数百美元订阅Cursor和Anthropic API。

**Steve的方案**：GitHub Copilot + VS Code

- 2023年购买的订阅，一直使用
- **关键技巧**：微软按请求收费，而非按token收费
- **一个"请求"** = 在聊天框中输入的内容
- 即使智能体花费30分钟扫描整个代码库、映射依赖、修改数百个文件，仍只支付约0.04美元

**最佳策略**：
1. 编写详细提示词，包含严格的成功标准
2. 告诉智能体"继续直到所有错误修复"
3. 出去喝杯咖啡

### 2.6 使用SQLite处理一切

**传统思维**：需要一个独立进程数据库服务器。

**实际情况**：本地SQLite文件通过C接口或内存通信，比TCP网络跳转到远程Postgres服务器快几个数量级。

**并发问题**：许多人认为SQLite会在每次写入时锁定整个数据库。

**解决方案**：启用WAL模式

```sql
PRAGMA journal_mode=WAL;
PRAGMA synchronous=NORMAL;
```

开启后：
- 读者不再阻塞写作者
- 写作者不再阻塞读者
- 轻松处理数千并发用户

**构建的工具**：[smhanov/auth](https://github.com/smhanov/auth)
- 管理用户注册、会话、密码重置
- 支持Google、Facebook、X登录
- 支持公司特定SAML提供商
- 无臃肿依赖，只有简单、可审计的代码

## 三、成本对比

| 项目 | 传统方式 | Steve方式 |
|------|----------|-----------|
| 服务器 | $300+/月（AWS EKS + RDS + NAT） | $5-10/月（VPS） |
| 编程语言 | Python/Ruby（需虚拟环境、依赖管理） | Go（静态编译） |
| AI处理 | OpenAI/Anthropic API（按token收费） | 本地GPU（一次投资） |
| AI IDE | Cursor $100+/月 | GitHub Copilot $10/月 |
| 数据库 | Postgres云服务 | SQLite（免费） |
| **总计** | **$400+/月** | **$20/月** |

## 四、结论

技术行业希望你相信：建立真正的业务需要复杂的编排、巨额AWS账单和数百万美元的风险投资。

**Steve的答案是否定的。**

通过以下方式，你可以创业一家高度可扩展的公司，成本低于几杯咖啡的价格：

- 单个VPS
- 静态编译的二进制文件
- 本地GPU硬件处理批量AI任务
- SQLite的原速度

这为你的项目提供了无限的生存期，让你有时间真正解决用户的问题，而不是为燃烧率担忧。
