**洞察链接**：https://jpmens.net/2026/04/03/ssh-certificates-the-better-ssh-experience/

**发布日期**：2026年4月3日

**作者**：Jan-Piet Mens

---

## 摘要

本文详细介绍了SSH证书（SSH Certificates）如何解决传统SSH密钥认证的诸多痛点，包括首次使用时信任（TOFU）问题、主机密钥轮换警告、以及需要在每台服务器上部署公钥的繁琐操作。SSH证书由OpenSSH自5.4版本（2010年3月）起支持，其设计简洁，仅需一对SSH密钥配合`ssh-keygen`工具即可完成配置。

---

## 传统SSH密钥认证的缺点

1. **公钥部署繁琐**：每台服务器每个用户都需要在`authorized_keys`文件中添加公钥
2. **TOFU问题**：首次连接时用户通常盲目输入"yes"确认主机指纹
3. **主机密钥变更警告**：服务器重装或密钥轮换时，客户端会显示吓人的警告信息

---

## SSH证书的优势

- **无需部署公钥**：用户无需使用`ssh-copy-id`或手动编辑`authorized_keys`
- **安全控制**：可防止未经授权的密钥被添加到`authorized_keys`
- **主机密钥可轮换**：服务器主机密钥变更时客户端不会再显示警告
- **消除TOFU**：用户无需确认服务器指纹，默认信任由CA保证
- **主体名称（Principals）**：用户证书可指定允许登录的用户名
- **强制命令**：可限制用户只能执行特定命令
- **IP限制**：可限制连接来源IP段（如仅允许从特定IP以root用户登录）
- **时效性**：证书可设置有效期（如30分钟、24天、34周），到期自动失效
- **统一配置**：客户端只需一行配置即可信任所有由CA签名的服务器

## 实施步骤

### 1. 创建SSH CA

在CA机器上生成CA密钥对（使用ECDSA算法）：

```bash
umask 077; mkdir CA
ssh-keygen -t ecdsa -C "JP's SSH CA" -f CA/ssh-ca
chmod -w CA/ssh-ca*
```

### 2. 签发用户证书

使用CA私钥签署用户公钥，指定身份、主体、序列号和有效期：

```bash
ssh-keygen -s CA/ssh-ca -I "Jane Jolie" -n jane -z 001 -V +1w jane.pub
```

### 3. 配置服务器

将CA公钥复制到目标服务器并配置`sshd_config`：

```bash
install -m444 /tmp/ssh-ca.pub /etc/ssh/ssh-ca.pub
echo "TrustedUserCAKeys /etc/ssh/ssh-ca.pub" >> /etc/ssh/sshd_config
```

### 4. 签发主机证书

签署服务器主机密钥（使用`-h`选项）：

```bash
ssh-keygen -h -s CA/ssh-ca -V +52w -I deadbeef01 -z 1000 -n alice.example.com ssh_host_ed25519_key.pub
```

### 5. 配置主机证书

在服务器上安装主机证书并配置`HostCertificate`：

```bash
install -m444 tmpfile /etc/ssh/ssh_host_ed25519_key-cert.pub
echo "HostCertificate /etc/ssh/ssh_host_ed25519_key-cert.pub" >> /etc/ssh/sshd_config
systemctl restart sshd
```

### 6. 配置客户端

在客户端的`known_hosts`中添加CA公钥信任行：

```bash
echo "@cert-authority *.example.com $(cat CA/ssh-ca.pub)" > ~/.ssh/known_hosts
```

## 高级功能

### 强制命令

签发证书时可指定强制执行的命令：

```bash
ssh-keygen -s CA/ssh-ca -I "Jane Jolie" -n jane -z 2 -V +1w -O force-command=/usr/bin/date jane.pub
```

### IP地址限制

可限制证书仅能从特定网段使用：

```bash
ssh-keygen -s CA/ssh-ca -I "Jane Jolie" -n jane -z 3 -V +1w -O force-command=/usr/bin/date -O source-address=192.0.2.0/24 jane.pub
```

### 扩展权限控制

可精确控制是否允许X11转发、-agent转发、端口转发、PTY分配等：

```bash
ssh-keygen -U -s CA/ssh-ca -I "Jane Jolie" -n jane,root -z 4 -V +1w -O clear -O extension:permit-agent-forwarding -O extension:permit-port-forwarding jane.pub
```

## 故障排查清单

**服务器端**：
- CA公钥必须在服务器上且sshd可读，配置在`TrustedUserCAKeys`
- 主机密钥必须已签名并与主机密钥放在一起
- `sshd_config`需要`HostCertificate`指向主机证书
- 服务需要重启

**客户端**：
- 用户SSH密钥必须由CA签署并与密钥文件放在一起
- `known_hosts`需要正确的`@cert-authority`配置
- 连接时使用的主机名/地址必须与主机证书中的主体名匹配

**常见错误**：
- 提示确认主机指纹：证书可能已过期
- 提示输入密码：证书中未配置该用户的主体名

## 总结

SSH证书提供了更优的SSH体验：
- 消除TOFU，客户端和服务器默认互信
- 可签发短时效证书，到期自动失效
- 无需在服务器上清理公钥
- 支持强制命令、PTY控制和来源IP限制
