## 核心内容摘要

### 政策变更要点

从 **2026年4月24日** 起，GitHub Copilot 将更新其数据使用政策：

- **适用范围**：Copilot Free、Pro 和 Pro+ 用户的交互数据（输入内容、输出内容、代码片段及相关上下文）将被用于训练和改进 GitHub 的 AI 模型，除非用户选择退出。
- **不受影响群体**：Copilot Business 和 Copilot Enterprise 用户不受此更新影响。
- **退出方式**：用户可在 GitHub 设置中的 "Privacy"（隐私）选项下选择退出。
- **历史偏好保留**：如果用户之前已选择退出数据收集设置，其偏好将被保留。

### 参与计划将收集的数据类型

用户参与后，可能收集和使用的交互数据包括：

- 用户接受或修改的输出内容
- 发送给 GitHub Copilot 的输入内容（包括展示给模型的代码片段）
- 光标位置周围的代码上下文
- 用户编写的注释和文档
- 文件名、仓库结构和导航模式
- 与 Copilot 功能的交互（聊天、内联建议等）
- 用户对建议的反馈（点赞/点踩评分）

### 不会收集的数据

- Copilot Business、Copilot Enterprise 或企业所属仓库的交互数据
- 在 Copilot 设置中选择退出模型训练的用户数据
- 静态状态下的 Issues、讨论或私有仓库内容（"静态"指未主动使用 Copilot 时）

### 数据共享说明

该计划中使用的数据可能会与 GitHub 关联公司（包括 Microsoft）共享，但不会与第三方 AI 模型提供商或其他独立服务提供商共享。

### 政策背景

GitHub 初期模型使用公开数据和手工编写的代码样本构建。过去一年已开始使用 Microsoft 员工的交互数据，模型在多种编程语言中的接受率都有显著提升。

---

## 洞察结论

此次政策更新是 GitHub 改进 AI 辅助开发能力的重要举措。通过利用真实世界的开发者交互数据训练模型，可以：
- 更好地理解开发工作流程
- 提供更准确和安全的代码模式建议
- 提高在生产前发现潜在 bug 的能力

用户可根据个人隐私偏好选择是否参与该计划，选择不参与的用户仍可正常使用 Copilot 的所有 AI 功能。
