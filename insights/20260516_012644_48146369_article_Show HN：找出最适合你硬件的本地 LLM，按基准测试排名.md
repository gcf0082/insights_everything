## 项目简介

**whichllm** 是一个命令行工具，旨在帮助用户找到在其硬件上真正能够运行且性能最佳的本地大语言模型（LLM）。该项目通过真实的、基于时间因素的基准测试对模型进行排名，而非简单地根据参数数量来选择模型。用户只需执行一条命令，即可快速了解最适合其硬件的模型。

## 核心功能

### 1. 自动硬件检测
支持自动检测 NVIDIA GPU、AMD GPU、Apple Silicon 和 CPU 仅运行模式。

### 2. 智能模型排名
根据 VRAM 适配性、推理速度和基准测试质量对模型进行综合评分和排名。

### 3. 一键运行模型
通过 `whichllm run` 命令直接下载模型并启动交互式聊天会话，无需手动安装。

### 4. 代码片段生成
通过 `whichllm snippet` 命令生成可立即运行的 Python 代码片段。

### 5. 实时数据更新
直接从 HuggingFace API 获取模型数据，并带有缓存机制以确保性能。

### 6. 基准感知评分
整合多个真实基准测试（LiveBench、Artificial Analysis、Aider、Chatbot Arena ELO 等），并根据置信度进行评分衰减。

### 7. 任务配置筛选
支持按通用、编码、视觉或数学等用例筛选模型。

### 8. GPU 模拟测试
支持通过 `--gpu` 参数模拟任何 GPU 配置，方便在购买前进行测试。

### 9. 硬件规划功能
支持反向查询：`whichllm plan "llama 3 70b"` 可查询运行特定模型所需的 GPU 配置。

### 10. JSON 输出
支持 `--json` 参数，便于在管道和脚本中集成使用。

## 技术特点

### 评分系统
每个模型获得 0-100 的评分，综合考虑以下因素：

- **基准测试质量**：核心因素，合并多个基准来源的评分
- **模型规模**：最高 35 分，使用 log2 缩放
- **量化折扣**：低比特量化会获得乘法折扣
- **证据置信度**：根据证据类型（direct/variant/base/interpolated/self-reported）给予 0.55-1.0 的乘数
- **运行时适配性**：部分卸载 0.72，CPU 仅运行 0.50
- **速度评分**：-8 到 +8 分
- **来源可信度**：官方组织加分，已知重打包者扣分
- **流行度**：作为平局破局者

### 数据管道
1. 从 HuggingFace API 获取热门模型
2. 获取多个基准测试数据（LiveBench、Artificial Analysis、Aider 等）
3. 应用置信度衰减和继承验证
4. 缓存结果（模型数据 6 小时 TTL，基准数据 24 小时 TTL）

## 使用示例

```bash
# 自动检测硬件并显示最佳模型
whichllm

# 模拟特定 GPU
whichllm --gpu "RTX 4090"

# CPU 仅模式
whichllm --cpu-only

# 运行最佳模型
whichllm run

# 生成代码片段
whichllm snippet "qwen 7b"

# 硬件规划查询
whichllm plan "llama 3 70b"

# JSON 输出
whichllm --json
```

## 总结

whichllm 是一个功能强大且实用的本地 LLM 选择工具，它通过真实基准测试而非参数数量来帮助用户找到最适合其硬件的模型。其智能评分系统考虑了量化、速度、证据置信度等多个因素，能够为用户提供可靠的模型推荐建议。支持多种安装方式和完善的 CLI 接口，使其成为本地运行 LLM 的理想选择工具。
