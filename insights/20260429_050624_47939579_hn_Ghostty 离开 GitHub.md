## 核心内容

Ghostty 终端应用的开发者 Mitchell Hashimoto 在博客中宣布将 Ghostty 项目从 GitHub 迁移离开。这是一篇带有强烈情感的长文，作者表示在写这篇博文时甚至落泪，认为自己对 GitHub 有着"不健康的"情感依赖。

**核心原因：**
- GitHub 目前频繁宕机，已经影响到日常开发工作
- 作者表示："我想工作，但 GitHub 不让我工作；我想发布软件，但 GitHub 不让我发布"
- GitHub 已经不再是曾经的样子

## 热门评论洞察

### 1. 对 GitHub 现状的普遍共鸣

多位长期用户（ID 22723、1371、34967 等早期用户）表达了对 GitHub 现状的失望。一位用户（idan）指出： GitHub 在过去几年经历了根本性范式转变（代理式编码）+ 多个增长爆发期，导致问题频发。

### 2. Enshittification（平台腐化）讨论

许多评论提到了 "enshittification"（平台质量下降）现象：
- "Enshittification 已成为公司的获胜策略"
- "GitHub 是天然垄断，用户无处可去"
- 除非像 Mitchell 这样有名气的人，否则普通人很难离开

### 3. 技术问题分析

用户指出 GitHub 的具体技术问题：
- Issues 和 PRs 标签页搜索使用了 Elasticsearch 索引，存在数据延迟和丢失问题
- 近期宕机后承认 Elasticsearch 索引丢失了数据
- 后端服务响应缓慢

### 4. 替代方案讨论

评论中提到的替代平台：
- Codeberg
- 自托管方案
- 有用户提到 GitHero.app（一个非官方的更快的前端）

### 5. 对微软收购的影响

多位用户认为 GitHub 的问题始于微软收购后的迁移（从 AWS 到 Azure），导致服务可靠性下降。

## 结论

这一事件反映了开源社区对 GitHub 平台服务质量持续下降的广泛关注。Mitchell 作为知名开发者离开 GitHub 的决定引发了关于"平台依赖"和"工具民主化"的讨论。许多用户认为这是 enshittification（平台腐化）的又一个例证，GitHub 作为天然垄断，用户难以轻易迁移。此事件可能推动更多开发者考虑自托管或其他替代方案。
