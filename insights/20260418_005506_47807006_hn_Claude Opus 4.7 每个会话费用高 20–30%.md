# 洞察报告：Claude Opus 4.7 费用上涨 20-30%

## 基本信息

- **来源**: Hacker News
- **原文链接**: https://news.ycombinator.com/item?id=47807006
- **原始链接**: [Claude Opus 4.7 costs 20–30% more per session](https://www.claudecodecamp.com/p/i-measured-claude-4-7-s-new-tokenizer-here-s-what-it-costs-you)
- **发布时间**: 2026年4月18日
- **讨论热度**: 117 points, 51 comments

---

## 内容摘要

### 核心发现

Claude Opus 4.7 采用的新分词器导致每个会话的 token 消耗增加了 20-30%。这一变化主要源于分词效率的降低，而非模型推理成本的实际增加。

### 主要讨论观点

**1. 性能与成本的关系**
- LLM 存在于对数性能/成本前沿上，Opus 4.5+ 是代表该曲线上性能更高但成本快速增加的位置
- 增量模型质量可能触及边际收益递减点
- 大多数用户（99%）在日常工作中无法感知到显著差异

**2. 定价策略争议**
- Anthropic 快速提价可能反映出其近期领先地位是以更高的运营成本为代价
- 另一个解释是 Anthropic 不像 OpenAI 那样愿意燃烧大量资金
- 这被认为是"后门提价"策略，就像冰淇淋公司缩小包装而非直接涨价

**3. 用户体验反馈**
- 部分用户报告会话 token 使用增加了 20-30%
- 有用户反映 Opus 4.7 更加冗长，系统提示可能被修改以移除"简洁回复"的指示
- 一些用户认为产出质量没有相应提升，看不到投资回报率
- 也有用户表示 xhigh 模式下实际使用更少 token，运行速度更快

**4. 分词器效率问题**
- Claude 的分词器效率实际上在逐年下降（这已经是第三次迭代）
- 非英语语言的 token 成本更高，对于复杂文字如泰米尔语或日语，成本可能高出数倍

**5. IPO 相关考量**
- Anthropic 正在接近 IPO，用户基础不断增长
- 无法在招股说明书中解释大量亏损
- 这可能推动他们提高每位用户的收入

**6. 模型版本选择**
- Opus 4.6 和 Sonnet 4.6 仍然是有效选项
- Opus 4.5 至今仍可用（已发布约 6 个月）
- 旧版本的支持期限问题引发讨论

## 关键数据点

| 指标 | 数值/观察 |
|------|-----------|
| Token 消耗增加 | 20-30% |
| 计划内缓存输入 | 免费（实际费用增加约 5-34%） |
| 新增努力级别模式 | 5 种 |
| 1M token 限制消耗速度 | 大幅增加 |

---

## 结论

社区对 Opus 4.7 的定价策略普遍持批评态度，主要原因是：
1. 成本增加与性能提升之间的对应关系不明确
2. 缺乏透明的改进数据支持提价
3. 分词器效率下降而非推理优化的技术问题

用户建议在决定是否升级前，评估 4.7 相比 4.6 的实际能力提升。
