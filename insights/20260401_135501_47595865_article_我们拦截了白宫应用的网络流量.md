## 执行摘要

本报告是针对白宫iOS应用的动态行为分析。研究者通过设置MITM代理，拦截并解密了该应用的所有HTTPS流量，旨在验证静态代码分析中发现的潜在行为是否真实发生。分析结果表明，该应用在用户不知情的情况下，向多个第三方服务商发送大量用户数据和设备信息，与应用隐私标签声明严重不符。

---

## 关键发现

### 1. 第三方流量占比过高

在单次浏览会话（覆盖所有五个标签页：Home、News、Live、Social、Explore）中，应用共发出**206次请求**：

- **48次（23%）** 请求发送至 whitehouse.gov（WordPress API）
- **158次（77%）** 请求发送至第三方服务

第三方服务商包括：Elfsight、OneSignal、YouTube、Google DoubleClick、Facebook、Twitter/X

### 2. 应用共计联系31个不同的域名

| 域名 | 请求次数 | 用途 |
|------|----------|------|
| www.whitehouse.gov | 48 | WordPress API（新闻、主页、优先事项图库、直播） |
| www.youtube.com | 25 | YouTube嵌入视频 |
| phosphor.utils.elfsightcdn.com | 19 | Elfsight工具脚本 |
| static.elfsight.com | 12 | Elfsight静态资源 |
| storage.elfsight.com | 10 | Elfsight文件存储 |
| api.onesignal.com | 9 | OneSignal分析与用户画像 |
| i.ytimg.com | 9 | YouTube视频缩略图 |
| rr6—.googlevideo.com | 9 | Google视频CDN |
| scontent-lax7-1.xx.fbcdn.net | 7 | Facebook CDN（图片） |
| pbs.twimg.com | 7 | Twitter/X图片 |
| apis.google.com | 7 | Google APIs |

### 3. OneSignal收集的用户数据

通过拦截OneSignal的实际请求body，应用向`api.onesignal.com`发送的数据包括：

```json
{
    "properties": {
        "language": "en",
        "timezone_id": "America/[地区已删除]",
        "country": "US",
        "first_active": 1774908688,
        "last_active": 1774909124,
        "ip": "[IP地址已删除]"
    },
    "identity": {
        "onesignal_id": "[标识符已删除]"
    },
    "subscriptions": [
        {
            "id": "[订阅ID已删除]",
            "session_time": 61,
            "session_count": 3,
            "sdk": "050500",
            "device_model": "iPhone[型号已删除]",
            "device_os": "[系统版本已删除]",
            "rooted": false,
            "app_version": "47.0.4",
            "net_type": 1,
            "carrier": ""
        }
    ]
}
```

**单次应用启动，OneSignal即可收集：**
- 语言和时区（定位用户所在区域）
- 国家
- IP地址（IPv4或IPv6）
- 首次打开应用的时间和最后活跃时间（精确时间戳）
- 设备型号和操作系统版本（设备指纹）
- 网络类型（WiFi或蜂窝）
- 运营商信息
- 设备是否越狱
- 应用打开次数
- 每次会话时长（秒）
- 持久唯一标识符（跨会话追踪）

**关键发现：** 应用在每次启动时先执行GET请求获取用户现有档案，然后用PATCH请求更新。GET返回的档案包含先前会话的IPv6地址，后续PATCH则更新为当前的IPv4地址。这表明OneSignal维护的持久档案会追踪用户IP地址的变化——每次从不同网络打开应用，新的IP地址都会被记录到同一标识符下。

单次启动发送18个PATCH请求，完整浏览会话共9个OneSignal请求。

### 4. 13个Elfsight控制的域名

动态流量分析确认了静态分析中发现的六款Elfsight小部件和两阶段JavaScript加载器。打开Social标签页时，应用联系以下Elfsight控制的域名：

1. elfsightcdn.com（platform.js CDN）
2. core.service.elfsight.com（引导API，返回待注入脚本）
3. static.elfsight.com（静态资源）
4. storage.elfsight.com（文件存储）
5. phosphor.utils.elfsightcdn.com（工具脚本）
6. universe-static.elfsightcdn.com（静态CDN）
7. widget-data.service.elfsight.com（小部件数据服务）
8. video-proxy.wu.elfsightcompute.com（视频代理）
9. cors-proxy.utils.elfsightcdn.com（CORS代理）
10. apps.elfsight.com（应用服务）
11. dash.elfsight.com（仪表板）
12. service-reviews-ultimate.elfsight.com（评论服务）
13. elfsight.com域名级Cookie

**两阶段加载器确认：** `/p/boot/`请求确认了每个小部件ID被发送至`core.service.elfsight.com`，服务器响应包含小部件配置和待注入的JavaScript文件URL数组。

单次会话中，Elfsight基础架构共设置**10+个追踪Cookie**，包括：
- elfsight_viewed_recently
- Cloudflare追踪Cookie（`_cfuvid`、`__cf_bm`）
- 会话标识符

### 5. Google DoubleClick广告追踪

YouTube嵌入内容加载了Google的广告追踪基础架构：
- `googleads.g.doubleclick.net`（Google广告）
- `static.doubleclick.net`（DoubleClick广告脚本）

DoubleClick是Google的广告投放和追踪平台。其存在意味着Google的广告基础架构正在官方白宫应用内运行，追踪用户与视频内容的互动。应用隐私清单中未披露此行为。

### 6. 隐私标签与实际情况严重不符

**隐私清单声明：**
```
NSPrivacyCollectedDataTypes: []
NSPrivacyTracking: false
```

**实际行为：**
- 将设备型号、操作系统、IP地址、时区、语言、会话次数、会话时长和持久唯一标识符发送给OneSignal（第三方分析公司）
- 联系13个Elfsight控制的域名并接收10+个追踪Cookie
- 加载Google DoubleClick广告追踪基础架构
- 向Facebook CDN、Twitter/X CDN、YouTube和Google APIs发出请求

**隐私标签显示"无数据收集"，与事实完全相反。**

| 项目 | 详情 |
|------|------|

## 结论

本次动态流量分析证实了以下问题：

1. **数据收集远超隐私标签声明**：应用向第三方发送大量用户数据，包括设备信息、位置信息（IP地址、时区）、使用习惯等。

2. **追踪器高度集中**：单一应用集成了OneSignal（分析）、Elfsight（ widgets）、Google DoubleClick（广告）、Facebook和Twitter（社交CDN）等多种追踪组件。

3. **隐私欺骗**：应用隐私清单声称"No Data Collected"（无数据收集），但实际上在后台大量收集和传输用户数据。

4. **持续性追踪**：OneSignal的持久标识符能够跨会话追踪用户，并通过记录IP地址变化建立用户的使用轨迹。

5. **第三方控制**：Elfsight的两阶段加载机制意味着服务器端可以随时决定向用户设备注入何种脚本，用户对此毫无控制权。
