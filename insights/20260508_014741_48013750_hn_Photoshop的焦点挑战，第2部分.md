## 摘要

一篇关于 Photoshop UI 迁移（即从旧框架转向基于 Web 的 Spectrum UI 框架）的深度批评文章引发了社区热议。Adobe 内部团队成员亲自下场回应，承认问题并表示正在修复，但用户对此并不买账。讨论揭示了专业软件在"现代化"过程中面临的深层矛盾。

---

## 主要洞察

### 1. UI 现代化代价：功能倒退而非进步

用户反映新版 Photoshop 的模态弹窗存在大量交互问题：
- 空字段时按退格键会触发错误弹窗并劫持焦点
- 焦点处理逻辑混乱（hover 状态与 focus 状态不一致）
- 失去键盘驱动的工作流支持（Tab 键导航失效）
- 色相/饱和度面板的"全局颜色"下拉菜单被不恰当地拆分为单选按钮
- "风"滤镜至今仍只能左右应用，无法上下应用（十年未变）

一位评论者尖锐地指出：Photoshop UI 现在使用 HTML 等 Web 技术渲染，这导致了传统桌面应用交互模式的系统性崩溃。

### 2. Adobe 工程师的内部回应与辩解

Adobe 团队成员（Principal Scientist）Foster Brereton 亲自回应，承认团队确实发现了这些问题，但以"发布压力"为由为其辩护。他强调团队确实使用 Photoshop（但"未达到大多数用户的程度和深度"），并表示迁移工作"远未结束"，已在修复部分问题。

然而，这种回应引发了更强烈的批评：
- "他们明知有缺陷却仍选择发布给付费客户"
- "根本没有人真正使用 Photoshop"
- "这就是将未完成的生产阶段构建错误地推向用户的行为"
- "Adobe 用一个破损的模态框、几个圆角滑块和加粗的色条来定义'现代 UI'，简直荒谬"

### 3. 专业软件用户的特殊诉求

讨论指出，Photoshop 是专业人士维持生计的工具，不能像初创公司那样以"寻找市场契合"的心态对待 UI 变更。关键观点：

- 用户每天使用 Photoshop 8 小时以上，肌肉记忆和高效工作流极其重要
- "幂等用户（power users）"是软件的"基石物种"，改善他们的工作流最终会惠及所有人
- 改变稳定运行的软件功能不属于"发布与完美的平衡"，而是"首先不造成伤害"

### 4. Adobe 的产品战略问题

多位评论者指出 Adobe 面临的深层问题：

- **AI 优先战略**：Adobe 将大量资源投入 AI 功能集成，导致核心产品体验被忽视
- **订阅制陷阱**：用户无法逃脱 Adobe 的生态锁定，即使不满也无替代选择
- **缺乏一致性**：不同 Adobe 应用间的 UI 风格和键盘快捷键存在巨大差异
- **内存管理糟糕**：应用在几乎不工作时也消耗大量 RAM

### 5. 软件腐化的广泛趋势

有评论感叹整个软件行业正在"腐化"（enshittification）：
- 所有产品都在被糟糕的 UI 重构所侵蚀
- 从旧框架迁移到 Web 技术的过程往往意味着功能的系统性损失
- 后端 JavaScript 和 Web 技术正在"谋杀"桌面软件的交互体验

### 6. 测试与质量管控的缺失

讨论揭示了一个核心问题：这些严重的交互问题竟然通过了所有测试环节。原因包括：
- 开发者不充分使用自己开发的产品
- 缺乏真正的 power user 参与测试
- UI 迁移决策缺乏充分的产品管理审核

## 关键引述

1. "如果你的产品色条之间有灰色空隙很难判断是否相同，那去掉空隙是个改进。但这种改变不应该以牺牲键盘导航为代价。"

2. "真正的问题是：Adobe 将现有的 UI 引擎替换为基于 Web 框架的引擎，并试图在视觉上尽可能匹配，同时引入了一些设计师导致的变化，但完全没有考虑交互设计。"

3. "他们应该被晋升到挖沟的岗位上——那工作太重要了，不能交给这些人。"

4. "Adobe 正忙着想办法留住付费客户、蹭 AI 浪潮。它的每一个应用现在都是一团糟。"

5. "用户可以拒绝'升级'——旧版 Photoshop 仍然很好用。"

## 结论

此次讨论折射出专业创意软件领域的一个核心矛盾：当软件已经高度成熟时，任何"现代化"都面临巨大的体验倒退风险。Adobe 在推进 UI 迁移时未能充分尊重用户的工作流和习惯，暴露出内部测试流程和质量管控的重大缺陷。用户社区的反应表明，对于影响数百万人谋生工具的重大变更，简单的"我们正在修复"承诺是不够的——用户要求的是系统性重构，而非补丁式修复。
