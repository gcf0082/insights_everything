# 洞察报告：Swift 6.3 发布

**洞察链接：** https://www.swift.org/blog/swift-6.3-released/

**发布日期：** 2026年3月24日

**作者：** Holly Borla（Holly Borla 是 Swift 核心团队和语言指导组成员，也是 Apple Swift 语言团队的工程经理）、Joe Heck（Joe Heck 在 Apple 开源项目办公室从事 Swift 相关工作）

## 概述

Swift 6.3 版本进一步将 Swift 扩展到新的领域，并全面改进了开发者的人体工程学体验。本次发布的主要亮点包括：

- 更灵活的 C 互操作性
- 跨平台构建工具的改进
- 嵌入式 Swift 的改进
- 官方 Swift SDK for Android

---

## 语言和标准库

### C 互操作性

Swift 6.3 引入了 `@c` 属性，允许你将 Swift 函数和枚举暴露给项目中的 C 代码。使用 `@c` 标注函数或枚举会提示 Swift 在生成的头文件中包含相应的声明：

```swift
@c
func callFromC() { ... }
```

生成的 C 头文件：

```c
void callFromC(void);
```

你还可以提供自定义名称：

```swift
@c(MyLibrary_callFromC)
func callFromC() { ... }
```

`@c` 还可以与 `@implementation` 结合使用，允许你用 Swift 为 C 头文件中声明的函数提供实现。

### 模块名选择器

Swift 6.3 引入了模块选择器，用于指定 Swift 应该在哪个导入模块中查找代码中使用的 API。如果导入多个提供相同名称 API 的模块，模块选择器可以帮助你消除歧义：

```swift
import ModuleA
import ModuleB

let x = ModuleA::getValue() // 调用 ModuleA 的 'getValue'
let y = ModuleB::getValue() // 调用 ModuleB 的 'getValue'
```

Swift 6.3 还支持使用 `Swift` 模块名来访问并发和 String 处理库 API。

### 库 API 的性能控制

Swift 6.3 引入了新属性，让库作者能够更精细地控制客户端 API 的编译器优化：

- **函数特化：** 使用 `@specialize` 为常见的具体类型提供泛型 API 的预特化实现。
- **内联：** 使用 `@inline(always)` 保证直接调用函数时进行内联。**注意：** 仅当你确定内联的收益大于代码大小的增加时，才使用此属性。
- **函数实现可见性：** 使用 `@export(implementation)` 将 ABI 稳定库中函数的实现暴露给客户端。这允许函数参与更多编译器优化。

## 包和构建改进

Swift 6.3 包含了 Swift Build 集成到 Swift Package Manager 的预览版。这个预览版带来了统一的构建引擎，支持所有平台，提供更一致的跨平台开发体验。

Swift Package Manager 的改进还包括：

- **用于共享宏库的预编译 Swift Syntax：** 将共享宏实现代码提取到支持 swift-syntax 预编译二进制的库中（仅供宏使用）。
- **灵活的继承文档：** 控制命令插件生成的符号图中是否包含继承文档。
- **可发现的包特性：** 使用新的 `swift package show-traits` 命令发现包支持的特性。

## 核心库更新

### Swift Testing

Swift Testing 进行了多项改进，包括警告问题、测试取消和图像附件。

- **警告问题：** 使用新的 `severity` 参数指定测试问题的严重级别。你可以使用 `Issue.record("Something suspicious happened", severity: .warning)` 记录警告。这会反映在测试结果中，但不会将测试标记为失败。
- **测试取消：** 在测试开始后使用 `try Test.cancel()` 取消测试（及其任务层次结构）。这对于跳过参数化测试的单个参数或在测试期间响应表明不应继续的条件很有帮助。
- **图像附件：** 在 Apple 和 Windows 平台的测试期间附加常见图像类型。这通过多个新的跨导入覆盖模块与 UIKit 等 UI 框架暴露。

### DocC

Swift 6.3 为 DocC 添加了三个新的实验性功能：

- **Markdown 输出：** 生成文档页面的 Markdown 版本以及标准的渲染 JSON。尝试使用 `docc convert --enable-experimental-markdown-output`。
- **每页静态 HTML 内容：** 将每个页面的轻量级 HTML 摘要（包括标题、描述、可用性、声明和讨论）直接嵌入 index.html 文件的 `<noscript>` 标签中。这改善了搜索引擎的可发现性和屏幕阅读器的无障碍性。尝试使用 `docc convert --transform-for-static-hosting --experimental-transform-for-static-hosting-with-content`。
- **代码块注释：** 为代码块解锁新的格式注释，包括 `nocopy`（禁用复制到剪贴板）、`highlight`（按行号突出显示特定行）、`showLineNumbers`（显示行号）和 `wrap`（按列宽换行）。

## 平台和环境

### 嵌入式 Swift

嵌入式 Swift 在 Swift 6.3 中有广泛改进，从增强的 C 互操作性到更好的调试支持，再到向完整链接模型迈出的有意义的一步。

### Android

Swift 6.3 包含首个官方发布的 Swift SDK for Android。利用此 SDK，你可以：

- 开始使用 Swift 开发原生 Android 程序
- 更新 Swift 包以支持构建到 Android
- 使用 Swift Java 和 Swift Java JNI Core 将 Swift 代码集成到用 Kotlin/Java 编写的现有 Android 应用程序中

这是 Swift 跨平台开发的一个重要里程碑。
