## 内容概述

GitButler 宣布获得了 1700 万美元 A 轮融资，旨在开发 Git 的继任者。该公司认为当前的 Git 已经无法满足 AI 时代的需求，需要全新的版本控制工具。

## 主要观点

### 质疑与批评

- **资金使用质疑**：评论者认为 1700 万美元可以解决更实际的问题，比如开发能整理 LEGO 玩具的扫地机器人
- **VC 投资逻辑**：有评论指出 VC 投资往往看中人脉和背景，而非实际的技术价值
- **Git 够用论**：许多开发者表示对当前的 Git 感到满意，认为它工作得很好
- **商业模式担忧**：有人担心新产品最终会变成"和 Git 一样，但每月收费"的订阅服务

### 支持与讨论

- **AI 时代需求**：支持者认为随着 AI 代理(Agent)参与开发，现有的 Git 工作流程确实需要重新思考
- **Git 历史局限性**：Git 最初是为通过邮件发送补丁而设计的，不适合现代代码托管平台如 GitHub
- **创新机会**：尽管 Git 足够好用，但历史上也出现过更好的替代方案（如 Google 相对于 AltaVista）

### 技术讨论

- Git 的 UI 仍然混乱
- 大文件支持和子模块/子树功能较差
- Pull Request 在 Git 本身并不是原生概念，而是 GitHub 添加的功能

## 结论

这场讨论反映了科技行业对 VC 投资方向的反思，以及对现有工具是否真的需要"革命性"替代的质疑。大多数评论者认为当前的 Git 已经足够好用，而 1700 万美元本可以用于解决更实际的现实问题。
