# 洞察报告：€54k spike in 13h from unrestricted Firebase browser key accessing Gemini APIs

**基本信息**：
- **来源**：Hacker News

## 事件概述

帖子作者分享了一个令人震惊的经历：由于使用不受限制的Firebase浏览器密钥访问Gemini API，在13小时内产生了€54,000的费用。

### 关键事实
- 作者设置了€80的预算警报和成本异常警报，但都延迟了几个小时才触发
- 当作者采取行动时，费用已经达到约€28,000
- 由于成本报告延迟，最终结算金额超过€54,000

## 讨论要点

### 1. 预算警报的延迟问题
多位用户指出，Google Cloud Platform (GCP) 的预算警报存在严重延迟：
- 预算警报触发延迟数小时
- 即使设置了$100的预算，也只能在超过预算5小时后收到邮件通知
- 这种延迟警报对防止意外超支毫无作用

### 2. API密钥安全问题
- **历史背景**：Google十多年来一直告诉开发者，Google API密钥（如Maps、Firebase使用的）不是秘密
- **问题所在**：Gemini现在接受相同的密钥来访问LLM服务，而这些密钥原本是为公共API设计的
- **GitHub泄露**：大量包含Gemini API令牌的仓库公开存在于GitHub上

### 3. Google的回应与争议
- Google声称已主动阻止"泄露"的API密钥访问Gemini API
- 用户指出这并非新问题，Firebase密钥本身就是"按设计公开"的
- Google文档明确表示"Firebase服务的API密钥不是秘密"

### 4. 缺乏硬性支出上限
- Amazon、Microsoft和Google都不提供硬性支出上限
- 其他较小的云服务提供商提供此功能
- 唯一的解决方法是设置Pub/Sub消息触发云函数来禁用项目计费

### 5. 其他云服务商的问题
- AWS和Azure同样没有真正的成本熔断机制
- 这似乎是整个行业的普遍问题

### 6. 欺诈者的动机
- 生成式AI可用于编写大量垃圾邮件来绕过过滤器
- 有人可能想要在代码库中搜索安全漏洞，而不愿花费自己的$50,000代币
- 用于自动化社会工程等犯罪目的

## 社区建议

1. **API密钥限制**：对HTTP引用者和API进行密钥限制（仅限`generativelanguage.googleapis.com`）
2. **计费预算**：设置带有Pub/Sub"cap"操作的计费预算，而不仅仅是邮件警报
3. **预付费选项**：Google提供预付费积分，当余额为0时所有API密钥将停止工作
4. **客户端密钥处理**：将API调用移至服务器端，避免在客户端暴露密钥
