# Railway 服务中断事件报告

## 基本信息

- **洞察链接**: https://status.railway.com/?date=20260519
- **日期**: 2025年5月19日
- **事件类型**: 重大服务中断 (Major Outage)
- **受影响服务**: Edge Network、Dashboard、API、内部网络控制平面

## 事件摘要

Railway 于 2025年5月19日发生大规模服务中断，用户可能遇到以下错误：
- "no healthy upstream"（无可用上游）
- "unhealthy upstream"（不健康的上游）
- "unconditional drop overload"（无条件丢弃过载）
- 登录失败
- 无法访问控制面板

## 根本原因

Google Cloud封锁了Railway的账户，导致基于Google Cloud的基础设施（Dashboard、API、控制平面）无法访问。

## 影响范围

- Railway Dashboard
- Railway API
- 内部网络控制平面
- 基于Google Cloud的托管基础设施

## 应对措施

1. 直接联系Google Cloud支持团队
2. 恢复Google Cloud计算资源
3. 评估替代方案以恢复受影响服务
4. 临时限制非企业版构建以保护构建基础设施

## 后续状态

截至最后更新（5月20日03:05 UTC），服务正在逐步恢复，部分用户可能仍会遇到间歇性问题。
