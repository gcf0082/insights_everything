## 内容摘要

本文讲述了开发者 tech4bot 如何将一台仅需80美元的 Doogee U10 Android 平板（搭载 Rockchip RK3562 芯片）转变为 Debian Linux 工作站。

### 项目亮点

1. **零文档起步**: 没有BSP、没有内核源码、没有厂商文档，仅从 stock Android 固件中提取 DTB（设备树二进制文件）进行重建

2. **SD卡启动**: Linux 直接从 SD 卡启动，无需修改内部 Android 存储。移除 SD 卡后仍可正常启动 Android

3. **AI 辅助开发**: 大量使用 Claude、Gemini 和 ChatGPT 进行驱动调试、DT 语法和内核配置问题排查

4. **硬件信息**:
   - 设备: Doogee U10 (RK3562)
   - 购买渠道: Amazon DE
   - 售价: 约80美元
   - 内存: 4GB RAM（广告宣称的9GB实际为4GB+5GB扩展内存，即 swap）

5. **性能表现**: 用于终端工作、轻量浏览、VS Code 和小型实验均可用

### 关键技术细节

- Rockchip 设备的 BootROM 默认优先检查 SD 卡，再回退到 eMMC
- 外部启动路径在 Rockchip 启动流程中被更早接受，不受 Android 验证启动约束
- 项目使用上游未修改的 u-boot 和 rockchip-linux 仓库工具

## 社区讨论要点

### 1. 4GB RAM 下的软件运行能力

- **Firefox** 在低内存环境下表现良好，可静默丢弃标签页
- 基于 Electron 的应用和 Chromium 浏览器是内存杀手
- DNS 广告拦截可显著改善低端设备网页浏览体验
- 轻量级桌面环境（如 xmonad + dmenu）配合 Chrome/Emacs 在 8GB 内存下运行良好

### 2. AI 在逆向工程中的角色

- AI 可用于保守辅助：理解错误、对比下游源码、提出小补丁、审查、测试
- 需要具备 C 语言和底层知识才能安全使用 AI 进行内核代码编写
- 目前 AI 尚无法直接将一台平板变成可启动的镜像，仍需大量实际测试和调试

### 3. 设备获取
