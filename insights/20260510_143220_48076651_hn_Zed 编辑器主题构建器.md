## 内容摘要

Zed 编辑器团队发布了新的 Theme-Builder（主题构建器）工具，这是一个可视化的主题编辑器，让用户可以直观地定制 Zed 编辑器的外观主题。该工具的亮点在于提供了 "inspector" 功能，用户可以直接点击预览中的特定元素来查看和调整对应的主题配置。

## 社区反馈分析

### 积极反馈

1. **Theme Builder 实用性**：用户普遍对这款工具表示欢迎，认为它大大简化了主题定制过程。有用户提到仅需几分钟就能创建自己的主题。
2. **Inspector 功能**：许多用户认为这个交互式检查器是"游戏规则改变者"（game changer），可以直观地看到每个UI元素对应的颜色配置。
3. **整体编辑器体验**：部分用户表示 Zed 的默认设置已经很优秀，无需太多修改。一些用户从 Vim 转投 Zed，认为它在许多细节上做得很到位。

### 改进建议与问题

1. **语法高亮不够完善**
   - 有用户指出 Zed 对 Svelte 等语言的语法高亮不如 VSCode 细致，例如尖括号、大小写组件、布尔属性等颜色区分不够
   - 原因可能是 treesitter 解析器版本过旧或缺少语义高亮

2. **深色主题对比度不足**
   - 多个用户反映 Zed 默认的深色主题对比度太低，"灰 on 灰"导致视觉体验沉闷
   - 社区贡献了高对比度主题，如 zedokai-darkest-machine

3. **滚动体验问题**
   - 使用鼠标滚轮时滚动不够平滑，只有使用触控板才能获得流畅体验
   - 有用户提到早在 1986 年的 Amiga 编辑器就已实现键盘平滑滚动

4. **Mac 字体渲染**
   - 部分用户认为 Zed 在 Mac 上的字体渲染偏细，视觉效果不如 Sublime Text

5. **AI 功能配置**
   - 有用户关心是否能禁用 Zed AI 并使用自己的 LLM 提供商
   - 可以禁用内置 AI 并连接 GitHub Copilot，也支持本地运行 zeta-2 模型

6. **远程开发插件缺失**
   - 用户希望 Zed 推出支持连接到 Windows 远程主机的远程开发插件

## 总结

Zed Theme-Builder 的发布获得了社区的积极响应，用户对可视化主题编辑功能表示认可。同时也暴露出 Zed 在语法高亮、主题对比度、滚动体验等方面仍有改进空间。这些反馈反映了开发者对 Zed 成为 VSCode 有力替代品的期待。
