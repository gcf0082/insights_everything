# 256字节BOSS战斗：Endbot 256字节DOS演示分析报告

**洞察链接**: https://hellmood.111mb.de//A_whole_boss_fight_in_256_bytes.html

**基本信息**:
- **项目名称**: Endbot
- **作者**: HellMood/Desire
- **平台**: DOS (通过DosBox-X运行)
- **发布日期**: Revision 2026
- **文件大小**: 256字节
- **类型**: 完整的音频视觉演示/游戏

---

## 概述

Endbot是一个仅用256字节实现的完整BOSS战斗演示，它能够在DOS环境下实时渲染一个机器人Sprite、渐进式子弹伤害、不断成长的爆炸效果、滚动的棋盘格地形，以及MIDI配乐，所有功能都封装在一个微小的.com文件中。

---

## 技术架构

### 运行环境
- **模拟器**: DosBox-X (需要其MPU-401 MIDI模拟)
- **显示模式**: VGA模式13h (320×200像素, 256色)
- **音频**: MIDI端口0x330

### 构建工具
- FASM (Flat Assembler) - 直接从ASM生成COM文件

---

## 代码实现分析

### 1. 文件头与初始化
- 使用BP寄存器作为时间基准 (initbp=0x91C)，所有动画都相对于此时间计算
- 设置VGA模式13h: 320×200分辨率，64KB帧缓冲区位于A000:0000
- 通过`rep outsb`将整个代码块(包括音乐和Sprite)一次性传输到MIDI端口

### 2. 主循环与帧同步
- 外层循环每个像素执行一次
- 像素计数器DI每65536次(320×200帧)归零时执行每帧处理
- **计时器同步**: 写入PIT端口40h后执行`hlt`，使CPU暂停到下一个硬件中断，锁定约30FPS
- **MIDI节拍**: 当BP模256==0时输出四个MIDI音符
- **键盘检测**: 读取端口60h的扫描码，ESC(扫描码1)减去2产生0xFF并置位进位触发退出

### 3. 像素级处理：XY解码与爆炸检测
- **Rrrola技巧**: 将线性像素索引DI乘以0xCCCD，从高位获取近似的X(DL)和Y(DH)，避免使用除法指令
- 时间计算: AX = BP - (768 + initbp)，负值=未爆炸，零=精确撞击时刻，正值=扩散中的圆

### 4. 爆炸效果
- 爆炸是一个数学上精确的扩散圆: X² + Y² = TSI (TSI=撞击后时间)
- 圆边缘像素绘制为橙色(0x2A)
- 圆内部像素溶解到背景中
- 机器人爆炸后向下漂移

### 5. Sprite处理
- 机器人Sprite由Steffest/Desire设计，16行16位位掩码
- 每个像素使用2位 → 4个值: 0=透明，1-3=查找Colors表
- 锯齿形水平移动通过定时器的位5切换实现
- `btc`指令从第512帧开始每帧翻转一个Sprite位，累积可见弹孔

### 6. 背景渲染
- DH>0(地平线以下)为纯蓝色天空
- 地形使用`div dh`获得透视深度值
- 使用XOR获得滚动的棋盘图案
- `aam 9`提供不规则的颜色带

## 数据结构

### Sprite数据
- 16行16位位掩码，与MIDI数据共享连续内存块
- 第8行的字节0xC0和0x7F是有效的MIDI程序切换命令(通道0, 音色127=枪声)

### 音乐数据
- 包含完整的MIDI设置和播放指令
- 枪声、爆炸声、机器人噪声等音效

## 字节节省技巧

| 技巧 | 节省字节数 |
|------|-----------|
| `push imm / pop es` | 4字节 vs 5字节 |
| BP作为全局时钟 | 复用现有寄存器 |
| `mul 0xCCCD`获取XY | 5字节 vs 16位除法序列 |
| `stosb`写像素 | 1字节，自动递增DI |
| `hlt`帧同步 | 1字节暂停到IRQ |
| `xlat`查表 | 1字节查表 |
| `btc`Sprite伤害 | 位翻转+内存地址一步完成 |
| `aam 9`取模 | 2字节实现模9 |
| Sprite兼作MIDI数据 | 同一字节双重用途 |
| `ret`退出DOS | 1字节，栈顶=0=int 20h |
