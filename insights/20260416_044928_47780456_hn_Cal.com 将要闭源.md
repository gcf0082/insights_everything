# Cal.com 转闭源：洞察报告

- **洞察链接**: https://news.ycombinator.com/item?id=47780456
- **标题**: Cal.com is going closed source
- **发布时间**: 5小时前
- **得分**: 112 points
- **评论数**: 109 comments
- **提交者**: Benjamin_Dobell

## 摘要

Cal.com 宣布将从开源转向闭源，官方声称唯一原因是安全。该公司联合创始人声称，16岁的邻居孩子只需15分钟和100美元的Claude Code积分就能"黑掉"他们的开源项目。此举引发社区热议，质疑闭源是否真的是安全解决方案，还是商业决策的借口。

## 核心观点

### 1. AI时代的安全悖论

Drew Breunig的观点引发广泛讨论：开源实际上在AI时代更有价值，因为安全审计预算可以由社区共享，而闭源软件必须自己承担所有漏洞发现成本。

> "要强化一个系统，你需要花费比攻击者更多的tokens来发现漏洞"

### 2. 安全性与攻击面的经济学

- 防御者必须发现系统中的所有漏洞
- 攻击者只需找到一个漏洞即可
- 闭源只是增加了攻击成本，但不能消除风险
- 稳定软件的相对攻击面会随时间缩小

### 3. 真实动机存疑

社区普遍认为闭源决定的真实原因可能包括：
- 避免AI用其代码进行版权清洗
- 难以将用户转化为付费套餐
- 产品正被Google等功能完善的竞争对手 commoditize
- 典型的开源"先开源后闭源"策略

### 4. 闭源真的更安全吗？

- LLMs已经可以反编译二进制文件并进行安全分析
- 闭源只是增加了攻击成本，无法阻止有决心的攻击者
- 这本质上是"通过模糊实现安全"(security through obscurity)
- 作为额外的威慑层是有效的，但不应是主要防御手段

### 5. 对开源生态的影响

- 开源作为营销手段已不再可行
- 投资者越来越质疑开源产品的知识产权
- 未来可能更多"开放规范"软件，源代码由模型按需生成
- 安全和治理将在模型层面进行

## 有趣的讨论点

1. **Xata的反向案例**：同时期有公司(Xata)选择开源，理由是开源能获得更多更快的代码审查

2. **Cal.com的coss.com**：该公司去年刚推出coss.com，现在转向闭源被视为"言行不一"

3. **LLM的双刃剑**：同样的工具可以被开发者用于自测，也会被攻击者用于寻找漏洞

4. **虚假的安全感**：闭源应用仍然可能有漏洞被发现，只是发现难度增加

## 结论

Cal.com的闭源决定引发了关于AI时代开源未来的广泛讨论。大多数评论认为该公司真正的动机是商业考量而非安全，闭源并不能真正解决AI驱动的安全问题。这一决定可能预示着开源商业模式的重大转变。
