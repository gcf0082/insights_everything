## 项目概述

BrightBean Studio 是一个社交媒体管理平台，作者使用 Claude 和 Codex 在3周内完成开发。该项目原本通常需要一个独立开发者花费近一年时间完成，却在3周内实现。

## 技术栈

- **后端**: Django 5.x
- **前端**: HTMX + Alpine.js + Tailwind CSS 4
- **数据库**: PostgreSQL
- **部署**: Docker Compose（4个容器）
- **开源协议**: AGPL-3.0

## 功能特性

- 12个社交平台的一级API集成（Facebook、Instagram、LinkedIn、TikTok等）
- 多租户认证
- 加密凭证存储
- 后台任务处理
- 审批工作流
- 统一收件箱

## AI开发经验总结

### 成功的领域

- Django模型、视图、序列化器、标准CRUD操作
- 文档完善的API提供者模块（如Facebook和LinkedIn）
- Tailwind布局和HTMX交互
- 测试生成
- 跨文件重构

### 存在问题的领域

- TikTok的内容发布API文档不佳，需两步上传流程
- 多租户权限逻辑容易产生跨租户数据泄露
- OAuth边缘情况（令牌刷新、权限撤销、平台特定错误码）
- 后台任务编排（重试逻辑、速率限制退避、错误处理）

### 开发流程

1. 花费4天时间制定详细规格、架构文档和样式指南
2. 将任务分为可并行执行和需按依赖顺序执行两类
3. 使用Opus 4.6进行后端和UI的首轮构建
4. 使用Codex 5.3进行代码审查和bug发现
5. 最多4个并行代理，但这是作者能够处理的最大并行度

## 社区讨论焦点

1. **"氛围编程"(Vibe Coding)的定义和态度**: 有人认为完全由LLM生成且不了解代码的项目风险很高，也有人认为只要自己懂技术就可以完全委托给AI
2. **维护成本**: 软件交付便宜，但维护昂贵且困难
3. **与现有SaaS竞争**: 如Buffer每年每个账户收费约$50，自托管是否值得投资
4. **用户期望**: 多数用户希望社交媒体管理工具能聚合内容到单一位置，而非发布工具
5. **技术选择**: Django/HTMX被某些人认为"过时"，但作者认为这些技术简单、成熟、资源丰富

## 结论

该案例展示了AI辅助开发在复杂生产级项目中的潜力和局限性。关键成功因素包括：详细的规格说明、合理的任务分解、以及对AI局限性的人为监督。项目开源为社区提供了有价值的参考，但也面临着任何人都可以复制的挑战。
