## 文章摘要

原文来自 dbreunig.com，文章讨论了随着AI模型（如Anthropic的Mythos）在网络安全领域的能力提升，网络安全正在变成一种"工作量证明"系统。

**核心观点**：
- AI安全研究所(AISI)的测试显示，Mythos是唯一能完成32步企业网络攻击模拟的模型
- 要保护系统，需要花费比攻击者更多的tokens来发现漏洞
- "你不能因为聪明而得分，你只能通过花更多钱来获胜"

**关键数据**：
- AISI为每次测试投入1亿tokens
- 每次Mythos尝试成本约$12,500
- 10次运行总成本$125,000
- 没有模型显示出收益递减的迹象

## 评论要点

### 支持方观点

1. **somesortofthing**: 攻击者通常只能获得有限访问权限（网络工具、未文档API、二进制文件），而 defenders 可以控制源代码端到端，只需扫描变更的文件，效率更高。

2. **btown**: 指出供应链攻击的风险——开发者被攻击后，整个源代码会被自动审计。

3. **tptacek**: 认为LLM在漏洞研究方面与fuzzing一样具有变革性意义。

4. **jerf**: 认为原则上网络安全是防守方优势，可能实现用有限tokens保护系统。

### 质疑方观点

1. **j2kun**: 质疑AI Security Institute的背景（主要是AI行业人士），指出文章和报告中未提及正式验证等替代方案。

2. **snowwrestler**: 指出该研究针对32步复杂任务，对于简单代码库可能存在收益递减点。

3. **chromacity**: 认为文章犯了分类错误，日常信息安全工作大部分与代码漏洞查找无关。

4. **DerSaidin**: 认为网络安全看起来更像形式化验证系统，而不是token竞赛。

### 建设性建议

1. **tekacs**: 建议从网络转向端点安全，更谨慎地控制输入。

2. **singpolyma3**: 提出如果持续运行LLM找到所有漏洞并打补丁，最终可能没有漏洞。

3. **umvi**: 认为Wireguard（一人编写）比OpenVPN（团队编写）更安全，代码简洁性很重要。

4. **jzelinskie**: 提到Go编程 proverb："一点点复制比一点点依赖更好"。

### 反驳观点

1. **mapontosevenths**: 反驳"防守方优势"说法——攻击者只需要幸运一次，防守方必须每次都对。

2. **devmor**: 更担心社会工程攻击而非LLM寻找漏洞。

3. **saidnooneever**: 批评人们轻信公司对自己产品的宣传。

4. **stringfood**: 对学习困难和令人困惑的主题感到疲惫。

## 结论

这场讨论反映了AI在网络安全领域带来的范式转变。虽然LLM大幅提升了漏洞发现能力，但评论中也提出了多种不同观点：有人认为token竞赛将成为新现实，有人建议通过形式化验证或代码简化来规避风险，还有人质疑这是否是AI公司推销GPU的营销手段。讨论还涉及供应链攻击风险、开源代码安全性等重要话题。
