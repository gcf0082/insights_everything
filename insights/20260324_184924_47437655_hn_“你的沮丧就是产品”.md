## 核心洞察

### 原文主要内容

John Gruber 基于 Shubham Bose 的文章 "[The 49MB Web Page](https://thatshubham.com/blog/news-audit)" 撰写了这篇评论文章，揭示了现代网页设计的严重问题：

1. **网页膨胀严重**: 访问纽约时报查看四个标题，需要加载 422 个网络请求和 49MB 数据，页面需要两分钟才能稳定。

2. **用户沮丧即产品**: "可见性"和"页面停留时间"是当今重要的指标。每个敌对用户体验的决策都源于这一事实——读者在页面上停留越久，出版商可以收取的 CPM 费用就越高。**你的沮丧就是产品**。

3. **移动端问题严重**: The Guardian 的移动端页面只有 11% 的屏幕用于显示文章内容，仅相当于四行文字。

4. **印刷版vs网页版**: 同一出版物（纽约时报、卫报、华尔街日报、大西洋月刊、纽约客等）的印刷版对读者非常尊重，但网页版却充斥着自动播放视频、弹窗广告、订阅请求等干扰。

5. **行业悖论**: 人们因为网站体验越来越差而减少上网时间，但出版商却用更多的敌对用户体验来试图"摆脱"这个困境——这实际上是驱赶用户的策略。

## HN 热门评论摘要

### 1. 广告系统的混乱 (最高赞评论)

**用户 jes5199**: 曾在一个试图用其他方式取代广告作为新闻资金来源的创业公司工作（最终失败）。他们发现：

- 新闻网站运营者大多不知道自己的广告来自哪里
- 忘记当初广告系统是如何安装的
- 即使想关掉广告也无法做到

他们甚至为合作伙伴开发了一个服务端广告拦截器，因为该合作伙伴已经完全失去对自身平台的控制。

### 2. 广告插件开发者的困惑

**用户 browningstreet**: 有一个流量不错的网站，使用了一个流行的高级广告插件。当他取消插件订阅时，开发者问他"有什么难的？"——这反映了行业内人士对普通用户面对的复杂性的无知。

### 3. Tag Manager 泛滥问题

**用户 evilduck**: 在一个知名网站工作，每年营销领导层都会更换，新的 CMO 会带来全新的 Google Tag Manager 脚本。每年有 50 个新的脚本注入，但没有人负责移除之前的脚本。到年底时会有 250 个 Tag Manager 注入。

### 4. 新闻行业的技术困境

**用户 Eric_WVGG**: 在新闻行业工作过，网站的 web 技术栈极其陈旧——有人离开 20 年后，网站上还有他留下的代码，有的甚至用 eval() 执行。Web 从未找到像印刷时代那样盈利的方式。

### 5. 付费订阅的困境

**用户讨论**: 许多人提到了 Apple News+、Netflix for news 等服务，但普遍认为：

- Apple News+ 虽然包含优质内容，但广告依然很多
- YouTube Premium 定价 $14/月，但广告收入仅约 $2/月/用户
- "Netflix for news" 的想法很好，但一旦出现就会有 50 个类似服务，导致内容分散

### 6. 无广告手机的创意

**用户 jes5199**: 他们曾为一个大型手机公司做方案——如果用户每次浏览网页时网站选择不显示广告，手机公司就支付费用。算下来每个用户每月约 $20。但由于无人能解决物流问题，这个想法最终未能实现。

### 7. RSS 的复兴

**用户 Eric_WVGG**: 已经 20 年不在网页上读新闻了，推荐使用 RSS（Feedly、NetNewsWire、Reeder、Feedbin 等）。

## 结论

这篇洞察揭示了现代互联网的核心悖论：

1. **出版商被广告系统绑架**: 他们无法控制自己平台上的广告，甚至不知道广告从何而来

2. **用户体验与商业利益的冲突**: 停留时间成为关键指标，导致用户体验被系统性地破坏

3. **行业知识流失**: 技术人员离开后，网站成为无人理解的"遗留系统"

4. **解决方案的困境**: 订阅服务面临碎片化问题，广告拦截反而成为必需

正如 Gruber 所言："这个世界上没有任何印刷出版物会这样做。网页是世界上唯一其最高决策者 despise 这个媒体并试图让人们离开它的媒介。"
