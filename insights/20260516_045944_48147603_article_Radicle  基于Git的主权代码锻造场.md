## 项目概述

Radicle 是一个基于 Git 构建的自治代码工坊（sovereign code forge），属于开源、去中心化的代码协作技术栈。与中心化的代码托管平台不同，Radicle 网络不受单一实体控制，仓库以去中心化方式在节点间复制，用户对自己的数据和协作流程拥有完全控制权。

## 核心特性

### 1. 数据安全与永久性
所有社交制品（issues、discussions、patches 等）都存储在 Git 中，并使用公钥密码学进行签名验证。Radicle 会验证所有数据的真实性和来源，确保用户数据的长期安全。

### 2. 无与伦比的自治性
用户可以运行自己的节点（seed），实现抗审查的代码协作，构建不依赖第三方的弹性网络。

### 3. 本地优先（Local-first）
Radicle 采用本地优先架构，即使没有网络连接也能正常工作。用户拥有自己的数据，便于迁移、备份和离线访问。

### 4. 可演化与可扩展
Radicle 的协作对象（Collaborative Objects, COBs）是其社交原语，允许将 issues、discussions、code review 等功能实现为 Git 对象。开发者可以扩展 Radicle 的功能，构建任何需要的协作流程。

### 5. 模块化设计
Radicle 技术栈包含 CLI、Web 界面、TUI，以及后端的 Radicle Node 和 HTTP Daemon。各组件可独立替换，便于开发自定义客户端。

## 技术架构

Radicle 协议使用加密身份来管理代码和社交制品，利用 Git 进行节点间的数据交换，并采用自定义的 gossip 协议来传播仓库元数据。

**栈结构**：
- **客户端层**：Radicle CLI、Radicle Web
- **仓库层**：代码、issues、patches 等
- **存储层**：Git
- **网络层**：Radicle Node（NoiseXK 协议）、Radicle HTTPD（HTTP + JSON）
