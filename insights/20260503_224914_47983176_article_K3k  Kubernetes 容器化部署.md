## 项目简介

k3k (Kubernetes in Kubernetes) 是由 Rancher 团队开发的一个工具,它允许用户在现有的 Kubernetes 环境中创建和管理隔离的 K3s 集群。该项目实现了高效的多租户、简化的实验流程和强大的资源隔离,通过在同一物理主机上运行多个轻量级 Kubernetes 集群来最小化基础设施成本。

k3k 提供两种模式:

- **共享模式(shared)**：优化资源利用率,多个 K3s 集群共享底层资源
- **虚拟模式(virtual)**：提供完全隔离,为每个嵌入式集群分配专用的 K3s server pods

## 核心特性

1. **资源隔离**：确保工作负载隔离,防止团队或应用程序之间的资源争用,可为每个嵌入式集群定义资源限制和配额

2. **简化的多租户**：轻松为不同用户或项目创建专用的 Kubernetes 环境,简化访问控制和管理

3. **轻量级和快速**：利用 K3s 的轻量特性快速启动和销毁集群,加速开发和测试周期

4. **优化的资源利用率(共享模式)**：在同一物理主机上运行多个 K3s 集群,最大化基础设施投资

5. **完全隔离(虚拟模式)**：为每个嵌入式集群提供专用的 K3s server pods,确保工作负载的完全分离

6. **Rancher 集成**：可与 Rancher 无缝集成,通过 Rancher 的直观 UI 管理和监控嵌入式集群

## 安装和使用

### 安装 K3k Controller

```bash
helm repo add k3k https://rancher.github.io/k3k
helm repo update
helm install --namespace k3k-system --create-namespace k3k k3k/k3k
```

### 安装 k3kcli

```bash
wget -qO k3kcli https://github.com/rancher/k3k/releases/download/v1.0.2/k3kcli-linux-amd64 && \
  chmod +x k3kcli && \
  sudo mv k3kcli /usr/local/bin
```

### 创建集群

```bash
k3kcli cluster create mycluster
```

### 删除集群

```bash
k3kcli cluster delete mycluster
```

## 应用场景

1. **多租户环境**：为不同团队或项目提供隔离的 Kubernetes 环境
2. **开发和测试**：快速创建和销毁测试环境,加速 CI/CD 流程
3. **资源优化**：在有限硬件资源上运行多个隔离的 Kubernetes 集群
4. **边缘计算**：在边缘节点上轻量部署 Kubernetes 集群
5. **教学和培训**：为学员提供独立的实验环境

## 优缺点分析

### 优点

- 轻量级,启动快速
- 资源利用率高
- 与 Kubernetes 和 Rancher 无缝集成
- 支持两种隔离模式,灵活满足不同需求
- 完整的 RBAC 和资源配额支持

### 缺点

- 依赖于 K3s,功能是 K3s 的子集
- 需要预先配置存储提供者
- 在共享模式下可能存在资源争用风险

## 总结

k3k 是一个创新性的项目,它通过在 Kubernetes 中运行 K3s 集群的方式,实现了高效的多租户和资源隔离。该项目特别适合需要在单一 Kubernetes 集群上运行多个隔离环境的场景,如开发测试、多租户 SaaS、边缘计算等。结合 Rancher 使用,可以获得更好的管理体验。
