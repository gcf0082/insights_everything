## 报告摘要

GnuPG 2.5.19 版本已正式发布，该版本新增了若干功能并修复了多个错误。

---

## 主要内容

### 版本背景

- 2.5 系列的主要改进包括：64位 Windows 平台优化以及引入 Kyber（又称 ML-KEM 或 FIPS-203）作为后量子密码（PQC）加密算法
- 值得注意的是，旧的 2.4 系列将在两个月后终止支持（end-of-life），建议用户及时升级到 2.5.19
- 新版本与旧版本完全兼容

### 新增功能

- **gpg**: 新增 `--use-ocb-sym` 选项
- **gpg**: 新增 `--show-[only-]session-hash` 选项
- **gpgsm**: 允许在 `--cipher-algo` 选项中使用密码模式参数
- **gpgsm**: 检查 CRL 分发点失败时输出更多详细信息
- **agent**: 改进智能卡场景下的 pinentry 行为和文本
- **dirmngr**: 为 `--keyserver` 添加新的 "clear" 关键字

### 错误修复

- 修复了 `--refresh-keys` 中的边缘情况
- 修复了空密码时的 `gcry_kdf_derive` 调用问题
- 修复了 PKCS#12 PBES2 密钥长度参数问题（支持德国电信证书导入）
- 修复了证书使用不同算法签名的 bug
- 修复了 DE-VS 模式下 GCM 兼容性问题
- 修复了证书链检查问题
- 修复了 trustlist.txt 缺少结尾换行符的问题
- 修复了 SSH 签名中的 RSA 填充问题
- 修复了 `-C` (--directory) 选项的输出目录检查问题

### 其他变更

- 当 RSA 密钥中 p >= q 时主动报错，以检测错误生成的 PGP 密钥

### 下载信息

- 源代码: https://gnupg.org/ftp/gcrypt/gnupg/gnupg-2.5.19.tar.bz2
- Windows 安装包: https://gnupg.org/ftp/gcrypt/binary/gnupg-w32-2.5.19_20260424.exe

### 国际化和支持

- 该版本支持 26 种语言，包括中文、捷克语、荷兰语、法语、格鲁吉亚语、德语、意大利语、日语、挪威语、波兰语、葡萄牙语、俄语、土耳其语和乌克兰语

## 关于 GnuPG

GNU Privacy Guard (GnuPG/GPG) 是 OpenPGP 和 S/MIME 标准的完整且免费实现。GnuPG 允许加密和签名数据及通信，具有强大的密钥管理系统，并可访问公钥目录。GnuPG 是自由软件，可免费使用、修改和分发，受 GNU 通用公共许可证条款约束。
