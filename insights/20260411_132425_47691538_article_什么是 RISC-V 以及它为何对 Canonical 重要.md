## 摘要

RISC-V 是一种开放标准指令集架构（ISA）。近年来，对 RISC-V 的关注快速增长。2026 年，预计将有更多支持 Linux 的芯片和开发板面世。本文分析了 RISC-V 增长的原因、价值主张，以及 Canonical 为何要支持 RISC-V。

---

## 什么是 RISC-V？

RISC-V 是一个开放标准指令集架构（ISA）。ISA 描述了 CPU 执行程序所需的指令集。其他现代 ISA 的例子包括 Armv8-A 或 Intel x86_64。RISC-V 创建于 2010 年，RISC-V International 于 2015 年成立，作为规范的管理机构。这些规范通过与行业、学术界甚至爱好者的社区参与而开发。

作为开放标准，任何人都可以创建 RISC-V CPU。作为规范，它提供了基础技术标准，同时允许通过 ISA 扩展和商业模型进行创新。它不是 CPU 的实现，而是类似 USB 或以太网的架构规范。

如今，RISC-V 已广泛使用并批量出货。大多数 RISC-V 使用场景是深度嵌入的——这意味着它们绑定在产品中，个人开发者无法访问。但这种情况正在改变。2026 年，预计将有多个 vendors 提供支持 RVA23 配置文件的开发板，可运行 Linux。

考虑使用 RISC-V 的原因有很多，从采用开放标准架构的哲学，到技术主权的关注。还有基本的业务和技术驱动因素，我将在下文详细解释。

---

## 推动新的商业模式

作为许可的 ISA，RISC-V 为企业和开源社区提供了最大的灵活性。RISC-V 的实现可以是开源的、闭源的、许可的 IP，或为内部使用而开发。

有许多公司提供商业 IP 形式的 RISC-V CPU，[Qualcomm](https://www.youtube.com/watch?v=t6_9pbgg1LI&list=PL85jopFZCnbPPRyjl_qMQ50DPq_iQKhFg&index=39) 和 [NVIDIA](https://riscv.org/blog/how-nvidia-shipped-one-billion-risc-v-cores-in-2024/) 等公司在其产品中使用 RISC-V 核心。

Google 通过 [OpenTitan](https://opentitan.org/) 项目对 RISC-V 给予了强有力的支持，该项目使用完全开源的 CPU 作为安全信任根。Google 最近宣布在 Chromebook 中批量生产硅，并在数据中心中使用。

---

## 可扩展性推动技术创新

与大多数其他 ISA 不同，RISC-V ISA 专门设计为可扩展的，它也被拆分为多个扩展集，可以选择使用。这为用户提供了更多选择和力量，可以使用该 ISA 来：

- 使用新型数据类型进行 AI/ML
- 使用新技术或自定义指令进行安全控制
- 控制自定义加速器
- 使用最小指令集创建系统以降低功耗/面积
- 进行新型 CPU 架构和微架构的学术研究

随着 AI/ML 领域的快速发展，拥有允许创新和实验的硬件架构变得越来越重要。虽然这种灵活性可能会给软件生态系统带来问题，但 RISC-V 有多种方式来管理这个问题，例如将指令子集分组（如"F"表示浮点指令），或者配置文件（如 RVA23 将多组指令组合在一起）。此外，对于深度嵌入的场景（开发者同时控制软件和硬件），这不那么成问题。

---

## 软件生态系统的成熟度

RISC-V 新手的一个常见问题是"虽然硬件方面很有趣，但我如何确信我的应用程序能在上面运行？"这个问题有多种解答方式，未来的博客将探讨在 Ubuntu 中支持自定义指令的��体细节。简短的回答是，开源社区已经广泛采用 RISC-V，并在生态系统的许多部分提供了出色的支持。这包括 Linux 内核、GCC 和 LLVM 等工具链，以及大多数实时操作系统，Ubuntu 自 2021 年起就支持 RISC-V。

配置文件方面的标准化工作确保了不同实现之间的兼容性——例如，RVA23 合规软件可在任何 RVA23 硬件上移植。

---

## 为什么 RISC-V 对 Canonical 很重要

从 Canonical 的角度来看，我们希望支持我们的社区和客户想要使用的 ISA。只要有开源移植到 RISC-V，我们就会尝试提供支持，并提供与其他架构相同标准的支持。这意味着 Ubuntu 的长期支持（LTS）版本将通过 Ubuntu Pro 订阅支持 RISA-V长达 15 年。Ubuntu 24.04 LTS 支持 RVA20 配置文件，而从 25.10 开始（包括 26.04 LTS）我们将支持 RVA23。解释配置文件是未来博客的主题，但可以说，大多数支持 Linux 的 RISC-V 硬件将在未来多年得到我们的支持。

## 结论

RISC-V 正在颠覆半导体行业，为定制硅启用新的应用和用例。虽然重点一直在硬件上，软件社区也非常积极地开发对 RISC-V 的支持，并且已经达到良好的成熟度。Canonical 将 RISC-V 视为一等公民，我们的目标是将其支持到与竞争架构相同的水平。我们已经很好地实现了这一目标。

如果您在下一个项目中考虑使用 RISC-V，从用于物联网和边缘设备的 Ubuntu Core 到 Ubuntu Pro 和 Ubuntu Server，我们都为您覆盖。为什么不与我们谈谈您的需求？
