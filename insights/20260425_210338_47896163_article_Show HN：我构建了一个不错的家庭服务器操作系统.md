## 项目概述

Lightwhale 是一个专为运行 Docker 容器而设计的目的构建操作系统。它可以从 ISO 直通引导，进入完全配置的 Docker Engine，无需安装或配置。核心系统是不可变的，天生无需维护，同时增强安全性和可靠性。数据和自定义内容完全隔离存储在专用设备上，确保永远不会与核心系统文件纠缠在一起。

## 核心特性

### 1. 即插即用
只需下载 ISO 并直引导服务器进入可运行的 Docker Engine，所有必要工具立即可用。

### 2. 简约设计
移动部件数量降至最低，使系统易于学习并快速掌握。

### 3. 安全可预测
由于核心是不可变的且无状态的，系统提供了最小的恶意软件攻击面，同样能够抵抗无意修改。每次启动都是一致的。

### 4. 可选持久化
持久化发生在数据文件系统上，该系统在物理上与不可变根文件系统分离。默认情况下，数据文件系统位于 RAM 中且易失。但是，当启用持久化后，Lightwhale 会自动检测、格式化并在单独存储设备上挂载数据文件系统，确保更改在重启后持久化。

### 5. 高效环保
所有不必要的进程都被移除，确保最小的资源占用和功耗。Lightwhale 延长了旧设备或低端机器的使用寿命，减少了环境影响。

### 6. 数字主权赋能
Lightwhale 让各种规模的组织都能轻松自托管，摆脱大型科技锁定，并收回隐私和数据。

## 启动流程

Lightwhale ISO 可以在裸机或虚拟机上启动，支持 UEFI 和传统 BIOS。它使用经典的 sysv 风格 init 系统，保持启动过程简单透明。

1. 引导加载程序将 Linux 内核和根文件系统加载到内存中
2. 内核初始化硬件，然后将控制权交给 /init
3. init 进程读取 /etc/inittab，挂载标准的可写 tmpfs 用于 /tmp 和 /run，然后执行 /etc/init.d 中的 init 脚本
4. 早期阶段，可写数据文件系统被挂载
5. 所有文件系统就位后，其余服务启动，Lightwhale 准备就绪

## 不可变性设计

根文件系统是静态的 squashfs 镜像，压缩以节省内存，并且天生不可变。不可变的内核和根文件系统立即带来了简约、安全性和可靠性方面的优势：

- **零安装**：所有必要软件和配置都已预烘焙
- **零维护**：无需额外软件或更新
- **减少攻击面**：能够抵抗无意识和恶意修改
- **无垃圾**：只读根文件系统防止混乱
- **自由实验**：重启即可撤销
- **轻松恢复**：设备丢失或损坏可简单替换

## 持久化机制

Lightwhale 通过自动化子系统在启动早期激活时提供临时和持久两种可写性。数据文件系统挂载在 /mnt/lightwhale-data。

- 所有由 Lightwhale 写入的数据都保存在 /mnt/lightwhale-data/lightwhale-state
- 这作为 overlayfs 堆栈中的可写上层，不可变根作为下层
- 默认使用易失的 tmpfs；启用持久化后使用存储设备

### 关键目录

数据文件系统 overlay 只应用于以下战略性目录：
- /etc：系统配置定制
- /var：日志和其他应用数据
- /home：用户账户定制

### Docker 数据

Docker 配置将其数据根目录直接位于数据文件系统上，包括镜像、容器、卷和网络状态：/mnt/lightwhale-data/lightwhale-state/docker

### 启用持久化

通过将魔数写入存储设备来启用持久化：
```bash
echo "lightwhale-please-format-me" | sudo dd conv=notrunc of=/dev/sdx
```

## 常见问题

| 问题 | 答案 |
|------|------|
| 如何登录？ | 使用默认登录：用户名 op，密码 opsecret（首次使用后务必修改） |
| 支持 WiFi 吗？ | 支持，使用 sudo setup-wifi --ssid="my wifi name" --password="my wifi secret" |
| 支持树莓派吗？ | 不支持，目前仅支持 x86-64 |
| 可以虚拟化运行吗？ | 支持 QEMU/KVM（Proxmox）和 VMware ESXi |
| 如何安装软件？ | 只能安装 Docker 容器，不能直接安装到文件系统 |
| 默认编辑器是什么？ | vi（不是 nano） |

## 技术规格

- **系统类型**：最小化、容器优先、优化可靠性和乐趣
- **引导方式**：ISO 直通引导
- **init 系统**：经典 sysv 风格
- **根文件系统**：squashfs（只读）
- **数据文件系统**：tmpfs（默认）或 btrfs（持久化）
- **容器运行时**：Docker Engine
- **支持架构**：x86-64（BIOS 和 EFI）
- **存储支持**：SSD、HDD、NVMe

Lightwhale 3 是一个创新的、以容器为中心的轻量级服务器操作系统，通过不可变根文件系统设计实现了零维护、零安装的目标。它特别适合家庭实验室、企业边缘节点或集群部署，强调简约、安全和数字主权。对于寻求简化服务器管理并避免传统 Linux 系统维护负担的用户来说，Lightwhale 提供了一个独特的选择。
