## 摘要

本报告揭示了 Vercel 为 Claude Code 开发的插件存在严重的隐私和安全问题。该插件会默认收集用户的所有 bash 命令和工具调用信息，无论用户是否正在处理 Vercel 相关项目。这一行为引发了社区的广泛批评和担忧。

## 问题概述

### 数据收集范围

根据原文分析，Vercel 插件会收集以下数据：

| 数据类型 | 收集方式 | 默认状态 |
|---------|---------|---------|
| Bash 命令 | 完整命令字符串（包括文件路径、项目名称、环境变量等） | 默认开启 |
| 原生工具调用 | 所有工具调用 | 默认开启 |
| 提示内容 | 用户 prompts | 需要用户同意 |

### 关键问题点

1. **跨项目数据收集**: 插件会在所有会话中运行，包括与 Vercel 无关的项目，如纯后端工作、数据科学等
2. **性能开销**: 每个会话约有 19k token 的开销，无论项目是否与 Vercel 相关
3. **技能注入**: 每个技能的触发规则都会在每个 prompt 和每个工具调用时评估
4. **默认开启**: Bash 命令和工具调用的收集是默认开启的，用户需要手动设置 `VERCEL_PLUGIN_TELEMETRY=off` 才能关闭

## Vercel 工程师回应

Vercel 工程师 `andrewqu` 在评论中做出以下回应：

- 插件一旦安装就会始终启用
- 不希望限制为仅检测 Vercel 项目，因为也希望帮助"绿色field"项目
- 所有数据是匿名的，使用随机 UUID
- 提示遥测是可选的，默认关闭
- 设置 `VERCEL_PLUGIN_TELEMETRY=off` 可以关闭所有遥测

## 社区反应

### 主要批评观点

1. **安全风险**: 插件会收集包含敏感信息（如 API 密钥、凭据）的 bash 命令，这些数据会被发送到 Vercel 服务器
2. **违反 GDPR**: 在默认情况下收集 bash 命令可能违反 GDPR 规定
3. **信任破裂**: 用户认为这是对信任的严重违反
4. **dark pattern**: 征询权限的方式被批评为 "dark pattern"（黑暗模式）

### 典型评论摘要

- "A deployment plugin shipping raw bash command strings off your machine. 'actively defend against it' is just normal hygiene"
- "If you log in with OAuth2, it also connects to platform.claude.com"
- "The degree of contempt for your (presumably quite technical) users necessary to do this is astounding"

## 建议

1. **立即行动**: 用户应设置 `VERCEL_PLUGIN_TELEMETRY=off` 环境变量来禁用遥测
2. **安全评估**: 如果用户曾经使用过该插件，应考虑将相关机器上的 secrets 视为已泄露
3. **关注发展**: 社区正在关注 Anthropic 是否会对此采取行动，因为这可能违反了 Claude Code 的插件政策
