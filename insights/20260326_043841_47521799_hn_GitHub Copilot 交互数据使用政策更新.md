## 摘要

GitHub 宣布将从 2024 年 4 月 24 日起，使用 GitHub Copilot Free、Pro 和 Pro+ 用户的交互数据（包括输入、输出、代码片段和相关上下文）来训练和改进其 AI 模型，除非用户主动选择退出。

## 主要内容

### 政策变更要点

1. **默认启用**: "允许 GitHub 将我的数据用于 AI 模型训练"选项默认开启
2. **数据收集范围**: 包括 Copilot Free、Pro 和 Pro+ 用户的交互数据
3. **企业用户保护**: 企业和企业客户的协议禁止将 Copilot 交互数据用于模型训练，GitHub 表示会遵守这些承诺
4. **隐私风险**: 数据可能与 GitHub  affiliates（包括 Microsoft）共享

### 用户争议与反馈

1. **黑暗模式**: 用户批评 GitHub 将数据收集包装成"用户功能"，启用=可以使用功能，禁用=无法使用，仿佛分享数据是一种特权

2. **隐私担忧**:
   - 收集的信息可能包含个人身份信息 (PII)
   - 可能涉及私有仓库中的代码和敏感信息
   - 用户担心即使关闭选项，数据可能已被使用

3. **法律合规性**:
   - 用户质疑在欧盟的合法性
   - 欧盟法律要求数据收集必须基于"自愿、特定、知情和明确"的同意

4. **信任问题**:
   - 用户对 GitHub 是否真正遵守承诺表示怀疑
   - 担心即使禁用也可能已被用于训练

### 建议的应对措施

1. 手动关闭数据共享设置：https://github.com/settings/copilot/features
2. 迁移到替代平台（如 Codeberg）或自托管 Git 服务
3. 在代码中添加版权声明，更改许可证

## 社区情绪

社区普遍对这一政策变化表示不满，认为这是：
- 对个人用户的"剥削"
- 典型的黑暗模式营销
- 可能违反数据保护法规
