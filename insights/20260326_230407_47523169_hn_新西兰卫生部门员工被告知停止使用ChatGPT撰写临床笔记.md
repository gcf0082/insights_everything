# 洞察报告：Health NZ 工作人员被要求停止使用 ChatGPT 撰写临床笔记

- **标题**: Health NZ staff told to stop using ChatGPT to write clinical notes
- **发布时间**: 18小时前
- **分数**: 154 points
- **评论数**: 50 comments
- **来源**: RNZ (Radio New Zealand)

## 核心事件

新西兰卫生局（Health NZ）工作人员被要求停止使用 ChatGPT 等公共AI工具撰写临床笔记。与此同时，卫生局正在大力推广AI转录和笔记解决方案。

## 主要讨论观点

### 1. 隐私与数据安全问题
- **数据流向**: 卫生机构声称数据留在本地，但实际上会传输到澳大利亚的 Azure OpenAI
- **缺乏保护**: 新西兰的医疗记录保护措施薄弱，之前发生过大面积医疗数据泄露并被在网上出售的事件
- **监管不力**: 虽然隐私法规存在，但执法不严

### 2. AI生成笔记的质量问题
- **幻觉问题**: AI会生成不存在的信息（幻觉），某位医生的笔记完全不可靠
- **过度详细**: AI生成的内容往往过于冗长，且遗漏重要细节
- **需要更多时间修复**: 医生花费在检查和修正AI笔记上的时间比手写笔记还多

### 3. 患者知情同意问题
- **强迫同意**: 患者被要求允许医生使用AI转录，拒绝时会遭到阻力
- **误导宣传**: 宣传称数据"留在本地"，但实际上并非如此
- **缺乏透明度**: 医护人员对AI系统的工作原理知之甚少，无法充分告知患者

### 4. 医疗专业人士的看法
- **部分医生已停止使用**: 如Heidi AI等工具，因错误太多而被一些全科医生放弃
- **责任问题**: 如果医生因AI生成的内容出错而受到投诉，将面临困境
- **技能退化风险**: 过度依赖AI可能导致医生技能退化

### 5. 支持AI应用的观点
- **提高效率**: AI可帮助总结和概述，适合非关键场景
- **减轻文档负担**: 医疗系统对文档要求越来越高，AI可能帮助医生回归患者护理
- **行业趋势**: 软件行业早已接受"足够好"的理念，医疗行业可能也需要适应

---

## 关键结论

1. **禁止公共AI工具**的指令与**推广商业AI解决方案**之间存在明显矛盾
2. AI在临床环境中的应用仍不成熟，存在幻觉和数据隐私问题
3. 患者知情同意机制不完善，缺乏对数据处理方式的了解
4. 需要建立更严格的监管框架，确保AI在医疗领域的安全应用
5. 原始音频应与AI生成笔记一起保存，以便验证和追责
