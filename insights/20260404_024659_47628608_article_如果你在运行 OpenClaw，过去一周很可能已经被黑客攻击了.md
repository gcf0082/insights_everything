# OpenClaw 安全漏洞洞察报告

## 基本信息

- **洞察链接**: https://old.reddit.com/r/sysadmin/comments/1sbdw29/if_youre_running_openclaw_you_probably_got_hacked/
- **报告生成日期**: 2026-04-04
- **主题**: OpenClaw 安全漏洞与攻击事件

## 概述

Reddit r/sysadmin 社区发布的这篇帖子警示运行 OpenClaw 的用户可能已经遭到入侵。OpenClaw 是一个开源的本地运行 AI 助手平台，在 GitHub 上已获得超过 34.6 万星标，成为最受欢迎的 AI 代理项目之一。然而，该项目在 2026 年爆发了大规模安全危机。

## 主要安全漏洞

### 1. CVE-2026-25253 - WebSocket 令牌泄露
- **严重程度**: 高危
- **漏洞描述**: OpenClaw 在 2026.1.29 之前的版本存在 WebSocket 令牌泄露漏洞，攻击者可通过恶意网站窃取用户令牌
- **影响**: 攻击者可劫持 OpenClaw 代理，完全控制用户设备

### 2. ClawJacked 漏洞
- **漏洞类型**: 0-Click 远程代码执行
- **影响**: 恶意网站可通过 WebSocket 直接控制本地 OpenClaw 代理，窃取敏感数据

### 3. 恶意技能（Malicious Skills）
- **发现数量**: 341 个恶意技能
- **影响**: 15% 的社区技能包含恶意指令
- **攻击方式**: 通过 ClawHub 商店分发恶意技能

### 4. 配置安全问题
- **暴露实例**: 约 13.5 万个 OpenClaw 实例暴露在互联网上
- **泄露数据**: 超过 150 万个 API 令牌、OAuth 令牌和聊天历史记录被泄露
- **根本原因**: 默认配置不安全，默认信任 localhost

| 2026年2月 | "ClawJacked" 漏洞允许网站劫持代理 |
| 2026年3月 | 4天内披露 9 个 CVE，其中 1 个严重程度达 9.9 |

## 受影响版本

- OpenClaw 2026.1.29 之前的所有版本
- 使用默认配置的实例尤其容易受到攻击

## 建议措施

1. **立即更新**: 将 OpenClaw 升级到最新版本（2026.2.26 或更高）
2. **检查配置**: 确保默认配置已加固，不要暴露在公网
3. **审查技能**: 检查已安装的社区技能，移除可疑技能
4. **轮换凭证**: 如怀疑已泄露，立即轮换所有 API 密钥和令牌
5. **网络隔离**: 建议将 OpenClaw 部署在隔离网络环境中
