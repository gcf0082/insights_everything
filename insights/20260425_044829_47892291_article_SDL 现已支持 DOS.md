## 功能摘要

此 PR 为 SDL3 添加了完整的 DOS 平台支持，使用 DJGPP（一个针对 DOS 的 32 位 C/C++ 编译器）进行编译。这是一个相当完整的移植，仅缺少部分次要功能（如音频录制）。

## 已支持的功能

### 视频 (Video)
- VGA 和 VESA 1.2+ 帧缓冲
- RGB 色彩模式和 8 位索引色彩（通过 VGA DAC 调色板编程）
- 硬件页翻转（page-flipping）配合垂直同步
- VBE 状态保存/恢复（退出时恢复原始显示模式）
- 支持 LFB（线性帧缓冲）和银行切换（banked）模式

### 音频 (Audio)
- Sound Blaster 16：16 位立体声，最高 44.1 kHz
- Sound Blaster Pro：8 位立体声，最高 22 kHz
- Sound Blaster 2.0/1.x：8 位单声道
- 所有音频通过 IRQ 驱动的 DMA 实现，使用双缓冲自动初始化

### 输入 (Input)
- PS/2 键盘：支持扩展扫描码（0xE0 前缀）、Pause 键等
- INT 33h 鼠标：支持灵敏度查询
- 游戏端口摇杆：通过 BIOS INT 15h 实现自动校准

### 线程 (Threading)
- 使用 `setjmp`/`longjmp` 实现合作式调度器并配合栈修补（stack patching）
- 真实的支持互斥锁（mutex）、信号量（semaphore）、TLS 和条件变量（使用通用回退实现）
- 事件泵和延迟函数中设置让出点（yield points）以保持音频和其他线程的响应性

### 计时器 (Timer)
- 使用 DJGPP 的 `uclock()` 实现原生 PIT 计时器，约 1.19 MHz 分辨率

### 文件系统 (Filesystem)
- `SDL_GetBasePath()` / `SDL_GetPrefPath()` 通过 DJGPP 的 `searchpath()` 实现
- POSIX 文件系统操作回退支持

### 构建 (Build)
- CMake 交叉编译工具链文件：`i586-pc-msdosdjgpp.cmake`
- DJGPP CI 任务
- 预种子缓存（preseed cache）加速配置

## 未包含的功能

- **音频录制**：仅支持播放
- **SDL_TIME 原生实现**：通过 DJGPP 的 POSIX 层复用 Unix `gettimeofday`
- **动态库加载支持**：`SDL_LoadObject` 不可用

## 关键技术实现细节

### 音频混音
- 初始在 IRQ 处理程序中执行混音，后重构移至主循环以提高稳定性并避免重入问题
- 使用 `SDL_DOS_PumpAudio` 函数管理音频泵送
- 静音陈旧的 DMA 缓冲区半区以防止加载期间出现卡顿

### VBE 视频处理
- 实现 VBE 模式的硬件页翻转（需要多于 1 个图像页面）
- 中断服务程序（ISR）代码/数据锁定以防止中断期间发生页面错误
- 单缓冲模式下始终在 blit 时进行垂直同步以减少撕裂

### 构建兼容性处理
- `SDL_runapp.c`：添加 `SDL_PLATFORM_DOS` 到排除列表
- `SDL.c`：排除 `SDL_Gtk_Quit()`（DJGPP 定义 `__unix__` 但 DOS 无 GTK/显示服务器）
- `sdlplatform.cmake`：添加 DOS 情况以正确检测平台
- 工具链文件支持 `i386-pc-msdosdjgpp-gcc` 回退编译器名称

### 测试支持
- 针对 DOS 平台的测试资源映射为 8.3 文件名格式
- 新增 DJGPP CI 任务

## 重要讨论与问题

### 1. 全屏模式与调色板问题
- `SDL_GetClosestFullscreenDisplayMode()` 倾向于选择最低色深（INDEX8），导致未设置调色板的应用程序显示黑屏
- 开发者建议修改 `SDL_GetClosestFullscreenDisplayMode()` 优先选择最高色深而非最低色深
- 暂时通过 `SDL_HINT_DOS_ALLOW_INDEX8_MODES` 控制，后又移除该 hint（因为主分支已修复 bpp 选择问题）
- 相关修复在 PR #15442 中处理

### 2. 测试程序兼容性问题
- `sprite.exe`：某些环境下立即退出
- `wm.exe` 和 `draw.exe`：某些环境下渲染空白
- `testpalette`：存在段错误问题
- 部分问题依赖于 PR #15442 的合并才能正确渲染

### 3. 真实硬件测试反馈
- 用户 `sulix` 报告在 DOSBox 和真实硬件（Vortex86 板，DOS 6.22）上均能正常运行
- 非全屏窗口模式工作良好，全屏模式仍有黑屏问题待解决
- 计划在更多系统上进行扩展测试

## 构建方法

```bash
cmake -S. -Bbuild-dos \
  -DCMAKE_TOOLCHAIN_FILE=build-scripts/i586-pc-msdosdjgpp.cmake \
  -DCMAKE_BUILD_TYPE=Release
cmake --build build-dos -j$(nproc)
```
