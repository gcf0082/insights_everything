## 项目概述

该项目是对David Ng的RYS（Repeated Y-Scaling）方法的复现与扩展。核心发现是：**通过复制Transformer模型中的特定层块（无需训练、无需修改权重），可以使模型在特定能力上显著提升**。

### 核心理念

Transformer模型在训练过程中会自组织形成功能回路（functional circuits）——执行完整认知操作的多层处理单元。这些回路是不可分割的：复制单个层几乎无效，但复制正确的3-4层块可以为模型提供第二次推理管道处理。

## 主要实验结果

### 1. Devstral-Small-2-24B

- **复制层**: 12, 13, 14（重复一次）
- **硬件环境**: H200实例（Vast.ai）
- **测试结果**:

| 指标 | 基础模型 | 手术后模型 | 变化 |
|------|----------|------------|------|
| bbh/causal_judgement | 0.5775 | 0.6364 | +5.88% |
| gsm8k_cot (strict-match) | 0.8408 | 0.8704 | +2.96% |
| ifeval (prompt_level_strict_acc) | 0.6229 | 0.5767 | -4.62% |
| mbpp (pass_at_1) | 0.7000 | 0.6700 | -3.00% |

**结论**: 手术提升了数学推理和因果推理能力，但代价是指令跟随和代码生成能力下降。模型"思考"更多但精确遵循指令的能力下降。

### 2. Qwen2.5-Coder-32B

- **复制层**: 7, 8, 9（重复一次）
- **测试集**: 自定义探测套件（BBH衍生 + EQ-Bench风格 + GSM8K）

| 探测类型 | 基础模型 | +3层 | 变化 |
|----------|----------|------|------|
| 推理（因果+逻辑+导航） | 76.5% | 94.1% | +23% |
| EQ（情商） | 92.1% | 93.6% | +1.6% |

## "模式"发现

不同的复制模式可以从相同权重创建不同的认知配置文件：

| 模式 | 数学 | EQ | 特征 |
|------|------|-----|------|
| 双次通过 13-16 | ↑↑ | ↑ | 数学专家 |
| 三次通过 13-16 | ↑ | ↑↑ | EQ专家 |
| 交错 13,13,14,14,15,15,16 | ↑↑↑ | ↓ | 纯数学模式 |
| 四次通过 13-16 | — | ↑↑ | EQ模式，数学中立 |

## 技术实现

### 工具文件

| 文件 | 功能 |
|------|------|
| `sweep.py` | 主扫描工具 - 查找最优层复制配置 |
| `layer_path.py` | 构建具有显式层执行路径的GGUF |
| `gguf_surgery.py` | 低级GGUF层复制（被sweep.py使用） |
| `math_probe.py` | 硬算术探测（Ng的部分信用评分） |
| `eq_probe.py` | 情商探测（EQ-Bench风格） |
| `reasoning_probe.py` | BBH衍生的因果/逻辑/导航/数学应用题 |
| `compare_eval.py` | 比较lm-evaluation-harness结果 |
| `visualize.py` | 扫描结果的文本和PNG热力图 |

### 扫描工作流程

1. 对于每个层配置(i, j):
   - GGUF手术创建层i..j-1被物理复制的模型
   - 新的前向传递: `layers 0..j-1 → layers i..j-1再次 → layers j..N-1`
   - llama-server加载修改后的模型
   - 运行三个探测套件: 数学、EQ、推理
   - 与基线比较分数
   - 服务器终止，修改后的GGUF删除

2. 搜索策略:
   - **第一轮**: 大块（8层），宽步幅 → 找到热点区域
   - **第二轮**: 小块（3-5层），步幅1 → 找到精确边界
   - **第三轮**: 尝试多通道、交错和复合配置

## 关键技术细节

### 资源需求

- **额外VRAM**: 是的，复制的层是GGUF中的物理副本。24B模型上3个额外层预计需要约1.5 GiB。
- **推理速度**: 是的，与额外层数量成比例。40层模型上3个额外层 ≈ 慢7.5%。

### 模型差异

不同模型在不同位置有不同的回路：
- **Devstral-24B** (40层): 推理回路在层**12-14**
- **Qwen2.5-32B** (64层): 推理回路在层**7-9**

边界非常精确。在任一方向上移动一个层，改进就会消失或反转。

## 快速开始

### 查找模型中的回路

```bash
pip install gguf requests tqdm

python sweep.py \
    --model /path/to/model.gguf \
    --llama-server /path/to/llama-server \
    --tmpdir /dev/shm/rys \
    --results pass.jsonl \
    --block-sizes 3 4 5 \
    --stride 1 \
    --start-min 10 --start-max 20 \
    --skip-baseline \
    --port 8099 \
    --server-args --device Vulkan1,Vulkan2
```

### 应用已知回路

```bash
# 复制Devstral中的层12-14
python layer_path.py model.gguf improved.gguf \
    -p "0..14,12,13,14,15..39" -v

# 复制Qwen2.5-32B中的层7-9
python layer_path.py model.gguf improved.gguf \
    -p "0..9,7,8,9,10..63" -v
```

## FAQ

**Q: 这是否使用更多VRAM?**
A: 是的，复制的层是GGUF中的物理副本。24B模型上3个额外层预计需要约1.5 GiB额外VRAM。

**Q: 这会减慢推理速度吗?**
A: 是的，与额外层数量成比例。40层模型上3个额外层≈慢7.5%。

**Q: 这对我的模型有效吗?**
A: 可能有效。已在Mistral架构（Devstral）和Qwen2架构上测试。Ng的原始工作在Qwen2-72B上。所有Transformer模型中都存在回路——问题在于它们在哪里、有多大。

**Q: 为什么不进行微调呢?**
A: 这与微调是正交的。你可以同时做两者。实际上，Ng的RYS模型后来被其他人微调并登上了HuggingFace排行榜。层复制改变架构；微调改变权重。可以叠加使用。

## 总结

该项目展示了一种无需训练、无需修改权重即可提升LLM推理能力的创新方法。通过识别并复制Transformer模型中的特定"推理回路"，可以在数学推理、因果推理等方面获得显著提升（高达23%），同时保持相同的模型权重和磁盘占用。这种方法与微调正交，可以叠加使用以获得更好的效果。
