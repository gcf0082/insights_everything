## 主题摘要

本帖讨论了 **FIM (FBI Improved)** —— 一个 Linux framebuffer 图像查看器。该工具允许用户直接在 Linux framebuffer 上查看图像，无需运行 X11 或 Wayland 图形桌面环境。

## 主要讨论内容

### 1. Framebuffer 基础工具生态

- **FIM**: 主要讨论的图像查看器
- **mplayer -fp**: 可在 framebuffer 上播放视频
- **fbpdf2/fbgs**: 在 framebuffer 上查看 PDF
- **fbtv**: 观看电视
- **mpv --vo=gpu --gpu-context=drm**: 无桌面环境的硬件加速视频播放

### 2. 无桌面环境的使用场景

用户分享了数周不运行图形桌面环境的经验，使用以下工具组合：
- framebuffer 图像/视频查看器
- 文本浏览器 (links, lynx, elinks)
- LLM 抓取内容

### 3. 技术实现细节

- 设置 SDL 应用使用 framebuffer:
  ```
  export SDL_VIDEODRIVER=fbcon
  # 或
  export SDL_VIDEODRIVER=kmsdrm
  ```
- framebuffer 兼容广泛，从1990年代的显卡到小型的i2c显示器都支持
- kmsdrm 相比 fbdev 有更好的性能，但也需要更多基础设施

### 4. 其他有趣项目

- **ascsee**: 基于细胞自动机的终端图像查看器，可让图像以动画方式呈现
- **timg**: 支持 sixels 的图像查看器

## 结论

FIM 和相关 framebuffer 工具代表了一种极简主义的 Linux 使用方式，允许用户在完全无需图形桌面环境的情况下完成图像查看、视频播放甚至游戏等常见任务。这种方式对老旧硬件尤其有用，也受到追求简洁和低资源占用的用户青睐。
