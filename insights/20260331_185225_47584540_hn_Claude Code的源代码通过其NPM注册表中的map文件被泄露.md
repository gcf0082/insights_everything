## 事件概述

Claude Code 的源代码通过其 NPM 注册表中的 source map 文件被泄露。泄露的代码仓库位于 `github.com/chatgptprojects/claude-code`。

## 主要发现

### 1. 产品路线图泄露
- **Kairos**: 一个未发布的"助手模式"，通过功能标志（feature flags）暴露
- **Buddy System**: 类似于 Tamagotchi 的宠物伴侣系统，包含 ASCII 艺术精灵
- **Undercover Mode**: 用于员工开源贡献时strip所有 Anthropic 内部信息

### 2. 敏感功能发现
- 代码中存在一个正则表达式用于检测用户提示中的负面情绪词汇并进行日志记录
- 发现了名为 "capybara" 的相关代码

### 3. 代码质量讨论
- 代码被描述为"vibe coded"（直觉编程）
- 存在大量嵌套的 if-else 语句
- 重复代码多，如多个 ad-hoc 实现的哈希函数和 PRNG
- 几乎没有高层级注释

## 社区观点

### 关于泄露的影响
- 有观点认为这不是真正的"泄露"，而是像网站一样可以检查源代码
- 计算机生成的代码无法获得版权保护
- 真正的价值在于 Opus 模型权重，而非前端代码

### 关于开源
- Codex 和 Gemini CLI 已经开源
- 有人呼吁 Anthropic 开源 Claude Code
- 已有第三方仓库如 `github.com/instructkr/claude-code` 获得关注

### 关于安全与责任
- 使用订阅服务可能面临被检测和封禁的风险
- 通过 AWS Bedrock 使用 Claude 可能是替代方案

## 总结

此次泄露事件暴露了 Claude Code 的完整源代码，揭示了 Anthropic 未发布的产品功能和代码质量状况。社区对此次泄露的看法不一，有人认为是重大安全事件，有人则认为源代码客户端代码的价值有限。
