# 邮件地址混淆技术：2026年什么方法有效？

**洞察链接**：https://spencermortensen.com/articles/email-obfuscation/  
**更新时间**：2026年1月30日  
**报告日期**：2026年4月2日

---

## 摘要

本报告基于Spencer Mortensen的实证研究，总结了各种邮件地址混淆技术的有效性。研究通过蜜罐机制收集统计数据，测试了318个垃圾邮件发送者对纯文本邮件地址的保护效果，以及299个对可点击邮件链接的保护效果。

---

## 一、纯文本邮件地址混淆技术

以下技术保护以纯文本形式呈现的邮件地址（如"email@example.com"）。

### 1.1 无保护

- **屏蔽率**：0%（318个垃圾邮件发送者中）
- **结论**：完全无效，邮件地址会被立即收集

### 1.2 HTML实体编码

- **屏蔽率**：95%
- **实现方式**：`&#97;&#98;&#64;&#101;&#109;&#97;&#105;&#108;&#46;&#115;&#112;&#101;&#110;&#99;&#101;&#114;&#109;&#111;&#114;&#116;&#101;&#110;&#115;&#101;&#110;&#46;&#99;&#111;&#109;`
- **结论**：令人意外地有效，虽然服务器端库可以自动解码，但仍然阻止了大多数爬虫

### 1.3 HTML注释

- **屏蔽率**：98%
- **实现方式**：`ac@email.spencermortensen<!--.example-->.com`
- **结论**：可阻止大多数基本的HTML解析器

### 1.4 HTML SVG

- **屏蔽率**：100%
- **实现方式**：使用`<object>`元素引用SVG文件
- **结论**：将邮件地址隐藏在非常规位置，完全屏蔽所有爬虫
- **优点**：对视障用户完全可访问（使用屏幕阅读器）

### 1.5 CSS Display None

- **屏蔽率**：100%
- **实现方式**：`<div class="email">ad@<span>email.</span>spencermortensen.<span>example.</span>com</div>`配合`display: none`
- **结论**：大多数爬虫无法应用CSS规则，是最佳技术之一
- **注意**：必须使用`display: none`，不能使用视觉隐藏技术

### 1.6 JavaScript字符串拼接

- **屏蔽率**：100%
- **实现方式**：通过`document.write`拼接字符串
- **缺点**：邮件地址仍直接出现在HTML源代码中

### 1.7 JavaScript ROT18

- **屏蔽率**：100%
- **实现方式**：对邮件地址进行ROT18编码
- **结论**：可被不解析JavaScript的爬虫破解，建议使用非标准偏移量

### 1.8 JavaScript转换法（推荐）

- **屏蔽率**：100%
- **实现方式**：HTML中包含乱码，通过自定义函数转换为真实邮件地址
- **结论**：被认为是最有效的技术之一，简单但极其有效

### 1.9 JavaScript AES加密

- **屏蔽率**：100%
- **实现方式**：使用AES 256位加密
- **优点**：使用浏览器内置密码库（SubtleCrypto），安全可靠
- **限制**：必须在HTTPS环境下使用（localhost除外）

### 1.10 JavaScript用户交互

- **屏蔽率**：100%
- **实现方式**：邮件地址仅在用户与页面交互后才显示
- **结论**：大幅提高爬虫破解门槛，需要完整Web客户端和交互能力

### 1.11 符号替换（破坏可用性）

- **屏蔽率**：97%
- **示例**：`ag AT email DOT spencermortensen DOT com`
- **结论**：用户需要手动还原地址，严重影响可用性

### 1.12 说明文字（破坏可用性）

- **屏蔽率**：100%
- **示例**：`au.fluff@email.spencermortensen.com (remove the ".fluff" before writing to me)`
- **结论**：只有人类或AI能理解，严重影响可用性

### 1.13 图片形式（破坏可用性）

- **屏蔽率**：100%
- **结论**：视障用户无法使用，所有用户都需要手动输入

### 1.14 CSS Content属性（破坏可用性）

- **屏蔽率**：100%
- **结论**：邮件地址可见但不可复制，完全破坏可用性

### 1.15 CSS文本方向（破坏可用性）

- **屏蔽率**：100%
- **实现方式**：`unicode-bidi: bidi-override; direction: rtl;`
- **结论**：复制的文本是反向的，用户体验极差

## 二、可点击邮件链接混淆技术

以下技术保护点击后打开邮件客户端的链接（如`mailto:email@example.com`）。

### 2.1 无保护

- **屏蔽率**：0%（299个垃圾邮件发送者中）

### 2.2 HTML实体编码

- **屏蔽率**：100%
- **结论**：在链接保护中效果完美

### 2.3 URL编码

- **屏蔽率**：96%
- **结论**：服务器端库可轻松解码，效果较差

### 2.4 HTTP重定向（推荐）

- **屏蔽率**：100%
- **实现方式**：将`mailto:`链接转换为普通链接，通过服务器重定向
- **优点**：隐藏真实邮件地址在普通链接中
- **注意**：使用`nofollow, noindex`标签防止搜索引擎索引

### 2.5 HTML SVG

- **屏蔽率**：100%
- **结论**：与纯文本保护相同，完全有效

### 2.6 JavaScript拼接

- **屏蔽率**：100%
- **缺点**：源代码中仍包含完整地址

### 2.7 JavaScript ROT18

- **屏蔽率**：100%
- **结论**：可被基本爬虫破解

### 2.8 JavaScript转换法（推荐）

- **屏蔽率**：100%
- **结论**：最佳技术之一，简单有效

### 2.9 JavaScript AES加密

- **屏蔽率**：100%
- **限制**：需要HTTPS环境

### 2.10 JavaScript用户交互

- **屏蔽率**：100%
- **结论**：最高安全级别

## 三、关键发现

### 3.1 大多数爬虫容易被击败

每个混淆技术都可能被破解，但这并不意味着混淆无用。**大多数爬虫并不先进**，即使最简单的技术也能有效阻止它们。

### 3.2 爬虫跟踪链接而非页面

爬虫通常直接访问高流量页面，忽略低流量页面和链接。这导致人们可能错误地认为某些页面不需要保护——但页面可能突然走红。

## 四、最佳实践建议

1. **组合使用多种技术**：将邮件地址分段，每段使用不同的混淆技术
2. **推荐纯文本保护**：CSS Display None、SVG、JavaScript转换法或AES加密
3. **推荐链接保护**：HTTP重定向、JavaScript转换法或AES加密
4. **保持可用性**：避免使用破坏用户体验的技术（符号替换、图片等）
5. **使用HTTPS**：如使用AES加密，必须在安全上下文中运行

---

## 五、研究方法

本研究采用蜜罐机制：
- 每种技术保护一个真实的邮件地址
- 当该地址收到垃圾邮件时，说明对应的混淆技术被破解
- 作者使用自建邮件服务器以避免过滤，收集真实统计数据
- 对消息进行去重处理，仅跟踪爬虫已知的邮件地址

**注意**：样本量仍在增长，链接到本文的人越多，统计数据越准确。
