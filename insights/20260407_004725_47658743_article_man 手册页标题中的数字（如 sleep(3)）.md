# 洞察报告：Man 页面标题中的数字含义

**洞察链接**：https://lalitm.com/til-number-in-man-page-titles-e-g-sleep-3/

**基本信息**：

- **发布日期**：2025年11月17日
- **主题**：Linux man 页面标题中数字的含义（如 sleep(3)、read(2)）
- **标签**：TIL、Linux

---

## 洞察内容总结

### 问题背景

作者在使用 Linux 系统编程时经常查看 man 页面，但对标题中的数字（如 `sleep(3)`、`read(2)`）的含义感到好奇却从未深究。一次 PR 评审中的评论促使他查找答案：

> 代码注释：`// Behaves like man 2 basename`  
> 评审意见：`nit: it's not a syscall, so "man 2" is incorrect`

### 核心发现

根据 `man(1)` 手册，man 页面编号代表不同的内容类型：

| 编号 | 类型 |
|------|------|
| 1 | 可执行程序或 shell 命令 |
| 2 | 系统调用（内核提供的函数） |
| 3 | 库调用（程序库中的函数） |

因此，`basename(3)` 应该是 libc 库调用，而不是系统调用，正确的注释应为 `man 3 basename`。

### 扩展知识：后缀字母

数字后还可附加字母后缀：
- **p**：POSIX man 页面，如 `free(3p)`
- **x**：X 文档，如 `ncurses(3x)`
