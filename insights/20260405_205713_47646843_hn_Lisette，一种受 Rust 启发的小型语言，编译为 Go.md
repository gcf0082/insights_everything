## 项目概述

Lisette 是一个受 Rust 启发的小型编程语言，可以编译为 Go 代码。它的目标是将 Rust 的类型系统优势引入 Go，同时保留 Go 的运行时和生态系统。

## 主要特性

- **模式匹配 (Pattern Matching)**: 支持穷尽性检查
- **代数数据类型 (ADT)**: 支持 Rust 风格的枚举和联合类型
- **Option 类型**: 替代 Go 的 nil 指针
- **更好的类型系统**: 改进 Go 有限的类型系统
- **Go 互操作性**: 可以导入 Go 标准库

## 讨论要点

### 支持者的观点

1. **结合最佳特性**: 将 Rust 的安全性和 Go 的运行时结合
2. **错误消息优秀**: 编译器的错误提示非常有用
3. **适合现有 Go 项目**: 可以在现有 Go 代码库中逐步采用
4. **解决 Go 的痛点**: 提供了模式匹配、ADT 和结构体未初始化值等 Go 缺失的功能

### 批评者的观点

1. **语法差异**: 使用 `import "foo.bar"` 而非 `use foo::bar`，使用 `Bar.Baz` 而非 `Bar::Baz`
2. **typed nil 问题**: 从 Go 导入接口类型时需要 `Option<Option<http.Handler>>`
3. **不能消除 defer**: 无法像 RAII 那样处理资源清理
4. **为什么不用 Rust**: 既然可以直接用 WASM 运行 Rust，为什么还要让 Go 变得更像 Rust？

### 技术考量

- **Go 运行时的优势**: 高效的并发 GC，适合需要 GC 的场景
- **Rust 异步的复杂性**: 由于缺乏 GC，Rust 的异步故事不如 Go 人体工程学
- **已有语言竞争**: XGo, Borgo, Soppo 等也在做类似的事情

### 未来计划

根据作者在评论中的回复，支持第三方 Go 包的导入功能已在路线图上。

## 适用场景

1. 现有 Go 代码库，想用更好的语言特性
2. 需要使用 Go 运行时（如并发模型）但想要更强的类型系统
3. 渐进式迁移到更安全的语言
