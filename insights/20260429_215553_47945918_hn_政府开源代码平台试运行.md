## 摘要

荷兰政府宣布推出开源代码平台 code.overheid.nl，该平台基于 Forgejo 技术构建（与 Codeberg 使用相同技术）。这一举措标志着荷兰政府从 GitHub 向自托管开源平台的迁移，显示出其在数字主权和开源方面的努力。

## 主要讨论内容

### 1. 荷兰政府的开源举措

- **平台技术**：使用 Forgejo（开源 Git 仓库管理系统）
- **平台地址**：https://code.overheid.nl/
- **项目示例**：
  - Regelrecht：将法规编码为结构化 YAML，用于确定性决策逻辑
  - Partijgedrag：分析荷兰政党投票行为的数据项目

### 2. 欧洲开源对比

| 国家 | 开源项目数量 |
|------|-------------|
| 挪威 | 约3000+仓库 |
| 英国 | 约17000+项目 |
| 德国 | opencode.de |
| 荷兰 | 逐步增加中 |

评论中提到，荷兰、德国和比利时对开源软件较为支持。NLnet 每年为开源项目提供数千万欧元资金支持。

### 3. 政府技术依赖问题

#### 微软依赖情况：
- 荷兰市政当局大多数使用微软云服务（mxmap.nl 显示）
- 比利时72%的市政当局使用 Microsoft 365
- 挪威64%、瑞典57%、芬兰77%的市政当局依赖微软

#### DigiD 认证系统争议：
- DigiD 是荷兰政府的公民认证系统
- 当前托管服务商 Solvinity 正在被美国公司 Kyndryl 收购
- 引发数据主权和隐私担忧（美国 CLOUD/PATRIOT 法案）
- 荷兰议会已通过动议，要求在2028年前更换提供商

### 4. 技术实现细节

- 平台运行在开发版 Forgejo v16（非稳定版）
- 部分 UI 采用英文而内容为荷兰语，存在国际化问题
- Dark mode 存在可访问性问题（深紫色文字配深色背景）

## 社区观点

**积极观点**：
- 荷兰政府终于开始开源工作，值得肯定
- Forgejo 是良好的技术选择（与 Codeberg 相同技术栈）
- 小型国家在开源和数字独立方面领先

**批评观点**：
- 政府仍计划将公民认证系统交给美国公司
- code.overheid.nl 运行在住宅 IP 地址
- 微软云依赖仍然严重
