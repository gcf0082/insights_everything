## 核心洞察

### DSPy的现状与困境

DSPy每月下载量470万次，而LangChain达2.22亿次。尽管使用DSPy的公司（JetBlue、Databricks、VMware、Sephora、Replit等）都报告了显著收益，但采用率仍然较低。

**核心问题**：DSPy的抽象概念陌生，需要不同的思维方式。开发者当前只想快速解决痛点，不想重新学习。

作者提出了一个有趣的"Khattab定律"：
> 任何足够复杂的AI系统都会包含一个临时性的、非正式的、漏洞百出的DSPy半成品实现。

意思是：工程师最终都会自己实现这些模式，只是做得更差，耗时更长，过程更痛苦。

## AI系统的演进阶段

文章通过一个简单的公司名称提取任务，展示了每个团队如何最终实现自己的"DSPy家庭版"：

### 阶段1：直接发货
直接调用OpenAI API，编写简单prompt。

### 阶段2：无部署调整prompt
将prompt存入数据库，添加版本历史，避免每次修改都重新部署。

### 阶段3：结构化输出
使用Pydantic定义输出格式，确保返回结构化数据。

### 阶段4：失败处理
添加重试逻辑处理临时性错误（如529错误）。

### 阶段5：RAG检索
添加向量数据库检索，为模型提供上下文支持。

### 阶段6：评估体系
建立评估数据集和指标，量化系统改进效果。

### 阶段7：模型切换
重构代码以支持不同模型（GPT→Claude），最终发现"恭喜你，你刚刚构建了一个更差的DSPy版本"。

## DSPy的解决方案

使用DSPy实现相同功能：

```python
# 一行配置模型
lm = dspy.LM("openai/gpt-5.2")
dspy.configure(lm=lm)

# Signature定义类型化输入输出
class CompanyExtraction(dspy.Signature):
    text: str = dspy.InputField()
    company_name: str = dspy.OutputField()
    confidence: float = dspy.OutputField()

# Module组合签名形成管道
class CompanyExtractor(dspy.Module):
    def __init__(self):
        self.retrieve = dspy.Retrieve(k=5)
        self.extract = dspy.ChainOfThought(CompanyExtraction)

    def forward(self, text):
        context = self.retrieve(text).passages
        return self.extract(text=text, context=context)
```

切换模型只需一行：
```python
dspy.configure(lm=dspy.LM("anthropic/claude-sonnet-4-20250514"))
```

## DSPy的三大核心模式

| 模式 | 描述 |
|------|------|
| **Signatures** | 类型化输入输出，定义数据结构 |
| **Modules** | 可组合、可测试、可替换的单元 |
| **Optimizers** | 自动化改进prompt的逻辑，与执行逻辑分离 |

这些本质上是软件工程基础原则：关注点分离、可组合性、声明式接口。

---

## 为什么优秀工程师写出糟糕的AI代码

1. **奇怪的反馈循环**：无法单步调试prompt，输出具有概率性，成功后不敢再改动
2. **交付压力**：让LLM工作起来就像一种成就，干净的架构感觉是以后的事
3. **边界不清晰**：prompt既是代码又是数据，一切都不熟悉

## 建议

### 选项1：使用DSPy
接受学习曲线，阅读文档，构建一些示例项目直到理解抽象概念。

### 选项2：借鉴其理念
即使不使用DSPy，也要从第一天就采用其模式：

- ✓ 每次LLM调用都使用类型化I/O（Pydantic）
- ✓ 将prompt与代码分离
- ✓ 构建可组合单元，每个LLM调用可测试、可模拟、可链式调用
- ✓ **第一天就建立评估基础设施**
- ✓ 抽象化模型调用，使模型切换成为一行代码的改变

## 结论

DSPy之所以采用率不高，是因为它在开发者真正感受到痛苦之前，就要求他们以不同的方式思考。

DSPy所体现的模式并非可选项。如果AI系统足够复杂，你必然会重新发明这些模式。唯一的问题是你是主动去做还是被动应对。

**你不必使用DSPy，但应该像理解它为何存在那样去构建系统。**
