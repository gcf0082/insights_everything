## 摘要

本文作者通过宝可梦游戏讲解Prolog（逻辑编程语言）的基础知识。作者认为宝可梦对战系统本质上是一个极其复杂的规则引擎，非常适合用逻辑编程来建模。通过这个"有些愚蠢"的项目，作者终于理解了Prolog的核心理念，并从中获得了关于接口设计的宝贵经验。

## 一、引言：为什么选择宝可梦？

作者在阅读Bruce Tate的《七周七语言》后，一直渴望对Prolog有更深入的理解。宝可梦对战系统提供了完美的学习场景：

- 超过1000种宝可梦物种
- 每种宝可梦有独特的属性（types）、技能（moves）、特性（abilities）和基础属性（base stats）
- 属性之间存在复杂的相克关系（2x、0.5x、4x、0伤害等）
- 伤害计算涉及多种因素

## 二、宝可梦基础

### 2.1 核心机制

- **对战格式**: 6v6队伍对战
- **技能**: 每只宝可梦配备4个技能
- **属性**: 宝可梦可以拥有单属性或双属性
- **速度**: 决定谁先手；攻击/特攻影响物理/特殊技能伤害；防御/特防影响承受伤害

### 2.2 属性相克示例

- 火系技能对草系造成2倍伤害（草系弱火）
- 水系技能对草系只造成0.5倍伤害（草系抗水）
- Scizor（虫/钢）承受4倍火系伤害（因为虫和钢都弱火）
- 水/地面属性的Swampert完全免疫电系技能

## 三、Prolog基础

### 3.1 事实（Facts）与谓词（Predicates）

```prolog
pokemon(bulbasaur).
pokemon(ivysaur).
pokemon(venusaur).
```

- 谓词`pokemon/1`表示"是一个宝可梦"的关系
- 使用小写表示原子（atoms），大写表示变量

### 3.2 类型建模

```prolog
type(bulbasaur, grass).
type(bulbasaur, poison).
```

- 双属性宝可梦用两个`type`事实表示
- 类似SQL的一对多关系

### 3.3 查询示例

```prolog
?- type(squirtle, water).
true.

?- type(squirtle, grass).
false.

?- type(squirtle, Type).
Type = water.

?- type(venusaur, Type).
Type = grass ; Type = poison.
```

### 3.4 复合查询

使用逗号连接多个谓词：

```prolog
?- type(Pokemon, water), type(Pokemon, ice).
Pokemon = dewgong ; Pokemon = cloyster ; ...
```

### 3.5 规则（Rules）

```prolog
damaging_move(Move) :-
  move_category(Move, physical)
; move_category(Move, special).
```

规则由头部（head）和主体（body）组成，主体为真时头部 unify。

## 四、与SQL的对比

### 4.1 SQL查询示例

查找特攻大于120且学会冰冻光束的冰属性宝可梦：

```sql
SELECT DISTINCT pokemon, special_attack
FROM pokemon as p
WHERE p.special_attack > 120
  AND EXISTS (SELECT 1 FROM pokemon_moves WHERE ...)
  AND EXISTS (SELECT 1 FROM pokemon_types WHERE ...);
```

### 4.2 等价Prolog查询

```prolog
?- pokemon_spa(Pokemon, SpA),
   SpA #> 120,
   learns(Pokemon, freezedry),
   type(Pokemon, ice).
```

### 4.3 对比结论

- Prolog版本更加简洁、灵活、易读
- SQL查询随着条件增加会变得难以管理
- Prolog的查询模型和数据模型是同一个，非常优雅

## 五、进阶应用：选秀联盟工具

### 5.1 项目背景

作者参与宝可梦选秀联盟（Draft League），需要追踪复杂的游戏机制。

### 5.2 定义玩家队伍

```prolog
alex(meowscarada).
alex(weezinggalar).
alex(swampertmega).
...
```

### 5.3 优先度技能查询

宝可梦的速度决定先手顺序，但技能本身有优先度（priority）属性：

```prolog
learns_priority(Pokemon, Move, P) :-
  learns(Pokemon, Move),
  move_priority(Move, P),
  P #> 0.
```

### 5.4 排除双打技能

```prolog
doubles_move(helpinghand).
doubles_move(allyswitch).
...
```

### 5.5 添加特性判断

具有"恶作剧"特性的宝可梦，状态技能优先度+1：

```prolog
learns_priority(Mon, Move, Priority) :-
  learns(Mon, Move),
  move_priority(Move, BasePriority),
  ( pokemon_ability(Mon, prankster), move_category(Move, status) ->
      Priority #= BasePriority + 1
    ; Priority #= BasePriority
  ),
  Priority #> 0.
```

### 5.6 实用查询示例

查找Tornadus能学习的所有对Justin队伍有效的特效技能：

```prolog
?- justin(Target), learns(tornadus, Move),
   super_effective_move(Move, Target),
   move_category(Move, special).
```

## 六、与电子表格的对比

### 6.1 现有工具

宝可梦社区使用Google Sheets（如"Techno's Prep Doc"）来管理队伍信息，功能非常强大。

### 6.2 电子表格公式

```excel
={IFERROR(ARRAYFORMULA(VLOOKUP(FILTER(INDIRECT(Matchup!$S$3&"!$AV$4:$AV"),...)))}
```

### 6.3 两种方案对比

| 方面 | Prolog | 电子表格 |
|------|--------|----------|
| 可扩展性 | 任意技能查询 | 硬编码列表 |
| 查询灵活性 | 任意方向查询 | VLOOKUP限制 |
| 维护性 | 代码清晰 | 公式复杂 |

### 6.4 重要思考

> "一个非常重要的问题是：为什么这类问题的最佳维护解决方案是电子表格？"
>
> "我相信世界上还有很多类似的问题有待更好的编程范式来解决。"

## 八、结论

1. **逻辑编程的优势**: 对于描述关系型数据，Prolog是目前最简洁、最具表达力的编程系统
2. **声明式DSL的价值**: 在特定领域使用声明式领域特定语言可以大大简化问题
3. **工具设计**: 为非程序员构建工具时，电子表格仍是主流，但有很大的改进空间
4. **未来方向**: 作者计划构建Web应用来提供更好的宝可梦工具（如果能将Scryer Prolog编译为WASM的话）
