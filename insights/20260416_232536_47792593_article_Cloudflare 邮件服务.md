# 洞察报告：Cloudflare Email Service for Agents

## 基本信息

- **洞察链接**: https://blog.cloudflare.com/email-for-agents/
- **发布日期**: 2026-04-16
- **作者**: Thomas Gauvin, Eric Falcão
- **标签**: Agents Week, Agents, Developers, Developer Platform, AI, Workers AI, Email

## 核心洞察

### 1. 邮件成为代理的核心接口

邮件是世界上最易访问的界面，具有普遍性。无需自定义聊天应用或SDK，每个人都有邮件地址，可以与应用或代理交互。在私有测试期间，开发者构建了：客户支持代理、发票处理流程、账户验证流程、多代理工作流等。邮件正成为代理的核心接口。

### 2. Email Sending 公测

- 直接从 Workers 发送事务性邮件，使用原生 Workers 绑定，无需 API 密钥
- 支持 REST API 和 TypeScript、Python、Go SDK
- 自动配置 SPF、DKIM、DMARC 记录，确保邮件送达而非被标记为垃圾邮件
- 基于 Cloudflare 网络的全球服务，低延迟投递

### 3. Agents SDK 原生邮件支持

- 第一个类的 `onEmail` 钩子用于接收和处理入站邮件
- 每个代理从单一域获得自己的身份
- 基于地址的解析器将 support@yourdomain.com 路由到"support"代理实例
- 使用子地址路由到不同的代理命名空间和实例
- 状态跨邮件持久化，代理记住对话历史、联系信息
- 使用 HMAC-SHA256 签名的安全回复路由

### 4. 代理工具集成

- **MCP 服务器**: 通过 Cloudflare MCP 服务器发现和调用邮件端点
- **Wrangler CLI**: 解决 MCP 上下文窗口问题，零上下文开销
- **Skills**: 提供完整的配置和构建指导

### 5. 开源工具

- **Agentic Inbox**: 参考应用，具有完整对话线程、邮件渲染、收件存储、附件处理、自动回复
- 包含内置 MCP 服务器，外部代理可在发送前起草邮件供审核

## 主要功能

| 功能 | 描述 |
|------|------|
| Email Routing | 接收邮件到应用或代理 |
| Email Sending | 回复邮件或发送通知 |
| onEmail Hook | Agents SDK 原生邮件处理 |
| MCP Server | 代理访问邮件端点 |
| Wrangler CLI | 命令行发送邮件 |
| Agentic Inbox | 开源参考应用 |

## 总结

Cloudflare Email Service 成为双向通信的完整平台，使收件箱成为代理的一级接口。开发者可以构建客户支持代理、与客户在收件箱中会面，或构建后台流程实时更新团队。
