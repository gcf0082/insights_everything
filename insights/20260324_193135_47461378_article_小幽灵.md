## 项目概述

Ghostling 是一个演示项目，旨在展示基于 libghostty C API 构建的最小可行终端仿真器。项目名称取自 "Ghost"（幽灵）+ "hatching"（孵化），寓意"从最小处诞生无限"。

## 技术架构

### 核心技术栈

1. **libghostty-vt**: 零依赖的终端仿真库，负责 VT 序列解析、终端状态管理（光标位置、样式、文本重排、回滚等）以及渲染器状态管理
2. **Raylib**: 用于窗口管理和2D渲染
3. **CMake**: 构建系统
4. **Zig**: 0.15.x 版本

### 架构特点

- **单线程设计**: 虽然 libghostty-vt 支持多线程，但示例采用单线程实现
- **2D 图形渲染器**: 使用 Raylib 的 2D 渲染器，而非 Ghostty 主程序的直接 GPU 渲染器
- **零依赖**: libghostty-vt 本身不依赖 libc，甚至可以实现 WASM 兼容

## 功能特性

### 已实现功能

- 文本重排的窗口大小调整
- 完整的 24 位彩色和 256 色调色板支持
- 粗体、斜体和反色文本样式
- Unicode 和多码点字素处理
- 键盘输入及修饰键支持（Shift、Ctrl、Alt、Super）
- Kitty 键盘协议支持
- 鼠标追踪（X10、normal、button、any-event 模式）
- 鼠标报告格式（SGR、URxvt、UTF8、X10）
- 滚轮支持（视口回滚或转发到应用程序）
- 滚动条支持鼠标拖动滚动
- 焦点报告（CSI I / CSI O）

### 即将推出的功能

- Kitty 图形协议
- OSC 剪贴板支持
- OSC 标题设置

### 不会提供的功能

libghostty 专注于核心终端仿真功能，不提供以下 GUI 层面功能：
- 标签页
- 多窗口
- 窗口分屏
- 会话管理
- 配置文件或 GUI
- 搜索 UI

## 应用场景

libghostty 的设计目标是让任何应用都能嵌入正确、快速的终端仿真功能，可应用于：

1. **TUI 桌面化打包**: 将命令行工具包装成原生桌面应用
2. **嵌入式终端**: 在各种应用内嵌入终端功能
3. **WASM 兼容**: 可在浏览器和其他环境中运行

## 总结

Ghostling 展示了 libghostty 库的灵活性和强大功能，通过一个最小化的示例证明了 libghostty 可以与各种渲染器和窗口系统配合使用。该项目为开发者提供了一个轻量级的终端仿真解决方案，区别于传统的复杂终端仿真器实现。
