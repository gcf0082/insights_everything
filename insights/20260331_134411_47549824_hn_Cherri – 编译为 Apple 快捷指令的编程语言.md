## 洞察摘要

### 项目简介

Cherri 是一种编程语言，可以编译成 Apple Shortcuts（快捷指令）。该项目旨在解决 Apple Shortcuts 应用在复杂自动化场景下面临的维护困难问题。

### 核心用途

1. **文本化编程**: 允许开发者使用代码而非点击式 GUI 创建 Shortcuts
2. **版本控制**: 可以用 Git 进行版本管理
3. **LLM 集成**: Claude 等大语言模型可以学习该语言并编写可编译的 Shortcuts

### 实际应用案例

- 用户 alin23 使用 Cherri 构建了 200 个 Shortcuts，用于其 macOS 事件驱动自动化应用
- Claude 能够从零学会该语言并生成完整可编译的 Shortcuts
- 开发者可以使用 Shortcuts 完成一些其他自动化工具无法做到的操作：调整 Focus Mode、切换辅助功能如 Color Filters、访问 Private Cloud Compute 模型等

## 社区讨论热点

### 1. Apple Shortcuts 的痛点

| 痛点 | 描述 |
|------|------|
| 编辑器简陋 | 在小屏幕上通过无代码界面操作，缺乏真正的文本编辑器 |
| 无版本控制 | 无法使用 Git 进行版本管理 |
| 维护困难 | 复杂的 Shortcuts 几乎无法维护和改进 |
| 兼容性差 | Apple 经常更新导致功能失效，缺乏向后兼容性 |
| UI 动画缓慢 | 每次交互都有很长的动画延迟，难以进入心流状态 |
| 变量命名混乱 | 占位符变量没有实际名称，上下文容易被覆盖 |

### 2. 与其他工具的对比

- **AppleScript**: 老旧但仍被很多 Mac 应用支持，但处理文本编码等基础问题时会遇到麻烦
- **Hammerspoon**: 使用 Lua 和直接 macOS API，可以实现更深入的功能
- **Jelly/JellyCuts**: 曾存在但现已闭源并转向广告，Open-Jellycore 未能跟上最新 Shortcuts 操作
- **Scriptable**: JavaScript 自动化工具

### 3. Shortcuts 能做到而其他工具做不到的事

- 设置/获取 Focus Mode
- 更改系统设置如 Airdrop 接收、Color Filters、Background Sounds
- 一些应用导出的 Shortcuts 操作可以运行应用内代码

## 结论

Cherri 项目代表了将 Apple Shortcuts 从可视化编程转向文本化编程的趋势，这对于需要复杂自动化的 power user 来说是一个重要的进步。社区普遍认为，虽然 Shortcuts 功能强大，但其编辑体验对于专业开发者来说仍然不够友好。AI 时代可能会改变这一现状，使 Shortcuts 的创建和维护变得更加容易。
