## 核心观点

### 1. 设计工具的分化

作者认为设计工具将分叉成两种截然不同的形态：

- **Claude Design**：采用"truth to materials"（诚实于材料）原则，完全基于HTML和JS构建
- **另一种工具**：不期望任何代码，纯探索性质的工具

### 2. Figma 的困境

随着真相来源从设计文件回归到代码，Figma 陷入尴尬境地：

- 基本上变成手动、pre-agentic 的工具
- 仍然假装设计文件是权威的（canonical）
- 90% 的生命期只有两种图层效果（drop shadow 或 blur）

### 3. Claude Design 的优势

- **诚实性**：至少诚实于它是什么 — HTML 和 JS 到底层
- **与 Claude Code 的协同**：可以与 Claude Code 直接打通，形成设计与实现的单一反馈循环
- **Arts and Crafts 传统**：像 Gustav Stickley 的家具，榫卯外露，木材就是木材

### 4. 预测

- Figma 的 "Sketch 时刻" 正在迅速逼近
- 37signals 可能推出一款支持 Pencil 的 iPad 纯探索应用

## 总结

作者认为 Claude Design 标志着设计工具范式转变的开始 — 从"假装成某种东西"回归到"诚实于它是什么"。随着 AI 代理的兴起，设计与工程之间的界限正在模糊，单一对话将取代传统的多角色协作流程。
