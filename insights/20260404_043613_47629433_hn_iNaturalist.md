## 洞察摘要

iNaturalist 是一个公民科学平台，用户可以通过拍照和记录来识别和分享野生动植物观察。本次讨论主要围绕其 API、隐私问题和竞品展开。

## 主要讨论内容

### 1. API 优势

- **无需认证**：读取操作不需要认证
- **开放 CORS**：非常适合演示和教程
- **示例项目**：OwlsNearMe.com（猫头鹰定位网站）

### 2. Seek 应用问题

- 每次启动都会弹出"请勿打扰自然"的弹窗
- 该功能现已整合到 iNaturalist 主应用中

### 3. 隐私安全隐患

- 用户家庭地址在地图上公开可见
- 老年人用户可能不了解这一风险
- 建议：默认模糊位置数据，添加随机偏移
- 参考 Strava 的做法：隐藏家庭/工作地点的起点和终点

### 4. 替代应用推荐

| 应用 | 类型 | 特点 |
|------|------|------|
| Merlin Bird ID | 鸟类识别 | 音频识别 |
| Flora Incognita | 植物识别 | 准确率80-98% |
| Pl@ntNet | 植物识别 | 有API |
| BirdNet | 鸟类识别 |  Cornell开发 |

### 5. 技术亮点

- **计算机视觉模型**：基于社区验证数据训练，支持约76,000个物种分类
- **训练数据集**：已在GitHub公开，成为细粒度视觉分类研究的标准基准
- **地图性能**：使用服务端渲染+Elasticsearch+PNG瓦片，支持数百万标记
- **数据共享**：观测数据通过GBIF与全球共享

### 6. 批评意见

- 模型未公开，不符合"科学"精神
- 照片会被压缩并剥离元数据
- 桌面端应用缺失
