## 项目概述

GoModel 是一个用 Go 语言开发的开源 AI 网关，位于应用程序与模型提供商（如 OpenAI、Anthropic 等）之间。该项目由华沙的独立创始人 Jakub 自 2025 年 12 月起开始构建，并与少数贡献者合作开发。

## 核心功能

1. **追踪 AI 使用量和成本** - 按客户或团队维度统计
2. **模型切换** - 无需修改应用代码即可切换模型
3. **调试请求流程** - 可见且易于检查
4. **精确和语义缓存** - 减少 AI 开支

## 用户反馈与讨论要点

### 1. 架构设计获好评
- 用户 `hgo` 评价：设计紧凑，让人想起 Traefik
- 支持 User-Path 统一概念，可按最终用户、内部用户或服务追踪使用量

### 2. 语义缓存实现
- 通过请求嵌入和向量相似度查找实现
- 缓存失效机制：模型作为命名空间的一部分，TTL 处理清理

### 3. 统一 API 讨论
- 开发者询问：是否提供真正统一的 API，支持 temperature、reasoning effort、tool choice modes 等参数
- GoModel 采用 OpenAI 兼容 API，遵循 Postel 法则（灵活接收，严格发送）
- 开发者 `simonw` 指出：2025 年因各家引入推理机制导致 API 差异较大

### 4. 本地模型支持
- 可通过 vLLM（OpenAI 兼容 API）集成本地模型
- 示例配置已提供

### 5. 竞品对比
- 主要竞品：Bifrost（功能丰富但 Docker 镜像更大，部分功能需付费）
- GoModel 定位：更轻量、更简单、完全开源

### 6. 批评与质疑
- `lackoftactics` 指出：基准测试设计存在一些问题
- `goodkiwi` 吐槽 LiteLLM：全局导入实现糟糕
- 关于维护难度：`jedisct1` 表示最大的挑战是维护，因为 API 变化频繁

### 7. 未来规划
- 支持 AI 订阅（如 ChatGPT、GitHub Copilot）的潜力
- 考虑与 Vault 集成生成密钥

## 技术亮点

1. **轻量级**：~17MB Docker 镜像
2. **供应链安全**：Go 编译时明确控制供应链
3. **灵活性**：额外字段会被正确传递（Postel 法则）
4. **回退机制**：包含 passthrough API 直接转发请求
