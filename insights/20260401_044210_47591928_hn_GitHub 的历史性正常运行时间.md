## 报告摘要

本报告总结了Hacker News上关于GitHub历史正常运行时间数据的讨论。该数据来源于 damrnelson.github.io/github-historical-uptime/ 网站，展示了GitHub自创立以来的可用性记录。

## 主要讨论内容

### 1. 微软收购前后的可用性对比

多个用户指出，GitHub在微软收购后（2018年）的可用性明显下降。数据显示：
- 微软收购前：可用性接近100%
- 微软收购后：可用性显著下降，特别是最近90天仅为90.84%

### 2. 统计方法的争议

用户对"Missing GitHub Status Page"网站的统计方法提出质疑：
- 聚合统计使用的是"维恩图"而非平均值
- 如果任意服务故障，整个系统被视为不可用
- 部分用户认为应该按服务类型加权计算

### 3. GitHub Actions 是主要问题源

多个用户指出，GitHub Actions是导致可用性下降的主要原因：
- Actions 于2019年8月正式发布
- 发布后几乎每次重大故障都与Actions相关
- 部分用户建议在统计中排除Actions以获得更准确的数据

### 4. Azure 迁移的影响

有用户观察到GitHub宕机与Azure服务问题高度相关：
- 几乎每次GitHub宕机时，Azure也存在问题
- 这可能与GitHub迁移到Azure平台有关
- 用户抱怨Azure迁移过程漫长且问题频发

### 5. 产品复杂度增加

部分用户认为不能简单对比现在与过去：
- GitHub现在提供的产品比2018年前复杂得多
- Copilot等AI服务依赖OpenAI，存在额外的不可靠因素
- 用户规模增长100倍，产品表面积也相应增加

### 6. 自托管 vs GitHub

有用户戏言：
- "现在我自己的自托管服务都比GitHub更可靠"
- 建议如果在意可靠性，可以考虑自托管方案

Hacker News上的讨论普遍认为GitHub近年来的可用性有所下降，尽管原因可能有多种解释。核心问题似乎集中在GitHub Actions和Azure迁移上，但也有用户认为简单地对比现在和过去是不公平的，因为GitHub的产品组合已经大大扩展。
