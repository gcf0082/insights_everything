## 核心发现

### 性能测试结果

作者在文中展示了在 Windows Server 2025 上运行的测试结果：

- **DNS 解析**：Snapdragon 优于 Intel 约 **20%**
- **IIS 服务**：Snapdragon 优于 Intel 约 **50%**
- **其他测试**：Snapdragon 优于 Intel 约 **80%**

### 关键观察

1. **性能稳定性**：Snapdragon 系统在多次测试中表现出一致的结果，而 Intel 系统结果波动较大
2. **睿频策略**：与 Intel 积极 boost 和降频不同，ARM 提供稳定、持续的 Performance
3. **Segment Heap**：ARM64 上默认启用 Segment Heap，这是 Windows 在 ARM 上性能更好的重要原因之一

## 热门评论摘要

### 1. Segment Heap 的重要性

**kh9000** (5小时前, 58 points) 指出：

> Windows 有两种堆分配实现：较旧的 "NT heap" 和较新的 "segment heap"。Segment heap 因为更低的内存碎片化而更高效。由于 ARM64 二进制文件不会有传统遗留代码问题，所以 segment heap 在 ARM 上默认启用。

**如何启用 Segment Heap**：
- 通过应用清单：`<heapType>SegmentHeap</heapType>`
- 通过注册表：在 `HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Segment Heap` 创建值为 3 的 DWORD

**使用效果**：内存占用节省约 **15%**

### 2. 电源管理配置问题

**jiggawatts** (4小时前) 指出：

> 即使使用"高性能"电源计划笔记本电脑通常也有固件级别的限制。如果强制关闭散热保护，笔记本会过热损坏。实际上你在测试风扇而不是 CPU！

**jasoneckert** 回复：

> 两个 Windows 11 系统都配置了"高性能"电源计划，Windows Server VMs 也是如此。

### 3. 虚拟化开销

**c0l0** (1小时前) 提出疑问：

> Windows on ARM 是否使用 VBS（基于虚拟化的安全）？ARM 是否支持嵌套虚拟化？这些因素是否会影响 VM 性能？

### 4. ARM 服务器的实际应用

**SOLAR_FIELDS** (3小时前) 分享实际经验：

> 在云平台中，从 x86 迁移到 ARM 实例通常能节省约 **15%** 的成本。无需其他代码修改，CPU 密集且不依赖 x86 架构的 workload 受益更大。

### 5. Windows Server 的使用场景

**phwbikm** 提问：竟然还有人用 Windows Server？

**jayd16** 回复：

> 构建 Unreal 游戏、运行 Windows 容器。Windows Server 是真正的 Windows 专业版——稳定、没有垃圾功能。

**mopsi** 补充：

> Windows Server 是运行集中管理办公室的支柱：从证书颁发机构到部署 PC 镜像，从虚拟桌面到文件服务器，一切 integrated。

## 技术讨论要点

| 话题 | 要点 |
|------|------|
| **内存** | 两系统都使用 DDR5 和 NVMe SSD，Intel 系统使用更快的 Samsung SSD |
| **CPU** | Intel 使用第 14 代 Core i9 |
| **电源设置** | "High Performance" 模式将 CPU 锁定在最高性能状态，包括睿频频率 |
| **Segment Heap** | ARM64 默认启用，可通过清单或注册表在 x64 上启用 |
| **Linux 对比** | MSSQL 已在 Linux 上运行，Azure 大量使用 Linux |

## 结论

这篇 HN 讨论���示了 Windows Server 2025 在 ARM 处理器上相对于 Intel 的性能优势。关键因素包括：

1. **Segment Heap** 在 ARM64 上默认启用，减少内存碎片化
2. **ARM 更稳定的性能曲线**，不像 Intel 那样频繁升降频
3. **电源管理配置**对性能影响巨大

同时，社区也指出了测试方法的局限性：在笔记本电脑上测试服务器操作系统，并未控制所有变量（如内存带宽、SSD 性能等）。
