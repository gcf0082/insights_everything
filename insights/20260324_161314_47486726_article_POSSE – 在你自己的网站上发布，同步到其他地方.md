## 什么是 POSSE

POSSE 是 "Publish (on your) Own Site, Syndicate Elsewhere"（在自己的网站上发布，向其他地方同步）的缩写。它是一种内容发布实践，要求用户首先在自己的网站上发布内容，然后通过原始帖子链接将内容分享到第三方平台（如社交媒体平台）。

## 核心原则

### 为什么要使用 POSSE

1. **减少第三方依赖**: 在自己的网站上发布内容，不依赖第三方服务来发布内容
2. **保持数据所有权**: 首先在个人网站上发布内容，建立直接的所有权链
3. **拥有规范链接**: 内容的规范 URL 存在于用户自己的域名上
4. **更好的搜索**: 在自己的域名上搜索公开内容比依赖平台搜索效果更好
5. **利用社交功能**: 可以利用其他服务的社交层和聚合功能

### 为什么要链接到原始内容

- 使第三方服务上的内容可以被发现
- 防止垃圾邮件复制者意外帮助分发原始内容
- 有助于搜索引擎提高原始帖子的排名

## 实现方式

### 发布流程

两种主要的实现方式：

1. **客户端 → 网站 → 平台**: 用户通过发布客户端编写内容，发布到自己的服务器，然后服务器自动将内容同步到第三方平台
2. **客户端 → 网站 和 平台**: 用户发布到自己的网站后，客户端再将内容发布到第三方平台（需要手动步骤）

### 支持的平台

- **Twitter**: 最流行的 POSSE 目标平台
- **Facebook**: 手动或半自动发布
- **Medium**: 支持 API 发布
- **WordPress**: WordPress.com 账号
- **Mastodon**: 使用 Bridgy Publish
- **Bluesky**: 开源工具支持

### 常用工具

- **Bridgy Publish**: POSSE 即服务
- **SiloRider**: Python 命令行工具
- **Feed2Toot**: 将 RSS 内容发布到 Mastodon
- **POSSE Party**: 自托管软件

## 相关概念对比

| 模式 | 描述 |
|------|------|
| **POSSE** | 首先在自己的网站上发布，然后同步到其他平台 |
| **PESOS** | 首先在第三方平台发布，然后同步到个人网站 |
| **COPE** | 创建一次，发布到所有地方（缺少"首先发布到自己网站"这一步） |
| **PESETAS** | 类似 PESOS，但复制到特定平台 |

## 总结

POSSE 是 IndieWeb 运动的核心概念之一，它强调用户对自己的内容拥有所有权，同时能够利用社交平台的传播能力。这种方法让用户能够在保持对内容控制的同时，与使用不同平台的受众保持联系。
