## 项目概述

Craft 是一个轻量级的 C/C++ 构建工具和工作流工具，旨在消除 C 和 C++ 开发中繁琐的 CMake 配置和依赖管理问题。用户只需在一个简单的 `craft.toml` 文件中描述项目，Craft 就会自动生成 CMake 配置、管理依赖，并提供流畅的现代化命令行界面。

## 核心功能

### 1. 简洁的项目定义

只需在一个 `craft.toml` 文件中定义项目，Craft 会自动将其转换为可工作的 `CMakeLists.txt`。

```toml
[project]
name = "my_app"
version = "0.1.0"
language = "cpp"
cpp_standard = 17

[build]
type = "executable"
include_dirs = ["include"]
source_dirs = ["src"]
```

### 2. 自动化依赖管理

通过单一命令即可添加或移除依赖：

```bash
craft add --git https://github.com/raysan5/raylib.git --links raylib
craft add --path ../PhysicsEngine
craft remove raylib
craft update
```

### 3. 项目模板

支持保存和复用项目模板，快速创建新项目：

```bash
craft template save my_game_template
craft project new_game --template my_game_template
```

## 命令列表

| 命令 | 功能 |
|------|------|
| `craft project <path>` | 创建新项目 |
| `craft init [path]` | 初始化现有项目 |
| `craft build` | 构建项目 |
| `craft run [executable]` | 运行可执行文件 |
| `craft gen <file>` | 生成模板代码文件 |
| `craft clean` | 清理构建目录 |
| `craft add` | 添加依赖 |
| `craft remove <name>` | 移除依赖 |
| `craft update [name]` | 更新依赖 |
| `craft template <subcommand>` | 管理模板 |
| `craft config <subcommand>` | 全局配置 |
| `craft upgrade` | 升级 Craft 本身 |

## 使用场景

1. **简化 C/C++ 项目构建流程** - 无需手动编写 CMakeLists.txt
2. **自动化依赖管理** - 一行命令添加 Git 依赖
3. **标准化项目结构** - 通过模板快速创建项目
4. **迁移现有项目** - 自动检测并生成配置

## 技术亮点

- **单文件配置**: 所有项目信息集中在 `craft.toml`
- **自动 CMake 生成**: 用户无需接触 CMake 底层
- **智能依赖管理**: 自动克隆和管理 Git 依赖
- **模板系统**: 支持自定义和内置模板
- **目录遍历**: 支持从任意子目录运行命令

## 总结

Craft 为 C/C++ 开发者提供了一个现代化、简洁的构建工具，类似于 Rust 的 Cargo。它通过自动生成 CMake 配置和简化依赖管理，大大降低了 C/C++ 项目的构建门槛。对于不想深入 CMake 配置细节的开发者来说，Craft 是一个值得尝试的选择。
