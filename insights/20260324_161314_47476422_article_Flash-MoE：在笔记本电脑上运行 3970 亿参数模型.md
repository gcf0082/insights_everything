## 项目概述

Flash-MoE 是一个纯 C/Metal 推理引擎，能够在 MacBook Pro 上运行 **Qwen3.5-397B-A17B**（3970 亿参数的混合专家模型），达到 **4.4+ tokens/秒** 的速度，支持生产级输出包括工具调用功能。

整个 209GB 的模型通过自定义 Metal 计算管线从 SSD 流式传输。无 Python，无框架，仅使用 C、Objective-C 和手写的 Metal 着色器。

## 硬件配置

- **机器**：MacBook Pro，Apple M3 Max
- **芯片**：16 核 CPU（12P + 4E），40 核 GPU，16 核 ANE
- **内存**：48 GB 统一内存（~400 GB/s 带宽）
- **SSD**：1TB Apple Fabric，**17.5 GB/s 顺序读取**（实测）
- **macOS**：26.2 (Darwin 25.2.0)

## 架构特点

模型有 60 个 transformer 层：45 个 GatedDeltaNet（线性注意力）+ 15 个标准全注意力。每层 512 个专家，其中 K=4 个专家被激活（加一个共享专家）。隐藏维度为 4096。

### 核心技术

1. **SSD 专家流式传输** — 按需从 NVMe SSD 读取专家权重（4-bit 下 209GB），通过并行 `pread()` 和 GCD 调度组。每层仅加载 K=4 个活跃专家（各约 6.75MB）。OS 页面缓存管理缓存——无需自定义缓存（"信任 OS"原则）。

2. **FMA 优化反量化内核** — 4-bit 反量化矩阵向量乘法的内循环将数学重新排列为 `fma(nibble, scale*x, bias*x)`。预计算 `scale*x` 和 `bias*x` 让 GPU 融合乘加单元在一个指令中完成反量化和乘法。比朴素公式快 12%。

3. **Metal 计算着色器** — 手写 Metal 内核用于：
   - 4-bit 和 2-bit 反量化矩阵向量乘法
   - 融合 SwiGLU 激活
   - RMS 归一化
   - 批量 GPU 注意力
   - GPU RoPE
   - MoE 组合 + 残差 + sigmoid 门控

4. **延迟 GPU 专家计算** — CMD3（专家前向传递）提交后不等待。GPU 在 CPU 准备下一层时执行。

5. **使用 Accelerate BLAS 进行线性注意力** — GatedDeltaNet 循环使用 `cblas_sscal`、`cblas_sgemv` 和 `cblas_sger`。

6. **信任 OS** — 无自定义专家缓存。OS 页面缓存管理专家数据缓存。自然达到约 71% 命中率。

## 实验结果总结

### 保留的技术

| 方法 | 结果 | 影响 |
|------|------|------|
| FMA 反量化内核 | GPU 计算 -12% | **+12% tok/s** |
| 信任 OS 页面缓存 | 删除 Metal LRU → +38% | **基础性** |
| GPU combine+norm in CMD3 | 消除 CPU 往返 | **管线** |
| BLAS delta-net (Accelerate) | cpu_attn 0.78→0.28ms | **+64% attn** |
| C BPE 分词器 | 180ms vs 3500ms 启动 | **20x 启动** |

### 放弃的技术（58 个实验）

| 方法 | 结果 | 原因 |
|------|------|------|
| LZ4 专家压缩 | -13% | 解压开销 > 热缓存节省 |
| F_RDADVISE 预取 | 净 0% | 统一内存：SSD DMA 减慢 GPU |
| 时间专家预测 | -18% | 25% 命中率，SSD 带宽浪费 |
| MLP 路由预测器 | 31% 准确率 | 比时间基线更差 |
| GPU LUT 反量化内核 | -2% | 间接寄存器访问串行化 |
| GPU 私有缓冲区压缩 | -20% 管线 | Blit 成本 4×7MB > 矩阵向量节省 |
| spin-poll GPU 等待 | -23% | CPU 热竞争 GPU |
| 专家文件聚类 | 0% | NVMe 在 7MB 粒度忽略散射 |
| dispatch_io | -70% | dispatch_data 管理开销 |
| mmap 专家文件 | -5x | 冷数据每页错误开销 |
| 推测性早期路由 | -38% | 缓存污染 + 开销 |

## 安全特性

这是主要开发机器。引擎显式控制内存：
- 非专家权重：5.5GB（mmap'd，只读）
- Metal 暂存缓冲区：~200MB
- 总计：~6GB，保留 42GB 用于 OS + 页面缓存
- 无 OOM 风险。专家数据按需从 SSD 流式传输。
- 无自定义缓存。信任 OS。

## 总结

Flash-MoE 是一个令人印象深刻的技术演示，展示了如何在消费级硬件上运行超大规模语言模型。通过巧妙的工程设计——利用 Apple Silicon 的统一内存架构、SSD 流式传输、Metal GPU 加速——实现了在笔记本电脑上运行 3970 亿参数模型的能力。该项目包含 58 个实验的详细记录，为研究高效 LLM 推理提供了宝贵的参考资料。
