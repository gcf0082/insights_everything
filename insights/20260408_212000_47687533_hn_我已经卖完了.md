## 内容摘要

本文是 Mario Zechner（网名 badlogic）发布的博客文章，宣布他将加入 Earendil 公司，并将他开发的 pi coding agent（编码代理工具）一并带入该公司。

### 背景介绍

Mario Zechner 是知名的开源开发者，曾创建：
- **libGDX**：跨平台游戏开发框架，2011年是Android最流行的游戏开发框架之一
- **pi**：一个极简的编码代理工具（类似 Claude Code，但更简洁优雅）

### 出售原因

1. **家庭因素**：Mario 有一个四岁的孩子，过去两个月孩子经常因"爸爸不在家"而哭泣。他不想再经历这种生活，希望有更多时间陪伴家人。

2. **商业化困境**：VC资助的初创公司需要找到产品市场匹配、建立公司文化、处理人际关系等，这些与他陪伴孩子的愿望相冲突。

3. **避免重蹈覆辙**：他曾经历过 RoboVM 被出售给 Xamarin 后关闭源代码的糟糕经历，不希望类似事件再次发生。

### 关于 Earendil

Earendil 是由以下人创建的公司：
- **Armin Ronacher**（Flask 框架创始人）
- **Peter Steinberger**（OpenClaw 创建者）
- **Colin**

尽管公司名称来源于《指环王》，Mario 强调 Earendil 不是具有法西斯倾向的科技公司，而是"充满善意的嬉皮士"，认为软件应该服务人类而非取代人类。

### pi 的未来安排

1. **代码迁移**：GitHub 仓库从 `badlogic/pi-mono` 迁移至 `earendil-works/pi`
2. **包名变更**：从 `@mariozechner/pi-coding-agent` 变为 `@earendil/pi`
3. **许可证**：保持 MIT 许可证，未来可能引入 Fair Source 许可证
4. **治理结构**：pi 现在由 Earendil 所有，Mario 作为股东负责所有 pi 决策
5. **开源贡献**：外部贡献流程保持不变，无需 CLA 或 DCO

## 社区反应

### 主要讨论话题

1. **Pi vs Claude Code 对比**
   - Pi 被描述为更极简、更优雅的编码工具
   - 系统提示词仅 1500 tokens，只有 read、edit、bash 三个工具
   - 有用户认为 Claude Code 是"vibe-coded bloat"（氛围编程的臃肿产物）

2. **对"出售"的看法**
   - 部分用户认为这是正确的选择
   - 有人担心开源项目商业化后会走 RoboVM 的老路
   - 也有人认为可以 fork Pi 来规避风险

3. **Tolkien 命名争议**
   - Earendil 名字来源于《指环王》
   - 有人将其与 Palantir（知名科技公司）联想，产生警惕
   - 也有人认为不应让纳粹夺走托尔金的命名权

4. **技术细节讨论**
   - 讨论 Anthropic 封禁其他 harness 的问题
   - 用户考虑在迁移前 fork 仓库

### 正面观点

- Armin Ronacher 有成功的开源商业化经验（Flask、Sentry）
- Pi 保持 MIT 许可证，用户可以随时 fork
- 团队成员大多有孩子，公司会对家庭友好
- Mario 和 Armin 有 14 年的友谊基础

### 负面观点

- 文章太长且难以理解
- 公司产品方向不明确
- 担心走 RoboVM 的老路
- 有人因此决定停止使用 Pi

## 技术背景

### Pi 是什么？

Pi 是一个极简的 AI 编码代理框架，具有以下特点：
- 极小的系统提示词（1500 tokens）
- 简洁的工具集（read、edit、bash）
- 可配合多种模型使用（如 Qwen3.5-35B）
- 支持任意分支的 git 操作

### OpenClaw 事件

OpenClaw 是基于 Pi 构建的 AI 编程工具。Armin Ronacher 公开介绍了 Pi 在 OpenClaw 中的作用，导致更多人联系 Mario Zechner，间接促成了这次收购。

## 总结

这是开源社区一个典型的"出售"案例。Mario Zechner 选择加入已有团队而非独立创业，既保持了 Pi 的开源性质，又为项目提供了商业支持。社区反应不一，但多数人认可 MIT 许可证提供的保护，并准备好在必要时 fork 代码。
