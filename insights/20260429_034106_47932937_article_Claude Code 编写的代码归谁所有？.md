## 一、核心结论

如果你本周交付了代码，其中部分可能由 AI 完成。谁合法拥有这些代码的问题比大多数开发者想象的更加复杂。答案取决于三个与代码质量无关的因素：

1. **你的就业状况**（雇佣合同中的 IP 转让条款）
2. **你使用的工具的许可方式**（免费版 vs 商业版）
3. **你贡献的人类创意程度**

---

## 二、雇佣关系与工作成果归属

### 2.1 雇佣作品 doctrine（Work-for-Hire Doctrine）

你的雇佣合同几乎肯定规定，你在工作期间构建的任何东西都属于雇主。版权法中的"雇佣作品"原则支持这一观点：

- 任何在雇佣范围内由员工创建的代码，其所有权归雇主
- 无论代码是手写、由 Claude Code 生成，还是两者的组合
- 在工作时间、工作项目、工作机器上使用 AI 编码工具，不会改变结果的所有权

### 2.2 IP 转让条款需重点关注

大多数雇佣合同超出了 doctrine 的默认范围。应查找以下部分：

- "Intellectual Property"（知识产权）
- "IP Assignment"（知识产权转让）
- "Work Product"（工作成果）

以下条款几乎肯定涵盖你使用 AI 辅助编写的代码：

- "任何在雇佣期间创造的发明、设计、作品"
- "任何利用公司资源创造的知识产权"
- "任何与公司业务相关的创造"

**特别需要警惕的是第三条**：如果你的雇主为团队许可了 Claude Code、Cursor 或 Copilot，然后你用同样的工具构建副业项目，即使你在自己的时间完成，广泛 IP 转让条款也可能使雇主对该项目拥有权利。

## 三、AI 生成代码的版权问题

### 3.1 版权保护的前提

根据美国版权局和 2026 年欧盟法规：

- **100% 纯 AI 生成的代码缺乏版权保护**
- 版权保护不会因使用 AI 而消失，但需要证明**可证明的人类创意贡献**
- 仅仅复制粘贴 Claude Code 的输出是不够的

### 3.2 DABUS 案例的先例

DABUS（AI 作为发明者）确立了关键先例：

- 美国、欧盟和英国的专利局都拒绝承认 AI 为作者
- 这一逻辑延伸至版权：只有人类才能符合资格

### 3.3 Anthropic 的争议

- Anthropic 的首席工程师公开表示，他对 Claude Code 的贡献**100% 由 AI 编写**
- Anthropic 泄露的代码库（他们发出 8,000 份 DMCA 删除通知来压制）可能主要是 AI 编写的
- Anthropic 对该代码库的版权主张是否合法有效，仍是**未经法院解决**的开放问题

## 四、Claude Code 泄露事件与法律矛盾

### 4.1 事件回顾

2026 年 3 月 31 日，Anthropic 意外将 Claude Code 的完整源代码发布到公共 npm 注册表，暴露了：

- 1,906 个文件中的 512,000 行 TypeScript
- 内部系统提示逻辑
- 44 个完全构建但未发布的功能标志
- Hooks 和 MCP 服务器集成的编排架构

### 4.2 关键矛盾

泄露的源代码中有一个名为 `undercover.ts` 的文件实现了隐形模式：

- 当 Anthropic 员工使用 Claude Code 为外部开源仓库做出贡献时
- 系统指示 AI **不要添加共同作者署名**
- **不要在提交消息中提及 Claude 或 Anthropic**

这意味着 Anthropic 一方面通过 DMCA 对 Claude Code 的源代码主张版权，另一方面系统性地抹去 AI 参与的证据——而这些证据本可成为法院评估人类与 AI 贡献比例的直接材料。

### 4.3 版权悖论

当 Codex（从 Claude Code 的源代码"学习"并生成功能等效的 Python 版本）时，Anthropic 使用 DMCA 删除相关仓库。

Layer5 的分析捕捉到了核心矛盾：

- 如果"AI 从受版权保护的作品学习并生成新输出"属于合理使用（Anthropic 在训练数据诉讼中的立场）
- 那么"Codex 从 Claude Code 学习并生成 Python 重写"在逻辑上应该适用相同原则

## 五、商业用户的权利

### 5.1 免费版 vs 商业版

| 版本类型 | 代码所有权 | 用途限制 |
|---------|-----------|---------|
| 免费消费者版 | 仅限"内部、非商业使用" | 不得直接销售主要由 AI 生成代码的产品 |
| 付费商业版 | **用户拥有输出成果** | 可自由使用和商业化 |

### 5.2 商业条款的关键保障

对于付费客户（尤其是通过 AWS Bedrock 等商业 API 使用）：

- Anthropic 在服务条款中明确：**客户拥有所有 Outputs（输出）**
- Anthropic 放弃对客户内容的所有权利
- Anthropic 承诺不对客户的输入或输出进行训练

例如，Anthropic 在 Bedrock（AWS）上的商业服务条款写道：

> "在各方之间且在适用法律允许的范围内，Anthropic 同意客户拥有所有 Outputs，并放弃其根据本条款获得的任何客户内容的权利。"

## 六、实用建议

### 6.1 开发者应采取的行动

1. **检查你的雇佣合同**：查找"知识产权"、"IP 转让"或"工作成果"条款
2. **记录人类贡献**：保留设计决策、代码审查、重构的证据
3. **了解你的工具许可**：免费版和商业版的权利大不相同
4. **谨慎处理副业项目**：使用雇主许可的工具构建个人项目可能带来风险

### 6.2 企业的 IP 策略

- **不能仅依赖代码作为竞争优势**：保护应在于模型权重、数据、用户群、基础设施
- **约 1.6% 的 Claude Code 泄露代码库直接涉及 AI 模型**：其余是编排、上下文管理和工具协调
- 编排模式现在是公开知识，但背后的模型不是

### 6.3 代码保护的关键

- 代码保护不在于代码本身
- 可保护与不可保护代码的区别在于**如何记录人类干预**

## 七、总结

AI 生成代码的法律所有权是一个尚未解决的领域。以下是关键要点：

| 场景 | 所有权归属 |
|------|-----------|
| 员工在工作范围内使用 AI 工具 | 雇主（根据雇佣作品 doctrine） |
| 员工使用雇主许可的工具做副业 | 可能归属雇主（取决于 IP 条款） |
| 纯 AI 生成的代码（无人类贡献） | 无版权保护，进入公共领域 |
| 付费商业用户使用 Claude Code | 用户拥有输出成果 |

**核心建议**：使用 Claude Code 和类似工具并不违法，但没有知识产权战略的盲目使用是你的公司无法承担的风险。
