## 摘要

2024年6月，Fisker Inc. 申请破产保护，留下约11,000名 Ocean SUV 车主。这些价值40,000至70,000美元的车辆因制造商倒闭而面临失去软件支持的风险。然而，这些车主没有放弃，而是自发组织起来，反向工程专有软件，建立开源工具，实际上从 Fisker 的废墟中建立了一个志愿者运营的开源汽车公司。

---

## 背景

Fisker Inc. 曾被视为特斯拉的竞争对手，获得了超过31,000份 Ocean 订单，总潜在收入达17亿美元。该公司仅生产了11,000辆汽车便耗尽资金破产，债务超过10亿美元。

Fisker Ocean 是一款"软件驱动型汽车"，几乎所有子系统（刹车、安全气囊、换挡、电池管理、车门锁）都需要定期连接 Fisker 的云服务器进行诊断或常规操作。当这些服务器关闭时，车辆不仅失去信息娱乐屏幕，还失去关键功能。

---

## 车主组织的行动

### Fisker 车主协会 (FOA)

破产后几个月内，数千名 Ocean 车主成立了 Fisker 车主协会（Fisker Owners Association, FOA）——一个非营利组织，迅速发展到4,000名成员，开始运作介于汽车俱乐部、科技初创公司和独立汽车制造商之间的实体。

### 主要成就

FOA 取得了一系列显著成就：
- **反向工程软件**: 聘请独立技术专家反向工程 Fisker 的专有软件补丁
- **团体采购**: 组织批量采购替换零件，将钥匙扣价格从每个约1,000美元降至一小部分
- **免费钥匙配对活动**: 在全球举办免费钥匙配对活动，为每位车主节省100至250美元
- **欧洲"飞行医生"计划**: 创建移动维修网络，技术熟练的成员帮助其他车主保持车辆运行
- **安全召回**: 推动将安全召回纳入破产程序
- **零件供应**: 通过 Tsunami/Tidal Wave 等公司建立零件供应渠道
- **保险维持**: 说服几家保险公司为不再有制造商的车辆维持保险覆盖

## 开源技术生态

### GitHub 项目

- **Home Assistant 集成**: 开发者 MichaelOE 反向工程了 Fisker 官方"My Fisker"移动应用背后的 API，构建了 Home Assistant 集成，暴露所有云 API 值作为传感器。该项目有135次提交、20个发布版本，采用 Apache 2.0 许可证。
- **CAN 总线文件**: CAN 总线文件已发布在 GitHub 上，包括用于 CAN 查看器过滤和处理的 DBC 文件。Ocean 运行多个 CAN 总线——CCAN、PTCAN、Inverter CAN 和 BCAN，均为500kbps。
- **诊断教程**: Majd Srour 在 Medium 上发布了一个多部分系列，介绍如何嗅探 CAN 流量和解码诊断故障代码（DTC）。

### 社区讨论

在 CH4RGE 和 Ocean Forums 等社区论坛上，车主们讨论是否可能完全开源 Ocean OS。共识是混合的——核心汽车软件由 Magna 和其他供应商开发，意味着安全关键系统不能像网络应用程序一样简单分叉。但信息娱乐层、连接栈和诊断都是可行的，社区已将精力集中在这些领域。

---

## 失败的交易

2024年10月，Fisker 剩余库存出售给 American Lease 公司，该交易包括访问 Fisker 的专有源代码和云服务。American Lease 额外花费250万美元获取软件访问权限，并同意通过与 FOA 的协议向私人 Ocean 车主延长连接服务。

问题是：**这笔交易从未正式签署**，只是基于握手协议。当 American Lease 要求 FOA 支付所有运营成本的58%——LTE 连接、Microsoft Cloud 服务和软件工具时，关系破裂。FOA 想要详细的发票，以便成员确切知道他们支付了什么，但 American Lease 拒绝了。结果：Ocean 车主的远程连接被撤销，云功能被切断，待定的软件召回被阻止。

## 行业影响与未来展望

### 为何这将再次发生

Fisker Ocean Saga 并非孤立事件。Nikola 也申请了破产，使车主陷入类似困境。Canoo 和 Arrival 正在进行清算拍卖。分析师预计，随着行业整合和投资者对资本密集型制造业的热情降温，更多 EV 初创公司将会倒闭。

### 改革呼声

消费者倡导者正在推动结构性变革：
- **强制软件托管基金**: 即使制造商消失，也能保持车辆软件运行
- **开源强制令**: 破产程序中的开源规定
- **共享维修数据要求**: 允许独立维修
- **右 repair 法案**: 俄勒冈州的右 repair 法案已经禁止"零件配对"，使 Ocean 等车辆的独立维修变得困难

### 欧洲方向

欧洲汽车制造商采取了不同方向——大众、宝马、奔驰和八家供应商于2025年签署了一份备忘录，开发共享开源汽车软件平台。

## 结论

Fisker Ocean 车主所完成的工作令人钦佩，但迫使他们这样做的系统存在严重问题。以太坊联合创始人 Vitalik Buterin 和数字权利活动家 Cory Doctorow 都正确指出汽车行业需要更多开源思维。

Fisker 的故事表明：对于任何依赖云连接才能正常运行的车辆，都需要强制软件托管和开源后备条款。如果制造商死亡，软件应该向公众发布。

Fisker 车主协会证明了专门的社区可以让被遗弃的电动汽车继续在道路上行驶。但他们本不应该需要这样做。下一次 EV 初创公司倒闭时——还会有下一次——车主不应该需要成为黑客、零件经销商和准制造商，才能继续驾驶他们已经付费购买的汽车。
