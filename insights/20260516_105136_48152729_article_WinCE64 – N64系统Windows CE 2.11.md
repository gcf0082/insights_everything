## 项目简介

WinCE64 是一个令人惊叹的黑客项目：将微软 Windows CE 2.11 操作系统运行在真实的 Nintendo 64 游戏机上。这是一个完全未经微软官方支持的移植项目，所有的硬件抽象层（HAL）、OEM 适配层（OAL）、显示驱动程序、文件系统、输入设备驱动和音频驱动均由项目作者从头编写。

项目作者通过自定义的 HAL，将未经修改的 `nk.lib` 内核加载到 N64 的 VR4300 处理器上，成功启动了 CE 2.11 的 GWES 桌面和 shell，并实现了通过 N64 手柄作为鼠标操作、SD 卡挂载、音频输出等核心功能。

## 技术架构

该项目采用了分层架构设计：

1. **内核层**：使用微软官方的 `nk.lib`，这是 CE 2.11 的核心内核，负责 PSL 分派、调度和 TLB 管理
2. **系统模块层**：加载标准 CE 2.11 用户态模块，包括 coredll.dll、gwes.exe、filesys.exe、device.exe、shell.exe
3. **硬件抽象层 (HAL/OAL)**：位于 `bsp/hal/`，负责启动、异常向量、MIPS 启动、定时器、USB 调试等
4. **驱动程序层**：
   - 显示驱动：VI 帧缓冲区 + RDP 填充加速 + 软件光标合成器
   - 鼠标/键盘 PDD：SI Joybus 轮询，支持 N64 手柄和官方 N64 鼠标
   - SD 文件系统：基于 FatFS 的 FSD，通过 PI-DMA 驱动 EverDrive-X7
   - Wave 音频驱动：轮询模式的 AI 驱动
5. **Shell 层**：Win9x 风格的桌面、任务栏、文件浏览器

## 功能特性

该项目已在真实的 N64 + EverDrive-64 X7 上完全启动，以下功能正常工作：

- 桌面、任务栏、文件浏览器；窗口拖动、关闭按钮、模态对话框
- N64 手柄驱动可见光标；A 键 = 左键点击，B 键 = 右键点击
- 通过 FatFS 将 EverDrive-64 X7 的 SD 卡挂载到 `\SDCard`
- 通过 `sndPlaySoundW` / `waveOutOpen` 实现 Wave 音频输出到 N64 AI 硬件
- 可直接从 SD 卡启动第三方 CE 2.11 EXE 程序
- RDP 加速的 3D 演示程序

## 构建与运行

### 构建要求

项目不附带任何微软或任天堂专有组件，需要用户自行提供以下外部依赖：

1. **wince211_sdk/**：微软 Windows CE 2.11 Platform Builder / Embedded Toolkit（已停产，需自行从归档中获取）
2. **libdragon/**：N64 家用开发工具链（mips64-elf-gcc、n64.mk）+ FatFS 源码
3. **EverDrive-64 X7**：用于实际硬件部署的 N64 存储卡

### 构建步骤

执行以下命令进行构建：

```bash
bash bsp/build.sh
```

这会编译 HAL 和所有驱动 DLL，运行 LINK.EXE 和 ROMIMAGE.EXE 生成 `bsp/build/nk.bin`，然后与 bootloader 打包生成约 3.5MB 的 `bootloader/n64ce.z64` 文件。

### 运行方式

1. **真实硬件**：使用 USB 上传工具将固件刷入 EverDrive-64 X7
2. **模拟器**：作者使用 Ares 进行早期开发（需要针对 LE 模式 RDRAM 奇怪问题进行修补）
