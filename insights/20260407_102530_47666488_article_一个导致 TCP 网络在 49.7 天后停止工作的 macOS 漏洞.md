# 洞察报告：macOS TCP 网络定时炸弹

**洞察链接**: https://photon.codes/blog/we-found-a-ticking-time-bomb-in-macos-tcp-networking  
**来源**: Photon Blog  
**发布日期**: 2026年4月6日  
**Hacker News 讨论**: https://news.ycombinator.com/item?id=47666488

## 1. 漏洞概述

研究团队发现 macOS 内核存在一个经典的整数溢出漏洞，会导致 TCP 网络功能在系统运行约 **49.7 天** 后完全失效。该漏洞被称为"定时炸弹"，因为它会在特定时间周期后"引爆"，导致所有 TCP 连接无法正常工作。

## 2. 漏洞详情

### 2.1 根本原因

macOS 内核使用一个 32 位无符号整数 (`uint32_t`) 来存储 `tcp_now` 计数器，该计数器以毫秒为单位记录系统启动以来的时间。

- **最大值**: 4,294,967,296 毫秒 (约 49.7 天)
- **溢出阈值**: 当 `tcp_now` 达到 `2^32 - 30000` 时，溢出发生

### 2.2 技术细节

```c
tcp_now = 4,294,960,000  // 溢出前的冻结值
timer = tcp_now + 30,000 = 4,294,990,000
// 超出 uint32 max 后回绕到一个较小的数值
// TSTMP_GEQ(4294960000, small_number) 始终返回 false
// 导致 TIME_WAIT 连接永远无法被清理
```

### 2.3 影响

1. **TIME_WAIT 连接泄漏**: 已关闭的 TCP 连接停留在 TIME_WAIT 状态，无法被释放
2. **新连接无法建立**: 系统达到连接上限后，所有新 TCP 连接失败
3. **唯一解决方案**: 重启系统

## 3. 受影响版本

- **仅影响 macOS 26** (2025年发布)
- 旧版本 macOS 不受此漏洞影响
- 该 bug 是在 macOS 26 中新引入的

## 4. 触发条件

1. 系统需持续运行约 49.7 天以上
2. 系统期间不能进入睡眠状态（睡眠会暂停计时器）
3. 活跃的 TCP 流量会持续更新 `tcp_now` 值

## 5. 规避方案

1. **定期重启**: 建议每 30 天重启一次 Mac
2. **利用睡眠**: 笔记本电脑定期合盖睡眠可重置计时器
3. **监控连接**: 使用 `netstat -an | grep TIME_WAIT` 监控连接状态

## 6. 类似历史案例

| 系统/设备 | 溢出周期 |
|----------|---------|
| Windows 95 | 49.7 天 |
| Boeing 787 | 51 天 |
| Linux 内核调度器 | 208 天 |

这些案例表明，经典的整数溢出问题在各种系统中反复出现。

## 7. 社区反应

- 该漏洞已报告给 Apple，内部工单已创建
- 部分用户报告在 50 天以上 uptime 后遇到类似症状
- 社区认为该问题被过度戏剧化，实际影响用户有限
