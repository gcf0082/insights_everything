## 项目概述

cell 是一个纯 Rust 编写的终端电子表格编辑器，具有 Vim 按键绑定。主要特点：
- Vim 风格操作：Normal/Insert/Visual 模式，hjkl 导航，dd/yy/p，:w，:q
- 支持 CSV/TSV 导入导出，以及原生 .cell 格式（保留公式）
- 公式引擎支持：SUM、AVERAGE、COUNT、MIN、MAX、IF
- 架构：Cargo workspace，包含 cell-sheet-core（核心库）和 cell-sheet-tui（终端界面，基于 ratatui）
- 安装：`cargo install cell-sheet-tui`

## 热门评论与反馈

### 1. XLSX/ODS 支持需求
**用户**：freedomben（54 points）

建议支持 xlsx 和 open document 格式 spreadsheets，以便更实用。

**作者回应**：已添加到路线图（Issue #22），承认不是简单任务但看到了价值。

### 2. 专注编辑体验
**用户**：rpassos（36 points）

建议专注于"写"体验而非公式引擎，因为 xan 已是优秀的分析工具。建议与 xan 互补合作。

**作者回应**：认同这一愿景，Cell 定位为"Excel 但它是(Neo)Vim"。

### 3. 数值稳定性问题
**用户**：nanis（30 points）

建议对 SUM 和 AVERAGE 使用数值稳定算法而非简单累加除法。并报告 Windows Terminal 上按键重复的 bug。

**作者回应**：已创建 Issue #43（数值稳定算法）和 #44（Windows 按键重复）。

### 4. 架构挑战讨论
**用户**：SilentEditor

询问重新计算、依赖跟踪和撤销/重做的实现方案。

**作者详细回应**：
- 当前实现：每次编辑全量重算，使用拓扑排序
- 问题：dirty flag 未实际使用，公式每次重新解析
- 依赖跟踪：范围如 SUM(A1:A1000) 展开为1000个位置，大数据集有性能问题
- 撤销/重做：仅支持单单元格，多单元格编辑（d/p/P）尚未支持
- 存储：HashMap<CellPos, Cell>，适合稀疏数据但有开销

**其他用户补充**：建议范围作为第一类 AST 节点，使用区间树优化反向查找。

### 5. 命令行编程访问
**用户**：marcyb5st（13 points）

建议支持命令行操作：`cell file.cell --write A2 "42"` 或 `cell file.cell --read "=SUM(A1:A10)"`

**作者回应**：v0.1.7 已实现该功能！

### 6. 与现有工具比较

## 总结

这是一个有前景的终端电子表格编辑器，填补了 Vim 用户处�� CSV/表格数据的空白。社区反馈积极，主要关注点：

1. **格式支持**：XLSX/ODS 是最高优先级的功能需求
2. **性能优化**：大规模数据的重新计算和依赖跟踪需要改进
3. **功能完善**：拖动填充、命令行访问、可视化等功能正在路线图上
4. **长期维护**：作者表示会长期维护该项目
