# Hacker News 洞察报告

## 基本信息

- **洞察链接**: https://news.ycombinator.com/item?id=47575212
- **原始链接**: https://www.neowin.net/news/microsoft-copilot-is-now-injecting-ads-into-pull-requests-on-github-gitlab/
- **标题**: Over 1.5 million GitHub PRs have had ads injected into them by Copilot
- **得分**: 220 points
- **评论数**: 114 comments
- **作者**: bundie
- **发布时间**: 1小时前

## 事件概述

微软Copilot被发现在超过150万个GitHub Pull Request中注入了广告（产品提示）。这些"product tips"最初仅在Copilot生成的PR中显示，但后来扩展到任何提及Copilot的PR，引发了用户强烈不满。

## 后续进展

GitHub已经禁用了该功能。GitHub员工Martin Woodward在Twitter上表示："我们已经禁用了它。原本在Copilot生成的PR上显示产品提示还可以接受，但当添加了让Copilot处理任何PR的功能后，这种行为就变得令人不适了。"

## 热门评论观点

### 1. 对Copilot广告的看法
- 有用户指出，这类似于"发自iPhone"风格的签名，虽然不完全是广告，但默认设置下确实具有广告性质
- 有人认为这是对用户信任的严重背叛
- 部分用户将此与Ubuntu登录时显示广告的motd进行比较

### 2. AI代码署名讨论
- 许多开发者支持AI工具在提交中标注自己（如Claude添加Co-authored-by）
- 支持者认为这提供了有价值的信号，可以识别"vibe coded"（感觉编码）的PR
- 反对者认为代码质量才是关键，无论是否由AI生成
- 讨论还涉及法律责任问题：如果AI生成代码涉及版权诉讼，署名可能成为证据

### 3. 代码审查态度
- 有评论者表示会区别对待AI生成的代码和人类代码
- AI生成的代码可能看起来质量不错但实际有误，更难检测
- 提交代码的人应对代码质量负责，无论是否使用AI

## 总结

这一事件反映了开发者社区对AI工具日益增长的警惕心。虽然GitHub已禁用该功能，但关于AI代码的透明度、责任归属和审查方式的讨论仍在继续。
