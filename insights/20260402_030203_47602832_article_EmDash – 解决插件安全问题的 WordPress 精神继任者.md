# EmDash — 解决插件安全问题的 WordPress 精神继任者

## 基本信息

- **发布时间**：2026年4月1日
- **发布机构**：Cloudflare
- **版本**：v0.1.0 预览版
- **许可证**：MIT（开源）
- **GitHub**：https://github.com/emdash-cms/emdash

## 报告摘要

Cloudflare 发布了 EmDash，这是一个基于 Astro 6.0 构建的全栈无服务器 JavaScript CMS，旨在成为 WordPress 的精神继任者。EmDash 完全使用 TypeScript 编写，可部署在 Cloudflare Workers 或任何 Node.js 服务器上。其核心创新在于通过 Dynamic Workers 将插件运行在隔离的沙箱中，从根本上解决了 WordPress 插件的安全问题。

## 核心洞察

### 1. WordPress 插件安全危机

- **96%** 的 WordPress 安全问题源于插件
- 2025 年发现的高危漏洞数量超过前两年总和
- 原因：插件直接访问数据库和文件系统，无任何隔离

### 2. EmDash 的安全解决方案

- 每个插件运行在独立的 **Dynamic Worker** 隔离沙箱中
- 插件通过 **capabilities**（能力声明）获得权限
- 严格保证：插件只能执行其清单中明确声明的操作
- 安装前可完全了解插件的权限范围

### 3. 解决市场锁定问题

- 插件可以采用任何许可证（不强制 GPL）
- 插件代码在安全沙箱中独立运行
- 无需依赖中心化市场审核

### 4. 内置 x402 支付支持

- 每个 EmDash 网站原生支持 x402 标准
- 无需订阅即可按需收费
- 为 AI 时代的内容变现提供解决方案

### 5. 真正的无服务器架构

- 基于 Cloudflare Workers 的 v8  isolate 架构
- 自动扩展至零，按 CPU 时间计费
- 可在 Cloudflare for Platforms 上运行数百万实例

### 6. AI 原生 CMS

- 内置 Agent Skills 描述插件能力
- 提供 EmDash CLI 供编程管理
- 内置 MCP 服务器支持

### 7. 现代主题架构

- 基于 Astro 框架
- 主题无法执行数据库操作（与 WordPress 不同）
- 对前端开发者友好，LLM 更容易理解和生成

### 8. 认证与迁移

- 默认使用 Passkey（无密码）
- 支持 SSO 集成
- 支持从 WordPress 导入内容（WXR 格式）

## 技术细节

### 插件示例

```typescript
export default () => definePlugin({
  id: "notify-on-publish",
  version: "1.0.0",
  capabilities: ["read:content", "email:send"],
  hooks: {
    "content:afterSave": async (event, ctx) => {
      // 发送邮件通知
    },
  },
});
```

### 部署方式

```bash
npm create emdash@latest
```

或通过 Cloudflare Dashboard 部署。

## 结论

EmDash 代表了 CMS 领域的重要创新，它不仅解决了 WordPress 长期存在的插件安全问题，还为 AI 时代的内容创作和分发提供了新的可能性。通过无服务器架构、内置支付支持和 AI 原生特性，EmDash 有望成为下一代内容管理系统的标杆。
