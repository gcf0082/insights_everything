## 摘要

这是一个关于使用 Zig 语言编写 C 编译器的 Hacker News 讨论帖子。讨论涉及 Zig 语言与其内置 LLVM/Clang 的关系、以及用 Zig 编写纯 Zig C 编译器的最新进展。

## 关键信息

1. **Zig 与 LLVM/Clang 的关系**
   - Zig 目前捆绑了 LLVM 的 Clang，使用 `zig cc` 命令来编译 C 代码
   - 长期目标似乎是与 LLVM 解耦，不那么紧密地耦合
   - Zig 在编译器运行时（compiler-rt）方面做了些巧妙的工作，使其在交叉编译方面比原生 Clang 更有优势

2. **纯 Zig 编写的 C 编译器**
   - 还有一个用 Zig 编写的 C 编译器项目 **Aro**，看起来比 TFA 更加完整
   - Zig 从 0.16 版本开始将 Aro 作为其 TranslateC 功能的库（用于将 C 头文件转换为 Zig，而非整个程序）

3. **未来计划**
   - Clang 作为 `zig cc` 前端将继续保留（并且将成为可选项）
   - 讨论暗示未来可能会减少对 LLVM/Clang 的依赖

4. **相关背景**
   - 用 Zig 编写 C 编译器符合 Zig 早期价值主张中"用 Zig 维护"的理念
   - 这是一个令人感兴趣的项目，让人们关注 Zig 语言未来的发展方向
