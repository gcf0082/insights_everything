## 项目概述

**Ratty** 是一个实验性的终端模拟器，支持在终端中直接渲染内联3D图形。该项目直接受到 **TempleOS** 的启发，后者由已故开发者 Terry Davis 创建，以其独特的图形界面和内联3D功能而闻名。

### 核心特性

- **GPU 加速渲染**：使用现代图形技术实现终端3D渲染
- **Ratty Graphics Protocol**：项目提出的新图形协议
- **TempleOS 遗产**：演示中使用的3D对象直接来自 TempleOS 代码库
- **旋转鼠标准 cursor**：将终端光标替换为3D旋转老鼠

## 社区讨论要点

### 1. 技术渊源与灵感来源

社区用户普遍指出 Ratty 与 **TempleOS** 的联系。TempleOS 早在1981年就实现了类似的内联图形功能，被认为是"疯狂与天才的一线之隔"。用户提到：
- Xerox 工作站和 Lisp 机器的 REPL 体验
- Oberon 操作系统
- Plan 9 操作系统
- Smalltalk 环境

### 2. 潜在应用场景

社区讨论了多个可能的用例：

| 应用场景 | 描述 |
|----------|------|
| **3D模型预览** | 在终端中直接预览 STL/STEP 等3D打印文件 |
| **数据可视化** | 3D图形与数据科学笔记本结合 |
| **文件系统浏览** | 网格风格的文件列表，支持3D模型缩略图 |
| **游戏开发** | 终端内游戏开发环境 |
| **恶作剧同事** | 趣味性应用 |

### 3. 技术实现讨论

- **Kitty Graphics Protocol**：现有协议支持图片显示，但3D支持有限
- **Sixels**：老式 vt340 支持的图像协议
- **WebGL/WebGPU**：Ghostty-web 已开始探索
- **SSH 远程使用**：关于 GPU 加速终端在远程场景的讨论

### 4. 相关项目对比

| 项目 | 特点 |
|------|------|
| **Kitty** | 最积极创新的终端模拟器 |
| **euporie** | 支持数据科学笔记本的图形输出 |
| **Ghostty** | 完全实现 Kitty 图形协议 |
| **eza** | 增强版 ls，支持图标 |
| **chafa** | 终端图像渲染工具 |

### 5. 争议与担忧

- **标准化问题**：有人担忧图形扩展会破坏现有文本终端标准的一致性
- **实用性质疑**：部分用户认为"不需要旋转的3D鼠标准 cursor"
- **TempleOS 争议**：Terry Davis 的个人言论和行为引发讨论，但社区普遍认为应将软件与其创作者分开看待
