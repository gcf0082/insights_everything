## 内容摘要

这篇文章讨论了如何使用 AI 编程辅助工具（如 Claude Code、Cursor）来重启那些你曾经放弃的项目。作者分享了自己在 2020 年因难度过大而搁置的 React 迁移到 Svelte 项目，如今通过 Claude 成功完成的故事。

---

## 核心观点

### 1. AI 降低了启动门槛
- AI 能够帮助克服重新加载项目情境的"激活能量"
- 不需要花时间重新理解旧的代码结构
- 可以快速连接旧 API 到新 API

### 2. 适合个人工具类应用
- 对于个人使用的工具，代码质量并不重要
- 快速实现功能比完美更重要
- 适合那些"只是需要一个功能"的具体需求

### 3. 游戏开发可行性
- Claude + Godot 组合被证明对游戏开发有效
- AI 擅长生成程序化算法、自动机、四叉树等
- 不擅长直接生成视觉效果

## 技术讨论

### 本地部署方案
- opencode + Qwen 3.6 本地运行
- 需要正确配置本地工具链
- 在适当硬件上可以处理中等复杂的规划

### 云端方案
- Claude Code Pro: $20/月
- Mac Studio 或 AMD 395+ 前期硬件成本高

---

## 结论

社区对 AI 辅助编程的态度两极分化：
1. **支持方**: 认为这是让个人项目重见天日的好机会，特别适合那些"只是需要一个功能"的应用
2. **反对方**: 认为这会让人失去对项目的掌控感和学习机会，产生的是"无价值的垃圾"

关键在于使用场景：如果是实用工具类需求，AI 辅助是高效方案；如果是为了学习和成长，可能需要亲手编写代码。
