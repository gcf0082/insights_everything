## 概述

Qite.js 是一个面向讨厌 React 且热爱 HTML 的人的前端框架。它不依赖构建步骤、无需 Virtual DOM、不需要 npm，也不将 JavaScript 与 HTML 混合使用。

## 核心理念

### 1. DOM 作为数据源
- Qite.js 直接操作真实 DOM，不保留 UI 的影子副本
- 每个组件直接附加到真实的 DOM 元素上
- 无需 diffing 步骤或 reconciliation 过程

### 2. 无构建步骤
- 可直接在浏览器中运行
- 无需 JSX、无转译器、无打包器要求
- HTML 与 JavaScript 完全分离

### 3. SSR 优先，按需 SPA
- 天然适合服务端渲染的应用
- 可保持大多数页面完全 SSR
- 可选择性地让特定区域像小 SPA 一样运行

### 4. 声明式状态管理
- 定义依赖字段和标志的规则
- 字段（Fields）：结构化值，如 status、email、price
- 标志（Flags）：简单开关状态，如 open、saving、locked

### 5. 事件系统简洁明确
- 子组件发出事件，父组件负责监听和决策
- 统一的事件模型处理浏览器事件和自定义组件信号
- 无全局总线、无隐藏魔法

### 6. 模板系统
- 使用原生 HTML `<template>` 元素
- 动态创建的组件与初始组件行为完全一致

## 技术特点

| 特性 | 说明 |
|------|------|
| 无需构建步骤 | 直接在浏览器中运行 |
| 无 Virtual DOM | 直接操作真实 DOM |
| SSR 友好 | 天然支持服务端渲染 |
| 声明式状态 | 通过规则定义 UI 行为 |
| 单一事件模型 | 统一处理 DOM 和自定义事件 |
| 零依赖 | 可作为 git 子模块或直接复制到项目中 |

## 适用场景

- 想要 UI 感觉像结构化 HTML，带有清晰的声明式行为
- 不想使用编译抽象层
- 希望避免前端过度复杂化的开发者

## 不适合场景

- 需要 Virtual DOM 框架的项目
- 需要模板编译器的项目
- 需要强制 SPA 架构的项目

---

## 总结

Qite.js 是一个轻量级、自包含但功能强大的前端框架，它采用与主流框架相反的思路：把 DOM 作为真实的数据源，直接操作而非模拟。该框架特别适合追求简洁、避免前端过度工程化的开发团队。
