## 摘要

本文作者详细介绍了如何通过Traceroute验证家庭Tailscale出口节点的工作原理，包括路由变化、信任边界、DERP回退机制，以及为什么这种模式可以免费使用。作者在Proxmox LXC容器中搭建了一个小型出口节点（1 vCPU, 512 MB RAM），通过实际测试验证了流量走向。

## 核心要点

### 1. 什么是出口节点？

- **无出口节点时**：Tailscale仅用于设备发现，普通网络流量通过本地ISP发送
- **启用出口节点后**：设备将默认互联网路径指向选定的出口节点，所有流量通过该节点发送
- **流量特征**：出口节点发出的流量显示出口节点的公网IP，而非设备当前网络的IP

### 2. 技术架构

#### 控制平面 vs 数据平面

- **控制平面**：负责身份认证、peer发现、NAT穿透协调、ACL分发、路由分发、DNS管理
- **数据平面**：基于WireGuard的加密数据包传输
- Tailscale = WireGuard（数据平面）+ 控制平面基础设施

#### 连接流程

1. 设备认证到Tailscale控制平面
2. 控制平面共享各peer的可达端点和密钥
3. 通过UDP进行NAT打孔
4. 成功后建立直接的WireGuard加密路径
5. 失败时回退到DERP中继（仍为端到端加密）

### 3. 路由变化机制

#### Linux系统
- 使用内核策略路由（`ip rule` + 单独路由表）
- 将默认流量导向`tailscale0`接口
- 为出口节点公网IP添加"逃逸路线"，避免隧道循环

#### macOS/Windows
- 使用OS VPN框架
- 创建带有默认路由的VPN适配器

### 4. 为什么Tailscale可以免费？

- Tailscale协调服务主要是控制平面，非数据包传输平面
- 用户流量直连peer，仅在直接连接失败时才使用DERP中继
- 估算：5%回退率下，月成本约$10,000-$25,000；1%回退率下仅$2,000-$5,000
- 对比传统VPN：传统VPN需承担所有带宽费用（$0.02-$0.05/GB）

### 5. 与OpenVPN对比

| 方面 | OpenVPN | Tailscale出口节点 |
|------|---------|-------------------|
| 地址稳定性 | 需要DDNS | 自动发现 |
| 端口暴露 | 需要端口转发 | 无需暴露 |
| 证书管理 | 需要PKI | SSO设备身份 |
| NAT处理 | 困难 | 自动打孔+DERP |

### 6. 验证方法

```bash
# 1. 出口身份检查
curl ifconfig.me

# 2. 路径形状检查
traceroute github.com

# 3. 出口节点健康检查
tailscale status
tailscale ping <exit-node-name>

# 4. DNS行为检查
dig <内部域名>
dig github.com
```

### 7. 信任边界

| 参与方 | 通常可见 | 通常不可见 |
|--------|----------|------------|
| 咖啡店WiFi/本地ISP | 连接状态、流量时间/体积 | 加密流量内容 |
| 出口节点运营者 | 目标元数据（IP/域名模式）、流量时间/体积 | HTTPS页面内容 |
| 访问的网站 | 请求和出口节点公网IP | 设备当前网络公网IP |

### 8. DNS行为

- 出口节点改变流量出口位置，但DNS设置决定哪个解析器响应查询
- MagicDNS和分割DNS将匹配域名发送到Tailscale配置的DNS服务器
- 公共查询仍沿系统解析器路径

#### 分割DNS示例
- `.home.arpa`域名通过AdGuard DNS解析
- 即使在咖啡店网络，也能访问内部服务（如`nas.home.arpa`）

### 9. Proxmox LXC配置要点

需在宿主机添加：
```
lxc.cgroup2.devices.allow: c 10:200 rwm
lxc.mount.entry: /dev/net/tun dev/net/tun none bind,create=file
```

### 10. 出口节点 vs 子网路由

- **出口节点**：发送所有互联网流量（`0.0.0.0/0`, `::/0`）
- **子网路由**：仅发送特定私有网络（如`192.168.1.0/24`）

## 结论

作者认为出口节点使Tailscale像全隧道VPN一样工作，信任权衡很简单：不再信任当前所在网络，开始信任自己控制的出口节点。对于作者来说，这值得信任，因为运行在一个极简的专用LXC容器中。

** sanity check三命令**：
```bash
curl ifconfig.me  # 应显示出口节点公网IP
traceroute        # 第一跳应为出口节点路径

```
