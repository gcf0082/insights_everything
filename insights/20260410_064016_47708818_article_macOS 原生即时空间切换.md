## 洞察摘要

本文探讨了 macOS 虚拟桌面（Space）切换动画问题的解决方案。

## 问题背景

macOS 的 Space 切换动画令许多用户感到困扰，尽管动画时间不长，但频繁切换时累积的延迟仍然明显且令人不适。Apple 长期忽视用户禁用该动画的请求。

## 现有方案评估

作者对当前主流解决方案逐一进行了分析：

1. **启用"减少动态效果"设置**  
   这是网络上最常见的建议，但作者认为这完全无效——它只是用另一个无用的淡入动画替代切换动画，还会触发网页的 `prefers-reduced-motion` 媒体查询，造成副作用。

2. **使用 yabai 窗口管理器**  
   yabai 提供即时切换功能，但存在两个问题：
   - 需要通过二进制补丁修改系统组件
   - 必须禁用系统完整性保护（SIP）
   - 作者已使用 PaperWM.spoon，与 yabai 不兼容

3. **第三方虚拟空间管理器**  
   如 FlashSpace、AeroSpace 等通过隐藏/显示窗口来模拟即时切换。作者认为这些方案虽有效，但并非原生解决方案，对于"仅需禁用动画"的需求来说过于复杂。

4. **BetterTouchTool**  
   需要付费，功能"Move Right/Left Space (Without Animation)"可解决问题。

## 推荐解决方案

**InstantSpaceSwitcher** 是作者发现的最佳方案：

- **开发者**：jurplel（GitHub）
- **工作原理**：模拟带有大量速度的触控板滑动手势
- **额外功能**：支持直接跳转到指定编号的 Space；提供命令行界面

### 安装方法

```bash
git clone https://github.com/jurplel/InstantSpaceSwitcher
cd InstantSpaceSwitcher
./build.sh
```

### 命令行用法

```bash
.build/release/ISSCli --help
Usage: .build/release/ISSCli [left|right|index <n>]
```
