# 洞察报告：LiteLLM Python 包供应链攻击

## 基本信息

- **洞察链接**: https://news.ycombinator.com/item?id=47501729
- **标题**: LiteLLM Python package compromised by supply-chain attack
- **来源**: Hacker News
- **发布时间**: 2小时前
- **得分**: 276 points
- **评论数**: 143 comments

## 事件概述

LiteLLM 是一个流行的 Python 包，用于统一调用多种大语言模型 API。2026年3月24日，该包被发现存在供应链攻击漏洞，攻击者通过恶意代码窃取用户凭证。

## 攻击来源

1. 攻击源于 LiteLLM 项目的 CI/CD 流程中使用的 Trivy 安全扫描工具
2. Trivy 此前已被 TeamPCP 攻击组织入侵（相关事件：https://news.ycombinator.com/item?id=47475888）
3. 攻击者通过被污染的 Trivy 获取了项目的 GitHub 访问凭证

## 受影响版本

- **受影响版本**: 1.82.7、1.82.8
- **未受影响**: 使用 proxy docker 的用户（他们在 requirements.txt 中固定了版本）
- PyPI 已对该包进行隔离，阻止所有下载

## 恶意代码行为

根据分析，恶意代码执行以下操作：

1. 在后台生成 Python 进程
2. 解码嵌入的有效载荷
3. 运行内部收集器
4. 搜索系统中的凭证（特别是加密钱包相关）
5. 将收集到的数据写入攻击者的公钥
6. 使用 AES 加密被盗数据
7. 使用攻击者的 RSA 公钥加密 AES 密钥
8. 将加密后的数据 POST 到远程主机 `https://models.litellm.cloud/`

## 影响范围

- **直接用户**: 使用 LiteLLM 的开发者
- **下游项目**: DSPy、CrewAI、browser-use 等项目依赖 LiteLLM
- **潜在风险**: GitHub 令牌、HuggingFace 令牌、加密钱包密钥等可能被窃取

## 开发者回应

LiteLLM 维护者 Krrish 表示：
- 正在调查此问题
- 正在研究如何加强安全措施
- 对此事件表示歉意

## 安全建议

1. **立即行动**:
   - 检查是否使用了受影响版本 (1.82.7、1.82.8)
   - 轮换所有可能暴露的 API 密钥和凭证
   - 检查 .git-credentials 文件

2. **长期措施**:
   - 固定依赖版本，避免自动更新到最新版本
   - 使用静态分析工具检测恶意代码
   - 考虑使用更轻量的替代方案，减少依赖链
   - 在隔离环境中运行不受信任的代码

## 总结

这是一起典型的供应链攻击事件，攻击者通过入侵 CI/CD 工具链中的依赖包，最终成功向终端用户的项目中注入恶意代码。此事件再次提醒开发者：
- 依赖供应链的安全性至关重要
- 不应盲目信任最新版本的依赖包
- 需要建立多层防御机制来保护开发环境
