# 洞察报告：Claude 4.7 忽略 stop hooks

**基本信息**
- **链接**: https://news.ycombinator.com/item?id=47895029
- **标题**: Tell HN: Claude 4.7 is ignoring stop hooks
- **发布时间**: 2026-04-24
- **类型**: Tell HN

## 内容摘要

这是一篇关于 Claude Code 4.7 版本忽略 Anthropic hook 功能的用户投诉。

### 问题描述

用户从 Anthropic 推出 hook 功能起就开始使用它，这允许在工作流程中注入确定性行为。该功能在 4.7 版本之前运行完美，但现在 Claude 经常忽略 hook 规则。例如，用户设置了一个 stop hook，用于在源文件已修改但未运行测试时阻止 Claude 停止。

### 技术细节

- 退出码 2 表示阻塞错误
- Claude Code 忽略 stdout 和其中的任何 JSON
- stderr 文本作为错误消息反馈给 Claude

### 社区讨论

评论者指出：
- "你永远不允许 contradict 一个 stop hook，声称它错误触发，或以任何方式忽略它"
- stop hook 是正确的，如果认为它错，那就是用户错了
- 确定性不是由发送的消息添加的，而是在执行机制中
- 建议将 hook 设置为 informative 而非 blocking 的方式

用户提到在 4.6 版本中 hook 正常工作，但 4.7 有显著变化。有人在讨论是否应该切换回 Sonnet / Opus 4.6。
