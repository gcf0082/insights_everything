## 核心发现

研究人员发现了一个影响所有基于 Firefox 浏览器的隐私漏洞。该漏洞允许网站通过 `indexedDB.databases()` API 返回的数据库名称排序，推导出一个**唯一的、确定性的、稳定的进程级别标识符**，即使在用户期望更强隔离的上下文环境中也能生效。

### 技术原理

1. **问题 API**：`indexedDB.databases()` 原本应返回数据库元数据，但 Firefox 的实现问题导致返回顺序携带了可识别的熵信息。

2. **根本原因**：在隐私浏览模式下，数据库名称被映射为 UUID 并存储在全局哈希表中，哈希表的内部结构（非排序后的迭代）决定了返回顺序。

3. **关键代码位置**：`dom/indexedDB/ActorsParent.cpp` 中的 `StorageDatabaseNameHashtable` 全局映射。

4. **熵容量**：16 个可控数据库名可产生约 44 位的理论熵，足够唯一识别浏览器实例。

---

## 隐私影响

### 跨域影响
- 不相关网站可以独立推导出相同的标识符
- 无需 cookies 或共享存储即可跨域链接用户活动

### 隐私浏览模式影响
- 即使关闭所有隐私窗口，只要 Firefox 进程仍在运行，标识符仍可保留
- 后续访问可能被识别为来自同一"新鲜"的隐私会话

### Tor Browser 影响（特别严重）
- 该稳定标识符实际上破坏了 Tor Browser 的"新建身份"隔离功能
- 即使用户使用"New Identity"功能（应清除 cookies、历史记录并使用新 Tor 电路），标识符仍然保持

---

## 修复方案

Mozilla 已发布修复版本：
- **Firefox 150**
- **ESR 140.10.0**

**修复方式**：在返回结果前对数据库名进行规范排序（如字典序），消除内部存储布局带来的熵泄露。

---

## 启示

1. **隐私漏洞不一定来自直接访问**：有时来自内部实现细节的确定性泄露
2. **看似无害的 API 也能成为追踪向量**：只要泄露稳定的进程级别状态
3. **修复简单有效**：通过规范输出即可消除熵源并恢复预期的隐私边界
