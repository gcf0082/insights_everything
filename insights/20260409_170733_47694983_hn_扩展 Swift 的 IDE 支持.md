## 内容摘要

这是一个关于 Swift 官方 VSCode 扩展现在可以在 Open VSX Registry 上安装的公告。由于微软去年禁止 forks 访问官方 VSCode Marketplace，用户之前需要手动下载和安装 .vsix 文件。现在通过 Open VSX Registry，用户可以通过正常方式安装扩展了。

需要注意的是，这个扩展仅支持"纯"Swift 开发，而非 iOS 开发。iOS 开发可以非官方地通过 LSP 支持来实现，例如 Spotify 的 sourcekit-bazel-bsp 项目。

## 热门评论要点

### 1. 扩展的意义
- 这是一个受欢迎的改进，但标题有点误导性
- 在 Cursor 等编辑器中使用 Swift 一直是可行的，只是安装方式变得麻烦
- 现在通过 Open VSX Registry 解决了这个问题

### 2. iOS vs Android 开发
- **iOS**: 官方不支持在 Xcode 之外进行 iOS 开发，可以通过非官方方式实现但很麻烦
- **Android**: 更加开放，可以在 VS Code/Codium 中开发，使用 Gradle CLI
- Android SDK 是高质量的开发者工具，而 iOS SDK 是封闭的 GUI 依赖

### 3. Swift 语言本身
- Swift 是一门设计优秀的语言
- 过去的 C 互操作存在问题，但最近版本（6.2.3）有显著改进
- Swift 不仅用于 iOS 开发，还有其他用途（如服务器端、游戏开发等）

### 4. 开发者工具
- 许多开发者怀念 JetBrains 的 AppCode
- 存在一些非官方解决方案：
  - Sweetpad：VSCode 的 iOS 开发扩展
  - xcode-build-server：将 xcodebuild 日志解析为 LSP
  - xcodebuild.nvim：Neovim 的 Xcode 构建插件
  - Noctule：JetBrains 的 Swift IDE 插件

### 5. 关于 AI 编码
- 评论中提到 AI 极大地加速了开发工作
- 以前需要"多周重构"的任务现在几小时就能完成

---

## 总结

这次更新主要是让 Swift VSCode 扩展的安装更加便捷，对纯 Swift 开发有帮助，但对 iOS 开发支持有限。开发者社区对苹果生态系统的封闭性仍有抱怨，同时也期待更多跨平台工具支持。
