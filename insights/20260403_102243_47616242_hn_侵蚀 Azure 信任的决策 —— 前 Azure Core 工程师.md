## 内容摘要

这篇文章由一位前Azure Core工程师撰写，详细描述了微软Azure内部存在的系统性问题，这些问题导致了信任的流失。原文是一篇长文系列，据称涉及微软蒸发万亿美元市值的内容。

### 主要问题领域

#### 1. 架构问题
- 一个122人的团队需要173个代理来管理Azure节点
- 团队致力于将Windows移植到Linux以支持现有的VM管理代理
- 混合SKU使用经常受限，某些组件是"高级"而其他不是时无法兼容

#### 2. 工程质量问题
- Azure文档由AI编写，经常过时或错误
- GraphAPI文档与实际行为不符
- 平台极不稳定，相同参数调用会随机失败并返回HTTP 400错误
- 使用Terraform配置时问题尤为严重

#### 3. 可靠性问题
- 80-90%的云提供商问题来自Azure（来自SRE的经验分享）
- AKS（Azure Kubernetes Service） pods随机崩溃
- 数据库节点磁盘延迟随机飙升
- Azure承认问题可能需要数周甚至数月

#### 4. 安全问题
- Azure OpenAI曾在高负载下向其他客户泄露其他客户的prompt响应
- 容器逃逸后可访问管理控制器，发现存在多年未修补的关键漏洞

#### 5. 管理层问题
- 公司正式认可缺陷可以通过人工干预在生产系统上修复
- 快速发布、稍后修复的文化
- 短期指标优先于长期质量
