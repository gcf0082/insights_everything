# 洞察报告：This Month in Ladybird — April 2026

**链接：** https://ladybird.org/newsletter/2026-04-30/

**来源：** Ladybird Newsletter

**日期：** 2026年4月30日

---

## 概述

本月 Ladybird 浏览器项目取得了显著进展，共合并了 333 个 PR，来自 35 位贡献者，其中包括 7 位首次贡献者。项目继续在浏览器功能、JavaScript 引擎性能和 Web 标准兼容性方面取得突破。

## 主要更新

### 新增赞助商

- **人权基金会** (Human Rights Foundation)：通过"AI for Individual Rights"项目捐赠 $50,000
- **Jakub Stęplowski**：个人捐赠 $1,000

### 核心功能

1. **内联 PDF 查看器**：集成 pdf.js，支持页面导航、文本选择、缩放和文档内搜索功能

2. **浏览历史和智能地址栏自动补全**：基于 SQLite 的 HistoryStore 持久化访问记录，支持带图标和标题的富建议

3. **增量 HTML 解析**：实现流式解码和投机性解析，提前预取资源

4. **后台 JavaScript 编译**：将脚本编译移至后台线程，YouTube 加载时约节省 200ms 主线程时间

5. **每Navigable独立光栅化**：iframe 现在独立渲染，为进程隔离做准备

### JavaScript 引擎优化

- 更快 JS-to-JS 调用：使用 ARM64 配对加载/存储指令
- O(1) 字节码寄存器分配器：加载 x.com 时节省约 800ms
- 缓存 for-in 迭代：Speedometer 2 从 67.7 提升至 73.6
- 零拷贝标识符名称共享：解析提速 1.14 倍，内存减少 282MB
- 短字符串拼接优化：提速 2.13 倍
- 稀疏数组优化：创建 2000 万元素数组不再有性能惩罚

### 新前端：GTK4/libadwaita

Linux 平台新增基于 GTK4 和 libadwaita 的新前端，支持地址栏自动补全、安全图标、页面内搜索、全屏、主题切换等功能。

### 其他改进

- **书签管理**：新增 about:bookmarks 页面，支持导入/导出
- **Cache 和 CacheStorage**：完整实现所有 9 个方法
- **CSS 功能**：image-set()、CSS anchor positioning、颜色插值重写、RTL 列表标记等
- **DNS 优化**：getaddrinfo 不再阻塞事件循环，x.com 加载性能大幅提升
- **mimalloc 分配器**：C++ 和 Rust 代码共享单一分配器实例
- **Linux GPU 渲染**：通过 dmabuf 实现 GPU 零拷贝渲染

### 网站兼容性提升

- **Reddit**：图片轮播和无限滚动修复
- **YouTube**：多项优化，显著提升加载速度
- **gocomics.com**：头部图片显示正常
- **yandex.com/maps**：WebGL 矢量瓦片渲染修复

### WPT 测试

WPT 通过数从 2,003,537 提升至 2,067,263，新增 63,726 个子测试。

## 总结

Ladybird 浏览器在 2026 年 4 月继续快速发展，在渲染性能、JavaScript 引擎和 Web 标准兼容性方面都取得了重要进展。新增的 GTK4 前端和多项性能优化使浏览器更加完善。
